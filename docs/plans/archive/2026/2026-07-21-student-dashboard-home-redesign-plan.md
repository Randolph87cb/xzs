# 学生端首页任务优先改版方案

状态：completed
创建日期：2026-07-21
完成日期：2026-07-21
验证摘要：学生端类型检查和生产构建通过；学生端构建产物已同步到后端静态资源目录；`scripts/verify-web-static-consistency.ps1 -SkipHttpCheck -SkipAdmin` 通过。未启动本地后端服务，因此未做 HTTP 页面一致性检查。

## 背景与现状

- 学生端首页位于 `frontend/apps/student/src/views/dashboard/DashboardView.vue`。
- 当前首页已经同时加载老师任务、首页试卷和班级排行：
  - `getDashboardTasks()`：返回按学生年级或班级过滤后的老师任务，并按学生目标科目过滤任务内试卷。
  - `getDashboardIndex()`：返回按学生 `targetSubjectId` 过滤后的固定试卷和当前时段试卷。
  - `getClassRanking()`：返回当前班级排行。
- 当前首屏仍以学习概览、统计卡、折叠任务列表为主。学生能看到入口，但不够直接回答“现在最应该做哪套题”。
- 右侧排行可以保留，但应作为辅助信息，不应和主学习任务抢层级。

## 结论

推荐把首页从“数据概览页”改为“下一步学习页”：左侧优先展示老师布置任务，其次展示当前学生级别或科目对应考试；右侧保留班级排行和轻量学习数据。第一阶段以学生端前端重排为主，不新增后端字段；若后续要严格按 GESP 级别推荐，需要再确认试卷模型是否已有明确级别字段。

## 需求拆解

### 1. 老师布置任务优先

- 当前现状：任务中心使用折叠面板展示，任务内容需要展开后才看清试卷与操作。
- 判断：老师任务是学生首页最重要的行动入口，应在首屏左上直接展示。
- 修改方案：
  - 将“老师布置任务”作为主卡片放在首页首屏左侧第一块。
  - 默认展示前 3 到 4 个任务，每个任务直接露出试卷名、完成状态和主操作按钮。
  - 试卷状态保持现有逻辑：未答题显示“开始答题”，已完成显示“查看试卷”，待批改显示标签。
  - 任务为空时显示“暂无老师任务”，并给出进入本级别考试或智能训练的行动入口。
- 影响范围：`DashboardView.vue` 模板、计算属性和样式。
- 验证方案：
  - 有任务时，首屏第一块能直接看到任务与按钮。
  - 无任务时，空状态不占据过大空间，并能继续进入考试或智能训练。
  - 点击任务试卷仍正确携带 `id` 和 `taskId`。

### 2. 本级别考试入口

- 当前现状：固定试卷、时段试卷分散展示，统计卡占据靠前位置；`pushPaper` 已在类型中存在，但首页没有实际展示。
- 判断：学生不应先理解试卷类型，而应先看到与自己当前科目或级别相关的可做试卷。
- 修改方案：
  - 合并 `fixedPaper`、`timeLimitPaper`、`pushPaper` 为 `recommendedPapers`。
  - 展示为“本级别考试”，用轻量标签区分“固定试卷 / 时段试卷 / 推送试卷”。
  - 优先排序建议：时段试卷、推送试卷、固定试卷；首屏展示有限数量，更多跳转到试卷中心。
  - 如果没有推荐试卷，显示清晰空状态，并保留“试卷中心”入口。
- 影响范围：`DashboardView.vue` 计算属性、模板、样式；必要时同步 `@xzs/api-client` 类型。
- 验证方案：
  - 固定试卷、时段试卷、推送试卷都能正常展示和跳转。
  - 时段试卷展示时间范围。
  - 空数据场景页面不出现空白断层。

### 3. 右侧班级排行保留但降权

- 当前现状：班级排行位于右侧，但右侧还存在重复的“学习任务”侧栏，和主任务区信息重复。
- 判断：排行可以提升参与感，但不应成为学生首页的主要任务入口。
- 修改方案：
  - 保留右侧“班级排行”，展示前 5 名和“查看全部”。
  - 移除右侧重复的“学习任务”侧栏。
  - 在排行下方增加轻量“学习数据”，承载任务数、推荐考试数等摘要，替代首屏四个统计大卡。
- 影响范围：`DashboardView.vue` 模板和样式。
- 验证方案：
  - 有排行数据时显示前 5 名。
  - 无排行数据或接口失败时仍能显示空状态，不影响左侧任务和考试。
  - 移动端排行排在主任务和本级别考试之后。

### 4. 响应式与视觉层级

- 当前现状：首页多个卡片并列，任务卡、排行卡、试卷卡的层级差异不明显。
- 判断：学生端应更强调行动按钮和扫描效率。
- 修改方案：
  - 桌面端采用左主右辅：左侧任务和考试，右侧排行和学习数据。
  - 移动端单列：老师任务、本级别考试、班级排行、学习数据。
  - 减少嵌套卡片和大面积统计卡，使用行分隔、标签和按钮建立层级。
- 影响范围：`DashboardView.vue` 样式。
- 验证方案：
  - 桌面宽度下主任务区和排行区不重叠。
  - 移动宽度下文本不溢出，按钮可点击区域清晰。
