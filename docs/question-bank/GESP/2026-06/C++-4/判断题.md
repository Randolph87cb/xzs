## 第1题

运行以下程序后，变量 `a` 的值最终会变为 `20`。

```cpp
void modify(int *p) {
    *p = *p + 10;
}
int main() {
    int a = 10;
    modify(&a);
    return 0;
}
```

答案：A

解析：本题判断为正确。函数 `modify` 的参数 `p` 是指针，调用 `modify(&a)` 时把变量 `a` 的地址传入函数。函数内部的 `*p` 就是对 `a` 本身的访问，执行 `*p = *p + 10` 后，`a` 从 `10` 变为 `20`。虽然程序没有输出语句，但题干问的是变量 `a` 的最终值，因此说法正确。

## 第2题

在 C++ 中，引用一旦初始化并绑定到某个变量后，可以通过赋值语句将其重新绑定到另一个变量。

答案：B

解析：本题判断为错误。C++ 引用在定义时必须初始化，初始化后就一直作为原变量的别名，不能再改为绑定另一个变量。对引用执行赋值语句，改变的是它所引用的那个变量的值，而不是让引用重新绑定。例如 `int &r = a; r = b;` 的含义是把 `b` 的值赋给 `a`，并不会让 `r` 改为引用 `b`。

## 第3题

下面程序可以正确计算并输出 `3` 名学生的平均成绩。

```cpp
struct Student {
    int id;
    int score;
};

int main() {
    Student students[3] = {
        {1, 90},
        {2, 80},
        {3, 100}
    };

    int sum = 0;
    for (int i = 0; i < 3; i++) {
        sum += students[i].score;
    }
    double average = sum / 3.0;
    cout << average << endl;
    return 0;
}
```

答案：A

解析：本题判断为正确。结构体数组 `students` 中保存了 3 名学生的信息，循环 `i = 0, 1, 2` 正好把三名学生的 `score` 累加到 `sum` 中，得到 `90 + 80 + 100 = 270`。随后用 `sum / 3.0` 计算平均值，因为除数是浮点数 `3.0`，结果会按浮点数计算，得到 `90`，所以程序可以正确计算并输出平均成绩。

## 第4题

选择排序算法在寻找每一轮最小值时，如果遇到相等的元素不进行交换，则选择排序是一种稳定的排序算法。

答案：B

解析：本题判断为错误。排序是否稳定，要求相等关键字的元素在排序前后的相对顺序一定不变。普通选择排序每一轮会把找到的最小元素与当前位置元素交换，即使寻找最小值时遇到相等元素不更新最小位置，最后这次跨位置交换仍可能把某个元素换到相等元素的后面，从而破坏相对顺序。因此普通选择排序通常不是稳定排序。

## 第5题

如果使用带 `flag` 的冒泡排序，且待排序数组一开始就是有序的，那么算法只需一轮扫描即可结束，时间复杂度为 $O(n)$。

答案：A

解析：本题判断为正确。带 `flag` 的冒泡排序会在每一轮扫描中记录是否发生过交换。如果数组一开始已经有序，第一轮从头到尾比较相邻元素时不会发生任何交换，`flag` 会保持为“未交换”的状态，算法即可判断数组已经有序并提前结束。这一轮只扫描了相邻元素，比较次数与 $n$ 成正比，因此时间复杂度为 $O(n)$。

## 第6题

在 C++ 中定义二维数组并初始化时，可以省略第一维，但不能省略第二维。因此 `int a[][2] = {{1, 2}, {3, 4}};` 是合法的，而 `int a[][] = {{1, 2}, {3, 4}};` 是不合法的。

答案：A

解析：本题判断为正确。二维数组本质上是“若干个固定长度的一维数组”组成的数组。初始化时，编译器可以根据初始化列表推断第一维有多少行，但必须知道每一行有多少列，才能计算元素地址并确定每行的边界。所以 `int a[][2] = {{1, 2}, {3, 4}};` 可以由初始化内容推出有 2 行、每行 2 列，而 `int a[][] = {{1, 2}, {3, 4}};` 缺少第二维长度，编译器无法确定每行大小，因此不合法。

## 第7题

下面代码的时间复杂度是 $O(2^n)$。

```cpp
int cnt = 0;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++) {
        cnt++;
    }
}
```

答案：B

解析：本题判断为错误。外层循环执行 $n$ 次；当外层变量为 `i` 时，内层循环执行 `i` 次。因此 `cnt++` 的总执行次数是 $1 + 2 + \cdots + n = \frac{n(n+1)}{2}$，数量级是 $O(n^2)$。这段代码没有递归分支或每层成倍增长的过程，不能看成 $O(2^n)$。

## 第8题

假设文件 `output.txt` 能正常打开，下面代码通过 `rdbuf` 将 `cout` 的输出重定向到了文件中。

```cpp
ofstream fout("output.txt");
streambuf* old_buf = cout.rdbuf();
cout.rdbuf(fout.rdbuf());
cout << "GESP Exam";
cout.rdbuf(old_buf);
```

答案：A

解析：本题判断为正确。`cout.rdbuf()` 可以取得或设置 `cout` 当前使用的缓冲区。代码先用 `old_buf` 保存 `cout` 原来的缓冲区，再执行 `cout.rdbuf(fout.rdbuf())`，让 `cout` 的输出写入文件流 `fout` 对应的缓冲区。随后 `cout << "GESP Exam";` 会写入 `output.txt`。最后 `cout.rdbuf(old_buf);` 把输出目标恢复为原来的标准输出。

## 第9题

小杨想通过下面程序给饭卡充值，程序会输出 `70`。

```cpp
void recharge(int money) {
    money += 20;
}
int main() {
    int card = 50;
    recharge(card);
    cout << card;
    return 0;
}
```

答案：B

解析：本题判断为错误。函数 `recharge` 的参数 `money` 是普通 `int` 形参，调用 `recharge(card)` 时会把 `card` 的值复制一份传给 `money`。函数中执行 `money += 20` 只会修改这份副本，不会修改 `main` 中的变量 `card`。函数返回后，`card` 仍然是 `50`，所以程序输出 `50`，不是 `70`。

## 第10题

下面代码可以通过编译。

```cpp
int a[5];
a++;
```

答案：B

解析：本题判断为错误。数组名 `a` 在表达式中常会转换为指向首元素的指针，但数组名本身不是可以被修改的变量，不能像普通指针变量那样执行自增操作。`a++` 试图改变数组名所表示的地址，这是不允许的，因此这段代码不能通过编译。正确说法是：可以定义指针 `int *p = a;`，然后对指针变量 `p++`，但不能对数组名 `a++`。
