## 第1题

小杨正在编写一个“数字交换器”程序，他希望通过函数交换两个变量的值。请问运行以下代码后，屏幕上输出的是（ ）。

```cpp
void exchange(int *a, int &b) {
    int t = *a;
    *a = b;
    b = t;
}
int main() {
    int x = 100, y = 200;
    exchange(&x, y);
    cout << x << " " << y;
    return 0;
}
```

A. `100 200`
B. `200 100`
C. `200 200`
D. 编译错误

答案：B

解析：本题选 B。调用 `exchange(&x, y)` 时，形参 `a` 指向 `x`，引用形参 `b` 直接绑定到 `y`。函数中先用 `t` 保存 `x` 的原值 `100`，再执行 `*a = b`，把 `x` 改为 `200`，最后 `b = t` 把 `y` 改为 `100`。因此输出 `200 100`。A 是没有交换时的结果；C 只把 `x` 改成了 `y`，忽略了最后一步；D 错在代码中指针和引用的使用都是合法的。

## 第2题

下面程序想通过函数计算三门课总分，横线处应填入的是（ ）。

```cpp
int sumScore(int a, int b, int c) {
    return a + b + c;
}
int main() {
    int chinese = 88, math = 95, english = 90;
    int total = __________;
    cout << total;
    return 0;
}
```

A. `sumScore`
B. `sumScore(chinese, math, english)`
C. `sumScore(int chinese, int math, int english)`
D. `sumScore(a, b, c)`

答案：B

解析：本题选 B。`sumScore` 是一个需要 3 个 `int` 参数的函数，调用时应写函数名和实参列表，所以应填 `sumScore(chinese, math, english)`，计算结果为 `88 + 95 + 90`。A 只有函数名，没有真正调用函数。C 把函数定义时的类型声明写到了调用位置，语法不对。D 中 `a`、`b`、`c` 是函数形参名，在 `main` 函数中并没有定义，不能直接使用。

## 第3题

下面程序输出结果是（ ）。

```cpp
int addOne(int x) {
    return x + 1;
}
int main() {
    int a = 6;
    cout << addOne(a) + addOne(3);
    return 0;
}
```

A. `9`
B. `10`
C. `11`
D. `12`

答案：C

解析：本题选 C。`addOne(a)` 中 `a` 的值是 `6`，返回 `7`；`addOne(3)` 返回 `4`。输出表达式计算的是 `7 + 4 = 11`。A、B 都少算了某次函数调用加 `1` 的效果；D 则把结果多加了 `1`。

## 第4题

关于下面程序，说法正确的是（ ）。

```cpp
void show() {
    int stars = 5;
}
int main() {
    cout << stars;
    return 0;
}
```

A. 程序输出 `5`
B. 程序可以通过编译，但输出随机值
C. 程序不能通过编译，因为 `stars` 只在 `show` 函数中有效
D. 程序不能通过编译，因为 `cout` 不能输出变量

答案：C

解析：本题选 C。`stars` 是定义在 `show` 函数内部的局部变量，它的作用域只在 `show` 函数体内。`main` 函数中没有名为 `stars` 的变量，因此编译器找不到这个标识符，程序不能通过编译。A、B 都把 `stars` 当成了 `main` 中可访问的变量。D 错在 `cout` 本来就可以输出变量，问题不在 `cout`，而在变量作用域。

## 第5题

小杨在调试一个“等级提升”系统，代码逻辑如下，执行后 `*p` 的值是（ ）。

```cpp
int lv = 5, next_lv = 6;
int *p = &lv;
*p = *p + 1;
p = &next_lv;
```

A. `5`
B. `6`
C. `lv` 的地址
D. `next_lv` 的地址

答案：B

解析：本题选 B。开始时 `p` 指向 `lv`，所以 `*p = *p + 1` 会把 `lv` 从 `5` 改为 `6`。随后 `p = &next_lv` 让指针改为指向 `next_lv`，此时再看 `*p`，取得的是 `next_lv` 的值，也就是 `6`。A 是 `lv` 修改前的值。C、D 是地址，而题目问的是 `*p` 的值，即指针指向位置中存放的数据。

## 第6题

小杨正在开发一款名为“星际网格”的游戏，他用二维数组 `int map[5][4];` 来表示地图。已知 `int` 占 `4` 字节，如果 `map` 的内存地址是 `0x2000`，则表达式 `&map + 1` 的地址值是（ ）。

A. `0x204c`
B. `0x205c`
C. `0x2050`
D. `0x2058`

答案：C

解析：本题选 C。`map` 是一个 `5 × 4` 的二维数组，共有 `20` 个 `int`，每个 `int` 占 `4` 字节，所以整个数组占 `20 × 4 = 80` 字节。`&map` 的类型是指向整个二维数组的指针，`&map + 1` 会跳过整个数组，而不是只跳过一个元素。`80` 的十六进制是 `0x50`，所以地址从 `0x2000` 变为 `0x2050`。A、B、D 都不是按整个数组大小移动得到的结果。

## 第7题

执行完下面代码后，变量 `val` 的值是（ ）。

```cpp
int data[] = {10, 20, 30, 40, 50};
int *ptr = data + 2;
int val = *(ptr - 1) + *(ptr + 1);
```

A. `50`
B. `60`
C. `70`
D. `80`

答案：B

解析：本题选 B。数组 `data` 的下标 `0` 到 `4` 分别是 `10, 20, 30, 40, 50`。`ptr = data + 2` 后，`ptr` 指向 `data[2]`，也就是 `30`。`ptr - 1` 指向 `data[1]`，值为 `20`；`ptr + 1` 指向 `data[3]`，值为 `40`。因此 `val = 20 + 40 = 60`。A、C、D 都没有正确对应指针偏移后访问到的两个元素。

## 第8题

某班 `3` 个小组、每组 `4` 名同学的分数存入下面的二维数组 `score`，则 `score[1][2]` 的值是（ ）。

```cpp
int score[3][4] = {
    {80, 81, 82, 83},
    {90, 91, 92, 93},
    {70, 71, 72, 73}
};
```

A. `81`
B. `90`
C. `92`
D. `72`

答案：C

解析：本题选 C。C++ 数组下标从 `0` 开始。`score[1]` 表示第 2 行，也就是 `{90, 91, 92, 93}`；在这一行中，`score[1][2]` 表示下标为 `2` 的第 3 个数，值为 `92`。A 是第 1 行第 2 个数，B 是第 2 行第 1 个数，D 是第 3 行第 3 个数。

## 第9题

小杨定义了一个结构体 `Hero` 来表示游戏角色，下面哪种初始化方式会由于语法错误导致编译失败？（ ）。

```cpp
struct Hero {
    string name;
    int hp;
};
```

A. `Hero h = {"Arthur", 100};`

B.

```cpp
Hero h;
h.name = "Arthur";
h.hp = 100;
```

C. `Hero h = new Hero{"Arthur", 100};`

D. `Hero *p = new Hero{"Arthur", 100};`

答案：C

解析：本题选 C。`new Hero{"Arthur", 100}` 会在动态内存中创建一个 `Hero` 对象，并返回它的地址，类型是 `Hero*`。选项 C 却用这个指针去初始化普通对象 `Hero h`，类型不匹配，会编译失败。A 是聚合初始化，可以直接给结构体成员赋值。B 是先定义对象，再逐个成员赋值，语法正确。D 使用 `Hero*` 指针接收 `new` 返回的地址，类型匹配。

## 第10题

下面程序输出结果是（ ）。

```cpp
struct Book {
    string title;
    int pages;
};
int main() {
    Book books[2] = {{"Math", 120}, {"Science", 150}};
    cout << books[1].title;
    return 0;
}
```

A. `Math`
B. `Science`
C. `120`
D. `150`

答案：B

解析：本题选 B。`books` 是长度为 `2` 的结构体数组，下标从 `0` 开始。`books[0]` 是 `{"Math", 120}`，`books[1]` 是 `{"Science", 150}`。程序输出的是 `books[1].title`，也就是第二本书的标题 `Science`。A 是第一个元素的标题，C、D 是 `pages` 成员，不是 `title` 成员。

## 第11题

小杨在对“能量晶石”按亮度进行排序。如果两块晶石亮度相同，他希望保持它们在原始序列中的相对顺序。下列关于排序算法稳定性的说法，错误的是（ ）。

A. 冒泡排序是稳定的，因为只有在左边比右边大时才交换。
B. 插入排序是稳定的，因为它将元素插入到相等元素的右侧。
C. 选择排序是稳定的，因为它每次选出最小元素放在前面。
D. 稳定性是指排序后相等元素的相对位置不发生改变。

答案：C

解析：本题选 C。稳定排序要求相等关键字的元素在排序前后的相对顺序不变。普通选择排序每一轮会把当前最小元素和前面的某个位置交换，这个交换可能越过与它相等的元素，从而破坏相对顺序，所以不能说选择排序是稳定的。A 中的冒泡排序若只在左边大于右边时交换，相等元素不会交换，通常是稳定的。B 中插入排序把当前元素插到相等元素后面，也能保持稳定。D 正是稳定性的定义。

## 第12题

小杨的机器人正在能量踏板上跳跃，踏板编号为 `1, 2, 3, ...`。跳到第 `n` 块踏板的方案数满足递推式 $f(n)=f(n-1)+f(n-2)$。若 $f(1)=1, f(2)=2$，则运行以下代码计算 `jump(5)` 的结果是（ ）。

```cpp
int jump(int n) {
    if (n <= 2)
        return n;
    int a = 1, b = 2, c = 0;
    for (int i = 3; i <= n; i++) {
        c = a + b;
        a = b;
        b = c;
    }
    return c;
}
```

A. `5`
B. `8`
C. `13`
D. `21`

答案：B

解析：本题选 B。根据递推关系，`f(1)=1`、`f(2)=2`，所以 `f(3)=1+2=3`，`f(4)=2+3=5`，`f(5)=3+5=8`。代码中的 `a` 和 `b` 分别维护相邻的前两项，循环到 `i=5` 后 `c` 就是 `f(5)`。A 是 `f(4)`，C 是 `f(6)`，D 是 `f(7)`，都不是 `jump(5)` 的结果。

## 第13题

在“模拟实验室”程序中，为了防止除以 `0` 导致崩溃，小杨使用了异常处理机制。执行以下代码将输出（ ）。

```cpp
try {
    int x = 10, y = 0;
    if (y == 0) throw "Zero Error";
    cout << x / y;
} catch (int e) {
    cout << "Error Code: " << e;
} catch (const char* msg) {
    cout << "Caught: " << msg;
}
```

A. `0`
B. `Error Code: 0`
C. `Caught: Zero Error`
D. 程序直接崩溃

答案：C

解析：本题选 C。`y == 0` 成立后，程序执行 `throw "Zero Error";`，抛出的是字符串字面量，对应类型可由 `const char*` 捕获。第一个 `catch (int e)` 类型不匹配，会被跳过；第二个 `catch (const char* msg)` 匹配成功，输出 `Caught: Zero Error`。因为在除法之前已经抛出并捕获异常，所以不会输出 A，也不会直接崩溃。B 是把异常当成整数 `0` 捕获时才可能想到的错误结果。

## 第14题

下面代码使用某种排序算法，将数组中的元素按从小到大排序。这段代码使用的排序算法是（ ）。

```cpp
void mystery_sort(double arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minPos = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minPos]) {
                minPos = j;
            }
        }
        double temp = arr[i];
        arr[i] = arr[minPos];
        arr[minPos] = temp;
    }
}
```

A. 冒泡排序
B. 插入排序
C. 选择排序
D. 非典型排序

答案：C

解析：本题选 C。代码每一轮固定一个位置 `i`，在未排序区间 `i` 到 `n-1` 中寻找最小元素的位置 `minPos`，然后把最小元素交换到第 `i` 个位置。这正是选择排序的典型过程。冒泡排序通常是相邻元素两两比较并交换；插入排序通常把当前元素插入到前面已经有序的区间；因此 A、B 不符合代码结构，D 也不对，因为这段代码是标准选择排序。

## 第15题

小杨正在读取“冒险日志”文件 `quest.txt`。若文件内容为 `Level 10`，执行以下程序后输出为（ ）。

```cpp
ifstream fin("quest.txt");
string s;
int v;
fin >> s >> v;
cout << s.length() * v;
```

A. `50`
B. `15`
C. `70`
D. `5`

答案：A

解析：本题选 A。文件内容 `Level 10` 中，`fin >> s >> v` 会把字符串 `Level` 读入 `s`，把整数 `10` 读入 `v`。`Level` 有 5 个字符，所以 `s.length()` 的值是 `5`，最终输出 `5 × 10 = 50`。B 把长度和数值相加了，D 只取了字符串长度，C 与题目中的字符串长度和整数都不匹配。
