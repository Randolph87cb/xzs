## 第1题

关于计算机的数据编码，下面说法正确的是（ ）。

A. 机器数 `1000 1101B` 的真值可能是 `-13`（如果它是原码）或 `141`（如果它是无符号数）等数，这取决于我们如何解释它。
B. 计算机中，所有数据最终都以二进制、八进制、十六进制的形式存储和运算。
C. 字节（Byte）是计算机中最小的数据单位。位（bit）是计算机中最小的存储单位。
D. 计算机中，`1k` 字节是 `1000` 字节的意思。

答案：A

解析：本题选 A。同一串二进制位要看采用哪种编码解释：`1000 1101B` 如果按 8 位原码看，最高位 `1` 表示负号，后 7 位 `0001101` 是 13，所以真值为 `-13`；如果按无符号数看，它的值是 $128 + 8 + 4 + 1 = 141$。B 错在八进制、十六进制主要是给人阅读的书写形式，计算机内部仍以二进制位存储和运算。C 把 byte 和 bit 的概念说反了，bit 是最小的信息单位。D 中计算机语境下通常 `1KB = 1024B`，不是 1000B。

## 第2题

计算机厂商为了计算方便，一般采用 `1000` 进制。如果我们买的厂商标注的是 `1 TB` 的硬盘，它实际的存储容量是（ ）。

A. $1000 \times 1000 \times 1000 \times 1000 \div 1024 \div 1024 \div 1024b = 931Gb$
B. $1000 \times 1000 \times 1000 \times 1000 \div 1024 \div 1024 \div 1024B = 931GB$
C. $1024 \times 1024 \times 1024 \times 1024 \div 1000 \div 1000 \div 1024B = 1049GB$
D. $1000 \times 1024 \times 1024 \times 1024 \div 1024 \div 1024 \div 1024b = 977Gb$

答案：B

解析：本题选 B。厂商标注的 `1 TB` 按十进制计算是 $1000^4$ 字节，也就是 $1,000,000,000,000$ 字节。操作系统常按 $1024^3$ 字节作为 1GB 来换算，所以容量约为 $1000^4 \div 1024^3 \approx 931$ GB。A 和 D 使用了小写 `b`，表示 bit（位），单位不对；C 把起始容量按 $1024^4$ 计算，和题目“厂商采用 1000 进制”不符。

## 第3题

低 4 位、高 4 位压缩技术，适用于数据仅使用字节的一部分（如仅用低 4 位）的场景。字节结构：一个字节为 8 位，分为高 4 位（高位）和低 4 位（低位）。当数据是十六进制数（0 ~ 15，即 `0x0` 到 `0xF`），每个值仅需 4 位表示，高 4 位全为 `0`。将两个相邻的 4 位值合并为一个字节。四个数据 `0x1`、`0x2`、`0x3`、`0x4` 采用上述压缩技术压缩以后是（ ）。

A. `12D`、`34D`
B. `12Q`、`34Q`
C. `12H`、`34H`
D. `00010011B`、`00110101B`

答案：C

解析：本题选 C。每个数据只占 4 位，`0x1` 和 `0x2` 合并成一个字节时，高 4 位为 `1`、低 4 位为 `2`，得到十六进制 `12H`；`0x3` 和 `0x4` 合并后得到 `34H`。A 的 `D` 表示十进制，B 的 `Q` 不是这里要表达的十六进制，D 给出的二进制分别对应 `13H` 和 `35H`，不是把相邻值 `1,2` 与 `3,4` 正确拼接后的结果。

## 第4题

关于计算机编码中反码和补码，下面说法错误的是（ ）。

A. 负数的补码，一个快速方法是从右往左扫描正数的二进制形式，遇到第一个 `1` 之后，左边的所有位都取反。
B. 对于一个 $n$ 位的二进制数：最大表示范围：$[-(2^{n-1}) - 1, +(2^{n-1} - 1)]$。
C. 反码减法可以统一为加法。符号位可以直接参与运算。
D. 反码表示中，0 的表示不唯一：`0000 0000B` 和 `1111 1111B`。

答案：B

解析：本题选 B。$n$ 位补码的表示范围是 $[-2^{n-1}, 2^{n-1}-1]$；$n$ 位反码的负数范围到 $-(2^{n-1}-1)$，也就是 $[-(2^{n-1}-1), 2^{n-1}-1]$，并不是选项 B 写的 $-(2^{n-1}) - 1$。A 是求负数补码的常用快速方法。C 说明反码可以把减法转化为加法。D 正确指出反码有 `+0` 和 `-0` 两种 0 的表示。

## 第5题

一种加密方式是字符数组与密钥 `KEY`、运算方式分开传输，比如字符数组 `char text[4] = {'G', 'E', 'S', 'P'};` 由一种传输方式发送，密钥 `KEY = 2026` 通过另一种发送方式发送，运算方式 `char function[4] = {'|', '-', '^', '+'};` 又是另一种发送方式发送。三种数据都到达目的地以后，分别进行例如 `'G' | 6`、`'E' - 2`、`'S' ^ 0`、`'P' + 2` 等计算，来得到相应的真实内容，上述 GESP 通过这种加密方式，加密以后最终的内容是（ ）。

A. `GCSR`
B. `RSCG`
C. `GCSA`
D. `BCSR`

答案：A

解析：本题选 A。按题目给出的对应运算逐个计算：`'G' | 6` 中 `'G'` 的低位已经包含需要置 1 的位，结果仍是 `'G'`；`'E' - 2` 得到 `'C'`；`'S' ^ 0` 与 0 异或不改变字符，仍是 `'S'`；`'P' + 2` 得到 `'R'`。四个字符合起来是 `GCSR`。B 把字符顺序倒置了，C 把最后一个字符算成了 `A`，D 第一个字符也不符合按位或的结果。

## 第6题

关于位运算，下列说法错误的是（ ）。

A. 找唯一数：数组中唯一出现一次的数，其余出现两次，全部异或结果即为该数。例如：数组 `[5, 7, 9, 7, 5]`（唯一数是 `9`）。
B. 交换两个数：`a ^= b; b ^= a; a ^= b;`（无需临时变量）。
C. 将二进制位整体左移 $n$ 位，高位溢出舍弃，低位补 `0`；等价于 `num` 乘以 $2^n$。
D. 对每一个二进制位取反，包括符号位，简单运算规则是 `~n = -n - 1`。

答案：C

解析：本题选 C。左移 $n$ 位在没有溢出、且类型语义允许的情况下，常可理解为乘以 $2^n$；但选项 C 同时说“高位溢出舍弃”，一旦发生溢出，结果已经不再等价于普通乘法结果。A 利用的是 `x ^ x = 0`、`x ^ 0 = x`。B 是经典异或交换写法。D 是补码表示下按位取反的常见规律。

## 第7题

关于字符串和字符数组，下列说法正确的是（ ）。

A.

```cpp
char str[] = "GESP";
int len1 = sizeof(str);
int len2 = strlen(str);
```

上面程序能够正确执行，`len1` 与 `len2` 相等。

B.

```cpp
char str1[4] = "GESP";
char str2[4] = {'G', 'E', 'S', 'P'};
```

这段程序将能够正确执行。

C.

```cpp
char str2[4] = {'G', 'E', 'S', 'P'};
strcpy(str2, "HELLO,GESP");
cout << str2 << endl;
```

这段程序即使能够运行，但是存在覆盖数组以外的内存空间的行为，可能会引起严重错误。

D.

```cpp
char dest[4] = {'G', 'E', 'S', 'P'};
char src[] = "HELLO";
strcat(dest, src);
cout << dest << endl;
```

这段程序能够正确执行，不存在数组越界行为。

答案：C

解析：本题选 C。`str2` 只有 4 个字符空间，而 `"HELLO,GESP"` 还包含字符串结束符 `'\0'`，长度远超 4，`strcpy` 会向数组之外写入数据，即使某些环境下看似运行，也属于危险的越界行为。A 中 `sizeof(str)` 统计整个字符数组大小，`"GESP"` 会带末尾 `'\0'`，所以 `len1` 为 5，而 `strlen(str)` 为 4。B 中 `char str1[4] = "GESP";` 没有空间存放结尾的 `'\0'`，不符合 C 风格字符串初始化要求。D 中 `dest` 既没有结尾 `'\0'`，也没有足够空间拼接 `"HELLO"`，同样存在越界风险。

## 第8题

计算机中的 `2 KB` 等于多少 bit（ ）。

A. `16384`
B. `20000`
C. `2000`
D. `2048`

答案：A

解析：本题选 A。计算机中通常 $1KB = 1024B$，所以 $2KB = 2048B$。又因为 $1B = 8bit$，所以 $2048 \times 8 = 16384bit$。B 把 `K` 当成 1000 且再乘 10，C 没有正确换算字节和位，D 只是字节数，不是 bit 数。

## 第9题

在 C++ 中，对于 32 位有符号整数 `int` 类型数据 `n`，关于按位取反运算符 `~`，下列说法正确的是（ ）。

A. `~6` 的结果是 `5`。
B. 按位取反满足公式 `~n = -n - 1`。
C. `~0` 的结果是 `1`。
D. `~(-2)` 的结果是 `-1`。

答案：B

解析：本题选 B。在补码表示下，对所有二进制位取反后，数值规律可以写成 `~n = -n - 1`。例如 `~6 = -7`，不是 5；`~0 = -1`，不是 1；`~(-2) = 1`，不是 -1。理解这个公式后，不需要逐位画出 32 位二进制也能快速判断。

## 第10题

关于计算机中的二进制编码表示，下列说法错误的是（ ）。

A. 原码是最直观的一种有符号数表示方法。最高位（最左边的位）为符号位：`0` 表示正数，`1` 表示负数，其余位为数值位（真值的绝对值）。
B. 补码完美解决了原码和反码的缺陷，是现代计算机中表示有符号整数的标准方式。正数的补码与其原码、反码相同；负数的补码是将其对应正数的原码按位取反（得到反码），然后加 `1`。
C. 计算补码的一个更快的技巧：从右往左扫描正数的二进制形式，遇到第一个 `1` 之后，左边的所有位都取反。
D. 对于一个 $n$ 位的二进制数，补码最大表示范围为 $[-2^{n-1}, +2^{n-1}]$。

答案：D

解析：本题选 D。$n$ 位补码一共有 $2^n$ 种状态，其中范围是 $[-2^{n-1}, 2^{n-1}-1]$，正数端少 1，因为还要表示 0。选项 D 把最大正数写成了 $+2^{n-1}$，超出了 $n$ 位补码能表示的范围。A 是原码定义，B 是补码的基本性质和负数补码求法，C 是从右向左求负数补码的快捷方法，都正确。

## 第11题

下面选项中提到的变量都是正整数，关于位运算，下面说法错误的是（ ）。

A. `num & 1`，结果为 `1` 则奇数，`0` 则偶数（仅看最低位）。
B. `num & 0xFF` 保留低 8 位。
C. `num & b` 的结果一定小于等于 `num`。
D. 若 `num` 左移导致高位溢出（如超过整型范围），结果符合乘法规律。

答案：D

解析：本题选 D。左移在没有溢出时可以和乘以 2 的幂联系起来；但一旦超过整型范围或高位被丢弃，就不能保证结果仍符合普通乘法规律。A 正确，因为整数奇偶只看最低二进制位。B 中 `0xFF` 的低 8 位全为 1，和它按位与会保留低 8 位。C 对正整数成立，因为按位与只会把某些 1 变成 0，不会把 `num` 中原来为 0 的位变成 1。

## 第12题

`a=7, b=3, c=14, d=15, e=8`，对于运算表达式 `!a << b & c ^ d | e` 的结果是（ ）。

A. `0`
B. `15`
C. `7`
D. `14`

答案：B

解析：本题选 B。运算优先级从高到低依次可按 `!`、`<<`、`&`、`^`、`|` 来看，所以表达式等价于 `(((!a) << b) & c) ^ d | e`，也就是最后一步再与 `e` 做按位或。因为 `a = 7` 为真，`!a` 得到 `0`；`0 << 3` 仍为 `0`；`0 & 14` 为 `0`；`0 ^ 15` 为 `15`；`15 | 8` 仍为 `15`。因此结果是 15。A 忽略了后面的异或和或运算，C、D 都不是按 C++ 位运算优先级逐步计算得到的结果。

## 第13题

关于 `string` 的成员函数，下面说法错误的是（ ）。

A. `size()`：返回字符串长度（字符个数，不含 `'\0'`）。
B. `length()` 与 `size()` 功能完全一致，返回字符串长度。
C. `empty()`：判断字符串是否为空（非空返回 `true`，空返回 `false`）。
D. `s.append(s2, 0, 3);` 从 `s2` 下标 `0` 开始，截取 `3` 个字符。

答案：C

解析：本题选 C。`empty()` 的含义是“是否为空”，空字符串返回 `true`，非空字符串返回 `false`，选项 C 把返回值说反了。A 正确，`string` 的 `size()` 返回字符个数，不把 C 风格字符串的结尾 `'\0'` 当作长度。B 正确，`length()` 和 `size()` 在 `string` 中作用相同。D 正确，这个 `append` 调用会从 `s2` 的下标 0 开始取 3 个字符追加到 `s` 后面。

## 第14题

以下数组定义，符合 C++ 语法的是（ ）。

A. `int [10] a;`
B. `int b['&'];`
C. `int c[*];`
D. `double d[10.0];`

答案：B

解析：本题选 B。C++ 数组定义应写成“元素类型 变量名[长度]”，而字符常量 `'&'` 是整型常量表达式，可以作为数组长度，因此 `int b['&'];` 语法上成立。A 把数组长度放在变量名前，顺序错误，正确形式应类似 `int a[10];`。C 中 `*` 不能作为数组长度。D 中 `10.0` 是浮点数，不是合法的数组长度表达式。

## 第15题

A. `cout << (max >= '1') << " " << (max >= '7') << " " << (max >= '9') << " " << (max >= 'F') << endl;`
