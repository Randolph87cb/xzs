# 判断题

来源：
- 官网原卷 PDF：https://gesp.ccf.org.cn/101/attach/1753295297183776.pdf

说明：本文件依据官网 PDF 渲染截图视觉核对整理。判断题中 `A` 表示正确，`B` 表示错误。

## 第1题

反码运算时，产生的进位需要循环进位，即最高位产生的进位要加回到结果的最低位。

答案：A

解析：本题判断为正确。反码加法和普通二进制加法的一个重要区别是“循环进位”：如果最高位相加后产生了进位，这个进位不能直接丢掉，而要再加回到结果的最低位。题干描述的正是反码运算中的循环进位规则。

## 第2题

`-11` 的补码的一种计算方式是从右往左扫描正数的二进制形式，遇到第一个 `1` 之后，左边的所有位都取反。

答案：A

解析：本题判断为正确。求一个负数的补码时，可以先写出对应正数在固定位数下的二进制表示，再从右往左找到第一个 `1`，这个 `1` 以及它右边的位保持不变，左边的位全部取反。以 8 位为例，`11` 是 `00001011`，按这个方法得到 `11110101`，这就是 `-11` 的 8 位补码。

## 第3题

一个 8 位的二进制数补码，最大的表示范围是从 $-127$ 到 $+127$。

答案：B

解析：本题判断为错误。8 位补码一共有 $2^8=256$ 种状态，表示范围是 $-128$ 到 $+127$，不是 $-127$ 到 $+127$。原因是补码中只有一个 0，最高位为 1 的状态比最高位为 0 的正数状态多出一个最小负数 `10000000`，它表示 $-128$。

## 第4题

判断某个数是否是质数，枚举范围可适当缩小（遍历到 $\sqrt{i}$ 而非 $i$），提升效率。

答案：A

解析：本题判断为正确。如果一个数 $i$ 有大于 $1$ 且小于 $i$ 的因子，那么这些因子一定成对出现，例如 $a \times b = i$。若两个因子都大于 $\sqrt{i}$，乘积就会大于 $i$，不可能成立。因此只需要检查到 $\sqrt{i}$ 是否存在因子，就能判断是否为质数，枚举范围变小后效率更高。

## 第5题

如果 `a` 为 `int` 类型的变量，且表达式 `((a & 1) == 1)` 的值为 `true`，则说明 `a` 是奇数。

答案：A

解析：本题判断为正确。整数的二进制最低位可以判断奇偶性：最低位为 `1` 表示奇数，最低位为 `0` 表示偶数。`a & 1` 会保留 `a` 的最低位，其他位都变成 0；当结果等于 1 时，说明最低位为 1，所以 `a` 是奇数。

## 第6题

十六进制数 `CCF` 对应的二进制数、八进制数、十进制数分别是：`110011001111`、`6317`、`3269`。

答案：B

解析：本题判断为错误。十六进制 `C` 等于二进制 `1100`，`F` 等于二进制 `1111`，所以 `CCF` 的二进制表示确实是 `110011001111`。它转成十进制应为 $12 \times 16^2 + 12 \times 16 + 15 = 3279$，不是 `3269`；`3279` 转成八进制是 `6317`。题干中的十进制数写错了，因此整体为错误。

## 第7题

下列程序如果能够正确执行，那么输出的结果是 `GESP`。

```cpp
int main() {
    string name = "GESP";
    cout << name[false] << name[true] << name[1 << 1] << name[7 >> 1] << endl;
}
```

答案：A

解析：本题判断为正确。题干已经说明“如果能够正确执行”，因此重点看下标计算。`false` 转成整数是 `0`，`true` 转成整数是 `1`，`1 << 1` 的结果是 `2`，`7 >> 1` 的结果是 `3`。字符串 `"GESP"` 的下标 `0, 1, 2, 3` 分别是 `G, E, S, P`，所以输出结果为 `GESP`。

## 第8题

某个初学 C++ 的学生，在对照参考程序写了一个程序之后，信心满满地进行编译，他敲进编译器的全部代码如下，这个程序能够正常编译运行。

```cpp
1 #include<bits/stdc++.h>
2 using namespace std;
3 int main() {
4     cout << "Hello world!" << endl;
5     return 0;
6 }
```

答案：B

解析：本题判断为错误。题干说这些是他敲进编译器的全部代码，因此每一行开头的 `1`、`2`、`3` 等数字也是代码内容，而不是编辑器自动显示的行号。C++ 程序不能在 `#include`、`using namespace std;`、`int main()` 等语句前随意加这些数字，否则会造成语法错误，程序不能正常编译运行。

## 第9题

代码逐行复制于手工运算步骤，中间不能增加任何判断逻辑，否则就不属于模拟算法。

答案：B

解析：本题判断为错误。模拟算法的核心是用程序按照题目规则或实际过程一步步执行，并不要求机械地逐行复制手工运算步骤。很多模拟题都需要根据当前状态加入 `if` 判断、循环控制或分支处理，只要这些逻辑是在正确模拟规则，就仍然属于模拟算法。

## 第10题

判断一个 `int` 型数字 `s[i]` 是不是在 `0` 和 `9` 之间（可以是 `0` 和 `9`），判断条件可以写为 `if (s[i] >= '0' && s[i] <= '9')`。

答案：B

解析：本题判断为错误。题干说 `s[i]` 是 `int` 型数字，要判断它是否在数值 `0` 到 `9` 之间，应写成 `s[i] >= 0 && s[i] <= 9`。`'0'` 和 `'9'` 是字符常量，在常见 ASCII 编码中分别对应整数 48 和 57，用它们比较会判断 `s[i]` 是否在 48 到 57 之间，而不是是否在 0 到 9 之间。
