# 判断题

来源：

说明：本文件依据官网原卷 PDF 整理；判断题答案按 A=正确、B=错误 标注，导入脚本会自动生成选项。

## 第1题

使用 cmath 或 math.h 中的三角函数时，角度参数默认采用角度制。

答案：B

解析：本题判断为错误。C++ 的 `<cmath>` 或 `math.h` 中，`sin`、`cos`、`tan` 等三角函数的参数默认按弧度制解释，而不是角度制。例如要计算 90 度的正弦值，通常应先把角度转换为弧度，即使用 $\pi / 2$ 附近的值。

## 第2题

使用 cmath 或 math.h 中的 pow(2, 10) 计算 $2^{10}$ 时，由于参数均为整型 int，返回值类型也为整型 int。

答案：B

解析：本题判断为错误。`pow` 是数学库中的幂函数，常见调用 `pow(2, 10)` 会把整型实参转换到浮点运算对应的重载中，返回值通常是浮点类型，如 `double`，而不是因为两个参数写成整数就返回 `int`。表达式的数值可以表示 $2^{10}=1024$，但题干关于返回值类型的说法不成立。

## 第3题

0/1 背包使用一维数组优化时，容量从小到大枚举也能保证每件物品最多被选一次。

答案：B

解析：本题判断为错误。0/1 背包要求每件物品最多选一次，一维优化时处理第 $i$ 件物品通常要让容量从大到小枚举，这样 `dp[c - w[i]]` 仍然来自上一轮物品的结果。如果从小到大枚举，当前物品刚更新出的状态可能马上又被更大的容量使用，相当于同一件物品在同一轮被重复选择。

## 第4题

哈希表采用开放定址法时，即使哈希函数设计合理，也仍然可能发生冲突。

答案：A

解析：本题判断为正确。哈希函数设计合理只能尽量让元素分布更均匀，降低冲突概率，不能保证不同关键字一定映射到不同位置。开放定址法本身就是为了解决冲突而设计的：发生冲突后，再按照线性探测、二次探测等规则寻找其他空位置。

## 第5题

同一个图从同一个起点进行深度优先搜索，访问序列一定与邻接点的枚举顺序无关。

答案：B

解析：本题判断为错误。深度优先搜索会沿着当前枚举到的邻接点继续向深处递归或入栈，因此邻接点的枚举顺序会直接影响先访问哪条分支。同一个图、同一个起点，如果邻接表中相邻顶点的排列顺序不同，DFS 得到的访问序列可能不同。

## 第6题

泛洪算法可以用递归 DFS 实现，但地图很大时可能由于递归层数过深导致调用栈溢出等运行时错误。

答案：A

解析：本题判断为正确。泛洪算法常用于从一个格子向四周或八个方向扩展连通区域，可以用递归 DFS 写得很直观。但如果可达区域很大，递归调用层数可能接近格子数量，超过运行环境允许的调用栈深度时，就可能出现栈溢出等运行时错误，此时常改用显式栈或队列实现。

## 第7题

哈夫曼树中不存在度为 1 的结点。

答案：A

解析：本题判断为正确。标准哈夫曼树通过不断取出两个权值最小的结点或子树并合并成一个新父结点构造，因此每个新产生的内部结点都有两个孩子，叶子结点没有孩子。按树中“度”表示孩子个数来理解，结点的度只会是 0 或 2，不会出现度为 1 的结点。

## 第8题

冒泡排序的常见实现是稳定排序，选择排序也是。

答案：B

解析：本题判断为错误。冒泡排序在常见实现中只交换相邻的逆序元素，相等元素不会互相越过，因此是稳定排序。简单选择排序通常每一轮选出最小元素后与当前位置交换，这个交换可能让某个相等元素跨过另一个相等元素，所以选择排序一般不是稳定排序。

## 第9题

在无权图中从起点执行 BFS 时，某个顶点第一次被访问到的层数等于起点到该顶点经过的最少边数。

答案：A

解析：本题判断为正确。在无权图中，每条边的代价都可以看作 1。BFS 从起点开始按层扩展，先访问距离为 0 的起点，再访问距离为 1 的顶点，之后依次访问距离更远的顶点。因此一个顶点第一次被访问到时，对应的层数就是从起点到它的最少边数。

## 第10题

在二维动态规划中，状态 dp[i][j] 的计算常常依赖其他状态，这些状态的计算必须在完成 dp[i][j] 的计算前完成。

答案：A

解析：本题判断为正确。动态规划的核心是用已经求出的状态推出新的状态。如果 `dp[i][j]` 依赖 `dp[i - 1][j]`、`dp[i][j - 1]` 或其他状态，那么这些被依赖的状态必须先有正确结果，否则 `dp[i][j]` 的计算就会使用未定义或尚未更新完成的数据。二维 DP 常常需要按依赖关系选择合适的遍历顺序。
