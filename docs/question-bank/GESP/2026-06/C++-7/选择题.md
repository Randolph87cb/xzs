## 第1题

下列 C++ 代码的输出结果是（ ）。

```cpp
#include <iostream>
#include <cmath>
using namespace std;
int main() {
    cout << (int)(sqrt(50) + log2(8));
    return 0;
}
```

A. 9
B. 10
C. 11
D. 12

答案：B

解析：本题选 B。`sqrt(50)` 约为 $7.07$，`log2(8)` 等于 $3$，两者相加约为 $10.07$。外层的 `(int)` 是强制转换为整数，会直接舍去小数部分，得到 `10`。A 把结果算小了，C 和 D 则没有正确处理强制类型转换后的截断效果。

## 第2题

下列关于 `<cmath>` 或 `<math.h>` 中的数学库函数的说法，正确的是（ ）。

A. `sqrt(49)` 的返回值可以参与浮点运算。
B. `log2(32)` 的返回值类型为 `int`。
C. `pow(2, 5)` 的返回值类型一定为 `int`。
D. `sin(90)` 的参数 `90` 表示 90 度。

答案：A

解析：本题选 A。`sqrt`、`log2`、`pow`、`sin` 等数学函数通常返回浮点类型的结果，因此 `sqrt(49)` 的返回值当然可以继续参与浮点运算。B 错在 `log2(32)` 的返回值不是 `int`；C 错在 `pow` 的返回值通常是浮点类型，不能因为数学结果是整数就说返回类型一定为 `int`；D 错在 C++ 三角函数的角度参数默认使用弧度制，`sin(90)` 表示 90 弧度而不是 90 度。

## 第3题

下列关于 C++ 函数参数传递的说法，正确的是（ ）。

A. 函数形参一定和实参使用同一块内存。
B. 值传递时，在函数内修改形参一定会修改实参。
C. 引用形参绑定到实参后，在函数内修改引用形参通常会影响实参。
D. 指针形参不能用于修改实参指向的数据。

答案：C

解析：本题选 C。引用形参相当于给实参起了一个别名，函数内通过引用形参修改值，通常会直接影响对应实参。A 和 B 都忽略了值传递：值传递会复制一份形参，形参和实参不是同一块内存，修改形参不会改变实参。D 也不正确，指针形参虽然本身可以是拷贝，但它保存的地址可以指向实参相关的数据，函数内通过解引用指针可以修改该地址处的数据。

## 第4题

有 5 个字符，它们出现的次数分别为 3、4、7、8、9。使用哈夫曼编码时，最小的带权路径长度 WPL 为（ ）。

A. 62
B. 64
C. 67
D. 69

答案：D

解析：本题选 D。哈夫曼树每次合并当前最小的两个权值，并把合并代价累加到 WPL 中。权值为 3、4、7、8、9 时，先合并 $3+4=7$，再合并 $7+7=14$，再合并 $8+9=17$，最后合并 $14+17=31$。总 WPL 为 $7+14+17+31=69$。A、B、C 都少算了某次合并代价，或没有始终选择当前最小的两个权值合并。

## 第5题

已知网格上每个网格点有一个数字，`a[i][j]` 表示第 `i` 行第 `j` 列处网格点上的数字。若 `dp[i][j]` 表示从网格左上角（第 0 行第 0 列）走到第 `i` 行第 `j` 列时能取得的最大数字和，且每次只能向右或向下移动。对于 `i > 0` 且 `j > 0` 的位置，正确的状态转移代码为（ ）。

A. `dp[i][j] = a[i][j] + min(dp[i - 1][j], dp[i][j - 1])`
B. `dp[i][j] = max(dp[i - 1][j - 1], dp[i][j])`
C. `dp[i][j] = a[i][j] + max(dp[i - 1][j], dp[i][j - 1])`
D. `dp[i][j] = a[i][j] + dp[i - 1][j - 1]`

答案：C

解析：本题选 C。只能向右或向下走到当前位置，因此到达 `(i, j)` 的上一步只可能来自上方 `(i - 1, j)` 或左方 `(i, j - 1)`。要求最大数字和，就应取这两个状态的较大值，再加上当前位置的 `a[i][j]`。A 用了 `min`，求的是较小路径和；B 没有加当前位置的值，也错误使用了对角线状态；D 只考虑从左上对角线来，但题目不允许斜着走。

## 第6题

已知 `f[0] = 0`，`f[1] = 2`，并且对 `i >= 2` 有 `f[i] = max(f[i - 1], f[i - 2] + a[i])`。若 `a[1 .. 5] = {2, 7, 9, 3, 1}`，则 `f[5]` 的值为（ ）。

A. 10
B. 11
C. 12
D. 22

答案：C

解析：本题选 C。按递推式逐项计算：`f[2] = max(f[1], f[0] + a[2]) = max(2, 7) = 7`；`f[3] = max(7, 2 + 9) = 11`；`f[4] = max(11, 7 + 3) = 11`；`f[5] = max(11, 11 + 1) = 12`。B 是算到 `f[4]` 后停止的结果，D 相当于把不应同时选的值直接相加，没有遵守递推式。

## 第7题

下面代码是一维数组优化 0/1 背包的核心片段，其中 `w[i]` 表示第 `i` 件物品的重量，`v[i]` 表示第 `i` 件物品的价值。横线处应填入（ ）。

```cpp
for (int i = 1; i <= n; i++) {
    for (int c = W; c >= w[i]; c--) {
        __________;
    }
}
```

A. `dp[c] = max(dp[c], dp[c + w[i]] + v[i])`
B. `dp[c] = min(dp[c], dp[c - w[i]] + v[i])`
C. `dp[c] = dp[c - w[i]] + v[i]`
D. `dp[c] = max(dp[c], dp[c - w[i]] + v[i])`

答案：D

解析：本题选 D。0/1 背包中，容量为 `c` 时有两种选择：不选第 `i` 件物品，价值保持 `dp[c]`；选第 `i` 件物品，则从容量 `c - w[i]` 的旧状态转移来，得到 `dp[c - w[i]] + v[i]`。两者取最大值，所以是 D。A 使用了 `c + w[i]`，方向错误；B 用 `min` 不符合最大价值目标；C 会强制选择当前物品，可能把原本更优的 `dp[c]` 覆盖掉。

## 第8题

下面程序片段主要体现的算法思想是（ ）。

```cpp
void dfs(int x, int y) {
    vis[x][y] = true;
    for (int k = 0; k < 4; k++) {
        int nx = x + dx[k], ny = y + dy[k];
        if (inside(nx, ny) && a[nx][ny] == 1 && !vis[nx][ny])
            dfs(nx, ny);
    }
}
```

A. 泛洪算法
B. 二分查找
C. 贪心算法
D. 归并排序

答案：A

解析：本题选 A。代码从一个格子出发，标记已访问，并递归扩展到上下左右相邻且满足条件的格子，这是在网格中扩展同一连通区域的典型泛洪算法，也常用 DFS 实现。B 的二分查找需要在有序范围中不断缩小区间；C 的贪心算法强调每一步做局部最优选择；D 的归并排序会分治拆分并合并序列，都与这段网格递归扩展代码不符。

## 第9题

下列关于排序稳定性的说法，正确的是（ ）。

A. 冒泡排序在只交换相邻逆序元素时是稳定排序
B. 选择排序一定是稳定排序
C. 快速排序一定是稳定排序
D. 稳定排序一定会改变相等元素的相对顺序

答案：A

解析：本题选 A。稳定排序要求相等关键字的元素排序后相对顺序不变。常见冒泡排序只在相邻元素逆序时交换，遇到相等元素不交换，因此相等元素不会越过彼此，是稳定的。B 错在选择排序把后面的最小元素直接换到前面，可能让相等元素相对顺序改变；C 错在常见快速排序的交换过程也可能破坏稳定性；D 正好说反了，稳定排序是不改变相等元素的相对顺序。

## 第10题

无向图的边为 `(1, 2), (1, 3), (2, 4), (3, 4), (4, 5)`。从顶点 1 开始进行 BFS，每轮根据出队顶点，将与其相邻顶点按编号从小到大入队，则顶点 4 第一次入队时，队列的状态为（ ）。

A. 1, 2, 3, 4
B. 2, 3, 4
C. 3, 4
D. 3, 4, 5

答案：C

解析：本题选 C。BFS 从 1 开始，先将 1 出队，再按编号把 2、3 入队，此时队列为 `2, 3`。接着 2 出队，检查 2 的邻点 1 和 4，其中 1 已访问，4 第一次入队。此时 2 已经出队，3 还在队列中，4 刚加入队尾，所以队列为 `3, 4`。A 和 B 都保留了已经出队的顶点，D 中的 5 要等顶点 4 出队后才可能入队。

## 第11题

一个长度为 11、下标为 0 到 10 的哈希表采用线性探测法处理冲突，哈希函数为 `h(x) = x % 11`。依次插入 22、33、4、15、26，则 26 最终存放在下标（ ）。

A. 0
B. 4
C. 5
D. 6

答案：D

解析：本题选 D。按顺序插入：`22 % 11 = 0`，放在下标 0；`33 % 11 = 0`，下标 0 已占用，线性探测到下标 1；`4 % 11 = 4`，放在下标 4；`15 % 11 = 4`，下标 4 已占用，探测到下标 5。最后 `26 % 11 = 4`，下标 4、5 都已占用，继续探测到下标 6，因此 26 放在 6。A 是 22 的位置，B 是 4 的位置，C 是 15 的位置。

## 第12题

关于哈希表处理冲突的方法，下列说法正确的是（ ）。

A. 线性探测法发生冲突后，只能放弃插入该元素。
B. 链地址法可以把哈希到同一位置的多个元素组织在同一个桶中。
C. 只要哈希表长度是素数，就一定不会发生冲突。
D. 开放定址法查找元素时不需要考虑冲突位置。

答案：B

解析：本题选 B。链地址法会在每个哈希桶中维护一个链表或类似结构，多个映射到同一位置的元素可以放在同一个桶里。A 错在线性探测法发生冲突后会继续向后寻找空位置，而不是放弃插入。C 错在表长取素数只能帮助改善分布，不能保证不同元素不会映射到同一位置。D 错在开放定址法查找时必须沿着探测序列检查冲突后可能存放的位置。

## 第13题

某算法需要枚举 `n` 个对象；对每个对象，还需要进行一次二分查找。若二分查找的对象规模也是 `n`，则该算法的时间复杂度通常为（ ）。

A. $O(n)$
B. $O(n \log n)$
C. $O(n^2)$
D. $O(\log n)$

答案：B

解析：本题选 B。枚举 `n` 个对象需要执行 `n` 次循环；每个对象内部做一次规模为 `n` 的二分查找，单次二分查找复杂度是 $O(\log n)$。两层工作相乘，总复杂度为 $O(n \log n)$。A 少算了每次二分查找的代价，D 只算了一次二分查找，C 则把二分查找误当成了线性查找。

## 第14题

在升序数组中用二分查找第一个大于等于 `x` 的位置。若当前中点 `mid` 满足 `a[mid] < x`，下一步应（ ）。

A. 令闭区间右边界变为 `mid - 1`
B. 令闭区间左边界变为 `mid + 1`
C. 立即返回 `mid`
D. 交换 `a[mid]` 与 `x`

答案：B

解析：本题选 B。数组升序排列，如果 `a[mid] < x`，那么 `mid` 以及它左侧的位置都不可能是“第一个大于等于 `x` 的位置”，答案只能在 `mid` 右侧，所以闭区间左边界应更新为 `mid + 1`。A 会错误地丢掉真正可能存在答案的右半部分；C 错在 `mid` 当前还小于 `x`；D 与二分查找无关，查找过程不会交换数组元素。

## 第15题

在如下网格中，`#` 表示不能经过的格子，`.` 表示可以经过的格子。从左上角走到右下角，每次只能向右或向下移动，不同路径共有（ ）条。

```text
. . . . .
. # . # .
. . . . .
# . # . .
. . . . .
```

A. 5
B. 6
C. 7
D. 8

答案：D

解析：本题选 D。可以用动态规划计数：起点记为 1，障碍格记为 0，每个可走格子的路径数等于上方路径数加左方路径数。逐行计算得到第 1 行为 `1, 1, 1, 1, 1`；第 2 行为 `1, 0, 1, 0, 1`；第 3 行为 `1, 1, 2, 2, 3`；第 4 行为 `0, 1, 0, 2, 5`；第 5 行为 `0, 1, 1, 3, 8`。右下角为 8，所以选 D。A、B、C 都少统计了部分绕过障碍后重新汇合的路径。
