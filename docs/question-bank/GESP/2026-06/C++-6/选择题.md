## 第1题

下列关于 C++ 中继承和多态的描述中，错误的是（ ）。

A. 通过基类指针调用虚函数时，会根据对象实际类型决定调用版本。
B. 基类析构函数常声明为虚函数，以便通过基类指针正确释放派生类对象。
C. 派生类可以重写基类中的虚函数。
D. 构造函数可以声明为 `virtual`，以便在构造对象时实现动态绑定。

答案：D

解析：本题选 D。C++ 的构造函数不能声明为虚函数，因为对象在构造过程中其动态类型尚未完整形成，虚函数机制不能用于“虚构造”。A 描述的是虚函数的动态绑定，正确；B 是多态基类常见做法，否则通过基类指针删除派生类对象可能析构不完整；C 也是虚函数重写的基本用法。

## 第2题

下列代码中，`d1->work();` 和 `d2->work();` 输出不同结果的主要原因是（ ）。

```cpp
class Device {
public:
    virtual void work() {
        cout << "Device is working" << endl;
    }
    virtual ~Device() {}
};

class Printer : public Device {
public:
    void work() override {
        cout << "Printer is printing" << endl;
    }
};

class Scanner : public Device {
public:
    void work() override {
        cout << "Scanner is scanning" << endl;
    }
};

int main() {
    Device* d1 = new Printer();
    Device* d2 = new Scanner();

    d1->work();
    d2->work();

    delete d1;
    delete d2;
    return 0;
}
```

A. `Printer` 和 `Scanner` 使用了相同的构造函数。
B. `work()` 是虚函数，且 `d1` 和 `d2` 实际指向不同派生类对象，发生动态绑定。
C. `d1` 和 `d2` 是不同的指针变量。
D. 程序中使用了 `delete` 释放对象。

答案：B

解析：本题选 B。`work()` 在基类中声明为 `virtual`，因此通过 `Device*` 调用时会按实际对象类型绑定：`d1` 指向 `Printer`，调用 `Printer::work()`；`d2` 指向 `Scanner`，调用 `Scanner::work()`。A 与输出版本无关；C 只说明变量不同，不足以导致多态调用；D 发生在输出之后，也不是输出不同的原因。

## 第3题

下面代码在 `main()` 中有一行会导致编译错误，请找出来。

```cpp
class Student {
public:
    Student(string n, int s) : name(n), score(s) {}

    string getName() {
        return name;
    }

    void setScore(int s) {
        score = s;
    }

private:
    string name;
    int score;
};

int main() {
    Student stu("Tom", 85);
    cout << stu.getName();     // ①
    stu.setScore(90);          // ②
    stu.score = 100;           // ③
    cout << stu.getName();     // ④
    return 0;
}
```

A. 第 ① 行
B. 第 ② 行
C. 第 ③ 行
D. 第 ④ 行

答案：C

解析：本题选 C。`score` 被声明在 `private` 区域，只能由类的成员函数或友元访问，`main()` 中直接写 `stu.score = 100;` 会违反访问控制并导致编译错误。① 和 ④ 调用的是公开的 `getName()`，合法；② 调用公开的 `setScore()` 修改私有成员，也是封装的正确用法。

## 第4题

某文本编辑器把用户输入的字符依次压入栈 `S`。用户依次输入 `X, Y, Z, W` 后，连续执行两次撤销操作。每次撤销都会弹出栈顶一个字符。此时栈从栈底到栈顶的内容是（ ）。

A. `X Y`
B. `X Y Z`
C. `Y Z`
D. `X Z`

答案：A

解析：本题选 A。栈的特点是后进先出，依次压入后从栈底到栈顶为 `X Y Z W`。第一次撤销弹出栈顶 `W`，第二次撤销再弹出 `Z`，剩下 `X Y`。B 只弹出了一次；C 少了最早压入且未被弹出的 `X`；D 则错误保留了已经被第二次撤销弹出的 `Z`。

## 第5题

假设循环队列数组长度为 `N = 7`，队空判断条件为 `front == rear`。入队和出队操作如下：

```cpp
const int N = 7;
int q[N];
int front = 3, rear = 3;

void enqueue(int x) {
    q[rear] = x;
    rear = (rear + 1) % N;
}

void dequeue() {
    front = (front + 1) % N;
}
```

依次执行：

```cpp
enqueue(10);
enqueue(20);
enqueue(30);
dequeue();
enqueue(40);
dequeue();
enqueue(50);
```

最终 `(front, rear)` 的值是（ ）。

A. `(5, 1)`
B. `(4, 0)`
C. `(5, 0)`
D. `(3, 1)`

答案：A

解析：本题选 A。初始为 `(3, 3)`。三次入队只移动 `rear`，依次变为 4、5、6；第一次出队使 `front` 变为 4；再入队后 `rear = (6 + 1) % 7 = 0`；第二次出队使 `front` 变为 5；最后一次入队使 `rear` 变为 1，所以最终是 `(5, 1)`。B 少算了一次出队后的 `front` 移动；C 少算了最后一次入队后的 `rear` 移动；D 则基本停留在初始 `front`。

## 第6题

以下函数 `check()` 用于判断一棵二叉树是否为（ ）。

```cpp
bool check(TreeNode* root) {
    if (!root) return true;

    queue<TreeNode*> q;
    q.push(root);

    bool hasNull = false;
    while (!q.empty()) {
        TreeNode* cur = q.front();
        q.pop();

        if (cur == nullptr) {
            hasNull = true;
        } else {
            if (hasNull) return false;
            q.push(cur->left);
            q.push(cur->right);
        }
    }

    return true;
}
```

A. 满二叉树
B. 完全二叉树
C. 二叉搜索树
D. 平衡二叉树

答案：B

解析：本题选 B。函数按层序遍历把空孩子也入队，一旦见到空指针后，如果后面又出现非空结点，就说明树的最后一层没有从左到右连续填充，应返回 `false`。这正是完全二叉树的判定思路。满二叉树要求每个结点要么没有孩子、要么有两个孩子，本函数并不检查这个条件；二叉搜索树要比较左右子树值的大小；平衡二叉树要检查高度差，这些都不是代码逻辑。

## 第7题

以下代码实现了二叉树的哪种遍历方式？

```cpp
void traverse(TreeNode* root) {
    if (root == nullptr) return;

    cout << root->val << " ";
    traverse(root->left);
    traverse(root->right);
}
```

A. 前序遍历
B. 中序遍历
C. 后序遍历
D. 层序遍历

答案：A

解析：本题选 A。代码先访问根结点，再递归遍历左子树，最后递归遍历右子树，顺序是“根、左、右”，即前序遍历。中序遍历应为“左、根、右”；后序遍历应为“左、右、根”；层序遍历通常需要队列按层访问，而本题代码使用递归深度优先遍历。

## 第8题

已知一棵二叉树的先序遍历序列为：`A B D E H C F G`，中序遍历序列为：`D B H E A F C G`，则该二叉树的后序遍历序列是（ ）。

A. `D H E B F G C A`
B. `D E H B F G C A`
C. `H D E B F C G A`
D. `D H E B G F C A`

答案：A

解析：本题选 A。先序第一个结点 `A` 是根，在中序中 `A` 左侧为 `D B H E`，右侧为 `F C G`。左子树先序为 `B D E H`，根为 `B`，其中左孩子是 `D`，右子树由 `E H` 构成且 `H` 在 `E` 左边，所以左子树后序为 `D H E B`。右子树根为 `C`，左孩子 `F`、右孩子 `G`，后序为 `F G C`。合并为 `D H E B F G C A`。B 把 `E` 与 `H` 的父子顺序处理错；D 把右子树的 `F`、`G` 顺序处理错。

## 第9题

有 6 个字符，它们出现的次数分别为：`{3, 4, 7, 8, 12, 15}`，现在用哈夫曼编码为这些字符编码，最小加权路径长度 `WPL` 的值为（ ）。

A. 113
B. 119
C. 126
D. 31

答案：B

解析：本题选 B。哈夫曼树每次合并当前最小的两个权值，合并代价会计入 `WPL`。过程为：`3 + 4 = 7`，此时有 `7, 7, 8, 12, 15`；再合并 `7 + 7 = 14`；合并 `8 + 12 = 20`；合并 `14 + 15 = 29`；最后合并 `20 + 29 = 49`。总加权路径长度为 `7 + 14 + 20 + 29 + 49 = 119`。A、C 都不是按每轮最小合并累加得到的结果；D 只接近部分权值之和，明显没有计算整棵哈夫曼树的路径代价。

## 第10题

对 `n` 个不同符号进行哈夫曼编码。若生成的哈夫曼树共有 63 个结点，则 `n` 的值是（ ）。

A. 31
B. 32
C. 63
D. 64

答案：B

解析：本题选 B。哈夫曼树是每个内部结点都有两个孩子的二叉树，若叶子数为 `n`，内部结点数为 `n - 1`，总结点数为 `2n - 1`。由 `2n - 1 = 63` 得 `n = 32`。A 会得到 61 个结点；C、D 把总结点数或接近的数直接当作符号数，忽略了哈夫曼树内部结点。

## 第11题

在格雷码中，相邻两个编码只能有一位不同。若当前编码为 `110`，则它的下一个编码不可能是（ ）。

A. `010`
B. `111`
C. `100`
D. `001`

答案：D

解析：本题选 D。格雷码相邻编码只能相差一位。`110` 与 `010` 只第 1 位不同，与 `111` 只第 3 位不同，与 `100` 只第 2 位不同，都可能相邻；而 `110` 与 `001` 三位都不同，不满足格雷码相邻条件，所以不可能是下一个编码。

## 第12题

给定一棵二叉树，采用广度优先搜索 BFS 返回其右视图，其中右视图中的每个节点都是该层最右侧的节点。横线处应填写（ ）。

```cpp
vector<int> rightSideView(TreeNode* root) {
    vector<int> result;
    if (!root) return result;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int sz = q.size();

        for (int i = 0; i < sz; ++i) {
            TreeNode* node = q.front();
            q.pop();

            __________________________

            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }

    return result;
}
```

A. `if (i == 0) result.push_back(node->val);`
B. `if (i == sz - 1) result.push_back(node->val);`
C. `result.push_back(q.front()->val);`
D. `if (node->right) result.push_back(node->right->val);`

答案：B

解析：本题选 B。每一轮外层循环处理一整层，`sz` 是当前层结点数；由于代码按队列顺序从左到右弹出本层结点，`i == sz - 1` 时的 `node` 就是该层最右侧结点，应加入右视图。A 记录的是每层最左侧结点；C 在弹出当前结点后直接看队首，可能是同层下一个结点，也可能已经进入下一层，含义不稳定；D 记录的是右孩子而不是当前层最右结点，遇到没有右孩子但本身可见的结点会漏掉。

## 第13题

下面代码实现二叉搜索树的插入操作。假设树中不存在重复值，横线处应填写（ ）。

```cpp
TreeNode* insertNode(TreeNode* root, int x) {
    if (root == nullptr) {
        return new TreeNode(x);
    }

    if (x < root->val) {
        __________________________
    } else {
        root->right = insertNode(root->right, x);
    }

    return root;
}
```

A. `root->left = insertNode(root->left, x);`
B. `root = insertNode(root->left, x);`
C. `root->right = insertNode(root->left, x);`
D. `insertNode(root->left, x);`

答案：A

解析：本题选 A。二叉搜索树中，小于根结点的值应插入左子树；递归插入后还要把返回的新左子树根接回 `root->left`，否则原来为空的位置无法挂上新结点。B 会把当前根指针改成左子树，破坏整棵树；C 把左侧插入结果接到右孩子位置，方向错误；D 虽然递归了左子树，但忽略返回值，当左孩子为空时新结点会丢失。

## 第14题

给定一个整数数组 `a`，每个元素表示一个位置上的数值。要求从数组中选择若干个元素，使得任意两个被选择的元素在原数组中都不相邻，并且所选元素的总和最大。函数 `choose(vector<int>& a)` 返回能够得到的最大总和，则横线处应填写（ ）。

```cpp
int choose(vector<int>& a) {
    if (a.empty()) return 0;

    int n = a.size();
    if (n == 1) return a[0];

    vector<int> dp(n, 0);
    dp[0] = a[0];
    dp[1] = max(a[0], a[1]);

    for (int i = 2; i < n; ++i) {
        dp[i] = __________________________;
    }

    return dp[n - 1];
}
```

A. `dp[i - 1] + a[i]`
B. `max(dp[i - 1], dp[i - 2] + a[i])`
C. `max(dp[i - 2], a[i])`
D. `dp[i - 1] + dp[i - 2]`

答案：B

解析：本题选 B。设 `dp[i]` 表示考虑到下标 `i` 时能得到的最大和。对位置 `i`，要么不选它，答案为 `dp[i - 1]`；要么选它，则不能选相邻的 `i - 1`，答案为 `dp[i - 2] + a[i]`，取两者最大值。A 把 `dp[i - 1]` 和 `a[i]` 同时选入，可能选择相邻元素；C 忽略了“不选当前位置但保留前面最优解”的情况；D 把两个重叠子问题相加，会重复计算且不对应任何合法选择。

## 第15题

下面代码实现 0/1 背包的一维动态规划。第 `i` 个物品重量为 `wt[i]`，价值为 `val[i]`，背包容量为 `W`。横线处应填写（ ）。

```cpp
int knapsack(int W, vector<int>& wt, vector<int>& val) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);

    for (int i = 0; i < n; ++i) {
        for (int w = W; w >= wt[i]; --w) {
            __________________________
        }
    }

    return dp[W];
}
```

A. `dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);`
B. `dp[w] = max(dp[w - 1], dp[w - wt[i]] + val[i]);`
C. `dp[w] = dp[w] + val[i];`
D. `dp[w - wt[i]] = max(dp[w], val[i]);`

答案：A

解析：本题选 A。一维 0/1 背包中，`dp[w]` 表示当前已处理物品下容量为 `w` 的最大价值。处理第 `i` 个物品时，要么不选它，保持 `dp[w]`；要么选它，从容量 `w - wt[i]` 的状态转移来并加上 `val[i]`，所以更新为 `max(dp[w], dp[w - wt[i]] + val[i])`。循环容量从大到小，是为了保证每个物品最多使用一次。B 用 `dp[w - 1]` 代替原来的 `dp[w]`，会错误丢掉容量 `w` 的已有最优值；C 无条件加价值，相当于每个可放物品都必须选；D 更新了错误的容量位置。
