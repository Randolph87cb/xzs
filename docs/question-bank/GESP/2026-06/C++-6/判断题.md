## 第1题

C++ 中构造函数可以声明为虚函数，从而实现运行时多态。

A. 正确
B. 错误

答案：B

解析：C++ 的构造函数不能声明为 `virtual`。运行时多态通常依靠基类中的虚成员函数，并通过基类指针或引用调用派生类重写后的函数实现；构造对象时对象本身还在建立过程中，不能用“虚构造函数”来实现动态绑定。因此题干说法错误。

## 第2题

通过指向 `Base` 的指针删除 `Derived` 对象时，一定会先调用 `Derived` 的析构函数，再调用 `Base` 的析构函数。

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    ~Base() {
        cout << "Base destructor" << endl;
    }
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Derived destructor" << endl;
    }
};

int main() {
    Base* p = new Derived();
    delete p;
    return 0;
}
```

A. 正确
B. 错误

答案：B

解析：只有当基类析构函数声明为虚析构函数时，通过基类指针删除派生类对象，才会按派生类析构函数、基类析构函数的顺序正确析构。本题代码中的 `Base` 析构函数不是 `virtual`，通过 `Base*` 删除实际指向的 `Derived` 对象不能保证调用 `Derived` 的析构函数。题干中的“一定会”不成立。

## 第3题

在 C++ STL 中，`stack` 的 `pop()` 函数会返回栈顶元素并将其删除。

A. 正确
B. 错误

答案：B

解析：`stack::pop()` 的作用是删除栈顶元素，但它不返回被删除的元素，返回类型是 `void`。如果需要取得栈顶元素，应先调用 `top()` 读取，再调用 `pop()` 删除。因此题干把 `pop()` 的功能说成“返回并删除”，是错误的。

## 第4题

程序运行后会输出 `2`。

```cpp
int main() {
    queue<int> q;
    q.push(1);
    q.push(2);
    q.push(3);
    q.pop();
    cout << q.front() << endl;
    return 0;
}
```

A. 正确
B. 错误

答案：A

解析：队列是先进先出的结构。代码依次入队 `1, 2, 3`，此时队首是 `1`；执行一次 `q.pop()` 会删除队首元素 `1`，新的队首变为 `2`。随后 `q.front()` 取得队首元素，所以输出为 `2`，题干正确。

## 第5题

下列函数试图将整数 `x` 插入到一棵二叉搜索树中。假设二叉搜索树满足如下性质：对于任意结点，左子树中所有结点的值均小于该结点的值，右子树中所有结点的值均大于或等于该结点的值。判断该函数是否能够在插入后保持二叉搜索树性质。

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

TreeNode* insertNode(TreeNode* root, int x) {
    if (root == nullptr) {
        return new TreeNode(x);
    }

    if (x < root->val) {
        root->right = insertNode(root->right, x);
    } else {
        root->left = insertNode(root->left, x);
    }

    return root;
}
```

A. 正确
B. 错误

答案：B

解析：按照题目给出的二叉搜索树性质，若 `x < root->val`，新值应插入左子树；若 `x >= root->val`，新值应插入右子树。代码却把这两个方向写反了：小于根结点的值被放到右子树，大于或等于根结点的值被放到左子树。这样插入后会破坏二叉搜索树性质，所以题干错误。

## 第6题

哈夫曼编码一定唯一，只要字符频率相同，得到的编码也一定完全相同。

A. 正确
B. 错误

答案：B

解析：哈夫曼编码的最小加权路径长度可以是确定的，但具体编码不一定唯一。当多个结点权值相同，合并顺序、左右孩子放置顺序不同，都可能得到不同的哈夫曼树；即使是同一棵树，左右分支分配 `0` 和 `1` 的方式不同，编码也会变化。因此“频率相同就一定得到完全相同的编码”不成立。

## 第7题

若用数组按层序存储完全二叉树，且根节点下标为 `0`，则下标为 `i` 的节点左孩子下标为 `2 * i + 1`，右孩子下标为 `2 * i + 2`。

A. 正确
B. 错误

答案：A

解析：完全二叉树按层序连续存入数组，并且根结点使用下标 `0` 时，结点之间的下标关系正是左孩子为 `2 * i + 1`，右孩子为 `2 * i + 2`。这个公式依赖于“层序存储”和“从 0 开始编号”这两个条件；在本题条件下，题干说法正确。

## 第8题

以下代码可以正确地按层换行输出二叉树的节点值。

```cpp
void printByLevel(TreeNode* root) {
    if (!root) return;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        for (int i = 0; i < q.size(); ++i) {
            TreeNode* cur = q.front();
            q.pop();
            cout << cur->val << " ";

            if (cur->left) q.push(cur->left);
            if (cur->right) q.push(cur->right);
        }
        cout << endl;
    }
}
```

A. 正确
B. 错误

答案：B

解析：按层换行时，应在进入每一层循环前先记录当前层的结点个数，例如 `int sz = q.size();`，再循环 `sz` 次。本题代码在 `for` 条件中直接使用 `q.size()`，而循环过程中会不断弹出当前结点并把下一层孩子加入队列，队列大小会动态变化，导致一行中可能混入下一层结点，不能稳定地按层换行输出。因此题干错误。

## 第9题

使用栈非递归实现二叉树前序遍历时，若希望先访问左子树，通常应先将右孩子入栈，再将左孩子入栈。

A. 正确
B. 错误

答案：A

解析：前序遍历的访问顺序是根、左子树、右子树。栈是后进先出结构，后入栈的结点会先被弹出并访问。因此在非递归实现中，如果一个结点的左右孩子都存在，通常先把右孩子入栈，再把左孩子入栈，这样左孩子会先出栈，才能先访问左子树。题干正确。

## 第10题

动态规划问题通常要求具有最优子结构，并且常常存在重叠子问题。

A. 正确
B. 错误

答案：A

解析：动态规划适合把原问题拆成若干子问题，并用子问题的最优解推出原问题的最优解，这就是最优子结构；同时，许多动态规划问题会反复遇到相同子问题，因此可以记录状态结果，避免重复计算。题干中的“通常要求具有最优子结构，并且常常存在重叠子问题”符合动态规划的典型适用条件。
