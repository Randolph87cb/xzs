## 第1题

从 7 本不同的算法书和 5 本不同的数学书中选出 4 本，要求两类书都至少选 1 本，共有（ ）种不同选法。

A. 420
B. 455
C. 465
D. 495

答案：B

解析：本题选 B。先不管“两类书都至少选 1 本”的限制，从 12 本书中选 4 本有 $C(12,4)=495$ 种。需要排除全是算法书的 $C(7,4)=35$ 种，以及全是数学书的 $C(5,4)=5$ 种，所以答案是 $495-35-5=455$。D 是没有排除不合法情况的总数；A 少算了部分合法混合选择。

## 第2题

6 个人排成一排照相，其中甲、乙两人不能相邻，共有（ ）种不同排法。

A. 240
B. 480
C. 600
D. 720

答案：B

解析：本题选 B。6 个人任意排列共有 $6!=720$ 种。若甲、乙相邻，把两人看成一个整体，则有 $5!$ 种整体排列，且甲乙内部有 2 种顺序，所以相邻排法为 $2 \times 5! = 240$ 种。不能相邻的排法是 $720-240=480$。A 正好是相邻的情况；D 是没有扣除限制的总排列数。

## 第3题

展开式 $(x^2-\frac{1}{x})^6$ 中，常数项的系数为（ ）。

A. 6
B. 12
C. 15
D. 20

答案：C

解析：本题选 C。设从 6 个因子中选出 $k$ 个 $-\frac{1}{x}$，其余 $6-k$ 个选 $x^2$，这一项的 $x$ 指数为 $2(6-k)-k=12-3k$。要成为常数项，指数必须为 0，所以 $k=4$。常数项系数为 $C(6,4)(-1)^4=15$。A 只相当于选 1 个位置，B 和 D 都没有满足二项式项数与指数条件。

## 第4题

下面代码用于预处理组合数，横线处应填入的是（ ）。

```cpp
for (int i = 0; i <= n; i++) {
    c[i][0] = c[i][i] = 1;
    for (int j = 1; j < i; j++)
        c[i][j] = __________;
}
```

A. `c[i - 1][j - 1] + c[i - 1][j]`
B. `c[i][j - 1] + c[i - 1][j - 1]`
C. `c[i - 1][j] + c[i][j + 1]`
D. `c[i][j - 1] * c[i - 1][j]`

答案：A

解析：本题选 A。组合数满足杨辉三角递推式 $C(i,j)=C(i-1,j-1)+C(i-1,j)$，也就是当前位置由上一行左上方和上一行正上方相加得到。B 使用了当前行的左侧值，不是标准组合数递推；C 还依赖当前行右侧尚未计算的位置；D 把递推关系写成乘法，含义不对。

## 第5题

下列程序输出的值为（ ）。

```cpp
#include <iostream>
using namespace std;

long long qpow(long long a, long long b, long long mod) {
    long long ans = 1 % mod;
    while (b) {
        if (b & 1)
            ans = ans * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return ans;
}

int main() {
    cout << qpow(3, 20, 17) << endl;
    return 0;
}
```

A. 1
B. 4
C. 13
D. 16

答案：C

解析：本题选 C。函数 `qpow(3, 20, 17)` 计算的是 $3^{20} \bmod 17$。可以逐步化简：$3^4=81 \equiv 13 \pmod {17}$，$3^8 \equiv 13^2=169 \equiv 16 \pmod {17}$，$3^{16} \equiv 16^2=256 \equiv 1 \pmod {17}$，所以 $3^{20}=3^{16}\cdot 3^4 \equiv 1\cdot 13=13$。A 是只看到 $3^{16}\equiv 1$ 后漏乘 $3^4$；D 是 $3^8$ 的余数，不是最终结果。

## 第6题

归并排序每次把长度为 $n$ 的序列分成两个规模约为 $n/2$ 的子序列，递归排序后再用线性时间合并。该算法的时间复杂度通常为（ ）。

A. $O(n)$
B. $O(n^2)$
C. $O(\log n)$
D. $O(n \log n)$

答案：D

解析：本题选 D。归并排序每一层递归中，所有子问题合并的总工作量是 $O(n)$；序列长度每次约减半，递归层数是 $O(\log n)$，所以总时间复杂度为 $O(n \log n)$。A 忽略了多层递归；B 是常见简单排序的量级，归并排序不会达到平方级；C 只像是递归层数，没有计算每层的合并代价。

## 第7题

在平面直角坐标系中，三角形三个顶点为 $A(1,1)$、$B(5,2)$、$C(3,6)$，该三角形面积为（ ）。

A. 9
B. 10
C. 12
D. 18

答案：A

解析：本题选 A。取向量 $\overrightarrow{AB}=(4,1)$，$\overrightarrow{AC}=(2,5)$，平行四边形面积为叉积绝对值 $|4\times 5-1\times 2|=18$，三角形面积是其一半，即 9。D 是把平行四边形面积直接当成三角形面积；B、C 与叉积计算结果不符。

## 第8题

某程序需要判断点 $P(x,y)$ 是否在以原点为圆心、半径为 5 的圆内或圆上。下列判断条件正确的是（ ）。

A. `x * x + y * y <= 25`
B. `abs(x) + abs(y) <= 5`
C. `x * x - y * y <= 25`
D. `x + y <= 5`

答案：A

解析：本题选 A。点到原点的距离满足 $\sqrt{x^2+y^2}\le 5$，两边平方后就是 $x^2+y^2\le 25$，对应代码为 `x * x + y * y <= 25`。B 判断的是菱形区域，例如点 $(4,3)$ 在圆上但 `abs(x)+abs(y)=7`，会被误判；C 把加法写成减法，不能表示距离平方；D 没有处理坐标符号和平方，点 $(100,-100)$ 也可能满足 `x + y <= 5`，显然不在圆内。

## 第9题

某无向带权图有边 $(1,2,4)$、$(1,3,2)$、$(2,3,1)$、$(2,4,5)$、$(3,4,8)$、$(3,5,10)$、$(4,5,2)$。该图最小生成树的总权值为（ ）。

A. 7
B. 8
C. 9
D. 10

答案：D

解析：本题选 D。按 Kruskal 算法从小到大选边：先选 $(2,3,1)$，再选 $(1,3,2)$，再选 $(4,5,2)$。边 $(1,2,4)$ 会让 1、2、3 形成环，不能选；接着选 $(2,4,5)$ 把两个连通块连接起来。此时 5 个顶点已经用 4 条边连通，总权值为 $1+2+2+5=10$。A、B、C 都少算了连接两个连通块所必须付出的权值 5。

## 第10题

有向非负权图边为 $1 \to 2(3)$、$2 \to 4(4)$、$1 \to 3(10)$、$3 \to 4(1)$、$2 \to 3(2)$。使用 Dijkstra 算法从 1 号顶点出发到 4 号顶点的最短距离为（ ）。

A. 6
B. 7
C. 8
D. 11

答案：A

解析：本题选 A。从 1 出发，先得到到 2 的距离 3、到 3 的距离 10。确定顶点 2 后，可更新到 4 的距离为 $3+4=7$，到 3 的距离为 $3+2=5$。再确定顶点 3 后，可把到 4 的距离更新为 $5+1=6$。因此最短路径是 $1\to2\to3\to4$，距离为 6。B 是路径 $1\to2\to4$ 的长度；D 是路径 $1\to3\to4$ 的长度，二者都不是最短。

## 第11题

下列代码片段的时间复杂度为（ ）。

```cpp
long long s = 0;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j * j <= n; j++) {
        s += i + j;
    }
}
```

A. $O(n)$
B. $O(n \log n)$
C. $O(n\sqrt{n})$
D. $O(n^2)$

答案：C

解析：本题选 C。外层循环执行 $n$ 次。内层循环条件是 `j * j <= n`，因此 $j$ 最多增长到约 $\sqrt n$，每次外层循环中内层执行 $O(\sqrt n)$ 次，总复杂度为 $O(n\sqrt n)$。B 的 $O(n\log n)$ 通常来自倍增或折半过程，这里 `j` 是每次加 1；D 则相当于内层也循环到 $n$，与题目条件不符。

## 第12题

某优化问题的答案是 $[1,M]$ 内的整数，存在单调判定函数 `check(x)`，且每次判定的时间复杂度为 $O(n)$。使用二分答案求最小可行值，整体时间复杂度通常为（ ）。

A. $O(nM)$
B. $O(n \log M)$
C. $O(M \log n)$
D. $O(n + M)$

答案：B

解析：本题选 B。二分答案在范围 $[1,M]$ 上每次把搜索区间约缩小一半，需要 $O(\log M)$ 次判定；每次 `check(x)` 的时间是 $O(n)$，所以总时间复杂度为 $O(n\log M)$。A 更像是从 1 到 $M$ 逐个检查；C 把二分对象和判定代价混淆了；D 不是二分答案的乘法型代价。

## 第13题

下列线性筛的代码片段中，当枚举到质数 `p` 且 `i % p == 0` 时，使用 `break;` 语句停止继续枚举。这样做的主要目的是（ ）。

```cpp
for (int i = 2; i <= n; ++i) {
    if (!is_composite[i])
        primes.push_back(i);
    for (int p : primes) {
        if (i * p > n)
            break;
        is_composite[i * p] = true;
        if (i % p == 0)
            break; // 这条语句的目的是什么？
    }
}
```

A. 保证递归深度不超过 $O(\log n)$。
B. 保证每个合数只被它的最小质因子筛去一次。
C. 保证每个素数都被标记为合数。
D. 把筛法时间复杂度提高到 $O(n \log n)$。

答案：B

解析：本题选 B。线性筛的核心是让每个合数只在“最小质因子 × 另一个数”的那一次被标记。当 `i % p == 0` 时，说明 `p` 已经是 `i` 的一个质因子；继续用更大的质数去乘 `i`，会导致一些合数以后被重复标记，所以要 `break`。A 与这段循环无关，代码没有递归；C 说反了，素数不应被标记为合数；D 中“提高到 $O(n\log n)$”反而比线性筛目标更差。

## 第14题

在 C++ 中，关于类的继承和构造、析构顺序，下列说法正确的是（ ）。

A. 派生类可以直接访问基类的 `private` 成员。
B. 基类的 `protected` 成员在私有继承后会变成派生类的 `public` 成员。
C. 创建派生类对象时，会先调用基类构造函数，再调用派生类构造函数。
D. 销毁派生类对象时，会先调用基类析构函数，再调用派生类析构函数。

答案：C

解析：本题选 C。创建派生类对象时，需要先把基类部分构造好，再构造派生类自己的部分，因此构造顺序是“基类构造函数先，派生类构造函数后”。A 错在派生类不能直接访问基类的 `private` 成员；B 错在私有继承会使基类的 `public`、`protected` 成员在派生类中表现为私有访问级别，不会变成 `public`；D 的析构顺序也写反了，销毁时应先调用派生类析构函数，再调用基类析构函数。

## 第15题

将 4 个元素按 1, 2, 3, 4 的顺序入栈，在该过程中可随时插入出栈操作。下列序列中不可能作为出栈序列的是（ ）。

A. 1, 2, 3, 4
B. 2, 1, 4, 3
C. 3, 2, 1, 4
D. 3, 1, 2, 4

答案：D

解析：本题选 D。若第一个出栈元素是 3，必须先依次把 1、2、3 入栈，然后弹出 3。此时栈顶是 2，1 在 2 的下面，因此下一次只能先弹出 2，不能直接弹出 1，所以序列 3, 1, 2, 4 不可能出现。A 可以每入栈一个就出栈；B 可先入 1、2，弹出 2、1，再入 3、4，弹出 4、3；C 可入 1、2、3 后连续弹出 3、2、1，再弹出 4。
