## 第1题

数组的存储空间在物理上通常是连续的，而链表的结点可以存储在不连续的内存空间中。

答案：A

解析：本题判断为正确。数组在内存中通常按下标顺序连续存放，因此可以根据起始地址和下标快速计算元素位置。链表的每个结点只需要保存数据和指向其他结点的指针，结点本身可以分散在不同内存位置，通过指针连接起来即可。所以题干对数组和链表存储特点的描述是正确的。

## 第2题

带哨兵头尾节点的双向循环链表，在表头插入节点 `p`，以下四步操作无论什么顺序执行结果都正确。

```cpp
① p->next = head->next;
② p->prev = head;
③ head->next->prev = p;
④ head->next = p;
```

答案：B

解析：本题判断为错误。在双向链表表头插入结点时，需要先让 `p` 记录原来的后继结点，再修改原后继结点和 `head` 的指针，操作顺序会影响 `head->next` 表示的是“原来的第一个结点”还是“新结点 `p`”。例如如果先执行 `④ head->next = p;`，再执行 `③ head->next->prev = p;`，此时 `head->next` 已经是 `p`，就会把 `p->prev` 设成 `p`，链表结构错误。正确说法是：这四步应按能保持原后继信息不丢失的顺序执行，不能任意调换。

## 第3题

对任意正整数 `a`、`b`，以下两种写法的 `gcd` 函数返回值完全相同。

```cpp
int gcd1(int a, int b) {
    return b ? gcd1(b, a % b) : a;
}

int gcd2(int a, int b) {
    while (b) {
        int t = b;
        b = a % b;
        a = t;
    }
    return a;
}
```

答案：A

解析：本题判断为正确。两个函数都使用欧几里得算法，核心等式是 $\gcd(a,b)=\gcd(b,a\bmod b)$。`gcd1` 用递归不断把参数变为 `(b, a % b)`，直到 `b` 为 `0` 时返回 `a`；`gcd2` 用循环完成同样的状态转移，变量 `t` 只是临时保存原来的 `b`。对正整数 `a`、`b`，二者终止时得到的都是最大公约数，因此返回值完全相同。

## 第4题

在归并排序的合并操作中，如下代码片段可以正确地将两个已排序的子数组 `L` 和 `R` 合并回原数组 `arr` 中。

```cpp
void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;
    vector<int> L(n1), R(n2);

    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];
    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}
```

答案：A

解析：本题判断为正确。代码先把原数组中左右两个有序区间分别复制到临时数组 `L` 和 `R`，再用双指针比较两个数组当前最小的未合并元素，把较小者放回 `arr[k]`。当某一边先用完后，另一边剩余元素本来已经有序，直接依次复制回去即可。因此这段代码能正确完成归并排序中的合并操作。

## 第5题

分治法通常将一个规模较大的问题拆分为若干个规模较小、结构相似的子问题，分别求解后再合并子问题的结果。

答案：A

解析：本题判断为正确。分治法的典型过程就是“分解、解决、合并”：先把原问题拆成若干个规模更小且形式相似的子问题，分别求出子问题答案，再把这些答案合成为原问题答案。归并排序把数组分成两半分别排序后合并，快速排序围绕枢轴划分后分别处理两边，都是这种思想的体现。

## 第6题

贪心算法只要每一步选择当前最优解，就一定能得到全局最优解。

答案：B

解析：本题判断为错误。贪心算法的做法确实是每一步作出当前看来最优的选择，但不是所有问题都能由局部最优推出全局最优。只有问题满足贪心选择性质等条件时，贪心策略才一定正确。例如某些背包问题中，先选当前性价比最高或价值最高的物品，可能会导致后续空间无法组合出最优总价值。因此正确说法是：贪心算法需要证明局部最优选择不会破坏全局最优，不能只凭“每步当前最优”就保证正确。

## 第7题

二分查找不仅可以应用于有序数组，也可以在不增加时间复杂度的情况下应用于有序的单链表，因为链表也支持 $O(1)$ 时间内的随机访问。

答案：B

解析：本题判断为错误。二分查找要求能够快速访问中间位置，数组可以通过下标在 $O(1)$ 时间内随机访问任意元素，所以适合二分查找。单链表只能从某个结点沿 `next` 指针逐个向后走，不能在 $O(1)$ 时间内直接访问中间结点；每次寻找中点都可能需要线性遍历。因此把二分查找直接用于单链表通常会增加时间开销，题干中“链表也支持 $O(1)$ 随机访问”的说法错误。

## 第8题

以下函数 `f1` 的时间复杂度比函数 `f2` 的更高。

```cpp
void f1(int n) {
    for (int i = 1; i < n; i *= 2);
}

void f2(int n) {
    if (n <= 1) return;
    f2(n - 1);
    f2(n - 1);
}
```

答案：B

解析：本题判断为错误。`f1` 中变量 `i` 每次乘以 `2`，循环次数大约是 $\log_2 n$，时间复杂度为 $O(\log n)$。`f2` 每次在规模减少 `1` 后递归调用两次，递归调用数量会成倍增长，时间复杂度是指数级的 $O(2^n)$。因此 `f2` 的时间复杂度远高于 `f1`，题干把二者高低关系说反了。

## 第9题

唯一分解定理表明，任何一个大于 `1` 的自然数都可以唯一地分解为若干个质数的乘积，如果不考虑质因数的顺序，这种分解方式是唯一的。

答案：A

解析：本题判断为正确。唯一分解定理说明，每个大于 `1` 的整数都能写成若干个质数相乘的形式，并且除质因数排列顺序不同外，分解结果唯一。例如 `60 = 2 × 2 × 3 × 5`，也可以写成 `3 × 2 × 5 × 2`，但质因数的种类和出现次数没有改变。因此“不考虑质因数顺序时分解唯一”的说法正确。

## 第10题

归并排序和快速排序在平均情况下的时间复杂度均为 $O(n\log n)$。但在稳定性方面，归并排序通常是不稳定的，而快速排序是稳定的。

答案：B

解析：本题判断为错误。归并排序和快速排序的平均时间复杂度通常都可以达到 $O(n\log n)$，这部分说法正确；错误在稳定性的描述。标准归并排序在合并时如果相等元素优先取左半部分，就能保持相等元素的原相对顺序，通常可以实现为稳定排序。普通快速排序在划分和交换元素时可能改变相等元素的相对顺序，通常是不稳定排序。因此题干把二者的稳定性说反了。
