## 第1题

执行以下 C++ 代码后，`c` 的数值是（ ）。

```cpp
int a = 10, b = 3;
double c = a / b;
```

【知识点：运算符、表达式与类型转换】

A. 3.33333
B. 3.333
C. 3.0
D. 3.3

答案：C

解析：表达式 `a / b` 的两个操作数 `a` 和 `b` 都是 `int`，所以先按整数除法计算，`10 / 3` 的结果是 `3`，小数部分会被舍去。之后把这个整数结果赋给 `double c`，才转换成浮点数，因此 `c` 的值是 `3.0`，不是 `3.333...`。



## 第2题

下列 C++ 表达式的结果为 `true` 的是（ ）。

【知识点：运算符、表达式与类型转换】

A. `(5 <= 5) && (7 < 5)`
B. `!(10 > 5)`
C. `(10 != 10) || (5 >= 3)`
D. `(5 == 3) && (4 > 2)`

答案：C

解析：逐项计算逻辑表达式。A 中 `5 <= 5` 为 `true`，但 `7 < 5` 为 `false`，`true && false` 为 `false`。B 中 `10 > 5` 为 `true`，取反后为 `false`。C 中 `10 != 10` 为 `false`，`5 >= 3` 为 `true`，`false || true` 为 `true`。D 中 `5 == 3` 为 `false`，因此整个 `&&` 表达式为 `false`。所以只有 C 的结果为 `true`。



## 第3题

以下关于 C++ 数组的说法，错误的是（ ）。

【知识点：数组与字符数组】

A. 数组的下标通常从 0 开始。
B. `int arr[5];` 声明了一个包含 5 个整数的数组。
C. 数组的大小必须在编译时确定，不能使用变量定义大小。
D. 可以通过 `arr[5]` 来访问 `int arr[5];` 数组的最后一个元素。

答案：D

解析：`int arr[5];` 定义了 5 个元素，合法下标是 `0, 1, 2, 3, 4`，最后一个元素应写作 `arr[4]`。`arr[5]` 已经越过数组末尾，访问它属于越界访问，因此 D 的说法错误。A、B 是数组的基本规则；C 在标准 C++ 的普通内置数组中也成立，数组长度应为编译期可确定的常量表达式。



## 第4题

执行以下 C++ 代码后，变量 `sum` 的值是（ ）。

```cpp
int sum = 0;
for (int i = 1; i <= 5; i += 2) {
    sum += i;
    int sum=0;
}
```

【知识点：控制结构与程序阅读】

A. 6
B. 9
C. 15
D. 死循环

答案：B

解析：外层的 `sum` 初始为 `0`。循环变量 `i` 依次取 `1, 3, 5`，因为每次执行后 `i += 2`，到 `7` 时不满足 `i <= 5`。三次循环中先执行 `sum += i`，外层 `sum` 依次变为 `1, 4, 9`。循环体内的 `int sum=0;` 又声明了一个新的局部变量，它只在本次循环体后半段有效，不会把外层 `sum` 清零。因此循环结束后外层 `sum` 的值是 `9`。



## 第5题

要正确定义一个返回两个整数中较大值的函数 `max`，应该使用（ ）。

【知识点：控制结构与程序阅读】

A. `void max(int a, int b) { return a > b ? a : b; }`
B. `int max(int a, int b) { if (a > b) return a; else return b; }`
C. `int max(a, b) { if (a > b) return a; else return b; }`
D. `void max(a, b) { cout << (a > b ? a : b); }`

答案：B

解析：题目要求函数“返回”两个整数中的较大值，因此函数返回类型应是 `int`，两个参数也都要写明类型。B 中 `int max(int a, int b)` 参数和返回类型都正确，并且 `if (a > b)` 时返回 `a`，否则返回 `b`。A 的返回类型是 `void` 却返回整数，不符合要求；C、D 的参数没有写类型，不是正确的 C++ 函数定义；D 还只是输出结果，并没有返回较大值。



## 第6题

执行以下 C++ 代码后，数组 `arr` 的内容是（ ）。

```cpp
int arr[4] = {1, 2, 3};
arr[3] = arr[0] + arr[2];
```

【知识点：数组与字符数组】

A. {1, 2, 3, 3}
B. {1, 2, 3, 4}
C. {1, 2, 3, 5}
D. {1, 2, 3, 6}

答案：B

解析：`int arr[4] = {1, 2, 3};` 只显式给出前三个元素，剩下的 `arr[3]` 会被初始化为 `0`，所以开始时数组是 `{1, 2, 3, 0}`。随后执行 `arr[3] = arr[0] + arr[2];`，其中 `arr[0] = 1`，`arr[2] = 3`，相加得到 `4`。最终数组内容为 `{1, 2, 3, 4}`。



## 第7题

以下关于 C++ 函数的描述，正确的是（ ）。

【知识点：函数与标准库】

A. 函数必须要有参数。
B. 函数通过 `return` 语句只能返回一个值。但是可以通过很多间接的方式返回多个值。
C. `main` 函数可以被其他函数调用。
D. 函数的定义可以直接嵌套，即一个函数内部可以真正定义另一个函数。

答案：B

解析：函数可以没有参数，例如 `void f()`，所以 A 错。一次 `return` 语句只能把一个返回值交回调用处，但这个返回值可以是结构体、容器等复合对象，也可以通过引用参数、指针参数等方式间接带回多个结果，所以 B 正确。标准 C++ 中不应在程序中调用 `main` 函数，C 错。C++ 不支持在一个函数体内真正定义另一个普通函数，D 错。



## 第8题

以下 C++ 代码 `count++` 执行的次数是（ ）。

```cpp
int i = 10;
int count=0;
while (i > 0) {
    i -= 3;
    continue;
    count++;
}
```

【知识点：控制结构与程序阅读】

A. 2
B. 3
C. 4
D. 0

答案：D

解析：`continue` 会立即结束本轮循环，跳到下一次循环条件判断处，因此它后面的 `count++` 在每一轮都会被跳过。`i` 的变化过程是 `10 -> 7 -> 4 -> 1 -> -2`，之后 `i > 0` 不成立，循环结束。虽然循环体执行了 4 次，但每次都在 `count++` 前执行了 `continue`，所以 `count++` 执行次数为 `0`。



## 第9题

以下 C++ 代码段的输出是（ ）。

```cpp
for (int i = 0; i < 4; i++) {
    for (int j = 0; j <= i; j++) {
        cout << j;
    }
    cout << "#";
}
```

【知识点：进制转换与进制字面量】

A. 0#01#012#0123#
B. 1#12#123#1234#
C. 0#1#2#3#
D. 0#01#012#012343#

答案：A

解析：外层循环中 `i` 依次为 `0, 1, 2, 3`。当 `i = 0` 时，内层 `j` 只输出 `0`，再输出 `#`，得到 `0#`；当 `i = 1` 时输出 `01#`；当 `i = 2` 时输出 `012#`；当 `i = 3` 时输出 `0123#`。把四段连起来就是 `0#01#012#0123#`。



## 第10题

以下关于 C++ 变量作用域的说法，错误的是（ ）。

【知识点：控制结构与程序阅读】

A. 在 `for` 循环语句中声明的变量，其作用域仅限于该循环体内。
B. 在函数内部声明的变量（局部变量），仅在函数内部有效。
C. 在所有函数外部声明的变量，在整个程序中都有效。
D. 不同函数中的局部变量可以同名，它们代表不同的内存单元。

答案：C

解析：在函数外部声明的变量通常称为全局变量，但“在整个程序中都有效”这个说法过于绝对。名字的可见范围从声明处开始，并不包括声明之前的位置；跨源文件使用时还涉及链接属性和额外声明，并不是随处直接可用。因此 C 错。循环语句中声明的变量离开该循环后不能继续使用，函数内部局部变量只在函数内有效，不同函数里的同名局部变量也互不影响，A、B、D 的方向是正确的。



## 第11题

关于以下代码的说法正确的是（ ）。

```cpp
int reversed = 0;
while (x != 0) {
    int digit = x % 10;
    x /= 10;
    reversed = reversed * 10 + digit;
}
```

【知识点：枚举与基础算法】

A. 能够反转任何位数的整数
B. 能够反转的最大位数正整数是 $2147483647$
C. 能够反转的最大位数正整数是 $2147483648$
D. 能够反转的最大位数正整数是 $1463847412$

答案：D

解析：这段代码每次取出 `x` 的个位 `digit = x % 10`，再执行 `reversed = reversed * 10 + digit`，相当于把十进制数字从低位到高位依次接到 `reversed` 后面。若使用常见的 32 位 `int`，最大正数是 `2147483647`，反转较大的 10 位数时可能溢出。选项 D 中 `1463847412` 反转后是 `2147483641`，仍不超过 `2147483647`；而 `2147483647` 反转会得到 `7463847412`，已经超过 `int` 范围，`2147483648` 本身也超过 32 位有符号 `int` 的最大值。因此在给出的选项中应选 D。实际编写这类程序时，应在执行 `reversed * 10 + digit` 前先判断是否会溢出。



## 第12题

以下 C++ 代码试图查找数组中的最大值，划线处应填入（ ）。

```cpp
#include <iostream>
using namespace std;

int findMax(int arr[], int size) {
    int maxVal = _______; // 划线处
    for (int i = 1; i < size; i++) {
        if (arr[i] > maxVal) {
            maxVal = arr[i];
        }
    }
    return maxVal;
}
```

【知识点：数组与字符数组】

A. 0
B. `arr[-1]`
C. `arr[0]`
D. `size`

答案：C

解析：查找数组最大值时，初始最大值应该取自数组中的一个合法元素，然后再与后续元素比较。循环从 `i = 1` 开始，说明 `arr[0]` 已经被用作初始值最合适。若填 `0`，当数组元素全为负数时会得到错误结果；`arr[-1]` 是非法下标；`size` 表示数组长度，不是数组中的元素值。因此应填 `arr[0]`。



## 第13题

以下关于 C++ 函数的说法，正确的是（ ）。

【知识点：函数与标准库】

A. 函数参数传递只有值传递一种方式。
B. 函数的形参在函数调用结束后依然占用内存空间
C. 没有返回值的函数必须声明为 `void` 类型，且不能包含 `return` 语句
D. C++11 及之后标准要求函数必须显式声明返回类型，不允许默认返回 `int`

答案：D

解析：C++ 函数的参数传递不只有值传递，还可以使用引用、指针等方式，所以 A 错。普通形参通常是函数调用期间的局部对象，函数调用结束后其生命周期也结束，B 错。没有返回值的函数应声明为 `void`，但可以写不带表达式的 `return;` 提前结束函数，C 错。C++11 及之后的函数定义必须有明确的返回类型，不能像早期 C 语言那样省略返回类型并默认当作 `int`，所以 D 正确。



## 第14题

以下 C++ 代码中存在几处错误（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    const int SIZE = 5;
    int arr[SIZE];
    for (int i = 0; i <= SIZE; i++) {
        arr[i] = i * 2;
    }
    cout << arr[SIZE] << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. 0 处
B. 1 处
C. 2 处
D. 3 处

答案：C

解析：`arr` 的长度是 `5`，合法下标只有 `0` 到 `4`。第一处错误在循环条件 `i <= SIZE`：当 `i` 等于 `5` 时仍会进入循环，执行 `arr[5] = i * 2;`，造成越界写入。第二处错误是 `cout << arr[SIZE]`，也就是读取 `arr[5]`，再次越界。`const int SIZE = 5;` 可用于定义数组长度，`return 0;` 也没有问题，所以共有 2 处错误。



## 第15题

以下关于 C++ 中 `string` 类和字符数组（`char[]`）的说法，错误的是（ ）。

【知识点：数组与字符数组】

A. `string` 对象可以使用 `=` 进行赋值，而字符数组需要使用 `strcpy`。
B. `string` 对象的长度可以使用 `length()` 成员函数获取，而字符数组需要使用 `strlen()` 函数。
C. `string` 对象在内存中是动态分配空间的，因此可以自动处理字符串长度的变化。
D. `string` 对象和字符数组都可以使用 `==` 运算符来直接比较两个字符串的内容是否相同。

答案：D

解析：`string` 重载了 `==`，所以两个 `string` 对象可以用 `==` 比较内容。但字符数组名在表达式中通常会转换为指向首元素的指针，两个字符数组用 `==` 比较时比较的是地址关系，不会逐字符比较内容；比较 C 风格字符串内容应使用 `strcmp` 等函数。因此 D 的说法错误。A、B、C 都体现了 `string` 相比字符数组更方便管理和操作字符串的特点。
