## 第1题

表达式 `sizeof('a')` 的结果总是 $1$，因为 `'a'` 是一个字符。

【知识点：运算符、表达式与类型转换】

答案：正确

解析：题干表述正确。这里需要特别区分 C 和 C++：在 C++ 中，普通字符字面量 `'a'` 的类型是 `char`，而 `sizeof(char)` 的结果按定义为 `1`，所以 `sizeof('a')` 的结果应为 `1`。官方答案表给出“错误/×”，疑似混入了 C 语言规则或其他语境；本题库按 C++ 标准结论处理，因此判断为正确。



## 第2题

在 C++ 中，所有全局变量如果没有显式初始化，都会被自动初始化为 $0$。

【知识点：C++语法规则与编译模型】

答案：正确

解析：题干表述正确。全局变量具有静态存储期，如果没有显式初始化，会先进行零初始化。对整数类型来说就是初始化为 `0`；对指针类型来说会初始化为空指针。它们不是像普通局部变量那样留下不确定值，因此本题判断为正确。



## 第3题

`do { ... } while (false);` 循环体内的语句至少会被执行一次。

【知识点：控制结构与程序阅读】

答案：正确

解析：题干表述正确。`do while` 的执行顺序是先执行一次循环体，再判断 `while` 后面的条件。即使条件一开始就是 `false`，第一次循环体也已经执行完了，然后才因为条件为假而退出。因此 `do { ... } while (false);` 的循环体至少执行一次。



## 第4题

在 C++ 中，`++i` 是一个左值表达式，而 `i++` 是一个右值表达式。

【知识点：运算符、表达式与类型转换】

答案：正确

解析：题干表述正确。前置自增 `++i` 会先修改 `i`，表达式结果仍表示被修改后的这个对象本身，因此是左值表达式。后置自增 `i++` 会保存自增前的旧值作为表达式结果，再修改 `i`，这个旧值结果不是可继续赋值的对象，因此是右值表达式。



## 第5题

对于 `enum Color { RED, GREEN, BLUE };`，RED 的类型是 `int`。

【知识点：C++语法规则与编译模型】

答案：错误

解析：题干表述错误。`RED` 是枚举 `Color` 中的枚举项，它的类型属于这个枚举类型，而不是直接等同于 `int`。无作用域枚举的枚举值通常可以隐式转换为整数，底层表示也常用整数保存，但“可以转换为 `int`”不等于“类型就是 `int`”。



## 第6题

`#define SQUARE(x) x * x` 是一个安全的宏定义，`SQUARE(2+3)` 会正确计算出 $25$。

【知识点：C++语法规则与编译模型】

答案：错误

解析：题干表述错误。宏只是文本替换，`SQUARE(2+3)` 会展开为 `2+3 * 2+3`，乘法优先级高于加法，所以计算结果是 `2 + 6 + 3 = 11`，不是 `25`。安全的写法至少应给参数和整体结果加括号，例如 `#define SQUARE(x) ((x) * (x))`，因此原宏不安全。



## 第7题

在 C++ 中，`char` 类型的取值范围总是 $-128$ 到 $127$。

【知识点：运算符、表达式与类型转换】

答案：错误

解析：题干表述错误。C++ 中普通 `char` 是否按有符号数解释由实现决定：有的环境中 `char` 的范围类似 `signed char`，有的环境中类似 `unsigned char`。因此不能说 `char` 的取值范围总是 `-128` 到 `127`；如果需要明确范围，应使用 `signed char` 或 `unsigned char`。



## 第8题

表达式 `a > b ? a : b = 10;` 一定是合法的 C++ 代码。

【知识点：运算符、表达式与类型转换】

答案：错误

解析：题干表述错误，关键在于“一定合法”过于绝对。这个表达式涉及条件运算符、比较和赋值，是否合法取决于 `a`、`b` 是否已经声明、类型是否能比较、`b` 是否能被赋值，以及条件运算符两侧类型是否满足规则。例如 `b` 是 `const` 对象时，`b = 10` 就不合法。因此不能脱离上下文断言 `a > b ? a : b = 10;` 一定是合法的 C++ 代码；实际书写时也应加括号明确意图。



## 第9题

`#include "file.h"` 和 `#include <file.h>` 在编译器查找头文件时的搜索策略是完全相同的。

【知识点：C++语法规则与编译模型】

答案：错误

解析：题干表述错误。`#include "file.h"` 通常会优先在当前源文件所在目录或项目指定的本地目录中查找，再到系统包含路径中查找；`#include <file.h>` 通常用于标准库或系统头文件，主要按系统包含路径查找。两者的搜索策略不完全相同，所以不能说完全相同。



## 第10题

在同一个作用域内，`extern` 声明的变量可以多次定义。

【知识点：C++语法规则与编译模型】

答案：错误

解析：题干表述错误。`extern int x;` 这样的语句通常只是声明，表示变量 `x` 的定义在别处；同一个变量可以有多次声明。但“声明”不等于“定义”，变量的定义仍然必须遵守单一定义规则，不能在同一个作用域内多次定义同一个变量。若写成带初始化的 `extern int x = 0;`，它就是定义，重复出现会造成重定义问题。
