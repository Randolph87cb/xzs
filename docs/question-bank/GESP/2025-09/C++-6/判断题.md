## 第1题

当基类可能被多态使用，其析构函数应该声明为虚函数。

答案：正确

解析：正确。若基类对象可能被当作多态基类使用，例如用基类指针指向派生类对象，并通过基类指针删除对象，那么析构函数需要支持动态绑定。把基类析构函数声明为 `virtual` 后，`delete` 基类指针时会先调用派生类析构函数，再调用基类析构函数，资源释放顺序才完整。因此题干说“当基类可能被多态使用，其析构函数应该声明为虚函数”是成立的。



## 第2题

哈夫曼编码是最优前缀码，且编码结果唯一。

答案：错误

解析：错误。哈夫曼编码确实能构造最优前缀码，即在给定频率下使带权路径长度最小。但编码结果不唯一：合并时若有相同或可并列选择的频率，选择顺序可能不同；即使树的结构相同，每个内部节点的左右子树交换后，`0` 和 `1` 的分配也会变化。这些不同编码仍然可能都是最优前缀码，所以“编码结果唯一”不成立。



## 第3题

一个含有 100 个节点的完全二叉树，高度为 8。

答案：错误

解析：错误。完全二叉树按层从左到右填充。若高度按“层数”计算，高度为 `h` 的完全二叉树节点数范围是 $2^{h-1}$ 到 $2^h-1$。因为 $2^6 = 64 \le 100 < 128 = 2^7$，所以 100 个节点的完全二叉树共有 7 层，高度不是 8。若按“边数”计算高度，则对应为 6，也同样不是 8。



## 第4题

在 C++ STL 中，栈（`std::stack`）的 `pop` 操作返回栈顶元素并移除它。

答案：错误

解析：错误。在 C++ STL 的 `std::stack` 中，`top()` 用来访问栈顶元素，`pop()` 只负责移除栈顶元素且返回值类型是 `void`。如果需要“取出并删除”栈顶元素，通常要先保存 `top()` 的结果，再调用 `pop()`。题干说 `pop` 会返回栈顶元素并移除它，和 STL 接口不符。



## 第5题

循环队列通过模运算循环使用空间。

答案：正确

解析：正确。循环队列通常用固定大小数组存储元素，并用队首、队尾下标表示当前队列范围。当下标向后移动到数组末尾后，通过 `(下标 + 1) % 容量` 回到数组开头，从而重复利用已经出队后空出来的位置。这正是“循环使用空间”的含义。



## 第6题

一棵有 $n$ 个节点的二叉树一定有 $n - 1$ 条边。

答案：正确

解析：正确。树是一种连通且无环的结构。除根节点外，每个节点都有且只有一条边连接到它的父节点，因此 `n` 个节点中有 `n - 1` 个非根节点，也就对应 `n - 1` 条父子边。二叉树只是每个节点最多有两个孩子的树，仍满足树的这一基本性质。



## 第7题

以下代码实现了二叉树的中序遍历。输入以下二叉树，中序遍历结果是 4 2 5 1 3 6 。

```cpp
//       1
//      / \
//     2   3
//    / \   \
//   4   5   6

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

void inorderIterative(TreeNode* root) {
    stack<TreeNode*> st;
    TreeNode* curr = root;

    while (curr || !st.empty()) {
        while (curr) {
            st.push(curr);
            curr = curr->left;
        }
        curr = st.top(); st.pop();
        cout << curr->val << " ";
        curr = curr->right;
    }
}
```

答案：正确

解析：正确。中序遍历顺序是“左子树、根节点、右子树”。这段迭代代码先不断把当前节点及其左孩子压栈，直到最左节点；然后弹栈访问节点，再转向它的右子树，正是在模拟中序遍历。对题中二叉树，先访问 `4`，回到 `2`，再访问 `5`，然后访问根 `1`，最后访问右子树中的 `3` 和 `6`，结果为 `4 2 5 1 3 6`，所以判断为正确。



## 第8题

下面代码实现的二叉排序树的查找操作时间复杂度为 $O(h)$，其中 $h$ 为树高。

```cpp
TreeNode* searchBST(TreeNode* root, int val) {
    while (root && root->val != val) {
        root = (val < root->val) ? root->left : root->right;
    }
    return root;
}
```

答案：正确

解析：正确。二叉排序树查找时，每次比较当前节点值和目标值，只会向左子树或右子树中的一个方向继续走，不会同时搜索两边。循环每执行一次，最多向下移动一层；从根走到目标节点或空指针，最多经过树高 `h` 层。因此时间复杂度是 $O(h)$。如果树平衡，`h` 约为 $\log n$；如果退化成链，`h` 可达 `n`，但用树高表示就是 $O(h)$。



## 第9题

下面代码实现了动态规划版本的斐波那契数列计算，其时间复杂度为 $O(2^n)$。

```cpp
int fib_dp(int n) {
    if (n <= 1) return n;
    vector<int> dp(n+1);
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}
```

答案：错误

解析：错误。这段代码用 `dp[i]` 保存第 `i` 项斐波那契数，先初始化 `dp[0]` 和 `dp[1]`，再用循环从 `2` 计算到 `n`，每个状态只计算一次。循环次数与 `n` 成正比，所以时间复杂度是 $O(n)$，空间复杂度是 $O(n)$。$O(2^n)$ 是朴素递归反复计算子问题时常见的复杂度，不是这个动态规划版本的复杂度。



## 第10题

有一排香蕉，每个香蕉有不同的甜度值。小猴子想吃香蕉，但不能吃相邻的香蕉。以下代码能找到小猴子吃到最甜的香蕉组合。

```cpp
// bananas: 香蕉的甜度
void findSelectedBananas(vector<int>& bananas, vector<int>& dp) {
    vector<int> selected;
    int i = bananas.size() - 1;

    while (i >= 0) {
        if (i == 0) {
            selected.push_back(0);
            break;
        }

        if (dp[i] == dp[i-1]) {
            i--;
        } else {
            selected.push_back(i);
            i -= 2;
        }
    }

    reverse(selected.begin(), selected.end());
    cout << "小猴子吃了第: ";
    for (int idx : selected)
        cout << idx+1 << " ";
    cout << "个香蕉" << endl;
}

int main() {
    vector<int> bananas = {1, 2, 3, 1}; // 每个香蕉的甜
    vector<int> dp(bananas.size());
    dp[0] = bananas[0];
    dp[1] = max(bananas[0], bananas[1]);
    for (int i = 2; i < bananas.size(); i++) {
        dp[i] = max(bananas[i] + dp[i-2], dp[i-1]);
    }
    findSelectedBananas(bananas, dp);
    return 0;
}
```

答案：正确

解析：正确。主函数中的 `dp[i]` 表示只考虑第 `0` 到第 `i` 个香蕉时，在不吃相邻香蕉的限制下能得到的最大甜度。转移 `dp[i] = max(bananas[i] + dp[i-2], dp[i-1])` 分别对应“吃第 `i` 个”和“不吃第 `i` 个”。`findSelectedBananas` 从最后一个位置倒推：如果 `dp[i] == dp[i-1]`，说明最优解可以不选第 `i` 个；否则选第 `i` 个，并跳过相邻的 `i-1`。样例 `{1, 2, 3, 1}` 的 `dp` 为 `[1, 2, 4, 4]`，回溯会选择第 1 个和第 3 个香蕉，甜度总和为 4，符合最优且不相邻，所以判断为正确。
