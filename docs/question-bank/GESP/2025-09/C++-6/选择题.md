## 第1题

下列关于类的说法，错误的是（ ）。

A. 构造函数不能声明为虚函数，但析构函数可以。
B. 函数参数如声明为类的引用类型，调用时不会调用该类的复制构造函数。
C. 静态方法属于类而不是某个具体对象，因此推荐用 类名::方法(...) 调用。
D. 不管基类的析构函数是否是虚函数，都可以通过基类指针/引用正确删除派生类对象。

答案：D

解析：构造函数负责创建对象，不能声明为虚函数；析构函数可以声明为虚函数，并且当基类可能通过指针指向派生类对象并被删除时，通常应声明为虚析构函数。引用参数不会按值复制实参，因此不会因此调用复制构造函数；静态成员函数属于类本身，用 `类名::方法(...)` 调用也合理。D 项说“无论基类析构函数是否为虚函数都能正确删除派生类对象”，这不成立：通过基类指针 `delete` 派生类对象时，如果基类析构函数不是虚函数，派生类析构过程不能得到正确保证，行为不安全，所以选 D。



## 第2题

假设变量 `veh` 是类 `Car` 的一个实例，我们可以调用 `veh.move()`，是因为面向对象编程有（ ）性质。

```cpp
class Vehicle {
private:
    string brand;

public:
    Vehicle(string b) : brand(b) {}

    void setBrand(const string& b) { brand = b; }
    string getBrand() const { return brand; }

    void move() const {
        cout << brand << " is moving..." << endl;
    }
};

class Car : public Vehicle {
private:
    int seatCount;

public:
    Car(string b, int seats) : Vehicle(b), seatCount(seats) {}

    void showInfo() const {
        cout << "This car is a " << getBrand()
             << " with " << seatCount << " seats." << endl;
    }
};
```

A. 继承 (Inheritance)
B. 封装 (Encapsulation)
C. 多态 (Polymorphism)
D. 链接 (Linking)

答案：A

解析：`Car` 使用 `public Vehicle` 继承了 `Vehicle`，因此 `Vehicle` 中的 `public` 成员函数 `move()` 也成为 `Car` 对象可以访问的接口。变量 `veh` 若是 `Car` 的实例，调用 `veh.move()` 的原因是它继承了基类的成员函数。封装强调把数据如 `brand` 设为 `private` 并通过接口访问；多态强调同一接口在不同动态类型上表现不同；“链接”不是面向对象的基本性质，所以选 A。



## 第3题

下面代码中 `v1` 和 `v2` 调用了相同接口 `move()`，但输出结果不同。这体现了面向对象编程的（ ）特性。

```cpp
class Vehicle {
private:
    string brand;

public:
    Vehicle(string b) : brand(b) {}

    void setBrand(const string& b) { brand = b; }
    string getBrand() const { return brand; }

    virtual void move() const {
        cout << brand << " is moving..." << endl;
    }
};

class Car : public Vehicle {
private:
    int seatCount;

public:
    Car(string b, int seats) : Vehicle(b), seatCount(seats) {}

    void showInfo() const {
        cout << "This car is a " << getBrand()
             << " with " << seatCount << " seats." << endl;
    }

    void move() const override {
        cout << getBrand() << " car is driving on the road!" << endl;
    }
};

class Bike : public Vehicle {
public:
    Bike(string b) : Vehicle(b) {}

    void move() const override {
        cout << getBrand() << " bike is cycling on the path!" << endl;
    }
};

int main() {
    Vehicle* v1 = new Car("Toyota", 5);
    Vehicle* v2 = new Bike("Giant");

    v1->move();
    v2->move();

    delete v1;
    delete v2;
    return 0;
}
```

A. 继承 (Inheritance)
B. 封装 (Encapsulation)
C. 多态 (Polymorphism)
D. 链接 (Linking)

答案：C

解析：`Vehicle::move()` 被声明为 `virtual`，`Car` 和 `Bike` 又分别用 `override` 重写了这个函数。`v1`、`v2` 的静态类型都是 `Vehicle*`，但它们实际指向的对象分别是 `Car` 和 `Bike`，调用 `v1->move()`、`v2->move()` 时会根据运行时对象类型执行不同版本的 `move()`。这种“同一个接口、不同对象表现出不同行为”的机制就是多态，所以选 C。严格来说，这段完整代码若要通过基类指针安全删除派生类对象，还应给基类添加虚析构函数；但本题横线考查的是 `move()` 的动态绑定。



## 第4题

栈的操作特点是（ ）。

A. 先进先出
B. 先进后出
C. 随机访问
D. 双端进出

答案：B

解析：栈只允许在栈顶进行主要的插入和删除操作。先入栈的元素会被压在下面，后入栈的元素位于栈顶，因此出栈时后进入的元素先出来，也就是“后进先出”。“先进后出”和“后进先出”是同一含义，所以选 B；“先进先出”是队列的特点。



## 第5题

循环队列常用于实现数据缓冲。假设一个循环队列容量为 $ 5 $（即最多存放 $ 4 $ 个元素，留一个位置区分空与满），依次进行操作：入队数据 $ 1 $、$ 2 $、$ 3 $，出队 $ 1 $ 个数据，再入队数据 $ 4 $ 和 $ 5 $，此时队首到队尾的元素顺序是（ ）。

A. `[2, 3, 4, 5]`
B. `[1, 2, 3, 4]`
C. `[3, 4, 5, 2]`
D. `[2, 3, 5, 4]`

答案：A

解析：循环队列的物理下标可能会绕回数组开头，但从队首到队尾的逻辑顺序仍按入队、出队规则判断。先依次入队 `1, 2, 3`，队列顺序为 `[1, 2, 3]`；出队 1 个数据后，队首元素 `1` 被删除，剩下 `[2, 3]`；再入队 `4` 和 `5`，它们依次排在队尾，得到 `[2, 3, 4, 5]`。容量为 5 且留一个空位时最多存 4 个元素，正好能容纳这 4 个数据，所以选 A。



## 第6题

以下函数 `createTree()` 构造的树是什么类型？

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

TreeNode* createTree() {
    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);
    return root;
}
```

A. 满二叉树
B. 完全二叉树
C. 二叉排序树
D. 其他都不对

答案：B

解析：代码构造出的树为：根节点 `1`，第二层有 `2` 和 `3`，第三层有 `2` 的两个孩子 `4`、`5`。完全二叉树要求除最后一层外，其余层都填满，最后一层节点从左到右连续排列；这棵树的第三层节点正好占据最左边的两个位置，因此是完全二叉树。它不是通常意义上的满二叉树，因为第三层没有填满；也不是二叉排序树，因为根节点 `1` 的左孩子 `2` 大于根节点，不满足左子树小于根的规则，所以选 B。



## 第7题

已知二叉树的中序遍历是 `[D, B, E, A, F, C]`，先序遍历是 `[A, B, D, E, C, F]`。请问该二叉树的后序遍历结果是（ ）。

A. `[D, E, B, F, C, A]`
B. `[D, B, E, F, C, A]`
C. `[D, E, B, C, F, A]`
D. `[B, D, E, F, C, A]`

答案：A

解析：先序遍历的第一个节点是根，所以根为 `A`。在中序序列 `[D, B, E, A, F, C]` 中，`A` 左边 `[D, B, E]` 是左子树，右边 `[F, C]` 是右子树。左子树的先序为 `[B, D, E]`，根是 `B`，中序中 `B` 左边是 `D`、右边是 `E`，所以左子树后序为 `[D, E, B]`。右子树的先序为 `[C, F]`，根是 `C`，中序 `[F, C]` 表明 `F` 是 `C` 的左孩子，所以右子树后序为 `[F, C]`。整棵树后序按“左、右、根”得到 `[D, E, B, F, C, A]`，选 A。



## 第8题

完全二叉树可以用数组连续高效存储，如果节点从 $ 1 $ 开始编号，则对有两个孩子节点的节点 $ i $，（ ）。

A. 左孩子位于 $ 2i $，右孩子位于 $ 2i+1 $
B. 完全二叉树的叶子节点可以出现在最后一层的任意位置
C. 所有节点都有两个孩子
D. 左孩子位于 $ 2i+1 $，右孩子位于 $ 2i+2 $

答案：A

解析：完全二叉树按层序存入数组且编号从 `1` 开始时，根节点编号为 `1`，每向下一层，左孩子和右孩子分别对应父节点编号的两倍及两倍加一。因此编号为 `i` 的节点若有两个孩子，左孩子在 `2i`，右孩子在 `2i+1`。D 项是从 `0` 开始编号时常见的孩子下标形式；B 项错在最后一层节点必须从左到右连续出现，不能任意位置出现；C 项描述的是每个节点都有两个孩子，也不是完全二叉树的要求，所以选 A。



## 第9题

设有字符集 `{a, b, c, d, e, f}`，其出现频率分别为 `{5, 9, 12, 13, 16, 45}`。哈夫曼算法构造最优前缀编码，以下哪一组可能是对应的哈夫曼编码？（非叶子节点左边分支记作 0，右边分支记作 1，左右互换不影响正确性）。

A. a:00; b:01; c:10; d:110; e:111; f:0
B. a:1100; b:1101; c:100; d:101; e:111; f:0
C. a:000; b:001; c:01; d:10; e:110; f:111
D. a:10; b:01; c:100; d:101; e:111; f:0

答案：B

解析：哈夫曼算法每次合并频率最小的两个节点：先合并 `5` 和 `9` 得到 `14`，再合并 `12` 和 `13` 得到 `25`，接着合并 `14` 和 `16` 得到 `30`，再合并 `25` 和 `30` 得到 `55`，最后与 `45` 合并成根。因此频率最大的 `f=45` 应该在离根最近的位置，可得到长度为 1 的编码；`a=5`、`b=9` 最早被合并，编码长度应最长，且只在最后一位不同。B 项中 `f:0`，`a:1100`、`b:1101`，`c:100`、`d:101`、`e:111` 的长度关系和合并结构一致，也没有前缀冲突。A、D 中存在某个编码是另一个编码前缀的情况，C 中频率最大的 `f` 没有最短编码，都不符合哈夫曼编码特征，所以选 B。

下面代码生成格雷编码。则横线上应填写（ ）。

    if (n == 0) return {"0"};
    if (n == 1) return {"0", "1"};

    vector<string> prev = grayCode(n-1);
    vector<string> result;
    for (string s : prev) {
        result.push_back("0" + s);

A. `int i = 0; i < prev.size(); i++`

C. `auto s : prev`

答案：B

## 第12题

令 $ n $ 是树的节点数目，下列代码实现了树的广度优先遍历，其时间复杂度是（ ）。

```cpp
void bfs(TreeNode* root) {
    if (!root) return;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        TreeNode* node = q.front();
        q.pop();
        cout << node->val << " ";
        if (node->left) q.push(node->left);
        if (node->right) q.push(node->right);
    }
}
```

A. $ O(n) $
B. $ O(\log n) $
C. $ O(n^2) $
D. $ O(2^n) $

答案：A

解析：广度优先遍历中，每个节点最多入队一次、出队一次，访问节点值和检查左右孩子都是常数时间操作。整棵树共有 `n` 个节点，因此总操作次数与 `n` 成正比，时间复杂度为 $O(n)$。队列中同时保存的节点数会随树形变化，但不影响每个节点只被处理一次这一时间复杂度结论，所以选 A。

        root->left = deleteNode(root->left, key);

    else if (key > root->val) {
        root->right = deleteNode(root->right, key);
    }
    else {
        if (!root->left) return root->right;
        if (!root->right) return root->left;
        TreeNode* temp = ___________;  // 在此处填写代码

A. `root->left`

C. `findMin(root->right)`

答案：C

解析：删除二叉排序树中有两个孩子的节点时，常见做法是用“中序后继”或“中序前驱”替换它。当前代码后面执行的是 `root->right = deleteNode(root->right, temp->val);`，说明替换值来自右子树，并且替换后要到右子树中删除那个节点。右子树中最小的节点正好是当前节点的中序后继，既大于左子树所有节点，又不大于右子树其他节点，替换后仍能保持 BST 性质。因此横线应填 `findMin(root->right)`，选 C。若选 `findMax(root->left)`，后续就应该从左子树删除该前驱节点，而不是从右子树删除。



## 第15题

给定 $ n $ 个物品和一个最大承重为 $ W $ 的背包，每个物品有一个重量 $ wt[i] $ 和价值 $ val[i] $，每个物品只能选择放或不放。目标是选择若干个物品放入背包，使得总价值最大，且总重量不超过 $ W $，则横线上应填写（ ）。

```cpp
int knapsack(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<int> dp(W+1, 0);
    for (int i = 0; i < n; ++i) {
        for (int w = W; w >= wt[i]; --w) {
            _______________________  // 在此处填写代码
        }
    }
    return dp[W];
}
```

A. `dp[w] = max(dp[w], dp[w] + val[i]);`
B. `dp[w] = dp[w - wt[i]] + val[i];`
C. `dp[w] = max(dp[w - 1], dp[w - wt[i]] + val[i]);`
D. `dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);`

答案：D

解析：这是一维数组优化的 0/1 背包。令 `dp[w]` 表示当前已处理物品中，在容量不超过 `w` 时能得到的最大价值。处理第 `i` 个物品时，对每个容量 `w` 有两种选择：不放该物品，价值保持 `dp[w]`；放该物品，则需要先留出 `wt[i]` 的容量，价值为 `dp[w - wt[i]] + val[i]`。取二者最大值就是 `dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);`。内层循环从 `W` 递减到 `wt[i]`，是为了避免同一个物品在本轮被重复使用，所以选 D。
