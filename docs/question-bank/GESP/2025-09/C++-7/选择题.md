## 第1题

已知小写字母 `b` 的 ASCII 码为 $ 98 $，下列 C++ 代码的输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;
int main() {
    char a = 'b' + 1;
    cout << a;
    return 0;
}
```

A. `b`
B. `c`
C. `98`
D. `99`

答案：B

解析：字符常量 `'b'` 参与加法时会按其 ASCII 码参与运算。已知 `'b'` 的 ASCII 码是 $98$，所以 `'b' + 1` 得到 $99$，再赋给 `char a` 时表示 ASCII 码为 $99$ 的字符，即小写字母 `c`。`cout << a` 输出的是字符本身，不是整数编码，因此答案是 B。



## 第2题

已知 `a` 为 `int` 类型变量，`p` 为 `int *` 类型变量，下列表达式不符合语法的是（ ）。

A. `a * a`
B. `p * p`
C. `a && a`
D. `p && p`

答案：B

解析：`a * a` 是两个 `int` 相乘，语法正确；`a && a` 是两个整数参与逻辑与，整数会转换为布尔含义，语法正确；`p && p` 是两个指针参与逻辑与，指针也可以按“是否为空指针”转换为布尔含义，语法正确。`*` 乘法运算符不能直接作用于两个 `int *` 指针，两个指针之间没有“相乘”的语义，因此 `p * p` 不符合语法，答案是 B。



## 第3题

下列关于 C++ 类的说法，错误的是（ ）。

A. 如果一个类包含纯虚函数，则它不能包含成员变量。
B. 如果一个类包含纯虚函数，则不能用它定义对象。
C. 派生类对象占用的内存总是不小于基类对象。
D. 派生类可以不实现基类的虚函数。

答案：A

解析：包含纯虚函数的类称为抽象类，它的限制是不能直接定义该类对象，但仍然可以有普通成员函数、构造函数和成员变量，所以 A 的说法错误。B 正是抽象类的基本限制，成立。C 中派生类对象至少要包含基类子对象，通常还会增加自己的成员或虚函数相关信息，因此占用内存不会小于基类对象。D 中派生类可以不实现基类的虚函数；如果未实现纯虚函数，它自己仍是抽象类，但语法上允许这样定义派生类。因此答案是 A。



## 第4题

已知数组 `a` 的定义 `int a[10] = {-1};`，下列说法不正确的是（ ）。

A. 数组 `a` 至少占用 $ 10 $ 个 `int` 大小的内存，一般为 $ 40 $ 个字节。
B. 数组 `a` 的所有元素均被初始化为 $ -1 $。
C. 语句 `a[-1] = 0;` 不会产生编译错误，但会导致难以预测的运行结果。
D. 语句 `a[13] = 0;` 不会产生编译错误，但会导致难以预测的运行结果。

答案：B

解析：`int a[10] = {-1};` 是列表初始化，只显式给第一个元素赋值为 $-1$，没有写出的其余元素会被值初始化为 $0$，所以并不是所有元素都是 $-1$，B 错误。数组有 $10$ 个 `int` 元素，常见环境下 `int` 为 $4$ 字节，所以通常占 $40$ 字节。`a[-1]` 和 `a[13]` 都属于越界访问，编译器一般不会仅因为下标表达式越界就报语法错误，但运行时行为未定义、结果不可预测。因此答案是 B。



## 第5题

一棵完全二叉树有 $ 165 $ 个结点，则叶结点有多少个？（ ）

A. 38
B. 82
C. 83
D. 84

答案：C

解析：在完全二叉树中，若结点按层序从 $1$ 到 $n$ 编号，则有孩子的结点编号最多到 $\lfloor n/2 \rfloor$，编号大于 $\lfloor n/2 \rfloor$ 的结点都是叶结点。这里 $n=165$，非叶结点数为 $\lfloor 165/2 \rfloor=82$，叶结点数为 $165-82=83$。因此答案是 C。



## 第6题

下列关于二叉树的说法，错误的是（ ）。

A. 二叉排序树的中序遍历顺序与元素排序的顺序是相同的。
B. 自平衡二叉查找树（AVL 树）是一种二叉排序树。
C. $ n $ 个元素的二叉排序树，其高一定为 $ \lfloor \log_2 n \rfloor $。
D. 任意的森林，都可以映射为一颗二叉树进行表达和存储。

答案：C

解析：二叉排序树只要求每个结点满足“左子树元素小于该结点、右子树元素大于该结点”的顺序关系，并不保证平衡。若按递增顺序插入，二叉排序树可能退化成一条链，高度达到 $n$ 级别，而不是一定为 $\lfloor \log_2 n \rfloor$。A 是二叉排序树中序遍历的性质，B 中 AVL 树是在二叉排序树基础上加入平衡条件，D 是森林与二叉树的常见孩子兄弟表示法。因此错误项是 C。



## 第7题

下列关于树和图的说法，错误的是（ ）。

A. 保留树的所有节点，并把树的每个节点指向其父节点，则可以将树转换为一个有向弱连通图。
B. 保留树的所有节点，并把树的每个节点指向其子节点，则可以将树转换为一个有向无环图。
C. 每个连通图都存在生成树。
D. 每个存在生成树的有向图，都一定是强连通的。

答案：D

解析：把树的边统一指向父节点或子节点后，忽略方向时仍然连通，所以可以得到有向弱连通图；若边都从父节点指向子节点，则不会形成有向环，所以是有向无环图。无向连通图一定可以通过删去多余边得到一棵生成树。D 错在“存在生成树”并不推出强连通，例如一个有向树从根指向所有子节点，能覆盖所有顶点，但子节点无法沿有向边回到根，因此不是强连通。答案是 D。



## 第8题

对一个包含 $ V $ 个顶点、$ E $ 条边的图，执行广度优先搜索，其最优时间复杂度是（ ）。

A. $ O(V + E) $
B. $ O(V) $
C. $ O(E) $
D. $ O(V^2) $

答案：A

解析：使用邻接表实现 BFS 时，每个顶点最多入队、出队一次，贡献 $O(V)$；遍历邻接表时，每条边最多被检查常数次，贡献 $O(E)$。两部分相加得到 $O(V+E)$。只写 $O(V)$ 会漏掉扫描边的代价，只写 $O(E)$ 会漏掉顶点初始化和入队出队的代价，因此答案是 A。



## 第9题

以下哪个方案不能合理解决或缓解哈希表冲突（ ）。

A. 用新元素覆盖发生冲突的哈希表项。
B. 在每个哈希表项处，使用单链表管理该表项的冲突元素。
C. 建立额外的单链表，用来管理所有发生冲突的元素。
D. 使用不同的哈希函数再建立一个哈希表，用来管理所有发生冲突的元素。

答案：A

解析：哈希冲突的核心问题是多个元素映射到同一个表项时仍要保留并能查找到它们。用新元素直接覆盖旧表项会丢失原来的元素，既不能正确查找，也不能算合理的冲突处理。B 是链地址法；C 可以把冲突元素额外组织起来；D 类似再哈希或建立辅助结构，虽然实现方式不同，但都没有直接丢弃已有元素。因此不能合理解决或缓解冲突的是 A。



## 第10题

以下关于贪心法和动态规划的说法中，错误的是（ ）。

A. 对特定的问题，贪心法不一定适用。
B. 当特定的问题适用贪心法时，通常比动态规划的时间复杂度更低。
C. 对很多问题，递推实现和递归实现动态规划方法的时间复杂度相当。
D. 采用动态规划的算法一定具有多项式时间复杂度。

答案：D

解析：贪心法需要问题具有贪心选择性质，并不是所有问题都适用，所以 A 正确。若某个问题确实能用贪心解决，通常只需局部选择和排序等步骤，常比枚举大量状态的动态规划更简单、更快，B 作为一般性说法成立。动态规划既可以自底向上递推，也可以自顶向下递归加记忆化，状态相同时时间复杂度通常相当，C 正确。D 过于绝对：动态规划的复杂度取决于状态数量和转移代价，状态空间可能不是输入规模的多项式，因此不能说一定是多项式时间。答案是 D。



## 第11题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;
int fib(int n) {
    if (n == 0)
        return 1;
    return fib(n - 1) + fib(n - 2);
}
int main() {
    cout << fib(6) << endl;
    return 0;
}
```

A. 8
B. 13
C. 21
D. 无法正常结束。

答案：D

解析：这个递归函数只有 `n == 0` 一个终止条件。计算 `fib(6)` 会不断调用到 `fib(1)`，而 `fib(1)` 又会调用 `fib(0) + fib(-1)`。其中 `fib(-1)` 会继续调用 `fib(-2) + fib(-3)`，参数越来越小，永远不会等于 $0$，递归无法正常终止，最终通常导致栈溢出。因此程序无法正常结束，答案是 D。



## 第12题

下面程序的时间复杂度为（ ）。

```cpp
int rec_fib[MAX_N];
int fib(int n) {
    if (n <= 1)
        return n;
    if (rec_fib[n] != 0)
        return rec_fib[n];
    return fib(n - 1) + fib(n - 2);
}
```

A. $ O(\phi^n), \phi = \frac{\sqrt{5}+1}{2} $
B. $ O(2^n) $
C. $ O(n^2) $
D. $ O(n) $

答案：A

解析：虽然代码里有 `rec_fib` 数组和查询语句，但函数从未把 `fib(n)` 的结果写回 `rec_fib[n]`。如果 `rec_fib` 初始为 $0$，那么对 $n>1$ 的调用不会命中记忆化，仍会递归计算 `fib(n - 1)` 和 `fib(n - 2)`。调用次数满足类似 $T(n)=T(n-1)+T(n-2)+O(1)$ 的递推式，增长量级为 $O(\phi^n)$，其中 $\phi=\frac{\sqrt{5}+1}{2}$。因此答案是 A。



## 第13题

下面 `init_sieve` 函数的时间复杂度为（ ）。

```cpp
int sieve[MAX_N];
void init_sieve(int n) {
    for (int i = 1; i <= n; i++)
        sieve[i] = i;
    for (int i = 2; i <= n; i++)
        for (int j = i; j <= n; j += i)
            sieve[j]--;
}
```

A. $ O(n) $
B. $ O(n \log \log n) $
C. $ O(n \log n) $
D. $ O(n^2) $

答案：C

解析：第一个循环执行 $n$ 次，复杂度为 $O(n)$。第二部分中，固定 $i$ 时，内层变量 `j` 取 $i,2i,3i,\ldots$，执行次数约为 $\lfloor n/i \rfloor$。总次数为 $\sum_{i=2}^{n}\lfloor n/i \rfloor$，约等于 $n\sum_{i=2}^{n}1/i=O(n\log n)$。与前面的 $O(n)$ 合并后仍为 $O(n\log n)$，答案是 C。



## 第14题

下面 `count_triple` 函数的时间复杂度为（ ）。

```cpp
int gcd(int m, int n) {
    if (m == 0) return n;
    return gcd(n % m, m);
}
int count_triple(int n) {
    int cnt = 0;
    for (int v = 1; v * v * 4 <= n; v++)
        for (int u = v + 1; u * (u + v) * 2 <= n; u += 2)
            if (gcd(u, v) == 1) {
                int a = u * u - v * v;
                int b = u * v * 2;
                int c = u * u + v * v;
                cnt += n / (a + b + c);
            }
    return cnt;
}
```

A. $ O(n^2) $
B. $ O(n^2 \log n) $
C. $ O(n \log n) $
D. $ O(n) $

答案：C

解析：外层循环条件 `v * v * 4 <= n` 给出 $v=O(\sqrt n)$。对每个固定的 $v$，内层条件 `u * (u + v) * 2 <= n` 也限制 $u=O(\sqrt n)$，所以两层循环的组合次数最多为 $O(n)$。循环体里调用欧几里得算法 `gcd(u, v)`，其时间复杂度为 $O(\log n)$，其余赋值和加法是常数时间。因此总体复杂度为 $O(n\log n)$，答案是 C。



## 第15题

下列选项中，哪个不可能是下图的深度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/2q8w8leg.png)

A. 2, 3, 5, 7, 8, 9, 6, 4, 1
B. 5, 7, 8, 9, 1, 2, 4, 3, 6
C. 6, 8, 9, 5, 7, 1, 2, 3, 4
D. 8, 5, 7, 9, 1, 2, 3, 6, 4

答案：B

解析：图中与本题相关的有向边包括 `1->2`、`2->3`、`1->4`、`4->7`、`1->5`、`3->5`、`3->6`、`6->8`、`6->9`、`8->5`、`5->7`、`7->8`、`8->9`。B 序列从 `5, 7, 8, 9` 开始是可能的；但随后访问到 `1` 后，若 DFS 选择 `1->2`，就必须从 `2` 继续访问尚未访问的 `3`，不能先回到 `1` 再访问 `4`。因此 B 中的片段 `1, 2, 4, 3` 违反了深度优先搜索“沿未访问邻点尽量深入”的规则，所以 B 不可能。A、C、D 都可以通过选择不同的起点和邻接点访问顺序得到。
