# 判断题

来源：

## 第1题

C++语言中，表达式 `9 && 12` 的结果类型为 int、值为 8。

答案：错误

解析：`&&` 是逻辑与运算符，不是按位与运算符。表达式 `9 && 12` 中，`9` 和 `12` 都是非零值，都会按逻辑含义转换为 `true`，所以逻辑与结果为 `true`，再以整型值表示时为 $1$。题干说值为 $8$，这是把它误当成按位与 `9 & 12` 的结果了，因此判断为错误。



## 第2题

C++语言中，在有 `int a[10];` 定义的范围内，通过表达式 `a[-1]` 进行访问将导致编译错误。

答案：错误

解析：`a[-1]` 在语法上等价于 `*(a + (-1))`，表达式形式本身能通过编译。问题在于它访问了数组首元素之前的位置，超出了 `int a[10]` 的合法下标范围 `0` 到 `9`，运行时行为未定义。题干说“将导致编译错误”不正确，因此判断为错误。



## 第3题

选择排序一般是不稳定的。

答案：正确

解析：稳定排序要求相等关键字元素在排序前后的相对顺序不变。普通选择排序每一轮会从未排序区间中选出最小元素并与当前位置交换，这个交换可能把后面的相等元素换到前面。例如序列 `2a, 2b, 1` 第一轮会把 `1` 与 `2a` 交换，得到 `1, 2b, 2a`，两个值为 `2` 的元素相对顺序被改变。因此选择排序一般是不稳定的，判断为正确。



## 第4题

C++语言中，`float` 和 `int` 类型一般都是 4 字节，因此 `float` 类型能够表达不同的浮点数值的数量，与 `int` 类型能够表达不同的整数值的数量是相同的。

答案：错误

解析：即使常见环境中 `float` 和 `int` 都占 $4$ 字节，也不能推出二者能表示的“不同数值数量”相同。`int` 通常用所有位组合表示整数；而 `float` 要把位划分为符号位、指数位和尾数位，还包含正负零、无穷大、NaN 等特殊编码。浮点数的编码方式和整数不同，能表示的有效数值集合也不同。题干仅凭字节数相同推出数量相同，结论不成立，因此判断为错误。



## 第5题

使用 `math.h` 或 `cmath` 头文件中的对数函数，表达式 `log(256)` 的结果类型为 double、值约为 8.0。

答案：错误

解析：`math.h` 或 `cmath` 中的 `log` 默认表示自然对数，即以 $e$ 为底的对数，返回类型为 `double`。`log(256)` 的值是 $\ln 256$，约为 $5.545$，不是以 $2$ 为底的 $\log_2 256=8$。题干的返回类型部分基本正确，但数值约为 $8.0$ 错误，因此判断为错误。



## 第6题

一棵有 $N$ 个节点的完全二叉树，则树的深度为 $\lfloor \log_2(N) \rfloor + 1$。

答案：正确

解析：完全二叉树按层从上到下、从左到右填充。深度为 $d$ 的二叉树最多有 $2^d-1$ 个结点，深度为 $d-1$ 的完全二叉树最多有 $2^{d-1}-1$ 个结点。若有 $N$ 个结点，则其深度正好是能容纳 $N$ 个结点的最小层数，即 $d=\lfloor \log_2 N \rfloor+1$。因此题干表述正确。



## 第7题

邻接表和邻接矩阵都是图的存储形式。通常，使用邻接表比使用邻接矩阵的时间复杂度更低。

答案：错误

解析：邻接表和邻接矩阵确实都是图的存储方式，但“通常使用邻接表比使用邻接矩阵的时间复杂度更低”这个说法过于笼统。复杂度取决于具体操作：遍历所有边时，邻接表常为 $O(V+E)$，稀疏图上优于邻接矩阵的 $O(V^2)$；但判断两点之间是否有边时，邻接矩阵可 $O(1)$ 查询，邻接表通常要扫描链表或集合。不能脱离操作直接比较“存储形式的时间复杂度”，因此判断为错误。



## 第8题

C++ 语言中，类的构造函数可以声明为私有（private）。

答案：正确

解析：构造函数也是类的成员函数，可以放在 `public`、`protected` 或 `private` 访问控制区中。私有构造函数常用于限制外部直接创建对象，例如单例模式、工厂函数控制创建过程等。它并不违反 C++ 语法，只是类外代码不能直接调用该私有构造函数。因此题干表述正确。



## 第9题

泛洪算法的递归实现容易造成溢出，因此大的二维地图算法中，一般使用广度优先搜索实现。

答案：正确

解析：泛洪算法在二维地图上常用于从一个格子扩展到相邻格子。若用递归 DFS 实现，在地图很大、连通区域很长时，递归深度可能接近格子数量，容易超过调用栈限制而溢出。BFS 使用显式队列保存待扩展格子，虽然也需要内存，但不依赖函数递归栈，更适合大地图的稳定实现。因此题干表述正确。



## 第10题

很多游戏中为玩家设置多种可供学习的技能，要学习特定技能又往往需要先学习 1 个或以上的前置技能。尽管这样的技能间依赖关系常被玩家称为“技能树”，但它并不一定是树，更可能是有向无环图。

答案：正确

解析：树结构要求除根外每个结点通常只有一个父结点，并且不存在环；而技能依赖中，一个技能可能同时需要多个前置技能，也可能被多个后续技能共同依赖。只要依赖关系不能形成“学习自己之前还要先学自己”的环，它更适合抽象成有向无环图：边表示“先学前置技能，再学后续技能”。所以“技能树”只是习惯叫法，不一定是真正的树，题干表述正确。
