## 第1题

在集成开发环境里调试程序时，要注意不能修改源程序，因为如果修改，就要终止调试、关闭该文件并重新打开，才能再次开始调试。（ ）

答案：错误

解析：在集成开发环境中调试时，通常可以继续编辑源程序；是否需要重新编译、重新启动调试，取决于语言、编译器和 IDE 的调试能力。题目把“修改源程序”说成必须终止调试、关闭文件并重新打开，限制过于绝对，因此判断为错误。



## 第2题

在 C++ 代码中，假设 $N$ 为正整数且大于 $100$，则 $N / 100$ 将舍弃个位和十位，如 $N$ 为 $1234$ 则 `cout << (N / 100)` 将输出 $12$。如果 $N$ 小于 $100$，则其值为 $0$。（ ）

答案：正确

解析：`N` 是整数时，`N / 100` 执行整数除法，小数部分会被舍弃，相当于去掉十进制的个位和十位。$1234 / 100 = 12$，而小于 $100$ 的正整数除以 $100$ 的整数商为 $0$，所以判断为正确。



## 第3题

下列 C++ 代码执行后将输出 $1$，因为 $a$ 确实小于 $20$ 和 $10$。（ ）

```cpp
a = 5;
cout << (a < 10 and 20);
```

答案：错误

解析：`and` 在 C++ 中等价于 `&&`，表达式 `a < 10 and 20` 实际是 `(a < 10) && 20`。当 `a = 5` 时，输出结果确实为 $1$，但原因不是“`a` 同时小于 $20$ 和 $10$”，因为代码里并没有写 `a < 20`，而是把非零整数 `20` 当作真值参与逻辑与。题目给出的理由错误，因此判断为错误。



## 第4题

下面的 C++ 代码中变量都是整型，则执行后将输出 $1$。（ ）

```cpp
x, y, z = 5, 10, 15;
result = x < y < z;
cout << result;
```

答案：正确

解析：C++ 不会把 `x < y < z` 当作数学中的连写不等式，而是从左到右计算为 `(x < y) < z`。第一次比较的结果是 $0$ 或 $1$，再与正整数 `z` 比较时，$0 < 15$ 和 $1 < 15$ 都为真，因此 `result` 为 $1$，输出 $1$。



## 第5题

下面 C++ 代码执行时如输入 $99.99$，将输出 及格 两个汉字。（ ）

```cpp
int score;
cout << "请输入学生成绩：";
cin >> score;
if (score >= 60)
    printf("及格");
else
    printf("不及格");
```

答案：正确

解析：`score` 是 `int`，用 `cin >> score` 读取 `99.99` 时，会先成功读取整数部分 $99$，小数点及后面的内容留在输入缓冲区。此时 `score >= 60` 为真，程序执行 `printf("及格");`，所以判断为正确。这里不是四舍五入成 $100$。



## 第6题

执行下面的 C++ 代码时输入 $123$，则输出是 DCB。（ ）

```cpp
int a;
cin >> a;
while(a){
    cout << 'A' + a%10;
    a /= 10;
}
```

答案：错误

解析：循环会按个位、十位、百位的顺序处理 $123$，但 `cout << 'A' + a % 10` 中字符和整数相加后的结果类型是整数，不是字符。第一次输出的是 `'A' + 3` 的编码值 $68$，后两次分别是 $67$、$66$，连起来是 `686766`，不是 `DCB`，因此判断为错误。



## 第7题

下面的 C++ 代码执行后将输出 `+#+#3#`。（ ）

```cpp
int i;
for (i = 0; i < 3; i++){
    if (i == 2)
        continue;
    printf("+#");
}
cout << i << '#';
```

答案：正确

解析：`i = 0` 和 `i = 1` 时不会触发 `continue`，各输出一次 `+#`；`i = 2` 时跳过 `printf("+#");`。随后 `for` 循环自增到 `i = 3` 后结束，最后输出 `3#`，合并结果为 `+#+#3#`，所以判断为正确。



## 第8题

下列 C++ 代码用于求斐波那契数列，即第 $1$ 个数 $0$、第 $2$ 个数 $1$，从第三个数开始，是前两个数之和。如果输入的值为大于 $1$ 的正整数，该代码能实现。（ ）

```cpp
cin >> n;
a = 0, b = 1;
for (int j = 0; j < n; j++){
    cout << a << " ";
    b = b + a;
    a = b - a;
}
```

答案：正确

解析：初始 `a = 0, b = 1`，每轮先输出当前 `a`，再用 `b = b + a; a = b - a;` 把 `(a, b)` 更新为原来的 `(b, a + b)`。因此输出序列依次为 $0, 1, 1, 2, 3, ...$，正是题目定义的斐波那契数列前若干项；输入大于 $1$ 时可以实现。



## 第9题

下面的 C++ 不能实现如下输出，但如果将 L1 标记的 `cout << 0` 行移动 if 块外面，或者说移动到 L2 标记行，则可以。（ ）

```plaintext
请输入矩阵大小n: 9
100000000
020000000
003000000
000400000
000050000
000006000
000000700
000000080
000000009
```

```cpp
int n, i, j;
cout << "请输入矩阵大小n: ";
cin >> n;

for (i = 0; i < n; i++){
    for (j = 0; j < n; j++){
        if (i == j){
            cout << i + 1;
            continue;
            cout << 0; // L1
        }
        // L2
    }
    printf("\n");
}
```

答案：正确

解析：按原卷代码位置，`cout << 0; // L1` 位于 `if (i == j)` 块内且在 `continue` 后面，执行到对角线位置时已经跳过本轮，`L1` 不会执行；非对角线位置又进不了这个 `if`，所以原代码不能输出矩阵中的 $0$。把这行移到 `if` 块外、仍放在内层 `j` 循环中时，非对角线位置会输出 $0$，对角线位置由 `continue` 跳过，从而得到示例矩阵，所以判断为正确。



## 第10题

C++ 代码 `cout << ('5' + 4);` 执行后的输出为 $9$。（ ）

答案：错误

解析：字符 `'5'` 的编码值是 $53$，表达式 `'5' + 4` 做的是整数加法，结果是整数 $57$。`cout` 输出这个整数时会打印 `57`，不会自动把它当作字符 `'9'` 输出；只有显式转成 `char` 时才会得到字符 `9`，因此判断为错误。
