## 第1题

C++ 语言中，表达式 `'9' ^ 3` 的结果值为 `'999'`。

答案：错误

解析：题干表述错误。C++ 中 `^` 是按位异或运算符，不是重复字符或乘方运算。字符常量 `'9'` 的 ASCII 码是 $57$，表达式 `'9' ^ 3` 会先把 `'9'` 提升为整数，再计算 $57 \mathbin{\mathrm{xor}} 3$。因为 $57$ 的二进制低位为 `1001`，与 `0011` 异或后得到低位 `1010`，整体结果为整数 $58$，对应字符 `':'`，不是多字符常量 `'999'`。所以判断为错误。



## 第2题

下列 C++ 语言代码，能够安全地输出 `arr[5]` 的值。

```cpp
int n = 5;
int arr[n] = {1, 2, 3};
std::cout << arr[5];
```

答案：错误

解析：题干表述错误。首先，标准 C++ 中普通数组长度必须是编译期常量，`int n = 5; int arr[n]` 属于变长数组写法，不是标准 C++ 语法。即使在某些编译器扩展下能够通过，`arr` 的有效下标也只有 `0` 到 `4`，访问 `arr[5]` 已经越界，行为未定义，不能说“安全地输出”。所以判断为错误。



## 第3题

对 $n$ 个元素的数组进行排序，最差情况的时间复杂度为 $O(n^2)$。

答案：错误

解析：题干表述错误。“对 $n$ 个元素的数组进行排序”没有指定具体算法，不能统一说最差情况一定是 $O(n^2)$。例如归并排序、堆排序的最差时间复杂度都是 $O(n\log n)$；冒泡排序、插入排序、选择排序的最差情况通常是 $O(n^2)$；快速排序在朴素选主元时最差可达 $O(n^2)$。由于不同排序算法的最差复杂度不同，题干把所有排序一概归为 $O(n^2)$ 是错误的。



## 第4题

有 4 个红球、3 个蓝球和 2 个绿球排成一排（相同色球视为完全相同），则不同的排列方案数为 1260 种。

答案：正确

解析：题干表述正确。共有 $4+3+2=9$ 个球排成一排，但同色球完全相同，所以不能按 $9!$ 直接计数。先把 $9$ 个位置全部排列，再除去红球内部 $4!$ 种交换、蓝球内部 $3!$ 种交换、绿球内部 $2!$ 种交换带来的重复，方案数为 $\frac{9!}{4!3!2!}=1260$。所以判断为正确。



## 第5题

使用 `math.h` 或 `cmath` 头文件中的函数，对于 `int` 类型的变量 $x$，表达式 `fabs(x)` 和 `sqrt(x * x)` 的结果总是近似相等的。

答案：错误

解析：题干表述错误。`fabs(x)` 会把 `int` 类型的 $x$ 转为浮点数后求绝对值，数学含义接近 $|x|$。`sqrt(x * x)` 先在 `int` 类型中计算 `x * x`，如果 $x$ 的绝对值较大，乘法可能发生整数溢出，之后再传给 `sqrt` 就已经不是原来的 $x^2$。另外当溢出结果为负数时，`sqrt` 还可能得到非法结果。因此不能说二者“总是近似相等”，判断为错误。



## 第6题

运算符重载是 C++ 语言静态多态的一种典型体现，而使用 C 语言则无法实现运算符重载。

答案：正确

解析：题干表述正确。静态多态指在编译期确定调用目标的多态形式，函数重载、函数模板和运算符重载都属于典型例子。C++ 允许为自定义类型重载很多运算符，使同一个运算符符号在不同参数类型下对应不同函数。C 语言没有类和 `operator` 重载机制，不能像 C++ 这样定义运算符重载。因此判断为正确。



## 第7题

存在一个简单无向图满足：顶点数为 6，边数为 8，6 个顶点的度数分别为 3、3、3、3、2、2。

答案：正确

解析：题干表述正确。度数和为 $3+3+3+3+2+2=16$，等于 $2\times8$，满足握手定理给出的必要条件。还可以构造一个例子：令顶点为 $1,2,3,4,5,6$，取边 $(1,2),(1,3),(1,5),(2,3),(2,6),(3,4),(4,5),(4,6)$。此时 $1,2,3,4$ 的度数都是 $3$，$5,6$ 的度数都是 $2$，边数正好为 $8$，且没有重边和自环，所以这样的简单无向图确实存在。



## 第8题

已知两个 `double` 类型的变量 $r$ 和 $theta$ 分别表示一个扇形的圆半径及圆心角（弧度），则扇形的周长可以通过表达式 `(2 + theta) * r` 求得。

答案：正确

解析：题干表述正确。扇形周长由两条半径和一段弧长组成。圆心角用弧度表示时，弧长公式为 $r\theta$，两条半径长度为 $2r$，所以周长为 $2r+r\theta=(2+\theta)r$。表达式 `(2 + theta) * r` 与这个公式一致，且 `2` 会在计算中转换为 `double`，所以判断为正确。



## 第9题

Dijkstra 算法的时间复杂度为 $O(V^2)$，其中 $V$ 为图中顶点的数量。

答案：错误

解析：题干表述错误。Dijkstra 算法的时间复杂度取决于图的存储方式和用于取最小距离顶点的数据结构。若使用邻接矩阵并每次线性扫描找最小顶点，复杂度可以是 $O(V^2)$；若使用邻接表配合堆优化，常见复杂度为 $O((V+E)\log V)$ 或类似形式，其中还与边数 $E$ 有关。因此不能不加条件地说 Dijkstra 算法的时间复杂度就是 $O(V^2)$，判断为错误。



## 第10题

从 32 名学生中选出 2 人分别担任男生班长和女生班长（男生班长必须是男生，女生班长必须是女生），则共有 $C(32, 2) / 2$ 种不同的选法。

答案：错误

解析：题干表述错误。题目要求分别选出男生班长和女生班长，并且男生班长必须从男生中选、女生班长必须从女生中选。若男生有 $a$ 人、女生有 $32-a$ 人，则选法数应为 $a(32-a)$，这个值取决于男女生人数。表达式 $C(32,2)/2$ 只是在 $32$ 人中不区分性别地选两人后再除以 $2$，既没有保证一男一女，也没有区分两个岗位的性别限制。因此判断为错误。
