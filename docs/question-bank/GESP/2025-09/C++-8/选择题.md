## 第1题

小杨想点一杯奶茶外卖，但还差 $5$ 元起送。于是，小杨决定点一些小料。可选的小料包括：珍珠 $1$ 元、椰果 $2$ 元、奶冻 $3$ 元、奶盖 $4$ 元。每种小料最多点 $1$ 份。请问共有多少种满足起送条件的点小料方案？（ ）。

A. $16$
B. $10$
C. $9$
D. $7$

答案：C

解析：每种小料最多点 $1$ 份，四种小料分别记为价格 $1,2,3,4$。所有选择方案共有 $2^4=16$ 种，其中需要总价至少为 $5$ 元。反过来数总价小于 $5$ 的方案：不点为 $0$ 元；只点一种有 $1,2,3,4$ 共 $4$ 种；点两种且小于 $5$ 的有 $1+2=3$、$1+3=4$ 共 $2$ 种，一共 $7$ 种。不满足起送的方案有 $7$ 种，所以满足条件的方案数为 $16-7=9$，选 C。



## 第2题

小杨和小刘是好朋友，她们在逛商场时发现新设置的大头贴自拍机，于是决定一起拍一组照片。一组照片包括 $4$ 张，这 $4$ 张照片没有顺序区分。拍每张照片时，可以选择有相框或无相框、两人可以分别选择有头饰或无头饰、还可以从 $2$ 种位置（小杨在左，或小刘在左）中选出一种。她们不希望一组照片中出现完全相同的相框、头饰、位置的组合。请问一组照片共有多少种不同的方案？（ ）。

A. $1820$
B. $70$
C. $24$
D. $16$

答案：A

解析：先计算单张照片可以形成多少种不同组合。相框有 $2$ 种；小杨是否有头饰 $2$ 种，小刘是否有头饰 $2$ 种；左右位置有 $2$ 种。因此单张照片共有 $2\times2\times2\times2=16$ 种组合。一组照片要从这 $16$ 种中选出 $4$ 种，且 $4$ 张没有顺序区分、不能完全相同，所以是组合数 $\binom{16}{4}=\frac{16\times15\times14\times13}{4\times3\times2\times1}=1820$，选 A。



## 第3题

下列关于 C++ 类的说法，错误的是（ ）。

A. 派生类对象占用的内存总是不小于基类对象。
B. 派生类可以不实现基类的虚函数。
C. 如果一个类包含纯虚函数，则它不能包含成员变量。
D. 如果一个类包含纯虚函数，则不能用它定义对象。

答案：C

解析：含有纯虚函数的类是抽象类，抽象类不能直接定义对象，但它仍然可以拥有普通成员变量、普通成员函数、构造函数等。因此 C 中“不能包含成员变量”是错误说法。A 中派生类对象通常包含基类子对象，所以占用空间不会小于只含基类子对象本身的大小；B 中如果派生类不实现基类纯虚函数，它仍然是抽象类，这种情况允许存在；D 是抽象类的基本性质。因此选 C。



## 第4题

下列关于树和图的说法，错误的是（ ）。

A. 每个连通图都存在生成树。
B. 每个存在生成树的有向图，都一定是强连通的。
C. 保留树的所有节点，并把树的每个节点指向其父节点，则可以将树转换为一个有向弱连通图。
D. 保留树的所有节点，并把树的每个节点指向其子节点，则可以将树转换为一个有向无环图。

答案：B

解析：连通无向图一定可以删去多余边，保留所有顶点并得到一棵生成树，所以 A 正确。对有向图来说，“存在生成树”并不推出任意两个顶点都能互相到达；例如一棵从根指向子节点的有向树，只能从根到达后代，后代不能反向到达根，因此不是强连通，B 错误。C 把每个节点指向父节点后，若忽略方向仍然是原树，所以是有向弱连通图。D 把节点指向子节点，边总是从父到子，不可能沿有向边回到祖先，因此是有向无环图。选 B。



## 第5题

一对夫妻生男生女的概率相同。这对夫妻希望儿女双全。请问这对夫妻生下三个孩子时，实现儿女双全的概率是多少？（ ）。

A. $ \frac{1}{4} $
B. $ \frac{1}{2} $
C. $ \frac{3}{4} $
D. $ \frac{7}{8} $

答案：C

解析：三个孩子每个孩子男、女概率相同，共有 $2^3=8$ 种等可能性别序列。没有实现儿女双全的情况只有两类：全男或全女，共 $2$ 种。因此实现儿女双全的情况有 $8-2=6$ 种，概率为 $\frac{6}{8}=\frac{3}{4}$，选 C。



## 第6题

二项式 $ (x + y)^6 $ 的展开式中 $ x^2y^4 $ 项的系数是（ ）。

A. $720$
B. $120$
C. $20$
D. $15$

答案：D

解析：在 $(x+y)^6$ 中，每一项来自 $6$ 个括号各选一个字母。要得到 $x^2y^4$，需要从 $6$ 个括号中选 $2$ 个贡献 $x$，其余 $4$ 个贡献 $y$。系数就是组合数 $\binom{6}{2}=\frac{6\times5}{2}=15$，选 D。



## 第7题

对一个包含 $V$ 个顶点、$E$ 条边的图，执行广度优先搜索，其最优时间复杂度是（ ）。

A. $ O(V) $
B. $ O(V + E) $
C. $ O(V^2) $
D. $ O(E) $

答案：B

解析：广度优先搜索使用队列按层访问顶点。若用邻接表存图，每个顶点最多入队、出队一次，对应 $O(V)$；每条边在扫描邻接表时最多被检查常数次，对应 $O(E)$。两部分相加得到 $O(V+E)$。题目问最优时间复杂度，应选邻接表下的 $O(V+E)$，选 B。



## 第8题

以下关于贪心法和动态规划的说法中，错误的是（ ）。

A. 动态规划能解决大部分多阶段决策问题。
B. 对特定的问题，贪心法不一定适用。
C. 当特定的问题适用贪心法时，通常比动态规划的时间复杂度更低。
D. 对很多问题，递推实现和递归实现动态规划方法的时间复杂度相当。

答案：A

解析：动态规划适用于具有最优子结构、重叠子问题，并且能合理定义状态和转移的多阶段决策问题，不能笼统说它“能解决大部分多阶段决策问题”，A 表述过强，所以错误。B 正确，贪心需要局部最优能推出全局最优，很多问题不满足这个性质。C 通常成立，能用贪心解决的问题往往只需排序或一次扫描，比枚举状态的动态规划更低。D 也成立，同一 DP 递推若用记忆化递归或自底向上递推，每个状态和转移次数相同，时间复杂度通常相当。选 A。



## 第9题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;
int main() {
    int N = 15, cnt = 0;
    for (int x = 1; x + x + x <= N; x++)
        for (int y = x; x + y + y <= N; y++)
            for (int z = y; x + y + z <= N; z++)
                cnt++;
    cout << cnt << endl;
    return 0;
}
```

A. $45$
B. $102$
C. $174$
D. $3375$

答案：B

解析：三重循环枚举满足 $1\le x\le y\le z$ 且 $x+y+z\le15$ 的整数三元组。固定 $x,y$ 后，$z$ 从 $y$ 枚举到 $15-x-y$，个数为 $(15-x-y)-y+1=16-x-2y$。按 $x$ 分类：$x=1$ 时 $y=1\sim7$，个数为 $13+11+9+7+5+3+1=49$；$x=2$ 时 $y=2\sim6$，个数为 $10+8+6+4+2=30$；$x=3$ 时 $y=3\sim6$，个数为 $7+5+3+1=16$；$x=4$ 时 $y=4\sim5$，个数为 $4+2=6$；$x=5$ 时 $y=5$，个数为 $1$。总数为 $49+30+16+6+1=102$，程序输出 `102`，选 B。



## 第10题

下面程序的时间复杂度为（ ）。

```cpp
int primes[MAXP], num = 0;
bool isPrime[MAXN] = {false};
void sieve() {
    for (int n = 2; n <= MAXN; n++) {
        if (!isPrime[n])
            primes[num++] = n;
        for (int i = 0; i < num && n * primes[i] <= MAXN; i++) {
            isPrime[n * primes[i]] = true;
            if (n % primes[i] == 0)
                break;
        }
    }
}
```

A. $ O(n \log n) $
B. $ O(n \log \log n) $
C. $ O(n) $
D. $ O(\log n) $

答案：C

解析：这段代码是线性筛。外层枚举 $n$ 从 $2$ 到 `MAXN`，内层用已得到的质数标记 `n * primes[i]`，并在 `n % primes[i] == 0` 时立即 `break`。这个 `break` 保证每个合数只会被它的最小质因子对应的一次标记生成，不会像普通埃氏筛那样被多个因子反复标记。因此总标记次数是 $O(n)$，外层也是 $O(n)$，整体时间复杂度为 $O(n)$，选 C。若严格按 C++ 数组下标检查，`isPrime[MAXN]` 的长度与 `<= MAXN` 的访问边界还需要在实际代码中配套开到足够大。



## 第11题

下列 Dijkstra 算法，假设图 `graph` 中顶点数 $v$、边数 $e$，则程序的时间复杂度为（ ）。

```cpp
typedef struct Edge {
    int in, out;   // 从下标in顶点到下标out顶点的边
    int len;       // 边长度
    struct Edge * next;
} Edge;
// v: 顶点个数, graph: 出边邻接表, start: 起点下标, dis: 输出每个顶点的最短距离
void dijkstra(int v, Edge * graph[], int start, int * dis) {
    const int MAX_DIS = 0x7fffffff;
    for (int i = 0; i < v; i++)
        dis[i] = MAX_DIS;
    dis[start] = 0;
    int * visited = new int[v];
    for (int i = 0; i < v; i++)
        visited[i] = 0;
    visited[start] = 1;
    for (int t = 0; ; t++) {
        int min = MAX_DIS, minv = -1;
        for (int i = 0; i < v; i++) {
            if (visited[i] == 0 && min > dis[i]) {
                min = dis[i];
                minv = i;
            }
        }
        if (minv < 0)
            break;
        visited[minv] = 1;
        for (Edge * e = graph[minv]; e != NULL; e = e->next)
            if (dis[e->out] > e->len)
                dis[e->out] = e->len;
    }
    delete[] visited;
}
```

A. $ O(v^2) $
B. $ O(v \log v + e) $
C. $ O((v + e) \log v) $
D. $ O(v + e) $

答案：A

解析：本题选 A。这段程序考查的是朴素 Dijkstra 的复杂度，复杂度分析关注循环结构。外层最多进行 $v$ 轮；每一轮先用 `for (int i = 0; i < v; i++)` 线性扫描所有顶点，找出当前未访问且距离最小的顶点，因此找最小点的总时间是 $O(v^2)$。选中一个顶点后，再扫描它的邻接边，所有顶点的邻接表合计扫描量为 $O(e)$。所以总复杂度可写成 $O(v^2+e)$；在简单图下，$e$ 不超过平方级，选项中对应的是 $O(v^2)$，因此选 A。



## 第12题

下面 `count_triple` 函数的时间复杂度为（ ）。

```cpp
int gcd(int m, int n) {
    if (m == 0) return n;
    return gcd(n % m, m);
}
int count_triple(int n) {
    int cnt = 0;
    for (int v = 1; v * v * 4 <= n; v++)
        for (int u = v + 1; u * (u + v) * 2 <= n; u += 2)
            if (gcd(u, v) == 1) {
                int a = u * u - v * v;
                int b = u * v * 2;
                int c = u * u + v * v;
                cnt += n / (a + b + c);
            }
    return cnt;
}
```

A. $ O(n^2) $
B. $ O(n^2 \log n) $
C. $ O(n) $
D. $ O(n \log n) $

答案：D

解析：外层循环条件 $4v^2\le n$，所以 $v$ 最大约为 $\frac{\sqrt n}{2}$。内层条件 $2u(u+v)\le n$，可粗略看作 $u=O(\sqrt n)$，因此两重循环的枚举规模至多为 $O(n)$。每次满足条件时调用一次欧几里得算法 `gcd(u, v)`，其时间复杂度为 $O(\log n)$。所以整体上界为 $O(n\log n)$。虽然实际可进一步做更细估计，但在给出的选项中应选 D。

## 第15题

下面的程序使用出边邻接表表达的带权无向图，则从顶点 0 到顶点 3 的最短距离为（ ）。

```cpp
#include <vector>
using namespace std;
class Edge {
public:
    int dest;
    int weight;
    Edge(int d, int w) : dest(d), weight(w) {}
};
class Graph {
private:
    int num_vertex;
    vector<vector<Edge>> vve;
public:
    Graph(int v) : num_vertex(v), vve(v) {}
    void addEdge(int s, int d, int w) {
        vve[s].emplace_back(d, w);
        vve[d].emplace_back(s, w);
    }
};
int main() {
    Graph g(4);
    g.addEdge(0, 1, 8);
    g.addEdge(0, 2, 5);
    g.addEdge(1, 2, 1);
    g.addEdge(1, 3, 3);
    g.addEdge(2, 3, 7);
    return 0;
}
```

A. $ 12 $
B. $ 11 $
C. $ 10 $
D. $ 9 $

答案：D

解析：无向图中的边为 $0-1$ 权 $8$、$0-2$ 权 $5$、$1-2$ 权 $1$、$1-3$ 权 $3$、$2-3$ 权 $7$。从 $0$ 到 $3$ 的可行路径中，$0\to1\to3$ 距离为 $8+3=11$；$0\to2\to3$ 距离为 $5+7=12$；$0\to2\to1\to3$ 距离为 $5+1+3=9$。其他绕路都会包含正权边，只会更长。因此最短距离为 $9$，选 D。
