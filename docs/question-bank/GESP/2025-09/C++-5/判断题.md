## 第1题

基于下面定义的函数，通过判断 `isDivisibleBy9(n) == isDigitSumDivisibleBy9(n)` 代码可验算如果一个数能被 9 整除，则它的各位数字之和能被 9 整除。

```cpp
bool isDivisibleBy9(int n) {
    return n % 9 == 0;
}

bool isDigitSumDivisibleBy9(int n) {
    int sum = 0;
    string numStr = to_string(n);
    for (char c : numStr) {
        sum += (c - '0');
    }
    return sum % 9 == 0;
}
```

答案：正确

解析：判断一个整数是否能被 $9$ 整除，可以利用十进制数的性质：一个数对 $9$ 取模的结果，与它各位数字之和对 $9$ 取模的结果相同。代码中 `isDivisibleBy9(n)` 直接判断 `n % 9 == 0`，`isDigitSumDivisibleBy9(n)` 把每个字符转成数字后求和，再判断和是否能被 $9$ 整除。两者结果相同，因此用这两个函数的返回值相等来验算该性质是成立的，判断为正确。



## 第2题

假设函数 `gcd()` 能正确求两个正整数的最大公约数，则下面的 `findMusicalPattern(4, 6)` 函数返回 2。

```cpp
void findMusicalPattern(int rhythm1, int rhythm2) {
    int commonDivisor = gcd(rhythm1, rhythm2);
    int patternLength = (rhythm1 * rhythm2) / commonDivisor;
    return patternLength;
}
```

答案：错误

解析：题干结论不成立。若先按题意假设 `gcd(4, 6)` 正确返回最大公约数 $2$，则 `patternLength = (4 * 6) / 2 = 12`，不是 $2$；这个表达式实际计算的是最小公倍数。此外，代码把函数声明成 `void`，却写了 `return patternLength;`，这在标准 C++ 中本身也是不合法的返回写法。无论从计算结果还是函数声明看，题干说返回 $2$ 都是错误的。



## 第3题

下面递归实现的斐波那契数列的时间复杂度为 $O(2^n)$。

```cpp
long long fib_memo(int n, long long memo[]) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo);
    return memo[n];
}

int main() {
    int n = 40;
    long long memo[100];
    fill_n(memo, 100, -1);
    long long result2 = fib_memo(n, memo);
    return 0;
}
```

答案：错误

解析：题干说时间复杂度为 $O(2^n)$，这是错误的。普通递归斐波那契会反复计算同一个 `fib(k)`，才会出现指数级复杂度；这段代码使用 `memo` 记忆化数组，`fib_memo(n)` 计算出某个 `n` 的结果后会保存下来，之后再次遇到同一个 `n` 会直接返回 `memo[n]`。从 $0$ 到 $n$ 的每个状态最多真正计算一次，每次只做常数次操作，所以时间复杂度是 $O(n)$，空间复杂度也是 $O(n)$。



## 第4题

链表通过更改指针实现高效的结点插入与删除，但结点访问效率低、占用内存较多，且对缓存利用不友好。

答案：正确

解析：题干表述正确。链表插入、删除结点时，若已经知道操作位置，只需要调整相邻结点的指针关系，不需要像数组那样移动后面的一串元素，因此适合频繁插入和删除。但链表不能按下标直接跳到第 $i$ 个结点，访问时通常要从头逐个走，访问效率低；每个结点还要额外保存指针，内存开销更大；结点在内存中往往不连续，CPU 缓存命中率也不如连续数组。



## 第5题

二分查找依赖数据的有序性，通过循环逐步缩减一半搜索区间来进行查找，且仅适用于数组或基于数组实现的数据结构。

答案：正确

解析：题干表述正确。二分查找每次比较中间位置后，能够舍弃一半搜索区间，前提是数据具有有序性或判断条件具有单调性；否则无法确定目标一定在哪一侧。标准二分还需要能快速访问中间元素，因此最适合数组、`vector` 这类支持随机访问的结构，或底层基于数组并能按下标访问的数据结构。链表虽然也能存有序数据，但找中点和随机访问代价高，不符合这里“逐步缩减一半区间”的高效二分模型。



## 第6题

线性筛关键是“每个合数只会被最小质因子筛到一次”，因此为 $O(n)$。

答案：正确

解析：题干表述正确。线性筛在枚举 `i` 和已找到的质数 `p` 时，用 `p * i` 标记合数，并在 `i % p == 0` 时停止继续枚举更大的质数。这样可以保证每个合数只会由“最小质因子 × 另一个数”这一种方式被标记，不会像埃氏筛那样可能被多个质因子重复标记。由于每个合数只被处理常数次，整体时间复杂度为 $O(n)$。



## 第7题

快速排序和归并排序都是稳定的排序算法。

答案：错误

解析：题干表述错误。排序算法“稳定”指相等关键字的元素在排序前后的相对顺序不变。归并排序在合并时如果相等元素优先取左半段元素，就可以保持稳定；而常见快速排序的划分过程会交换元素，可能改变相等元素的相对顺序，因此通常不稳定。题干把快速排序和归并排序都说成稳定算法，结论不成立。



## 第8题

下面代码采用分治算法求解标准 3 柱汉诺塔问题，时间复杂度为 $O(n \log n)$。

```cpp
void move(vector<int> &src, vector<int> &tar) {
    int pan = src.back();
    src.pop_back();
    tar.push_back(pan);
}

void dfs(int n, vector<int> &src, vector<int> &buf, vector<int> &tar) {
    if (n == 1) {
        move(src, tar);
        return;
    }

    dfs(n - 1, src, tar, buf);
    move(src, tar);
    dfs(n - 1, buf, src, tar);
}

void solveHanota(vector<int> &A, vector<int> &B, vector<int> &C) {
    int n = A.size();
    dfs(n, A, B, C);
}
```

答案：错误

解析：题干中“采用分治算法求解标准 3 柱汉诺塔问题”这一点是对的，但时间复杂度为 $O(n\log n)$ 是错的。递归过程要先移动 $n-1$ 个盘子，再移动最大的盘子，最后再移动 $n-1$ 个盘子，所以移动次数满足 $T(n) = 2T(n-1) + 1$，且 $T(1) = 1$。解得 $T(n) = 2^n - 1$，时间复杂度为 $O(2^n)$，不是 $O(n\log n)$，因此判断为错误。



## 第9题

所有递归算法都可以转换为迭代算法。

答案：正确

解析：题干表述正确。递归本质上依赖调用栈保存“当前执行到哪里、局部变量是什么、返回后继续做什么”。这些信息可以由程序员显式维护，例如使用栈、队列或循环中的状态变量来模拟递归调用过程，所以从可计算性和实现方式上看，递归算法都可以转换为迭代形式。只是有些递归转换后代码会更复杂，可读性也可能变差。



## 第10题

贪心算法总能得到全局最优解。

答案：错误

解析：题干表述错误。贪心算法每一步都选择当前看起来最优的方案，但只有当问题具有贪心选择性质和最优子结构时，局部最优选择才能推出全局最优。例如活动选择中按结束时间最早选择可以证明正确，而很多背包、路径或调度问题中，眼前收益最大的选择可能导致后续无法得到最优结果。因此不能说贪心算法总能得到全局最优解。
