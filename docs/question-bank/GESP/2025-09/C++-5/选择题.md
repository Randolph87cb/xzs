## 第1题

以下哪种情况使用链表比数组更合适？

A. 数据量固定且读多写少
B. 需要频繁在中间或开头插入、删除元素
C. 需要高效随机访问元素
D. 存储空间必须连续

答案：B

解析：链表的结点在内存中不要求连续，插入或删除一个已定位的结点时，主要操作是修改前后结点的指针，不需要像数组那样整体移动大量元素。因此当需要频繁在开头或中间插入、删除时，链表更合适。数组的优势是连续存储和按下标随机访问，适合数据量固定、读多写少或需要高效访问第 $i$ 个元素的场景，所以 A、C、D 都不是链表相对数组的主要优势。

## 第4题

函数 `isPerfectNumber` 判断一个正整数是否为完全数（该数是否即等于它的真因子之和），则横线上应填写（ ）。一个正整数 $n$ 的真因子包括所有小于 $n$ 的正因子，如 $28$ 的真因子为 $1, 2, 4, 7, 14$。

```cpp
bool isPerfectNumber(int n) {
    if(n <= 1) return false;
    int sum = 1;
    for(int i = 2; ______; i++) {
        if(n % i == 0) {
            sum += i;
            if(i != n/i) sum += n/i;
        }
    }
    return sum == n;
}
```

A. `i <= n`
B. `i*i <= n`
C. `i <= n/2`
D. `i < n`

答案：B

解析：如果 $i$ 是 $n$ 的一个因子，那么与它配对的另一个因子是 $n / i$。一对因子中至少有一个不超过 $\sqrt n$，所以只需要枚举到 `i*i <= n`，再在发现整除时同时加入 `i` 和 `n/i`。当 `i == n/i` 时说明是平方根，只能加一次，代码中的判断正是为了避免重复。A、C、D 虽然可能得到结果，但枚举范围过大，不符合这段成对枚举因子的写法。



## 第5题

以下代码计算两个正整数的最大公约数 (GCD)，横线上应填写（ ）。

```cpp
int gcd0(int a, int b) {
    if (a < b) {
        swap(a, b);
    }
    while(b != 0) {
        int temp = a % b;
        a = b;
        b = temp;
    }
    return ______;
}
```

A. `b`
B. `a`
C. `temp`
D. `a * b`

答案：B

解析：欧几里得算法每轮把 `(a, b)` 变成 `(b, a % b)`，不会改变两数的最大公约数。当循环结束时条件是 `b == 0`，此时最后一个非零的数保存在 `a` 中，所以应返回 `a`。返回 `b` 会得到 $0$；`temp` 的作用域只在循环体内，循环外不能使用；`a * b` 在结束时也是 $0$，不是最大公约数。

```cpp
            is_prime[p * i] = false;

```

B. `p % i == 0`
C. `i == p`
D. `i * p == n`

## 第8题

关于埃氏筛和线性筛的比较，下列说法错误的是（ ）。

A. 埃氏筛可能会对同一个合数进行多次标记
B. 线性筛的理论时间复杂度更优，所以线性筛的速度往往优于埃氏筛
C. 线性筛保证每个合数只被其最小质因子筛到一次
D. 对于常见范围 ($n \leq 10^7$)，埃氏筛因实现简单，常数较小，其速度往往优于线性筛

答案：B

解析：本题问“错误的是”。线性筛的理论复杂度是 $O(n)$，埃氏筛通常记作 $O(n\log\log n)$，但实际运行速度还受常数、内存访问、实现复杂度等影响；在常见数据范围内，埃氏筛实现简单、访问模式直接，未必比线性筛慢。因此“理论更优，所以速度往往优于埃氏筛”这个因果说法过于绝对，是错误的。A 说埃氏筛可能重复标记合数，C 说线性筛按最小质因子唯一标记，D 说明常见范围内埃氏筛可能更快，这些都成立。

## 第11题

下述 C++ 代码实现了快速排序算法，下面说法错误的是（ ）。

```cpp
int partition(vector<int>& arr, int low, int high) {
    int i = low, j = high;
    int pivot = arr[low];                    // 以首元素为基准
    while (i < j) {
        while (i < j && arr[j] >= pivot) j--;  // 从右往左查找
        while (i < j && arr[i] <= pivot) i++;  // 从左往右查找
        if (i < j) swap(arr[i], arr[j]);
    }
    swap(arr[i], arr[low]);
    return i;
}

void quickSort(vector<int>& arr, int low, int high) {
    if (low >= high) return;
    int p = partition(arr, low, high);
    quickSort(arr, low, p - 1);
    quickSort(arr, p + 1, high);
}
```

A. 快速排序之所以叫“快速”，是因为它在平均情况下运行速度较快，常数小、就地排序，实践中通常比归并排序更高效。
B. 在平均情况下，划分的递归层数为 $ \log n $，每层中的总循环数为 $ n $，总时间为 $ O(n \log n) $。
C. 在最坏情况下，每轮划分操作都将长度为 $ n $ 的数组划分为长度为 0 和 $ n - 1 $ 的两个子数组，此时递归层数达到 $ n $，每层中的循环数为 $ n $，总时间为 $ O(n^2) $。
D. 划分函数 `partition` 中“从右往左查找”与“从左往右查找”的顺序可以交换。

答案：D

解析：本题问“错误的是”。这段 `partition` 选取 `arr[low]` 为基准，并在最后执行 `swap(arr[i], arr[low])`，所以必须先从右侧找小于 `pivot` 的元素，再从左侧找大于 `pivot` 的元素，才能保证最终停下的位置适合放回基准。如果随意交换查找顺序，可能把不该放到基准左侧或右侧的元素留错位置，划分结果就不正确。A 说明快速排序实践中常数小、就地排序，B 是平均情况 $O(n\log n)$ 的推导，C 是极端不平衡划分导致 $O(n^2)$ 的推导，均正确。

## 第14题

给定一个整数数组 `nums`，下面代码找到一个具有最大和的连续子数组，并返回该最大和。则下面说法错误的是（ ）。

```cpp
int crossSum(vector<int>& nums, int left, int mid, int right) {
    int leftSum = INT_MIN, rightSum = INT_MIN;
    int sum = 0;
    for (int i = mid; i >= left; i--) {
        sum += nums[i];
        leftSum = max(leftSum, sum);
    }
    sum = 0;
    for (int i = mid + 1; i <= right; i++) {
        sum += nums[i];
        rightSum = max(rightSum, sum);
    }
    return leftSum + rightSum;
}

int helper(vector<int>& nums, int left, int right) {
    if (left == right)
        return nums[left];
    int mid = left + (right - left) / 2;
    int leftMax = helper(nums, left, mid);
    int rightMax = helper(nums, mid + 1, right);
    int crossMax = crossSum(nums, left, mid, right);
    return max({leftMax, rightMax, crossMax});
}

int maxSubArray(vector<int>& nums) {
    return helper(nums, 0, nums.size() - 1);
}
```

A. 上述代码采用分治算法实现
B. 上述代码采用贪心算法
C. 上述代码时间复杂度为 $ O(n \log n) $
D. 上述代码采用递归方式实现

答案：B

解析：本题问“错误的是”。代码中的 `helper` 每次把区间 `[left, right]` 分成左右两半，分别递归求左半最大子数组和、右半最大子数组和，再用 `crossSum` 求跨过中点的最大子数组和，最后取三者最大值。这是分治算法，并且通过递归实现。每一层所有 `crossSum` 的扫描总长度为 $O(n)$，递归层数为 $O(\log n)$，所以时间复杂度为 $O(n\log n)$。它没有使用“每一步局部最优选择推出全局最优”的贪心策略，因此 B 错误。

给定一个由非负整数组成的数组 `digits`，表示一个非负整数的各位数字，其中最高位在数组首位，且 `digits` 不含前导 0（除非是 0 本身）。下面代码对该整数执行 +1 操作，并返回结果数组，则横线上应填写（ ）。

```cpp
        if (digits[i] < 9) {
            digits[i] += 1;
            return digits;
```

A. `digits[i] = 0;`
B. `digits[i] = 9;`
C. `digits[i] = 1;`

解析：代码从最低位，也就是数组末尾开始处理进位。如果某一位小于 $9$，直接加 $1$ 后返回；如果这一位等于 $9$，加 $1$ 后会变成 $10$，当前位应写成 $0$，并继续向更高位进位。因此横线处应为 `digits[i] = 0;`。如果所有位都是 $9$，循环会把它们都置为 $0$，最后在开头插入 $1$，例如 `999` 变为 `1000`。写成 $9$、$1$ 或 $10$ 都不符合十进制进位规则。
