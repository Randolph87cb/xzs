## 第1题

在集成开发环境里调试程序时，要注意不能修改源程序，因为如果修改，就要终止调试、关闭该文件并重新打开，才能再次开始调试。（ ）

答案：错误

解析：调试时可以根据发现的问题修改源程序，现代集成开发环境通常支持停止调试、修改代码、重新编译后继续调试，有些环境还支持一定范围内的热重载。修改代码并不要求关闭文件再重新打开；真正需要的是让运行中的程序使用修改后的新版本。因此题目把调试流程说得过于绝对。



## 第2题

执行 C++ 表达式 `10 % 0.5` 将报错，因为 `0.5` 所在位置只能是整数。（ ）

答案：正确

解析：`%` 是 C++ 的取余运算符，两个操作数必须是整数类型。`0.5` 是浮点常量，不能作为 `%` 的右操作数参与取余运算，因此表达式 `10 % 0.5` 会在编译阶段报错。这里不是“除以小数结果如何”的问题，而是运算符本身不接受浮点操作数。



## 第3题

下面 C++ 代码执行后将输出 $9$。（ ）
```cpp
for (i = 0; i < 10; i++)
    break;
cout << i;
```

答案：错误

解析：`for` 循环开始时先执行 `i = 0`，判断 `i < 10` 为真后进入循环体。循环体第一句就是 `break`，会立即跳出整个循环，不会执行本轮结束后的 `i++`。因此跳出后 `i` 仍然是 `0`，`cout << i;` 输出 `0`，不是 `9`。



## 第4题

下面 C++ 代码执行后将输出 $55$。（ ）
```cpp
n = 0;
for (int i = 0; i > -10; i--)
    n = n + i * -1;
cout << n;
```

答案：错误

解析：循环中 `i` 的取值是 `0, -1, -2, ..., -9`，到 `-10` 时条件 `i > -10` 不再满足。每次累加的是 `i * -1`，也就是 `0, 1, 2, ..., 9`，总和为 $0 + 1 + \cdots + 9 = 45$。所以输出应为 `45`，不是 `55`。



## 第5题

将下面 C++ 代码中的 L1 行的 $i = 0$ 修改为 $i = 1$，其输出与当前代码输出相同。（ ）
```cpp
cnt = 0;
for (int i = 0; i < 100; i++) // L1
    cnt += i;
cout << cnt;
```

答案：正确

解析：原程序累加的是 `0 + 1 + 2 + ... + 99`。把初值改为 `i = 1` 后，累加的是 `1 + 2 + ... + 99`，少掉的只有第一项 `0`，而加上或不加 `0` 不会改变总和。因此两段程序输出相同。



## 第6题

将下面 C++ 代码中的 `i < 10` 修改为 `i <= 10`，其执行后输出相同。（ ）
```cpp
int n, i;
n = i = 0;
while (i < 10){
    n += i;
    i += 1;
}
cout << n;
```

答案：错误

解析：原条件 `i < 10` 时，循环累加 `0` 到 `9`，结果是 $45$。改成 `i <= 10` 后，`i = 10` 时还会多执行一轮，先把 `10` 加入 `n`，再把 `i` 变成 `11`，最终结果变为 $55$。多出的这一轮会改变输出。



## 第7题

下面的 C++ 代码执行后将输出 $45$。（ ）
```cpp
int n, i;
n = i = 0;
while (i < 10){
    i += 1;
    n += i;
}
cout << n;
```

答案：错误

解析：这段代码先把 `i` 加 `1`，再把新的 `i` 加到 `n`。因此 `n` 累加的是 `1 + 2 + ... + 10`，不是 `0 + 1 + ... + 9`。最终输出为 $55$，所以“输出 $45$”的判断是错的。



## 第8题

执行 C++ 代码 `cout << (12 + 12.12)` 将报错，因为 $12$ 是 `int` 类型，而 $12.12$ 是 `float` 类型，不同类型不能直接运算。（ ）

答案：错误

解析：C++ 允许整数和浮点数一起参与常见算术运算，计算时会进行类型转换，通常把整数转换为更适合表达结果的浮点类型。`12 + 12.12` 可以正常求值，结果约为 `24.12`，再由 `cout` 输出。题目把“类型不同”误认为“不能运算”，这是常见误区。



## 第9题

下面 C++ 代码执行时将导致无限循环（也称死循环）。（ ）
```cpp
int count = 0;
while (count < 5){
    count += 1;
    if (count == 3)
        continue;
    cout << count << ' ';
}
```

答案：错误

解析：`count` 在每轮循环一开始都会执行 `count += 1`，所以它依次变为 `1, 2, 3, 4, 5`。当 `count == 3` 时，`continue` 只会跳过本轮后面的输出语句，不会跳过下一轮开始前的条件判断，也不会把 `count` 改回去。之后 `count` 继续增长，到 `5` 后下一次判断 `count < 5` 为假，循环会结束。



## 第10题

下列 C++ 代码用于求斐波那契数列，即第 1 个数为 $0$、第 2 个数为 $1$，从第三个数开始，依次是其前两个数之和。如果输入的值为大于 1 的正整数，该代码能实现所求。（ ）
```cpp
cin >> n;
a = 0, b = 1;
for (int j = 0; j < n; j++){
    cout << a << " ";
    b = b + a;
    a = b - a;
}
```

答案：正确

解析：初始时 `a = 0`、`b = 1`，每轮先输出当前 `a`。随后 `b = b + a` 得到后一个数与当前数之和，再用 `a = b - a` 把 `a` 更新为原来的 `b`，也就是下一项。这样每轮结束后 `(a, b)` 都向斐波那契数列后移一位，输入大于 `1` 的正整数时，循环可以依次输出从第 $1$ 项开始的前 `n` 项。
