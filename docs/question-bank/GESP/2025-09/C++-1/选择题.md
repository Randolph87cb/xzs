## 第1题

人工智能现在非常火，小杨就想多了解一下，其中就经常听人提到“大模型”。那么请问这里说的“大模型”最贴切是指（ ）。

A. 大电脑模型
B. 大规模智能
C. 智能的单位
D. 大语言模型

答案：D

解析：“大模型”在人工智能语境中通常指参数规模大、训练数据量大、能力覆盖面广的模型。本题日常语境里最贴切的是“大语言模型”，也就是能够理解和生成自然语言的 AI 模型。A 把模型误解成“大电脑”的外形或硬件，B 不是规范名称，C 把“模型”误说成单位，都没有准确表达这个概念。



## 第2题

小杨这学期刚开学就选修了一门编程课，然后就想编写程序来计算 1 到 10001 之间的所有偶数的和。他希望程序采用简单累加整数的方法，同时希望源程序尽可能清晰、简洁，则下面关于实现累加时采用的主要控制结构的哪个说法最不合适（ ）。

A. 使用循环结构
B. 使用循环和分支的组合
C. 仅使用顺序结构
D. 不使用分支结构

答案：C

解析：要从 $1$ 到 $10001$ 中找出所有偶数并累加，清晰写法通常是让变量按固定步长循环，例如从 $2$ 开始每次加 $2$，也可以从 $1$ 到 $10001$ 循环并用分支判断偶数。因此“使用循环结构”“使用循环和分支的组合”都说得通；如果直接从 $2$ 开始步进为 $2$，还可以不写分支。仅使用顺序结构意味着把所有加法一条条写出来，既不清晰也不简洁，所以最不合适。



## 第3题

下面的 C++ 代码用于输入姓名，然后输出姓名，正确的说法是（ ）。

```cpp
string XingMing;
cout << "请输入您的姓名：";
cin >> XingMing;
cout << XingMing;
```

A. XingMing 是汉语拼音，不能作为变量名称
B. 可以将 XingMing 改为 Xing Ming
C. 可以将 XingMing 改为 xingming
D. 可以将 XingMing 改为 Xing-Ming

答案：C

解析：C++ 标识符可以由字母、数字和下划线组成，不能含空格或减号，也不能以数字开头。`XingMing` 和 `xingming` 都只由字母组成，都是合法变量名，只是大小写不同；C++ 区分大小写，改名后所有使用处也要同步改。`Xing Ming` 含空格，会被拆成两个标识符；`Xing-Ming` 中的 `-` 会被当成减号运算符；汉语拼音本身并不影响变量名合法性。



## 第4题

下列 C++ 代码中 a 和 b 都是整型变量，执行后，其结果是（ ）。

```cpp
a = 13;
b = 5;
cout << a / b << a %// b << a % b;
a * b;
```

A. 2 3
B. 23
C. 20
D. 以上都不准确

答案：C

解析：官方题面代码为 `cout << a / b << a %// b << a % b;`，这种写法非常不规范，存在排版和源码歧义。按给定答案 C 来理解，应把 `//` 视为 C++ 行注释：本行 `//` 后面的 `b << a % b;` 被注释掉。由于 `%` 前面还留下左操作数 `a`，下一行 `a * b;` 会接到表达式中，等价于形成类似 `cout << a / b << a % a * b;` 的效果。于是 `a / b = 13 / 5 = 2`，`a % a * b = 13 % 13 * 5 = 0`，输出没有空格，拼接为 `20`，所以选 C。真实编程中不推荐这种写法，应避免依赖注释和换行拼接造成歧义的代码。



## 第5题

C++ 表达式 `3 * 4 % 5 / 6` 的值是（ ）。

A. 10
B. 5
C. 2
D. 0

答案：D

解析：`*`、`%`、`/` 的优先级相同，按从左到右计算。先算 `3 * 4 = 12`，再算 `12 % 5 = 2`，最后算 `2 / 6`。这里两个操作数都是整数，整型除法会舍去小数部分，所以结果是 `0`，不是 $0.333\ldots$。



## 第6题

下面的 C++ 代码中变量 N 和 M 都是整型，则执行时如果先输入 10 并输入一个制表符后输入 20 并回车，其输出的数值是（ ）。

```cpp
scanf("%d", &N);
scanf("%d", &M);
printf("{%d}", N + M);
```

A. {30}
B. 1020
C. {N+M}
D. 不输出，继续等待输入

答案：A

解析：`scanf("%d", &N)` 读入整数时会自动跳过空白字符，空格、制表符和回车都可以作为分隔。因此先读到 `N = 10`，再读到 `M = 20`。`printf("{%d}", N + M)` 中的大括号是普通字符，`%d` 才会被替换成整数 `30`，所以输出为 `{30}`。



## 第7题

当前是 9 月，编写 C++ 代码求 N 个月后的月份。横线处应填入的代码是（ ）。

```cpp
int N, M;
cin >> N;
M = ___________;
if (M == 0)
    printf("%d个月后12月", N);
else
    printf("%d个月后是%d月", N, M);
```

A. `N % 12`
B. `9 + N % 12`
C. `(9 + N) / 12`
D. `(9 + N) % 12`

答案：D

解析：月份每 $12$ 个月循环一次，从当前 $9$ 月往后数 $N$ 个月，可以先算 `9 + N`，再对 `12` 取余得到月份编号。余数为 `0` 时表示正好落在 $12$ 月，代码下面已经专门处理了这种情况。`N % 12` 忽略了当前是 $9$ 月，`9 + N % 12` 可能得到大于 `12` 的数，`(9 + N) / 12` 求的是经过了多少个整年，不是月份。



## 第8题

下面 C++ 代码执行后的输出是（ ）。

```cpp
int n = 0;
for (int i = 0; i < 100; i++)
    n += i % 2;
cout << n;
```

A. 5050
B. 4950
C. 50
D. 49

答案：C

解析：循环变量 `i` 从 `0` 到 `99`，共 $100$ 个整数。`i % 2` 在偶数时为 `0`，在奇数时为 `1`，所以 `n` 实际统计的是这 $100$ 个数中奇数的个数。`0` 到 `99` 中奇数有 $50$ 个，因此最终输出 `50`。



## 第9题

下面的 C++ 代码执行后输出是（ ）。

```cpp
int N = 0, i;
for (i = -100; i < 100; i++)
    N += i % 10;
cout << N;
```

A. 900
B. 100
C. 0
D. -100

答案：C

解析：C++ 中整数取余的结果符号跟被除数相同，例如 `-9 % 10` 是 `-9`，`9 % 10` 是 `9`。循环从 `-100` 到 `99`，可以把 `-99` 到 `-1` 与 `1` 到 `99` 配对，它们对 `10` 的余数一正一负，正好抵消；`-100 % 10` 和 `0 % 10` 都是 `0`。所以总和为 `0`。



## 第10题

下面 C++ 代码执行后输出是（ ）。

```cpp
int i;
for(i = 1; i < 5; i++){
    if(i % 3 == 0)
        break;
    printf("%d#",i);
}
if(i > 5) printf("END\n");
```

A. 1#2#
B. 1#2#END
C. 1#2
D. 1#2#3#4#END

答案：A

解析：循环从 `i = 1` 开始。`i = 1`、`i = 2` 时都不满足 `i % 3 == 0`，分别输出 `1#` 和 `2#`；当 `i = 3` 时满足条件，执行 `break` 直接跳出循环，`3#` 不会输出。跳出后 `i` 的值是 `3`，不满足 `i > 5`，所以也不会输出 `END`。



## 第11题

下面的 C++ 代码用于求 $N$ 的镜面数（$N$ 的个位到最高位的各位数字依次反过来出现在数字中，但高位 $0$ 将被忽略，不输出），如输入 $1234$，则将输出 $4321$，又如输入 $120$，则将输出 $21$，错误的选项是（ ）。

```cpp
cout << "请输入个位数不为0的正整数：";
cin >> N;
rst = 0; // 保存逆序结果
while (_____________){
    rst = rst * 10 + N % 10;
    N = N / 10;
}
cout << rst << endl;
```

A. `N != 0`
B. `not (N == 0)`
C. `N = 0`
D. `N > 0`

答案：C

解析：循环条件应当在 `N` 还没有被除到 `0` 时继续取末位数字。`N != 0`、`not (N == 0)` 和 `N > 0` 在本题“正整数”的范围内都能表达这个意思。`N = 0` 是赋值，不是判断，它会把 `N` 改成 `0`，表达式值也为 `0`，循环体一次都不执行，因此不能得到镜面数。



## 第12题

下面 C++ 代码用于交换两个正整数 $a$ 和 $b$ 的值，不能实现交换的代码是（ ）。

A.
```cpp
cout << "输入第一个正整数：";
cin >> a;
cout << "输入第二个正整数：";
cin >> b;
temp = a;
a = b;
b = temp;
cout << "a=" << a << " b=" << b << endl;
```

B.
```cpp
cout << "输入第一个正整数：";
cin >> a;
cout << "输入第二个正整数：";
cin >> b;
b = a - b;
a = a - b;
b = a + b;
cout << "a=" << a << " b=" << b << endl;
```

C.
```cpp
cout << "输入第一个正整数：";
cin >> a;
cout << "输入第二个正整数：";
cin >> b;
a = a + b;
b = a - b;
a = a - b;
cout << "a=" << a << " b=" << b << endl;
```

D.
```cpp
cout << "输入第一个正整数：";
cin >> a;
cout << "输入第二个正整数：";
cin >> b;
a, b = b, a;
cout << "a=" << a << " b=" << b << endl;
```

答案：D

解析：A 用临时变量保存原来的 `a`，再把 `b` 赋给 `a`、临时变量赋给 `b`，可以交换。B 和 C 都是用加减关系把两个原值重新算回来，在不考虑溢出的前提下也能交换。D 中的 `a, b = b, a;` 不是 Python 式多变量交换，C++ 的逗号表达式会按逗号分隔顺序求值，实际不会把两个变量互换，因此它不能实现交换。



## 第13题

下面 C++ 代码用于获得正整数 $N$ 的第 $M$ 位数，约定个位数为第 1 位，如 $N$ 等于 $1234$，$M$ 等于 $2$，则输出 $3$。假设 $M$ 的值是大于等于 $1$ 且小于等于 $N$ 的位数。横线处应填入的代码是（ ）。

```cpp
int N, M, div=1;
cout << "请输入一个正整数：";
cin >> N;
cout << "请输入从右到左取第几位数：";
cin >> M;
for (int i = 0; i < (M - 1); i++) div *= 10;
cout << (_____________);
```

A. `N % div / 10`
B. `N / div / 10`
C. `N % div % 10`
D. `N / div % 10`

答案：D

解析：循环后 `div` 等于 $10^{M-1}$。要取从右往左第 $M$ 位，先用 `N / div` 去掉右侧 $M-1$ 位，再对 `10` 取余得到当前个位。例如 `N = 1234`、`M = 2` 时，`div = 10`，`1234 / 10 = 123`，`123 % 10 = 3`。其他选项要么少除了这一位右侧的数字，要么又多除了一次 `10`。



## 第14题

下面 C++ 代码执行后输出是（ ）。

```cpp
num = 0;
while (num <= 5){
    num += 1;
    if (num == 3)
        continue;
    printf("%d#", num);
}
```

A. 1#2#4#5#6#
B. 1#2#4#5#6
C. 1#2#3#4#5#6#
D. 1#2#3#4#5#6

答案：A

解析：`while` 判断发生在每轮开头，但 `num += 1` 在输出之前执行。`num` 依次变成 `1, 2, 3, 4, 5, 6`；当 `num == 3` 时执行 `continue`，跳过本轮的 `printf`，所以 `3#` 不输出。其余几次都会按 `"%d#"` 输出，最后一次 `num` 从 `5` 加到 `6` 后仍会完成本轮输出，因此结果是 `1#2#4#5#6#`。



## 第15题

下面 C++ 代码用于记录多个输入数中的最大数和最小数（输入 $-999$ 则输入结束），相关说法错误的是（ ）。

```cpp
cin >> now_num;
min_num = max_num = now_num;
while (now_num != -999){
    if (max_num < now_num)
        max_num = now_num;
    if (min_num > now_num)
        min_num = now_num;
    cin >> now_num;
}
cout << min_num << ' ' << max_num;
```

A. 程序运行时如果第一个数输入 $-999$，则输出将是 $-999$ $-999$
B. 程序输入过程中，如果输入的第一个数不是 $-999$，则如果待输入的数据中没有 $-999$，则程序能求出已输入整数中的最大数和最小数
C. 如果用于输入考试成绩，即成绩中不可能有 $-999$，则程序能求出已输入成绩中的最高成绩和最低成绩
D. 可以将 `cin >> now_num;` 移动到 `while (now_num != -999) {` 下面，结果不变

答案：D

解析：这段程序先读入第一个数，用它初始化最小值和最大值，然后在每轮循环末尾读入下一个数，这样哨兵值 `-999` 不会参与后续更新。若第一个数就是 `-999`，循环不执行，输出初始化得到的 `-999 -999`；若一直没有输入 `-999`，程序会继续等待新输入，已读过的数据中的最值会被正确维护。把循环末尾的 `cin >> now_num;` 移到 `while` 下面后，进入循环时 `now_num` 不会在本轮末尾更新，可能反复处理同一个数并造成死循环，所以“结果不变”是错误说法。
