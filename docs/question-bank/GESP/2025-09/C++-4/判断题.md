## 第1题

以下代码能正确初始化指针。

```cpp
int a = 5;
int *p = a;
```

答案：错误

解析：指针变量保存的应当是地址。`int *p = a;` 试图把普通整数 `a` 的值 `5` 直接赋给 `int*` 指针，类型不匹配，不能正确初始化指针。正确写法应是 `int *p = &a;`，其中 `&a` 才表示变量 `a` 的地址。因此题干说法错误。



## 第2题

执行下面 C++ 代码将输出 $11$。

```cpp
int x = 10;
void f() {
    int x = x + 1;
    cout << x << endl;
}

int main() {
    f();
}
```

答案：错误

解析：在函数 `f()` 中，语句 `int x = x + 1;` 正在声明一个新的局部变量 `x`。从声明点开始，这个局部 `x` 会遮蔽全局变量 `x`，初始化表达式右侧的 `x` 指的也是这个尚未初始化的局部变量，而不是全局的 `10`。读取未初始化的局部变量会导致未定义行为，不能确定输出为 `11`。因此题干说法错误。



## 第3题

以下 C++ 代码合法。

```cpp
struct Student {
    string name;
    int age;
    float score;
};
Student* students = new Student[20];
```

答案：正确

解析：`Student` 是一个合法的结构体类型，包含 `string`、`int`、`float` 三个成员。`new Student[20]` 会在动态存储区创建 20 个 `Student` 对象，并返回指向第一个元素的 `Student*` 指针，用 `students` 接收是合法的。因此这段代码在语法和类型上成立，题干说法正确。



## 第4题

执行下面 C++ 代码将输出 $10$。

```cpp
void func(int* p) {
    *p = 10;
}

int main() {
    int a = 5;
    func(&a);
    cout << a << endl;
    return 0;
}
```

答案：正确

解析：`main` 中 `a` 初始为 `5`，调用 `func(&a)` 时把 `a` 的地址传给形参 `p`。函数内 `*p = 10;` 通过指针访问并修改这个地址上的变量，也就是把 `a` 改为 `10`。回到 `main` 后输出 `a`，结果就是 `10`。因此题干说法正确。



## 第5题

下面代码将二维数组 `arr` 传递给函数 `f`，函数内部用 `arr[i][j]` 访问元素，函数参数声明为 `int arr[][4]` 是错误的。

```cpp
void f(int arr[][4], int rows) {
    // 访问 arr[i][j]
}

int main() {
    int arr[3][4] = { /* 初始化 */ };
    f(arr, 3);
}
```

答案：错误

解析：把二维数组作为函数参数时，第一维长度可以省略，但后续维度必须给出，因为编译器需要知道每一行有多少个元素，才能根据 `arr[i][j]` 计算地址。实参数组是 `int arr[3][4]`，每行正好有 4 个 `int`，与形参 `int arr[][4]` 匹配，所以这种声明是正确的。题干说它“是错误的”，因此题干说法错误。



## 第6题

递推是在给定初始条件下，已知前一项（或前几项）求后一项的过程。

答案：正确

解析：递推的核心是先给出一个或几个初始值，再利用相邻项之间的关系由已知项推出未知项。例如斐波那契数列中，知道前两项后，可以用前两项之和推出后一项。题干“已知前一项（或前几项）求后一项”正是递推的基本含义，因此说法正确。



## 第7题

虽然插入排序的时间复杂度为 $O(n^2)$，但由于单元操作相对较少，因此在小数据量的排序任务中非常受欢迎。

答案：正确

解析：插入排序最坏情况下需要大量移动元素，时间复杂度为 $O(n^2)$。但它的实现简单，内层主要是比较和移动相邻元素，常数开销较小；当数据量较小或数据本身接近有序时，实际运行效果往往很好。因此“小数据量排序任务中非常受欢迎”这个判断成立，题干说法正确。



## 第8题

对整数数组 $\{4, 1, 3, 1, 5, 2\}$ 进行冒泡排序（将最大元素放到最后），执行一轮之后是 $\{4, 1, 3, 1, 2, 5\}$。

答案：错误

解析：按“相邻逆序就交换、将最大元素放到最后”的冒泡排序执行一轮：初始为 `{4, 1, 3, 1, 5, 2}`。比较 `4` 和 `1`，交换得 `{1, 4, 3, 1, 5, 2}`；比较 `4` 和 `3`，交换得 `{1, 3, 4, 1, 5, 2}`；比较 `4` 和 `1`，交换得 `{1, 3, 1, 4, 5, 2}`；比较 `4` 和 `5`，不交换；比较 `5` 和 `2`，交换得 `{1, 3, 1, 4, 2, 5}`。题干给出的 `{4, 1, 3, 1, 2, 5}` 只是把 `5` 移到了最后，没有体现前面的相邻交换过程，因此说法错误。



## 第9题

以下代码只能捕获 int 类型异常。

```cpp
int main() {
    try {
        throw 42;
    } catch (...) {
        cout << "Caught" << endl;
    }
    return 0;
}
```

答案：错误

解析：`throw 42;` 抛出的确实是一个 `int` 类型异常，但 `catch (...)` 是省略号捕获器，含义是捕获任意类型的异常，而不是只捕获 `int`。如果要只捕获 `int`，应写类似 `catch (int e)` 的形式。因此题干“只能捕获 int 类型异常”的说法错误。



## 第10题

以下代码将 `Hello` 写入文件 `data.txt`。

```cpp
ofstream file("data.txt");
cout << "Hello" << endl;
file.close();
```

答案：错误

解析：`ofstream file("data.txt");` 打开了文件输出流，但后面的输出语句是 `cout << "Hello" << endl;`，它把 `Hello` 输出到标准输出，也就是屏幕或控制台，而不是写入 `file`。要写入文件，应使用 `file << "Hello" << endl;`。因此这段代码不会把 `Hello` 写入 `data.txt`，题干说法错误。
