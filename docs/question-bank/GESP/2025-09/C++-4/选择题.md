## 第1题

运行下面程序后变量 `a` 的值是（ ）。

```cpp
int a = 42;
int* p = &a;
*p = *p + 1;
```

A. 42
B. 43
C. 编译错误
D. 不确定

答案：B

解析：`p` 是指向变量 `a` 的指针，`int* p = &a;` 保存的是 `a` 的地址。表达式 `*p` 表示通过这个地址访问 `a` 本身，所以 `*p = *p + 1;` 等价于把 `a` 改成 `a + 1`。初始 `a = 42`，执行后变为 `43`，因此选 B。



## 第2题

以下关于数组的描述中，（ ）是错误的。

A. 数组名是一个指针常量
B. 随机访问数组的元素方便快捷
C. 数组可以像指针一样进行自增操作
D. `sizeof(arr)` 返回的是整个数组 `arr` 占用的字节数

答案：C

解析：数组元素在内存中连续存放，可以通过下标快速访问，`sizeof(arr)` 在 `arr` 是当前作用域中的真实数组时得到整个数组占用的字节数。数组名在很多表达式中会转换为指向首元素的指针，但它不是可以被修改的指针变量，不能写成 `arr++` 或 `++arr` 让它指向下一个元素。因此“数组可以像指针一样进行自增操作”是错误说法，选 C。严格地说，A 项“数组名是一个指针常量”是常见教学化表述，标准表述应是“数组表达式在多数情况下会转换为指向首元素的指针”。



## 第3题

给定如下定义的数组 `arr`，则 `*(*(arr + 1) + 2)` 的值是（ ）。

```cpp
int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};
```

A. 2
B. 5
C. 4
D. 6

答案：D

解析：`arr` 是一个 `2 x 3` 的二维数组，`arr + 1` 指向第 2 行，也就是 `{4, 5, 6}` 这一行。`*(arr + 1)` 取得第 2 行的首地址，再加 `2` 指向这一行下标为 `2` 的元素。最后再解引用得到 `arr[1][2]`，其值为 `6`，因此选 D。



## 第4题

下面这段代码会输出（ ）。

```cpp
int add(int a, int b = 1); // 函数声明

int main() {
    cout << add(2) << " " << add(2, 3);
    return 0;
}

int add(int a, int b) { // 函数定义
    return a + b;
}
```

A. 3 5
B. 编译失败：定义处少了默认参数
C. 运行错误
D. 链接失败：未定义引用

答案：A

解析：默认参数可以写在函数声明处，调用时如果省略对应实参，就使用声明中给出的默认值。`add(2)` 等价于 `add(2, 1)`，返回 `3`；`add(2, 3)` 使用显式传入的第二个参数，返回 `5`。两个结果中间输出一个空格，所以输出为 `3 5`，选 A。定义处不需要再次写默认参数。



## 第5题

下面这段代码会输出（ ）。

```cpp
int x = 5;

void foo() {
    int x = 10;
    cout << x << " ";
}

void bar() {
    cout << x << " ";
}

int main() {
    foo();
    bar();
}
```

A. 5 5
B. 10 10
C. 5 10
D. 10 5

答案：D

解析：全局变量 `x` 的值是 `5`。在 `foo()` 内部又定义了一个局部变量 `int x = 10;`，它会遮蔽同名全局变量，所以 `foo()` 输出的是局部变量 `10`。`bar()` 中没有局部变量 `x`，因此访问全局变量，输出 `5`。最终输出 `10 5`，选 D。



## 第6题

下面程序运行的结果是（ ）。

```cpp
void increaseA(int x) {
    x++;
}

void increaseB(int* p) {
    (*p)++;
}

int main() {
    int a = 5;
    increaseA(a);
    cout << a << " ";
    increaseB(&a);
    cout << a;
}
```

A. 6 7
B. 6 6
C. 5 6
D. 5 5

答案：C

解析：`increaseA(int x)` 是值传递，形参 `x` 只是实参 `a` 的一份拷贝，函数内 `x++` 不会改变 `main` 中的 `a`，所以第一次输出仍是 `5`。`increaseB(int* p)` 接收的是 `a` 的地址，`(*p)++` 通过指针直接修改 `a`，使它从 `5` 变为 `6`。因此输出 `5 6`，选 C。



## 第7题

关于结构体初始化，以下哪个选项中正确的是（ ）。

```cpp
struct Point {int x,y;};
```

A. `Point p = (1,2);`
B. `Point p = {1,2};`
C. `Point p = new {1,2};`
D. `Point p = <1,2>;`

答案：B

解析：`Point` 是一个包含两个 `int` 成员 `x`、`y` 的简单结构体，可以用花括号按成员声明顺序进行聚合初始化，`Point p = {1,2};` 会令 `p.x = 1`、`p.y = 2`。A 中 `(1,2)` 是逗号表达式，不是结构体初始化；C、D 都不是合法的这种结构体对象初始化写法。因此选 B。



## 第8题

运行如下代码会输出（ ）。

```cpp
struct Cat {
    string name;
    int age;
};

void birthday(Cat& c) {
    c.age++;
}

int main() {
    Cat kitty{"Mimi", 2};
    birthday(kitty);
    cout << kitty.name << " " << kitty.age;
}
```

A. Mimi 2
B. Mimi 3
C. kitty 3
D. kitty 2

答案：B

解析：`Cat kitty{"Mimi", 2};` 创建结构体变量，`name` 为 `"Mimi"`，`age` 为 `2`。函数 `birthday(Cat& c)` 的参数是引用，`c` 是 `kitty` 的别名，因此 `c.age++` 会直接把 `kitty.age` 改为 `3`。`name` 没有被修改，所以输出 `Mimi 3`，选 B。



## 第9题

关于排序算法的稳定性，以下说法错误的是（ ）。

A. 稳定的排序算法不改变相等元素的相对位置
B. 冒泡排序是稳定的排序算法
C. 选择排序是稳定的排序算法
D. 插入排序是稳定的排序算法

答案：C

解析：稳定排序要求相等关键字的元素在排序前后的相对顺序不变。冒泡排序只在相邻元素逆序时交换，插入排序通常把新元素插入到相等元素之后，所以可以保持稳定。选择排序每轮会把最小元素与当前位置交换，这个交换可能把前面的相等元素换到后面，破坏相对顺序，因此通常不是稳定排序。本题问“错误”的说法，选 C。



## 第10题

下面代码试图实现选择排序，使其能对数组 `nums` 排序为升序，则横线上应分别填写（ ）。

```cpp
void selectionSort(vector<int>& nums) {
    int n = nums.size();
    for (int i = 0; i < n - 1; ++i) {
        int minIndex = i;
        for (int j = i + 1; j < n; ++j) {
            if (__________) {  // 在此处填入代码
                minIndex = j;
            }
        }
        ______________; // 在此处填入代码
    }
}
```

A.  
```cpp
nums[j] < nums[minIndex]
swap(nums[i], nums[minIndex])
```

B.  
```cpp
nums[j] > nums[minIndex]
swap(nums[i], nums[minIndex])
```

C.  
```cpp
nums[j] <= nums[minIndex]
swap(nums[j], nums[minIndex])
```

D.  
```cpp
nums[j] <= nums[minIndex]
swap(nums[i], nums[j])
```

答案：A

解析：选择排序升序的思路是：第 `i` 轮在区间 `[i, n - 1]` 中找到最小元素的位置 `minIndex`，然后把它交换到下标 `i`。内层循环发现 `nums[j] < nums[minIndex]` 时，才应更新最小值位置；内层循环结束后应执行 `swap(nums[i], nums[minIndex])`。A 正好对应这个过程。B 找的是最大值，会排成降序；C、D 的交换位置不符合“每轮把最小值放到第 `i` 位”的选择排序过程。



## 第11题

下面程序实现插入排序（升序排序），则横线上应分别填写（ ）。

```cpp
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && ________________) { // 在此处填入代码
            arr[j + 1] = arr[j];
            j--;
        }
        ________________; // 在此处填入代码
    }
}
```

A.  
```cpp
arr[j] > key
arr[j + 1] = key
```

B.  
```cpp
arr[j] < key
arr[j + 1] = key
```

C.  
```cpp
arr[j] > key
arr[j] = key
```

D.  
```cpp
arr[j] < key
arr[j] = key
```

答案：A

解析：插入排序升序时，`key = arr[i]` 是待插入元素，左侧 `arr[0..i-1]` 已经有序。只要 `arr[j] > key`，说明这个较大的元素应该向右移动到 `arr[j + 1]`，然后继续向左找插入位置。循环结束时，`j + 1` 就是 `key` 应放入的位置，因此应写 `arr[j + 1] = key`。两处合起来对应 A。



## 第12题

关于插入排序的时间复杂度，下列说法正确的是（ ）。

A. 最好情况和最坏情况的时间复杂度都是 $O(n^2)$
B. 最好情况是 $O(n)$，最坏情况是 $O(n^2)$
C. 最好情况是 $O(n)$，最坏情况是 $O(2^n)$
D. 最好情况是 $O(n^2)$，最坏情况是 $O(2^n)$

答案：B

解析：插入排序的外层循环会从第 2 个元素开始逐个处理。最好情况下，数组本来就是升序，每次检查到 `arr[j] > key` 为假，内层循环几乎不移动元素，总次数与 `n` 成正比，所以是 $O(n)$。最坏情况下，数组逆序，第 `i` 轮可能要把前面 `i` 个元素都右移，总移动次数约为 $1 + 2 + \cdots + (n - 1)$，因此是 $O(n^2)$。选 B。



## 第13题

小杨正在爬楼梯，需要 $n$ 阶才能到达楼顶，每次可以爬 $1$ 阶或 $2$ 阶，求小杨有多少种不同的方法可以爬到楼顶，横线上应填写（ ）。

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    int prev2 = 1;
    int prev1 = 2;
    int current = 0;
    for (int i = 3; i <= n; ++i) {
        ________________ // 在此处填入代码
    }
    return current;
}
```

A.  
```cpp
prev2 = prev1;
prev1 = current;
current = prev1 + prev2;
```

B.  
```cpp
current = prev1 + prev2;
prev2 = prev1;
prev1 = current;
```

C.  
```cpp
current = prev1 + prev2;
prev1 = current;
prev2 = prev1;
```

D.  
```cpp
prev1 = current;
prev2 = prev1;
current = prev1 + prev2;
```

答案：B

解析：爬到第 `i` 阶的方法数等于爬到第 `i - 1` 阶后再走 1 阶的方法数，加上爬到第 `i - 2` 阶后再走 2 阶的方法数，所以递推式是 `f(i) = f(i - 1) + f(i - 2)`。代码中 `prev2` 表示 `f(i - 2)`，`prev1` 表示 `f(i - 1)`，应先计算 `current = prev1 + prev2`，再把状态向后滚动：`prev2 = prev1; prev1 = current;`。如果先更新 `prev1` 或 `prev2`，就会丢失旧值。选 B。



## 第14题

假设有一个班级的成绩单，存储在一个长度为 $n$ 的数组 `scores` 中，每个元素是一个学生的分数。老师想要找出所有满足 `scores[i] + scores[j] + scores[k] == 300` 的三元组，其中 $i < j < k$。下面代码实现该功能，请问其时间复杂度是（ ）。

```cpp
int cnt = 0;
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        for (int k = j + 1; k < n; k++) {
            if (scores[i] + scores[j] + scores[k] == 300) {
                cnt++;
            }
        }
    }
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(n^3)$
D. $O(2^n)$

答案：C

解析：这段代码有三层嵌套循环。第一层枚举 `i`，第二层枚举满足 `j > i` 的 `j`，第三层枚举满足 `k > j` 的 `k`，也就是枚举所有下标满足 `i < j < k` 的三元组。三元组数量是 $\binom{n}{3} = \frac{n(n-1)(n-2)}{6}$，最高次项为 $n^3$，每次检查只做常数次加法和比较，所以时间复杂度是 $O(n^3)$，选 C。



## 第15题

关于异常处理，以下说法错误的是（ ）。

A. `try` 块中的代码可能会抛出异常
B. `catch` 块可以有多个，处理不同类型的异常
C. `throw` 语句用于抛出异常
D. 所有异常都必须被捕获，否则程序会崩溃

答案：D

解析：`try` 块用于放置可能抛出异常的代码，`throw` 用于抛出异常，多个 `catch` 可以按类型处理不同异常，这些说法都成立。D 项的问题在于“所有异常都必须被捕获”说得过绝对：C++ 语法并不要求每个异常都必须在本处捕获，未被当前层捕获的异常会继续向调用栈外传播；如果最终仍无人处理，程序才会异常终止。因此本题问错误说法，选 D。
