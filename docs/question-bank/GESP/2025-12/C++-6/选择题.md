## 第1题

在面向对象编程中，下列关于虚函数的描述中，错误的是（ ）。

A. 虚函数用于支持运行时多态
B. 通过基类指针调用虚函数时，会根据对象实际类型决定调用版本
C. 构造函数可以声明为虚函数以支持多态
D. 基类析构函数常声明为虚函数以避免资源泄漏

答案：C

解析：虚函数的动态绑定发生在“对象已经构造完成、通过基类指针或引用调用虚成员函数”时。构造函数的任务正是创建对象，在构造开始时还没有一个完整的派生类对象可供动态绑定，因此 C++ 不允许把构造函数声明为 `virtual`。析构函数则不同：如果通过基类指针删除派生类对象，基类析构函数应声明为虚函数，才能先调用派生类析构函数再调用基类析构函数，避免派生类资源没有释放。



## 第2题

执行如下代码，会输出 钢琴：叮咚叮咚 和 吉他：咚咚当当 。这体现了面向对象编程的（ ）特性。

```cpp
class Instrument {
public:
    virtual void play() {
        cout << "乐器在演奏声音" << endl;
    }

    virtual ~Instrument() {}
};

class Piano : public Instrument {
public:
    void play() override {
        cout << "钢琴: 叮咚叮咚" << endl;
    }
};

class Guitar : public Instrument {
public:
    void play() override {
        cout << "吉他: 咚咚当当" << endl;
    }
};

int main() {
    Instrument* instruments[2];
    instruments[0] = new Piano();
    instruments[1] = new Guitar();

    for (int i = 0; i < 2; ++i) {
        instruments[i]->play();
    }

    for (int i = 0; i < 3; ++i) {
        delete instruments[i];
    }
    return 0;
}
```

A. 继承
B. 封装
C. 多态
D. 链接

答案：C

解析：数组中保存的是 `Instrument*`，静态类型都是基类指针，但实际指向的对象分别是 `Piano` 和 `Guitar`。因为 `Instrument::play()` 被声明为 `virtual`，执行 `instruments[i]->play()` 时会按对象的实际类型动态绑定：第一个对象调用 `Piano::play()`，第二个对象调用 `Guitar::play()`。同一个调用形式在运行时表现出不同实现，这正是运行时多态。需要注意，后面的删除循环写成 `i < 3` 会访问 `instruments[2]`，这是题面代码的边界问题；本题选项考察的是前一个播放循环体现出的面向对象特性。

## 第4题

某文本编辑器把用户输入的字符依次压入栈 $S$。用户依次输入 $A$，$B$，$C$，$D$ 后，用户按了两次撤销（每次撤销，弹出栈顶一个字符）。此时栈从栈底到栈顶的内容是：（ ）。

A. $A$ $B$
B. $A$ $B$ $C$
C. $A$ $B$ $D$
D. $B$ $C$

答案：A

解析：栈是后进先出结构。依次输入 $A,B,C,D$ 后，栈底到栈顶为 $A,B,C,D$，栈顶是最后压入的 $D$。第一次撤销弹出 $D$，第二次撤销再弹出 $C$，留下的元素从栈底到栈顶就是 $A,B$。



## 第5题

假设循环队列数组长度为 $N$，其中队空判断条件为：`front == rear`，队满判断条件为：`(rear + 1) % N == front`，出队对应的操作为：`front = (front + 1) % N`，入队对应的操作为：`rear = (rear + 1) % N`。循环队列长度 $N = 6$，初始 `front = 1`，`rear = 1`，执行操作序列为：入队，入队，入队，出队，入队，入队，则最终 `(front, rear)` 的值是（ ）。

A. $(2, 5)$
B. $(2, 0)$
C. $(3, 5)$
D. $(3, 0)$

答案：B

解析：循环队列每次入队只移动 `rear`，每次出队只移动 `front`，位置都要对 $N=6$ 取模。初始为 `(1,1)`；三次入队后 `rear` 依次变为 $2,3,4$，得到 `(1,4)`；一次出队后 `front` 变为 $2$，得到 `(2,4)`；再两次入队，`rear` 从 $4$ 到 $5$，再由 $(5+1)\bmod 6=0$ 回到 $0$。最终为 `(2,0)`。



## 第6题

以下函数 `check()` 用于判断一棵二叉树是否为（ ）。

```cpp
bool check(TreeNode* root) {
    if (!root) return true;

    queue<TreeNode*> q;
    q.push(root);
    bool hasNull = false;
    while (!q.empty()) {
        TreeNode* cur = q.front(); q.pop();
        if (!cur) {
            hasNull = true;
        } else {
            if (hasNull) return false;
            q.push(cur->left);
            q.push(cur->right);
        }
    }
    return true;
}
```

A. 满二叉树
B. 完全二叉树
C. 二叉搜索树
D. 平衡二叉树

答案：B

解析：这段代码按层序遍历二叉树，并且把空孩子指针也入队。完全二叉树的层序序列有一个关键性质：一旦遇到空位置，后面就不能再出现非空结点。变量 `hasNull` 记录是否已经遇到过空指针；如果之后又弹出一个非空结点，就说明某一层出现了“左边有空洞、右边还有结点”的情况，返回 `false`。这正是完全二叉树的判定方法。



## 第7题

以下代码实现了二叉树的（ ）。

```cpp
void traverse(TreeNode* root) {
    if (!root) return;
    traverse(root->left);
    traverse(root->right);
    cout << root->val << " ";
}
```

A. 前序遍历
B. 中序遍历
C. 后序遍历
D. 层序遍历

答案：C

解析：递归遍历时要看访问根结点的位置。代码先递归访问 `root->left`，再递归访问 `root->right`，最后才输出 `root->val`，顺序是“左子树、右子树、根”。这对应后序遍历；前序是“根、左、右”，中序是“左、根、右”，层序则需要队列按层访问。

## 第9题

以下关于哈夫曼编码的说法，正确的是（ ）。

A. 哈夫曼编码是定长编码
B. 哈夫曼编码中，没有任何一个字符的编码是另一个字符编码的前缀
C. 哈夫曼编码一定唯一
D. 哈夫曼编码不能用于数据压缩

答案：B

解析：哈夫曼编码由哈夫曼树的根到叶路径得到，每个字符对应一个叶子结点。一个叶子结点不可能同时是另一个叶子结点的祖先，所以任意一个字符编码都不会成为另一个字符编码的前缀，这叫前缀编码或即时可译码。它通常是变长编码；当多个字符频率相同或合并时左右孩子选择不同，编码结果可能不唯一；哈夫曼编码的主要用途之一就是压缩。



## 第10题

以下函数实现了二叉排序树（BST）的（ ）操作。

```cpp
TreeNode* op(TreeNode* root, int x) {
    if (!root) return new TreeNode(x);
    if (x < root->val)
        root->left = op(root->left, x);
    else
        root->right = op(root->right, x);
    return root;
}
```

A. 查找
B. 插入
C. 删除
D. 遍历

答案：B

解析：函数遇到空指针时 `return new TreeNode(x)`，说明它会在搜索路径末端创建一个新结点。若 `x < root->val`，递归插入左子树；否则递归插入右子树；递归返回后再把子树根接回 `root->left` 或 `root->right`，最后返回当前根。整个过程保持 BST 的“左小右大”方向，并把新值接入树中，因此是插入操作。

## 第13题

在二叉排序树（Binary Search Tree, BST）中，假设节点值互不相同。给定如下搜索函数，以下说法一定正确的是（ ）。

```cpp
bool find(Node* root, int x) {
    while (root) {
        if (root->val == x) return true;
        root = (x < root->val) ? root->left : root->right;
    }
    return false;
}
```

A. 最坏情况下，访问结点数是 $O(\log n)$
B. 最坏情况下，访问结点数是 $O(n)$
C. 无论如何，访问结点数都不超过树高的一半
D. 一定比在普通二叉树中搜索快

答案：B

解析：BST 查找每一步只会沿左子树或右子树继续走，访问结点数与查找路径长度成正比，也就是 $O(h)$，其中 $h$ 是树高。若 BST 很平衡，$h$ 可以是 $O(\log n)$；但题目只说它是 BST，没有保证平衡，插入有序数据时可能退化成一条链，树高达到 $n$。因此最坏情况下访问结点数是 $O(n)$。



## 第14题

0/1 背包（每件物品最多选一次）问题通常可用一维动态规划求解，核心代码如下。则下面说法正确的是（ ）。

```cpp
for each item (w, v):
    for (int j = W; j >= w; --j)
        dp[j] = max(dp[j], dp[j-w] + v);
```

A. 内层 `j` 必须从小到大，否则会漏解
B. 内层 `j` 必须从大到小，否则同一件物品会被用多次
C. `j` 从大到小或从小到大都一样
D. 只要 `dp` 初始为 `0`，方向无所谓

答案：B

解析：一维 0/1 背包中，处理当前物品时，`dp[j-w]` 必须表示“还没有使用当前物品”时的旧状态，所以 `j` 要从大到小枚举。若从小到大，较小容量的 `dp[j-w]` 可能已经在本轮加入过当前物品，再用它更新更大的 `dp[j]`，就等于同一件物品被选了多次，变成完全背包的效果。倒序枚举正是为了保证每件物品最多选一次。



## 第15题

以下关于动态规划的说法中，错误的是（ ）。

A. 动态规划方法通常能列出递推公式。
B. 动态规划方法的时间复杂度通常为状态的个数。
C. 动态规划方法有递推和递归两种实现形式。
D. 对很多问题，递推实现和递归实现动态规划方法的时间复杂度相当。

答案：B

解析：动态规划通常要先定义状态，再写出状态转移，这与 A 的说法一致；实现上可以自底向上递推，也可以用递归加记忆化搜索，很多问题两种写法访问的状态和转移数量相当。时间复杂度不能只看状态个数，还要乘上每个状态需要枚举或计算的转移数量。例如某些区间 DP 有 $O(n^2)$ 个状态，但每个状态还要枚举断点，时间可能达到 $O(n^3)$。因此“时间复杂度通常为状态的个数”遗漏了转移代价，是错误说法。
