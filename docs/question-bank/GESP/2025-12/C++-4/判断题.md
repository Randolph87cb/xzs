## 第1题

小杨正在调试他的温度传感器程序，其中变量 x 保存当前温度。下面这段代码运行后，变量 x 的值变成了 8。

```cpp
int x = 5;
int *p = &x;
*p = *p + 3;
```

答案：正确

解析：`p` 被初始化为 `&x`，因此它保存的是变量 `x` 的地址。表达式 `*p` 表示访问 `p` 指向的变量，也就是 `x` 本身。执行 `*p = *p + 3` 时，右侧先读到当前值 `5`，加 `3` 得到 `8`，再写回 `*p`，所以 `x` 的值变为 `8`。



## 第2题

一个结构体不能包含另一个结构体。

答案：错误

解析：结构体可以把另一个已经定义好的结构体作为成员，这是 C++ 中组织复合数据的常见写法。例如可以先定义 `Point`，再在 `Rectangle` 中写 `Point topLeft; Point bottomRight;`。需要注意的是，结构体不能直接包含“自身类型的普通成员”，否则大小无法确定；但它可以包含其他结构体，也可以包含指向自身类型的指针成员。



## 第3题

在 C++ 中，定义如下二维数组：`int a[3][4];`，数组 a 在内存中是按行优先连续存放的，即 `a[0][0]`、`a[0][1]`、`a[0][2]`、`a[0][3]` 在内存中是连续的。

答案：正确

解析：C++ 的普通二维数组按行优先顺序连续存放。对 `int a[3][4];` 来说，内存顺序先放完第 `0` 行的 `a[0][0]`、`a[0][1]`、`a[0][2]`、`a[0][3]`，再放第 `1` 行。因此题干说这四个元素连续存放是正确的。



## 第4题

执行下面程序后，变量 a 的值会变成 15。

```cpp
void add(int &x){
    x += 10;
}

int a = 5;
add(a);
```

答案：正确

解析：函数形参 `int &x` 是引用，调用 `add(a)` 后，`x` 就是变量 `a` 的别名。函数体中的 `x += 10` 等价于对 `a` 执行 `a += 10`。`a` 初值为 `5`，加上 `10` 后变为 `15`，所以题干判断正确。



## 第5题

执行下面的 C++ 代码，会输出 8，因为两个指针地址相差 8 个字节（假设 int 占 4 字节）。

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p1 = arr;
int* p2 = arr + 2;
cout << p2 - p1;  // 输出结果
```

答案：错误

解析：两个同一数组内的指针相减，结果不是字节数，而是两个指针之间相差的元素个数。`p1 = arr` 指向 `arr[0]`，`p2 = arr + 2` 指向 `arr[2]`，所以 `p2 - p1` 的值是 `2`。即使一个 `int` 占 $4$ 字节，地址相差 $8$ 字节，指针减法的输出仍按元素个数计算，不会输出 `8`。



## 第6题

考虑用如下递推方式计算斐波那契数列，时间复杂度是 $O(n)$。

```cpp
int n = 10;
int f[20];
f[0] = 0;
f[1] = 1;
for (int i = 2; i <= n; i++)
    f[i] = f[i - 1] + f[i - 2];
```

答案：正确

解析：这段程序用循环从 `i=2` 计算到 `i=n`，每一轮只做一次加法和一次赋值，单轮是常数时间。循环次数随 `n` 线性增长，大约是 `n-1` 次，所以时间复杂度为 $O(n)$。数组 `f` 保存中间项会影响空间使用，但不改变时间复杂度判断。



## 第7题

冒泡排序和插入排序都是稳定排序算法。

答案：正确

解析：稳定排序的核心要求是：两个关键字相等的元素，排序后相对先后顺序不改变。标准冒泡排序只有在相邻元素逆序时才交换，相等时不交换；标准插入排序通常只把大于待插入元素的项后移，相等元素不会被跨过去。因此这两种常见写法都保持相等元素的相对顺序，题干正确。



## 第8题

下面这段代码实现了选择排序算法。

```cpp
void sort(int a[], int n) {
    for (int i = 1; i < n; i++) {
        int x = a[i];
        int j = i - 1;
        while (j >= 0 && a[j] > x) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = x;
    }
}
```

答案：错误

解析：这段代码的流程是：从第 `i` 个元素取出 `x`，向左扫描已经有序的前缀区间，把所有大于 `x` 的元素右移，最后把 `x` 放到空出的位置。这正是插入排序的典型过程。选择排序则是在未排序区间中寻找最小或最大元素，再与当前位置交换；代码中没有寻找最值下标并交换的步骤，所以题干错误。



## 第9题

下面代码可以正常编译并输出 10。

```cpp
#include <iostream>
using namespace std;

int calculate(int x, int y = 10);

int main() {
    cout << calculate(5);  // 调用1
    return 0;
}

int calculate(int x, int y) {
    return x * y;
}

int calculate(int x) {  // 重载函数
    return x * 2;
}
```

答案：错误

解析：在 `main` 调用处，前面可见的声明是 `int calculate(int x, int y = 10);`，因此 `calculate(5)` 可以用默认参数补成 `calculate(5, 10)`，返回值是 `5 * 10 = 50`。后面定义的 `int calculate(int x)` 虽然也是一个重载函数，但它在这次调用之后才出现，调用点的重载决议不会使用尚未声明的函数。程序不是输出 `10`，所以题干错误。



## 第10题

执行下面代码会输出 100。

```cpp
int main() {
    ofstream fout("data.txt");
    fout << 10 << " " << 20 << endl;
    fout << 30 << " " << 40;
    fout.close();

    ifstream fin("data.txt");
    int a, b, c, d;
    fin >> a >> b >> c >> d;
    fin.close();

    cout << a + b + c + d;
    return 0;
}
```

答案：正确

解析：文件中依次写入四个整数：第一行 `10 20`，第二行 `30 40`。读取时使用 `fin >> a >> b >> c >> d`，提取运算符会自动跳过空格和换行，所以四个变量依次得到 `10`、`20`、`30`、`40`。它们的和为 $10+20+30+40=100$，因此会输出 `100`。
