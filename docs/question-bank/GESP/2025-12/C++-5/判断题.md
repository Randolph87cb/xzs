## 第1题

数组和链表都是线性表。链表的优点是插入删除不需要移动元素，并且能随机查找。

答案：错误

解析：数组和链表都可以按线性顺序存放一组元素，因此都属于线性表。链表的优势是插入、删除时主要修改指针，不需要像顺序数组那样整体搬移后续元素。但链表不能随机访问第 $i$ 个元素，通常必须从头结点沿 `next` 一个一个走过去，查找位置的时间复杂度是 $O(n)$。题干把“能随机查找”也说成链表优点，所以错误。



## 第2题

假设函数 `gcd()` 函数能正确求两个正整数的最大公约数，则下面的 `lcm(a, b)` 函数能正确找到两个正整数 $ a $ 和 $ b $ 的最小公倍数。

```cpp
int lcm(int a, int b) {
    return a / gcd(a, b) * b;
}
```

答案：正确

解析：两个正整数满足 $\gcd(a,b)\times \operatorname{lcm}(a,b)=a\times b$，所以 $\operatorname{lcm}(a,b)=a / \gcd(a,b) \times b$。代码先做除法再乘以 `b`，在数学结果上等价于最小公倍数，也比直接 `a * b / gcd(a, b)` 更能降低中间乘积溢出的风险。题干已假设 `gcd()` 正确，因此该函数能正确表达最小公倍数公式。



## 第3题

在单链表中，已知指针 $ p $ 指向要删除的结点（非尾结点），想在 $ O(1) $ 删除 $ p $，可行做法是用 `p->next` 覆盖 $ p $ 的值与 $ next $，然后删除 `p->next`。

答案：正确

解析：单链表通常删除某结点需要知道它的前驱；但如果已知 `p` 不是尾结点，可以把 `p->next` 结点的数据复制到 `p`，再让 `p->next` 指向下下个结点，最后删除原来的后继结点。这样链表中从外部看起来少了原本 `p` 位置的那个值，且只做常数次指针和数据操作，所以是 $O(1)$。该技巧不能用于尾结点，因为尾结点没有后继可复制。



## 第4题

在求解所有不大于 $ n $ 的素数时，线性筛法（欧拉筛）都应当优先于埃氏筛法使用，因为线性筛法的时间复杂度为 $ O(n) $，低于埃氏筛法的 $ O(n \log \log n) $。

答案：错误

解析：线性筛的理论时间复杂度是 $O(n)$，埃氏筛约为 $O(n\log\log n)$，但“都应当优先”说得过于绝对。埃氏筛实现更短、常数小、缓存表现好，很多数据范围下足够快；线性筛为了保证每个合数只被最小质因子筛一次，需要维护质数表和 `break` 条件，代码复杂度更高。选择算法要结合数据范围、实现成本和实际常数，不能只凭渐进复杂度一概而论。



## 第5题

二分查找仅适用于有序数据。若输入数据无序，当仅进行一次查找时，为了使用二分而排序通常不划算。

答案：正确

解析：二分查找依赖数据具有单调性，最常见条件就是数组已经有序；无序数据中间位置的大小不能判断答案应在左边还是右边。若只有一次查找，先排序通常要 $O(n\log n)$，再二分是 $O(\log n)$，总成本往往高于直接线性扫描的 $O(n)$。如果要在同一批数据上做很多次查找，排序后反复二分才更可能划算。



## 第6题

通过在数组的第一个、最中间和最后一个这 3 个数据中选择中间值作为枢轴（比较基准），快速排序算法可降低落入最坏情况的概率。

答案：正确

解析：快速排序最坏情况常出现在枢轴总是选到当前区间的极小值或极大值，导致划分极不平衡。三数取中会比较首元素、中间元素、尾元素，并选三者的中间值作为枢轴；在有序、逆序或某些接近有序的数据上，它能避免直接选到端点极值。它不能从理论上消除最坏情况，但确实能降低普通输入落入最坏划分的概率。



## 第7题

贪心算法在每一步都做出当前看来最优的局部选择，并且一旦做出选择就不再回溯；而分治算法将问题分解为若干子问题分别求解，再将子问题的解合并得到原问题的解。

答案：正确

解析：贪心算法的核心是每一步按当前信息选择一个局部最优方案，并把这个选择固定下来继续往后做；它是否能得到全局最优，还需要问题具有贪心选择性质和最优子结构。分治算法的核心则是把原问题拆成若干规模更小、结构相同或相近的子问题，分别求解后再合并，例如归并排序先排左右两半再合并。题干分别概括了二者的基本特征，因此正确。



## 第8题

以下 `fib` 函数计算第 $ n $ 项斐波那契数（$ fib(0)=0 $，$ fib(1)=1 $），其时间复杂度为 $ O(n) $。

```cpp
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}
```

答案：错误

解析：这个 `fib` 不是线性递推写法，而是每次同时递归计算 `fib(n-1)` 和 `fib(n-2)`。许多子问题会被重复计算，例如 `fib(n-2)` 会在计算 `fib(n-1)` 的过程中出现，又会被右支单独计算一次。其调用次数满足类似 $T(n)=T(n-1)+T(n-2)+O(1)$ 的递推，增长是指数级的，不是 $O(n)$。若想做到 $O(n)$，需要循环递推或记忆化保存已算过的项。



## 第9题

递归函数一定要有终止条件，否则可能会造成栈溢出。

答案：正确

解析：每一次递归调用都会在调用栈上保存当前函数的局部变量、返回地址等信息。终止条件用于让递归在某个规模下停止继续调用，并开始逐层返回。若没有终止条件，或者终止条件永远无法被触发，函数会不断调用自身，栈空间持续增长，最终很可能造成栈溢出。因此题干判断正确。



## 第10题

使用贪心算法解决问题时，通过对每一步求局部最优解，最终一定能找到全局最优解。

答案：错误

解析：贪心算法只保证每一步选择当前看起来最好的方案，并不自动保证全局最优。只有当问题满足贪心选择性质时，局部最优选择才能被证明可以扩展成全局最优解。反例可以看找零：若硬币面额为 $1,3,4$，要凑 $6$，每次取不超过剩余金额的最大硬币会取 $4+1+1$，共 $3$ 枚；最优方案是 $3+3$，只需 $2$ 枚。因此“最终一定能找到全局最优解”的说法错误。
