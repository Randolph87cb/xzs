## 第1题

下面关于 C++ 中形参、实参和定义域的说法中，正确的一项是（ ）。

A. 形参是函数定义时所指定的变量，它只在函数内部有效。
B. 在函数内部，可以修改传入的形参的值，即使该形参是一个常量引用。
C. 实参和形参的类型必须完全一致，否则会导致编译错误。
D. 使用指针作为形参时，形参是指向实参的地址，因此对该指针赋值会影响实参。

答案：A

解析：函数定义中的参数变量叫形参，例如 `void f(int x)` 中的 `x`。形参的作用域在函数体内，函数调用结束后这个形参变量也随之结束，因此 A 正确。常量引用形参不能在函数内被修改，所以 B 错；实参可以通过合法的隐式转换传给形参，并不要求类型完全一致，所以 C 错；指针形参本身也是一份拷贝，给这个指针变量重新赋值不会改变调用处的指针或对象地址，只有通过解引用修改指向对象时才可能影响实参，所以 D 错。



## 第2题

已知三个序列：`s1 = {3, 1, 8, 2, 5, 6, 7, 4}`，`s2 = {1, 5, 1, 8, 6, 4, 7, 5, 6}`，`s3 = {1, 8, 3, 5, 7, 6, 2, 4}`。以下哪个序列是它们的最长公共子序列（ ）。

A. `{1, 8, 5, 6}`
B. `{1, 5, 6, 7}`
C. `{1, 8, 6}`
D. `{1, 5, 7, 4}`

答案：A

解析：公共子序列要求在三个序列中都保持相同的相对顺序，不要求连续。A 中 `{1, 8, 5, 6}` 在 `s1` 中可取第 $2,3,5,6$ 项，在 `s2` 中可取第 $1,4,8,9$ 项，在 `s3` 中可取第 $1,2,4,6$ 项，因此是长度为 $4$ 的公共子序列。用三序列 LCS 动态规划按前缀计算，最终最大长度为 $4$，所以长度为 $4$ 的 A 已达到最长。B 在 `s3` 中 `7` 出现在 `6` 前面，D 在 `s2` 中 `4` 出现在 `7` 前面，都不是三个序列的公共子序列；C 只有长度 $3$，不是最长。



## 第3题

现有一个地址区间为 $0 \sim 10$ 的哈希表，当出现冲突情况，会往后找第一个空的地址存储（到 $10$ 冲突了就从 $0$ 开始往后），现在要依次存储 $(1,3,5,7,9)$，哈希函数为 $h(x) = (x^2 + x) \bmod 11$。其中 $9$ 存储在哈希表哪个地址中（ ）。

A. 1
B. 2
C. 3
D. 4

答案：D

解析：依次计算并按线性探测放入哈希表。$h(1)=(1^2+1)\bmod 11=2$，地址 $2$ 空，放入 $1$；$h(3)=12\bmod 11=1$，放入地址 $1$；$h(5)=30\bmod 11=8$，放入地址 $8$；$h(7)=56\bmod 11=1$，地址 $1,2$ 已被占用，继续往后到地址 $3$ 放入。最后 $h(9)=90\bmod 11=2$，地址 $2,3$ 已占用，继续往后第一个空地址是 $4$，所以 $9$ 存在地址 $4$。



## 第4题

在 0/1 背包问题中，给定一组物品，每个物品有一个重量和价值，背包的容量有限。假设背包的最大容量为 $W$，物品的数量为 $n$，其中第 $i$ 个物品的重量为 $w[i]$，价值为 $v[i]$。以下关于 0/1 背包问题的描述，正确的是（ ）。

A. 在解决 0/1 背包问题时，使用贪心算法可以保证找到最优解，因为物品只能放入一次。
B. 0/1 背包是 P 问题（多项式时间可解问题），它可以在 $O(nW)$ 的时间复杂度内解决。
C. 0/1 背包问题中，动态规划解法的空间复杂度为 $O(nW)$，但可以通过滚动数组技巧将空间复杂度优化到 $O(W)$。
D. 0/1 背包问题中，每个物品只能选择一次，并且子问题之间是独立的，无法重用计算结果。

答案：C

解析：标准 0/1 背包可以设 `dp[i][j]` 表示只考虑前 $i$ 个物品、容量为 $j$ 时的最大价值，二维状态需要 $O(nW)$ 空间。由于第 $i$ 层只依赖第 $i-1$ 层，可以改成一维 `dp[j]`，并且容量必须从大到小枚举，避免同一件物品被重复使用，空间降为 $O(W)$，所以 C 正确。贪心按价值或性价比选物品不能保证 0/1 背包最优，A 错；$O(nW)$ 是伪多项式复杂度，不能据此直接说 0/1 背包是一般意义上的 P 问题，B 错；动态规划正是利用重叠子问题重用计算结果，D 错。



## 第5题

一棵深度为 $6$（根节点深度为 $1$）的完全二叉树，节点总数最少有（ ）。

A. 31
B. 32
C. 63
D. 64

答案：B

解析：完全二叉树要求除最后一层外，其余各层都必须填满，最后一层从左到右连续填充。深度为 $6$ 说明第 $6$ 层至少有 $1$ 个结点；前 $5$ 层必须是满的，共有 $1+2+4+8+16=31$ 个结点。因此节点总数最少为 $31+1=32$。



## 第6题

对于如下二叉树，下面关于访问的顺序说法错误的是（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/f778m3c4.png)

A. D E B F H J I G C A 是它的后序遍历序列。
B. A B C D E F G H I J 是它的广度优先遍历序列。
C. A B D E C F G H I J 是它的先序遍历序列。
D. D B E A F C H G J I 是它的中序遍历序列。

G J I`，所以 D 错。



## 第7题

下面程序的运行结果为（ ）。

```cpp
#include <iostream>

int query(int n, int *a, int x) {
    int l = 0, r = n;
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (a[mid] >= x) r = mid;
        else l = mid + 1;
    }

    if (l == n) return -1;
    return l;
}

int main() {
    int n = 10;
    int x = 3;
    int num[] = {1, 2, 2, 3, 3, 4, 5, 5, 6, 7};

    std::cout << query(n, num, x) << "\n";
    return 0;
}
```

A. 2
B. 3
C. 4
D. 5

答案：B

解析：`query` 做的是二分查找中的 `lower_bound`：返回第一个满足 `a[i] >= x` 的下标。初始 `l=0,r=10`，第一次 `mid=5`，`a[5]=4>=3`，令 `r=5`；第二次 `mid=2`，`a[2]=2<3`，令 `l=3`；第三次 `mid=4`，`a[4]=3>=3`，令 `r=4`；第四次 `mid=3`，`a[3]=3>=3`，令 `r=3`。此时 `l==r==3`，函数返回下标 `3`。



## 第8题

下面程序中，函数 `query` 的时间复杂度是（ ）。

```cpp
#include <iostream>

int query(int n, int *a, int x) {
    int l = 0, r = n;
    while (l < r) {
        int mid = l + (r - l) / 2;
        if (a[mid] >= x) r = mid;
        else l = mid + 1;
    }

    if (l == n) return -1;
    return l;
}

int main() {
    int n = 10;
    int x = 3;
    int num[] = {1, 2, 2, 3, 3, 4, 5, 5, 6, 7};

    std::cout << query(n, num, x) << "\n";
    return 0;
}
```

A. $O(1)$
B. $O(\log n)$
C. $O(n)$
D. $O(n \log n)$

答案：B

解析：`query` 每轮循环都会把搜索区间 `[l,r)` 至少缩小到原来的一半：若 `a[mid] >= x`，右边界变成 `mid`；否则左边界变成 `mid+1`。长度从 $n$ 变成约 $n/2,n/4,\ldots$，直到区间为空或只剩一个位置，循环次数是 $O(\log n)$。循环外只做常数次判断和返回，因此总时间复杂度为 $O(\log n)$。



## 第9题

有 5 个字符，它们出现的次数分别为 2 次、2 次、3 次、3 次、5 次。现在要用哈夫曼编码的方式来为这些字符进行编码，最小加权路径长度 WPL（每个字符的出现次数 $\times$ 它的编码长度，再把每个字符结果加起来）的值为（ ）。

A. 30
B. 34
C. 43
D. 47

答案：B

解析：哈夫曼树每次合并当前最小的两个权值，所有合并代价之和就是最小 WPL。权值为 $2,2,3,3,5$：先合并 $2+2=4$，再合并 $3+3=6$，剩下 $4,5,6$；继续合并 $4+5=9$，最后合并 $6+9=15$。因此 WPL 为 $4+6+9+15=34$。即使相同权值的合并顺序有不同，最小 WPL 不变。



## 第10题

下面程序的运行结果为（ ）。

```cpp
#include <iostream>
using namespace std;
int f(int n) {
    if (n <= 2) return n * 2;
    return f(n - 1) + f(n - 2);
}
int main() {
    cout << f(5) << endl;
    return 0;
}
```

A. 10
B. 16
C. 26
D. 30

答案：B

解析：递归边界是 `n <= 2` 时返回 `n * 2`，所以 $f(1)=2$，$f(2)=4$。继续向上计算：$f(3)=f(2)+f(1)=4+2=6$，$f(4)=f(3)+f(2)=6+4=10$，$f(5)=f(4)+f(3)=10+6=16$。程序输出 `16`。



## 第11题

一个简单无向图 $G$ 有 $36$ 条边，且每个顶点的度数都为 $4$，则图 $G$ 的顶点个数为（ ）。

A. 9
B. 12
C. 18
D. 36

答案：C

解析：无向图中每条边会给两个端点各贡献 $1$ 度，所以所有顶点度数之和等于 $2E$。本题 $E=36$，度数总和为 $72$。每个顶点的度数都是 $4$，若顶点数为 $V$，则 $4V=72$，解得 $V=18$。



## 第12题

下面关于二叉树的说法正确的是（ ）。

A. 任意二叉树的中序遍历与后序遍历必定不相同。
B. 对任意二叉树，若已知先序遍历与后序遍历，则该二叉树唯一确定。
C. 若二叉树有 $n$ 个结点，根节点高度为 $1$，则其高度满足：$\lceil \log_2(n+1) \rceil \le h \le n$。
D. 在二叉树的先序遍历中，根后紧跟的结点一定是根的左孩子。

答案：C

解析：高度为 $h$ 的二叉树最多有 $2^h-1$ 个结点，因此有 $n \le 2^h-1$，推出 $h \ge \lceil \log_2(n+1) \rceil$；另一方面，二叉树最“瘦”时可以退化成一条链，高度最大为 $n$，所以 C 正确。A 可用单结点树反例，中序和后序都只有这个结点；B 错在任意二叉树仅由先序和后序不能唯一确定，例如根只有一个孩子时，孩子在左或在右会得到相同的先序、后序；D 错在根没有左孩子但有右孩子时，先序中根后面的结点是右孩子。



## 第13题

假设一个算法时间复杂度的递推式是 $T(n) = 8T(\frac{n}{4}) + n\sqrt{n}$（$n$ 为正整数），和 $T(0) = 1$，那么这个算法的时间复杂度是（ ）。

A. $O(n\sqrt{n})$
B. $O(n\sqrt{n} \log n)$
C. $O(n^2)$
D. $O(n^2 \log n)$

答案：B

解析：套用主定理，递推式中 $a=8$，$b=4$，所以子问题部分对应的指数为 $\log_4 8=\frac{3}{2}$，即 $n^{\log_4 8}=n^{3/2}=n\sqrt n$。非递归部分也是 $n\sqrt n$，与该量级相同，属于主定理第二种情况，需要再乘一个 $\log n$ 因子。因此时间复杂度为 $O(n\sqrt n\log n)$。



## 第14题

下面哪一个可能是下图的深度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/6q1m0uo5.png)

A. 1, 5, 6, 3, 2, 8, 9, 4, 7
B. 1, 5, 8, 9, 7, 4, 6, 3, 2
C. 3, 2, 1, 4, 7, 6, 9, 5, 8
D. 2, 5, 6, 3, 8, 7, 9, 4, 1

答案：B

解析：深度优先遍历的访问顺序取决于起点和每个点选择邻接点的顺序，只要每一步沿着尚未访问的出边尽量深入即可。对图中有向边，可以从 `1` 出发，先走 `1 -> 5 -> 8 -> 9`；`9` 没有可继续深入的未访问出边后回溯到 `8`，再访问 `7`、`4`；回到 `5` 后再走 `5 -> 6 -> 3 -> 2`。这样得到的首次访问序列正是 `1, 5, 8, 9, 7, 4, 6, 3, 2`，所以 B 可能是该图的一次深度优先遍历序列。



## 第15题

下面这个有向图的强连通分量的个数是（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/o4h0ic38.png)

A. 3
B. 4
C. 5
D. 6

答案：C

解析：强连通分量要求分量内任意两个点都能互相到达。图中 `1` 和 `2` 之间有双向可达关系，形成 `{1,2}`；左下部分 `4,5,7,8` 之间可通过 `4 -> 7 -> 8 -> 5 -> 4` 等路径互相到达，形成 `{4,5,7,8}`；`3` 能走向右侧和下方，但没有路径回到 `3`，所以 `{3}` 单独成分量；右侧 `6,9,10,11` 可以经 `6 -> 10 -> 9 -> 11 -> 6` 互相到达，形成 `{6,9,10,11}`；`12` 只有向外到 `9、11` 的边，没有从其他点回到 `12` 的路径，所以 `{12}` 单独成分量。共 $5$ 个强连通分量。
