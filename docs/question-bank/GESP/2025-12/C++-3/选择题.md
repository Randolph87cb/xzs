## 第1题

请将二进制数 $(1101.101)_2$，转换为等值的 $10$ 进制数（ ）。

【知识点：进制转换与进制字面量】

A. $14.25$
B. $13.625$
C. $13.5$
D. $14.5$

答案：B

解析：二进制整数部分按位权展开：$(1101)_2 = 1 \times 2^3 + 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 = 13$。小数部分从小数点后第一位开始，位权依次是 $2^{-1}, 2^{-2}, 2^{-3}$，所以 $(.101)_2 = 1 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3} = 0.5 + 0.125 = 0.625$。合起来是 $13.625$，选 B。



## 第2题

以下关于标准 C++ 一维数组的描述，哪一项是错误的？（ ）

【知识点：数组与字符数组】

A. 数组的下标从 $0$ 开始
B. 数组在内存中是连续存放的
C. 数组的长度只能在运行时才能确定
D. 可以在程序中修改数组某个元素的值

答案：C

解析：标准 C++ 的普通一维数组下标从 $0$ 开始，元素在内存中连续存放，某个元素也可以通过下标重新赋值，例如 `a[2] = 10`。易错点在数组长度：普通数组定义时就要确定元素个数，自动存储期数组的长度不能依赖运行时输入；如果要在运行时决定规模，应使用动态分配或 `vector` 等容器。因此“数组的长度只能在运行时才能确定”是错误描述，选 C。



## 第3题

下列代码段执行后，变量 `sum` 的值是（ ）

```cpp
{
    int sum = 0;
    for (int i = 1; i <= 10; i++) {
        if (i % 3 == 8 >> 2)
            continue;
        sum += i;
    }
}
```

【知识点：位运算】

A. $12$
B. $17$
C. $33$
D. $40$

答案：D

解析：表达式 `i % 3 == 8 >> 2` 中，取模 `%` 先算，右移 `>>` 的优先级高于相等比较 `==`，所以判断等价于 `(i % 3) == (8 >> 2)`。`8 >> 2` 表示二进制右移两位，结果为 $2$。循环从 $1$ 到 $10$，当 $i \bmod 3 = 2$ 时跳过，即跳过 $2,5,8$。$1+2+\cdots+10=55$，减去跳过的 $2+5+8=15$，得到 `sum = 40`，选 D。



## 第4题

执行以下程序段，输出值是（ ）

```cpp
int x = 5;
if (x == 3 >> 2) x = 8 >> 3;
cout << x << endl;
```

【知识点：位运算】

A. $0$
B. $5$
C. $120$
D. $1$

答案：B

解析：先看条件 `x == 3 >> 2`。右移运算先于相等比较计算，`3 >> 2` 等于 $0$，因此条件实际是 `x == 0`。初始 `x = 5`，条件为假，`if` 后面的赋值 `x = 8 >> 3` 不会执行。最后输出的仍是 $5$，选 B。



## 第5题

已知字符串 `s = "C++ Programming"`，下列程序运行的结果是（ ）

```cpp
cout << (int)(s.length() | 3) << endl;
```

【知识点：位运算】

A. $14$
B. $15$
C. $16$
D. $17$

答案：B

解析：字符串 `"C++ Programming"` 的长度包含中间的空格：`C++` 有 $3$ 个字符，空格 $1$ 个，`Programming` 有 $11$ 个字符，总长度为 $15$。按位或运算中，$15$ 的二进制低位已经全为 $1$，与 $3$ 做或运算仍为 $15$，即 `15 | 3 = 15`。强制转换为 `int` 后输出 $15$，选 B。



## 第6题

以下代码是一个程序的部分代码，能够正确执行，输出的结果是（ ）

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int p = arr[1];
cout << (p + 2) << endl;
```

【知识点：数组与字符数组】

A. $1$
B. $2$
C. $3$
D. $4$

答案：D

解析：数组下标从 $0$ 开始，`arr[0]` 是 $1$，`arr[1]` 是 $2$。语句 `int p = arr[1];` 将 $2$ 赋给 `p`，随后输出的是 `p + 2`，也就是 $2 + 2 = 4$。这里没有指针移动或数组整体变化，只是读取一个元素的值，选 D。



## 第7题

以下关于 C++ 求最小值函数 `min()` 的描述，不正确的是（ ）

【知识点：枚举与基础算法】

A. 函数必须返回一个值
B. 该函数可以嵌套调用
C. 调用 `min(5)` 是错误的，不能通过编译
D. 调用 `min()`（不传任何参数）可以通过编译，结果为 $0$

答案：D

解析：标准库中的 `min` 用来返回较小值，常见调用形式需要提供两个可比较的参数，例如 `min(3, 5)`；也可以把一次调用结果作为另一次调用的参数，例如 `min(min(a, b), c)`。`min(5)` 只有一个参数，无法匹配常见的二元 `min`；`min()` 没有参数，也没有“默认返回 $0$”的规则，不能据此得到结果。因此不正确的描述是 D。



## 第8题

在一个特定的计算机系统中，假如 `unsigned int` 类型需要占用 $2$ 个字节的存储空间（每个字节有 $8$ 位），则 `unsigned int` 可以表示的数据范围是（ ）

【知识点：位运算】

A. 0 ~ 65535
B. 0 ~ 65536
C. -65536 ~ 65535
D. 0 ~ 32767

答案：A

解析：题目给定 `unsigned int` 占 $2$ 个字节，每个字节 $8$ 位，所以一共有 $16$ 位。无符号整数没有符号位，$16$ 位可以表示 $2^{16}=65536$ 个不同的值，从 $0$ 开始编号，最大值是 $2^{16}-1=65535$。因此范围是 $0 \sim 65535$，选 A。



## 第9题

以下代码执行后，数组 `arr` 的内容是（ ）

```cpp
int arr[6] = {1, 2, 3, 4, 5, 6};
for (int i = 0; i < 6; i += 2) {
    arr[i] = arr[i] + arr[i+1];
    arr[i+1] = arr[i] - arr[i+1];
    arr[i] = arr[i] - arr[i+1];
}
```

【知识点：数组与字符数组】

A. $\{2,1,4,3,6,5\}$
B. $\{1,2,3,4,5,6\}$
C. $\{3,1,4,2,5,3\}$
D. $\{1,3,2,5,4,6\}$

答案：A

解析：循环变量依次取 $0,2,4$，每次处理一对相邻元素。设处理前这两个数是 $a,b$：第一句把前一个变成 $a+b$，第二句把后一个变成 $(a+b)-b=a$，第三句把前一个变成 $(a+b)-a=b$，所以三句的作用是交换相邻两个元素。原数组三组相邻元素分别是 $(1,2),(3,4),(5,6)$，交换后得到 $\{2,1,4,3,6,5\}$，选 A。



## 第10题

`a & b | (c ^ d)`，其中 $a = 3$，$b = 7$，$c = 15$，$d = 4$，计算结果是（ ）

【知识点：位运算】

A. 十进制 $11$
B. 二进制 $11$
C. 八进制 $11$
D. 十六进制 $11$

答案：A

解析：先按括号计算 `c ^ d`：$15$ 的二进制是 `1111`，$4$ 是 `0100`，异或得到 `1011`，即十进制 $11$。再算 `a & b`：$3$ 是 `0011`，$7$ 是 `0111`，按位与得到 `0011`，即 $3$。最后 $3 | 11$，二进制为 `0011 | 1011 = 1011`，结果仍是十进制 $11$。表达式的数值结果是 $11$，选 A；“二进制 11”“八进制 11”“十六进制 11”表示的数值分别不同，不能混同。



## 第11题

整型变量 $x$ 的初始值为 $10$，以下代码的输出结果是（ ）

```cpp
int t = x--;
t -= x;
cout << t << endl;
```

【知识点：控制结构与程序阅读】

A. $0$
B. $-1$
C. $1$
D. 死循环

答案：C

解析：`x--` 是后置自减，先取出当前值参与赋值，再把 `x` 减 $1$。因此 `int t = x--;` 执行后，`t = 10`，`x = 9`。接着 `t -= x` 等价于 `t = t - x`，所以 `t = 10 - 9 = 1`。最终输出 $1$，选 C。

## 第13题

以下代码运行后，`sum` 的结果是（ ）

```cpp
int arr[5] = {2, 4, 6, 8, 10};
int sum = 0;
for (int i = 0; i < 5; i++) {
    switch (arr[i] % (1|2)) {
        case 0: sum += 1; break;
        case 1: sum += 2; break;
        case 2: sum += 3; break;
    }
}
```

【知识点：位运算】

A. $10$
B. $11$
C. $14$
D. $15$

答案：B

解析：先计算 `1 | 2`，二进制 `01 | 10 = 11`，结果为 $3$，所以 `switch` 判断的是 `arr[i] % 3`。五个元素的余数依次为：$2 \bmod 3 = 2$，加 $3$；$4 \bmod 3 = 1$，加 $2$；$6 \bmod 3 = 0$，加 $1$；$8 \bmod 3 = 2$，加 $3$；$10 \bmod 3 = 1$，加 $2$。总和为 $3+2+1+3+2=11$，选 B。



## 第14题

以下关于 C++ 中 `abs()` 函数的描述，正确的是（ ）

【知识点：函数与标准库】

A. `abs()` 函数可用于计算 `int` 类型整数的绝对值，头文件为 `<cmath>` 或 `<cstdlib>`
B. 调用 `abs(3.14)` 可以四舍五入得到 $3$
C. `abs(-1, -2)` 的返回值是 $1$
D. 若传入负数浮点数，`abs()` 会自动截断小数部分后返回整数绝对值

答案：A

解析：`abs()` 的作用是求绝对值，常用于 `int` 等整数类型，相关声明可来自 `<cstdlib>` 或 `<cmath>`。它不是四舍五入函数，`abs(3.14)` 不表示把 $3.14$ 四舍五入成 $3$；`abs(-1, -2)` 参数个数也不符合一元绝对值函数的调用方式；对浮点数求绝对值时应理解为保留数值大小的绝对值，不是先截断小数再返回整数。因此正确描述是 A。



## 第15题

如果字符串 `s` 的值是 `GESP`，以下代码 `s` 的最后结果是（ ）

```cpp
for (int i = 0; i < s.length(); i++) {
    s[i] = toupper(s[i] + i);
}
```

【知识点：字符串处理】

A. `GESP`
B. `HFTR`
C. `hesp`
D. `GFUS`

答案：D

解析：循环中 `i` 从 $0$ 到 $3$，每次把当前字符的编码值加上 `i`，再转成大写。原字符串是 `GESP`：`i = 0` 时 `G + 0` 仍为 `G`；`i = 1` 时 `E + 1` 变为 `F`；`i = 2` 时 `S + 2` 变为 `U`；`i = 3` 时 `P + 3` 变为 `S`。这些结果本身都是大写字母，`toupper` 不再改变它们，所以最终字符串为 `GFUS`，选 D。
