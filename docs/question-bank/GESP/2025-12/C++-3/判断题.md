## 第1题

在 C++ 中，数组名不能改变，数组名不能被赋值。

【知识点：数组与字符数组】

答案：正确

解析：数组名在很多表达式中会转换成指向首元素的指针，但数组名本身不是可以重新赋值的变量。定义 `int a[5];` 后，不能写 `a = other;` 让它指向另一个数组，也不能改变这个数组名代表的存储位置。可以修改的是数组元素，例如 `a[0] = 10`。所以题干判断正确。



## 第2题

整型变量 a 的值为 3，浮点数变量 b 的值为 3.5，在 C++ 编译环境下，经过 `a *= b` 计算后，a 的值变为 10.5。此时如果输出 a，将输出 10.5。

【知识点：C++语法规则与编译模型】

答案：错误

解析：`a *= b` 是复合赋值，含义类似于先计算 `a * b`，再把结果赋回 `a`。这里 `3 * 3.5 = 10.5`，但 `a` 的类型是 `int`，赋回时会向整数转换，结果为 $10$，小数部分不会保留下来。输出 `a` 时输出的是整数 $10$，不是 $10.5$，所以题干判断错误。



## 第3题

`strcmp(str1, str2)` 返回 0 表示 str1 大于 str2，返回正数表示两者相等。

【知识点：字符串处理】

答案：错误

解析：`strcmp(str1, str2)` 比较的是两个 C 风格字符串的字典序。返回 $0$ 表示两个字符串相等；返回负数表示 `str1` 小于 `str2`；返回正数表示 `str1` 大于 `str2`。题干把“返回 $0$”和“返回正数”的含义说反了，所以判断错误。



## 第4题

以下代码输出结果为 8。

```cpp
int a = 5, b = 3;
int c = a++ + ++b;
cout << c << endl;
```

【知识点：控制结构与程序阅读】

答案：错误

解析：`a++` 是后置自增，表达式中先使用原值 $5$，然后 `a` 变为 $6$；`++b` 是前置自增，先把 `b` 从 $3$ 增加到 $4$，再使用 $4$。因此 `c = a++ + ++b` 得到的是 $5 + 4 = 9$，输出 $9$，不是 $8$。所以题干判断错误。



## 第5题

位运算符 `&`、`|`、`^`、`~` 的优先级高于算术运算符 `+`、`-`、`*`、`/`。

【知识点：位运算】

答案：错误

解析：在 C++ 运算符优先级中，`*`、`/` 高于 `+`、`-`，而二元位运算符 `&`、`^`、`|` 的优先级低于加减运算。虽然一元按位取反 `~` 的优先级较高，但题干把 `&`、`|`、`^`、`~` 全部说成高于算术运算符，这个整体表述不成立。所以判断错误。



## 第6题

在 C++ 中，`int a[] = {1, 2, 3, 4, 5};` 可以定义一个包含 5 个元素的整型数组。

【知识点：数组与字符数组】

答案：正确

解析：数组定义 `int a[] = {1, 2, 3, 4, 5};` 没有显式写长度时，编译器会根据初始化列表中的元素个数推断数组长度。初始化列表里有 $5$ 个整数，所以定义出的数组包含 $5$ 个 `int` 元素，合法且题干判断正确。



## 第7题

C++ 表达式 `z = a > b ? x : y` 等同于：

```cpp
if (a > b) {
    z = x;
} else {
    z = y;
}
```

以下代码执行后，输出将会是 2。

```cpp
int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    cout << arr[3 > 2 ? 1 : 2] << endl;
    return 0;
}
```

【知识点：数组与字符数组】

答案：正确

解析：条件运算符 `条件 ? 表达式1 : 表达式2` 会在条件为真时取表达式1，为假时取表达式2。这里 `3 > 2` 为真，所以数组下标表达式 `3 > 2 ? 1 : 2` 的结果是 $1$。数组下标从 $0$ 开始，`arr[1]` 的值是 $2$，因此程序输出 $2$，题干判断正确。



## 第8题

在 C++ 中，`++` 和 `--` 运算符只能用于整型变量，不能用于浮点数变量。

【知识点：运算符、表达式与类型转换】

答案：错误

解析：`++` 和 `--` 要求操作对象是可修改的变量，并不只限于整型。浮点变量同样可以自增或自减，例如 `double x = 1.5; x++;` 后 `x` 变为 $2.5$。它们也常用于指针移动。题干说只能用于整型变量，范围过窄，所以判断错误。



## 第9题

给定一个正整数 a，当需要计算 -a 的补码时，有这样一个计算技巧：将 a 的二进制形式从右往左扫描，遇到第一个 1 之后，将找到的第一个 1 左边的所有位都取反，能得到 -a 的补码。

【知识点：原码反码补码与整数表示】

答案：正确

解析：在固定二进制位数下，求 $-a$ 的补码可以看作“按位取反再加 $1$”。从右往左扫描正数 $a$ 的二进制表示时，最右侧第一个 $1$ 以及它右边的所有位在这个过程中保持不变，而这个 $1$ 左边的位全部取反。这正是题干描述的技巧，因此判断正确。使用这个技巧时要默认位数已经固定，例如按 $8$ 位、$16$ 位或 $32$ 位表示。



## 第10题

以下代码中，Hello 将被输出 5 次。

```cpp
for (int i = 0; i < 5; i++);
{
    cout << "Hello" << endl;
}
```

【知识点：控制结构与程序阅读】

答案：错误

解析：`for (int i = 0; i < 5; i++);` 末尾的分号表示循环体是一条空语句，循环本身只是在空转 $5$ 次。后面的花括号代码块并不属于 `for` 循环体，而是在循环结束后顺序执行一次。因此 `Hello` 只会输出 $1$ 次，不是 $5$ 次，题干判断错误。
