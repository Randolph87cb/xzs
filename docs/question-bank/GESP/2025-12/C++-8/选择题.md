## 第1题

某平台生成“取件码”由 $6$ 个字符组成：前 $4$ 位为数字（$0-9$），后 $2$ 位为大写字母（$A-Z$），其中字母不能为 $I$、$O$。假设数字和字母均可重复使用，要求整个取件码中恰好有 $2$ 个数字为奇数。共有多少种不同取件码？（ ）

A. 1,440,000
B. 2,160,000
C. 2,535,000
D. 8,640,000

答案：B

解析：前 $4$ 位数字中恰好 $2$ 个为奇数，先选出奇数所在位置，有 $\binom{4}{2}=6$ 种。每个奇数位可选 $1,3,5,7,9$ 共 $5$ 种，每个偶数位可选 $0,2,4,6,8$ 共 $5$ 种，所以数字部分为 $6 \times 5^2 \times 5^2$ 种。后 $2$ 位字母不能为 $I,O$，每位有 $26-2=24$ 种且可重复，因此共有 $6 \times 5^4 \times 24^2=2,160,000$ 种。



## 第2题

下列代码实现了归并排序（Merge Sort）的分治部分。为了正确地将数组 `a` 的 `[left, right]` 区间进行排序，横线处应该填入的是（ ）。

```cpp
void merge_sort(int a[], int left, int right) {
    if (left >= right) return;
    int mid = (left + right) / 2;
    merge_sort(a, left, mid);
    _______; // 在此处填入选项
    merge(a, left, mid, right); // 合并操作
}
```

A. `merge_sort(a, mid, right)`
B. `merge_sort(a, mid + 1, right)`
C. `merge_sort(a, left, mid + 1)`
D. `merge_sort(a, mid - 1, right)`

答案：B

解析：归并排序要把区间 $[left,right]$ 分成互不重叠且覆盖完整的两段。第一段已经递归排序为 $[left,mid]$，第二段就应是 $[mid+1,right]$，然后 `merge(a, left, mid, right)` 才能把这两段有序区间合并。若从 `mid` 开始，会和左半段重叠；若到 `mid + 1` 或从 `mid - 1` 开始，则会造成区间划分错误，甚至在小区间上无法缩小递归范围。



## 第3题

某社团有男生 $8$ 人、女生 $7$ 人。现需选出 $1$ 名队长（性别不限）、$1$ 名副队长（性别不限）、$2$ 名宣传委员（两人无角色区别，且必须至少 $1$ 名女生）。假如一人不能兼任多职，共有多少种不同选法？（ ）

A. 12012
B. 11844
C. 12474
D. 11025

答案：A

解析：可以先选 $2$ 名无角色区别的宣传委员，再安排有角色区别的队长和副队长。宣传委员至少 $1$ 名女生，分两类：选 $1$ 女 $1$ 男有 $\binom{7}{1}\binom{8}{1}=56$ 种，选 $2$ 女有 $\binom{7}{2}=21$ 种。选定宣传委员后还剩 $13$ 人，队长和副队长有先后角色，共 $13 \times 12=156$ 种安排。因此总数为 $(56+21)\times 156=12012$。



## 第4题

二项式 $(2x - y)^8$ 的展开式中 $x^5y^3$ 项的系数为（ ）。

A. -7168
B. 7168
C. -1792
D. 1792

答案：C

解析：展开 $(2x-y)^8$ 时，要得到 $x^5y^3$，需要从 $8$ 个因子中选 $3$ 个取 $-y$，其余 $5$ 个取 $2x$。系数为 $\binom{8}{3}\cdot 2^5 \cdot (-1)^3=56 \times 32 \times (-1)=-1792$。易错点是 $(-y)^3$ 带来负号，不能只计算绝对值 $1792$。



## 第5题

下面是使用邻接矩阵实现的 Dijkstra 算法的核心片段，用于求单源最短路径。在找到当前距离起点最近的顶点 $u$ 后，需要更新其邻接点 $j$ 的距离。横线处应填入的代码是（ ）。

```cpp
for (int j = 1; j <= n; j++) {
    if (!visited[j] && graph[u][j] < INF) {
        if (________) { // 在此处填入选项
            dis[j] = dis[u] + graph[u][j];
        }
    }
}
```

A. `dis[j] < dis[u] + graph[u][j]`
B. `dis[j] > dis[u] + graph[u][j]`
C. `graph[u][j] > dis[u] + dis[j]`
D. `dis[j] > graph[u][j]`

答案：B

解析：`dis[j]` 表示当前已知的起点到 $j$ 的最短距离上界。处理顶点 $u$ 时，若走“起点到 $u$ 的最短路”再接边 $(u,j)$，得到的新距离 `dis[u] + graph[u][j]` 更小，就应该把 `dis[j]` 改成这个值。因此判断条件应为 `dis[j] > dis[u] + graph[u][j]`。方向写反会在路径更长时更新，`dis[j] > graph[u][j]` 又漏掉了起点到 $u$ 的距离。



## 第6题

下面程序使用动态规划求两个字符串的最长公共子序列（LCS）长度，横线处应填入的是（ ）。

```cpp
#include <algorithm>
#include <string>
#include <vector>
using namespace std;

int lcs_len(const string &a, const string &b) {
    int n = (int)a.size(), m = (int)b.size();
    vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (a[i - 1] == b[j - 1])
                dp[i][j] = dp[i - 1][j - 1] + 1;
            else
                ________; // 在此处填入选项
    return dp[n][m];
}
```

A. `dp[i][j] = dp[i - 1][j] + dp[i][j - 1];`
B. `dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]);`
C. `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);`
D. `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]) + 1;`

答案：C

解析：`dp[i][j]` 表示 `a` 的前 $i$ 个字符与 `b` 的前 $j$ 个字符的 LCS 长度。若 `a[i - 1] == b[j - 1]`，这两个字符可以一起放入公共子序列，所以从左上角转移并加 $1$。若二者不相等，最后一个字符不可能同时使用，只能在“舍去 `a` 的第 $i$ 个字符”和“舍去 `b` 的第 $j$ 个字符”两种情况中取较大值，即 `max(dp[i - 1][j], dp[i][j - 1])`。



## 第7题

已知两个点 $A(x_1, y_1)$ 和 $B(x_2, y_2)$ 在平面直角坐标系中的坐标。下列 C++ 表达式中，能正确计算这两点之间直线距离的是（ ）。

A. `sqrt((x1 - x2) ^ 2 + (y1 - y2) ^ 2)`
B. `sqrt(pow(x1 - x2, 2) + pow(y1 - y2, 2))`
C. `pow(x1 - x2, 2) + pow(y1 - y2, 2)`
D. `abs(x1 - x2) + abs(y1 - y2)`

答案：B

解析：平面两点距离公式是 $\sqrt{(x_1-x_2)^2+(y_1-y_2)^2}$，在 C++ 中可以用 `pow` 计算平方，再用 `sqrt` 开方。选项 A 的 `^` 不是乘方运算，而是按位异或，不能表示平方；选项 C 少了外层开方，得到的是距离的平方；选项 D 是曼哈顿距离 $|x_1-x_2|+|y_1-y_2|$，不是直线距离。



## 第8题

已知 `int a = 10;`，执行 `int &b = a; b = 20;` 后，变量 `a` 的值是（ ）。

A. 10
B. 20
C. 30
D. 编译错误

答案：B

解析：`int &b = a;` 声明的是引用，`b` 成为变量 `a` 的别名，不会新建一个独立的 `int` 对象。执行 `b = 20;` 时，实际就是给 `a` 赋值为 $20$。引用必须在定义时初始化，初始化后也不能再改绑到别的变量。



## 第9题

下列代码的时间复杂度（以 $n$ 为自变量，忽略常数与低阶项）是（ ）。

```cpp
long long s = 0;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j * j <= i; j++) {
        s += j;
    }
}
```

A. $O(n)$
B. $O(n \log n)$
C. $O(n \sqrt{n})$
D. $O(n^2)$

答案：C

解析：外层循环执行 $n$ 次。固定某个 $i$ 时，内层条件是 `j * j <= i`，所以 `j` 最多到 $\lfloor\sqrt{i}\rfloor$，总次数为 $\sum_{i=1}^{n}\lfloor\sqrt{i}\rfloor$。上界可看作每项不超过 $\sqrt{n}$，得到 $O(n\sqrt{n})$；当 $i>n/4$ 时，每项至少约为 $\sqrt{n}/2$，又有 $\Omega(n\sqrt{n})$ 次。因此忽略常数后复杂度为 $O(n\sqrt{n})$。



## 第10题

下列程序实现了线性筛法（欧拉筛），用于在 $O(n)$ 时间内求出 $1 \sim n$ 之间的所有质数。为了保证每个合数只被其最小质因子筛掉，横线处应填入的语句是（ ）。

```cpp
for (int i = 2; i <= n; i++) {
    if (!not_prime[i]) primes[++cnt] = i;
    for (int j = 1; j <= cnt && i * primes[j] <= n; j++) {
        not_prime[i * primes[j]] = true;
        if (________) break; // 在此处填入选项
    }
}
```

A. `i + primes[j] == n`
B. `primes[j] > i`
C. `i % primes[j] == 0`
D. `i % primes[j] != 0`

答案：C

解析：线性筛的关键是不让同一个合数被多个质因子重复筛。枚举 `primes[j]` 标记 `i * primes[j]` 后，如果 `primes[j]` 已经是 `i` 的一个质因子，也就是 `i % primes[j] == 0`，那么它就是 `i * primes[j]` 的最小质因子；继续使用更大的质数去乘 `i`，会让得到的合数应该由更小质因子在别处筛掉，从而破坏“每个合数只被最小质因子筛一次”的性质，所以此时要 `break`。



## 第11题

在 C++ 语言中，关于类的继承和访问权限，下列说法正确的是（ ）。

A. 派生类可以访问基类的 `private` 成员。
B. 基类的 `protected` 成员在私有继承（private inheritance）后，在派生类中变为 `public`。
C. 派生类对象在创建时，会先调用基类的构造函数，再调用派生类自己的构造函数。
D. 派生类对象在销毁时，会先调用基类的析构函数，再调用派生类自己的析构函数。

答案：C

解析：创建派生类对象时，必须先把它内部的基类子对象构造好，派生类自己的构造函数才能使用这部分基础结构，所以调用顺序是先基类构造函数，再派生类构造函数。基类的 `private` 成员不能被派生类直接访问；`protected` 成员经过私有继承后对外访问级别会降低为私有继承下的成员访问，不会变成 `public`；销毁顺序与构造相反，先执行派生类析构函数，再执行基类析构函数。



## 第12题

当输入 $6$ 时，下列程序的输出结果为（ ）。

```cpp
#include <iostream>
using namespace std;

int f(int n) {
    if (n <= 3) return n;
    return f(n - 1) + f(n - 2) + 2 * f(n - 3);
}

int main() {
    int n;
    cin >> n;
    cout << f(n) << endl;
    return 0;
}
```

A. 14
B. 27
C. 28
D. 15

答案：B

解析：递归函数在 $n \le 3$ 时直接返回 $n$，所以 $f(1)=1$、$f(2)=2$、$f(3)=3$。继续计算：$f(4)=f(3)+f(2)+2f(1)=3+2+2=7$，$f(5)=f(4)+f(3)+2f(2)=7+3+4=14$，$f(6)=f(5)+f(4)+2f(3)=14+7+6=27$。输入为 $6$，程序输出 `27`。



## 第13题

从 $1$ 到 $999$ 这 $999$ 个正整数中，十进制表示中数字 $5$ 恰好出现一次的数有多少个？（ ）

A. 243
B. 271
C. 300
D. 333

答案：A

解析：把 $1$ 到 $999$ 统一看成三位数，不足三位的在前面补 $0$，例如 $7$ 看作 `007`。这样不会额外引入数字 $5$，也不会遗漏任何原数。恰好一个位置为 $5$ 有 $3$ 种选择，另外两个位置都不能为 $5$，各有 $9$ 种数字可选（$0$ 到 $9$ 中排除 $5$）。因此数量为 $3 \times 9 \times 9=243$。



## 第14题

当输入 $2023$ 时，下列程序的输出结果为（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int x, ans = 0;
    cin >> x;
    while (x != 0) {
        x -= x & -x;
        ans++;
    }
    cout << ans << endl;
    return 0;
}
```

A. 7
B. 8
C. 9
D. 11

答案：C

解析：表达式 `x & -x` 会取出 `x` 的二进制表示中最低位的 $1$ 所对应的值，循环每次减去它，相当于每轮删除一个二进制 $1$，因此 `ans` 最终等于 $2023$ 的二进制中 $1$ 的个数。$2023=1024+512+256+128+64+32+4+2+1$，共有 $9$ 个 $1$，所以输出为 `9`。



## 第15题

对连通无向图执行 Kruskal 算法。已按边权从小到大依次扫描到某条边 $e = (u, v)$。此时在已经构建的部分 MST 结构中，$(u, v)$ 已在同一连通块内。关于边 $e$ 的处理，下列说法正确的是（ ）。

A. 必须选入 MST，否则可能不连通。
B. 一定不能选入 MST（在此扫描顺序下）。
C. 若后续出现更大的边权，可以回溯改选 $e$。
D. 只有当 $e$ 是当前最小时才能舍弃。

答案：B

解析：Kruskal 按边权从小到大扫描边，并用并查集维护当前已选边形成的连通块。若扫描到边 $e=(u,v)$ 时，$u$ 与 $v$ 已经在同一连通块内，说明当前已选边中已经存在一条从 $u$ 到 $v$ 的路径；再加入 $e$ 会形成环。最小生成树不能包含环，所以在这次扫描顺序下应直接舍弃这条边，后续也不需要回溯把它加回来。
