## 第1题

鸿蒙是华为公司开发的一款操作系统，那么它能够将正确的源程序翻译成目标程序，并运行。（ ）

答案：错误

解析：鸿蒙是操作系统，主要负责管理硬件资源、运行应用程序和提供系统服务；把源程序翻译成目标程序通常是编译器的工作。题干把操作系统和编译器的作用混在一起，说鸿蒙能够将源程序翻译成目标程序并运行，这一表述不准确。因此判断为错误。



## 第2题

C++ 表达式 `5 < 10 && 20` 对应的逻辑值为 `true`。（ ）

答案：正确

解析：关系运算符 `<` 的优先级高于逻辑与 `&&`，所以表达式先计算 `5 < 10`，结果为 `true`。在 C++ 中，非 0 数值在逻辑判断中视为 `true`，因此 `20` 也视为 `true`。`true && true` 的结果为 `true`，所以题干正确。



## 第3题

C++ 表达式 `10 / 0.333333 == 10 / (1 / 3)` 的值为 `true`。（ ）

答案：错误

解析：左边 `10 / 0.333333` 是浮点除法，结果大约为 30.00003。右边括号内的 `1 / 3` 是两个整数相除，结果是整数 0；再计算 `10 / 0` 会发生除以 0 的错误，不会得到一个可以正常比较且等于左边的值。因此表达式的值不能说是 `true`，题干错误。



## 第4题

下面 C++ 代码中 $N$ 是整数，执行时无论输入负整数、0 或正整数，其输出都将是 0。（ ）

```cpp
cin >> N;
while (N)
    N /= 10;
cout << N;
```

答案：正确

解析：`while (N)` 表示只要 `N` 不为 0 就继续循环。循环体中执行整数除法 `N /= 10`，无论 `N` 是正整数还是负整数，反复除以 10 后绝对值都会变小，最终变成 0；如果一开始输入 0，则循环不执行，`N` 本来就是 0。最后输出的都是 0，所以题干正确。



## 第5题

下面的 C++ 代码执行后，其输出是 `4 0`。（ ）

```cpp
int a, b;
a = 4;
b = a == 5;
cout << a << ' ' << b;
```

答案：正确

解析：先执行 `a = 4`，所以 `a` 的值为 4。语句 `b = a == 5;` 中，比较运算 `a == 5` 的结果为假，因为 4 不等于 5；假在输出整数时通常表现为 0，因此 `b` 得到 0。最后按 `a`、空格、`b` 的顺序输出，即 `4 0`，题干正确。



## 第6题

C++ 代码中对表达式 `('Z' - 'A') < ('z' - 'A')` 的结果输出为 0。（ ）

答案：错误

解析：在 ASCII 编码中，`'A'`、`'Z'`、`'z'` 的编码值分别为 65、90、122，所以 `'Z' - 'A' = 25`，`'z' - 'A' = 57`。比较 `25 < 57` 的结果为真，作为整数输出时应为 1，而不是 0。因此题干错误。



## 第7题

下面的 C++ 代码可以用于判断正整数 $N$ 的位数（即几位数，如 123 是 3 位数，12 为 2 位数）。（ ）

```cpp
int N, N10, i;
cin >> N;
N10 = 10, i = 1;
while (1) {
    if (N % N10 == N) {
        printf("%d 是 %d 位数", N, i);
        break;
    }
    i++, N10 *= 10;
}
```

答案：正确

解析：程序用 `N10` 表示当前的 10、100、1000 等界限，用 `i` 表示位数。判断 `N % N10 == N` 的含义是：如果 `N` 小于当前界限，那么用 `N10` 取余后仍等于 `N`，说明已经找到位数。例如 `N=123` 时，`123 % 10` 不等于 123，`123 % 100` 不等于 123，`123 % 1000` 等于 123，此时 `i=3`，输出 3 位数。对正整数可以得到位数，所以题干正确。



## 第8题

计算交叉加减的结果，形如 $1 - 2 + 3 - 4 + 5 - \cdots$。下面 C++ 代码中的变量都是整型，则将 `Flag = -Flag` 改为 `Flag -= Flag` 效果相同。（ ）

```cpp
cin >> N;
Flag = -1;
tnt = 0;
for (i = 1; i < N + 1; i++) {
    Flag = -Flag;
    tnt += Flag * i;
}
cout << tnt;
```

答案：错误

解析：原代码中 `Flag = -Flag` 会让 `Flag` 在 `1` 和 `-1` 之间交替变化，从而实现 `+1 -2 +3 -4 ...` 的符号交替。若改成 `Flag -= Flag`，等价于 `Flag = Flag - Flag`，执行后 `Flag` 会变成 0，之后每一项 `Flag * i` 都是 0，无法实现交叉加减。因此两者效果不同，题干错误。



## 第9题

下列 C++ 代码段执行后将输出 55。（ ）

```cpp
int cnt = 0;
for (i = 0; i < 10; i++)
    for (j = i; j < 10; j++)
        cnt += 1;
cout << cnt;
```

答案：正确

解析：外层 `i` 从 0 到 9。内层 `j` 从 `i` 开始到 9，所以当 `i=0` 时执行 10 次，`i=1` 时执行 9 次，依此类推，`i=9` 时执行 1 次。`cnt` 增加的总次数是 `10 + 9 + 8 + ... + 1 = 55`，因此输出 55，题干正确。



## 第10题

下面 C++ 代码执行后输出如下，因为代码 `printf("\n")` 没有任何可读内容，删除不影响输出效果。（ ）

```
  1  2  3  4  5  6  7  8  9
  2  4  6  8 10 12 14 16 18
  3  6  9 12 15 18 21 24 27
  4  8 12 16 20 24 28 32 36
  5 10 15 20 25 30 35 40 45
  6 12 18 24 30 36 42 48 54
  7 14 21 28 35 42 49 56 63
  8 16 24 32 40 48 56 64 72
  9 18 27 36 45 54 63 72 81
```

```cpp
for (int i = 1; i < 10; i++) {
    for (int j = 1; j < 10; j++)
        printf("%3d", i * j);
    printf("\n");
}
```

答案：错误

解析：`printf("\n")` 输出的是换行符，虽然屏幕上看不到具体字符形状，但它会让下一行内容从新的一行开始。九九乘法表每完成一行内层循环后都需要换行，否则删除这句后所有数字会连续输出在同一行，版式与题干给出的多行表格不同。因此“删除不影响输出效果”是错误的。
