## 第1题

鸿蒙是华为公司开发的一款操作系统，那么它能够将正确的源程序翻译成目标程序，并运行。（ ）

答案：错误

解析：操作系统的主要作用是管理计算机硬件和软件资源，为程序运行提供环境。把正确的源程序翻译成目标程序，一般是编译器或解释器等开发工具完成的工作，不是操作系统本身的功能。鸿蒙是操作系统，所以题干把“操作系统”和“翻译源程序的工具”混在一起了，判断为错误。



## 第2题

C++ 表达式 `10 * 4 % 6` 和 `10 * 2 % 3` 的结果相同。（ ）

答案：错误

解析：两个表达式都要先做乘法，再做取余。`10 * 4 % 6 = 40 % 6 = 4`，而 `10 * 2 % 3 = 20 % 3 = 2`。两个结果分别是 4 和 2，并不相同，所以题干说法错误。



## 第3题

下面 C++ 代码执行后将输出 0。（ ）

```cpp
int i;
for (i = 0; i < 10; i++)
    if (i % 3 == 0)
        continue;
    else
        break;
cout << i;
```

答案：错误

解析：循环开始时 `i=0`，满足 `i % 3 == 0`，因此执行 `continue`，本轮结束后 `i++`，`i` 变成 1。下一轮 `i=1` 时，`i % 3 == 0` 不成立，执行 `else` 中的 `break` 退出循环。退出后输出当前的 `i`，也就是 1，不是 0，所以题干错误。



## 第4题

下面 C++ 代码中的 `i > -10; i--` 改为 `i < 10; i++` 执行结果相同。（ ）

```cpp
int tnt = 0;
for (int i = 0; i > -10; i--){
    if (i < 0)
        i = -i;
    tnt += i;
}
cout << tnt;
```

答案：错误

解析：原代码第一次循环 `i=0`，`tnt` 加 0，之后 `i--` 变成 -1；下一次进入循环后，因为 `i < 0`，代码把 `i` 改成 1，再加到 `tnt`，本轮结束 `i--` 又回到 0。这样会在 `0` 和 `-1` 相关状态之间反复进行，不能正常退出并输出结果。若改成 `i < 10; i++`，循环会从 0 加到 9 后退出，行为明显不同，因此题干说“执行结果相同”是错误的。



## 第5题

下面 C++ 代码执行后将输出 99，因为 `i < 100` 不包括 100，而是到 99。（ ）

```cpp
int cnt = 0;
for (int i = 0; i < 100; i++)
    cnt += 1;
cout << cnt;
```

答案：错误

解析：`i < 100` 的确表示 `i` 最大只取到 99，但循环次数要从 0 数到 99，一共有 100 个整数。每次循环 `cnt += 1`，所以 `cnt` 最终是 100。题干把“最后一个 `i` 是 99”和“循环次数是 99”混淆了，因此判断为错误。



## 第6题

下面的 C++ 代码能用于判断输入的正整数是否为对称数。所谓对称数是指从左到右和从右到左读该数，其值相同，$121$ 或 $414$ 等是对称数，而 $123$ 不是对称数。（ ）

```cpp
int n, new_number;
cout << "请输入正整数:";
cin >> n;

new_number = 0;
while (n != 0){
    new_number = new_number * 10 + n % 10;
    n /= 10;
}

if (n == new_number)
    cout << "对称数";
else
    cout << "非对称数";
```

答案：错误

解析：循环确实可以把原数的数字倒过来存入 `new_number`，例如输入 121 时，`new_number` 会变成 121。但是循环过程中 `n` 会不断执行 `n /= 10`，最后变成 0。后面判断的是 `if (n == new_number)`，也就是用已经变成 0 的 `n` 去和反转后的数比较，不能正确判断原来的输入是否对称。因此这段代码不能用于正确判断对称数，题干错误。



## 第7题

下面的 C++ 代码执行后将输出 $0$。（ ）

```cpp
int tnt = 0;
for (int i = -100; i < 100; i++)
    tnt += i;
cout << tnt;
```

答案：错误

解析：循环中 `i` 从 -100 一直取到 99，共 200 个数。可以把它们配对来看：`-99` 和 `99` 抵消，`-98` 和 `98` 抵消，一直到 `-1` 和 `1` 抵消，`0` 也不影响结果，最后还剩下一个 `-100`。所以总和是 -100，不是 0，题干错误。



## 第8题

某同学执行 C++ 代码时 `printf("%g\n", (3 + 3.1415926535));` 输出 $6.14159$ 而不是 $6.1415926535$，表明该同学的计算机存在故障需要重新安装 C++ 软件。（ ）

答案：错误

解析：`%g` 是 `printf` 的浮点数输出格式之一，默认会按一定有效数字位数输出，并不会把所有小数位都完整打印出来。`3 + 3.1415926535` 的数学结果约为 6.1415926535，使用 `%g` 输出成 `6.14159` 是正常的格式控制结果。它不能说明计算机故障，也不需要重新安装 C++ 软件，因此题干错误。



## 第9题

如果 $x$ 是 C++ 代码中的 `double` 型变量，则对 `cout << (x? 1227 : 12.27);` 编译时将报错，因为 $1227$ 和 $12.12$ 分别是 `int` 和 `double` 类型，导致识别混乱。（ ）

答案：错误

解析：条件运算符 `x ? 1227 : 12.27` 先把 `x` 作为条件判断，`double` 类型的 `x` 可以转换为真假值：0 表示假，非 0 表示真。后两个结果表达式一个是整数、一个是浮点数时，C++ 可以进行类型转换，把结果统一成合适的数值类型，不会因为类型不同就“识别混乱”并报错。题干后半句写的是 `12.12`，与表达式中的 `12.27` 不一致，但二者都是 `double` 字面量，不影响“不会因 `int`/`double` 混用而编译报错”的判断。因此题干说法错误。



## 第10题

在 C++ 代码中，不可以将变量命名为 `keyword`，因为它是 C++ 语言的关键字。（ ）

答案：错误

解析：`keyword` 这个单词的意思是“关键字”，但它本身并不是 C++ 语言规定的关键字。C++ 中不能用作变量名的是 `for`、`while`、`int` 等真正的保留关键字，而不是所有看起来像英文术语的单词。因此可以把变量命名为 `keyword`，题干说“不可以”是错误的。
