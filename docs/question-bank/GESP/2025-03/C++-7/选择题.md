## 第1题

下列哪个选项是 C++ 中的关键字？

A. `function`
B. `class`
C. `method`
D. `object`

答案：B

解析：C++ 的关键字是语言预先保留、不能作为普通标识符使用的单词，`class` 用来定义类，因此 B 正确。`function`、`method`、`object` 都是常见英文术语，在标准 C++ 中不是关键字，可以在不冲突的作用域里作为变量名或函数名使用。本题容易把面向对象概念和语言关键字混在一起，判断时应以 C++ 语法保留字为准。



## 第2题

下面代码输出的是（ ）

```cpp
int main() {
    int a = 5, b = 2;
    cout << (a >> b) << endl;
}
```

A. 1
B. 2
C. 5
D. 10

答案：A

解析：`>>` 是右移运算符，`a >> b` 表示把整数 `a` 的二进制表示向右移动 `b` 位。`5` 的二进制是 `101`，右移 `2` 位后变成 `1`，所以输出为 `1`。这里不是比较大小，也不是输入运算符；在表达式中它按位移动整数，不能把 `5 >> 2` 理解成除法写法，但对非负整数本例结果等同于 $\lfloor 5 / 2^2 \rfloor = 1$。



## 第3题

以下代码的输出是什么？

```cpp
int main() {
    int a = 10;
    int *p = &a;
    int *&q = p;
    *q = 20;
    cout << a << endl;
    return 0;
}
```

A. 10
B. 20
C. 地址值
D. 编译错误

答案：B

解析：`p` 是指向 `a` 的指针，`int *&q = p;` 表示 `q` 是指针变量 `p` 的引用，而不是一个新的普通指针副本。因此 `q` 和 `p` 指向同一个位置，即变量 `a`。执行 `*q = 20;` 时修改的是 `q` 所指向对象的值，也就是把 `a` 改为 `20`，最后输出 `20`。常见误区是只看到 `q` 是引用就以为输出地址；真正输出的是 `a`，不是 `p` 或 `q` 本身。



## 第4题

下面代码输出的是（ ）

```cpp
int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int *p = arr + 2;
    cout << *p << endl;
    return 0;
}
```

A. 1
B. 2
C. 3
D. 4

答案：C

解析：数组名 `arr` 在这里会转换为指向首元素 `arr[0]` 的指针。`arr + 2` 按元素个数做指针算术，指向 `arr[2]`，而不是把地址数值简单加 `2` 个字节。数组内容为 `{1, 2, 3, 4, 5}`，下标从 `0` 开始，所以 `arr[2]` 是 `3`，`*p` 输出 `3`。



## 第5题

下列关于排序的说法，正确的是（ ）。

A. 选择排序是最快的排序算法之一。
B. 归并排序通常是稳定的。
C. 最差情况，$N$ 个元素做快速排序的时间复杂度为 $O(N)$。
D. 最好情况，$N$ 个元素做插入排序的时间复杂度为 $O(N^2)$。

答案：B

解析：稳定排序要求相等关键字的元素在排序前后的相对顺序不改变。归并排序在合并两个有序段时，只要遇到相等元素优先取左段元素，就能保持原有相对顺序，因此通常可以实现为稳定排序，B 正确。选择排序一般需要把最小元素直接交换到前面，可能越过相等元素，并不稳定；快速排序最坏时间复杂度可达到 $O(N^2)$；插入排序在序列已有序时只需线性扫描，最好情况是 $O(N)$。



## 第6题

下面关于 C++ 类构造和析构函数的说法，错误的是（ ）。

A. 构造函数不能声明为虚函数。
B. 析构函数必须声明为虚函数。
C. 类的默认构造函数可以被声明为 `private`。
D. 类的析构函数可以被声明为 `private`。

答案：B

解析：析构函数并不是“必须”声明为虚函数，所以 B 是错误说法。只有当类可能被当作基类使用，并通过基类指针删除派生类对象时，基类析构函数才应声明为 `virtual`，否则会有析构不完整的风险。构造函数不能是虚函数，因为构造对象时虚函数机制尚未按完整动态类型工作；默认构造函数和析构函数都可以放在 `private` 区域，用来限制对象创建或销毁方式。



## 第7题

下列关于树和图的说法，错误的是（ ）。

A. 树是一种有向无环图，但有向无环图不都是一棵树。
B. 如果把树看做有向图，每个节点指向其子节点，则该图是强连通图。
C. $N$ 个顶点且连通的无向图，其最小生成树一定包含 $N - 1$ 个条边。
D. $N + 1$ 个顶点、$N$ 条边的有向图，一定不是强连通的。

答案：B

解析：强连通要求任意两个顶点之间都能沿有向边互相到达。若把一棵树的边都定向为“父节点指向子节点”，则子节点无法沿边回到父节点，更不可能和所有其他节点互相可达，所以 B 错误。A 说的是有向无环图范围更大，不一定满足树的连通性和唯一父节点等限制；C 中连通无向图的最小生成树一定含 $N - 1$ 条边；D 中 $N + 1$ 个顶点只有 $N$ 条有向边，边数不足以让每个点都处在同一个强连通结构中。

## 第11题

给定一个整数数组 `nums`，找到其中最长的严格上升子序列的长度。子序列是指从原数组中删除一些元素（或不删除）后，剩余元素保持原有顺序的序列。该程序的时间复杂度为（ ）

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    vector<int> dp(n, 1);

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[i] > nums[j]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
    }
    return *max_element(dp.begin(), dp.end());
}

int main() {
    int n;
    cin >> n;
    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }

    int result = lengthOfLIS(nums);
    cout << result << endl;

    return 0;
}
```

A. $O(n^2)$
B. $O(n)$
C. $O(\log(n))$
D. $O(n\log(n))$

答案：A

解析：核心耗时来自两层循环：外层 `i` 从 `1` 到 `n - 1`，内层 `j` 从 `0` 到 `i - 1`。比较与转移执行次数为 $1 + 2 + \cdots + (n - 1) = O(n^2)$。最后的 `max_element` 是 $O(n)$，不会改变总复杂度。因此该程序是经典的二重循环 LIS 动态规划，时间复杂度为 $O(n^2)$，不是更优的 $O(n\log n)$ 贪心加二分写法。

## 第15题

在哈希表的实现中，冲突解决是一个重要的问题。以下哪种方法 不是 常见的哈希表冲突解决策略？

A. 链地址法（Chaining）
B. 开放地址法（Open Addressing）
C. 二次哈希法（Double Hashing）
D. 二分查找法（Binary Search）

答案：D

解析：哈希冲突是指不同关键字被映射到同一个桶或同一个位置。链地址法会在同一桶中挂链表或其他容器；开放地址法会在表内继续探测空位；二次哈希法通常作为开放地址法的一种探测方式，用另一个哈希函数决定步长。二分查找要求数据有序，用来在有序序列中定位元素，不负责处理哈希表位置冲突，因此 D 不是常见的哈希冲突解决策略。
