## 第1题

链表不具备的特点是（ ）。

A. 可随机访问任何一个元素
B. 插入、删除操作不需要移动元素
C. 无需事先估计存储空间大小
D. 所需存储空间与存储元素个数成正比

答案：A

解析：链表中的结点通常通过指针串联，访问第 $k$ 个元素时需要从头结点沿 `next` 指针逐个走过去，不能像数组那样用下标在 $O(1)$ 时间直接随机访问，所以 A 是链表不具备的特点。B 正是链表插入、删除的优势：只要已经定位到相关结点，修改前驱和后继指针即可，不需要整体搬移元素。C 也符合链式存储的特点，结点可以按需要动态申请。D 说明每个元素对应一个结点，空间随元素个数增长，虽然还要额外存指针，但总体仍与元素个数成正比。



## 第2题

双向链表中每个结点有两个指针域 `prev` 和 `next`，分别指向该结点的前驱及后继结点。设 `p` 指向链表中的一个结点，它的前驱结点和后继结点均非空。要删除结点 `p`，则下述语句中错误的是（ ）。

A.
```cpp
p->next->prev = p->next;
p->prev->next = p->prev;
delete p;
```

B.
```cpp
p->prev->next = p->next;
p->next->prev = p->prev;
delete p;
```

C.
```cpp
p->next->prev = p->prev;
p->next->prev->next = p->next;
delete p;
```

D.
```cpp
p->prev->next = p->next;
p->prev->next->prev = p->prev;
delete p;
```

答案：A

解析：删除双向链表中的 `p`，正确目标是让 `p` 的前驱直接连到 `p` 的后继，即 `p->prev->next = p->next`，同时让 `p` 的后继直接连回 `p` 的前驱，即 `p->next->prev = p->prev`。A 把后继结点的 `prev` 指向后继自己，又把前驱结点的 `next` 指向前驱自己，形成两个自连，原链表在 `p` 两侧被断开，所以错误。B 直接完成两条连接，是标准写法。C 先令 `p->next->prev = p->prev`，此时 `p->next->prev` 就是前驱，再执行其 `next = p->next`，仍能接好。D 同理，先令前驱的 `next` 指向后继，再通过该指针把后继的 `prev` 改为前驱。



## 第3题

假设双向循环链表包含头尾哨兵结点（不存储实际内容），分别为 `head` 和 `tail`，链表中每个结点有两个指针域 `prev` 和 `next`，分别指向该结点的前驱及后继结点。下面代码实现了一个空的双向循环链表，横线上应填的最佳代码是（ ）。

```cpp

    T data;
    ListNode<T>* prev;
    ListNode<T>* next;

    // 构造函数
    explicit ListNode(const T& val = T())
        : data(val), prev(nullptr), next(nullptr) {}
};

struct LinkedList {
    ListNode<T>* head;
    ListNode<T>* tail;
};

void InitLinkedList(LinkedList* list) {
    list->head = new ListNode<T>();
    list->tail = new ListNode<T>();     
    // 在此处填入代码
}
```

A.
```cpp
list->head->prev = list->head;
list->tail->prev = list->head;
```

B.
```cpp
list->head->next = list->tail;
list->tail->prev = list->head;
```

C.
```cpp
list->head->next = list->tail;
list->tail->next = list->head;
```

D.
```cpp
list->head->next = list->tail;
list->tail->next = nullptr;
```

答案：B

解析：空的带头尾哨兵双向链表中，`head` 后面应直接是 `tail`，`tail` 前面应直接是 `head`，因此需要设置 `head->next = tail` 和 `tail->prev = head`，B 正确。A 没有设置 `head->next`，从头结点无法走到尾哨兵，还把 `head->prev` 指向自己，不符合这里的空链表连接。C 设置了 `tail->next = head`，但没有设置 `tail->prev`，无法通过前驱指针从 `tail` 回到 `head`。D 把 `tail->next` 设为 `nullptr`，同样缺少 `tail->prev = head`，不能构成题目要求的双向连接。



## 第4题

用以下辗转相除法（欧几里得算法）求 `gcd(84, 60)` 的步骤中，第二步计算的数是（ ）。

```cpp
int gcd(int a, int b) {
    int big = a > b ? a : b;
    int small = a < b ? a : b;
    if (big % small == 0) {
        return small;
    }
    return gcd(small, big % small);
}
```

A. 84 和 60
B. 60 和 24
C. 24 和 12
D. 12 和 0

答案：B

解析：第一次调用是 `gcd(84, 60)`，函数中 `big = 84`，`small = 60`，因为 $84 \bmod 60 = 24$，不能直接返回，于是递归调用 `gcd(60, 24)`。题目问“第二步计算的数”，指进入下一次递归时的一组参数，所以是 $60$ 和 $24$。继续算会有 $60 \bmod 24 = 12$，再进入 `gcd(24, 12)`，这属于第三步，不能选 C。



## 第5题

根据唯一分解定理，下面整数的唯一分解是正确的（ ）。

A. $18 = 3 \times 6$
B. $28 = 4 \times 7$
C. $36 = 2 \times 3 \times 6$
D. $30 = 2 \times 3 \times 5$

答案：D

解析：唯一分解定理要求把大于 $1$ 的整数写成若干质数的乘积，并且质因数分解在不考虑顺序时唯一。D 中 $2$、$3$、$5$ 都是质数，且 $2 \times 3 \times 5 = 30$，所以正确。A 的 $6$ 不是质数，完整质因数分解应为 $18 = 2 \times 3^2$。B 的 $4$ 不是质数，$28 = 2^2 \times 7$。C 的 $6$ 也不是质数，$36 = 2^2 \times 3^2$。

## 第7题

在程序运行过程中，如果递归调用的层数过多，会因为（ ）引发错误。

A. 系统分配的栈空间溢出
B. 系统分配的堆空间溢出
C. 系统分配的队列空间溢出
D. 系统分配的链表空间溢出

答案：A

解析：每发生一次函数调用，系统都要在调用栈上保存该层函数的局部变量、返回地址等信息。递归调用层数过多时，这些栈帧不断累积，超过系统为栈分配的空间，就会发生栈空间溢出，所以选 A。堆通常用于动态申请内存，队列和链表是数据结构名称，不是递归调用层数过多时首先被耗尽的系统调用空间。



## 第8题

对下面两个函数，说法错误的是（ ）。

```cpp
int factorialA(int n) {
    if (n <= 1) return 1;
    return n * factorialA(n-1);
}

int factorialB(int n) {
    if (n <= 1) return 1;
    int res = 1;
    for(int i=2; i<=n; i++)
        res *= i;
}
```

A. 两个函数的实现的功能相同。
B. 两个函数的时间复杂度均为 $O(n)$。
C. factorialA 采用递归方式。
D. factorialB 采用递归方式。

答案：D

解析：`factorialA` 在 `n > 1` 时调用 `factorialA(n - 1)`，直到 `n <= 1` 返回，因此它采用递归方式，计算过程是 $n \times (n-1) \times \cdots \times 1$。`factorialB` 使用 `for(int i=2; i<=n; i++)` 循环累乘 `res`，没有调用自身，所以不是递归方式，D 的说法错误。两者按题意都是求阶乘，循环和递归都要处理从 $2$ 到 $n$ 的每个数，时间复杂度均为 $O(n)$。严格写代码时 `factorialB` 循环后还应 `return res;`，但本题考查的是递归与循环方式的辨析。



## 第9题

以下算法中，（ ）是不稳定的排序。

A. 选择排序
B. 插入排序
C. 归并排序
D. 冒泡排序

答案：A

解析：排序稳定性指相等关键字的元素排序后相对顺序不变。选择排序每一轮会从未排序区间中选出最小值并与当前位置交换，这个交换可能把后面的相等元素换到前面。例如序列 $[2_a, 2_b, 1]$，第一轮把 $1$ 与 $2_a$ 交换后得到 $[1, 2_b, 2_a]$，两个值为 $2$ 的元素相对顺序改变，所以选择排序不稳定。插入排序、归并排序和冒泡排序在常规稳定实现中都可以保持相等元素的相对顺序。

```cpp
    for (int j = low; j < high; j++) {

// 快速排序
void quickSort(vector<int>& arr, int low, int high) {
```

A.
```cpp
if (arr[j] > pivot) {
    i++;
```

B.

```cpp
if (arr[j] < pivot) {
```

C.

```cpp
if (arr[j] < pivot) {
    swap(arr[i], arr[j]);
```

## 第11题

若用二分法在 $[1, 100]$ 内猜数，最多需要猜（ ）次。

A. 100
B. 10
C. 7
D. 5

答案：C

解析：二分法每猜一次最多把候选范围缩小约一半。要覆盖 $1$ 到 $100$ 共 $100$ 个数，需要最小的 $k$ 满足 $2^k \ge 100$。因为 $2^6 = 64 < 100$，而 $2^7 = 128 \ge 100$，所以最多需要 $7$ 次。选项 D 的 $5$ 次最多只能区分 $32$ 个数，B 的 $10$ 次不是最少的最大次数。



## 第12题

```cpp
int binarySearch(int arr[], int left, int right, int target) {
    while (left <= right) {
        _________________________  // 在此处填入代码

        if (arr[mid] == target)
            return mid;
        else if (arr[mid] < target)
            left = mid + 1;
        else
```

解析：二分查找每轮应取当前区间中点，比较 `arr[mid]` 与 `target` 后丢弃一半区间。`left + (right - left) / 2` 等价于 `(left + right) / 2` 的中点含义，但避免 `left + right` 在大下标时发生整型溢出，因此是最佳写法。B 和 D 每次只取端点，不能稳定地把区间折半，退化为逐步缩小。C 通常能工作，但不如 A 安全。



## 第13题

贪心算法的核心特征是（ ）。

A. 总是选择当前最优解
B. 回溯尝试所有可能
C. 分阶段解决子问题
D. 总能找到最优解

答案：A

解析：贪心算法的核心是在每一步按照某个规则选择当前看来最优的方案，并希望这些局部选择能组成全局最优解，所以 A 是特征描述。B 是回溯或暴力搜索的做法，会尝试多种可能。C 更接近动态规划或分治中“分阶段/分子问题”的描述，不突出“当前最优选择”。D 说法过强，贪心只有在问题满足贪心选择性质时才能保证最优，并不是对所有问题都总能得到最优解。



## 第14题

函数 `int findMax(int arr[], int low, int high)` 计算数组中最大元素，其中数组 `arr` 从索引 `low` 到 `high`，（ ）正确实现了分治逻辑。

A.
```cpp
if (low == high)
    return arr[low];
int mid = (low + high) / 2;
return arr[mid];
```

B.
```cpp
if (low >= high)
    return arr[low];
int mid = (low + high) / 2;
int leftMax = findMax(arr, low, mid - 1);
int rightMax = findMax(arr, mid, high);
return leftMax + rightMax;
```

C.
```cpp
if (low > high)
    return 0;
int mid = low + (high - low) / 2;
int leftMax = findMax(arr, low, mid);
int rightMax = findMax(arr, mid + 1, high);
return leftMax * rightMax;
```

D.
```cpp
if (low == high)
    return arr[low];
int mid = low + (high - low) / 2;
int leftMax = findMax(arr, low, mid);
int rightMax = findMax(arr, mid + 1, high);
return (leftMax > rightMax) ? leftMax : rightMax;
```

答案：D

解析：分治求区间最大值要先处理基本情况：当 `low == high` 时区间只有一个元素，最大值就是 `arr[low]`。否则把区间分成 `[low, mid]` 和 `[mid + 1, high]` 两部分，分别递归求最大值，再返回二者较大者，D 完整体现了“分解、求解、合并”。A 只返回中点元素，没有递归处理左右区间。B 的两个子区间虽然覆盖了原区间，但最后返回 `leftMax + rightMax`，求的是和不是最大值。C 返回乘积，同样不是最大值。
