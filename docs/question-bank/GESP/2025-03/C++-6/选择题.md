## 第1题

在面向对象编程中，类是一种重要的概念。下面关于类的描述中，不正确的是（ ）。

A. 类是一个抽象的概念，用于描述具有相同属性和行为的对象集合。
B. 类可以包含属性和方法，属性用于描述对象的状态，方法用于描述对象的行为。
C. 类可以被实例化，生成具体的对象。
D. 类一旦定义后，其属性和方法不能被修改或扩展。

答案：D

解析：类是对同类对象的抽象，通常把对象共有的数据成员和成员函数组织在一起，所以 A、B、C 都是类的基本特征。D 的问题在于说得过于绝对：类定义之后，可以通过修改类定义来增删成员，也可以通过继承在派生类中扩展行为；即使从“已编译的某个类定义”看不能在运行时随意变更，也不能推出类不能被扩展。因此不正确的是 D。



## 第2题

哈夫曼编码是一种数据压缩算法。以下关于哈夫曼编码的描述中，不正确的是（ ）。

A. 哈夫曼编码是一种变长编码，频率高的字符使用较短的编码，频率低的字符使用较长的编码。
B. 在构造哈夫曼树时，频率越低的字符离根节点越近，频率越高的字符离根节点越远。
C. 哈夫曼编码的生成过程基于贪心算法，每次选择频率最低的两个节点进行合并。
D. 哈夫曼编码是一种前缀编码，任何一个字符的编码都不会是另一个字符编码的前缀，因此可以实现唯一解码。

答案：B

解析：哈夫曼树每次取权值最小的两个结点合并，使权值大的字符尽量靠近根，得到较短编码；权值小的字符通常在更深的位置，编码更长。A 正确描述了“高频短码、低频长码”，C 正确描述了贪心合并过程，D 正确说明哈夫曼编码是前缀编码，任一字符编码都不是另一个字符编码的前缀，因此能唯一解码。B 把频率高低与离根远近说反了，所以不正确。



## 第3题

以下代码实现了树的哪种遍历方式？

```cpp
void traverse(TreeNode* root) {
    if (root == nullptr) return;
    cout << root->val << " ";
    traverse(root->left);
    traverse(root->right);
}
```

A. 前序遍历
B. 中序遍历
C. 后序遍历
D. 层次遍历

答案：A

解析：函数先访问当前结点 `root`，输出 `root->val`，再递归遍历左子树，最后递归遍历右子树，访问顺序是“根、左、右”。这正是前序遍历。中序遍历应为“左、根、右”，后序遍历应为“左、右、根”，层次遍历通常借助队列按层访问，因此选 A。



## 第4题

以下关于完全二叉树的代码描述，正确的是（ ）。

```cpp
bool isCompleteTree(TreeNode* root) {
    if (root == nullptr) return true;
    queue<TreeNode*> q;
    q.push(root);
    bool hasNull = false;
    while (!q.empty()) {
        TreeNode* node = q.front();
        q.pop();
        if (node == nullptr) {
            hasNull = true;
        } else {
            if (hasNull) return false;
            q.push(node->left);
            q.push(node->right);
        }
    }
    return true;
}
```

A. 该代码用于判断一棵树是否为满二叉树
B. 该代码用于判断一棵树是否为完全二叉树
C. 该代码用于判断一棵树是否为二叉搜索树
D. 该代码用于计算树的高度

答案：B

解析：代码用队列做层次遍历，并把空孩子 `nullptr` 也放入队列。完全二叉树的判定规则是：按层从左到右扫描时，一旦遇到空位置，后面就不能再出现非空结点。变量 `hasNull` 记录是否已经遇到空位置；若之后又取出非空 `node`，就返回 `false`。这正是完全二叉树判定，不是要求所有内部结点都有两个孩子的满二叉树，也没有比较键值大小或统计高度，因此选 B。



## 第5题

以下代码实现了二叉排序树的哪种操作？

```cpp
TreeNode* op(TreeNode* root, int val) {
    if (root == nullptr) return new TreeNode(val);
    if (val < root->val) {
        root->left = op(root->left, val);
    } else {
        root->right = op(root->right, val);
    }
    return root;
}
```

A. 查找
B. 插入
C. 删除
D. 遍历

答案：B

解析：当 `root == nullptr` 时，函数创建并返回 `new TreeNode(val)`，说明它会在空位置生成新结点。若 `val < root->val`，递归插入左子树；否则递归插入右子树，符合二叉排序树“左子树值小、右子树值大或不小于根”的插入规则。查找不会创建新结点，删除需要处理被删结点和子树重接，遍历只访问结点，因此选 B。



## 第6题

给定字符集 $\{A, B, C, D\}$ 的出现频率分别为 $\{5, 1, 6, 2\}$，则正确的哈夫曼编码是（ ）。

A. A: 0, B: 100, C: 11, D: 101
B. A: 11, B: 100, C: 0, D: 101
C. A: 0, B: 101, C: 11, D: 100
D. A: 10, B: 101, C: 0, D: 100

答案：B

解析：按频率从小到大合并：先合并 $B(1)$ 和 $D(2)$ 得到权值 $3$；再合并 $3$ 与 $A(5)$ 得到 $8$；最后合并 $C(6)$ 与 $8$ 得到根。于是最高频的 $C$ 可以在根的一侧取得 1 位编码，$A$ 在另一侧的下一层取得 2 位编码，$B$、$D$ 在更深层取得 3 位编码。B 中 $C:0$、$A:11$、$B:100$、$D:101$ 正好对应这种哈夫曼树结构，并且任一编码都不是另一个编码的前缀，是正确的哈夫曼编码方案。D 虽然也让 $C$ 使用 1 位编码，但 $A:10$ 是 $B:101$ 和 $D:100$ 的前缀，破坏前缀编码性质；例如读到前两位 `10` 时无法立即判断是字符 $A$，还是还要继续读成 $B$ 或 $D$，会造成解码歧义。因此选 B。



## 第7题

关于动态规划的描述，正确的是（ ）。

A. 动态规划算法的时间复杂度总是低于贪心算法。
B. 动态规划要求问题必须具有最优子结构和重叠子问题两个性质。
C. 动态规划通过递归实现时不需要存储中间结果。
D. 动态规划的核心思想是将问题分解为互不重叠的子问题。

答案：B

class MyClass {

    MyClass obj1;

## 第9题

以下代码实现了循环队列的哪种操作？

```cpp
class CircularQueue {
    int* arr;
    int front, rear, size;
public:
    CircularQueue(int k) {
        size = k;
        arr = new int[k];
        front = rear = -1;
    }

    bool enQueue(int value) {
        if (isFull()) return false;
        if (isEmpty()) front = 0;
        rear = (rear + 1) % size;
        arr[rear] = value;
        return true;
    }
};
```

A. 入队
B. 出队
C. 查看队首元素
D. 判断队列是否为空

答案：A

解析：函数名为 `enQueue`，参数 `value` 是要加入队列的新元素。代码先判断满队列，空队列时把 `front` 置为 $0$；随后用 `rear = (rear + 1) % size` 把队尾循环后移，并把 `value` 存入 `arr[rear]`。这些步骤都是循环队列的入队操作。出队应移动 `front`，查看队首只读取 `arr[front]`，判断空队列也不会写入新元素，因此选 A。

        if (current->val == target) {

## 第15题

关于下面代码，说法错误的是（ ）。

```cpp
class Shape {
protected:
    string name;

public:
    Shape(const string& n) : name(n) {}

    virtual double area() const {
        return 0.0;
    }
};

class Circle : public Shape {
private:
    double radius;

public:
    Circle(const string& n, double r) : Shape(n), radius(r) {}

    double area() const override {
        return 3.14159 * radius * radius;
    }
};

class Rectangle : public Shape {
private:
    double width;   // 宽度
    double height;  // 高度

public:
    Rectangle(const string& n, double w, double h) : Shape(n), width(w), height(h) {}

    double area() const override {
        return width * height;
    }
};

int main() {
    Circle circle("MyCircle", 5.0);
    Rectangle rectangle("MyRectangle", 4.0, 6.0);

    Shape* shapePtr = &circle;
    cout << "Area: " << shapePtr->area() << endl;

    shapePtr = &rectangle;
    cout << "Area: " << shapePtr->area() << endl;

    return 0;
}
```

A. 语句 `Shape* shapePtr = &circle;` 和 `shapePtr = &rectangle;` 出现编译错误
B. `Shape` 为基类，`Circle` 和 `Rectangle` 是派生类
C. 通过继承，`Circle` 和 `Rectangle` 复用了 `Shape` 的属性和方法，并扩展了新的功能
D. `Circle` 和 `Rectangle` 通过重写（override）基类的虚函数 `area` 和基类指针，实现了运行时多态

答案：A

解析：`Circle` 和 `Rectangle` 都以 `public Shape` 方式继承，因此派生类对象地址可以赋给基类指针，`Shape* shapePtr = &circle;` 与 `shapePtr = &rectangle;` 都能通过编译，A 的说法错误。`area` 在基类中声明为 `virtual`，两个派生类都用 `override` 重写；通过基类指针调用 `shapePtr->area()` 时，会根据指针实际指向的对象动态绑定到 `Circle::area()` 或 `Rectangle::area()`，这就是运行时多态。B、C、D 都符合代码含义，因此本题选错误项 A。
