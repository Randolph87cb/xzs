## 第1题

函数是 C++ 中的核心概念，用于封装可重用的代码块。

答案：正确

解析：函数可以把一段完成特定任务的代码封装起来，并通过函数名和参数在不同位置重复调用。例如计算、输出、交换等逻辑都可以写成函数，调用者不需要重复书写函数体内部的语句。因此“用于封装可重用的代码块”符合 C++ 中函数的基本作用，判断为正确。



## 第2题

在 C++ 中，函数的返回类型可以省略，默认为 `int`。

答案：错误

解析：在标准 C++ 中，函数声明和函数定义都必须明确写出返回类型，例如 `int f()`、`void g()`。早期 C 语言中曾有“省略返回类型默认为 `int`”的历史规则，但 C++ 不采用这种写法。若写成 `f() { return 1; }`，在 C++ 中不是合法的函数定义，因此题干判断为错误。



## 第3题

结构体的成员默认是 `public` 访问权限。

答案：正确

解析：C++ 中 `struct` 和 `class` 都可以有成员变量、成员函数和访问权限控制，但默认访问权限不同：`struct` 的成员默认是 `public`，`class` 的成员默认是 `private`。因此不额外写 `public:` 时，结构体成员可以被结构体外部直接访问，题干判断为正确。



## 第4题

假设整数数组 `arr[4] = {0, 1, 2, 3};` 的第一个元素在内存中的地址为 `0x7ffee4065820`，经过 `int* p = arr; p += 1;` 后，指针 `p` 的值是 1。

答案：错误

解析：`int* p = arr;` 后，`p` 指向数组首元素 `arr[0]`，也就是地址 `0x7ffee4065820`。执行 `p += 1;` 是指针运算，指针会移动到下一个 `int` 元素的位置，地址增加 `sizeof(int)` 个字节，通常为 $4$ 字节，因此新地址通常是 `0x7ffee4065824`。`p` 的值仍然是地址，不是数组元素值 $1$，所以题干判断为错误。



## 第5题

二维数组作为函数参数时，必须显式指定所有维度的大小。

答案：错误

解析：二维数组作为函数参数时，第一维大小可以省略，但从第二维开始必须显式指定。例如 `void f(int a[][3])` 或 `void f(int a[2][3])` 都能让编译器知道每一行有 $3$ 个元素，从而计算 `a[i][j]` 的地址。题干说“必须显式指定所有维度的大小”，把可省略的第一维也包括进去了，因此判断为错误。



## 第6题

递推是一种通过已知的初始值和递推公式，逐步求解目标值的算法。

答案：正确

解析：递推的核心是先给出可以直接知道的初始值，再用前面已经求出的结果推出后面的结果。例如爬楼梯问题中可设 $F_1 = 1$、$F_2 = 2$，再用 $F_n = F_{n-1} + F_{n-2}$ 逐步求到目标阶数。题干对递推的描述包含“初始值”“递推公式”“逐步求解目标值”这三个关键点，判断为正确。



## 第7题

考虑最坏情况下冒泡排序算法的时间复杂度，$ T(n) $ 为待排序数字的数目为 $ n $ 的复杂度，则其递推关系式为 $ T(n) = T(n - 1) + n $，$ T(0) = 1 $。

答案：正确

解析：最坏情况下，冒泡排序处理 $n$ 个数时，第一轮需要在相邻元素间进行约 $n - 1$ 次比较，把最大值放到末尾；剩下的 $n - 1$ 个数仍要继续冒泡排序。因此可写成“解决 $n$ 个数的问题 = 解决 $n - 1$ 个数的问题 + 本轮线性比较”，即 $T(n) = T(n - 1) + n$ 这一量级表达。初始条件 $T(0) = 1$ 表示空规模时为常数时间，不影响递推量级，题干判断为正确。



## 第8题

插入排序在最好情况（已有序）下的时间复杂度是 $ O(n^2) $。

答案：错误

解析：插入排序在最好情况，也就是序列已经有序时，每个新元素只需要和它前一个元素比较一次，就能确认不需要移动。第 $2$ 个到第 $n$ 个元素各做常数次检查，总比较次数与 $n$ 成正比，所以最好时间复杂度是 $O(n)$。$O(n^2)$ 是插入排序在逆序等最坏情况下的量级，题干判断为错误。



## 第9题

对数组 `arr[] = {4, 3, 1, 5, 2}` 进行升序排序，执行第一轮选择排序后数组 `arr` 中的内容是 `{1, 4, 3, 5, 2}`。

答案：错误

解析：升序选择排序第一轮会在整个数组 `{4, 3, 1, 5, 2}` 中找到最小值 $1$，再把它与第一个位置的元素 $4$ 交换。交换后数组应为 `{1, 3, 4, 5, 2}`。题干给出的 `{1, 4, 3, 5, 2}` 像是把 $1$ 前面的元素顺序后移，但这不是选择排序“找到最小值并与当前位置交换”的结果，因此判断为错误。



## 第10题

未捕获异常会调用 `std::terminate` 终止程序。

答案：正确

解析：C++ 异常抛出后会沿调用栈寻找匹配的 `catch` 处理语句。如果一直找不到能处理该异常的 `catch`，异常就属于未捕获异常，运行时会调用 `std::terminate()` 终止程序。题干描述的是未捕获异常的标准处理结果，判断为正确。
