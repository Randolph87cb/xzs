## 第1题

关于下述代码，说法错误的是（ ）。

```cpp
int multiply(int x, int y);

int main() {
    int a = 4;
    int b = 5;
    int result = multiply(a, b);
    std::cout << "The result is: " << result << std::endl;
    return 0;
}

int multiply(int x, int y) {
    return x * y;
}
```

A. 函数 `multiply` 的定义应该放到函数 `main` 之前。
B. 函数声明 `int multiply(int x, int y);` 中明确指定了函数 `multiply()` 的返回值为整数类型。
C. 在 `main` 函数中，函数 `multiply` 通过 `multiply(a, b)` 被调用，其中 `a` 和 `b` 是定义在 `main` 函数中的变量，它们作为实参传递给了 `multiply` 函数的形参 `x` 和 `y`。
D. 运行上述代码，将输出 `The result is: 20`。

答案：A

解析：代码最前面已经有函数声明 `int multiply(int x, int y);`，编译器在读到 `main` 中的 `multiply(a, b)` 时已经知道函数名、参数类型和返回类型，所以函数定义可以放在 `main` 后面，A 的“应该放到 `main` 之前”是错误说法。B 正确，声明开头的 `int` 表示返回值类型为整型。C 正确，`a`、`b` 是实参，调用时分别传给形参 `x`、`y`。D 正确，`multiply(4, 5)` 返回 $20$，输出内容为 `The result is: 20`。



## 第2题

执行下述代码将输出（ ）。

```cpp
int x = 10;
void func() { int x = 20; std::cout << x; }
int main() {
    func();
    std::cout << x;
    return 0;
}
```

A. 2020
B. 2010
C. 1010
D. 编译错误

答案：B

解析：全局变量 `x` 的值是 $10$。调用 `func()` 时，函数内部又定义了一个局部变量 `int x = 20;`，它会遮蔽同名全局变量，因此 `std::cout << x;` 先输出 `20`。`func()` 返回后，`main` 中的 `std::cout << x;` 访问的是全局变量 `x`，继续输出 `10`。两次输出之间没有空格或换行，所以结果是 `2010`。



## 第3题

执行下述代码后，变量 `a` 的值为（ ）。

```cpp
int a = 10;
int* p = &a;
*p = 20;
```

A. 10
B. 20
C. 随机值
D. 编译错误

答案：B

解析：`int* p = &a;` 使指针 `p` 保存变量 `a` 的地址，`*p` 表示通过这个地址访问到的变量本身。因此执行 `*p = 20;` 不是修改指针变量 `p`，而是把 `p` 指向的 `a` 改成 $20$。所以 `a` 的最终值为 $20$，不是原来的 $10$，也不是随机值或编译错误。



## 第4题

以下哪种参数传递方式可以避免拷贝大型对象？

A. 只能用值传递
B. 只能用引用传递
C. 只能用指针传递
D. 引用传递和指针传递均可

答案：D

解析：值传递会把实参对象复制一份给形参，大型对象复制成本较高，因此 A 错。引用传递让形参成为实参的别名，不复制对象；指针传递只复制一个地址，也不复制对象本身，所以二者都可以避免拷贝大型对象。B、C 的“只能”过于绝对，正确答案是 D。



## 第5题

执行下述代码，将输出（ ）。

```cpp
void swap(int a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}
int main() {
    int x = 1, y = 2;
    swap(x, y);
    std::cout << x << y;
    return 0;
}
```

A. 12
B. 21
C. 22
D. 11

答案：D

解析：调用 `swap(x, y)` 时，形参 `a` 是值传递，得到 `x` 的副本 $1$；形参 `b` 是引用传递，直接绑定到变量 `y`。函数内先令 `temp = a = 1`，再令 `a = b`，这一步只把局部副本 `a` 改为 $2$，不会影响 `x`；最后 `b = temp` 会把 `y` 改为 $1$。函数结束后 `x` 仍为 $1$，`y` 为 $1$，连续输出得到 `11`。



## 第6题

下面的描述中，（ ）正确定义一个名为 Person 的结构体并正确初始化了一个 Person 结构体的变量 p。

A.
```cpp
struct Person {
    string name;
    int age;
};
Person p("Yang", 10);
```

B.
```cpp
struct Person {
    string name,
    int age;
};
Person p;
p.name = "Yang";
p.age = 10;
```

C.
```cpp
struct Person {
    string name;
    int age;
};
Person p = { "Yang", 10 };
```

D.
```cpp
struct Person {
    string name;
    int age;
};
Person p = new Person("Yang", 10);
```

答案：C

解析：C 中 `Person` 有两个成员 `name` 和 `age`，`Person p = { "Yang", 10 };` 按成员声明顺序进行聚合初始化，能正确得到 `p.name == "Yang"`、`p.age == 10`。A 写成 `Person p("Yang", 10);` 需要匹配的构造函数，但结构体中没有定义这样的构造函数。B 的成员声明 `string name, int age;` 语法错误，不能在同一条声明中这样写不同类型。D 中 `new Person(...)` 的结果是指针，不能赋给普通变量 `Person p`，而且同样没有对应构造函数。



## 第7题

给定如下代码，

```cpp
struct Person {
    std::string name;
    int age;
    struct Address {
        std::string street;
        std::string city;
    };
    Address address;
};
```

下面描述错误的是（ ）。

A. 结构 `Person` 内嵌套结构 `Address`
B. `Person` 有一个 `Address` 类型的 `address` 成员
C. 一个 `Person` 类型的变量 `p` 的 `address` 的初始化可以写成：`p.address.street = "123 Main St"; p.address.city = "Anytown";`
D. 结构的嵌套可以减少命名冲突，因此可以不必控制嵌套层次

答案：D

解析：代码中 `Address` 定义在 `Person` 内部，所以 A 正确；随后 `Address address;` 声明了一个 `Address` 类型的成员变量，所以 B 正确。若有 `Person p;`，可以通过 `p.address.street` 和 `p.address.city` 访问嵌套成员，C 的写法成立。D 前半句“减少命名冲突”有一定道理，但“不必控制嵌套层次”不正确，嵌套过深会降低可读性和维护性，因此本题选错误描述 D。



## 第8题

假设 `int arr[2][3] = {{1,2,3},{4,5,6}};`，则 `arr[1][2]` 的值是（ ）。

A. 2
B. 3
C. 5
D. 6

答案：D

解析：二维数组下标从 $0$ 开始。`arr[0]` 是第一行 `{1, 2, 3}`，`arr[1]` 是第二行 `{4, 5, 6}`；在第二行中，`arr[1][0]` 为 $4$，`arr[1][1]` 为 $5$，`arr[1][2]` 为 $6$。因此 `arr[1][2]` 的值是 $6$。



## 第9题

下面（ ）正确定义了二维数组。

A. `int arr[3,4];`
B. `int arr[3][4];`
C. `int arr(3,4);`
D. `int a[3-4];`

答案：B

解析：C++ 中二维数组的每一维都要分别写在一对方括号中，`int arr[3][4];` 表示定义一个 $3$ 行 $4$ 列的整型二维数组。A 的 `int arr[3,4];` 不能用逗号在同一个方括号内写两个维度，编译不通过；C 的 `int arr(3,4);` 使用圆括号，不是数组声明；D 的 `int a[3-4];` 数组长度为负数，不是合法的普通数组长度。



## 第10题

小杨正在爬楼梯，需要爬 $ n $ 阶才能到达楼顶。如果每次可以爬 1 个或 2 个台阶，下面代码采用递推算法来计算一共有多少种不同的方法可以爬到楼顶，则横线上应填写（ ）。

```cpp
int f(int n) {
    if (n == 1 || n == 2)
        return n;

    int f1 = 1;
    int f2 = 2;
    int res = 0;
    for (int i = 3; i <= n; i++) {
        // 在此处填入代码
    }
    return res;
}
```

A.
```cpp
res += f1 + f2;
f1 = f2;
f2 = res;
```

B.
```cpp
res = f1 + f2;
f1 = f2;
f2 = res;
```

C.
```cpp
res += f1 + f2;
f2 = res;
f1 = f2;
```

D.
```cpp
res = f1 + f2;
f2 = res;
f1 = f2;
```

答案：B

解析：设爬到第 $i$ 阶的方法数为 $F_i$，最后一步只能从第 $i-1$ 阶爬 $1$ 阶，或从第 $i-2$ 阶爬 $2$ 阶，所以 $F_i = F_{i-1} + F_{i-2}$。代码中 `f1` 保存 $F_{i-2}$，`f2` 保存 $F_{i-1}$，循环内应先计算 `res = f1 + f2`，再把窗口向后移动：`f1 = f2; f2 = res;`。A、C 使用 `res += ...` 会把上一轮的 `res` 继续累加进去；D 先令 `f2 = res` 再令 `f1 = f2`，会使二者都变成当前值，丢失前一项。



## 第11题

给定如下算法，其时间复杂度为（ ）。

```cpp
bool f(int arr[], int n, int target) {
    for (int i = 0; i < (1 << n); i++) {
        int sum = 0;
        for (int j = 0; j < n; j++) {
            if (i & (1 << j)) {
                sum += arr[j];
            }
        }
        if (sum == target) return true;
    }
    return false;
}
```

A. $O(n^2)$
B. $O(n \times 2^n)$
C. $O(1)$
D. $O(n^3)$

答案：B

解析：外层循环 `i < (1 << n)`，一共枚举 $2^n$ 个二进制状态，每个状态代表数组元素的一个子集。对每个状态，内层循环 `j = 0` 到 `n - 1`，检查 $n$ 个位置是否被选中并累加。一次外层循环需要 $O(n)$，总共 $2^n$ 次，因此时间复杂度为 $O(n \times 2^n)$。`sum == target` 的判断是常数时间，不改变总量级。



## 第12题

下面关于排序稳定性的描述，正确的是（ ）。

A. 稳定性指算法的时间复杂度恒定
B. 稳定排序保证相同元素的相对顺序不变
C. 选择排序是稳定排序
D. 插入排序不是稳定排序

答案：B

解析：排序稳定性讨论的是“关键字相等的元素”排序前后的相对先后是否保持不变，与时间复杂度是否恒定无关，所以 A 错。稳定排序的定义正是相同元素的相对顺序不变，B 正确。选择排序会把后面的最小元素直接交换到前面，可能跨过与它相等的元素，通常不稳定，所以 C 错。插入排序在只把严格更大的元素后移时，相等元素不会互相越过，通常是稳定的，所以 D 错。



## 第13题

对数组 `arr[]={5, 3, 8, 1}` 进行升序排序，执行第一轮冒泡排序后数组 `arr` 中的内容为（ ）。

A. 3, 5, 1, 8
B. 3, 1, 5, 8
C. 3, 5, 8, 1
D. 5, 3, 8, 1

答案：A

解析：升序冒泡排序第一轮从左到右比较相邻元素，把当前未排序区间中的最大值交换到最后。初始为 `{5, 3, 8, 1}`：比较 $5$ 和 $3$，交换为 `{3, 5, 8, 1}`；比较 $5$ 和 $8$，不交换；比较 $8$ 和 $1$，交换为 `{3, 5, 1, 8}`。因此第一轮结束后数组内容为 `3, 5, 1, 8`。



## 第14题

运行下面的代码，将出现（ ）。

```cpp
double hmean(double a, double b) {
    if (a == -b)
        throw runtime_error("Runtime error occurred.");
    return 2.0*a*b/(a + b);
}

int main() {
    double x = 10;
    double y = -10;

    try {
        int result = hmean(x, y);
        cout << "hmean: " << result << endl;
    }

    catch (const runtime_error& e) {
        cout << "Caught: " << e.what() << endl;
    } catch (...) {
        cout << "Caught an unknown exception." << endl;
    }
    return 0;
}
```

A. 屏幕上输出 `Caught: Runtime error occurred.`
B. 屏幕上输出 `Caught an unknown exception.`
C. 程序调用 `std::terminate()`
D. 编译错误

答案：A

解析：`x = 10`、`y = -10`，调用 `hmean(x, y)` 时满足 `a == -b`，函数立即执行 `throw runtime_error("Runtime error occurred.");`，不会继续计算返回值。该异常类型是 `runtime_error`，会被第一个 `catch (const runtime_error& e)` 捕获，而不会进入 `catch (...)`，也不会因为未捕获而调用 `std::terminate()`。输出前缀为 `Caught: `，`e.what()` 返回异常消息，所以屏幕输出 `Caught: Runtime error occurred.`。



## 第15题

下面哪种方式不能实现将字符串 "Happy Spring!" 输出重定向到文件 log.txt （ ）。

A.
```cpp
freopen("log.txt", "w", stdout);
cout << "Happy Spring!" << endl;
fclose(stdout);
```

B.
```cpp
std::ofstream outFile("log.txt");
outFile << "Happy Spring!" << endl;
outFile.close();
```

C.
```cpp
std::ofstream outFile("log.txt");
cout << "Happy Spring!" << endl;
outFile.close();
```

D.
```cpp
ofstream log_file("log.txt");
streambuf* org_cout = cout.rdbuf();
cout.rdbuf(log_file.rdbuf());
cout << "Happy Spring!" << endl;
cout.rdbuf(org_cout);
```

答案：C

解析：A 通过 `freopen("log.txt", "w", stdout)` 把标准输出重定向到文件，随后 `cout` 的内容会写入 `log.txt`。B 直接创建 `std::ofstream outFile("log.txt")`，并把字符串写入这个文件流，也能实现文件输出。D 先保存 `cout` 原来的缓冲区，再把 `cout` 绑定到 `log_file` 的缓冲区，输出后恢复原缓冲区，也能重定向。C 虽然打开了 `outFile`，但实际执行的是 `cout << "Happy Spring!" << endl;`，没有把内容写入 `outFile`，因此不能实现题目要求。
