## 第1题

Base64 编码将每 3 字节的输入数据编码为 4 字节的输出数据。如果输入数据长度不是 3 的倍数，会用 = 号填充。在 Base64 编码中，如果输入字符串的长度为 10 字节，编码后的字符串长度是多少（ ）

【知识点：字符串处理】

A. 12 字节
B. 13 字节
C. 14 字节
D. 16 字节

答案：D

解析：Base64 每 $3$ 个输入字节变成 $4$ 个输出字节，不足 $3$ 字节时也要补齐成一组。$10 = 3 \times 3 + 1$，前 $9$ 字节编码成 $3 \times 4 = 12$ 字节，剩下 $1$ 字节仍占一组，编码成 $4$ 字节（其中会包含填充符 `=`）。所以总长度是 $12 + 4 = 16$ 字节，选 D；A、B、C 都没有按“向上补成完整 3 字节组”计算。



## 第2题

UTF-8 编码规则如下：

1 字节：0xxxxxxx

2 字节：110xxxxx 10xxxxxx

3 字节：1110xxxx 10xxxxxx 10xxxxxx

4 字节：11110xxx 10xxxxxx 10xxxxxx 10xxxxxx

以下哪个字节序列是合法的 UTF-8 编码（ ）

【知识点：进制转换与进制字面量】

A. 0xC0 0x80
B. 0xF0 0x90 0x80 0x80
C. 0x80 0x80 0x80
D. 0xFF 0xFE 0xFD

答案：B

解析：合法 UTF-8 多字节编码中，首字节决定总长度，后续字节必须形如 `10xxxxxx`。B 中 `0xF0 = 11110000`，符合 $4$ 字节首字节 `11110xxx`；三个后续字节 `0x90 = 10010000`、`0x80 = 10000000`、`0x80 = 10000000` 都符合 `10xxxxxx`，因此合法。A 的 `0xC0 0x80` 虽然外形像 $2$ 字节序列，但属于过长编码，不是合法 UTF-8；C 以 `10xxxxxx` 开头，缺少首字节；D 的 `0xFF`、`0xFE`、`0xFD` 不符合 UTF-8 首字节或续字节格式。



## 第3题

在 8 位二进制原码表示中，八进制数 -5 的二进制形式是什么（ ）

【知识点：原码反码补码与整数表示】

A. 10000101
B. 11111010
C. 11111011
D. 00000101

答案：A

解析：八进制数 $5$ 的数值仍是十进制的 $5$，写成 $8$ 位二进制数为 `00000101`。原码最高位是符号位，负数符号位为 `1`，数值部分保持 $5$ 的二进制 `0000101`，所以 $-5$ 的 $8$ 位原码是 `10000101`，选 A。B、C 分别像反码、补码形式，不是原码；D 是正数 $+5$ 的原码。



## 第4题

十进制数 $111.111$ 的二进制表示可以是下面的（ ）。

【知识点：进制转换与进制字面量】

A. 1101111.0001110001
B. 1101110.1001110001
C. 1101111.1001110001
D. 1101111.0011110001

答案：A

解析：整数部分 $111 = 64 + 32 + 8 + 4 + 2 + 1$，对应二进制为 `1101111`。小数部分用“乘 $2$ 取整”：$0.111 \times 2 = 0.222$ 取 `0`，再得 `0`、`0`，第 $4$ 次 $0.888 \times 2 = 1.776$ 取 `1`，继续可得前 $10$ 位为 `0001110001`。因此可写作 `1101111.0001110001`，选 A；B 的整数部分少了 $1$，C、D 的小数前几位与乘 $2$ 取整结果不符。



## 第5题

在 C++ 中，补码的主要作用是（ ）

【知识点：原码反码补码与整数表示】

A. 提高浮点数的精度
B. 简化整数的加减法运算
C. 增加整数的表示范围
D. 优化内存分配

答案：B

解析：补码把减法统一成加法处理，例如计算 $a - b$ 可以转化为 $a + (-b)$，硬件只需按同一套二进制加法规则运算并舍弃溢出的高位，因此主要作用是简化整数的加减法运算，选 B。A 讨论的是浮点数精度，D 是内存管理问题；C 不准确，同样 $8$ 位下补码范围是 $-128$ 到 $127$，并不是单纯“增加范围”，而是让负数表示和运算更统一。



## 第6题

在 C++ 中，一个 8 位有符号整数（使用补码表示）的范围是（ ）

【知识点：原码反码补码与整数表示】

A. -128 到 127
B. -127 到 128
C. -256 到 255
D. 0 到 255

答案：A

解析：$8$ 位一共有 $2^8 = 256$ 种位模式。补码中最高位为 `0` 的 $128$ 种表示 $0$ 到 $127$，最高位为 `1` 的 $128$ 种表示 $-128$ 到 $-1$，所以范围是 $-128$ 到 $127$，选 A。B 把负数和正数端点写反，C 超出了 $8$ 位能表示的 $256$ 种状态，D 是 $8$ 位无符号整数范围。



## 第7题

在 C++ 中，以下代码的输出是什么（ ）

```cpp
int a = -5;
unsigned int b = a;
cout << b;
```

【知识点：C++语法规则与编译模型】

A. -5
B. 5
C. 4294967291
D. 编译错误

答案：C

解析：常见环境下 `unsigned int` 为 $32$ 位，赋值时 $-5$ 按模 $2^{32}$ 转成无符号数：$2^{32} - 5 = 4294967291$，所以输出 C。A 不对，因为变量 `b` 已经是无符号类型；B 把负号简单去掉，不符合补码转换规则；D 不对，带符号整数转无符号整数是允许的。



## 第8题

下列程序的作用是（ ）

```cpp
int main() {
    int decimal = 25;
    cout << oct << decimal;
    return 0;
}
```

【知识点：进制转换与进制字面量】

A. 将十进制数转换成八进制数
B. 将八进制数转换成十进制数
C. 将二进制数转换成八进制数
D. 将八进制数转换成 16 进制数

答案：A

解析：变量 `decimal` 中保存的数值是十进制写法的 $25$，`oct` 是输出流操纵符，会让后面的整数按八进制形式输出。$25_{10} = 3 \times 8 + 1 = 31_8$，所以程序作用是把这个整数按八进制输出，选 A。它不是读取八进制再转十进制，也没有涉及二进制或十六进制转换。



## 第9题

下面程序目的是将十进制数 255 转换为十六进制输出，横线处应填写的语句是（ ）

```cpp
#include <iostream>
using namespace std;

int main() {
    int decimal = 255;
    _______________________
    return 0;
}
```

【知识点：进制转换与进制字面量】

A. cout << oct << decimal;
B. cout << decimal << decimal;
C. cout << hex << decimal;
D. 不能正确执行

答案：C

解析：`hex` 是十六进制输出操纵符，`cout << hex << decimal;` 会把变量 `decimal` 的值 $255$ 按十六进制输出，即 `ff`，所以选 C。A 使用 `oct`，会输出八进制 `377`；B 连续输出变量值，仍是十进制文本；D 错在程序可以正确执行。



## 第10题

以下代码的说法正确的是什么（ ）

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 0b1101;
    int b = 0b1011;
    cout << (a ^ b);
    return 0;
}
```

【知识点：位运算】

A. 进行的是整体异或运算
B. 进行的是按位同或运算
C. 进行的是按位与运算
D. 进行的是按位异或运算

答案：D

解析：C++ 中 `^` 是按位异或运算符，两位相同得 `0`，不同得 `1`。本题中 `a = 0b1101`，`b = 0b1011`，逐位异或得到 `0110`，也就是十进制 $6$；题目问“进行的是什么运算”，因此选 D。A 的“整体异或”不是 C++ 对整数 `^` 的含义；B 的同或是相同得 `1`；C 的按位与应使用 `&`。



## 第11题

下面枚举法查找最大值索引程序中，横线处应该填写的是（ ）

```cpp
#include <iostream>
using namespace std;

int main() {
    int arr[] = {3, 7, 2, 9, 5};
    int maxIndex = 0;
    for (int i = 1; i < 5; i++) {
        _______________________
        {
            maxIndex = i;
        }
    }
    cout << maxIndex;
    return 0;
}
```

【知识点：数组与字符数组】

A. `if (arr[maxIndex] > arr[i])`
B. `if (arr[i]-1 > arr[maxIndex])`
C. `if (arr[i]+1 > arr[maxIndex])`
D. `if (arr[i] > arr[maxIndex])`

答案：D

解析：`maxIndex` 保存当前最大元素的下标，循环 `i = 1` 到 `4` 枚举后续元素。只有当 `arr[i] > arr[maxIndex]` 时，才说明找到了更大的元素，需要把 `maxIndex` 更新为 `i`，所以选 D。A 判断方向相反，会在当前最大值比新元素大时错误更新；B、C 把 `arr[i]` 改成 `arr[i]-1` 或 `arr[i]+1`，比较的就不是原数组元素本身。



## 第12题

以下代码的功能是将数组中的奇数和偶数分别放在数组的前半部分和后半部分，横线处应该填入的是（ ）

```cpp
#include <iostream>
using namespace std;

int main() {
    int arr[] = {1, 2, 3, 4, 5};
    int left = 0, right = 4;
    while (left < right) {
        while (arr[left] % 2 == 1 && left < right) left++;
        _______________________
        if (left < right) {
            swap(arr[left], arr[right]);
        }
    }
    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";
    }
    return 0;
}
```

【知识点：数组与字符数组】

A. `while (arr[left] % 2 == 0 && left < right) right--;`
B. `while (arr[right] % 2 == 0 && left < right) left--;`
C. `while (arr[right] % 2 != 0 && left < right) right--;`
D. `while (arr[right] % 2 == 0 && left < right) right--;`

答案：D

解析：左侧循环已经让 `left` 跳过前面的奇数，停在一个需要交换的偶数位置；右侧应当让 `right` 从末尾向左跳过已经在后半部分的偶数，所以条件应是 `arr[right] % 2 == 0 && left < right`，循环体执行 `right--`，选 D。A 检查了 `arr[left]`，方向错；B 虽然检查右端偶数，却移动 `left--`，会破坏左指针；C 会跳过右端奇数，反而把应交换到前面的奇数留在右侧。



## 第13题

下面程序最后能够得到 HelloC++ 的是（ ）

```cpp
int main() {
    string str = "HelloWorld";
    _______________________
    cout << str;
    return 0;
}
```

【知识点：字符串处理】

A. `str.replace(0, 5, "C++");`
B. `str.replace(5, 5, "C++");`
C. `str.replace(1, 5, "C++");`
D. `str.replace(4, 5, "C++");`

答案：B

解析：`"HelloWorld"` 中下标从 $0$ 开始，`Hello` 占下标 $0$ 到 $4$，`World` 占下标 $5$ 到 $9$。`str.replace(5, 5, "C++")` 表示从下标 $5$ 开始删除 $5$ 个字符，把 `World` 替换为 `C++`，结果正好是 `HelloC++`，选 B。A 会把 `Hello` 替换掉；C 从下标 $1$ 开始替换，会破坏 `elloW`；D 从下标 $4$ 开始替换，会得到不是目标的字符串。



## 第14题

想要得到字符串 world，下面程序横线处应该填入的是（）

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string str = "HelloC++";
    _____________
    _____________
    return 0;
}
```

【知识点：字符串处理】

A.
```cpp
str.insert(4, "World");
cout << str.substr(4, 4);
```

B.
```cpp
cout << str.substr(5, 5);
```

C.
```cpp
str.insert("World");
cout << str.substr(5, 5);
```

D.
```cpp
str.insert(5, "World");
cout << str.substr(5, 5);
```

答案：D

解析：原串 `"HelloC++"` 的下标 $0$ 到 $4$ 是 `Hello`，下标 $5$ 开始是 `C++`。D 先执行 `str.insert(5, "World")`，把 `World` 插入到下标 $5$ 处，字符串变成 `HelloWorldC++`；再执行 `str.substr(5, 5)`，从下标 $5$ 取 $5$ 个字符，得到 `World`（题面写作 `world`，但选项实际产生的是首字母大写的 `World`）。A 从下标 $4$ 插入且只取 $4$ 个字符；B 没有插入，`substr(5, 5)` 得到 `C++`；C 的 `insert` 调用参数不匹配。



## 第15题

有 $n$ 个正整数，假设一个正整数是美丽数字当且仅当该正整数是 9 的倍数但不是 8 的倍数。下面的程序是编写计算 $n$ 个正整数中美丽数字的数量，横线处应该填入的是（ ）

```cpp
for (int i = 1; i <= n; i++) {
    cin >> a;
    _______________________
    cnt++;
}
```

【知识点：位运算】

A. `if (a % 9 != 0 && a % 8 != 0)`
B. `if (a % 9 == 0 && a % 8 == 0)`
C. `if (a % 9 == 0 && a % 8 != 0)`
D. `if (a % 9 == 0 & a % 8 != 0)`

答案：C

解析：美丽数字需要同时满足两个条件：是 $9$ 的倍数，即 `a % 9 == 0`；并且不是 $8$ 的倍数，即 `a % 8 != 0`。表达“并且”时规范写法是逻辑与 `&&`，所以应填 `if (a % 9 == 0 && a % 8 != 0)`，选 C。A 要求两个条件都为“不整除”，B 要求同时是 $9$ 和 $8$ 的倍数，均与题意不符。D 写成 `if (a % 9 == 0 & a % 8 != 0)`，由于 `==`、`!=` 的优先级高于按位与 `&`，实际等价于 `(a % 9 == 0) & (a % 8 != 0)`；在本题这两个比较表达式都没有副作用时，真值结果与 `&&` 相同，但 `&` 是按位与，不会短路，也不是表达逻辑条件的推荐/标准写法，因此本题按规范逻辑运算选择 C。
