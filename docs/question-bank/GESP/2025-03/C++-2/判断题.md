## 第1题

C++、Python 都是高级编程语言，它们的每条语句最终都要通过机器指令来完成。（ ）

答案：正确

解析：题干表述成立。C++、Python 这类高级语言的语句不能被 CPU 直接当作源代码执行，运行前或运行时都需要经过编译、解释、虚拟机等机制，最终落实为机器能够执行的指令序列。这里说“最终都要通过机器指令来完成”，强调的是底层执行结果，不是说两种语言的执行方式完全相同。



## 第2题

在 C++ 代码中，假设 $N$ 为正整数，则 `N - N / 10 * 10` 与 `N % 10` 都将获得 $N$ 的个位数。（ ）

答案：正确

解析：题干表述成立。$N / 10$ 是整数除法，会去掉个位；再乘以 $10$ 得到去掉个位后的整十部分，所以 $N - N / 10 * 10$ 剩下的正是个位。`N % 10` 也是求除以 $10$ 的余数。题目限定 $N$ 为正整数，因此二者都会得到 $N$ 的个位数。



## 第3题

C++ 语句 `cout << ((10 <= N <= 12) ? "true" : "false")` 中，假设整型变量 N 为 12，则其输出为 true。原因是执行 `10 <= N` 后其值为 true，true 与 12 相比仍然是 true。（ ）

答案：正确

解析：题干表述成立。C++ 不会把 `10 <= N <= 12` 当成数学里的连续比较，而是从左到右先算 `10 <= N`。当 $N = 12$ 时，`10 <= N` 的结果为 `true`，参与后续比较时可转换为整数 $1$，于是继续判断 $1 <= 12$，结果仍为 `true`，三目运算符输出 `"true"`。



## 第4题

C++ 表达式 `(sqrt(N) * sqrt(N)) == N` 中的 $N$ 如果为正整数，则表达式的值为 true，相当于开平方后平方是本身。（ ）

答案：错误

解析：题干表述不成立。`sqrt(N)` 返回的是浮点数，很多正整数开平方后并不是整数，计算结果只能用近似值表示；即使数学上先开方再平方等于原数，浮点运算中也可能因为精度误差导致 `(sqrt(N) * sqrt(N)) == N` 为 `false`。判断浮点结果通常应比较误差范围，而不是直接用 `==`。



## 第5题

下面 C++ 执行后将输出 `3*2=6`。（ ）

```cpp
int a=2, b = 3;
a = a - b;
b = a + b;
a = b - a;
printf("%d*%d=%d\n", a, b, a*b);
```

答案：正确

解析：题干表述成立。初始 $a = 2, b = 3$。执行 `a = a - b` 后 $a = -1$；执行 `b = a + b` 后 $b = -1 + 3 = 2$；执行 `a = b - a` 后 $a = 2 - (-1) = 3$。因此最终 $a = 3, b = 2$，`a*b` 为 $6$，格式串输出 `3*2=6`。



## 第6题

下面 C++ 代码执行后将输出 10。（ ）

```cpp
int i;
for (i = 0; i < 10; i++)
    continue;
cout << i << endl;
```

答案：正确

解析：题干表述成立。循环从 $i = 0$ 开始，只要 $i < 10$ 就进入循环体。循环体里的 `continue` 会跳过本轮剩余语句，但 `for` 循环的第三部分 `i++` 仍会执行。$i$ 依次增加到 $10$ 时条件不成立，循环结束后输出的就是 $10$。



## 第7题

下面 C++ 代码执行后将输出 1。（ ）

```cpp
int i;
for (i = 1; i < 10; i++) {
    break;
    continue;
}
cout << i << endl;
```

答案：正确

解析：题干表述成立。`for` 循环初始化后 $i = 1$，条件 $i < 10$ 成立，进入循环体后立刻执行 `break`，循环马上终止。`break` 后面的 `continue` 永远不会执行，循环的 `i++` 也不会执行，所以循环外输出的 $i$ 仍为 $1$。



## 第8题

下面的 C++ 代码执行后将输出 10 行 "OK"。（ ）

```cpp
for (int i = 0; i < 5; i++)
    for (int j = 0; j < i; j++)
        printf("OK\n");
```

答案：正确

解析：题干表述成立。外层 $i$ 取 $0,1,2,3,4$。内层循环次数分别是 $0,1,2,3,4$ 次，因为条件是 $j < i$。总输出次数为 $0 + 1 + 2 + 3 + 4 = 10$，每次输出一行 `OK`，所以会输出 10 行。



## 第9题

将下面 C++ 代码中的 for 循环中的 `i = 1` 调整为 `i = 0` 的输出结果相同。（ ）

```cpp
int tnt = 0;
for (int i = 1; i < 5; i++) // i=1
    tnt += i;
cout << tnt;
```

答案：正确

解析：题干表述成立。原循环累加 $1 + 2 + 3 + 4 = 10$。如果把初值改成 $i = 0$，循环累加的是 $0 + 1 + 2 + 3 + 4$，多加的 $0$ 不改变总和，结果仍为 $10$。循环上界和递增方式没有变化，因此输出相同。



## 第10题

下面 C++ 代码执行后将输出 0123。（ ）

```cpp
for (i = 0; i < 5; i++)
    for (i = 0; i < i; i++)
        continue;
    printf("%d\n", i);
```

答案：错误

解析：题干表述不成立。没有花括号时，外层 `for` 的循环体只有紧跟着的内层 `for`，`printf("%d\n", i);` 不属于外层循环体。外层第一次令 $i = 0$，进入内层后又执行 `i = 0`，内层条件 `i < i` 变成 $0 < 0$，不成立，所以 `continue` 不执行。随后外层执行 `i++`，使 $i$ 变为 $1$；下一轮外层条件仍成立，但又会被内层初始化重置为 $0$。程序会这样反复循环，永远到不了 `printf`，因此不会输出 `0123`。
