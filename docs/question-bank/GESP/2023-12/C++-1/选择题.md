## 第1题

以下 C++ 不可以作为变量的名称的是（）。

A. CCF GESP
B. ccfGESP
C. CCFgesp
D. CCF_GESP

答案：A

解析：C++ 标识符只能由字母、数字和下划线组成，且不能以数字开头，也不能包含空格。`CCF GESP` 中间有空格，会被看成两个独立的标识符，不能作为一个变量名。`ccfGESP`、`CCFgesp` 都只含字母，`CCF_GESP` 只额外含有合法的下划线，因此都可以作为变量名。

## 第2题

C++ 表达式 `10 - 3 * (2 + 1) % 10` 的值是（）。

A. 0
B. 1
C. 2
D. 3

答案：B

解析：表达式要按优先级计算：先算括号内 `2 + 1 = 3`，再算乘法 `3 * 3 = 9`。`*` 和 `%` 的优先级相同，并且从左到右结合，所以接着算 `9 % 10 = 9`，最后 `10 - 9 = 1`。选项 A 容易误把 `10 - 3 * 3` 后再取模，选项 C、D 也都没有按正确的优先级和结合方向完整计算。

## 第3题

假设现在是上午十点，求出 $N$ 小时（正整数）后是第几天几时，如输入 $20$ 小时则为第 $2$ 天 $6$ 点，如 $N$ 输入 $4$ 则为今天 $14$ 点。为实现相应功能，应在横线处填写代码是（）。

```cpp
int N, dayX, hourX;
cin >> N;
dayX = _____, hourX = _____;
if (dayX == 0)
    cout << "今天" << hourX << "点";
else
    cout << "第" << (dayX + 1) << "天" << hourX << "点";
```

A. ` (10 + N) % 24 `, ` (10 + N) / 24 `
B. ` (10 + N) / 24 `, ` (10 + N) % 24 `
C. ` N % 24 `, ` N / 24 `
D. ` 10 / 24 `, ` 10 % 24 `

答案：B

解析：从上午 10 点开始，经过 `N` 小时后的总小时数是 `10 + N`。经过了多少个完整的 24 小时应放入 `dayX`，即 `(10 + N) / 24`；当天的小时数应放入 `hourX`，即 `(10 + N) % 24`。例如 `N = 20` 时得到 `dayX = 1`、`hourX = 6`，输出第 2 天 6 点；`N = 4` 时得到 `dayX = 0`、`hourX = 14`，输出今天 14 点。A 把天数和小时数写反了，C 忘记了起点是上午 10 点，D 只使用常数 `10`，没有用到输入的 `N`。

## 第4题

下面的程序用于判断 $N$ 是否为偶数，横线处应填写代码是（ ）。

```cpp
cin >> N;
if (______)
    cout << "偶数";
else
    cout << "奇数";
```

A. ` N % 2 == 0 `
B. ` N % 2 = 0 `
C. ` N % 2 `
D. ` N % 2 != 0 `

答案：A

解析：偶数的判断条件是能被 `2` 整除，也就是余数为 `0`，应写成 `N % 2 == 0`。B 中 `N % 2 = 0` 把赋值号用在取模表达式结果上，既不是比较，也不能给这样的临时结果赋值；C 中 `N % 2` 在奇数时为 `1`、在偶数时为 `0`，判断方向正好相反；D 中 `N % 2 != 0` 明确判断的是奇数。

## 第5题

下面对 C++ 代码执行后输出的描述，正确的是（ ）。

```cpp
cin >> N;
cnt = 0;
for(int i = 1; i < N; i++)
    cnt += 1;
cout << cnt;
```

A. 如果输入的 $N$ 是小于等于 $2$ 整数，第 $5$ 行将输出 $0$。
B. 如果输入的 $N$ 是大于等于 $2$ 整数，第 $5$ 行将输出 $N-1$。
C. 如果输入的 $N$ 是大于等于 $2$ 整数，第 $5$ 行将输出 $N$。
D. 以上说法均不正确。

答案：B

解析：`cnt` 初始为 `0`，循环从 `i = 1` 开始，只要 `i < N` 就执行一次 `cnt += 1`。当 `N >= 2` 时，`i` 依次取 `1, 2, ..., N - 1`，一共执行 `N - 1` 次，因此输出 `N - 1`。A 错在 `N = 2` 时循环会执行一次，输出 `1`，不是 `0`；C 把循环次数误认为包含了 `i = N`；D 不成立，因为 B 已经给出了正确描述。

## 第6题

下面 C++ 代码执行后的输出是（ ）。

```cpp
cnt = 0;
for(int i = 1; i < 10; i++){
    cnt += 1;
    i += 2;
}
cout << cnt;
```

A. 10
B. 9
C. 3
D. 1

答案：C

解析：要注意循环体内和 `for` 末尾都会改变 `i`。第一次进入循环时 `i = 1`，`cnt` 变为 `1`，循环体内 `i += 2` 变为 `3`，再执行 `for` 的 `i++` 变为 `4`。之后以 `i = 4`、`i = 7` 两次进入循环，`cnt` 分别变为 `2`、`3`。第三次循环结束后 `i` 先变为 `9`，再由 `i++` 变为 `10`，不满足 `i < 10`，所以输出 `3`。A、B 都忽略了循环体内的 `i += 2`，D 则少算了后两次循环。

## 第7题

下面 C++ 代码执行后的输出是（ ）。

```cpp
cnt = 0;
for(int i = 1; i < 20; i++){
    if(i%2)
        continue;
    else if(i%3==0 && i%5==0)
        break;
    cnt += i;
}
cout << cnt;
```

A. 90
B. 44
C. 20
D. 10

答案：A

解析：`if(i%2)` 在奇数时为真，奇数会直接 `continue`，不会累加，也不会检查后面的 `else if`。偶数才会继续判断是否同时被 `3` 和 `5` 整除；在 `1` 到 `19` 中，偶数里没有 `15` 的倍数，所以不会触发 `break`。最终累加的是所有小于 `20` 的正偶数：`2 + 4 + 6 + 8 + 10 + 12 + 14 + 16 + 18 = 90`。如果把奇数也累加、或误以为遇到 `15` 会退出，都会得到错误结果。

## 第8题

下面 C++ 代码执行后的输出是（ ）。

```cpp
N = 10;
cnt = 0;
while(1){
    if(N == 0) break;
    cnt += 1;
    N -= 2;
}
cout << cnt;
```

A. 11
B. 10
C. 5
D. 4

答案：C

解析：初始 `N = 10`。每次循环先判断 `N == 0`，不为 `0` 时才让 `cnt` 加 `1`，然后执行 `N -= 2`。真正被计数的 `N` 依次是 `10`、`8`、`6`、`4`、`2`，共 `5` 次；减到 `0` 后进入下一轮，马上执行 `break`，不会再给 `cnt` 加一。因此输出 `5`，不是把 `0` 也计入后的 `6`，更不是按普通 `1` 到 `10` 计数。

## 第9题

下面 C++ 代码用于判断一个数是否为质数（素数），在横线处应填入代码是（ ）。

```cpp
cin >> N;
cnt = 0;
for (int i = 1; i < N + 1; i++)
    if (N % i == 0)
        _____;
if (cnt == 2)
    cout << N << "是质数。";
else
    cout << N << "不是质数。";
```

A. ` cnt = 1 `
B. ` cnt = 2 `
C. ` cnt =+ 1 `
D. ` cnt += 1 `

答案：D

解析：程序遍历 `1` 到 `N`，遇到能整除 `N` 的 `i` 就要把约数个数加一。质数恰好只有 `1` 和自身两个正约数，所以最后用 `cnt == 2` 判断。只有 `cnt += 1` 表示在原值基础上增加 `1`。A、B 的 `cnt = 1`、`cnt = 2` 是每次重新赋固定值，不能累计；C 的 `cnt =+ 1` 会被解析成 `cnt = (+1)`，也只是把 `1` 赋给 `cnt`，不是自增。

## 第10题

对下面的代码，描述正确的是（ ）。

```cpp
#include <stdlib.h>
using namespace std;

int main(){
    int arr[] = {2,6,3,5,4,8,1,0,9,10};
    for(int i = 0; i < 10; i++)
        cout << arr[i] << " ";
    cout << i << endl;
    cout << endl;
    return 0;
}
```

A. 输出 2 6 3 5 4 8 1 0 9 10 10
B. 输出 2 6 3 5 4 8 1 0 9 9
C. 输出 2 6 3 5 4 8 1 0 9 10
D. 提示有编译错误

答案：D

解析：关键问题在循环变量的作用域。`for(int i = 0; i < 10; i++)` 中声明的 `i` 只在这个 `for` 语句内部有效；循环结束后再执行 `cout << i << endl;` 时，`i` 已经不可见，因此程序会编译错误。A、B 假设循环结束后还能输出 `i`，C 假设只输出数组内容，但实际代码后面还访问了越出作用域的 `i`，所以三者都不是正确描述。

## 第11题

执行下面 C++ 代码后输出是（ ）。

```cpp
cnt = 0;
for (i = 10; i > 3; i -= 3)
    cnt = cnt + i;
cout << cnt;
```

A. 3
B. 21
C. 27
D. 49

答案：B

解析：循环开始时 `i = 10`，满足 `i > 3`，把 `10` 加入 `cnt`；之后 `i -= 3` 变为 `7`，再加 `7`；再变为 `4`，仍满足 `i > 3`，再加 `4`。下一次 `i` 变为 `1`，不满足条件，循环结束，所以输出 `10 + 7 + 4 = 21`。选项 A 少算了前面的累加，C 把不满足条件的 `1` 或其他值误算进去了，D 则不是这组循环取值的和。

## 第12题

定义变量 `char c`，下面对 c 赋值的语句，不符合语法的是（ ）。

A. ` c = (char)66; `
B. ` c = (char)(66); `
C. ` c = char(66); `
D. ` c = char 66; `

答案：D

解析：`(char)66` 和 `(char)(66)` 是 C 风格强制类型转换，`char(66)` 是 C++ 的函数式类型转换，它们都能把整数 `66` 转为字符后赋给 `c`。`char 66` 既不是合法的类型转换表达式，也不是合法的变量声明或赋值右值写法，缺少必要的括号或变量名，因此不符合语法。

## 第13题

下图所示 C++ 代码对大写字母 'A' 到 'Z' 分组，对每个字母输出所属的组号，那么输入 'C' 时将输出的组号是（ ）。

```cpp
char c;
while(1){
    cin >> c;
    if(c=='q') break;
    switch(c){
        case 'A': cout << "1 "; break;
        case 'B': cout << "3 ";
        case 'C': cout << "3 ";
        case 'D': cout << "5 "; break;
        case 'E': cout << "5 "; break;
        default: cout << "9 ";
    }
    cout << endl;
}
```

A. 3
B. 3 5
C. 3 5 9
D. 以上都不对

答案：B

解析：`switch` 会跳到匹配的 `case` 开始执行。输入 `'C'` 时进入 `case 'C'`，先输出 `3 `。这个分支后面没有 `break`，所以会继续向下执行 `case 'D'`，再输出 `5 `，随后遇到 `break` 退出 `switch`。因此结果是 `3 5`。A 忽略了 `case 'C'` 后的贯穿执行，C 则多算了 `default`，但程序在 `case 'D'` 的 `break` 处已经退出。

## 第14题

某公司新出了一款无人驾驶的小汽车，通过声控智能驾驶系统，乘客只要告诉汽车目的地，车子就能自动选择一条优化路线，告诉乘客后驶达那里。请问下面哪项不是驾驶系统完成选路所必须的。（ ）

A. 麦克风
B. 扬声器
C. 油量表
D. 传感器

答案：C

解析：题干强调的是声控智能驾驶系统完成“接收目的地、规划路线、告知乘客”这一选路过程。麦克风用于接收乘客说出的目的地，传感器用于感知车辆和环境信息，扬声器可用于把选择的路线告知乘客。油量表反映剩余燃油，可能影响车辆能否行驶很远，但不是完成选路功能本身所必须的部件。

## 第15题

现代计算机是指电子计算机，它所基于的是（ ）体系结构。

A. 艾伦·图灵
B. 冯·诺依曼
C. 阿塔纳索夫
D. 埃克特-莫克利

答案：B

解析：现代电子计算机通常基于冯·诺依曼体系结构，核心思想包括存储程序、程序和数据存放在存储器中，并由运算器、控制器、存储器、输入设备和输出设备协同工作。图灵、阿塔纳索夫、埃克特-莫克利都与计算机发展有关，但不是这道题所问的体系结构名称。
