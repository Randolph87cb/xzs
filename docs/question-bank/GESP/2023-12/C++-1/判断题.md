# 判断题

## 第1题

C++ 表达式 `int(3.14)` 的值为 3。

答案：正确

解析：`int(3.14)` 是把浮点数 `3.14` 显式转换为 `int`。C++ 中浮点数转整数时会直接舍去小数部分，而不是四舍五入；对 `3.14` 来说结果就是 `3`，因此说法正确。

## 第2题

C++ 语句 `cout << (2 * 3, 3 % 10, 2+3)` 的输出为 6,3,5。

答案：错误

解析：括号内不是三个独立的输出项，而是一个逗号表达式。它会从左到右计算 `2 * 3`、`3 % 10`、`2 + 3`，但整个表达式的值只取最后一项，也就是 `5`。因此 `cout` 只会输出 `5`，不会输出 `6,3,5`，也不会自动补逗号。

## 第3题

`for(int i = 1; i < 10; i += 3);` 表示从 1 开始到 10 结束间隔为 3，相当于 1、4、7、10。

答案：错误

解析：循环条件是 `i < 10`，不是 `i <= 10`。循环变量进入循环时依次为 `1`、`4`、`7`，之后执行 `i += 3` 变成 `10`，再次判断 `i < 10` 为假，循环结束，所以 `10` 不会作为循环中的取值出现。另外，语句末尾的分号表示循环体为空，即使循环执行了，也没有额外语句被重复执行。

## 第4题

C++ 表达式 `010+100+001` 的值为 111。

答案：错误

解析：在 C++ 整数字面量中，以 `0` 开头的数按八进制解释。`010` 等于十进制的 `8`，`001` 等于十进制的 `1`，而没有前导 `0` 的 `100` 是十进制的 `100`。三者相加为 `8 + 100 + 1 = 109`，不是 `111`。

## 第5题

if 语句中的条件表达式的结果可以为 int 类型。

答案：正确

解析：`if` 的条件表达式不要求本身就是 `bool` 类型，只要能转换为布尔值即可。`int` 类型可以用于条件判断，其中 `0` 会转换为 `false`，非 `0` 会转换为 `true`。例如 `if(1)` 会进入分支，`if(0)` 不会进入分支，所以题目说法正确。

## 第6题

在下面的 C++ 代码 `while(1) continue;` 中，由于循环中的 continue 是无条件被执行，因此将导致死循环。

答案：正确

解析：`while(1)` 的循环条件恒为真，循环体只有 `continue;`。每次执行到 `continue` 时，程序会直接跳到下一轮循环条件判断，而条件仍然是 `1`。循环中没有任何改变条件的语句，也没有 `break`，因此会一直重复执行，形成死循环。

## 第7题

C++ 表达式 `('1'+'1'=='2'? flag=1:flag=2)` 的结果值和表达式 `(flag==2)` 的相同。

答案：错误

解析：字符参与加法时使用字符编码值。字符 `'1'` 的 ASCII 值是 `49`，所以 `'1' + '1'` 的结果是 `98`，不等于字符 `'2'` 的值 `50`。因此三目运算符会执行 `flag = 2`，这个赋值表达式的结果值是 `2`；而表达式 `(flag == 2)` 的结果是布尔真，按数值输出时通常表现为 `1`。一个结果值为 `2`，一个为真值 `1`，所以二者不相同。

## 第8题

在 C++ 的程序中，不能用 `scanf` 作为变量名。

答案：错误

解析：`scanf` 是 C 标准库函数名，但它不是 C++ 关键字，也不是像 `int`、`for` 那样被语言语法保留的词。只要不在同一作用域里造成重定义冲突，程序员可以把它作为普通标识符使用，例如在函数内部定义局部变量 `int scanf;`。因此“不能用 `scanf` 作为变量名”的说法过于绝对，是错误的。

## 第9题

小杨最近在准备考 GESP，他用的 Dev C++ 来练习和运行程序，所以 Dev C++ 也是一个小型操作系统。

答案：错误

解析：Dev C++ 是用于编写、编译和运行 C++ 程序的集成开发环境，也就是 IDE。它需要运行在 Windows 等操作系统之上。操作系统负责管理硬件、进程、内存、文件等基础资源，而 Dev C++ 只是面向程序开发的应用软件，二者不是同一类软件。

## 第10题

任何一个 while 循环都可以转化为等价的 for 循环。

答案：正确

解析：`while(条件) 语句` 可以写成 `for(; 条件; ) 语句`。二者都是先判断条件，条件为真时执行循环体，执行完再回到条件判断。由于 `for` 的初始化部分和更新部分都可以省略，所以普通 `while` 循环可以转化为等价的 `for` 循环，题目说法正确。
