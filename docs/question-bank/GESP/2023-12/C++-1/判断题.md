# 判断题

来源：
- 洛谷有题页：https://ti.luogu.com.cn/problemset/1133
- 官网原卷 PDF：https://gesp.ccf.org.cn/101/attach/1585703601307680.pdf

说明：本文件优先采用洛谷结构化数据整理，并以官网 PDF 做交叉校对。

## 第1题

C++ 表达式 `int(3.14)` 的值为 3。

答案：正确

解析：`int(3.14)` 是把浮点数 `3.14` 显式转换为 `int`。C++ 中浮点数转整数会舍去小数部分，结果为 `3`，因此说法正确。

## 第2题

C++ 语句 `cout << (2 * 3, 3 % 10, 2+3)` 的输出为 6,3,5。

答案：错误

解析：括号内使用的是逗号表达式，会从左到右计算 `2 * 3`、`3 % 10`、`2 + 3`，但整个表达式的值只取最后一项 `5`。`cout` 只会输出 `5`，不会输出中间两个值，也不会自动加逗号。

## 第3题

`for(int i = 1; i < 10; i += 3);` 表示从 1 开始到 10 结束间隔为 3，相当于 1、4、7、10。

答案：错误

解析：循环变量进入循环时依次为 `1`、`4`、`7`，之后执行 `i += 3` 变成 `10`，再判断 `i < 10` 为假，循环结束，所以 `10` 不会作为循环中的取值出现。末尾的分号还表示循环体为空，并没有执行其他语句。

## 第4题

C++ 表达式 `010+100+001` 的值为 111。

答案：错误

解析：以 `0` 开头的整数字面量按八进制解释，`010` 等于十进制的 `8`，`001` 等于十进制的 `1`，而 `100` 是十进制的 `100`。三者相加为 `109`，不是 `111`。

## 第5题

if 语句中的条件表达式的结果可以为 int 类型。

答案：正确

解析：`if` 的条件不要求表达式本身就是 `bool` 类型，只要能转换为布尔值即可。`int` 类型可以用于条件判断，其中 `0` 表示假，非 `0` 表示真。

## 第6题

在下面的 C++ 代码 `while(1) continue;` 中，由于循环中的 continue 是无条件被执行，因此将导致死循环。

答案：正确

解析：`while(1)` 的循环条件恒为真，循环体只有 `continue;`，它会立即跳到下一次循环条件判断。循环中没有任何改变条件的语句，也没有 `break`，因此会一直重复执行。

## 第7题

C++ 表达式 `('1'+'1'=='2'? flag=1:flag=2)` 的结果值和表达式 `(flag==2)` 的相同。

答案：错误

解析：字符 `'1'` 的 ASCII 值是 `49`，所以 `'1' + '1'` 为 `98`，不等于字符 `'2'` 的值 `50`。条件表达式会执行 `flag = 2`，该赋值表达式的结果值是 `2`；而随后比较 `(flag == 2)` 的结果是布尔真，按数值看为 `1`，两者不相同。

## 第8题

在 C++ 的程序中，不能用 `scanf` 作为变量名。

答案：错误

解析：`scanf` 是标准库函数名，但不是 C++ 关键字或保留的变量名。只要不在同一作用域里造成重定义冲突，程序员可以把它作为普通标识符使用，例如局部变量名可以遮蔽外层函数名。

## 第9题

小杨最近在准备考 GESP，他用的 Dev C++ 来练习和运行程序，所以 Dev C++ 也是一个小型操作系统。

答案：错误

解析：Dev C++ 是用于编写、编译和运行 C++ 程序的集成开发环境，运行在 Windows 等操作系统之上。操作系统负责管理硬件、进程、文件等资源，二者不是同一类软件。

## 第10题

任何一个 while 循环都可以转化为等价的 for 循环。

答案：正确

解析：`while(条件) 语句` 可以写成 `for(; 条件; ) 语句`，二者都是先判断条件，条件为真时执行循环体，执行完再回到条件判断。由于 `for` 的初始化和更新部分都可以省略，所以普通 `while` 循环可以转成等价的 `for` 循环。
