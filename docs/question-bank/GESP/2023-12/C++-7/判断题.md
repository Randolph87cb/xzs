## 第1题

小杨这学期准备参加 GESP 的 7 级考试，其中有关于三角函数的内容，他能够通过下面的代码找到结束循环的角度值。（ ）

```cpp
int main()
{
    double x;

    do {
        cin >> x;
        x = x / 180 * 3.14;
    } while (int(sin(x) * sin(x) + cos(x) * cos(x)) == 1);
    cout << "//" << sin(x) << " " << cos(x);

    cout << endl;
    return 0;
}
```

答案：正确

解析：本题判断为正确。数学上恒有 $\sin^2 x+\cos^2 x=1$，但程序使用 `double` 和近似圆周率 `3.14` 计算，`sin(x) * sin(x) + cos(x) * cos(x)` 的结果可能因浮点误差略小于 $1$。此时再强制转换为 `int` 会截断为 $0$，循环条件 `== 1` 变为假，程序就会结束。因此它确实可能通过输入某些角度值找到结束循环的情况；这也说明浮点数不应直接拿来依赖精确等式。

## 第2题

小杨在开发画笔刷小程序（applet），操作之一是选中黄颜色，然后在下面的左图的中间区域双击后，就变成了右图。这个操作可以用图的泛洪算法来实现。（  ）

![](https://cdn.luogu.com.cn/upload/image_hosting/6hhmbnta.png)

答案：正确

解析：本题判断为正确。泛洪算法（flood fill）就是从一个起点出发，把与起点连通、且满足同一填充条件的区域整体改成目标颜色。图中从中间黄色格子开始，沿可连通的空白区域扩展，遇到黑色边界就停止，最终形成右图中一整片连通的黄色区域。这正是画图软件中“油漆桶/填充”操作的典型实现方式。

## 第3题

假设一棵完全二叉树共有 $N$ 个节点，则树的深度为 $\log(N) + 1$。（ ）

答案：错误

解析：本题判断为错误。完全二叉树的深度不是简单的 $\log(N)+1$。若根结点深度按 $1$ 计，深度应为 $\lfloor \log_2 N\rfloor+1$，也可写成满足 $2^{h-1}\le N\le 2^h-1$ 的整数 $h$。题干既没有说明对数底数，也没有取整；例如 $N=3$ 时深度为 $2$，而 $\log_2 3+1$ 不是整数深度，所以该说法错误。

## 第4题

给定一个数字序列 $A_1,\ A_2,\ A_3,\ \ldots,\ A_n$，要求 $i$ 和 $j$（$1 \leq i \leq j \leq n$），使 $A_i + \ldots + A_j$ 最大，可以使用动态规划方法来求解。（ ）

答案：正确

解析：本题判断为正确。这是经典的最大连续子段和问题，可以用动态规划求解。令 $dp[i]$ 表示“必须以 $A_i$ 结尾的最大连续子段和”，则 $dp[i]=\max(A_i,dp[i-1]+A_i)$：要么从 $A_i$ 重新开始，要么接在前一个最优连续段后面。所有 $dp[i]$ 的最大值就是答案，因此题干说可以使用动态规划是正确的。

## 第5题

若变量 $x$ 为 `double` 类型正数，则 $\log(\exp(x)) > \log_{10}(x)$。（ ）

答案：正确

解析：本题判断为正确。按数学含义，$\log(\exp(x))=x$，而右边是 $\log_{10}(x)$。当 $0<x<1$ 时，$\log_{10}(x)<0$，而 $x>0$，不等式成立；当 $x\ge 1$ 时，$\log_{10}(x)$ 的增长远慢于 $x$，也有 $x>\log_{10}(x)$。在 C++ 教材口径下，`log` 表示自然对数，`exp` 是其反函数，所以题干判断为真。实际机器上极大 $x$ 可能使 `exp(x)` 溢出为无穷，但这仍不改变本题按数学函数考查时的结论。

## 第6题

简单有向图有 $n$ 个顶点和 $e$ 条弧，可以用邻接矩阵或邻接表来存储，二者求节点 $u$ 的度的时间复杂度一样。（ ）

答案：错误

解析：本题判断为错误。用邻接矩阵存有向图时，求顶点 $u$ 的出度需要扫描第 $u$ 行，求入度需要扫描第 $u$ 列，复杂度通常是 $O(n)$。用邻接表时，出度可直接扫描 $u$ 的出边链表，代价与出度有关；若还要求入度，通常要扫描相关弧表，代价与弧数有关。两种存储结构的访问方式不同，不能说求度的时间复杂度一样。

## 第7题

某个哈希表键值 $x$ 为整数，为其定义哈希函数 $H(x) = x \% p$，则 $p$ 选择素数时不会产生冲突。（ ）

答案：错误

解析：本题判断为错误。哈希函数取模后不同键仍可能得到相同值，表长或模数为素数只能降低某些冲突概率，不能保证无冲突。

## 第8题

动态规划只要推导出状态转移方程，就可以写出递归程序来求出最优解。（ ）

答案：错误

解析：本题判断为错误。状态转移方程只是动态规划的一部分，还必须明确状态含义、边界条件、计算顺序，以及是否存在重复子问题和最优子结构。若直接写递归而没有记忆化或正确边界，可能重复计算、无限递归，或者在有环依赖时根本无法得到正确结果。因此不能说“只要”推导出状态转移方程，就一定可以写出递归程序求最优解。

## 第9题

广度优先搜索（BFS）能够判断图是否连通。（ ）

答案：正确

解析：本题判断为正确。对于无向图，从任意一个顶点开始做 BFS，可以访问到与它处在同一连通分量中的全部顶点；遍历结束后检查是否所有顶点都被访问，就能判断图是否连通。若题目讨论有向图，则还需区分强连通、弱连通等定义；但在本题通常的“图是否连通”教材语境下，BFS 可以完成判断。

## 第10题

在 C++ 中，如果定义了构造函数，则创建对象时先执行完缺省的构造函数，再执行这个定义的构造函数。（ ）

答案：错误

解析：本题判断为错误。创建对象时只会按匹配规则调用一个构造函数来初始化该对象，不会先执行“缺省的构造函数”再执行用户定义的构造函数。若程序员定义了有参构造函数，编译器甚至可能不再自动生成无参默认构造函数；若定义了无参构造函数，它本身就是该类的默认构造函数。对象的成员和基类会在构造函数体执行前初始化，但这不是“先执行缺省构造函数再执行这个构造函数”。
