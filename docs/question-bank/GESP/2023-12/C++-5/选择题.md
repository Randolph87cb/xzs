## 第1题

下面 C++ 代码用于求斐波那契数列，该数列第 1、2 项为 1，以后各项均是前两项之和。下面有关说法错误的是（ ）。
```cpp
int fiboA(int N)
{
    if (N == 1 || N == 2)
        return 1;
    return fiboA(N - 1) + fiboA(N - 2);
}
int fiboB(int N)
{
    if (N == 1 || N == 2)
        return 1;
    int last2 = 1, last1 = 1;
    int nowVal = 0;
    for (int i = 2; i < N; i++)
    {
        nowVal = last1 + last2;
        last2 = last1;
        last1 = nowVal;
    }
    return nowVal;
}
```

A. fiboA() 用递归方式，fiboB() 循环方式
B. fiboA() 更加符合斐波那契数列的数学定义，直观易于理解，而 fiboB() 需要将数学定义转换为计算机程序实现
C. fiboA() 不仅仅更加符合数学定义，直观易于理解，且因代码量较少执行效率更高
D. fiboB() 虽然代码量有所增加，但其执行效率更高

答案：C

解析：`fiboA()` 直接按 $F(N)=F(N-1)+F(N-2)$ 递归，形式上更接近数学定义；`fiboB()` 用循环保存前两项并逐步推出当前项。递归版会重复计算大量相同子问题，例如计算 `fiboA(5)` 时会多次计算 `fiboA(3)`、`fiboA(2)`，效率远低于线性循环版。代码短不等于执行效率高，因此错误说法是 C；A、B、D 都符合两段代码的实际特点。

## 第3题

阅读下面的 C++ 代码，执行后其输出是（ ）。
```cpp
int stepCount = 0;
int fracA(int N)
{
    stepCount += 1;
    cout << stepCount << "->";
    int rtn = 1;
    for (int i = 1; i <= N; i++)
        rtn *= i;
    return rtn;
}
int fracB(int N)
{
    stepCount += 1;
    cout << stepCount << "->";
    if (N == 1)
        return 1;
    return N * fracB(N - 1);
}
int main()
{
    cout << fracA(5);
    cout << "<==>";
    cout << fracB(5);
    return 0;
}
```

A. 1->120<==>2->120
B. 1->120<==>1->120
C. 1->120<==>1->2->3->4->5->120
D. 1->120<==>2->3->4->5->6->120

答案：D

解析：本题考查全局变量状态和递归调用顺序。`stepCount` 是全局变量，初值为 `0`。先执行 `fracA(5)`，它只进入一次函数，把 `stepCount` 加到 `1` 并输出 `1->`，循环算出 $5!=120$，所以前半段是 `1->120`。随后输出 `<==>`。再执行 `fracB(5)`，递归调用参数依次为 `5,4,3,2,1`，每次进入函数都先把 `stepCount` 加一并输出，所以依次输出 `2->3->4->5->6->`，递归返回值仍为 $120$。合起来是 D。

## 第6题

有关下面 C++ 代码说法正确的是（ ）。
```cpp
int rc;
int foo(int x, int y)
{
    int r;
    if(y == 0)
        r = x;
    else{
        r = foo(y, x % y);
        rc++;
    }
    return r;
}
```

A. 如果 $x$ 小于 $10$，$rc$ 值也不会超过 $20$
B. `foo` 可能无限递归
C. `foo` 可以求出 $x$ 和 $y$ 的最大公共质因子
D. `foo` 能够求出 $x$ 和 $y$ 的最小公倍数

答案：A

解析：`foo(x,y)` 是欧几里得算法：当 `y==0` 时返回 `x`，否则递归计算 `foo(y, x % y)`，返回的是最大公约数。每发生一次非终止递归，`rc` 在返回后加一，记录递归缩减的次数。对非负整数输入，余数会逐步变小，不会无限递归；它求的是最大公约数，不是最大公共质因子，也不是最小公倍数。若 $x<10$，进入以较小数为余数的欧几里得过程后步数很少，`rc` 不会超过 20，因此 A 正确。

## 第8题

下面C++代码中的 `isPrimeA()` 和 `isPrimeB()` 都用于判断参数 $N$ 是否素数，有关其时间复杂度的正确说法是（ ）。

```cpp
bool isPrimeA(int N)
{
    if (N < 2)
        return false;
    for (int i = 2; i <= N / 2; i++)
        if (N % i == 0)
            return false;
    return true;
}

bool isPrimeB(int N)
{
    if (N < 2)
        return false;
    for (int i = 2; i <= sqrt(N); i++)
        if (N % i == 0)
            return false;
    return true;
}
```

A. `isPrimeA()` 的最坏时间复杂度是 $O(\frac{N}{2})$，`isPrimeB()` 的最坏时间复杂度是 $O(\log N)$。`isPrimeA()` 优于 `isPrimeB()`
B. `isPrimeA()` 的最坏时间复杂度是 $O(\frac{N}{2})$，`isPrimeB()` 的最坏时间复杂度是 $O(N^{\frac{1}{2}})$，`isPrimeB()` 绝大多数情况下优于 `isPrimeA()`
C. `isPrimeA()` 的最坏时间复杂度是 $O(N^{\frac{1}{2}})$，`isPrimeB()` 的最坏时间复杂度是 $O(N)$，`isPrimeA()` 优于 `isPrimeB()`
D. `isPrimeA()` 的最坏时间复杂度是 $O(\log N)$，`isPrimeB()` 的最坏时间复杂度是 $O(N)$，`isPrimeA()` 优于 `isPrimeB()`

答案：B

解析：`isPrimeA()` 最坏情况下要从 `2` 检查到 `N/2`，例如 $N$ 为较大的素数时不会提前返回，时间复杂度数量级为 $O(N)$，写成 $O(\frac N2)$ 也表示同一数量级。`isPrimeB()` 只检查到 $\sqrt N$，因为若 $N$ 有一个大于 $\sqrt N$ 的因数，必然还配有一个小于 $\sqrt N$ 的因数，所以最坏时间复杂度为 $O(N^{1/2})$。因此 B 正确；把 $\sqrt N$ 说成 $\log N$ 或把两者优劣反过来都不对。

## 第9题

下面 C++ 代码用于有序 list 的二分查找，有关说法错误的是（ ）。
```cpp
int _binarySearch(vector<int> lst, int Low, int High, int Target)
{
    if (Low > High)
        return -1;
    int Mid = (Low + High) / 2;
    if (Target == lst[Mid])
        return Mid;
    else if (Target < lst[Mid])
        return _binarySearch(lst, Low, Mid - 1, Target);
    else
        return _binarySearch(lst, Mid + 1, High, Target);
}

int bSearch(vector<int> lst, int Val)
{
    return _binarySearch(lst, 0, lst.size(), Val);
}
```

A. 代码采用二分法实现有序 list 的查找
B. 代码采用分治算法实现有序 list 的查找
C. 代码采用递归方式实现有序 list 的查找
D. 代码采用动态规划算法实现有序 list 的查找

答案：D

解析：`_binarySearch` 每次取中点 `Mid`，若目标更小就递归查左半段，更大就递归查右半段，这是典型二分查找；把原问题拆成一个更小区间继续求解，也属于分治思想；实现形式上使用了递归。因此 A、B、C 都是正确描述。动态规划要求保存并复用重叠子问题的结果，这段代码没有状态表或记忆化过程，所以错误说法是 D。另需注意，`bSearch` 把上界传成 `lst.size()` 而不是 `lst.size()-1` 有越界风险，但这不改变本题所问的算法类型。

## 第12题

有关下面 C++ 代码的说法正确的是（ ）。

## 第15题

下面的排序算法都要处理多趟数据，哪种排序算法不能保证在下一趟处理时从待处理数据中选出最大或最小的数据？（ ）

A. 选择排序
B. 快速排序
C. 堆排序
D. 冒泡排序

答案：B

解析：选择排序每一趟从未排序部分选出最小或最大元素放到边界；堆排序每一趟从堆顶取出当前最大或最小元素；冒泡排序每一趟相邻交换后也会把当前最大或最小元素“冒”到一端。快速排序每一趟是选一个枢轴做划分，只保证枢轴左侧不大于它、右侧不小于它，并不保证枢轴就是当前待处理数据中的最大或最小值，所以选 B。
