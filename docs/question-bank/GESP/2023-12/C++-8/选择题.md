## 第1题

小杨要从 A 城到 B 城，又想顺路游览一番。他有两个选项：1、坐高铁路到 C 城游览，再坐高铁或飞机到 B 城；2、坐船到 D 城游览，再坐船、高铁或飞机到 B 城。请问小杨从 A 城到 B 城共有几种交通方案可以选择？（ ）。

A. $2$
B. $3$
C. $5$
D. $6$

答案：C

解析：本题选 C。按路线分类计数：方案 1 是 A 到 C，再从 C 到 B，可选高铁或飞机，共 $2$ 种；方案 2 是 A 到 D，再从 D 到 B，可选船、高铁或飞机，共 $3$ 种。两类方案互不重叠，因此总数为 $2 + 3 = 5$。A、B 少算了其中一类后续交通方式，D 则把两类路线错误地相乘或重复计数。

## 第2题

以下哪个函数声明是符合语法的，且在调用时可以将二维数组的名字作为实际参数传递给形式参数 $a$？（ ）。

A. `void QuickSort(int a[][10], int n);`
B. `void QuickSort(int a[5][], int m);`
C. `void QuickSort(int a[][], int n, int m);`
D. `void QuickSort(int ** a, int n, int m);`

答案：A

解析：本题选 A。二维数组作为函数参数时，第一维长度可以省略，但除第一维外的维度必须给出，否则编译器无法根据下标计算元素地址，所以 `int a[][10]` 合法。B、C 都省略了第二维长度，不符合二维数组形参语法。D 的 `int **a` 是“指向指针的指针”，不能直接接收普通二维数组名，因为普通二维数组名退化后类型类似 `int (*)[列数]`，二者内存解释方式不同。

## 第3题

下面有关 C++ 类和对象的说法，错误的是（ ）。

A. 对象的生命周期开始时，会执行构造函数。
B. 对象的生命周期结束时，会执行析构函数。
C. 类的析构函数可以为虚函数。
D. 类的构造函数可以为虚函数。

答案：D

解析：本题选 D。构造函数在对象生命周期开始时被调用，析构函数在对象生命周期结束时被调用，因此 A、B 正确。析构函数可以声明为虚函数，常用于通过基类指针删除派生类对象时保证析构完整，因此 C 正确。构造对象时对象的准确类型已经确定，C++ 不支持虚构造函数，所以 D 是错误说法。

## 第4题

使用邻接矩阵表达 $n$ 个顶点的有向图，则该矩阵的大小为（ ）。

A. $n \times (n + 1)$
B. $n \times n$
C. $n \times (n - 1)$
D. $n \times (n - 1)/2$

答案：B

解析：本题选 B。邻接矩阵用行、列分别表示起点和终点，对 $n$ 个顶点需要为每一对有序顶点 $(u, v)$ 留出一个位置，因此大小是 $n \times n$。A 多出一列，C 少了一列，D 是无向简单图只存上三角时可能出现的压缩计数，不能表示一般有向图的邻接矩阵。

## 第5题

$5$ 位同学排队，其中一位同学不能排在第一，则共有多少种可能的排队方式？（ ）。

A. $5$
B. $24$
C. $96$
D. $120$

答案：C

解析：本题选 C。设受限制的同学为甲。第一位不能放甲，所以第一位有 $4$ 种选择；剩下 $4$ 位同学依次排列，有 $4 \times 3 \times 2 \times 1 = 24$ 种。总数为 $4 \times 24 = 96$。D 是没有限制时的 $5! = 120$，B 只计算了甲固定不在某一位后的剩余排列，A 只是人数。

## 第6题

一个无向图包含 $n$ 个顶点，则其最小生成树包含多少条边？（ ）。

A. $n - 1$
B. $n$
C. $n + 1$
D. 最小生成树可能不存在。

答案：D

解析：本题选 D。若无向图连通，则它的生成树包含全部 $n$ 个顶点且有 $n - 1$ 条边，最小生成树也是如此。但题干只说“一个无向图包含 $n$ 个顶点”，没有保证连通；不连通图不存在覆盖所有顶点的生成树，也就不存在最小生成树。因此不能直接选 A，B、C 也不符合树的边数性质。

## 第7题

已知三个 `double` 类型的变量 $a$、$b$ 和 $theta$ 分别表示一个三角形的两条边长及二者的夹角（弧度），则下列哪个表达式可以计算这个三角形的面积？（ ）。

A. `a * b * sin(theta) / 2`
B. `(a + b) * sin(theta) / 2`
C. `a * b * cos(theta) / 2`
D. `sqrt(a * a + b * b - 2 * a * b * cos(theta))`

答案：A

解析：本题选 A。已知两边及夹角时，三角形面积公式为 $S = \frac{1}{2}ab\sin\theta$，对应表达式 `a * b * sin(theta) / 2`。B 使用的是两边和而不是两边乘积；C 把正弦误写成余弦；D 根据余弦定理计算的是第三边长度，不是面积。

## 第8题

对有 $n$ 个元素的二叉排序树进行中序遍历，其时间复杂度是（ ）。

A. $O(1)$
B. $O(\log(n))$
C. $O(n)$
D. $O(n^2)$

答案：C

解析：本题选 C。中序遍历会对二叉排序树中的每个结点访问一次，访问结点本身、递归进入左右子树的总工作量与结点数成正比，因此时间复杂度为 $O(n)$。A、B 低估了遍历全部结点的代价；D 高估了代价，中序遍历不会对每个结点再扫描整棵树。

## 第9题

假设输入参数 $m$ 和 $n$ 满足 $m \leq n$，则下面程序的最差情况的时间复杂度为（ ）。

```cpp
int gcd(int m, int n) {
    while (m > 0) {
        int t = m;
        m = n % m;
        n = t;
    }
    return n;
}
```

A. $O(\log(n))$
B. $O(n)$
C. $O(n \times m)$
D. $O(m \times \log(n))$

答案：A

解析：本题选 A。程序是辗转相除法：每轮把 $(m, n)$ 更新为 $(n \bmod m, m)$，较小的参数会快速变小。最差情况出现在相邻斐波那契数一类输入，此时每轮大致退到前一项，循环次数与数值大小的对数同阶，所以为 $O(\log n)$。B、C、D 都把它看成线性枚举或双重枚举，明显高估了循环次数。

## 第10题

下面程序的时间复杂度为（ ）。

```cpp
long long power_mod(long long a, long long n, long long mod) {
    if (n == 0)
        return 1;
    a = a % mod;
    if (n == 1)
        return a;
    long long pw = power_mod(a, n / 2, mod);
    long long pw2 = pw * pw % mod;
    if (n % 2 == 0)
        return pw2;
    return pw2 * a % mod;
}
```

A. $O(n)$
B. $O(a^n)$
C. $O(\log(n))$
D. $O(\log(n) \times a)$

答案：C

解析：本题选 C。函数每次递归只调用一次 `power_mod(a, n / 2, mod)`，之后只做常数次乘法、取模和判断，因此递推式是 $T(n) = T(n / 2) + O(1)$，解得 $T(n) = O(\log n)$。A 把指数逐次减一才会得到；B 是朴素枚举所有乘积的误判；D 中的 $a$ 只参与常数次算术运算，不影响递归层数。

## 第11题

下面程序的时间复杂度为（ ）。

```cpp
int record_choose[MAXN][MAXM];
int choose(int n, int m) {
    if (m == 0 || m == n)
        return 1;
    if (record_choose[n][m] == 0)
        record_choose[n][m] = choose(n - 1, m - 1) + choose(n - 1, m);
    return record_choose[n][m];
}
```

A. $O(2^n)$
B. $O(2^m \times (n - m))$
C. $O(C(n, m))$
D. $O(m \times (n - m))$

答案：D

解析：本题选 D。递归计算组合数时加入了 `record_choose` 记忆化，同一个状态 `(n, m)` 只会真正计算一次，所以不是指数级。能被访问的状态集中在从 `(n, m)` 向边界 `m == 0` 或 `m == n` 展开的三角区域内，规模与两条边界之间的宽度乘积同阶，即 $O(m \times (n - m))$。A、B、C 都更接近未记忆化递归或按组合数个数展开的估计，忽略了缓存去重。

## 第12题

下面的程序使用出边的邻接表表达有向图，则下列选项中哪个是它表达的图？（ ）。

```cpp
#include <iostream>

struct Edge {
    int e;
    Edge * next;
};

struct Node {
    Edge * first;
};

int main() {
    Edge e[5] = {{1, nullptr}, {2, &e[2]},
                 {3, nullptr}, {3, nullptr}, {0, nullptr}};
    Node n[4] = {&e[0], &e[1], &e[3], &e[4]};
    // 其他处理
    return 0;
}
```

A. A
B. B
C. C
D. D

答案：B

解析：本题选 B。`Node::first` 指向该顶点的第一条出边，`Edge::next` 指向同一顶点的下一条出边。由初始化可得：顶点 0 的链表是 `e[0]`，有边 $0 \to 1$；顶点 1 的链表是 `e[1] -> e[2]`，有边 $1 \to 2$、$1 \to 3$；顶点 2 的链表是 `e[3]`，有边 $2 \to 3$；顶点 3 的链表是 `e[4]`，有边 $3 \to 0$。只有图 B 同时包含这些有向边，其他图的方向或出边集合不一致。

## 第13题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int cnt = 0;
    for (int a = 1; a <= 10; a++)
        for (int b = 1; b <= 10; b++)
            for (int h = 1; h <= 10; h++)
                if ((a + b) * h == 20)
                    cnt++;
    cout << cnt << endl;
    return 0;
}
```

A. 12
B. 18
C. 36
D. 42

答案：B

解析：本题选 B。程序统计 $1 \le a,b,h \le 10$ 且 $(a + b)h = 20$ 的三元组。$h$ 必须是 $20$ 的因子且不超过 $10$，可能为 $1,2,4,5,10$。对应 $a+b$ 分别为 $20,10,5,4,2$，在 $1$ 到 $10$ 内的有序正整数对数量分别为 $1,9,4,3,1$，总数为 $1+9+4+3+1=18$。A、C、D 都没有正确处理有序对数量或 $h$ 的取值限制。

## 第14题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    const int N = 30;
    int cnt = 0;
    for (int a = 1; a <= N; a++)
        for (int b = a; a + b <= N; b++)
            for (int c = b; a + b + c <= N; c++)
                if (a * a + b * b == c * c)
                    cnt++;
    cout << cnt << endl;
    return 0;
}
```

A. 3
B. 6
C. 11
D. 22

答案：A

解析：本题选 A。循环条件保证 $1 \le a \le b \le c$ 且 $a+b+c \le 30$，因此每个直角三角形边长组只统计一次。满足 $a^2+b^2=c^2$ 且周长不超过 $30$ 的正整数三元组为 $(3,4,5)$、$(5,12,13)$、$(6,8,10)$，共 $3$ 个。B、C、D 都来自重复统计排列或把周长超过 $30$ 的勾股数组也算入。

## 第15题

下面的程序中，二维数组 $h$ 和 $v$ 分别代表如下图所示的网格中的水平边的时间消耗和垂直边的时间消耗。程序使用动态规划计算从左下角到右上角的最小时间消耗，则横线处应该填写下列哪个选项的代码？（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/ia7a4flz.png)

```cpp
int dis[MAXY][MAXX];
int shortest_path(int x, int y) {
    dis[0][0] = 0;
    for (int i = 0; i < y; i++)
        dis[i + 1][0] = dis[i][0] + v[i][0];
    for (int j = 0; j < x; j++)
        dis[0][j + 1] = dis[0][j] + h[0][j];
    for (int i = 0; i < y; i++)
        for (int j = 0; j < x; j++)
            __________________; // 在此处填写代码
    return dis[y][x];
}
```

A. `dis[i][j] = min(dis[i - 1][j] + v[i - 1][j], dis[i][j - 1] + h[i][j - 1]);`    
B. `dis[i][j] = min(dis[i - 1][j] + h[i - 1][j], dis[i][j - 1] + v[i][j - 1]);`    
C. `dis[i + 1][j + 1] = min(dis[i][j + 1] + v[i][j + 1], dis[i + 1][j] + h[i + 1][j]);`     
D. `dis[i + 1][j + 1] = min(dis[i][j + 1] + h[i][j + 1], dis[i + 1][j] + v[i + 1][j]);`

答案：C

解析：本题选 C。前两段初始化已经算出左边界 `dis[*][0]` 和下边界 `dis[0][*]`，内层循环的 `i < y`、`j < x` 应继续计算内部点 `dis[i + 1][j + 1]`，所以 A、B 的目标下标不对，还会在 `i=0` 或 `j=0` 时出现负下标。到达 `dis[i + 1][j + 1]` 只有两种最后一步：从下方 `dis[i][j + 1]` 走一条垂直边，代价为 `v[i][j + 1]`；或从左侧 `dis[i + 1][j]` 走一条水平边，代价为 `h[i + 1][j]`。这正是 C；D 把水平边和垂直边数组用反了。
