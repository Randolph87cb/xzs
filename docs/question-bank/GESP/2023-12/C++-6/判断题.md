## 第1题

在面向对象中，方法在 C++ 的 class 中表现为 class 内定义的函数。（ ）

答案：正确

解析：本题判断为正确。面向对象中的“方法”通常指对象能够执行的操作，在 C++ 的 `class` 中通常表现为成员函数。成员函数可以在类体内定义，也可以在类体内声明、再在类外用作用域限定符定义；无论哪种写法，都是类提供给对象使用的行为描述。因此题干说法符合 C++ 类的基本概念。

## 第2题

C++ 类的定义中，可以没有构造函数，会给出默认的构造函数（ ）

答案：正确

解析：本题判断为正确。一个类可以不显式写出构造函数；在没有用户声明构造函数等特定前提下，编译器会隐式声明默认构造函数，使对象可以被默认构造。需要注意的是，如果程序员声明了带参数构造函数等成员，是否还会自动得到可直接调用的无参默认构造函数要按 C++ 规则具体分析。本题按“类中可以不写构造函数，编译器可提供默认构造”的基础口径判断为正确。

## 第3题

如果某个 C++ 对象（object）支持下标运算符（方括号运算符），则该对象在所对应 class 中以成员函数的形式进行了重载。（ ）

答案：正确

解析：本题判断为正确。C++ 中下标运算符 `operator[]` 是必须作为非静态成员函数重载的运算符，不能像某些二元运算符那样写成普通非成员函数重载。对象能够使用 `obj[index]`，本质上就是调用该对象所属类中的 `operator[]` 成员函数。因此题干说法正确。

## 第4题

深度优先搜索（DFS，Depth First Search 的简写）属于图算法，其过程是对每一个可能的分支路径深入到不能再深入为止，而且每个节点只能访问一次。（ ）

答案：正确

解析：本题判断为正确。DFS 的核心思想是从一个节点出发，沿着某一条分支尽量向深处访问，直到不能继续再回溯去尝试其他分支。对于图来说，为避免环导致反复访问，通常会用 `visited` 数组标记已经访问过的节点，因此每个节点在一次遍历中只会被访问一次。若图不连通，则需要从尚未访问的节点再次启动 DFS 才能覆盖所有连通分量；这不影响题干对 DFS 过程的描述。

## 第5题

哈夫曼编码（Huffman Coding）具有唯一性，因此有确定的压缩率。（ ）

答案：错误

解析：本题判断为错误。哈夫曼编码只要求高频字符使用较短编码、低频字符使用较长编码，并通过前缀码保证可解码；当多个字符频率相同或合并时出现并列选择时，可能构造出不同的哈夫曼树和不同的具体编码，所以编码本身不具有唯一性。即使同一组频率下最优总长度可能相同，实际压缩率还会受字符频率、原始编码方式、是否存储码表等因素影响，不能由“哈夫曼编码唯一”推出确定压缩率。

## 第6题

在下面 C++ 代码中，由于删除了变量 `ptr`，因此 `ptr` 所对应的数据也随之删除，故第 8 行代码被执行时，将报错。（ ）

```cpp
int * ptr = new int(10);
cout << *ptr << endl;
delete ptr;
cout << *ptr << endl; // 第 8 行
```

答案：错误

解析：本题判断为错误。`delete ptr;` 会释放 `ptr` 指向的动态分配整数对象，但不会把指针变量 `ptr` 本身删除，也不会自动把它改成 `nullptr`。删除后 `ptr` 成为悬空指针，再执行 `*ptr` 是未定义行为：程序可能崩溃、可能输出旧值、也可能表现出其他异常结果。C++ 标准并不保证第 8 行“一定报错”，所以题干说法错误。

## 第7题

二叉搜索树查找的平均时间复杂度为 $O(\log N)$。（ ）

答案：正确

解析：本题判断为正确。二叉搜索树查找时，每比较一个节点，就会根据关键字大小只进入左子树或右子树中的一边；在树形较均衡或随机构造的平均情况下，树高通常为 $O(\log N)$，查找复杂度也为 $O(\log N)$。需要区分的是最坏情况：如果二叉搜索树退化成链，查找会变成 $O(N)$。题干说的是平均时间复杂度，因此按教材常见口径判断为正确。

## 第8题

二叉搜索树可以是空树（没有任何节点）或者单节点树（只有一个节点），或者多节点。如果是多节点，则左节点的值小于父节点的值，右节点的值大于父节点的值，由此推理，右节点树的值都大于根节点的值，左节点树的值都小于根节点的值。（ ）

答案：正确

解析：本题判断为正确。二叉搜索树可以没有节点，也可以只有一个节点；当有多个节点时，其有序性要求递归地作用于整棵树：任一节点的左子树中所有值都小于该节点，右子树中所有值都大于该节点。因此从根节点看，左子树所有值都小于根节点，右子树所有值都大于根节点。题干中的“左节点树、右节点树”按“左子树、右子树”理解，结论正确。

## 第9题

小杨想写一个程序来算出正整数 $N$ 有多少个因数，经过思考他写出了一个重复没有超过 $N/2$ 次的循环就能够算出来了。（ ）

答案：正确

解析：本题判断为正确。正整数 $N$ 的真因数除 $N$ 自身外都不会大于 $N/2$，因为若 $d > N/2$ 且 $d < N$，则 $2d > N$，不可能再作为 $N$ 的正因数。因此可以循环检查 $1$ 到 $\lfloor N/2 \rfloor$ 是否整除 $N$，统计完后再把 $N$ 本身这个因数计入，循环次数不超过 $N/2$。当 $N=1$ 时循环次数为 0，也可以单独处理或最终计入因数 1。

## 第10题

同样的整数序列分别保存在单链表和双向链中，这两种链表上的简单冒泡排序的复杂度相同。（ ）

答案：正确

解析：本题判断为正确。简单冒泡排序的主要工作是多轮相邻元素比较和必要时交换，比较次数随元素个数呈二次增长，时间复杂度为 $O(N^2)$。同样的整数序列保存在单链表或双向链表中，双向链表多了前驱指针，能让向前移动更方便，但简单冒泡排序仍然需要相邻比较的多轮扫描，复杂度数量级不变。因此两种链表上的简单冒泡排序复杂度相同。
