## 第1题

关于 C++ 类和对象的说法，错误的是（ ）。

A. 在 C++ 中，一切皆对象，即便是字面量如整数 $5$ 等也是对象
B. 在 C++ 中，可以自定义新的类，并实例化为新的对象
C. 在 C++ 中，内置函数和自定义函数，都是类或者对象
D. 在 C++ 中，可以在自定义函数中嵌套定义新的函数

答案：D

解析：本题选 D。C++ 不支持在一个普通函数体内再定义新的普通函数，函数只能在命名空间作用域或类作用域等允许的位置定义，因此 D 直接违反 C++ 语法。B 说明可以自定义类并创建对象，这是类和对象的基本用法。A、C 是面向对象入门语境中的概括性说法，实际学习时还要区分字面量、函数和对象在 C++ 标准术语中的不同含义；相比之下，D 是最明确的错误项。

## 第2题

有关下面 C++ 代码的说法，错误的是（ ）。

```cpp
class Rectangle
{
private:
    class Point
    {
    public:
        double x;
        double y;
    };
    Point a, b, c, d;
    double length;
    double width;
public:
```

A. C++ 中类内部可以嵌套定义类
B. 在类中定义的类被称为内部类，定义类的类被称为外部类
C. 内部类可以随便访问，不需要通过外部类来访问
D. 代码中 `Point` 被称为内部类，可以通过外部类 `Rectangle` 来访问，`Rectangle::Point`

答案：C

解析：本题选 C。C++ 允许在类中嵌套定义类，所以 A 正确；嵌套在另一个类中的类通常可称为内部类，外层的类可称为外部类，所以 B 符合教材表述。内部类的名字仍然受访问控制和作用域限制，类外访问时通常需要用外部类名限定，例如 `Rectangle::Point`，且本代码中 `Point` 位于 `private` 区域，类外实际使用还会受到私有访问权限限制。因此“内部类可以随便访问，不需要通过外部类来访问”的 C 是错误说法。

## 第3题

有关下面 C++ 代码的说法，正确的是（ ）。

```cpp
using namespace std;
class newClass
{
public:
    static int objCounter;
};
int newClass::objCounter = 2;
int main()
{
    newClass classA;
    newClass classB;
    cout << newClass::objCounter << endl; // 第 14 行
    cout << classB.objCounter << endl; // 第 15 行
}
```

A. 第 14 行代码错误，第 15 行正确
B. 第 15 行代码错误，第 14 行代码正确
C. 第 14、15 两行代码都正确
D. 第 6 行代码可修改为 `objCounter += 1`

答案：C

解析：本题选 C。`objCounter` 是 `public` 的静态数据成员，属于类本身，类外已经用 `int newClass::objCounter = 2;` 完成定义和初始化。访问静态成员既可以写成 `newClass::objCounter`，也可以通过对象写成 `classB.objCounter`，所以第 14 行和第 15 行都能访问同一个静态成员。D 不正确，因为类内第 6 行是成员声明位置，不能把声明直接改成没有类型说明的 `objCounter += 1` 语句。

## 第4题

有关下面 C++ 代码的说法，错误的是（ ）。

```cpp
struct BiNode {
    char data;
    BiNode* lchid, *rchid;
};
class BiTree {
private:
    BiNode* Creat();
    void Release(BiNode* bt);
    BiNode* root;
public:
    BiTree() {
        root = Creat();
    }
    ~BiTree() {
        Release(root);
    }
};
```

A. 上列 C++ 代码适用于构造各种二叉树
B. 代码 `struct BiNode` 用于构造二叉树的节点
C. 代码 `BiTree(){root=Creat();}` 用于构造二叉树
D. 析构函数不可以省略

答案：D

解析：本题选 D。`struct BiNode` 保存节点数据和左右孩子指针，是二叉树节点结构，B 正确；`BiTree()` 构造函数中调用 `Creat()` 并把返回的根指针赋给 `root`，用于建立一棵二叉树，C 正确；用节点指针递归连接左右孩子，可以表示不同形态的二叉树，A 也可以成立。析构函数并不是语法上必须写出的成员函数，若省略，编译器会生成默认析构函数；只是本类若动态申请节点，省略释放逻辑可能造成内存泄漏。因此“析构函数不可以省略”说得过于绝对，是错误项。

## 第5题

基于第 4 题的定义，有关下面 C++ 代码的说法正确的是（ ）。

```cpp
void Order(BiNode* bt) {
    if (bt == nullptr)
        return;
    else {
        cout << bt->data;
        Order(bt->lchid);
        Order(bt->rchid);
    }
}
```

A. 代码中 `Order()` 函数是中序遍历二叉树的方法
B. 代码中 `Order()` 先访问根节点，然后对左子树进行前序遍历，再对右子树前序遍历
C. 代码中 `Order()` 先访问中序遍历左子树，然后访问根节点，最后则是中序遍历右子树
D. 代码中 `Order()` 先后序遍历左子树，然后后序遍历右子树，最后访问根节点

答案：B

解析：本题选 B。函数先判断 `bt == nullptr`，空树直接返回；非空时先执行 `cout << bt->data;` 访问当前根节点，然后递归调用 `Order(bt->lchid)` 处理左子树，最后递归调用 `Order(bt->rchid)` 处理右子树。访问顺序是“根、左、右”，这正是前序遍历。A、C 描述的是中序遍历的相关顺序，D 描述的是后序遍历顺序，都与代码中先输出根节点的行为不符。

## 第6题

有关下面 C++ 代码的说法正确的是（ ）。

```cpp
typedef struct LinkList {
    int data;
    LinkList* next;
    LinkList* prev;
} LinkList, LinkNode;
bool ListInit(LinkList* &L) {
    L = new LinkNode;
    if (!L) return false;
    L->next = NULL;
    L->prev = NULL;
    L->data = -1;
    return true;
}
```

A. 上述代码构成单向链表
B. 上述代码构成双向链表
C. 上述代码构成循环链表
D. 上述代码构成指针链表

答案：B

解析：本题选 B。结构体中每个节点除了 `data` 外，同时有 `next` 和 `prev` 两个指针，分别用于指向后继节点和前驱节点；初始化函数也把这两个指针都置为 `NULL`。只有一个方向指针时才是单向链表，所以 A 不对；循环链表需要尾节点和头节点相连，代码没有建立这种关系，所以 C 不对；“指针链表”不是这里与单向、双向、循环并列的明确分类。该结构体现的是双向链表。

## 第7题

对 `hello world` 使用霍夫曼编码（Huffman Coding），最少 bit（比特）为（ ）。

A. $4$
B. $32$
C. $64$
D. $88$

答案：B

解析：本题选 B。字符串 `hello world` 含 11 个字符，各字符频次为：`l` 出现 3 次，`o` 出现 2 次，`h`、`e`、空格、`w`、`r`、`d` 各出现 1 次。霍夫曼编码的最小带权路径长度可按合并代价计算：初始频次为 $1,1,1,1,1,1,2,3$，依次合并得到的代价为 $2,2,2,4,4,7,11$，总和为 $32$，所以最少需要 32 bit。4 bit 不可能表示 11 个字符的整段编码，64 bit 和 88 bit 分别接近固定长度编码或按字节存储的结果，不是霍夫曼最优编码长度。

## 第8题

下面的 `fiboA()` 和 `fiboB()` 两个函数分别实现斐波那契数列，该数列第 $1$、第 $2$ 项值为 $1$，其余各项分别为前两项之和。下面有关说法错误的是（ ）。

```cpp
int fiboA(int n)
{
    if (n == 0)
        return 1;
    if (n == 1)
        return 1;
    else
    {
        return fiboA(n - 1) + fiboA(n - 2);
    }
}

int fiboB(int n)
{
    if ((n == 0) || (n == 1)) {
        fiboB[n] = n;
        return n;
    }
    else {
        if (fiboB[n] == 0) {
            fiboB[n] = fiboB(n - 1) + fiboB(n - 2);
        }
        return fiboB[n];
    }
}
```

A. `fiboA()` 采用递归方式实现斐波那契数列
B. `fiboB()` 采用动态规划算法实现斐波那契数列
C. 当 N 值较大时，`fiboA()` 存在大量重复计算
D. 由于 `fiboA()` 代码较短，其执行效率较高

答案：D

解析：本题选 D。`fiboA()` 在函数内部直接调用 `fiboA(n - 1)` 和 `fiboA(n - 2)`，属于递归实现，A 正确；当 $N$ 较大时，同一个子问题会被重复求很多次，例如求 `fiboA(5)` 会多次重新求 `fiboA(3)`、`fiboA(2)`，所以 C 正确。`fiboB()` 的意图是用数组保存已经算过的项，只有缓存值为 0 时才递归计算，属于记忆化搜索/动态规划思想，B 按题目意图成立。代码短不代表效率高，`fiboA()` 的重复递归会导致效率明显低于带缓存的写法，因此 D 是错误说法。严格按 C++ 编译看，题面中的 `fiboB[n]` 还需要对应的缓存数组，且不能与函数同名；这里按官方题意理解为缓存数组访问。

## 第9题

有关下面 C++ 代码不正确的说法是（ ）。

```cpp
int Depth(BiTree T)
{
    if (T == NULL)
    {
        return 0;
    }
    else
    {
        int m = Depth(T->lchild);
        int n = Depth(T->rchild);
        if (m > n)
        {
            return m + 1;
        }
        else
        {
            return n + 1;
        }
    }
}
```

A. 该代码可用于求解二叉树的深度
B. 代码中函数 `Depth()` 的参数 T 表示根节点，非根节点不可以作为参数
C. 代码中函数 `Depth()` 采用了递归方法
D. 代码中函数 `Depth()` 可用于求解各种形式的二叉树深度，要求该二叉树节点至少有 `left` 和 `right` 属性

答案：B

解析：本题选 B。函数对空指针返回 0，对非空节点分别递归求左子树和右子树深度，再返回较大值加 1，因此可以求以参数 `T` 为根的子树深度，A、C 正确。这里的 `T` 不一定必须是整棵树的总根节点，传入任意非空节点指针时，函数同样会计算以该节点为根的子树深度，所以 B 错误。D 按算法思想理解为节点需要有左右孩子指针；在真实 C++ 代码中，成员名还必须和代码里的 `lchild`、`rchild` 等实际名字一致。

## 第10题

下面有关树的存储，错误的是（ ）。

A. 完全二叉树可以用 `list` 存储
B. 一般二叉树都可以用 `list` 存储，空子树位置可以用 `None` 表示
C. 满二叉树可以用 `list` 存储
D. 树数据结构，都可以用 `list` 存储

答案：D

解析：本题选 D。完全二叉树和满二叉树的节点位置与数组下标有固定对应关系，适合用顺序表或列表形式存储，所以 A、C 可以成立。一般二叉树也可以按层序位置存入列表，并用 `null`/`None` 标记空孩子位置，B 描述的是常见的二叉树顺序表示方法。D 把范围扩大到所有树数据结构，过于绝对；普通多叉树或一般树的孩子数量不固定，通常还要用孩子链表、父节点数组、左孩子右兄弟等表示方式，不能简单说都适合只用 `list` 直接存储。

## 第11题

构造二叉树 `[1,2,3,null,4]`（ ）。

A. `1(2()(4))(3)`
B. `1(2(3)())(4)`
C. `(1,2(3),(4))`
D. `(1,(2)(3),(4))`

答案：A

解析：本题选 A。列表 `[1,2,3,null,4]` 按层序构造二叉树：`1` 是根节点，`2` 是根的左孩子，`3` 是根的右孩子；接着 `null` 表示节点 `2` 没有左孩子，`4` 表示节点 `2` 的右孩子。用括号表示就是根 `1` 的左子树为 `2()(4)`，右子树为 `3`，即 `1(2()(4))(3)`。B、C、D 都把节点 `3` 或 `4` 放到了与层序数组不一致的位置。

## 第12题

下面有关布尔类型的函数的说法，正确的是（ ）。

A. `bool` 类型函数只能返回 0 或者 1 两种值
B. `bool` 类型函数可以返回任何整数值
C. `bool` 类型函数必须有参数传递
D. `bool` 类型函数没有返回值

答案：A

解析：本题选 A。`bool` 类型函数的返回结果是布尔值，只表示 `false` 或 `true`，在输出或数值转换时通常对应 0 和 1，因此 A 正确。B 容易混淆“可以写 `return 2;` 这样的整数表达式”和“函数最终返回任何整数值”：在 C++ 中整数会被转换成 `bool`，非 0 变为 `true`，函数结果仍不是任意整数。C 错在函数是否有参数与返回类型无关；D 错在 `bool` 函数必须返回布尔结果。

## 第13题

通讯卫星在通信网络系统中主要起到（ ）的作用。

A. 信息过滤
B. 信号中继
C. 避免攻击
D. 数据加密

答案：B

## 第14题

小杨想编写一个判断任意输入的整数 $N$ 是否为素数的程序，下面哪个方法不合适？（ ）

A. 埃氏筛法
B. 线性筛法
C. 二分答案
D. 枚举法

答案：C

解析：本题选 C。判断一个整数 $N$ 是否为素数，可以枚举 $2$ 到 $\sqrt N$ 的因数，也可以在需要处理很多数时用埃氏筛法或线性筛法预处理素数表，所以 A、B、D 都与素数判断有关。二分答案适用于答案具有单调性的判定问题，而“一个数是否为素数”并不随候选答案呈现可二分的单调关系，用二分答案不能直接判断任意整数是否为素数，因此 C 不合适。

## 第15题

内排序有不同的类别，下面哪种排序算法和冒泡排序是同一类？（ ）

A. 希尔排序
B. 快速排序
C. 堆排序
D. 插入排序

答案：B

解析：本题选 B。按常见教材分类，冒泡排序属于交换排序，快速排序也通过不断交换元素并划分区间完成排序，同属于交换类排序。希尔排序和插入排序归入插入类排序，堆排序通常归入选择类排序。因此与冒泡排序同一类的是快速排序。
