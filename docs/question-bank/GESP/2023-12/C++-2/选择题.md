## 第1题

以下不可以作为 C++ 变量的是（ ）。

A. FiveStar
B. fiveStar
C. 5Star
D. Star5

答案：C

解析：C++ 变量名可以由字母、数字、下划线组成，但不能以数字开头。`FiveStar`、`fiveStar` 和 `Star5` 都以字母开头，大小写不同也只是不同变量名；`5Star` 以数字 `5` 开头，不符合变量命名规则，因此不能作为变量名。

## 第2题

在 C++ 中，与 `for(int i = 10; i < 20; i += 2) cout << i;` 输出结果相同的是 ( )。

A. `for(int i = 10; i < 19; i += 2) cout << i;`
B. `for(int i = 11; i < 19; i += 2) cout << i;`
C. `for(int i = 10; i < 21; i += 2) cout << i;`
D. 以上均不对

答案：A

解析：原循环中 `i` 依次为 `10, 12, 14, 16, 18`，输出这些数后再加到 `20`，此时 `20 < 20` 不成立而结束。A 中条件 `i < 19` 仍会输出 `10, 12, 14, 16, 18`，因为下一次已经是 `20`；B 从 `11` 开始，输出序列不同；C 的条件 `i < 21` 会多输出 `20`。

## 第3题

以下 C++ 代码实现从小到大的顺序输出能整除 $N$ 的数（$N$ 的因子），例如 $N=18$ 时输出 1 2 3 6 9 18，横线处应填入 ( )。

```cpp
cin >> N;
for(__________)
    if(N % i == 0)
        cout << i << " ";
```

A. `int i = 0; i < N; i++`
B. `int i = 1; i < N; i++`
C. `int i = 0; i < N+1; i++`
D. `int i = 1; i < N+1; i++`

答案：D

解析：枚举因子时要从 `1` 开始，因为 `1` 是所有正整数的因子；也要枚举到 `N` 本身，因为 `N % N == 0`。A、C 都从 `i = 0` 开始，会出现 `N % 0`，这是非法的除以 0；B 只枚举到 `N - 1`，会漏掉因子 `N`。D 等价于枚举 `1` 到 `N`，能按从小到大的顺序检查所有可能因子。

## 第4题

下面 C++ 代码用于判断输入的整数是否为对称数，如 1221、12321 是对称数，但 123、972 不是对称数。下面对该题对应代码的说法，正确的是 ( )。

```cpp
cin >> N;
newNum = 0;
while(N) {
    newNum = newNum * 10 + N % 10;
    N = N / 10;
}
if(newNum == N)
    cout << N << "为对称数";
```

A. 代码没有语法错误，如果 N 为对称数，第 8 行将能正确输出。
B. 代码没有语法错误，但如果 N 为负数，将导致死循环。
C. 代码存在语法错误，程序不能被执行。
D. 代码没有语法错误，但不能达到预期目标，因为循环结束 N 总为 0。

答案：D

解析：循环中用 `N % 10` 取末位、用 `N = N / 10` 去掉末位，确实能把原数反转到 `newNum` 中；但原来的 `N` 也在这个过程中被不断除以 10，循环结束时一定变成 0。因此最后比较的是 `newNum == 0`，不是“反转后的数是否等于原数”，对 1221、12321 这类正整数对称数也无法正确输出。代码本身没有语法错误，负数不断除以 10 也会逐步趋向 0，不会因此死循环。

## 第5题

下面 C++ 代码用于判断 $N$（大于等于 $2$ 的正整数）是否为质数（素数）。下面对如下代码的说法，正确的是 ( )。

```cpp
cin >> N;
for(i = 2; i < N/2; i++)
    if(N % i == 0) {
        cout << N << " 不是质数";
        break;
    }
if(i >= N/2)
    cout << N << " 是质数";
```

A. 代码能正确判断 N 是否为质数。
B. 代码总是不能判断 N 是否质数。
C. 删除第 5 行 `break`，将能正确判断 N 是否质数。
D. 代码存在漏洞，边界存在问题，应将第 2 行和第 7 行的 `N / 2` 改为 `N / 2 + 1`。

答案：D

解析：判断质数时需要检查是否存在 $2$ 到某个边界内的因子。原代码使用 `i < N/2`，会漏掉因子 `N/2` 本身，例如 $N = 4$ 时循环条件一开始就是 `2 < 2`，循环体不执行，程序会把 4 误判为质数。把边界改成 `N / 2 + 1` 后，条件 `i < N / 2 + 1` 能覆盖到 `i = N/2`。删除 `break` 不能修复边界漏检问题。

## 第6题

下面 C++ 代码执行后的输出是 ( )。

```cpp
N = 4;
for(int i = 0; i < N; i++) {
    for(int j = 1; j < i; j++)
        if (i * j % 2 == 0)
            cout << i << "#";
    continue;
}
cout << "0";
```

A. 2#3#0
B. 1#2#0
C. 1#0#
D. 2#3#

答案：A

解析：外层 `i` 取 `0, 1, 2, 3`。`i = 0` 和 `i = 1` 时内层 `j = 1; j < i` 不成立，不输出；`i = 2` 时 `j = 1`，`2 * 1` 为偶数，输出 `2#`；`i = 3` 时 `j = 1` 不满足偶数条件，`j = 2` 时 `3 * 2` 为偶数，输出 `3#`。循环结束后还会执行 `cout << "0";`，所以总输出是 `2#3#0`。循环体里的 `continue` 只结束当前外层循环的本轮，不会跳过循环后的最终输出。

## 第7题

下面 C++ 代码执行后的输出是 ( )。

```cpp
cnt = 0;
for(i = 1; i < 10; i++)
    for(j = 1; j < i; j += 2)
        if (i * j % 2 == 0) {
            cnt++;
            break;
        }
if(i >= 10) cout << cnt << "#";
cout << cnt;
```

A. 0
B. 8#8
C. 4
D. 4#4

答案：D

解析：内层 `j` 每次取奇数，`i * j` 要成为偶数，只能依靠 `i` 为偶数。`i = 2, 4, 6, 8` 时，第一次 `j = 1` 就满足条件，`cnt` 各增加 1 并 `break` 跳出内层循环；奇数 `i` 不会增加计数。因此循环结束时 `cnt = 4`，外层循环正常结束后 `i` 为 10，先输出 `4#`，再执行最后一行输出 `4`，结果为 `4#4`。

## 第8题

下面 C++ 代码执行后的输出是 ( )。

```cpp
N = 100;
while (N > 0)
    if (N % 2)
        break;
    else if (N % 3 == 0)
        N -= 5;
    else
        N -= 20;
cout << N;
```

A. 100
B. 95
C. 55
D. 0

答案：C

解析：初始 `N = 100`，不是奇数，也不能被 3 整除，所以走最后的 `else`，变成 `80`；`80` 同样走最后的 `else`，变成 `60`；`60` 不是奇数且能被 3 整除，执行 `N -= 5`，变成 `55`。下一轮 `55 % 2` 为真，执行 `break` 跳出循环，最后输出当前的 `N`，即 `55`。

## 第9题

下面 C++ 代码执行后的输出是 ( )。

```cpp
x = 1;
while (x < 100) {
    if (x % 3 != 0)
        cout << x << ",";
    else if (x / 10)
        break;
    else
        x += 5;
    x += 2;
}
cout << x;
```

A. 1
B. 1,3
C. 15,17
D. 1,10,12

答案：D

解析：`x = 1` 时不能被 3 整除，输出 `1,`，再统一执行 `x += 2`，变成 3。`x = 3` 时能被 3 整除，且 `x / 10` 为 0，所以执行 `x += 5` 变成 8，再统一加 2 变成 10。`x = 10` 时输出 `10,` 后变成 12。`x = 12` 能被 3 整除，且 `x / 10` 为 1，执行 `break`，最后输出 `12`，所以结果是 `1,10,12`。

## 第10题

下面 C++ 代码执行后的输出是 ( )。

```cpp
cnt = 0;
for (i = 0; i < 5; i++)
    for (j = 0; j < i; j++)
        cnt += 1;
cout << cnt;
```

A. 5
B. 10
C. 20
D. 30

答案：B

解析：外层 `i` 取 `0, 1, 2, 3, 4`。对应内层循环次数分别为 0 次、1 次、2 次、3 次、4 次，每次都让 `cnt` 加 1。因此总次数是 $0 + 1 + 2 + 3 + 4 = 10$，输出 `10`。

## 第11题

以下 C++ 代码用于输出 $1\sim 100$（含）的整数平方数（完全平方数），如 $16$ 是 $4$ 的平方，横线处应填写（ ）。

```cpp
for (i = 1; i < 100 + 1; i++)
    if (__________________)
        cout << i << " ";
```

A. `int(sqrt(i)) * int(sqrt(i)) = i`
B. `int(sqrt(i)) == sqrt(i)`
C. `int(sqrt(i)) * int(sqrt(i)) == i`
D. `int(sqrt(i)) = sqrt(i)`

答案：C

解析：完全平方数的特征是它的平方根取整后再平方，能回到原数。C 先计算 `int(sqrt(i))`，再判断这个整数平方是否等于 `i`，能准确筛出 $1, 4, 9, \ldots, 100$。A、D 使用的是赋值号 `=`，而且左侧表达式不是可修改变量，不能作为正确判断条件；B 直接比较浮点平方根和整数取整结果，表达不如“平方回原数”严谨，容易受到浮点比较方式影响。

## 第12题

下面的 C++ 代码用于实现如下左图所示的效果，应在以下右图 C++ 代码中填入

![](https://cdn.luogu.com.cn/upload/image_hosting/j15m3ylx.png)

```cpp
cin >> N;
for (i = 0; i < N; i++) {
    nowNum = 0;
    for (j = 0; j < i + 1; j++) {
        cout << nowNum;
        nowNum += 1;
        if (nowNum == 10)
            nowNum = 0;
    }
}
```

A. 于第 8 行下面填入一行：`cout << nowNum;`
B. 于第 2 行下面填入一行：`cout << endl;`
C. 于第 7 行下面填入一行：`cout << nowNum;`
D. 于第 9 行下面填入一行：`cout << endl;`

答案：D

解析：图中每一行都从 `0` 开始输出，行内数字个数依次增加，输入 `12` 时最后一行是 `012345678901`。已有内层循环负责输出一行中的数字，并在 `nowNum == 10` 时回到 0；缺少的是每一行结束后的换行。因此应在外层循环的一轮结束处，也就是第 9 行下面加入 `cout << endl;`。A、C 会额外输出数字，B 会在每行开始前换行，都不能得到图中的行结构。

## 第13题

某公司新出了一款无人驾驶的小汽车，通过声控智能驾驶系统，乘客只要告诉汽车目的地，车子就能自动选择一条优化路线，告诉乘客后驶达那里。请问下面哪项不是驾驶系统完成选路所必须的。（  ）

A. 麦克风
B. 扬声器
C. 油量表
D. 传感器

答案：C

## 第14题

现代计算机是指电子计算机，它所基于的是（ ）体系结构。

A. 艾伦·图灵
B. 冯·诺依曼
C. 阿塔纳索夫
D. 埃克特-莫克利

答案：B

## 第15题

输入一个正整数 $N$，想找出它所有相邻的因数对，比如，输入 $12$，因数对有 $(1,2)$、$(2,3)$、$(3,4)$。下面哪段代码找不到所有的因数对？( )

A. `for(i=1;i<N;i++) if(!(N%i) && !(N%(i+1))) printf("(%d,%d)\n", i, i+1);`
B. `for(i=2;i<N;i++) if(!(N%i) && !(N%(i+1))) printf("(%d,%d)\n", i, i+1);`
C. `for(i=2;i<N/2;i++) if(!(N%(i-1)) && !(N%i)) printf("(%d,%d)\n", i-1, i);`
D. `for(i=1;i<N/2;i++) if(!(N%i) && !(N%(i+1))) printf("(%d,%d)\n", i, i+1);`

答案：B

解析：官网答案表给出的答案是 B。按常见单选考查口径，本题主要考查枚举起点是否会漏掉相邻因数对 `(1, 2)`：B 的循环从 `i = 2` 开始，只检查 `(2, 3)`、`(3, 4)` 等形式，必然漏掉可能存在的 `(1, 2)`，例如输入 `12` 时 `(1,2)` 是合法因数对，但 B 不会检查，所以选 B。不过这里存在边界争议：若严格按题干“输入一个正整数 N”理解，不能把 A、C、D 都写成无条件正确。例如 C 的条件 `i < N/2` 在 `N = 6` 时不会检查到 `(2, 3)`，D 的条件 `i < N/2` 在 `N = 2` 时不会检查到 `(1, 2)`。因此本题保留官方单选答案 B，但解析应理解为依赖官网答案表和常见考查口径，而不是证明其他选项对所有正整数 `N` 都一定找全。
