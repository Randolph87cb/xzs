## 第1题

C++ 表达式 `-7/2` 的值为整数 $-3$。( )

答案：正确

解析：`-7` 和 `2` 都是整数，`/` 执行整数除法。C++ 中整数除法会舍去小数部分，也就是向 0 截断；`-7 / 2` 的数学结果是 $-3.5$，向 0 截断后得到 $-3$，所以题干说法正确。

## 第2题

C++ 表达式 `2*int('9')*2` 的值为 $36$。( )

答案：错误

解析：`'9'` 是字符常量，`int('9')` 得到的是字符 `'9'` 的 ASCII 码值 $57$，不是数字 $9$。因此表达式的值为 $2 \times 57 \times 2 = 228$，不是 $36$，所以题干说法错误。

## 第3题

C++ 表达式 `3+2 && 5-5` 的值为 `false`。( )

答案：正确

解析：先算加减法，`3 + 2` 得到 $5$，`5 - 5` 得到 $0$。在逻辑运算中，非 0 值当作 `true`，0 当作 `false`，所以整个表达式等价于 `true && false`，结果为 `false`，题干说法正确。

## 第4题

在 C++ 代码中，执行 `srand(0)` 后连续两次执行 `rand()` 的结果相等。( )

【知识点：函数】

答案：错误

解析：`srand(0)` 和 `rand()` 都是与随机数有关的库函数。`srand(0)` 的作用是设置随机数种子，使之后调用 `rand()` 产生的伪随机序列在同一实现中可重复；它并不表示每一次调用 `rand()` 都返回同一个数。连续两次调用 `rand()` 得到的是同一伪随机序列中的前两个值，这两个值不要求相等，所以题干说法错误。

## 第5题

C++ 代码中 `while(1){...}` 的判断条件不是逻辑值，将导致语法错误。( )

答案：错误

解析：`while` 的条件表达式不必直接写成 `bool` 类型，C++ 会把条件表达式转换为布尔意义判断。整数 `1` 会被视为 `true`，因此 `while(1){...}` 语法正确，并表示条件恒真、会不断循环，除非循环体内用 `break`、`return` 等方式退出。

## 第6题

执行以下 C++ 代码后将输出 $0$。( )

```cpp
Sum = 0;
for (i = -500; i < 500; i++)
    Sum += i;
cout << Sum;
```

答案：错误

解析：循环变量 `i` 依次取 `-500, -499, ..., 499`，不包含 `500`。这串数中 `-499` 到 `499` 可以两两抵消，剩下一个 `-500`，所以 `Sum` 最终为 `-500`。程序不会输出 $0$，题干说法错误。

## 第7题

在 C++ 代码中，运算符只能处理相同的数据类型，不同类型之间必须转换为相同的数据类型。( )

答案：错误

解析：C++ 的许多运算允许不同类型参与，并会按规则进行隐式类型转换。例如 `int` 和 `double` 相加时，`int` 通常会先转换为 `double` 再计算；字符也可以在算术表达式中转换成对应编码值。并不是所有不同类型都必须由程序员手动转成同一类型，所以题干说法错误。

## 第8题

在 C++ 代码中，虽然变量都有数据类型，但同一个变量也可以先后用不同类型的值赋值。( )

答案：正确

解析：变量声明后类型固定，但赋值表达式右侧的值可以是其他可转换类型，赋值时会转换成变量本身的类型。例如 `double x; x = 1; x = 2.5;` 可以先后赋整数和小数；`int y; y = 3.8;` 也能赋值，只是小数部分会被截断。因此“可以先后用不同类型的值赋值”这个说法成立，但变量本身的类型不会随赋值改变。

## 第9题

小杨最近在准备考 GESP，他用的 Dev C++ 来练习和运行程序，所以 Dev C++ 也是一个小型操作系统。( )

答案：错误

解析：Dev C++ 是用于编写、编译和运行 C++ 程序的集成开发环境，也就是 IDE。操作系统负责管理硬件、进程、文件、内存等基础资源，例如 Windows、Linux；Dev C++ 运行在操作系统之上，不是操作系统。

## 第10题

任何一个 while 循环都可以转化为等价的 for 循环。( )

答案：正确

解析：`while(条件){循环体}` 可以写成 `for(; 条件; ){循环体}`，两者都是先判断条件，再决定是否执行循环体。若原来的 `while` 循环体中有初始化或更新语句，也可以放在 `for` 的三个表达式位置或保留在循环体中，因此任意 `while` 循环都能改写成等价的 `for` 循环。
