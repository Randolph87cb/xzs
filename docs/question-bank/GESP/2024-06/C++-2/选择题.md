## 第1题

小杨父母带他到某培训机构给他报名参加 CCF 组织的 GESP 认证考试的第 1 级，那他可以选择的认证语言有几种？（ ）

A. 1
B. 2
C. 3
D. 4

答案：C

解析：GESP 第 1 级面向入门认证，支持图形化编程、C++、Python 三种认证语言，因此可选择的语言数量是 3 种。A、B 少算了可选语言，D 多算了不存在的第 4 种，本题选 C。

## 第2题

下面流程图在 yr 输入 2024 时，可以判定 yr 代表闰年，并输出 2 月是 29 天，则图中菱形框中应该填入（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/shumi9qb.png)

A. `(yr%400==0) || (yr%4==0)`
B. `(yr%400==0) || (yr%4==0 && yr%100!=0)`
C. `(yr%400==0) && (yr%4==0)`
D. `(yr%400==0) && (yr%4==0 && yr%100!=0)`

答案：B

解析：闰年的完整规则是“能被 400 整除，或者能被 4 整除但不能被 100 整除”。写成表达式就是 `(yr%400==0) || (yr%4==0 && yr%100!=0)`，2024 满足右半部分条件。A 会把 1900 这类能被 100 整除但不能被 400 整除的年份误判为闰年；C、D 都要求满足左半部分，2024 不能通过，本题选 B。

## 第3题

在 C++ 中，下列不可做变量的是（ ）。

A. `five-Star`
B. `five_star`
C. `fiveStar`
D. `_fiveStar`

答案：A

解析：C++ 标识符只能由字母、数字和下划线组成，并且不能以数字开头。`five-Star` 中的 `-` 不是标识符字符，在表达式里会被理解成减号；而 `five_star`、`fiveStar`、`_fiveStar` 都只含合法字符，所以不可做变量名的是 A。

## 第4题

在 C++ 中，与 `for(int i=0; i<10; i++)` 效果相同的是（ ）。

A. `for(int i=0; i<10; i+=1)`
B. `for(int i=1; i<=10; i++)`
C. `for(int i=10; i>0; i--)`
D. `for(int i=10; i<1; i++)`

答案：A

解析：原循环中 `i` 的取值依次为 0、1、2、...、9。A 把更新语句从 `i++` 写成 `i+=1`，含义仍是每轮加 1，初值、循环条件和每次循环时的 `i` 值都相同。B、C 虽然也可能执行 10 轮，但 `i` 的取值不同；D 初始 `i=10` 时就不满足 `i<1`，不会进入循环，本题选 A。

## 第5题

在 C++ 中，`cout << (5 % 2 && 5 % 3)` 的输出是（ ）。

A. `1`
B. `2`
C. `true`
D. `false`

答案：A

解析：先计算取余：`5 % 2` 得 1，`5 % 3` 得 2。逻辑运算中，非 0 值都当作真，因此表达式等价于 `true && true`，结果为 `true`。`cout` 在没有使用 `boolalpha` 时会把布尔值 `true` 输出为 `1`，不是字符串 `true`，本题选 A。

## 第6题

执行下面的 C++ 代码时输入 1，则输出是（ ）。

```cpp
int month;

cin >> month;

switch(month){
    case 1:
        cout << "Jan ";
    case 3:
        cout << "Mar ";
        break;
    default:
        ;
}
```

A. Jan
B. Mar
C. Jan Mar
D. 以上均不对

答案：C

解析：`switch` 根据 `month` 的值跳到对应分支。输入 1 时先进入 `case 1`，输出 `Jan `；但这一分支后面没有 `break`，会继续向下执行 `case 3`，再输出 `Mar `，然后遇到 `break` 才跳出 `switch`。因此实际输出包含两个单词，即 `Jan Mar`，本题选 C。

## 第7题

执行下面 C++ 代码后，有关说法错误的是（ ）。

```cpp
int a, b;
cin >> a >> b;
if (a && b)
    cout << "1";
else if (!(a || b))
    cout << "2";
else if (a || b)
    cout << "3";
else
    cout << "4";
```

A. 如果先后输入 1 和 1，则将输出 1
B. 如果先后输入 0 和 1 或者 1 和 0，则将输出 3
C. 如果先后输入 0 和 0，则将输出 2
D. 如果先后输入 0 和 0，则将输出 4

答案：D

解析：逐项判断：输入 `1 1` 时 `a && b` 为真，输出 `1`；输入 `0 1` 或 `1 0` 时第一个条件为假，`!(a || b)` 也为假，接着 `a || b` 为真，输出 `3`；输入 `0 0` 时 `a || b` 为假，所以 `!(a || b)` 为真，输出 `2`。因此 A、B、C 都正确，D 说 `0 0` 会输出 `4` 是错误说法，本题选 D。

## 第8题

某货币由 5 元、2 元和 1 元组成。输入金额（假设为正整数），计算出最少数量。为实现其功能，横线处应填入代码是（ ）。

```cpp
int N;
cin >> N;

int M5, M2, M1;
M5 = N / 5;
M2 = _______________;
M1 = _______________;
printf("5*%d+2*%d+1*%d", M5, M2, M1);
```

A. 第 1 横线处应填入：`N / 2`  第 2 横线处应填入：`N - M5 - M2`
B. 第 1 横线处应填入：`(N - M5 * 5) / 2`  第 2 横线处应填入：`N - M5 * 5 - M2 * 2`
C. 第 1 横线处应填入：`N - M5 * 5 / 2`  第 2 横线处应填入：`N - M5 * 5 - M2 * 2`
D. 第 1 横线处应填入：`(N - M5 * 5) / 2`  第 2 横线处应填入：`N - M5 - M2`

答案：B

解析：面值为 5、2、1 时，要让数量最少，应先尽量使用大面额。`M5 = N / 5` 后，剩余金额为 `N - M5 * 5`，因此 2 元数量应为 `(N - M5 * 5) / 2`。再扣掉已经使用的 5 元和 2 元金额，1 元数量是 `N - M5 * 5 - M2 * 2`。A 没扣除 5 元金额，C 受运算优先级影响不是“剩余金额除以 2”，D 最后一项减的是张数不是金额，本题选 B。

## 第9题

下面 C++ 代码执行后的输出是（ ）。

```cpp
int loopCount = 0;
for (int i=0; i < 10; i++)
    for (int j=1; j < i; j++)
        loopCount += 1;
cout << loopCount;
```

A. 55
B. 45
C. 36
D. 28

答案：C

解析：外层 `i` 的取值为 0 到 9。固定某个 `i` 时，内层 `j` 从 1 开始，满足 `j < i`，所以执行次数是 `i - 1` 次；当 `i=0,1` 时执行 0 次，当 `i=2` 到 9 时分别执行 1、2、...、8 次。总次数为 $1+2+\cdots+8=36$，因此 `loopCount` 最终为 36，本题选 C。

## 第10题

下面 C++ 代码执行后的输出是（ ）。

```cpp
int loopCount = 0;
for (int i=0; i < 10; i++){
    for (int j=0; j < i; j++)
        if (i * j % 2)
            break;
    loopCount += 1;
}
cout << loopCount;
```

A. 25
B. 16
C. 10
D. 9

答案：C

解析：注意 `loopCount += 1` 在内层 `for` 结束之后，但仍然属于外层循环体。无论内层循环执行多少次、是否因为 `break` 提前结束，外层每一轮最后都会执行一次 `loopCount += 1`。外层 `i` 从 0 到 9 共 10 轮，所以最终输出 10。A、B 把内层循环次数也算进了 `loopCount`，D 少算了一轮，本题选 C。

## 第11题

假设下面 C++ 代码执行过程中仅输入正负整数或 0，有关说法错误的是（ ）。

```cpp
int N, Sum = 0;
cin >> N;
while (N){
    Sum += N;
    cin >> N;
}
cout << Sum;
```

A. 执行上面代码如果输入 0，将终止循环
B. 执行上面代码能实现所有非 0 整数的求和
C. 执行上面代码第一次输入 0，最后将输出 0
D. 执行上面代码将陷入死循环，可将 `while (N)` 改为 `while (N==0)`

答案：D

解析：`while (N)` 的含义是“当前 `N` 非 0 时继续循环”。每轮先把当前非 0 整数累加到 `Sum`，再读入下一个 `N`，因此可以实现以 0 作为结束标记的求和；第一次输入 0 时循环体一次也不执行，输出仍为 0。D 说程序会陷入死循环不对，而且改成 `while (N==0)` 会在输入 0 时进入循环，正好违背结束标记的含义，本题选 D。

## 第12题

执行下面的 C++ 代码，有关说法正确的是（ ）【质数是指仅能被 1 和它本身整除的正整数】。

```cpp
int N;
cin >> N;
bool Flag = true;
for (int i = 2; i < N; i++){
    if (i * i > N)
        break;
    if (N % i == 0){
        Flag = false;
        break;
    }
}
if (Flag)
    cout << N << "是质数" << endl;
else
    cout << N << "不是质数" << endl;
```

A. 如果输入正整数，上面代码能正确判断 $N$ 是否为质数
B. 如果输入整数，上面代码能正确判断 $N$ 是否为质数
C. 如果输入大于等于 0 的整数，上面代码能正确判断 $N$ 是否为质数
D. 如将 `Flag = true` 修改为 `Flag = N>=2? true:false` 则能判断所有整数包括负整数、0、正整数是否为质数

答案：D

解析：质数必须是大于等于 2 的整数。原程序先把 `Flag` 设为 `true`，对于 1、0 或负数，循环无法找到因子，最后会错误输出“是质数”，所以 A、B、C 都不能成立。若改为 `Flag = N>=2? true:false`，小于 2 的数一开始就被标记为非质数；大于等于 2 的数再通过试除检查是否存在因子，因此可以覆盖负数、0 和正整数，本题选 D。

## 第13题

下面 C++ 代码用于实现如下图所示的效果，其有关说法正确的是（ ）。

```
1
2 4
3 6 9
4 8 12 16
5 10 15 20 25
```

```cpp
for (int i = 1; i < 6; i++) { // L1
    for (int j = 1; j < i+1; j++) // L2
        cout << i*j << " ";
    cout << endl;
}
```

A. 当前代码能实现预期效果，无需调整代码
B. 如果 `cout << endl;` 移到循环 L2 内部，则可实现预期效果
C. 如果 `cout << endl;` 移到循环 L1 外部，则可实现预期效果
D. 删除 `cout << endl;` 行，则可实现预期效果

答案：A

解析：外层 `i` 控制第几行，取值 1 到 5；内层 `j` 从 1 到 `i`，所以第 `i` 行会输出 `i*1, i*2, ..., i*i`，正好对应样例图形。`cout << endl;` 放在内层循环之后，表示一整行输出完再换行。若移到 L2 内部，会每输出一个数就换行；若移到 L1 外部或删除，则多行会连在一起，本题选 A。

## 第14题

下面 C++ 代码执行后，输出是（ ）。

```cpp
int cnt1 = 0, cnt2 = 0;
for (int i = 0; i < 10; i++){
    if (i % 2 == 0)
        continue;
    if (i % 2)
        cnt1 += 1;
    else if (i % 3 == 0)
        cnt2 += 1;
}
cout << cnt1 << " " << cnt2;
```

A. 5 2
B. 5 0
C. 0 2
D. 0 0

答案：B

解析：当 `i` 为偶数时，第一条 `if (i % 2 == 0)` 成立，执行 `continue` 后直接进入下一轮，后面的 `cnt1`、`cnt2` 判断都不会执行。能继续往下走的只有奇数 1、3、5、7、9，它们都满足 `i % 2`，所以 `cnt1` 加 5 次。由于这些奇数已经进入 `if (i % 2)` 分支，后面的 `else if` 没有机会执行，`cnt2` 保持 0，输出 `5 0`，本题选 B。

## 第15题

在下面的 C++ 代码中，$N$ 必须是小于 $10$ 大于 $1$ 的整数，$M$ 为正整数（大于 $0$）。如果 $M$ 被 $N$ 整除则 $M$ 为幸运数，如果 $M$ 中含有 $N$ 且能被 $N$ 整除，则为超级幸运数，否则不是幸运数。程序用于判断 $M$ 是否为幸运数或超级幸运数或非幸运数。阅读下面代码，有关说法正确的是（ ）。

```cpp
int N, M;
cout << "请输入幸运数字：";
cin >> N;
cout << "请输入正整数：";
cin >> M;

bool Lucky;
if (M % N == 0)
    Lucky = true;
else
    Lucky = false;
while (M){
    if (M % 10 == N && Lucky){
        printf("%d是%d的超级幸运数!", M, N);
        break;
    }
    M /= 10;
}
if (M == 0)
    if (Lucky)
        printf("%d是%d的幸运数!", M, N);
    else
        printf("%d非%d的幸运数!", M, N);
```

A. 如果 $N$ 输入 $3$，$M$ 输入 $36$ 则将输出：`36是3的超级幸运数!`
B. 如果 $N$ 输入 $7$，$M$ 输入 $21$ 则将输出：`21是7的幸运数!`
C. 如果 $N$ 输入 $8$，$M$ 输入 $36$ 则将输出：`36非8的超级幸运数!`
D. 如果 $N$ 输入 $3$，$M$ 输入 $63$ 则将输出：`63是3的超级幸运数!`

答案：D

解析：这段程序有一个关键细节：它直接用 `M /= 10` 修改原变量 `M`，没有保存最初输入的数。因此只要不是第一次循环就输出，后面 `printf` 中的 `M` 往往已经变小，甚至变成 0。A 中 `36` 先变成 `3` 后才发现数字 3，会输出 `3是3的超级幸运数!`；B 中 `21` 被除到 0 后输出 `0是7的幸运数!`；C 中 `36` 不是 8 的倍数，最后会输出 `0非8的幸运数!`，且不是“超级幸运数”。D 中 `N=3, M=63`，`63%3==0` 且个位就是 3，第一次循环立即输出 `63是3的超级幸运数!`，之后 `M` 仍非 0，不会再进入最后的 `if (M == 0)`，本题选 D。
