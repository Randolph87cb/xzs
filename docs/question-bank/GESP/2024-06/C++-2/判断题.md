## 第1题

执行 C++ 代码 `cout << '9' + '1';` 的输出为 $10$。（ ）

答案：错误

解析：`'9'` 和 `'1'` 是字符常量，参与 `+` 运算时会按字符编码对应的整数相加，不是把两个数字字符当作 9 和 1 直接相加。ASCII 中 `'9'` 为 57，`'1'` 为 49，相加输出 106，因此题干说输出 10 是错误的。

## 第2题

C++ 表达式 `-12 % 10` 的值为 $2$。（ ）

答案：错误

解析：C++ 中整数取余结果的符号与被除数一致。`-12 / 10` 的整数商向 0 截断为 -1，因此 `-12 % 10 = -12 - (-1)*10 = -2`，不是 2，题干错误。

## 第3题

C++ 表达式 `int(12.56)` 的值为 $13$。（ ）

答案：错误

解析：`int(12.56)` 是把浮点数转换为整数，转换时会直接舍去小数部分，而不是四舍五入。结果为 12，不是 13，题干错误。

## 第4题

C++ 的整型变量 $N$ 被赋值为 $10$，则语句 `cout << N / 3 << "-" << N % 3` 执行后输出是 $3-1$。（ ）

答案：正确

解析：`N` 为整型且值为 10，`N / 3` 是整数除法，结果为 3；`N % 3` 是余数，结果为 1。`cout` 中间输出字符串 `"-"`，所以整体输出正是 `3-1`，题干正确。

## 第5题

在 C++ 代码中，不可以将变量命名为 `scanf`，因为 `scanf` 是 C++ 语言的关键字。（ ）

答案：错误

解析：`scanf` 是 C 标准库函数名，不是 C++ 关键字。虽然把变量命名为 `scanf` 会造成可读性差、也可能遮蔽库函数名，但从关键字规则看它不是保留关键字，题干理由错误。

## 第6题

下面 C++ 代码执行后将导致死循环。（ ）

```cpp
for (int i = 0; i < 10; i++)
    continue;
```

答案：错误

解析：`continue` 会结束本轮循环体，随后执行 `i++` 并重新判断 `i < 10`。`i` 从 0 增加到 10 后条件为假，循环正常结束，因此不会死循环，题干错误。

## 第7题

下面 C++ 代码执行后将输出 $10$。（ ）

```cpp
int cnt = 0;
for (int i = 0; i < 10; i++)
    for (int j = 0; j < i; j++){
        cnt += 1;
        break;
    }
cout << cnt;
```

答案：错误

解析：外层 `i=0` 时内层 `j < i` 不成立，不加计数；`i=1` 到 9 时，内层第一次执行就 `cnt += 1`，随后 `break` 跳出内层，所以每个这样的 `i` 只加 1 次。总计加 9 次，输出 9，不是 10，题干错误。

## 第8题

下面 C++ 代码执行后，将输出 $5$。（ ）

```cpp
int cnt = 0;
for (int i = 1; i < 5; i++)
    for (int j = i; j < 5; j += i)
        if (i * j % 2 == 0)
            cnt += 1;
cout << cnt;
```

答案：正确

解析：逐个统计：`i=1` 时 `j=1,2,3,4` 中有 2 和 4 使乘积为偶数，计 2 次；`i=2` 时 `j=2,4` 都满足，计 2 次；`i=3` 时只有 `j=3`，乘积为奇数，计 0 次；`i=4` 时 `j=4` 满足，计 1 次。总数 `2+2+0+1=5`，题干正确。

## 第9题

下面 C++ 代码能实现正整数各位数字之和。（ ）

```cpp
int N, Sum = 0;
cin >> N;
while (N){
    Sum += N % 10;
    N /= 10;
}
cout << Sum;
```

答案：正确

解析：对正整数 `N`，`N % 10` 取出当前个位，累加到 `Sum`；`N /= 10` 去掉个位。循环直到所有数字都被处理完，因此能求出正整数各位数字之和，题干正确。

## 第10题

GESP 测试是对认证者的编程能力进行等级认证，同一级别的能力基本上与编程语言无关。（ ）

答案：正确

解析：GESP 是面向编程能力的等级认证，同一级别考查的算法思想、程序结构和问题解决能力基本一致，语言只是实现工具；不同语言会有语法差异，但能力等级目标基本相同，题干正确。
