## 第1题

下列代码中，输出结果是（ ）
```cpp
#include<iostream>
using namespace std;
int func(int x,int y)
{
    int a=x,b=y;
    int t;
    t=a;
    a=b;
    b=t;
    cout<<a<<" "<<b<<" ";
}
int main()
{
    int c,d;
    c=12;
    d=24;
    func(12,24);
    cout<<c<<" "<<d<<endl;
}
```

A. 12 24 24 12
B. 24 12 12 24
C. 12 12 24 24
D. 24 24 12 12

答案：B

解析：`func(12,24)` 中形参 `x=12,y=24`，局部变量 `a=12,b=24` 交换后先输出 `24 12`；`main` 中的 `c=12,d=24` 没有传入函数，也没有被修改，所以随后输出 `12 24`，合起来是 `24 12 12 24`。

## 第3题

下面程序输出的是（ ）

```cpp
#include<iostream>
using namespace std;
int func();
int main()
{
    int i=2;
    cout<<i<<endl;
    for(int x=0;x<1;x++)
    {
        int i=10;
        cout<<i<<endl;
    }
    i=i+1;
    cout<<i<<endl;
    {
        i=i*i;
        cout<<i<<endl;
    }
}
```

A. 2 2 3 9
B. 2 10 3 9
C. 2 10 11 121
D. 2 10 3 100

答案：B

解析：`main` 开始的 `i=2` 先输出 `2`；`for` 循环体内重新定义了一个局部 `int i=10`，它只在循环体内有效，所以输出 `10` 后外层 `i` 仍为 `2`；执行 `i=i+1` 后输出 `3`；最后一个代码块没有重新定义 `i`，执行 `i=i*i` 得到 `9`。

## 第4题

假设变量 `a` 的地址是 0x6ffe14，下面程序的输出是（ ）。
```cpp
#include<iostream>
using namespace std;
int main()
{
    int *p;
    int a=10;
    p=&a;
    p++;
    cout<<p<<endl;
}
```

A. 10
B. 0x6ffe14
C. 0x6ffe15
D. 0x6ffe18

答案：D

解析：`p=&a` 后 `p` 保存的是 `a` 的地址 `0x6ffe14`。`p` 的类型是 `int*`，执行 `p++` 不是地址加 `1`，而是前进一个 `int` 的大小；本题按 `int` 占 4 字节计算，所以地址变为 `0x6ffe18`。

## 第5题

如果下列程序输出的地址是 0x6ffe00，则 `cout<<a+1<<endl;` 输出的是（ ）

```cpp
#include<iostream>
using namespace std;
int main()
{
    int a[2][3]={0};
    cout<<a<<endl;
}
```

A. 0x6ffe04
B. 0x6ffe0C
C. 0x6ffe08
D. 0x6ffe00

答案：B

解析：`a` 是 `int a[2][3]`，在表达式中表示指向第 0 行的指针；`a+1` 会跳过一整行，也就是 `3` 个 `int`。每个 `int` 按 4 字节计算，共前进 `12` 字节，`0x6ffe00 + 0x0C = 0x6ffe0C`。

## 第6题

C++ 中，关于文件路径说法错误的是（ ）

A. "GESP.txt"：指定与当前工作目录中的程序文件相同目录中的 GESP.txt 文件
B. "../data/GESP.txt"：指定与当前工作目录中的程序文件上一级目录下的 data 目录中的 GESP.txt 文件
C. "./data/GESP.txt"：指定与当前工作目录中的程序文件同级目录下的 data 目录中的 GESP.txt 文件
D. "GESP.txt"是绝对路径

答案：D

解析：`"GESP.txt"` 不含盘符、根目录或以 `/`、`\` 开头的根路径信息，是相对路径，会相对当前工作目录查找文件，不是绝对路径。`../data/GESP.txt` 表示先到上一级目录再进入 `data`，`./data/GESP.txt` 表示当前目录下的 `data`。

## 第7题

关于直接插入排序，下列说法错误的是（ ）

A. 插入排序的最好情况是数组已经有序，此时只需要进行 $n - 1$ 次比较，时间复杂度为 $O(n)$
B. 最坏情况是数组逆序排序，此时需要进行 $n(n - 1)/2$ 次比较以及 $n - 1$ 次赋值操作（插入）
C. 平均来说插入排序算法的复杂度为 $O(n^2)$
D. 空间复杂度上，直接插入法是就地排序，空间复杂度为 $O(n)$

答案：D

解析：直接插入排序只需要一个临时变量保存待插入元素，排序过程在原数组中移动元素，所以是就地排序，额外空间复杂度应为 $O(1)$，不是 $O(n)$。最好情况数组已有序，只做相邻比较，时间为 $O(n)$；平均和逆序最坏情况都需要大量移动，时间为 $O(n^2)$。

## 第8题

下列程序横线处，应该输入的是（ ）。

```cpp
#include<iostream>
using namespace std;
int n,a[10001];
void swap(int &a,int &b)
{
    int t=a;
    a=b;
    b=t;
}
int main()
{
    cin>>n;
    for(int i=1;i<=n;i++)
        cin>>a[i];
    for(int i=n;i>1;i--)
        for(int j=1;j<i;j++)
            if(a[j]>a[j+1])
                ;
    for(int i=1;i<=n;i++)
        cout<<a[i]<<" ";
    cout<<endl;
    return 0;
}
```

A. swap(a[j],a[j+1]);
B. swap(a[j-1],a[j]);
C. swap(a[j-1],a[j+1]);
D. swap(&a[j-1],&a[j+1]);

答案：A

解析：这段代码是从前向后比较相邻元素的冒泡排序，条件 `a[j]>a[j+1]` 成立时应交换这两个相邻位置，所以填 `swap(a[j],a[j+1]);`。B、C 交换的下标与当前比较对象不一致，D 传入的是地址，和函数形参 `int&` 的类型不匹配。

## 第9题

下面关于递推的说法不正确的是（ ）。

A. 递推表现为自己调用自己
B. 递推是从简单问题出发，一步步的向前发展，最终求得问题。是正向的
C. 递推中，问题的 $n$ 要求是在计算中确定，不要求计算前就知道 $n$
D. 斐波那契数列可以用递推实现求解

答案：A

解析：自己调用自己描述的是递归，不是递推。递推通常从已知的初始值出发，按递推关系一步步求后面的值，例如斐波那契数列可以由前两项推出后一项，因此 B、D 符合递推思想；C 表示可以在计算过程中逐步确定需要的项数，不影响递推定义。

## 第10题

关于几种排序算法的说法，下面说法错误的是（ ）。

A. 选择排序不是一个稳定的排序算法
B. 冒泡排序算法不是一种稳定的排序算法
C. 插入排序是一种稳定的排序算法
D. 如果排序前2个相等的数在序列中的前后位置顺序和排序后它们2个的前后位置顺序相同，则称为一种稳定的排序算法

答案：B

解析：标准冒泡排序只在前一个元素大于后一个元素时交换，相等元素不会互相越过，所以冒泡排序是稳定排序，B 说“不稳定”是错误的。直接插入排序同样可以保持相等元素相对顺序，选择排序则可能把后面的较小元素换到前面，破坏相等元素顺序。

## 第11题

数组 $\{45,66,23,1,10,97,52,88,5,33\}$ 进行从小到大冒泡排序过程中，第一遍冒泡过后的序列是（ ）。

A. $\{45,23,1,10,66,52,88,5,33,97\}$
B. $\{45,66,1,23,10,97,52,88,5,33\}$
C. $\{45,66,23,1,10,52,88,5,33,97\}$
D. $\{45,66,23,1,10,97,52,88,33,5\}$

答案：A

解析：第一遍从左到右比较相邻元素，把当前较大的数不断向右交换。`66` 依次越过 `23,1,10` 后停在 `97` 前，随后 `97` 又依次越过 `52,88,5,33` 到达末尾，所以第一遍结束序列为 $\{45,23,1,10,66,52,88,5,33,97\}$。

## 第12题

下面的排序算法程序中，横线处应该填入的是（ ）。
```cpp
int a[8]={ 2,3, 4, 5, 6,2,3,1};
for (int i=1;i<8;i++)
{
    int key = a[i];
    int j=i-1;
    while(a[j]>key && j>=0)
    {
        ______;
        j -= 1;
    }
    a[j + 1]= key;
}
```

A. a[j]=a[j-1];
B. a[j]=a[j+1];
C. a[j+1]=a[j-1];
D. a[j+1]=a[j];

答案：D

解析：这是直接插入排序。`key` 保存待插入元素，`j` 从已排序部分末尾向前找位置；当 `a[j]>key` 时，需要把较大的 `a[j]` 向右移动一格，所以本题横线处仍应填 `D. a[j+1]=a[j];`，循环结束后再把 `key` 放到 `a[j+1]`。但题面条件 `while(a[j]>key && j>=0)` 按 C++ 中 `&&` 从左到右的短路求值规则，会先计算 `a[j]>key`；如果 `j` 已经减到 `-1`，就可能先访问 `a[-1]`，存在越界和未定义行为风险。严谨写法应改为 `while(j>=0 && a[j]>key)`，先判断下标合法，再访问 `a[j]`。

## 第13题

下面的程序中，如果输入 10 0 ，会输出（ ）。

```cpp
#include<iostream>
using namespace std;

double Division(int a, int b)
{
    if (b == 0)
        throw "Division by zero condition!";
    else
        return ((double)a / (double)b);
}

void func()
{
    int len, time;
    cin >> len >> time;
    cout << Division(len, time) << endl;
}

int main()
{
    try {
        func();
    }
    catch (const char* errmsg)
    {
        cout << errmsg << endl;
    }
    catch (const int errmsg)
    {
        cout << errmsg << endl;
    }
    return 0;
}
```

A. Division by zero condition!
B. 0
C. 10
D. 100

答案：A

解析：输入 `10 0` 后，`func` 调用 `Division(10,0)`，因为 `b==0`，函数执行 `throw "Division by zero condition!"`，不会返回除法结果。抛出的字符串字面量可由 `catch (const char* errmsg)` 捕获，于是输出这段错误信息。

## 第14题

10 条直线，最多可以把平面分为多少个区域（ ）。

A. 55
B. 56
C. 54
D. 58

答案：B

解析：要让 10 条直线分出的区域最多，应保证任意两条直线相交且不存在三线共点。第 $k$ 条直线会被前面 $k-1$ 条直线切成 $k$ 段，从而新增 $k$ 个区域，所以总数为 $1+1+2+\cdots+10=56$。

## 第15题

下面程序中，如果语句 `cout<<p<<endl;` 输出的是 `0x6ffe00`，则 `cout<<++p<<endl;` 输出的是（ ）

```cpp
int x[10][10][10]={{0}};
int *p;
p=&x[0][0][0];
cout<<p<<endl;
cout<<++p<<endl;
```

A. 0x6ffe0c
B. 0x6ffe09
C. 0x6ffe06
D. 0x6ffe04

答案：D

解析：`p` 的类型是 `int*`，初始指向 `x[0][0][0]`，输出为 `0x6ffe00`。`++p` 先把指针前进一个 `int` 再输出；多维数组在内存中连续存放，但这里指针步长仍按一个 `int` 计算，即前进 4 字节，得到 `0x6ffe04`。
