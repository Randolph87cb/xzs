# 判断题

来源：

说明：本文件优先采用洛谷结构化数据整理，并以官网 PDF 做交叉校对。

## 第1题

ABCDE 五个小朋友，排成一队跑步，其中 AB 两人必须排在一起，一共有 48 种排法。

答案：正确

解析：把 AB 两人看成一个整体，再与 C、D、E 共 $4$ 个单位排列，有 $4! = 24$ 种。AB 内部还可以是 AB 或 BA 两种顺序，所以共有 $24 \times 2 = 48$ 种。

## 第2题

已知 `double` 类型的变量 `a` 和 `b`，则执行语句 `a = a + b; b = a - b; a = a - b;` 后，变量 `a` 和 `b` 的值会互换。

答案：错误

解析：若按精确实数运算，这三句是常见的加减交换写法；但 `double` 是有限精度浮点数，`a + b` 可能发生舍入、溢出，甚至在数量级差距很大时丢失较小数的信息。因此题干说“会互换”过于绝对，不能保证总是成立。

## 第3题

一个袋子中有 3 个完全相同的红色小球、2 个完全相同的蓝色小球。每次从中取出 1 个，再放回袋子，这样进行 3 次后，可能的颜色顺序有 8 种。

答案：正确

解析：每次取出后放回，下一次仍然只能出现红或蓝两种颜色，且三次之间在颜色选择上都可独立考虑。长度为 $3$ 的红蓝序列共有 $2^3 = 8$ 种，例如红红蓝、蓝红蓝等都可能出现。

## 第4题

已知 `int` 类型的变量 `a` 和 `b` 中分别存储着一个直角三角形的两条直角边的长度，则斜边的长度可以通过表达式 `sqrt(a * a + b * b)` 求得。

答案：正确

解析：直角三角形斜边满足勾股定理 $c=\sqrt{a^2+b^2}$，表达式 `sqrt(a * a + b * b)` 正好对应这个公式。由于结果可能不是整数，用 `sqrt` 得到浮点结果也是合理的。

## 第5题

在一个包含 $v$ 个顶点、$e$ 条边的带权连通简单有向图上使用 Dijkstra 算法求最短路径，时间复杂度为 $O(v^2)$，可进一步优化至 $O(e + v \log(v))$。

答案：正确

解析：在 Dijkstra 适用的非负边权前提下，朴素实现每轮从未确定顶点中找最小距离，做 $v$ 轮，时间复杂度为 $O(v^2)$。若使用能支持高效 decrease-key 的堆结构（如斐波那契堆），可把复杂度优化到 $O(e + v\log v)$；普通二叉堆实现通常不能直接达到这一界。

## 第6题

在 $N$ 个元素的二叉排序树中查找一个元素，最差情况的时间复杂度是 $O(\log N)$。

答案：错误

解析：二叉排序树只有在比较平衡时查找路径长度才接近 $O(\log N)$。最差情况下，插入顺序可能让它退化成一条链，此时查找要沿着最多 $N$ 个节点走下去，时间复杂度为 $O(N)$。

## 第7题

C++ 语言中，可以为同一个类定义多个析构函数。

答案：错误

解析：C++ 中一个类只能有一个析构函数，形式固定为 `~类名()`，没有返回值，也不能带参数。因为不能通过参数列表区分多个析构函数，所以析构函数不能重载。

## 第8题

使用单链表和使用双向链表，查找元素的时间复杂度相同。

答案：正确

解析：若只知道要查找某个元素而没有额外索引，单链表和双向链表都必须从某个端点开始逐个比较节点。最坏情况下都要检查 $N$ 个节点，所以查找时间复杂度同为 $O(N)$。

## 第9题

为解决哈希函数冲突，可以使用不同的哈希函数为每个表项各建立一个子哈希表，用来管理该表项的所有冲突元素。这些子哈希表一定不会发生冲突。

答案：错误

解析：为冲突元素再建立子哈希表是一种再哈希思路，但新的哈希函数仍然可能把不同元素映射到同一位置。除非额外构造完美哈希并证明无冲突，否则不能说这些子哈希表一定不会发生冲突。

## 第10题

要判断无向图的连通性，在深度优先搜索和广度优先搜索中选择，深度优先的平均时间复杂度更低。

答案：错误

解析：用邻接表判断无向图连通性时，DFS 和 BFS 都会访问每个顶点，并检查每条边，时间复杂度都是 $O(V+E)$。两者遍历顺序不同，但不存在 DFS 平均时间复杂度更低这一结论。
