## 第1题

GESP 测试是对认证者的编程能力进行等级认证，同一级别的能力基本上与编程语言无关。

【知识点：计算机基础常识】

答案：正确

解析：GESP 是按等级考察编程能力，同一级别对应的知识范围和能力要求基本一致；虽然图形化、Python、C++ 的语法形式不同，但认证等级本身不是按某一种语言单独定义能力高低，所以题干说法正确。

## 第2题

整数 $-6$ 的 $16$ 位补码可用十六进制表示为 `FFFA`。

【知识点：原码反码补码与整数表示】

答案：正确

解析：$+6$ 的 $16$ 位二进制是 `0000 0000 0000 0110`，按位取反得 `1111 1111 1111 1001`，再加 $1$ 得 `1111 1111 1111 1010`。每 $4$ 位转一位十六进制，正好是 `FFFA`，所以题干正确。

## 第3题

补码的优点是可以将减法运算转化为加法运算，从而简化计算机的硬件设计。

【知识点：原码反码补码与整数表示】

答案：正确

解析：补码把负数设计成可以直接参与加法的形式，例如计算 $a-b$ 可以转化为 $a+(-b)$。这样硬件主要使用加法器即可同时处理加法和减法，确实能简化计算机的运算电路设计。

## 第4题

字符常量 `'\0'` 常用来表示字符串结束，和字符常量 `'0'` 相同。

【知识点：ASCII/字符编码】

答案：错误

解析：`'\0'` 是空字符，编码值为 $0$，常用作 C 风格字符串的结束标记；`'0'` 是字符数字 $0$，ASCII 编码是 $48$。两者写法相近但含义和编码都不同，所以题干错误。

## 第5题

数组的所有元素在内存中可以不连续存放。

【知识点：数组与字符数组】

答案：错误

解析：C++ 数组的元素在内存中按下标顺序连续存放，例如 `a[0]` 后面紧接着是 `a[1]`。随机访问数组下标正是依赖这种连续存储规则，因此“可以不连续存放”的说法错误。

## 第6题

C++ 中可以对数组和数组的每个基础类型的元素赋值。

【知识点：数组与字符数组】

答案：错误

解析：数组的单个基础类型元素可以赋值，例如 `a[0] = 3;`；但数组名不能作为整体被普通赋值，例如定义后写 `a = b;` 或 `a = {1,2,3};` 都不合法。题干把“数组整体”和“数组元素”都说成可以赋值，因此错误。

## 第7题

如果 $a$ 为 int 类型的变量，且表达式 `((a | 3) == 3)` 的值为 true，则说明 $a$ 在从 $0$ 到 $3$ 之间（可能为 $0$、可能为 $3$）。

【知识点：位运算】

答案：正确

解析：`3` 的二进制低两位是 `11`。如果 `(a | 3)` 仍等于 `3`，说明 `a` 不能在低两位以外有任何为 $1$ 的二进制位，低两位则可以是 `00`、`01`、`10`、`11`，对应 $0$、$1$、$2$、$3$，所以题干正确。

## 第8题

执行下面 C++ 代码后，输出的结果是 $8$。

```cpp
int a = 0b1010;
int b = 01100;
int c = a & b;
cout << c << endl;
```

【知识点：位运算】

答案：错误

解析：`0b1010` 是二进制整数，值为 $10$；`01100` 以 $0$ 开头，在 C++ 中是八进制整数，值为 $1\times 8^3+1\times 8^2=576$。$10$ 与 $576$ 做按位与没有共同的 $1$ 位，结果是 $0$，不是 $8$，所以题干错误。

## 第9题

执行下面 C++ 代码后，输出的结果不可能是 $89781$。

```cpp
#include <iostream>
#include <cstdlib>  // 为了使用 rand() 和 srand()
#include <ctime>    // 为了使用 time()

using namespace std;

int main() {
    // 设置随机种子
    srand(time(NULL));

    int i = 1;
    int s[5];
    while(i <= 5)
    {
        int a = rand() % 10;

        if(a % 3 == (i + 1) % 3)
            s[i++] = a;
    }
    for(int i = 1; i <= 5; i++)
        cout << s[i];
    cout << endl;
    return 0;
}
```

【知识点：控制结构与程序阅读】

答案：正确

解析：严格 C++ 中，`int s[5]` 的合法下标是 $0$ 到 $4$，代码却会访问 `s[5]`，存在数组越界；以下按考试意图，把数组视作可存放第 $1$ 到第 $5$ 个数字来分析。按题目考察的筛选条件看，第 $i$ 个输出数字必须满足 `a % 3 == (i + 1) % 3`。若输出 `89781`，前四位分别满足 $8\bmod3=2$、$9\bmod3=0$、$7\bmod3=1$、$8\bmod3=2$；但第 $5$ 位要求余数为 $0$，而 $1\bmod3=1$，不满足条件，所以该输出不可能出现，题干说法正确。

## 第10题

把整数 $3025$ 从中剪开分为 $30$ 和 $25$ 两个数，此时再将这两数之和平方，计算结果又等于原数。$(30 + 25) \times (30 + 25) = 55 \times 55 = 3025$，这样的数叫“雷劈数”。可以使用枚举的方法求出所有符合这样条件的四位数。

【知识点：枚举与基础算法】

答案：正确

解析：四位数只有 $1000$ 到 $9999$，数量有限。枚举每个四位数 $n$，用 `n / 100` 得到前两位、用 `n % 100` 得到后两位，再判断两部分之和的平方是否等于 $n$，就能找出所有“雷劈数”，所以题干正确。
