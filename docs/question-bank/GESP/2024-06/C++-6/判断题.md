# 判断题

来源：
- 洛谷有题页：https://ti.luogu.com.cn/problemset/1154
- 官网原卷 PDF：https://gesp.ccf.org.cn/101/attach/1621071589539872.pdf

说明：本文件优先采用洛谷结构化数据整理，并以官网 PDF 做交叉校对。

## 第1题

哈夫曼编码本质上是一种贪心策略。

答案：正确

解析：哈夫曼编码每一步都选择当前权值最小的两棵树合并，使低频字符处在更深位置、高频字符处在更浅位置。这种“每步做局部最优选择并能得到全局最优前缀编码”的过程正是贪心策略。

## 第2题

创建一个对象时，会自动调用该对象所属类的构造函数。如果没有定义构造函数，编译器会自动生成一个默认的构造函数。

答案：正确

解析：创建对象时，构造函数负责完成对象初始化，会被自动调用。若类中没有显式声明任何构造函数，编译器会隐式生成一个默认构造函数，使对象仍然可以按默认方式创建。

## 第3题

定义一个类时，必须手动定义一个析构函数，用于释放对象所占用的资源。

答案：错误

解析：类并不一定要手动定义析构函数；没有显式定义时，编译器会生成默认析构函数。只有类中管理了动态内存、文件句柄等需要自定义释放逻辑的资源时，才通常需要手写析构函数，因此“必须”过于绝对。

## 第4题

C++ 中类内部可以嵌套定义类。

答案：正确

解析：C++ 允许在一个类的内部再定义另一个类，这称为嵌套类。嵌套类的名字位于外层类的作用域中，例如可用 `Outer::Inner` 这样的形式访问类型名。

## 第5题

$000, 001, 011, 010, 110, 111, 101, 100$ 是一组格雷码。

答案：正确

解析：格雷码要求相邻两个编码只相差一位。题中序列从 `000` 到 `001`、`011`、`010`、`110`、`111`、`101`、`100`，每一步都只改变一个二进制位，所以是一组 3 位格雷码。

## 第6题

$n$ 个节点的双向循环链表，在其中查找某个节点的平均时间复杂度是 $O(\log n)$。

答案：错误

解析：双向循环链表只能通过指针沿前后方向逐个结点查找，不能像有序数组那样直接跳到中间位置二分查找。若没有额外索引，查找某个指定结点平均仍需检查线性数量的结点，时间复杂度是 $O(n)$，不是 $O(\log n)$。

## 第7题

完全二叉树可以用数组存储数据。

答案：正确

解析：完全二叉树除最后一层外都填满，最后一层从左到右连续排列，因此用数组按层序存储时不会产生大量空洞。若下标从 1 开始，结点 `i` 的左右孩子可对应 `2i` 和 `2i+1`，适合数组表示。

## 第8题

在 C++ 中，静态成员函数只能访问静态成员变量。

答案：正确

解析：静态成员函数不依赖某个具体对象调用，函数内部没有可用的 `this` 指针，因此不能直接访问普通成员变量。它只能直接访问同属于类本身的静态成员；若要操作非静态成员，必须通过传入或已知的对象来访问。

## 第9题

在深度优先搜索中，通常使用队列来辅助实现。

答案：错误

解析：深度优先搜索的特点是沿一条路径尽量深入，再回退到其他分支，通常用递归调用栈或显式栈辅助实现。队列按先进先出顺序处理结点，更符合广度优先搜索逐层扩展的过程。

## 第10题

对 0-1 背包问题，贪心算法一定能获得最优解。

答案：错误

解析：0-1 背包中每个物品只能选或不选，局部看起来最优的选择不一定组成全局最优。例如按性价比优先可能先选一个性价比最高的小物品，反而占用容量，错过价值更高的组合；该问题通常用动态规划求最优解，不能说贪心一定正确。
