## 第1题

下面 C++ 代码用于求斐波那契数列，该数列第 $1$、$2$ 项为 $1$，以后各项均是前两项之和。函数 `fibo()` 属于（ ）。
```cpp
int fibo(int n) {
    if (n <= 0)
        return 0;
    if (n == 1 || n == 2)
        return 1;

    int a = 1, b = 1, next;
    for (int i = 3; i <= n; i++) {
        next = a + b;
        a = b;
        b = next;
    }
    return next;
}
```

A. 枚举算法
B. 贪心算法
C. 迭代算法
D. 递归算法

答案：C

解析：函数先处理 $n \le 0$、$n=1$、$n=2$ 的边界，然后用 `for` 循环不断把 `a`、`b` 更新为相邻两项，直到算出第 $n$ 项。整个过程是反复执行同一组更新语句，没有调用 `fibo()` 自身，所以属于迭代算法，不是递归算法。

## 第2题

下面 C++ 代码用于将输入金额换成最少币种组合方案，其实现算法是（ ）。
```cpp
#include <iostream>
using namespace std;

#define N_COINS 7
int coins[N_COINS] = {100, 50, 20, 10, 5, 2, 1}; //货币面值，单位相同
int coins_used[N_COINS];

void find_coins(int money) {
    for (int i = 0; i < N_COINS; i++) {
        coins_used[i] = money / coins[i];
        money = money % coins[i];
    }
    return;
}

int main() {
    int money;
    cin >> money;  //输入要换算的金额

    find_coins(money);
    for (int i = 0; i < N_COINS; i++)
        cout << coins_used[i] << endl;

    return 0;
}
```

A. 枚举算法
B. 贪心算法
C. 迭代算法
D. 递归算法

答案：B

解析：`coins` 按面值从大到小排列，每一步都先尽量使用当前最大面值：`coins_used[i] = money / coins[i]`，再把剩余金额交给后面的更小面值。这种“当前一步选局部最优面值数量”的思想是贪心算法。

## 第3题

小杨采用如下双链表结构保存他喜欢的歌曲列表：
```cpp
struct dl_node {
    string song;
    dl_node* next;
    dl_node* prev;
};
```
小杨想在头指针为 `head` 的双链表中查找他喜欢的某首歌曲，采用如下查询函数，该操作的时间复杂度为（ ）。
```cpp
dl_node* search(dl_node* head, string my_song) {
    dl_node* temp = head;
    while (temp != nullptr) {
        if (temp->song == my_song)
            return temp;
        temp = temp->next;
    }
    return nullptr;
}
```

A. $O(1)$
B. $O(n)$
C. $O(\log n)$
D. $O(n^2)$

答案：B

解析：函数从 `head` 开始，沿着 `next` 指针一个结点一个结点比较 `song`。最坏情况下目标歌曲在最后一个结点，或根本不存在，需要检查全部 $n$ 个结点，因此时间复杂度是 $O(n)$。

## 第5题

下面是根据欧几里得算法编写的函数，它计算的是 $a$ 与 $b$ 的（ ）。
```cpp
int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}
```

A. 最小公倍数
B. 最大公共质因子
C. 最大公约数
D. 最小公共质因子

答案：C

解析：循环中把 `(a, b)` 更新为 `(b, a % b)`，这正是欧几里得算法的辗转相除过程，并且每次更新都保持 `gcd(a,b)` 不变。当 `b` 变为 $0$ 时，当前的 `a` 就是原来两数的最大公约数。

## 第6题

欧几里得算法还可以写成如下形式：
```cpp
int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
}
```
下面有关说法，错误的是（ ）。

A. 本题的 `gcd()` 实现为递归方式。
B. 本题的 `gcd()` 代码量少，更容易理解其辗转相除的思想。
C. 当 $a$ 较大时，本题的 `gcd()` 实现会多次调用自身，需要较多额外的辅助空间。
D. 当 $a$ 较大时，相比上题中的 `gcd()` 的实现，本题的 `gcd()` 执行效率更高。

答案：D

解析：这段代码用 `gcd(b, a % b)` 调用自身，是递归写法，含义和上题的循环版辗转相除相同。递归版通常代码更短，但每次递归调用都要占用调用栈空间；它并不会因为 `a` 较大就比循环版执行效率更高，甚至可能有函数调用开销，所以错误说法是 D。

## 第8题

A. $O(n^2)$

## 第9题

为了正确实现快速排序，下面横线上的代码应为（ ）。
```cpp
void qsort(vector<int>& arr, int left, int right) {
    int i, j, mid;
    int pivot;

    i = left;
    j = right;
    mid = (left + right) / 2; // 计算中间元素的索引
    pivot = arr[mid]; // 选择中间元素作为基准值

    do {
        while (arr[i] < pivot) i++;
        while (arr[j] > pivot) j--;
        if (i <= j) {
            swap(arr[i], arr[j]); // 交换两个元素
            i++; j--;
        }
    } ________________________; // 在此处填入代码
    if (left < j) qsort(arr, left, j); // 对左子数组进行快速排序
    if (i < right) qsort(arr, i, right); // 对右子数组进行快速排序
}
```

A. while (i <= mid)
B. while (i < mid)
C. while (i < j)
D. while (i <= j)

答案：D

解析：快速排序这段划分过程用 `i` 从左找不小于 `pivot` 的元素，用 `j` 从右找不大于 `pivot` 的元素，只要 `i <= j` 就交换并继续向中间推进。循环应在两个指针交错后结束，因此 `do ... while` 的条件应为 `while (i <= j)`。

## 第10题

关于分治算法，以下哪个说法正确？

A. 分治算法将问题分成子问题，然后分别解决子问题，最后合并结果。
B. 归并排序不是分治算法的应用。
C. 分治算法通常用于解决小规模问题。
D. 分治算法的时间复杂度总是优于 $O(n \log(n))$。

答案：A

解析：分治的基本步骤是把原问题拆成若干规模更小的子问题，分别求解，再把子问题结果合并成原问题答案。归并排序正是典型分治应用；分治不只用于小规模问题，时间复杂度也不一定总优于 $O(n \log n)$，所以只有 A 正确。

## 第11题

根据下述二分查找法，在排好序的数组 1, 3, 6, 9, 17, 31, 39, 52, 61, 79, 81, 90, 96 中查找数值 82，和82比较的数组元素分别是（ ）。
```cpp
int binary_search(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1;
    while (left <= right) {
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1; // 如果找不到目标元素，返回-1
}
```

A. 52, 61, 81, 90
B. 52, 79, 90, 81
C. 39, 79, 90, 81
D. 39, 79, 90

答案：C

解析：数组下标从 $0$ 到 $12$。第一次 `mid=(0+12)/2=6`，比较 `39`，因 $39<82$，转到右半边；第二次在下标 $7..12$ 中比较 `79`；第三次在下标 $10..12$ 中比较 `90`；最后在下标 $10..10$ 比较 `81` 后查找失败，所以比较序列为 `39, 79, 90, 81`。

## 第13题

设 $A$ 和 $B$ 是两个长度为 $n$ 的有序数组，现将 $A$ 和 $B$ 合并成一个有序数组，归并排序算法在最坏情况下至少要做（ ）次比较。

B. $n \log n$
C. $2n - 1$

答案：C

## 第14题

给定如下函数：
```cpp
int fun(int n) {
    if (n == 1) return 1;
    if (n == 2) return 2;
    return fun(n - 2) - fun(n - 1);
}
```
则当 $n = 7$ 时，函数返回值为（ ）。

A. 0
B. 1
C. 21
D. -11

答案：D

解析：由递推式 `fun(n)=fun(n-2)-fun(n-1)`，有 `fun(1)=1`、`fun(2)=2`，所以 `fun(3)=1-2=-1`，`fun(4)=2-(-1)=3`，`fun(5)=-1-3=-4`，`fun(6)=3-(-4)=7`，`fun(7)=-4-7=-11`。

## 第15题

给定如下函数（函数功能同上题，增加输出打印）：
```cpp
int fun(int n) {
    cout << n << " ";
    if (n == 1) return 1;
    if (n == 2) return 2;
    return fun(n - 2) - fun(n - 1);
}
```

则当 $n = 4$ 时，屏幕上输出序列为（ ）。

A. 4 3 2 1
B. 1 2 3 4
C. 4 2 3 1 2
D. 4 2 3 2 1

答案：C

解析：函数一进入就先输出当前 `n`。调用 `fun(4)` 先输出 `4`，再计算 `fun(2) - fun(3)`；`fun(2)` 输出 `2` 后返回；接着 `fun(3)` 输出 `3`，再依次调用并输出 `fun(1)` 的 `1`、`fun(2)` 的 `2`。因此输出顺序是 `4 2 3 1 2`。
