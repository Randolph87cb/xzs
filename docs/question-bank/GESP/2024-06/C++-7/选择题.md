## 第1题

下列 C++ 代码的输出结果是（ ）。
```cpp
#include <iostream>
#include <cmath>
using namespace std;
int main() {
    cout << sin(3.1415926 / 2);
    return 0;
}
```

A. $0$
B. $1$
C. $0.5$
D. $0.7071$

答案：B

解析：`3.1415926 / 2` 近似为圆周率的一半，即弧度制下的 $\frac{\pi}{2}$。`sin` 使用弧度计算，$\sin(\frac{\pi}{2}) = 1$，`cout` 输出的结果为接近 1 的值，本题按选项应选 B。

## 第2题

对于如下图的二叉树，说法正确的是（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/i65ct90r.png)

A. 先序遍历是 $132$。
B. 中序遍历是 $123$。
C. 后序遍历是 $312$。
D. 先序遍历和后序遍历正好是相反的。

答案：D

解析：图中根节点为 1，1 的右孩子是 2，2 的左孩子是 3。先序遍历按“根、左、右”为 `123`，中序遍历为 `132`，后序遍历为 `321`，所以 A、B、C 都不对，而先序 `123` 与后序 `321` 正好互为反序，选 D。

## 第3题

已知两个序列 $s1 = \{1,3,4,5,6,7,7,8,1\}$、$s2 = \{3,5,7,4,8,2,9,5,1\}$，则它们的最长公共子序列是（ ）。

A. $\{3,5,7,8,1\}$
B. $\{3,4,5,7,8\}$
C. $\{5,7,8\}$
D. $\{3,5,7,9,1\}$

答案：A

解析：$\{3,5,7,8,1\}$ 在 $s1$ 中可取第 2、4、6、8、9 项，在 $s2$ 中可取第 1、2、3、5、9 项，顺序都保持不变，长度为 5。B 中 `3,4,5` 在 $s2$ 中顺序不成立，D 含有 $s1$ 中不存在的 9，C 虽是公共子序列但长度只有 3，因此选 A。

## 第4题

关于序列 $\{2,7,1,5,6,4,3,8,9\}$，以下说法错误的是（ ）。

A. $\{2,5,6,8,9\}$ 是它的最长上升子序列
B. $\{1,5,6,8,9\}$ 是它的最长上升子序列
C. $\{7,5,4,3\}$ 是它的最长下降子序列
D. $\{1,5,6,8,9\}$ 是它的唯一最长上升子序列

答案：D

解析：该序列存在长度为 5 的上升子序列，例如 A 的 $\{2,5,6,8,9\}$ 和 B 的 $\{1,5,6,8,9\}$，所以 $\{1,5,6,8,9\}$ 不是唯一的最长上升子序列。C 中 $\{7,5,4,3\}$ 按原序列位置递减，长度为 4，是可达到的最长下降子序列；错误说法是 D。

## 第5题

关于图的深度优先搜索和广度优先搜索，下列说法错误的是（ ）。

A. 二叉树是也是一种图。
B. 二叉树的前序遍历和后序遍历都是深度优先搜索的一种。
C. 深度优先搜索可以从任意根节点开始。
D. 二叉树的后序遍历也是广度优先搜索的一种。

答案：D

解析：二叉树可以看作一种特殊的图，前序、后序遍历都会沿一条分支深入后再回溯，属于深度优先思想。广度优先搜索按层访问节点，典型结果是层序遍历，不会得到“左子树、右子树、根”的后序顺序，因此 D 是错误说法。

## 第6题

对于如下二叉树，下面访问顺序说法错误的是（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/5dofuogk.png)

A. $HDEBFIGCA$ 不是它的后序遍历序列
B. $ABCDEFGHI$ 是它的广度优先遍历序列
C. $ABDHECFGI$ 是它的深度优先遍历序列
D. $ABDHECFGI$ 是它的先序遍历序列

答案：A

解析：后序遍历按“左子树、右子树、根”访问：B 子树为 `HDEB`，C 子树为 `FIGC`，最后访问根 A，合起来正是 `HDEBFIGCA`。所以 A 说“不是它的后序遍历序列”是错误的；`ABCDEFGHI` 是按层的广度优先序列，`ABDHECFGI` 也正是先序 DFS 序列。

## 第7题

以下哪个方案不能合理解决或缓解哈希表冲突（ ）。

A. 丢弃发生冲突的新元素。
B. 在每个哈希表项处，使用不同的哈希函数再建立一个哈希表，管理该表项的冲突元素。
C. 在每个哈希表项处，建立二叉排序树，管理该表项的冲突元素。
D. 使用不同的哈希函数建立额外的哈希表，用来管理所有发生冲突的元素。

答案：A

解析：哈希冲突的处理目标是在不同关键字映射到同一位置时仍能保存并查找到这些元素。直接丢弃新元素会造成数据丢失，后续查找也无法保证正确，因此不能算合理解决冲突；B、C、D 分别类似二级哈希、桶内树结构和额外冲突区，都是可行的缓解思路。

## 第8题

在 C++ 中，关于运算符 `&`，下面说法正确的是（ ）。

A. $2$ & $3$ 的结果是 `true`
B. $011$ & $111$ 的结果是 $3$
C. $3$ & $6$ 的结果是 $2$
D. $110$ & $101$ 的结果是 $4$

答案：C

解析：`&` 是按位与，两个二进制位都为 1 时结果才为 1。$3$ 的二进制是 `011`，$6$ 的二进制是 `110`，按位与得到 `010`，即 $2$。A 的结果值是整数 2 不是 `true`；B 中 C++ 的 `011` 是八进制整数；D 中 `110`、`101` 不是二进制写法，结果也不是 4。

## 第9题

下面关于图的说法正确的是（ ）。

A. 在无向图中，环是指至少包含三个不同顶点，并且第一个顶点和最后一个顶点是相同的路径。
B. 在有向图中，环是指一个顶点经过至少另一个顶点到自身的路径。
C. 在有向图中，如果任意两个顶点之间都存在一条边，则这个图一定是强连通图。
D. 在有向图中，所有顶点的入度和出度的总和就是图的边数的两倍。

答案：D

解析：有向图中每条边都会给一个顶点贡献 1 个出度、给另一个顶点贡献 1 个入度，因此所有入度之和为边数，所有出度之和也为边数，二者相加是边数的两倍。A、B 对“环”的表述不够严谨，C 中任意两点之间有一条有向边也可能只是单向连通，不能保证强连通。

## 第10题

图的存储和遍历算法，下面说法错误的是（ ）。

A. 图的深度优先搜索和广度优先搜索对有向图和无向图都适用。
B. 图的深度优先搜索和二叉树的先序遍历道理是不一样的。
C. 图的深度优先搜索需要借助栈来完成。
D. 邻接表中，顶点 $v_i$ 对应链表中的边结点数目正好是顶点 $v_i$ 的度。

答案：B

解析：二叉树先序遍历是先访问当前节点，再递归深入子树，本质上就是深度优先搜索在二叉树上的一种固定访问顺序。图的 DFS 也可以用递归调用栈或显式栈实现，并且适用于有向图和无向图；邻接表中一个顶点链表里的边结点数反映该顶点相邻边的数量，所以错误说法是 B。

## 第13题

下面程序的输出为（ ）。
```cpp
#include <iostream>
using namespace std;
int fib(int n) {
    if (n <= 1)
        return n;
    return fib(n - 1) + fib(n - 2);
}
int main() {
    cout << fib(6) << endl;
    return 0;
}
```

A. 5
B. 8
C. 13
D. 无法正常结束。

答案：B

解析：函数定义的是斐波那契数列：`fib(0)=0`，`fib(1)=1`，之后 `fib(n)=fib(n-1)+fib(n-2)`。逐项计算得 `fib(2)=1`、`fib(3)=2`、`fib(4)=3`、`fib(5)=5`、`fib(6)=8`，递归参数不断减小能到达边界，所以输出为 8。

## 第14题

下面 `count_triple` 函数的时间复杂度为（ ）。
```cpp
int count_triple(int n) {
    int cnt = 0;
    for (int a = 1; a <= n; a++)
        for (int b = a; a + b <= n; b++) {
            int c = sqrt(a * a + b * b);
            if (a + b + c > n)
                break;
            if (a * a + b * b == c * c)
                cnt++;
        }
    return cnt;
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(n^3)$
D. $O(n^4)$

答案：B

解析：外层变量 `a` 最多循环 $n$ 次；对固定的 `a`，内层 `b` 从 `a` 开始递增，受 `a + b <= n` 和 `break` 限制，次数最多仍是 $O(n)$。循环体中的 `sqrt`、比较和加法都是常数时间，因此总时间为两层循环的 $O(n^2)$，`break` 只能减少常数或部分次数，不会把数量级降为 $O(n)$。

## 第15题

下列选项中，哪个可能是下图的深度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/31z0l3ll.png)

A. 1, 3, 7, 5, 4, 2, 6, 8, 9
B. 9, 4, 2, 1, 3, 5, 7, 6, 8
C. 1, 3, 4, 2, 7, 6, 8, 9, 5
D. 9, 7, 6, 8, 4, 2, 1, 5, 3

答案：C

解析：从 1 开始做 DFS，可以先沿有向边访问 `1->3`，再从 3 选择 `3->4`、从 4 到 `2`；回溯到 3 后再走 `3->7`，接着 `7->6->8->9`，最后回到 7 访问 `5`，得到 `1, 3, 4, 2, 7, 6, 8, 9, 5`。其他选项中存在从当前递归分支无法直接到达下一个点，或在回溯前跳过仍未访问邻点的情况。
