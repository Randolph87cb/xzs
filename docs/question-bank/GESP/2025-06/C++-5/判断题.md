## 第1题

下面 C++ 代码是用欧几里得算法（辗转相除法）求两个正整数的最大公约数，$a$ 大于 $b$ 还是小于 $b$ 都适用。

```cpp
int gcd(int a, int b) {
    while (b) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}
```

答案：正确

解析：欧几里得算法利用 $\gcd(a,b)=\gcd(b,a\bmod b)$。如果一开始 `a > b`，循环按常规辗转相除；如果一开始 `a < b`，第一次计算 `a % b` 得到 `a`，随后变量变成原来的 `(b, a)`，等价于自动交换了顺序。只要输入是正整数，循环最终会在 `b` 变为 $0$ 时返回最大公约数，因此判断为正确。



## 第2题

假设函数 `gcd()` 函数能正确求两个正整数的最大公约数，则下面的 `lcm()` 函数能求相应两数的最小公倍数。

```cpp
int lcm(int a, int b) {
    return a * b / gcd(a, b);
}
```

答案：正确

解析：对正整数 $a,b$，有 $\gcd(a,b)\times\operatorname{lcm}(a,b)=a\times b$，因此 `a * b / gcd(a, b)` 可以求出最小公倍数。题目已经假设 `gcd()` 正确，且没有把整型溢出作为考点，所以按数学公式判断该函数能得到相应结果，判断为正确。



## 第3题

下面的 C++ 代码用于输出每个数对应的质因数列表，输出形如：`{5: [5], 6: [2, 3], 7: [7], 8: [2, 2, 2]}`。

```cpp
int main() {
    int n, m;
    cin >> n >> m;
    if (n > m) swap(n, m);
    map<int, vector<int>> prime_factor;

    for (int i = n; i <= m; ++i) {
        int j = 2, k = i;
        while (k != 1) {
            if (k % j == 0) {
                prime_factor[i] = prime_factor[i] + j;
                k /= j;
            } else {
                ++j;
            }
        }
    }

    for (auto& p : prime_factor) {
        cout << p.first << ": ";
        for (int v : p.second)
            cout << v << " ";
        cout << endl;
    }

    return 0;
}
```

答案：错误

解析：`prime_factor` 的值类型是 `vector<int>`，而语句 `prime_factor[i] = prime_factor[i] + j;` 试图把一个 `vector<int>` 和一个 `int` 相加，C++ 标准库没有这种运算，代码无法通过编译。正确记录质因数应类似写成 `prime_factor[i].push_back(j);`。另外后面的输出格式也只是逐个打印数字和空格，并不会直接输出题干示例中的 `{5: [5], ...}` 形式，因此判断为错误。



## 第4题

下面的 C++ 代码实现归并排序。代码在执行时，将输出一次 HERE 字符串，因为 merge() 函数仅被调用一次。

```cpp
void merge(std::vector<int>& arr, int left, int mid, int right) {
    std::vector<int> temp(right - left + 1);
    int i = left;
    int j = mid + 1;
    int k = 0;

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }

    while (i <= mid) {
        temp[k++] = arr[i++];
    }

    while (j <= right) {
        temp[k++] = arr[j++];
    }

    for (int p = 0; p < k; ++p) {
        arr[left + p] = temp[p];
    }
}

void mergeSort(std::vector<int> arr, int left, int right) {
    if (left >= right) {
        return;
    }

    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    std::cout << "HERE";
    merge(arr, left, mid, right);
}
```

答案：错误

解析：`merge()` 并不是只调用一次。`mergeSort` 每把区间分成左右两半并递归排好后，都会执行一次 `std::cout << "HERE";` 并调用 `merge(arr, left, mid, right)` 合并当前区间。只要待排序区间包含多个元素，递归树中的多个内部结点都会触发这段输出，因此 `HERE` 会输出多次，判断为错误。



## 第5题

归并排序的最好、最坏和平均时间复杂度均为 $O(n \log n)$。

答案：正确

解析：归并排序每一层递归都会把所有元素参与一次合并，总合并代价为 $O(n)$；递归把规模不断二分，层数为 $O(\log n)$。无论原数组是否已经有序，标准归并排序通常仍会完成分割和合并过程，所以最好、最坏和平均时间复杂度都是 $O(n\log n)$，判断为正确。



## 第6题

查字典这个小学生必备技能，可以把字典视为一个已排序的数组。假设小杨要查找一个音首字母为 $g$ 的单词，他首先翻到字典约一半的页数，发现该页的首字母是 $m$，由于字母表中 $g$ 位于 $m$ 之前，所以排除字典后半部分，查找范围缩小到前半部分；不断重复上述步骤，直到找到首字母为 $g$ 的页码。这种查字典的一系列操作可看作二分查找。

答案：正确

解析：二分查找的核心是在有序范围中取中间位置比较，并根据比较结果排除一半候选范围。字典页码按字母顺序排列，可以视为有序数组；看到中间页首字母 `m`，目标首字母 `g` 在它之前，于是排除后半部分，之后继续用同样方法缩小范围。这正对应二分查找思想，判断为正确。



## 第7题

求解下图中 A 点到 D 点最短路径，其中 A 到 B 之间的 12 可以理解为距离。求解这样的问题常用 Dijkstra 算法，其思路是通过逐步选择当前距离起点最近的节点来求解非负权重图（如距离不能为负值）单源最短路径的算法。从该算法的描述可以看出，Dijkstra 算法是贪心算法。

![](https://cdn.luogu.com.cn/upload/image_hosting/q6lizie9.png)

答案：正确

解析：在边权非负的图中，Dijkstra 算法每一步从尚未确定最短距离的点里选出当前距离起点最近的点，并把它的距离确定下来，再用它更新相邻点。这个选择是局部最优选择，而且在非负权条件下不会被后续路径推翻，因此体现了贪心算法思想，判断为正确。



## 第8题

分治算法将原问题可以分解成规模更小的子问题，使得求解问题的难度降低。但由于分治算法需要将问题进行分解，并且需要将多个子问题的解合并为原问题的解，所以分治算法的效率通常比直接求解原问题的效率低。

答案：错误

解析：分治确实会带来拆分和合并的开销，但它的目的通常是把大问题化成结构相同、规模更小的子问题，从而降低总体复杂度或让问题更容易求解。例如归并排序通过分治达到 $O(n\log n)$，比直接反复选择最小值的某些简单排序思路更高效。不能笼统说分治通常比直接求解效率低，因此判断为错误。



## 第9题

函数 `puzzle` 定义如下，则调用 `puzzle(7)` 程序会无限递归。

```cpp
int puzzle(int n) {
    if (n == 1) return 1;
    if (n % 2 == 0) return puzzle(n / 2);
    return puzzle(3 * n + 1);
}
```

答案：错误

解析：`puzzle` 的终止条件是 `n == 1`。从 $7$ 开始按代码计算：$7\to22\to11\to34\to17\to52\to26\to13\to40\to20\to10\to5\to16\to8\to4\to2\to1$，最终会到达终止条件并返回，不会无限递归。易错点是只看到奇数分支 `3 * n + 1` 会变大，就误以为一定无法停止；对 $7$ 这一路径实际会回落到 $1$，因此判断为错误。



## 第10题

如下为线性筛法，用于高效生成素数表，其核心思想是每个合数只被它的最小质因数筛掉一次，时间复杂度为 $O(n)$。

```cpp
vector<int> linearSieve(int n) {
    vector<bool> is_prime(n + 1, true);
    vector<int> primes;

    for (int i = 2; i <= n; ++i) {
        if (is_prime[i]) {
            primes.push_back(i);
        }

        for (int j = 0; j < primes.size() && i * primes[j] <= n; ++j) {
            is_prime[i * primes[j]] = false;
            if (i % primes[j] == 0) {
                break;
            }
        }
    }
    return primes;
}
```

答案：正确

解析：线性筛在遍历 `i` 时，用已得到的质数 `primes[j]` 标记 `i * primes[j]` 为合数；当 `i % primes[j] == 0` 时立即 `break`，保证后续不会用更大的质数重复标记同一个合数。这样每个合数都会由它的最小质因子对应的方式筛掉一次，整体标记次数为线性级别，因此时间复杂度为 $O(n)$，判断为正确。
