## 第1题

C++ 中 `string` 的 `==` 运算符比较的是字符串的内存地址，而非内容。

【知识点：字符串处理】

答案：错误

解析：C++ 标准库中的 `string` 已经重载了 `==` 运算符，比较的是两个字符串的内容是否相同，而不是对象所在的内存地址。比如两个不同的 `string` 对象只要都保存 `"abc"`，用 `==` 比较就会得到 `true`。因此题目把内容比较说成地址比较，是错误的。



## 第2题

`string` 的 `substr(1, 3)` 返回从下标 $1$ 开始的 $3$ 个字符的子串。

【知识点：数组与字符数组】

答案：正确

解析：`substr(pos, len)` 的第一个参数 `pos` 表示起始下标，第二个参数 `len` 表示最多取多少个字符。`substr(1, 3)` 就是从下标 $1$ 的字符开始，连续取 $3$ 个字符。例如对 `"abcdef"` 调用它，结果是 `"bcd"`，所以判断为正确。



## 第3题

$x$ 是浮点数，`(x >> 1)` 等价于 `x / 2`。

【知识点：位运算】

答案：错误

解析：`>>` 是位移运算符，要求操作数是整数类型或可进行整数提升的类型，浮点数不能直接做右移运算。即使对整数来说，右移一位在很多情况下类似除以 $2$，也不能把它推广到浮点数 `x`。所以 `(x >> 1)` 不等价于 `x / 2`，判断为错误。



## 第4题

`string("hello") == "hello"` 的比较结果为 `true`。

【知识点：字符串处理】

答案：正确

解析：`string("hello")` 会构造一个内容为 `"hello"` 的 `string` 对象。它与右侧字符串字面量 `"hello"` 比较时，C++ 会按字符串内容进行比较，两个内容完全相同，所以表达式结果为 `true`。



## 第5题

`sort` 可以直接用于排序 `set` 中的元素。

【知识点：枚举与基础算法】

答案：错误

解析：`set` 本身会按照比较规则维护元素的有序结构，遍历时已经是有序的。标准库 `sort` 需要随机访问迭代器，而 `set` 的迭代器不是随机访问迭代器，不能像数组或 `vector` 那样直接传给 `sort` 排序。因此说 `sort` 可以直接用于排序 `set` 中的元素是错误的。



## 第6题

`(x & 1) == 0` 可以判断整数 $x$ 是否为偶数。

【知识点：位运算】

答案：正确

解析：整数的二进制最低位可以表示奇偶性：偶数最低位是 $0$，奇数最低位是 $1$。表达式 `x & 1` 只保留 `x` 的最低位，所以当 `(x & 1) == 0` 时，说明最低位为 $0$，`x` 是偶数。这个判断常用于整数奇偶判断，因此正确。



## 第7题

`string` 的 `substr(2, 10)` 在字符串长度不足时会抛出异常。

【知识点：字符串处理】

答案：错误

解析：`substr(2, 10)` 中起点下标 $2$ 只要没有超过字符串长度，就可以正常截取。若从下标 $2$ 开始后剩余字符不足 $10$ 个，`substr` 会一直取到字符串末尾，而不是因为长度不够就抛出异常。只有起始位置超过允许范围时才会出问题，所以本题说法错误。



## 第8题

在数学纸面计算中，`pow(2, 3)` 的计算结果一定是 $8$，但是在 C++ 中，如果遇到数据类型是浮点数，那就不一定正确。

【知识点：函数与标准库】

答案：正确

解析：纸面上 $2^3$ 一定等于 $8$，但 C++ 的 `pow` 通常使用浮点类型参与计算并返回浮点结果。浮点数在计算机中不能精确表示所有实数，连续运算后可能出现很小的舍入误差。虽然 `pow(2, 3)` 这个例子通常会得到 $8$，但题目强调遇到浮点数据时不一定完全精确，这个提醒是正确的。



## 第9题

在 C++ 中，枚举的底层类型可以是非整型（如 `float` 或 `double`）。

【知识点：枚举与基础算法】

答案：错误

解析：C++ 的枚举类型本质上使用某种整型作为底层类型，例如 `int`、`unsigned int` 等。即使在限定作用域枚举中手动指定底层类型，也必须指定整型类型，不能指定 `float` 或 `double`。所以“枚举的底层类型可以是非整型”是错误的。



## 第10题

函数声明 `double f();` 返回 `int` 时，会自动转换为 `double`。

【知识点：函数与标准库】

答案：正确

解析：函数声明 `double f();` 表示调用者认为 `f` 的返回类型是 `double`。如果函数体中写了 `return` 一个 `int` 表达式，C++ 会在返回时把这个 `int` 值转换成 `double` 类型，例如返回 `3` 会变成 `3.0`。因此这个说法是正确的。
