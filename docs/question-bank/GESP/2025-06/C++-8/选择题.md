## 第1题

一间的机房要安排 $6$ 名同学进行上机考试，座位共 $2$ 行 $3$ 列。考虑到在座位上很容易看到同一行的左右两侧的屏幕，安排中间一列的同学做 A 卷，左右两列的同学做 B 卷。请问共有多少种排座位的方案？（ ）。

A. $720$
B. $90$
C. $48$
D. $15$

答案：A

解析：座位本身是 $2$ 行 $3$ 列的 $6$ 个不同位置，A 卷和 B 卷只由列的位置决定，并没有限制某位同学必须坐哪一类试卷的位置。因此只需要把 $6$ 名不同同学排列到 $6$ 个不同座位上，共有 $6! = 720$ 种方案。若先去选“谁做 A 卷”会把固定座位的排列顺序漏掉，容易得到偏小的结果。



## 第2题

又到了毕业季，学长学姐们都在开心地拍毕业照。现在有 $3$ 位学长、$3$ 位学姐希望排成一排拍照，要求男生不相邻、女生不相邻。请问共有多少种拍照方案？（ ）。

A. $720$
B. $72$
C. $36$
D. $2$

答案：B

解析：男女各 $3$ 人且同类不能相邻，所以性别只能完全交替，形态为“男男女女”之类的排法都不允许。可行的性别框架只有男-女-男-女-男-女和女-男-女-男-女-男两种；在每个框架中，$3$ 位学长可任意排列，$3$ 位学姐也可任意排列。因此总数为 $2 \times 3! \times 3! = 72$。



## 第3题

下列关于 C++ 类和对象的说法，错误的是（ ）。

A. 通过语句 `const int x = 5;` 定义了一个对象 x。
B. 通过语句 `std::string t = "12345";` 定义了一个对象 t。
C. 通过语句 `void (*fp)() = NULL;` 定义了一个对象 fp。
D. 通过语句 `class MyClass;` 定义了一个类 MyClass。

答案：D

解析：`const int x = 5;` 定义了一个 `int` 类型对象，`std::string t = "12345";` 定义了一个字符串对象，`void (*fp)() = NULL;` 定义的是函数指针对象 `fp`。而 `class MyClass;` 只是类的前向声明，让编译器知道有这样一个类名，并没有给出类体成员，因此不是类定义。题目要求选错误说法，所以应选 D。



## 第4题

关于生成树的说法，错误的是（ ）。

A. 一个无向连通图，一定有生成树。
B. $n$ 个顶点的无向图，其生成树要么不存在，要么一定包含 $n - 1$ 条边。
C. $n$ 个顶点、$n - 1$ 条边的无向图，不可能有多颗生成树。
D. $n$ 个顶点、$n - 1$ 条边的无向图，它本身就是自己的生成树。

答案：D

解析：生成树必须包含原图全部 $n$ 个顶点，并且本身连通、无环。无向连通图一定能取出一棵生成树，且任何 $n$ 个顶点的生成树都有 $n - 1$ 条边；如果一个 $n$ 点 $n - 1$ 边的图连通，它本身就是一棵树，生成树也唯一。D 忽略了“连通”条件：例如有孤立点的无向图即使边数为 $n - 1$，也不存在生成树，更不能说它本身就是生成树。



## 第5题

一对夫妻生男生女的概率相同。这对夫妻希望儿女双全。请问这对夫妻生下两个孩子时，实现儿女双全的概率是多少？（ ）。

A. $\frac{2}{3}$
B. $\frac{1}{3}$
C. $\frac{1}{2}$
D. $\frac{1}{4}$

答案：C

解析：两个孩子的性别按出生顺序共有男男、男女、女男、女女 $4$ 种等可能情况。儿女双全对应男女和女男两种，所以概率为 $\frac{2}{4} = \frac{1}{2}$。这里不能把“至少有一个男孩”和“至少有一个女孩”当成两个独立事件相乘，否则会重复或漏算顺序情况。



## 第6题

已定义变量 `double a, b;`，下列哪个表达式可以用来判断一元二次方程 $x^2 + ax + b = 0$ 是否有实根？（ ）。

A. `4 * b - a * a < 0`
B. `4 * b <= a * a`
C. `a * a - 4 * b`
D. `b * 4 - a * a`

答案：B

解析：方程 $x^2 + ax + b = 0$ 有实根的条件是判别式 $\Delta = a^2 - 4b \ge 0$。把这个不等式移项得到 $4b \le a^2$，对应表达式 `4 * b <= a * a`，结果类型也是适合条件判断的 `bool`。A 判断的是无实根；C 和 D 是数值表达式，作为条件时只按是否为 $0$ 转成真假，不能正确表达“有实根”。



## 第7题

$n$ 个结点的二叉树，执行广度优先搜索的平均时间复杂度是（ ）。

A. $O(\log n)$
B. $O(n \log n)$
C. $O(n)$
D. $O(2^n)$

答案：C

解析：二叉树的广度优先搜索按层访问结点，每个结点入队一次、出队一次，检查左右孩子也只发生常数次。整棵树共有 $n$ 个结点，因此总操作次数与 $n$ 成正比，平均时间复杂度为 $O(n)$。树是否平衡只会影响队列峰值和层数，不会把“访问每个结点一次”变成 $O(\log n)$ 或 $O(n\log n)$。



## 第8题

以下关于动态规划的说法中，错误的是（ ）。

A. 动态规划方法通常能够列出递推公式。
B. 动态规划方法的时间复杂度通常为状态的个数。
C. 动态规划方法有递推和递归两种实现形式。
D. 对很多问题，递推实现和递归实现动态规划方法的时间复杂度相当。

答案：B

解析：动态规划通常需要写出状态定义和递推关系，也可以用自底向上的递推或带记忆化的递归实现；在很多问题中，两种实现访问的状态和转移相同，复杂度相当。B 的问题在于少算了“每个状态需要枚举多少转移”：总时间通常是状态数乘以单个状态的转移代价，例如朴素 LIS 有 $n$ 个状态但每个状态枚举前面的元素，总时间是 $O(n^2)$。



## 第9题

下面的 `sum_digit` 函数试图求出从 $1$ 到 $n$（包含 $1$ 和 $n$）的数中，包含数字 $d$ 的个数。该函数的时间复杂度为（ ）。

```cpp
#include <string>
int count_digit(int n, char d) {
    int cnt = 0;
    std::string s = std::to_string(n);
    for (int i = 0; i < s.length(); i++)
        if (s[i] == d)
            cnt++;
    return cnt;
}
int sum_digit(int n, char d) {
    int sum = 0;
    for (int i = 1; i <= n; i++)
        sum += count_digit(i, d);
    return sum;
}
```

A. $O(n \log n)$
B. $O(n)$
C. $O(\log n)$
D. $O(n^2)$

答案：A

解析：`sum_digit` 的外层循环从 $1$ 到 $n$ 调用 `count_digit`。一次 `count_digit(i, d)` 会把整数 $i$ 转成字符串并扫描所有十进制位，位数是 $O(\log i)$，最多为 $O(\log n)$。因此总时间可按 $\sum_{i=1}^{n} O(\log i)$ 估计，为 $O(n \log n)$。只看外层循环会漏掉每次数位扫描的代价。



## 第10题

下面程序的输出为（ ）。

```cpp
#include <iostream>
const int N = 10;
int ch[N][N][N];
int main() {
    for (int x = 0; x < N; x++)
        for (int y = 0; y < N; y++)
            for (int z = 0; z < N; z++)
                if (x == 0 && y == 0 && z == 0)
                    ch[x][y][z] = 1;
                else {
                    if (x > 0)
                        ch[x][y][z] += ch[x - 1][y][z];
                    if (y > 0)
                        ch[x][y][z] += ch[x][y - 1][z];
                    if (z > 0)
                        ch[x][y][z] += ch[x][y][z - 1];
                }
    std::cout << ch[1][2][3] << std::endl;
    return 0;
}
```

A. $60$
B. $20$
C. $15$
D. $10$

答案：A

解析：数组递推满足 `ch[x][y][z] = ch[x - 1][y][z] + ch[x][y - 1][z] + ch[x][y][z - 1]`，初值 `ch[0][0][0] = 1`。它等价于从 $(0,0,0)$ 走到 $(x,y,z)$，每步只让一个坐标加 $1$ 的路径数。到 $(1,2,3)$ 一共要走 $1 + 2 + 3 = 6$ 步，其中 $x$ 方向 $1$ 步、$y$ 方向 $2$ 步、$z$ 方向 $3$ 步，排列数为 $\frac{6!}{1!2!3!} = 60$。



## 第11题

下面 `count_triple` 函数的时间复杂度为（ ）。

```cpp
int gcd(int a, int b) {
    if (a == 0)
        return b;
    return gcd(b % a, a);
}
int count_triple(int n) {
    int cnt = 0;
    for (int v = 1; v * v * 4 <= n; v++)
        for (int u = v + 1; u * (u + v) * 2 <= n; u += 2)
            if (gcd(u, v) == 1) {
                int a = u * u - v * v;
                int b = u * v * 2;
                int c = u * u + v * v;
                cnt += n / (a + b + c);
            }
    return cnt;
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(n \log n)$
D. $O(n^2 \log n)$

答案：C

解析：外层变量 `v` 满足 $4v^2 \le n$，所以大约有 $O(\sqrt n)$ 种取值。对固定的 `v`，内层变量 `u` 满足 $2u(u+v) \le n$，也至多是 $O(\sqrt n)$ 次，因此两层循环组合上界为 $O(n)$。循环内的 `gcd(u, v)` 使用欧几里得算法，单次代价为 $O(\log n)$，所以总时间复杂度为 $O(n \log n)$。



## 第12题

下面 quick_sort 函数试图实现快速排序算法，两横线处分别应该填入的是（ ）。

```cpp
void swap(int & a, int & b) {
    int temp = a; a = b; b = temp;
}

int partition(int a[], int l, int r) {
    int pivot = a[l], i = l + 1, j = r;
    while (i <= j) {
        while (i <= j && a[j] >= pivot)
            j--;
        while (i <= j && a[i] <= pivot)
            i++;
        if (i < j)
            swap(a[i], a[j]);
    }
    ________;           // 在此处填入选项
    return ________;    // 在此处填入选项
}

void quick_sort(int a[], int l, int r) {
    if (l < r) {
        int pivot = partition(a, l, r);
        quick_sort(a, l, pivot - 1);
        quick_sort(a, pivot + 1, r);
    }
}
```

A.

```cpp
swap(a[l], a[i])
i
```

B.

```cpp
swap(a[l], a[j])
i
```

C.

```cpp
swap(a[l], a[i])
j
```

D.

```cpp
swap(a[l], a[j])
j
```

答案：D

解析：这段划分过程以 `a[l]` 为基准，右指针 `j` 停在小于基准的位置，左指针 `i` 停在大于基准的位置，交换后继续推进。循环结束时，`j` 左侧元素不大于基准，`j` 右侧元素不小于基准，应把基准 `a[l]` 与 `a[j]` 交换，并返回基准最终下标 `j`。若返回 `i`，递归区间会把基准位置算错；若交换 `a[i]`，还可能把大于基准的元素换到开头。



## 第13题

下面 `LIS` 函数试图求出最长上升子序列的长度，横线处应该填入的是（ ）。

```cpp
int max(int a, int b) {
    return (a > b) ? a : b;
}
int LIS(vector<int> & nums) {
    int n = nums.size();
    if (n == 0)
        return 0;
    vector<int> dp(n, 1);
    int maxLen = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++)
            if (nums[j] < nums[i])
                ________;  // 在此处填入选项
        maxLen = max(maxLen, dp[i]);
    }
    return maxLen;

```

A. `dp[j] = max(dp[j] + 1, dp[i])`
B. `dp[j] = max(dp[j], dp[i] + 1)`
C. `dp[i] = max(dp[i] + 1, dp[j])`
D. `dp[i] = max(dp[i], dp[j] + 1)`

答案：D

解析：`dp[i]` 表示以 `nums[i]` 结尾的最长上升子序列长度。当前面某个 `j < i` 且 `nums[j] < nums[i]` 时，可以把 `nums[i]` 接到以 `nums[j]` 结尾的序列后面，候选长度是 `dp[j] + 1`，因此要写 `dp[i] = max(dp[i], dp[j] + 1)`。修改 `dp[j]` 会改变已经计算好的旧状态；写成 `dp[i] + 1` 则不是从前驱状态转移而来。



## 第14题

下面 `LIS` 函数试图求出最长上升子序列的长度，其时间复杂度为（ ）。

```cpp
#define INT_MIN (-1000)
int LIS(vector<int> & nums) {
    int n = nums.size();
    vector<int> tail;
    tail.push_back(INT_MIN);
    for (int i = 0; i < n; i++) {
        int x = nums[i], l = 0, r = tail.size();
        while (l < r) {
            int mid = (l + r) / 2;
            if (tail[mid] < x)
                l = mid + 1;
            else
                r = mid;
        }
        if (r == tail.size())
            tail.push_back(x);
        else
            tail[r] = x;
    }
    return tail.size() - 1;
}
```

A. $O(\log n)$
B. $O(n)$
C. $O(n \log n)$
D. $O(n^2)$

答案：C

解析：外层 `for` 循环处理 `nums` 中的每个元素，共 $n$ 次。每次在 `tail` 中用二分查找第一个不小于 `x` 的位置，`tail` 的长度最多为 $n + 1$，所以单次查找是 $O(\log n)$；后续 `push_back` 或赋值是 $O(1)$。总时间复杂度为 $O(n \log n)$，不是 $O(n^2)$，因为没有对每个元素再线性枚举所有前驱。



## 第15题

下面的程序使用邻接矩阵表达的带权无向图，则从顶点 $0$ 到顶点 $3$ 的最短距离为（ ）。

```cpp
int weight[4][4] = {
    {0, 5, 8, 10},
    {5, 0, 1, 7},
    {8, 1, 0, 3},
    {10, 7, 3, 0}
};
```

A. $9$
B. $10$
C. $11$
D. $12$

答案：A

解析：从顶点 $0$ 到顶点 $3$ 的直接边权是 $10$，但可以经过中间点缩短距离。路径 $0 \to 1 \to 2 \to 3$ 的长度为 $5 + 1 + 3 = 9$；路径 $0 \to 2 \to 3$ 为 $8 + 3 = 11$，路径 $0 \to 1 \to 3$ 为 $5 + 7 = 12$。所有边权为正，绕更多边不会优于已找到的 $9$，所以最短距离为 $9$。
