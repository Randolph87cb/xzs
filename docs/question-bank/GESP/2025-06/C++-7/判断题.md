# 判断题

## 第1题

C++ 语言中，表达式 `9 & 12` 的结果类型为 int 、值为 8 。

答案：正确

解析：按位与 `&` 会逐位比较两个整数的二进制位，只有两边同为 $1$ 的位置结果才为 $1$。$9$ 的二进制是 `1001`，$12$ 的二进制是 `1100`，按位与得到 `1000`，也就是十进制 $8$。两个操作数都是整型字面量，表达式结果类型为 `int`，所以该说法正确。



## 第2题

C++ 语言中，指针变量指向的内存地址不一定都能够合法访问。

答案：正确

解析：指针变量保存的是一个地址值，但“有地址”不等于“当前程序可以合法访问”。例如空指针、未初始化指针、已经释放对象的悬空指针，或者指向数组末尾后一位的指针，都不能被随意解引用。只有指向有效对象或函数、并满足访问规则的指针，才能进行相应访问，所以该说法正确。



## 第3题

对 $n$ 个元素的数组进行快速排序，最差情况的时间复杂度为 $O(n \log n)$。

答案：错误

解析：快速排序平均情况下通常是 $O(n \log n)$，但最差情况会退化到 $O(n^2)$。例如每次划分都只把一个元素放到最终位置，剩下 $n-1$ 个元素继续递归，递推规模接近 $n + (n-1) + \cdots + 1$。因此把最差情况说成 $O(n \log n)$ 是错误的。



## 第4题

一般情况下， long long 类型占用的字节数比 float 类型多。

答案：正确

解析：在常见的 C++ 编译环境中，`long long` 通常占 $8$ 字节，`float` 通常占 $4$ 字节，因此一般情况下 `long long` 占用字节数更多。标准只规定了类型的最小范围和相对要求，具体大小与实现有关；题干使用“一般情况下”，符合常见平台上的判断，所以为正确。



## 第5题

使用 math.h 或 cmath 头文件中的函数，表达式 `pow(10, 3)` 的结果的值为 1000 、类型为 int 。

答案：错误

解析：`pow(10, 3)` 的数值结果确实是 $1000$，但 `pow` 是数学库函数，常用重载的返回类型是浮点类型，典型情况下为 `double`，不是 `int`。如果需要整数结果，需要再进行合适的类型转换或使用整数运算。因此题干把结果类型也说成 `int`，判断为错误。



## 第6题

二叉排序树的中序遍历序列一定是有序的。

答案：正确

解析：二叉排序树满足左子树中结点关键字小于根，右子树中结点关键字大于根，左右子树也分别是二叉排序树。中序遍历的顺序是“左子树、根、右子树”，正好先输出较小关键字，再输出根，再输出较大关键字。递归到整棵树后，得到的序列就是有序序列，所以该说法正确。



## 第7题

无论哈希表采用何种方式解决冲突，只要管理的元素足够多，都无法避免冲突。

答案：正确

解析：哈希函数把元素映射到有限的哈希地址或桶中；当要管理的元素数量足够多时，根据抽屉原理，必然会有至少两个元素落到同一个地址。链地址法、开放定址法、再哈希等方法只能在冲突发生后保存和查找这些元素，并不能在元素无限增加时保证永远没有冲突。因此该说法正确。



## 第8题

在 C++ 语言中，类的构造函数和析构函数均可以声明为虚函数。

答案：错误

解析：析构函数可以声明为虚函数，这样通过基类指针删除派生类对象时能够正确调用派生类析构函数。但构造函数不能声明为虚函数，因为对象在构造期间其动态类型和虚函数机制尚未按完整对象使用，构造过程必须从基类到派生类确定执行。题干说构造函数和析构函数均可以为虚函数，包含了错误部分，所以判断为错误。



## 第9题

动态规划方法将原问题分解为一个或多个相似的子问题，因此必须使用递归实现。

答案：错误

解析：动态规划确实会把原问题拆成相互关联的子问题，并保存子问题结果以避免重复计算，但实现方式不必须是递归。常见写法既可以是递归加记忆化搜索，也可以是按照状态顺序自底向上递推，例如背包问题常用循环填表完成。题干把“可递归实现”说成“必须递归实现”，因此错误。



## 第10题

如果将城市视作顶点，公路视作边，将城际公路网络抽象为简单图，可以满足城市间的车道级导航需求。

答案：错误

解析：简单图只记录顶点之间是否有边，通常不允许平行边，也缺少车道数、转向限制、匝道、限速、单行方向、实时路况等导航所需信息。把城市抽象成顶点、公路抽象成边，可以用于较粗粒度的连通性或路径分析，但无法表达车道级导航的细节。因此该说法错误。
