## 第1题

已知小写字母 `b` 的 ASCII 码为 $ 98 $，下列 C++ 代码的输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    char a = 'b' ^ 4;
    cout << a;
    return 0;
}
```

A. b
B. bbbb
C. f
D. 102

答案：C

解析：字符 `b` 的 ASCII 码是 $98$，二进制低位可看成 `1100010`。整数 $4$ 的二进制是 `0000100`，按位异或只会把从右往左第 $3$ 位翻转，得到 $98 \mathbin{\hat{}} 4 = 102$。`char a` 中存放的是编码值 $102$ 对应的字符，所以 `cout << a` 输出字符 `f`，不是数字 `102`；`b` 或 `bbbb` 都没有发生按位异或后的编码变化。



## 第2题

已知 `a` 为 `int` 类型变量，`p` 为 `int *` 类型变量，下列赋值语句不符合语法的是（ ）。

A. `*(p + a) = *p;`
B. `*(p - a) = a;`
C. `p + a = p;`
D. `p = p + a;`

答案：C

解析：赋值号左边必须是可修改的左值。`p + a` 是指针运算产生的临时结果，表示从 `p` 偏移 `a` 个 `int` 后得到的新地址，本身不能作为被赋值对象，因此 `p + a = p;` 语法错误。A、B 的左边 `*(p + a)`、`*(p - a)` 是解引用后的 `int` 对象，语法上可以赋值；D 是把新的指针值赋给指针变量 `p`，语法上也成立。



## 第3题

下列关于 C++ 类的说法，错误的是（ ）。

A. 如需要使用基类的指针释放派生类对象，基类的析构函数应声明为虚析构函数。
B. 构造派生类对象时，只调用派生类的构造函数，不会调用基类的构造函数。
C. 基类和派生类分别实现了同一个虚函数，派生类对象仍能够调用基类的该方法。
D. 如果函数形参为基类指针，调用时可以传入派生类指针作为实参。

答案：B

解析：构造派生类对象时，必须先构造其中的基类子对象，再构造派生类自身部分，所以基类构造函数会被调用，B 的说法错误。A 是释放多态对象时的常见规则：若通过基类指针 `delete` 派生类对象，基类析构函数应为虚函数。C 中派生类即使重写了虚函数，也仍可通过作用域限定如 `Base::func()` 调用基类版本。D 体现的是向上转型，派生类指针可以传给基类指针形参。



## 第4题

下列 C++ 代码的输出是（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int arr[5] = {2, 4, 6, 8, 10};
    int * p = arr + 2;
    cout << p[3] << endl;
    return 0;
}
```

A. 6
B. 8
C. 编译出错，无法运行。
D. 不确定，可能发生运行时异常。

答案：D

解析：数组 `arr` 的合法下标是 `0` 到 `4`，`p = arr + 2` 指向 `arr[2]`，也就是值为 $6$ 的位置。表达式 `p[3]` 等价于 `*(p + 3)`，实际访问的是 `arr[5]`，已经越过数组末尾并被解引用，属于未定义行为。程序可能打印某个偶然值，也可能运行异常；它不是确定输出 `6` 或 `8`，通常也不会因为这行语法本身而编译出错。



## 第5题

假定只有一个根节点的树的深度为 1，则一棵有 $ N $ 个节点的完全二叉树，则树的深度为（ ）。

A. $ \lfloor \log_2(N) \rfloor + 1 $
B. $ \lfloor \log_2(N) \rfloor $
C. $ \lceil \log_2(N) \rceil $
D. 不能确定。

答案：A

解析：完全二叉树按层从上到下、从左到右填充。深度为 $d$ 的完全二叉树节点数满足 $2^{d-1} \le N \le 2^d - 1$，因为前 $d-1$ 层至少要填满，最后一层至少有一个节点。由 $2^{d-1} \le N$ 可得 $d-1 \le \log_2 N$，对应深度为 $\lfloor \log_2(N) \rfloor + 1$。只写 $\lfloor \log_2(N) \rfloor$ 会少算根所在的一层。



## 第6题

对于如下图的二叉树，说法正确的是（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/75fek6di.png)

A. 先序遍历是 ABDEC。
B. 中序遍历是 BDACE。
C. 后序遍历是 DBCEA。
D. 广度优先遍历是 ABCDE。

答案：D

解析：图中根节点是 `A`，它的左、右孩子分别是 `B`、`C`，`B` 的孩子是 `D`，`C` 的孩子是 `E`。广度优先遍历按层访问，先访问第 $1$ 层 `A`，再访问第 $2$ 层 `B`、`C`，最后访问第 $3$ 层 `D`、`E`，得到 `ABCDE`。先序应为 `ABDCE`，中序应为 `DBAEC`，后序应为 `DBECA`，所以 A、B、C 都不对。



## 第7题

图的存储和遍历算法，下面说法错误的是（ ）。

A. 图的深度优先遍历须要借助队列来完成。
B. 图的深度优先遍历和广度优先遍历对有向图和无向图都适用。
C. 使用邻接矩阵存储一个包含 $ v $ 个顶点的有向图，统计其边数的时间复杂度为 $ O(v^2) $。
D. 同一个图分别使用出边邻接表和入边邻接表存储，其边结点个数相同。

答案：A

解析：深度优先遍历的核心是沿一条路径尽量向深处走，通常用递归或栈维护待回溯的顶点；队列是广度优先遍历的典型辅助结构。因此“深度优先遍历须要借助队列”这一说法错误。DFS 和 BFS 都可用于有向图、无向图；邻接矩阵统计边要扫描 $v \times v$ 个位置，时间为 $O(v^2)$；同一张图无论按出边表还是入边表存储，每条边都对应一个边结点。



## 第8题

一个连通的简单有向图，共有 28 条边，则该图至少有（ ）个顶点。

A. 5
B. 6
C. 7
D. 8

答案：B

解析：简单有向图不允许自环，也不允许同一方向的重边；若有 $v$ 个顶点，最多有 $v(v-1)$ 条有向边。要容纳 $28$ 条边，需要 $v(v-1) \ge 28$。当 $v=5$ 时最多 $20$ 条，不够；当 $v=6$ 时最多 $30$ 条，已经可以达到 $28$ 条。连通性只会增加最低结构要求，不会让 $5$ 个顶点突破简单有向图的最大边数，所以至少需要 $6$ 个顶点。



## 第9题

以下哪个方案不能合理解决或缓解哈希表冲突（ ）。

A. 在每个哈希表项处，使用不同的哈希函数再建立一个哈希表，管理该表项的冲突元素。
B. 在每个哈希表项处，建立二叉排序树，管理该表项的冲突元素。
C. 使用不同的哈希函数建立额外的哈希表，用来管理所有发生冲突的元素。
D. 覆盖发生冲突的旧元素。

答案：D

解析：哈希冲突的处理目标是让多个映射到同一位置的元素仍然能够被保存和查找。A 类似分层哈希，B 类似把桶内元素组织成二叉排序树，C 用额外哈希结构管理冲突元素，都还保留了发生冲突的元素。D 直接覆盖旧元素会丢失原来存入的数据，之后无法正确查询旧元素，因此不是合理的冲突处理方法。



## 第10题

以下关于动态规划的说法中，错误的是（ ）。

A. 动态规划方法通常能够列出递推公式。
B. 动态规划方法的时间复杂度通常为状态的个数。
C. 动态规划方法有递推和递归两种实现形式。
D. 对很多问题，递推实现和递归实现动态规划方法的时间复杂度相当。

答案：B

解析：动态规划的总时间通常由“状态个数”和“计算每个状态所需的转移代价”共同决定，而不一定只等于状态个数。例如每个状态要枚举 $k$ 种转移时，时间会多出一个 $k$ 的因子。A 中递推公式是动态规划的重要表达方式；C 中既可以自底向上递推，也可以用递归加记忆化；D 中很多问题的两种写法都会计算同一批状态，复杂度可以相当。



## 第11题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;
int rec_fib[100];
int fib(int n) {
    if (n <= 1)
        return n;
    if (rec_fib[n] == 0)
        rec_fib[n] = fib(n - 1) + fib(n - 2);
    return rec_fib[n];
}
int main() {
    cout << fib(6) << endl;
    return 0;
}
```

A. 8
B. 13
C. 64
D. 结果是随机的。

答案：A

解析：函数实现的是带记忆化的斐波那契数列，边界为 `fib(0)=0`、`fib(1)=1`。继续计算得到 `fib(2)=1`、`fib(3)=2`、`fib(4)=3`、`fib(5)=5`、`fib(6)=8`，所以输出 `8`。数组 `rec_fib` 是全局数组，初值为 $0$，用于记录已经算过的项，不会使结果随机；`13` 是 `fib(7)` 的值，`64` 与这里的递推关系无关。



## 第12题

下面程序的时间复杂度为（ ）。

```cpp
int rec_fib[MAX_N];
int fib(int n) {
    if (n <= 1)
        return n;
    if (rec_fib[n] == 0)
        rec_fib[n] = fib(n - 1) + fib(n - 2);
    return rec_fib[n];
}
```

A. $ O(2^n) $
B. $ O(\phi^n) $, $ \phi = \frac{\sqrt{5} - 1}{2} $
C. $ O(n^2) $
D. $ O(n) $

答案：D

解析：这段代码使用 `rec_fib[n]` 记忆化结果。对每个下标 $0$ 到 $n$，真正展开计算至多发生一次；之后再次访问同一项会直接返回数组中保存的值。每个状态只做常数次加法和判断，因此总时间复杂度为 $O(n)$。指数级复杂度对应的是没有记忆化的朴素递归斐波那契，不适用于这段代码。



## 第13题

下面 search 函数的平均时间复杂度为（ ）。

```cpp
int search(int n, int * p, int target) {
    int low = 0, high = n;
    while (low < high) {
        int middle = (low + high) / 2;
        if (target == p[middle]) {
            return middle;
        } else if (target > p[middle]) {
            low = middle + 1;
        } else {
            high = middle;
        }
    }
    return -1;
}
```

A. $ O(n \log(n)) $
B. $ O(n) $
C. $ O(\log(n)) $
D. $ O(1) $

答案：C

解析：`search` 每次取区间 `[low, high)` 的中点 `middle`，然后把搜索区间缩小到左半部分或右半部分，区间长度大约减半。长度为 $n$ 的区间最多经过约 $\log_2 n$ 次减半就会变为空或找到目标，每轮只做常数次比较和赋值，所以平均时间复杂度为 $O(\log n)$。线性或 $O(n\log n)$ 都没有体现二分查找的减半过程，$O(1)$ 又忽略了循环次数随 $n$ 增长。



## 第14题

下面程序的时间复杂度为（ ）。

```cpp
int primes[MAXP], num = 0;
bool isPrime[MAXN] = {false};
void sieve() {
    for (int n = 2; n <= MAXN; n++) {
        if (!isPrime[n])
            primes[num++] = n;
        for (int i = 0; i < num && n * primes[i] <= MAXN; i++) {
            isPrime[n * primes[i]] = true;
            if (n % primes[i] == 0)
                break;
        }
    }
}
```

A. $ O(n) $
B. $ O(n \times \log n) $
C. $ O(n \times \log \log n) $
D. $ O(n^2) $

答案：A

解析：这段筛法是线性筛的写法。外层枚举每个 `n`，内层用已找到的质数标记 `n * primes[i]`，并在 `n % primes[i] == 0` 时停止，保证每个合数只会被它的最小质因子对应地标记一次。若把筛选上界记为 $n$，所有标记操作总数为线性级别，因此时间复杂度是 $O(n)$，不是埃氏筛常见的 $O(n \log\log n)$，更不是平方级。



## 第15题

下列选项中，哪个不可能是下图的广度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/23zak3jy.png)

A. 1, 2, 4, 5, 3, 7, 6, 8, 9
B. 1, 2, 5, 4, 3, 7, 8, 6, 9
C. 1, 4, 5, 2, 7, 3, 8, 6, 9
D. 1, 5, 4, 2, 7, 3, 8, 6, 9

答案：B

解析：从顶点 `1` 开始广度优先遍历时，第一层可按某种邻接顺序访问 `2`、`4`、`5`。如果序列 B 的前三个邻接点顺序是 `2, 5, 4`，处理 `2` 时会先发现 `3`，处理 `5` 或 `4` 时会发现 `7`，因此队列中应先有 `3`、再有 `7`。访问 `3` 后会发现 `6`，访问 `7` 后才会发现 `8`，所以 `6` 必须排在 `8` 前面；B 把 `8` 放在 `6` 前，不可能由 BFS 得到。A、C、D 都可以通过不同的邻接点访问顺序产生。
