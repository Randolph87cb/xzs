## 第1题

人们现在参加很多闭卷考试时通常都不允许带智能手机、平板电脑等，此外很多种智能手表同样因为具有嵌入操作系统及通信等功能，所以也不允许随身携带。（ ）

答案：正确

解析：这个判断是正确的。智能手机、平板电脑和很多智能手表都不只是普通计时工具，它们通常带有操作系统，可以存储资料、运行程序，还可能通过蓝牙、蜂窝网络或无线网络通信。闭卷考试限制这类设备，是为了避免查资料、传递信息或使用计算功能。



## 第2题

在 C++ 代码中，假设 $N$ 为正整数，则 $N / 10$ 舍弃个位数。如果 $N$ 小于 $10$，则其值为 $0$，大于 $10$ 则是舍弃个位数的数。（ ）

答案：正确

解析：这个判断是正确的。`N` 是正整数时，`N / 10` 使用整数除法，小数部分会被舍去，相当于删掉十进制表示的个位。例如 `97 / 10` 得到 `9`，`123 / 10` 得到 `12`；如果 `N` 是 `1` 到 `9`，结果就是 `0`。



## 第3题

下列 C++ 代码执行后，其输出为 $10\ 20$，即 `a == b` 和 `b == a` 对 $a$ 和 $b$ 的值没有任何影响。（ ）

```cpp
int a = 10, b = 20;
a == b;
b == a;
cout << (a, b);
```

答案：错误

解析：`a == b;` 和 `b == a;` 确实只是比较，不会给变量赋值，`a` 仍是 `10`，`b` 仍是 `20`。但是 `cout << (a, b);` 里面的 `(a, b)` 是逗号表达式，整个表达式的值取最后一个 `b`，所以只输出 `20`，不会输出 `10 20`。因此本题判断为错误。



## 第4题

$a$ 和 $b$ 分别是 C++ 的整型变量，如果表达式 `max(a, b) == min(a, b)` 的值为真，则说明 $a$ 和 $b$ 相等。（ ）

答案：正确

解析：这个判断是正确的。`max(a, b)` 表示两数中较大的那个，`min(a, b)` 表示两数中较小的那个。如果较大的数和较小的数相等，就不可能一个大一个小，只能说明 `a` 和 `b` 本来就相等。



## 第5题

下面 C++ 代码编译时将报错，因为字符变量 $a$ 被赋值了浮点值。（ ）

```cpp
char a = '1';
a = 45.6;
cout << a;
```

答案：错误

解析：这段代码不一定会因为 `a = 45.6;` 编译报错。C++ 允许把数值赋给 `char` 变量，赋值时会发生类型转换；`45.6` 转成整数值时小数部分会被舍去，再作为字符编码保存到 `a` 中。这样的写法不推荐，因为输出字符不直观，但它不是“必然编译错误”。



## 第6题

下面 C++ 代码执行时如输入 $59.99$，将输出“及格”两个汉字。（ ）

```cpp
int score;
cout << "请输入学生成绩：";
cin >> score;
if (score < 60)
    cout << "不及格";
else
    cout << "及格";
```

答案：错误

解析：`score` 是 `int` 类型，输入 `59.99` 时，通常会先读入整数部分 `59`，后面的 `.99` 留在输入流中。随后判断 `score < 60`，也就是 `59 < 60`，条件成立，输出“不及格”。题目说会输出“及格”，所以判断为错误。



## 第7题

在 C++ 代码中，因为 `continue` 将被执行，因此不会有任何输出。（ ）

```cpp
int i;
for (i = 1; i < 10; i++)
    if (i % 2 == 0)
        continue;
if(i == 10)
    cout << "END";
```

答案：错误

解析：`for` 后面没有花括号，所以循环体只有紧跟着的 `if (i % 2 == 0) continue;`。偶数轮执行 `continue` 只是进入下一轮循环，奇数轮什么也不输出。循环正常结束后，`i` 已经变成 `10`，后面的 `if(i == 10)` 在循环外执行，于是会输出 `END`。因此“不会有任何输出”是错误的。



## 第8题

下面的 C++ 代码执行后将输出 $15$。（ ）

```cpp
int Sum = 0;
for (int i = 0; i < 5; i++)
    Sum += i;
cout << Sum;
```

答案：错误

解析：循环变量 `i` 从 `0` 开始，满足 `i < 5` 时执行，所以取到的是 `0, 1, 2, 3, 4`。累加结果为 $0 + 1 + 2 + 3 + 4 = 10$，不是 `15`。如果要加到 `5`，循环条件需要包含 `5`，但这里没有。



## 第9题

将下面 C++ 代码中的 `(int i = 5; i > 1; i--)` 调整为 `(int i = 1; i < 5; i++)` 输出结果相同，因为 $5$ 到 $1$ 与 $1$ 到 $5$ 的求和相同。（ ）

```cpp
int tnt;
tnt = 0;
for (int i = 5; i > 1; i--)
    tnt += i;
cout << tnt;
cout << endl;
```

答案：错误

解析：原循环是 `i = 5; i > 1; i--`，实际累加的是 `5, 4, 3, 2`，和为 $14$，并没有加到 `1`。改成 `i = 1; i < 5; i++` 后，累加的是 `1, 2, 3, 4`，和为 $10$。两个循环包含的数不同，输出结果也不同，所以判断为错误。



## 第10题

为实现如下效果，即 $N$ 行 $N$ 列字符。当输入是奇数时，中间列为 `*`，其他是 `-`；当输入是偶数时，则中间两列为 `*`，其他是 `-`。字符串列后的代码能实现其效果。（ ）

```cpp
/*
5
--*--
--*--
--*--
--*--
--*--
*/
```

```cpp
int N;
cin >> N;

int i, j;
for (i = 0; i < N; i++){
    for (j = 0; j < N; j++)
        if ((j == N / 2) || (j == (N-1) / 2))
            cout << "*";
        else
            cout << "-";
    cout << endl;
}
```

答案：正确

解析：这个判断是正确的。列下标 `j` 从 `0` 到 `N - 1`。当 `N` 为奇数，例如 `5`，`N / 2` 和 `(N - 1) / 2` 都是 `2`，正好只在中间一列输出 `*`。当 `N` 为偶数，例如 `6`，`N / 2` 是 `3`，`(N - 1) / 2` 是 `2`，正好对应中间两列。外层循环控制行数，所以每一行都会按同样规则输出。
