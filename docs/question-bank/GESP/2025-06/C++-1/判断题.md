## 第1题

人们现在参加很多闭卷考试时通常都不允许带智能手机、平板电脑等。此外很多智能手表同样因为具有嵌入操作系统及通信等功能，所以也不允许随身携带。（ ）

答案：正确

解析：智能手表不只是普通手表，很多型号内部有嵌入式操作系统，可以运行程序，还能通过蓝牙、蜂窝网络或无线网络通信。闭卷考试限制这类设备，是因为它们可能存储信息或与外界传递信息。题目说“很多智能手表同样因为具有嵌入操作系统及通信等功能，所以也不允许随身携带”是合理的，判断为正确。



## 第2题

如果 $N$ 是 C++ 的整型变量，值为 5，则表达式 `(N + !N)` 的值为 4。（ ）

答案：错误

解析：在 C++ 中，逻辑非 `!` 会把非零值变成 `false`，参与整数运算时就是 $0$。因为 `N = 5` 是非零数，所以 `!N` 的值为 $0$，`N + !N = 5 + 0 = 5`，不是 $4$。因此判断为错误。



## 第3题

在下面 C++ 代码中，删除 break 语句对程序执行结果没有影响。（ ）

```cpp
int i;
for (i = 0; i < 10; i++){
    continue;
    cout << i << "#";
    break;
}
if (i >= 10)
    cout << "END";
```

答案：正确

解析：循环体第一句是 `continue;`，每一轮都会直接跳到下一轮循环，后面的 `cout << i << "#";` 和 `break;` 都没有机会执行。既然原程序里的 `break` 永远执行不到，把它删掉也不会改变程序流程。循环正常走到 `i = 10` 后结束，仍会输出 `END`，所以判断为正确。



## 第4题

删除下面 C++ 代码中的 continue 后其输出是 0#2#4#6#8#。（ ）

```cpp
int i;
for (i = 0; i < 10; i++){
    if (i % 2 == 0){
        cout << i << "#";
        continue;
    }
}
if (i >= 10)
    cout << "END";
```

答案：错误

解析：如果删掉 `continue`，偶数时仍会输出 `0#2#4#6#8#`，奇数时不输出任何内容，但整个 `for` 循环还是会一直执行到 `i` 变成 $10$ 才结束。循环结束后 `i >= 10` 成立，还会继续输出 `END`。实际输出应包含末尾的 `END`，不是只有 `0#2#4#6#8#`，所以判断为错误。



## 第5题

将下面 C++ 代码中的 `i < 100; i = i + 1` 修改为 `i < 200; i += i + 1`，其输出与当前代码输出相同。（ ）

```cpp
int cnt = 0;
for (int i = 0; i < 100; i = i + 1)
    cnt += 1;
cout << cnt;
```

答案：错误

解析：原来的循环是 `i` 从 $0$ 到 $99$ 每次加 $1$，一共执行 $100$ 次，所以 `cnt` 输出 $100$。修改后更新语句是 `i += i + 1`，也就是 `i` 依次变为 $0,1,3,7,15,31,63,127,255$，在小于 $200$ 时只会执行有限的几次，次数不是 $100$。两次输出不同，判断为错误。



## 第6题

交换下面 C++ 代码中的 `i += 2` 和 `cnt += 1`，交换前后分别运行的两次输出相同。（ ）

```cpp
int i, cnt = 0;
cnt = 0;
while (i < 10){
    i += 2;
    cnt += 1;
}
cout << cnt;
```

答案：错误

解析：这段代码里 `int i, cnt = 0;` 只初始化了 `cnt`，没有给 `i` 初值，直接判断 `while (i < 10)` 本身就有风险，不能保证每次运行都从同一个 `i` 开始。即使暂时只看循环体顺序，先 `i += 2` 再 `cnt += 1` 与先 `cnt += 1` 再 `i += 2` 也不能在未初始化变量的程序中保证“交换前后输出相同”。题目说一定相同，判断为错误。



## 第7题

下面的 C++ 代码执行后将输出 45。（ ）

```cpp
int cnt;
for (int i = 0; i < 10; i++)
    cnt += 1;
cout << cnt;
```

答案：错误

解析：`int cnt;` 只是定义变量，没有把它初始化为 $0$。后面执行 `cnt += 1` 时，是在一个不确定的初始值上加 $1$，所以循环 $10$ 次后也不能保证得到某个固定结果。更不可能保证输出 $45$，因此判断为错误。



## 第8题

执行 C++ 代码 `cout << (12 + 12.12)` 将报错，因为 12 是 int 类型，而 12.12 是 float 类型，不同类型不能直接运算。（ ）

答案：错误

解析：C++ 允许整数和浮点数一起做普通算术运算。表达式 `12 + 12.12` 中，整数 `12` 会按通常算术转换参与浮点运算，结果是一个浮点值，不会因为类型不同就报错。题目说“不同类型不能直接运算”不对，判断为错误。



## 第9题

在 C++ 代码中，不可以将变量命名为 false，因为 false 是 C++ 语言的关键字。（ ）

答案：正确

解析：`false` 是 C++ 中表示布尔假值的关键字，和 `true` 一样有固定含义。关键字不能再拿来当变量名使用，否则会和语言本身的语法冲突。因此题目说不能将变量命名为 `false` 是正确的。



## 第10题

$X$ 是 C++ 的整型变量，则表达式 `3 < X < 5` 求值结果是 4。（ ）

答案：错误

解析：C++ 不会把 `3 < X < 5` 当作数学里的“$X$ 在 $3$ 和 $5$ 之间”。它会从左到右先算 `3 < X`，结果是布尔值 `true` 或 `false`，再把这个结果拿去和 $5$ 比较。`true` 可当作 $1$，`false` 可当作 $0$，所以第二步通常是 `1 < 5` 或 `0 < 5`，结果是布尔值，不是固定的 $4$。因此判断为错误。
