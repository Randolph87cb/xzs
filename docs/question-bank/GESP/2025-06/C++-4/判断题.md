## 第1题

下面 C++ 代码正确声明了一个返回 `int` 类型、接受两个 `int` 参数的函数。

```cpp
int add(int, int);
```

答案：正确

解析：`int add(int, int);` 是函数声明，也叫函数原型。最前面的 `int` 表示返回值类型是 `int`，括号里的两个 `int` 表示这个函数需要接收两个 `int` 类型参数。函数声明中可以只写参数类型，不写参数名，所以这句声明是正确的。



## 第2题

下面 C++ 代码的输出是 $15$。

```cpp
void foo(int x) {
    x += 5;
}

int main() {
    int a = 10;
    foo(a);
    cout << a << endl;
}
```

答案：错误

解析：`foo(int x)` 使用的是值传递，调用 `foo(a)` 时，`x` 得到的是 `a` 的一份副本。函数里执行 `x += 5;` 只会把副本从 `10` 改成 `15`，不会修改 `main` 中的原变量 `a`。回到 `main` 后输出的仍然是 `10`，不是 $15$。



## 第3题

下面 C++ 代码在一个结构体中又定义了别的结构体。这种结构体嵌套定义的方式语法不正确。

```cpp
#include <string>
#include <vector>
using namespace std;

struct Library {
    struct Book {
        struct Author {
            string name;
            int birthYear;
        };

        string title;
        int year;
        Author author;
    };

    string name;
    vector<Book> books;
};
```

答案：错误

解析：C++ 允许在结构体内部再定义结构体类型。代码中 `Library` 内部定义了 `Book`，`Book` 内部又定义了 `Author`，之后用 `Author author;` 和 `vector<Book> books;` 声明成员，语法上都是允许的。因此说“结构体嵌套定义语法不正确”是错误的。



## 第4题

在 C++ 中，相比于值传递，使用引用传递作的优点可以直接操作和修改原始变量，避免数据拷贝，提高效率。

答案：正确

解析：引用参数相当于给原变量起了一个别名，函数中通过这个别名操作的就是原来的变量，所以可以直接修改实参。对于较大的结构体或类对象，引用传递还避免了值传递时复制整个对象的开销，因此既能修改原数据，也可能提高效率。



## 第5题

下面这段代码不合法，因为每一行都必须显式初始化 $3$ 个元素。

```cpp
int arr[2][3] = {{1, 2}, {3}};
```

答案：错误

解析：二维数组初始化时，每一行不一定要把所有元素都显式写出来。`{{1, 2}, {3}}` 中第一行给了 `1, 2`，剩下的位置自动补 `0`；第二行给了 `3`，后两个位置也自动补 `0`。因此数组实际可看作 `{{1, 2, 0}, {3, 0, 0}}`，代码是合法的。



## 第6题

以下程序中使用了递推方式计算阶乘（$n! = 1 \times 2 \times \ldots \times n$），计算结果正确。

```cpp
int factorial(int n) {
    int res = 1;
    for (int i = 0; i < n; ++i) {
        res *= i;
    }
    return res;
}
```

答案：错误

解析：阶乘 $n!$ 应从 $1$ 乘到 $n$。这段循环从 `i = 0` 开始，第一次循环就执行 `res *= i`，使 `res` 变成 `0`。之后无论再乘什么，结果都还是 `0`。例如 `factorial(5)` 会返回 `0`，不是 $120$，所以计算结果不正确。



## 第7题

无论初始数组是否有序，选择排序都执行 $O(n^2)$ 次比较。

答案：正确

解析：选择排序每一轮都会在未排序区间中寻找最小元素。第 $1$ 轮比较约 $n-1$ 次，第 $2$ 轮比较约 $n-2$ 次，直到最后一轮，总比较次数是 $(n-1)+(n-2)+\cdots+1$，数量级为 $O(n^2)$。这个过程不依赖初始数组是否已经有序，因此判断为正确。



## 第8题

以下 C++ 代码，尝试对有 $n$ 个整数的数组 `arr` 进行排序。这个代码实现了选择排序算法。

```cpp
for (int i = 0; i < n - 1; ++i) {
    int minIndex = i;
    for (int j = i + 1; j < n; ++j) {
        if (arr[j] < arr[minIndex])
            minIndex = j;
    }
    if (minIndex != i)
        swap(arr[i], arr[minIndex]);
}
```

答案：正确

解析：这段代码的外层循环固定当前位置 `i`，先假设 `minIndex = i`。内层循环从 `i + 1` 到末尾寻找更小元素的位置，找到后更新 `minIndex`。一轮结束后，把当前位置 `arr[i]` 与未排序区间的最小元素交换。这正是选择排序“每轮选出最小值放到前面”的过程。



## 第9题

如果一个异常在 `try` 块中抛出但没有任何 `catch` 匹配，它将在编译时报错。

答案：错误

解析：异常是否有匹配的 `catch`，通常不是编译阶段能直接判定的问题。程序运行时，如果 `try` 中抛出的异常没有被当前的 `catch` 捕获，它会继续向外层调用者传播；如果最终仍没有匹配的处理程序，程序会异常终止。因此“将在编译时报错”的说法不对。



## 第10题

下面 C++ 代码实现将 `Hello` 写入 `data.txt`。

```cpp
ofstream out("data.txt");
out << "Hello";
out.close();
```

答案：正确

解析：`ofstream out("data.txt");` 创建输出文件流，用来向 `data.txt` 写入内容。语句 `out << "Hello";` 把字符串 `Hello` 写入这个文件，最后 `out.close();` 关闭文件流。因此这段代码实现了向 `data.txt` 写入 `Hello`。
