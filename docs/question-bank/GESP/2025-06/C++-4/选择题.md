## 第1题

在 C++ 中，声明一个指向整型变量的指针的正确语法是（ ）。

A. `int* ptr;`
B. `*int ptr;`
C. `int ptr*;`
D. `ptr int;`

答案：A

解析：声明指针时，要先写它所指向的数据类型，再用 `*` 表示这是一个指针变量。`int* ptr;` 的含义是：`ptr` 是一个指针，它保存的是 `int` 变量的地址。B、C、D 都不符合 C++ 声明语法，不能声明整型指针。



## 第2题

下面的函数接收一个 3 行 4 列的二维数组并输出其中元素，则横线上不能填写（ ）。

```cpp
void printArray(________) {
    for (int i = 0; i < 3; ++i)
        for (int j = 0; j < 4; ++j)
            std::cout << arr[i][j] << " ";
}
```

A. `int arr[3][4]`
B. `int arr[][4]`
C. `int (*arr)[4]`
D. `int** arr`

答案：D

解析：二维数组作为函数参数时，第一维可以省略，但第二维必须让编译器知道，否则无法根据 `arr[i][j]` 计算元素位置。A、B 都表示每行有 $4$ 个 `int`，C 中 `int (*arr)[4]` 是“指向含 $4$ 个 `int` 的数组的指针”，也能接收连续的二维数组。`int** arr` 是“指向 `int*` 的指针”，常用于指针数组或动态二级指针结构，不能直接代替 `int arr[3][4]` 这种连续存放的二维数组形参。



## 第3题

在 C++ 中，`int arr[3][4]` 和 `int* arr = new int[12]` 均可模拟一个 3 行 4 列的二维数组。关于这两种方式，下面说法错误的是（ ）。

A. `int arr[3][4]` 在栈上分配空间，适合数组较小的情况；
B. `int* arr = new int[12]` 在堆上分配空间，数组较大时也适用；
C. 这两种方式申请的内存空间都是连续的。
D. 这两种方式申请的内存都能自动释放。

答案：D

解析：`int arr[3][4]` 是局部数组时通常分配在栈上，离开它所在的作用域后会自动释放；`new int[12]` 申请的是堆空间，程序员需要在不用时写 `delete[] arr;` 释放。两者都可以连续存放 $12$ 个 `int`，但堆数组不会自动释放，所以“这两种方式申请的内存都能自动释放”是错误说法。



## 第4题

关于以下 C++ 代码，说法正确的是（ ）。

```cpp
int main() {
    greet();
    return 0;
}

void greet() {
    cout << "Hello!" << endl;
}
```

A. 正确编译并输出 Hello!
B. 编译错误：找不到函数 greet()
C. 编译警告但可以运行
D. 链接错误

答案：B

解析：C++ 编译到 `main` 中的 `greet();` 时，前面还没有出现 `greet` 的声明或定义，编译器不知道这是一个什么函数，因此会报“找不到函数”一类的编译错误。若要这样调用，应在 `main` 前添加函数声明 `void greet();`，或者把 `greet` 的完整定义放到 `main` 前面。



## 第5题

在 C++ 中，如果希望通过函数修改传入的结构体对象的内容，应该使用哪种参数传递方式？

A. 值传递或引用传递
B. 值传递或指针传递
C. 引用传递或指针传递
D. 仅指针传递

答案：C

解析：值传递会把结构体对象复制一份，函数里改的是副本，原对象不变。引用传递如 `void f(Student& s)`，参数 `s` 就是原对象的别名；指针传递如 `void f(Student* s)`，可以通过 `s->成员` 修改原对象。题目要求“通过函数修改传入对象的内容”，因此应使用引用传递或指针传递。



## 第6题

以下哪个选项正确描述了 C++ 中形参和实参的区别？

A. 形参是函数调用时传递给函数的具体值，实参是函数定义中声明的变量。
B. 形参是函数定义中声明的变量，实参是函数调用时传递给函数的具体值。
C. 形参和实参在函数调用时是完全相同的。
D. 形参只在函数内部可见，实参在函数外部可见。

答案：B

解析：形参出现在函数定义或声明中，用来说明函数接收什么数据，例如 `int add(int x, int y)` 中的 `x`、`y`。实参出现在函数调用时，是实际传给函数的数据，例如 `add(a, 3)` 中的 `a` 和 `3`。所以 B 对形参和实参的位置、作用描述最准确。



## 第7题

运行如下代码会输出（ ）。

```cpp
int value = 100;

void print1() {
    int value = 50;
    cout << value << " ";
    cout << ::value << " ";
}

void print2() {
    cout << value << " ";
}

print1();
print2();
```

A. 100 100 100
B. 50 50 50
C. 50 100 100
D. 50 50 100

答案：C

解析：全局变量 `value` 的值是 `100`。在 `print1` 中又定义了局部变量 `int value = 50;`，它会遮蔽同名全局变量，所以第一个 `cout << value` 输出 `50`；`::value` 使用全局作用域运算符，明确访问全局变量，输出 `100`。`print2` 中没有局部同名变量，直接访问全局 `value`，也输出 `100`。最终结果是 `50 100 100`。



## 第8题

小杨在整理一副扑克牌的所有红心扑克牌，使其从小到大排列。他的做法是：最开始抓到第 1 张扑克牌被认为已经排好序；然后抓第 2 张扑克牌，将其插入至有序部分的正确位置；不断循环步骤，每次将新抓到扑克牌插入至有序部分，直至抓完所有扑克牌，这样抓牌结束时就完成了扑克牌的排序。小杨这种整理扑克牌的方式与（ ）排序的方式最接近。

A. 冒泡排序
B. 插入排序
C. 选择排序
D. 直接排序

答案：B

解析：插入排序的核心过程是：前面一段先看作已经有序，每次取出一个新元素，把它插入到有序部分的正确位置。题目中“第 $1$ 张牌已排好序”“每次将新抓到的牌插入至有序部分”正是插入排序的典型描述，因此选 B。



## 第9题

以下哪种情况是使用插入排序的合适场景？

A. 数据量非常大，且乱序严重
B. 希望获得稳定排序，但不要求实时性
C. 数据几乎有序，只需少量调整
D. 想在交换次数最少的前提下排好大数组

答案：C

解析：插入排序在处理“几乎有序”的数据时很合适，因为每个元素通常只需要向前移动很少几步，实际操作次数会明显减少。若数据量非常大且乱序严重，插入排序的最坏时间复杂度是 $O(n^2)$，效率不理想；“稳定排序”不是插入排序唯一适用理由；大数组中想尽量少交换也不是选择插入排序的主要依据。



## 第10题

以下关于递推算法基本思想的描述，正确的是（ ）。

A. 递推算法通过将问题分解为相互独立的子问题来解决。
B. 递推算法从已知的基础情况出发，通过某种关系逐步推导出更大规模问题的解。
C. 递推算法通常用于穷举所有可能的解决方案。
D. 递推算法适用于在每一步做出局部最优选择以达到全局最优。

答案：B

解析：递推通常先给出最小规模的已知结果，再按照递推关系一步步算出后面的结果。例如已知 `f[1]`、`f[2]`，再用 `f[i] = f[i - 1] + f[i - 2]` 算更大的 `i`。A 更接近分治思想，C 更接近穷举搜索，D 描述的是贪心算法。



## 第11题

给定如下算法，其时间复杂度为（ ）。

```cpp
bool f(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        int sum = 0;
        for (int j = 0; j < n; j++) {
            if (i & (1 << j)) {
                sum += arr[j];
            }
        }
        if (sum == target) return true;
    }
    return false;
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(n^3)$
D. $O(2^n)$

答案：B

解析：复杂度要按给出的代码统计。外层循环是 `for (int i = 0; i < n; i++)`，执行 $n$ 次，不是枚举到 $2^n$；每次外层循环中，内层 `j` 从 $0$ 到 $n-1$，也执行 $n$ 次。`if` 和加法都是常数时间，因此总次数约为 $n \times n$，时间复杂度是 $O(n^2)$。



## 第12题

下述斐波那契数列计算的时间复杂度是（ ）。

```cpp
int fibonacci(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(n^3)$
D. $O(2^n)$

答案：D

解析：这个函数没有保存已经算过的结果。计算 `fibonacci(n)` 时会继续计算 `fibonacci(n - 1)` 和 `fibonacci(n - 2)`，而这两个分支内部又会继续分裂出更多递归调用，许多值会被重复计算。递归调用树的规模随 $n$ 近似成倍增长，所以时间复杂度通常记为 $O(2^n)$。



## 第13题

关于下面 C++ 程序的描述，（ ）最准确。

```cpp
ifstream in("data.txt");
string line;
while (getline(in, line)) {
    cout << line << endl;
}
```

A. 将从标准输入读取每行，并输出到屏幕
B. 程序无法运行，因为 getline 只能读取 cin
C. 将 data.txt 中的每一行读取并输出到屏幕
D. 程序将创建 data.txt 并写入默认文本

答案：C

解析：`ifstream in("data.txt");` 创建的是输入文件流，用来从文件 `data.txt` 读取数据。`getline(in, line)` 每次从文件流 `in` 中读取一整行到字符串 `line`，读到文件结束时循环停止。循环体中的 `cout << line << endl;` 会把每一行输出到屏幕。



## 第14题

在 C++ 中，异常处理机制（try-catch 块）的主要目的是（ ）。

A. 提高程序的运行速度。
B. 在程序发生运行时错误时，提供一种结构化的错误处理方式。
C. 确保程序在编译时没有错误。
D. 减少程序的内存占用。

答案：B

解析：`try-catch` 用来处理程序运行过程中可能出现的异常情况。可能出错的代码放在 `try` 块中，异常被抛出后，由匹配的 `catch` 块集中处理，这比到处写零散的错误处理语句更有结构。它不能保证编译一定通过，也不是为了提高速度或减少内存。



## 第15题

为了提高冒泡排序的效率，如果某轮“冒泡”中没有执行任何交换操作，说明数组已经完成排序，可直接返回结果，则两条横线上分别应该填写（ ）。

```cpp
void bubbleSortWithFlag(vector<int> &nums) {
    for (int i = nums.size() - 1; i > 0; i--) {
        bool flag;
        // 在此处填入代码

        for (int j = 0; j < i; j++) {
            if (nums[j] > nums[j + 1]) {
                swap(nums[j], nums[j + 1]);
                // 在此处填入代码
            }
        }
        if (!flag)
            break;
    }
}
```

A.
```cpp
flag = false;
flag = false;
```

B.
```cpp
flag = false;
flag = true;
```

C.
```cpp
flag = true;
flag = false;
```

D.
```cpp
flag = true;
flag = true;
```

答案：B

解析：`flag` 应表示“本轮是否发生过交换”。每一轮开始前还没有交换，所以先令 `flag = false;`。如果在内层循环中发现相邻元素逆序并执行了 `swap`，就把 `flag = true;`，表示本轮数组仍发生了调整。内层循环结束后若 `!flag`，说明这一轮一次交换都没有，数组已经有序，可以提前退出。因此两处应分别填 `false` 和 `true`。
