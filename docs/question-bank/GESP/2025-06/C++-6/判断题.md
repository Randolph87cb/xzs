## 第1题

构造函数可以被声明为 virtual。

答案：错误

解析：C++ 构造函数不能声明为 `virtual`。虚函数调用依赖对象的动态类型，而构造过程中对象还在按“先基类、后派生类”的顺序逐步建立，不能用虚机制来选择构造函数；构造函数也不被继承，必须由当前类或初始化列表明确调用。容易混淆的是析构函数可以声明为虚函数，而且通过基类指针删除派生类对象时通常应当这样做。



## 第2题

给定一组字符及其出现的频率，构造出的哈夫曼树是唯一的。

答案：错误

解析：哈夫曼树只要求每次合并当前权值最小的两个结点以取得最小带权路径长度，但在合并顺序出现并列权值时可能有不同选择；即使结构相同，左右子树交换也会得到不同的 0/1 编码。因此“构造出的哈夫曼树是唯一的”说得过强，最优带权路径长度可以唯一，具体树形或编码不一定唯一。



## 第3题

为了实现一个队列，使其出队操作（pop）的时间复杂度为 $O(1)$ 并且避免数组删除首元素的 $O(n)$ 问题，一种常见且有效的方法是使用环形数组，通过调整队首和队尾指针来实现。

答案：正确

解析：顺序数组如果每次出队都删除首元素，后面的元素需要整体前移，代价是 `$O(n)$`。环形数组用 `front` 记录队首、`rear` 记录队尾，出队时只把 `front` 按容量取模向后移动一格，不搬移元素，所以 `pop` 可以做到 `$O(1)$`。这是数组实现队列时避免首元素删除开销的常见方法。



## 第4题

对一棵二叉排序树进行中序遍历，可以得到一个递增的有序序列。

答案：正确

解析：二叉排序树满足左子树所有结点值小于根、右子树所有结点值大于根，并且左右子树本身也满足同样规则。中序遍历的顺序是“左子树、根、右子树”，因此会先输出较小值，再输出根，最后输出较大值，整体得到递增序列。若允许重复值，通常会得到非递减序列；本题表述的核心性质是正确的。



## 第5题

如果二叉搜索树在连续的插入和删除操作后，所有节点都偏向一侧，导致其退化为类似于链表的结构，这时其查找、插入、删除操作的时间复杂度会从理想情况下的 $O(\log n)$ 退化到 $O(n \log n)$。

答案：错误

解析：普通二叉搜索树查找、插入、删除的时间复杂度与树高 `$h$` 成正比。平衡时 `$h = O(\log n)$`，所以理想复杂度是 `$O(\log n)$`；如果结点全部偏向一侧，树高会退化为 `$n$`，操作复杂度退化为 `$O(n)$`。题干写成 `$O(n \log n)$` 多乘了一个对数因子。



## 第6题

执行下列代码， my_dog.name 的最终值是 Charlie 。

```cpp
class Dog {
public:
    std::string name;
    Dog(std::string str) : name(str) {}
};

int main() {
    Dog my_dog("Buddy");
    my_dog.name = "Charlie";
    return 0;
}
```

答案：正确

解析：`Dog my_dog("Buddy");` 调用构造函数，把成员 `name` 初始化为 `"Buddy"`。随后 `my_dog.name = "Charlie";` 直接给这个公开成员重新赋值，旧值会被新字符串替换。函数结束前没有其他语句再修改 `name`，所以最终值确实是 `Charlie`。



## 第7题

下列 C++ 代码可以成功编译，并且子类 Child 的实例能通过其成员函数访问父类 Parent 的属性 value。

```cpp
class Parent {
private:
    int value = 100;
};
class Child : public Parent {
public:
    int get_private_val() {
        return value; // 尝试访问父类的私有成员
    }
};
```

答案：错误

解析：`Parent` 中的 `value` 被声明在 `private` 区域，只有 `Parent` 自己的成员函数和友元可以直接访问。`Child` 虽然以 `public` 方式继承了 `Parent`，但继承方式不会把基类私有成员变成派生类可直接访问的成员，所以 `return value;` 会因为访问权限不足而编译失败。若希望派生类访问，应改为 `protected` 或提供公开接口。



## 第8题

下列代码中的 tree 向量，表示的是一棵完全二叉树（-1 代表空节点）按照层序遍历的结果。

```cpp
#include <vector>
std::vector<int> tree = {1, 2, 3, 4, -1, 6, 7};
```

答案：错误

解析：按层序数组理解，`-1` 出现在下标 `4`，表示结点 `2` 的右孩子为空；但后面下标 `5`、`6` 仍有结点 `6`、`7`。完全二叉树要求最后一层从左到右连续填充，出现空位后，其右侧不能再有实际结点。这个数组在中间留下空缺后又继续出现结点，所以不能表示完全二叉树。



## 第9题

在树的深度优先搜索（DFS）中，使用栈作为辅助数据结构以实现“先进后出”的访问顺序。

答案：正确

解析：深度优先搜索会沿着一条路径尽量向深处访问，回退后再处理其他分支。显式实现时常用栈保存待访问结点，后压入的结点先被弹出，正好体现“后进先出”，也就是题干所说的“先进后出”。递归 DFS 本质上也依赖函数调用栈完成同样的回退过程。



## 第10题

下面代码采用动态规划求解零钱兑换问题：给定 $n$ 种硬币，第 $i$ 种硬币的面值为 $coins[i - 1]$，目标金额为 $amt$，每种硬币可以重复选取，求能够凑出目标金额的最少硬币数量；如果不能凑出目标金额，返回 -1。

```cpp
int coinChangeDPComp(vector<int> &coins, int amt) {
    int n = coins.size();
    int MAX = amt + 1;
    vector<int> dp(amt + 1, MAX);
    dp[0] = 0;

    for (int i = 1; i <= n; i++) {
        for (int a = 1; a <= amt; a++) {
            if (coins[i - 1] > a)
                dp[a] = dp[a];
            else
                dp[a] = min(dp[a], dp[a - coins[i - 1]] + 1);
        }
    }
    return dp[amt] != MAX ? dp[amt] : -1;
}
```

答案：正确

解析：`dp[a]` 表示凑出金额 `a` 所需的最少硬币数，`dp[0] = 0` 是起点，`MAX = amt + 1` 作为不可达标记。外层枚举硬币，内层金额从小到大枚举，转移 `dp[a] = min(dp[a], dp[a - coins[i - 1]] + 1)` 允许当前硬币在同一轮被重复使用，符合“每种硬币可以重复选取”。最后若 `dp[amt]` 仍为 `MAX` 返回 `-1`，否则返回最少数量。
