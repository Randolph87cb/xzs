## 第1题

下列哪一项不是面向对象编程的基本特征？

A. 继承
B. 封装
C. 多态
D. 链接

答案：D

解析：面向对象编程通常强调封装、继承和多态：封装把数据和操作组织在类中，继承复用和扩展已有类，多态让同一接口在不同对象上表现出不同行为。`链接` 是程序构建阶段把目标文件和库组合成可执行程序的过程，不是面向对象的基本特征，因此应选 D。



## 第2题

为了让 `Dog` 类的构造函数能正确地调用其父类 `Animal` 的构造方法，横线线处应填入（ ）。

```cpp
class Animal {
public:
    std::string name;

    Animal(std::string str) : name(str) {
        std::cout << "Animal created\n";
    }

    virtual void speak() {
        std::cout << "Animal speaks" << endl;
    }
};

class Dog : public Animal {
    std::string breed;
public:
    Dog(std::string name, std::string b) : ______________, breed(b) {
        std::cout << "Dog created\n";
    }
    void speak() override {
        std::cout << "Dog barks" << endl;
    }
};

int main() {
    Animal* p = new Dog("Rex", "Labrador");
    p->speak();
    delete p;
    return 0;
}
```

A. `Animal(name)`
B. `super(name)`
C. `Animal::Animal(name)`
D. `Animal()`

答案：A

解析：派生类构造函数要在初始化列表中直接写基类名并传入实参，所以应写 `Dog(...) : Animal(name), breed(b)`。`super(name)` 不是 C++ 语法，`Animal::Animal(name)` 不能这样调用构造函数，`Animal()` 又要求基类存在无参构造函数而题中没有提供，因此应选 A。



## 第3题

代码同上一题，代码执行结果是（ ）。

A. 输出 Animal speaks
B. 输出 Dog barks
C. 编译错误
D. 程序崩溃

答案：B

解析：`p` 的静态类型是 `Animal*`，实际指向的是 `Dog` 对象；由于 `speak()` 在基类中被声明为 `virtual`，调用 `p->speak()` 会发生动态绑定，执行 `Dog::speak()`，输出 `Dog barks`。如果没有 `virtual` 才会按静态类型调用基类版本；这里不会因为基类指针指向派生类对象而编译错误，因此应选 B。



## 第4题

以下关于栈和队列的代码，执行后输出是（ ）。

```cpp
stack<int> s;
queue<int> q;

for (int i = 1; i <= 3; ++i) {
    s.push(i);
    q.push(i);
}
cout << s.top() << " " << q.front() << endl;
```

A. 1 3
B. 3 1
C. 3 3
D. 1 1

答案：B

解析：循环依次把 `1, 2, 3` 压入栈和队列。栈是后进先出，`top()` 取得最后压入的 `3`；队列是先进先出，`front()` 取得最早入队的 `1`。输出中间还有一个空格，所以结果为 `3 1`，应选 B。



## 第5题

在一个循环队列中，`front` 是指向队头的指针，`rear` 指向队尾的指针，队列最大容量为 `maxSize`。判断队列已满的条件是（ ）。

A. `rear == front`
B. `(rear + 1) % maxSize == front`
C. `(rear - 1 + maxSize) % maxSize == front`
D. `(rear - 1) == front`

答案：B

解析：常见循环队列会空出一个位置来区分“空”和“满”。当 `rear` 再向后移动一格就会追上 `front` 时，说明已经不能继续入队，即 `(rear + 1) % maxSize == front`。`rear == front` 通常表示空队列；减一相关的条件是在看 `rear` 的前一个位置，不能判断下一次入队是否会覆盖队头，因此应选 B。



## 第6题

（ ）只有最底层的节点未被填满，且最底层节点尽量靠左填充。

A. 完美二叉树
B. 完全二叉树
C. 完满二叉树
D. 平衡二叉树

答案：B

解析：完全二叉树要求除最后一层外其余层都填满，最后一层的结点从左到右连续排列，允许右侧缺少结点。完美二叉树要求所有层都满；完满二叉树强调每个结点要么没有孩子、要么有两个孩子；平衡二叉树关注左右子树高度差。题干描述的是最后一层可不满但靠左填充，因此应选 B。



## 第7题

在使用数组表示完全二叉树时，如果一个节点的索引为 $ i $（从 0 开始计数），那么其左子节点的索引通常是（ ）。

A. $ (i - 1) / 2 $
B. $ i + 1 $
C. $ i \times 2 $
D. $ 2 \times i + 1 $

答案：D

解析：完全二叉树按层序存入数组且下标从 `0` 开始时，结点 `i` 的左孩子下标是 `$2 \times i + 1$`，右孩子下标是 `$2 \times i + 2$`。`$(i - 1) / 2$` 是父结点下标，`i + 1` 只是数组中的下一个位置，`$2 \times i$` 是一基下标写法下的左孩子形式，不适用于本题，因此应选 D。



## 第8题

已知一棵二叉树的前序遍历序列为 `GDAFEMHZ`，中序遍历序列为 `ADFGEHMZ`，则其后序遍历序列为（ ）。

A. ADFGEHMZ
B. ADFGHMEZ
C. AFDGEMZH
D. AFDHZMEG

答案：D

解析：前序序列第一个字符 `G` 是根；在中序 `ADFGEHMZ` 中，`G` 左侧是左子树 `ADF`，右侧是右子树 `EHMZ`。左子树前序为 `DAF`，后序为 `AFD`；右子树根为 `E`，其右子树由 `M` 带左右孩子 `H`、`Z`，后序为 `HZME`。最后再访问根 `G`，得到 `AFDHZMEG`，应选 D。



## 第9题

第9题 设有字符集 $\{a, b, c, d, e\}$，其出现频率分别为 $\{5, 8, 12, 15, 20\}$，得到的哈夫曼编码为（ ）。

A.

```plaintext
a: 010
b: 011
c: 00
d: 10
e: 11
```

B.

```plaintext
a: 00
b: 10
c: 011
d: 100
e: 111
```

C.

```plaintext
a: 10
b: 01
c: 011
d: 100
e: 111
```

D.

```plaintext
a: 100
b: 01
c: 011
d: 100
e: 00
```

答案：A

解析：按哈夫曼算法先合并最小的 `5` 和 `8` 得到 `13`，再与 `12` 合并得到 `25`，同时 `15` 与 `20` 合并得到 `35`，最后合并 `25` 和 `35`。因此频率较小的 `a`、`b` 编码长度为 3，`c`、`d`、`e` 编码长度为 2，且所有编码不能互为前缀。A 正好满足这个结构；B、C 中存在短编码成为长编码前缀的问题，D 还把 `a` 和 `d` 设成同一编码，均不是有效哈夫曼编码。



## 第10题

3 位格雷编码中，编码 $ 101 $ 之后的下一个编码不可能是（ ）。

A. 100
B. 111
C. 110
D. 001

答案：C

解析：格雷编码相邻两个编码只允许有一位不同。从 `101` 出发，`100` 只变最后一位，`111` 只变中间一位，`001` 只变最高位，都可能成为相邻编码；`110` 与 `101` 的中间位和最后一位都不同，变化了两位，不可能紧接在 `101` 后面，因此应选 C。

```cpp
    TreeNode* right;

void dfs(TreeNode* root, vector<int>& result) {
    if (root == nullptr) return;
```

A.

```cpp
result.push_back(root->val);
dfs(root->left);
dfs(root->right);
```

B.

```cpp
result.push_back(root->left->val);
dfs(root->right);
dfs(root->left);
```

C.

```cpp
result.push_back(root->left->val);
```

```cpp
result.push_back(root->right->val);
```

```cpp
if (key < root->val)
```

## 第14题

以下关于动态规划算法特性的描述，正确的是（ ）。

A. 子问题相互独立，不重叠
B. 问题包含重叠子问题和最优子结构
C. 只能从底至顶迭代求解
D. 必须使用递归实现，不能使用迭代

答案：B

解析：动态规划适用的典型前提是问题能拆成重叠子问题，并且整体最优解可以由子问题最优解推出，也就是最优子结构。子问题相互独立更像普通分治；动态规划既可以自底向上迭代，也可以自顶向下递归加记忆化；实现方式并不限定为递归或迭代中的某一种。因此应选 B。



## 第15题

给定 $ n $ 个物品和一个最大承重为 $ W $ 的背包，每个物品有一个重量 $ wt[i] $ 和价值 $ val[i] $，每个物品只能选择放或不放。目标是选择若干个物品放入背包，使得总价值最大，且总重量不超过 $ W $。关于下面代码，说法正确的是（ ）。

```cpp
int knapsack1D(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<int> dp(W+1, 0);
    for (int i = 0; i < n; ++i) {
        for (int w = W; w >= wt[i]; --w) {
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}
```

A. 该算法不能处理背包容量为 $ 0 $ 的情况
B. 外层循环 `i` 遍历背包容量，内层遍历物品
C. 从大到小遍历 `w` 是为了避免重复使用同一物品
D. 这段代码计算的是最小重量而非最大价值

答案：C

解析：`dp[w]` 表示当前已处理物品在容量不超过 `w` 时能取得的最大价值。每个物品只能选一次，所以处理第 `i` 个物品时，容量 `w` 必须从大到小更新，保证 `dp[w - wt[i]]` 仍然来自处理该物品之前的状态；如果从小到大更新，同一物品可能在本轮被重复使用。`W = 0` 时 `dp[0]` 正常为 0，外层循环遍历的是物品，代码求的是最大价值，因此应选 C。
