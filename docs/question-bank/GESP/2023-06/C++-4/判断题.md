## 第1题

域名是由一串用点分隔的名字来标识互联网上一个计算机或计算机组的名称，CCF 编程能力等级认证官方网站的域名是 gesp.ccf.org.cn，其中顶级域名是 gesp。

答案：错误

解析：本题判断为错误。域名的层级通常从右向左看，最右侧的 `cn` 才是顶级域名，`org`、`ccf`、`gesp` 是更靠左的下级名称。因此 `gesp.ccf.org.cn` 的顶级域名不是 `gesp`。

## 第2题

数列 1, 1, 2, 3, 5, 8 … 是以意大利数学家列昂纳多·斐波那契命名的数列，从第三个数开始，每个数是前面两项之和。如果计算该数列的第 $n$ 项（其中 $n>3$）$fib(n)$，我们采用如下方法：① 令 $fib(1)=fib(2)=1$ ②用循环 `for i=3 to n` 分别计算 $f(i)$ ③输出 $fib(n)$。这体现了递推的编程思想。

答案：正确

解析：本题判断为正确。题目先给出 $fib(1)$ 和 $fib(2)$，再从 $i=3$ 到 $n$ 依次用前两项推出后一项，这正是“已知初始值，按递推关系逐步求后续项”的递推思想。

## 第3题

在 C++ 语言中，函数的参数默认以引用传递方式进行传递。

答案：错误

解析：本题判断为错误。C++ 函数参数默认是值传递，调用函数时会把实参的值复制给形参。只有在形参类型中显式写出引用符号，例如 `int &x`，才是引用传递。

## 第4题

在 C++ 语言中，可以定义四维数组，但在解决实际问题时不可能用到，因为世界是三维的。

答案：错误

解析：本题判断为错误。C++ 数组可以有多维，例如 `int a[2][3][4][5];` 就是四维数组。数组维数只是程序组织数据的方式，和现实世界是否三维没有必然关系，实际问题中也可能用四维数组表示多重状态。

## 第5题

在 C++ 语言中，一个函数没有被调用时，它的参数不占用内存。

答案：正确

解析：本题判断为正确。函数的参数是在函数被调用时才为本次调用准备的，例如局部形参通常存在于调用过程对应的运行时存储中。如果函数从未被调用，就不会产生这次调用的形参对象，也就不会为这些参数占用运行时内存。

## 第6题

在 C++ 语言中，如果一个函数可能抛出异常，那么一定要在 try 子句里调用这个函数。

答案：错误

解析：本题判断为错误。`try` 语句用于捕获和处理异常，但不是所有可能抛出异常的函数调用都必须直接写在 `try` 中。异常可以向外层调用者继续传递；如果一直没有被捕获，程序才会异常终止。

## 第7题

如果希望记录 10 个最长为 99 字节的字符串，可以将字符串数组定义为 `char s[100][10];`。

答案：错误

解析：本题判断为错误。`char s[100][10]` 表示 100 个字符串位置，每个位置最多存 9 个普通字符再加结尾的 `'\0'`。如果要记录 10 个最长为 99 字节的字符串，通常应为每个字符串预留 100 个 `char`，形如 `char s[10][100];`。

## 第8题

字符常量 '0' 和 '\0' 是等价的。

答案：错误

解析：本题判断为错误。`'0'` 是字符数字 0，对应的是可显示字符；在常见编码中它的编码值不是 0。`'\0'` 是空字符，编码值为 0，常用作 C 风格字符串的结束标记，二者含义和数值都不同。

## 第9题

`>=` 和 `>>=` 都是 C++ 语言的运算符。

答案：正确

解析：本题判断为正确。`>=` 是“大于等于”关系运算符，例如 `a >= b`；`>>=` 是右移赋值运算符，例如 `x >>= 1` 等价于把 `x` 右移后再赋回 `x`。它们都属于 C++ 运算符。

## 第10题

由于文件重定向操作，程序员在使用 C++ 语言编写程序时无法确定通过 `cout` 输出的内容是否会被输出到屏幕上。

答案：正确

解析：本题判断为正确。`cout` 写入的是标准输出流，标准输出可以在运行程序时被重定向到文件或其他设备。由于这种重定向可能发生在程序外部，仅从程序中的 `cout` 语句不能保证内容一定显示在屏幕上。
