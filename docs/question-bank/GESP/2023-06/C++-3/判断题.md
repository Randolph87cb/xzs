## 第1题

一个算法可以用不同的形式来描述，但要求描述比较规范，因此不能用自然语言描述。

【知识点：算法描述与流程图】

答案：错误

解析：本题判断为错误。算法可以用自然语言、流程图、伪代码、程序设计语言等多种形式描述。只要步骤清楚、顺序明确、没有歧义，自然语言也可以规范地描述算法；题干把“规范”理解成“不能用自然语言”，说法过于绝对。

## 第2题

域名是由一串用点分隔的名字来标识互联网上一个计算机或计算机组的名称，CCF 编程能力等级认证官方网站的域名是 gesp.ccf.org.cn，其中顶级域名是 gesp。

【知识点：计算机基础常识】

答案：错误

解析：本题判断为错误。域名按层级通常从右向左看，`gesp.ccf.org.cn` 中最右侧的 `cn` 是顶级域名，`org`、`ccf`、`gesp` 依次是更左侧的下级名称。因此 `gesp` 不是顶级域名。

## 第3题

数据编码方式只有原码、反码、补码三种。

【知识点：原码反码补码与整数表示】

答案：错误

解析：本题判断为错误。原码、反码、补码主要用于讨论有符号整数在计算机中的表示方式，不能代表所有“数据编码方式”。字符编码中的 ASCII、Unicode，十进制数的 BCD 编码等也都是编码方式，所以“只有三种”的说法不成立。

## 第4题

在 C++语言中，长度为 $n$ 的数组，合理的下标范围是从 $0$ 到 $n$，包括 $0$ 和 $n$。

【知识点：数组与字符数组】

答案：错误

解析：本题判断为错误。C++ 数组下标从 `0` 开始，长度为 $n$ 的数组共有 $n$ 个元素，合法下标是 `0` 到 `n - 1`。下标 `n` 已经越过最后一个元素，不能作为合理下标使用。

## 第5题

字符常量 '\0' 常用来表示字符串结束，它和字符常量 '0' 是不同的。

【知识点：ASCII/字符编码】

答案：正确

解析：本题判断为正确。`'\0'` 是空字符，字符编码值为 0，常作为 C 风格字符串的结束标记；`'0'` 是数字字符 0，在 ASCII 中编码值为 48。二者写法相近，但含义和编码值都不同。

## 第6题

在 C++ 语言中，可以使用字符（如 '0'）作为数组下标。

【知识点：数组与字符数组】

答案：正确

解析：本题判断为正确。C++ 中字符类型属于整型类别，作为数组下标时会转换成对应的整数编码值。例如 `'0'` 在 ASCII 中通常是 48，因此 `a['0']` 实际相当于访问 `a[48]`；只要数组下标没有越界，语法上是允许的。

## 第7题

在 C++ 语言中，数组被定义时，它的大小就确定了。

【知识点：数组与字符数组】

答案：正确

解析：本题判断为正确。普通 C++ 数组在定义时就要确定元素个数，例如 `int a[10];` 定义后数组长度就是 10，不能再把同一个数组改成长度 20。后续可以修改数组元素的值，但不能改变这个数组本身的大小。

## 第8题

计算机中存储的数据都是二进制形式。因此，在使用 C++ 语言编写程序时，将所有十进制数改写为相同数值的二进制数，会使得程序运行效率更高。

【知识点：进制转换与进制字面量】

答案：错误

解析：本题判断为错误。十进制、二进制、十六进制只是程序中书写整数常量的不同形式，相同数值经过编译后表示的是同一个值。把 `10` 改写成 `0b1010` 不会让相同逻辑的程序天然运行得更快，程序效率主要取决于算法和实际执行的操作。

## 第9题

在 C++语言中，表达式 `(0xf == 015)` 的值为 true。

【知识点：进制转换与进制字面量】

答案：错误

解析：本题判断为错误。`0xf` 是十六进制整数，等于十进制的 15；`015` 以 `0` 开头，是八进制整数，等于十进制的 $1 \times 8 + 5 = 13$。15 不等于 13，所以表达式 `(0xf == 015)` 的值为 `false`。

## 第10题

如果 a 为 int 类型的变量，且表达式 `((a | 3) == 3)` 的值为 true，则说明 a 在从 0 到 3 之间（可能为 0、可能为 3）。

【知识点：位运算】

答案：正确

解析：本题判断为正确。按位或 `|` 会把某一位中至少有一个 `1` 的结果置为 `1`。`3` 的二进制低两位是 `11`，其他高位都是 `0`；若 `(a | 3) == 3`，说明 `a` 的高位不能出现 `1`，低两位可以是 `00`、`01`、`10` 或 `11`，所以 `a` 只能是 0、1、2、3。
