## 第1题

高级语言编写的程序需要经过以下（ ）操作，可以生成在计算机上运行的可执行代码。

【知识点：C++语法规则与编译模型】

A. 编辑
B. 保存
C. 调试
D. 编译

答案：D

解析：本题选 D。用高级语言写出的源程序不能直接作为机器可执行代码运行，通常需要经过编译等处理，生成计算机能够执行的目标代码或可执行程序。编辑和保存只是编写、存放源文件，调试是查找和修正错误，都不是生成可执行代码的关键操作。

## 第2题

二进制数 $11.01$ 在十进制下是（ ）。

【知识点：进制转换与进制字面量】

A. $3.01$
B. $3.05$
C. $3.125$
D. $3.25$

答案：D

解析：本题选 D。二进制 `11.01` 的整数部分 `11` 等于 $1 \times 2^1 + 1 \times 2^0 = 3$；小数部分 `.01` 等于 $0 \times 2^{-1} + 1 \times 2^{-2} = 0.25$。合起来是 $3.25$，所以选 D。

## 第3题

已知大写字符 'A' 的 ASCII 编码的十六进制表示为 $0x41$，则字符 'F' 的 ASCII 编码的十六进制表示为（ ）。

【知识点：ASCII/字符编码】

A. 46
B. 47
C. 48
D. 49

答案：A

解析：本题选 A。大写英文字母在 ASCII 编码中是连续排列的，`'A'` 是 `0x41`，`'F'` 比 `'A'` 往后数 5 个位置，所以编码是 `0x41 + 5 = 0x46`。题目问十六进制表示，对应选项为 46。

## 第4题

以下哪个不是 C++ 语言中的运算符？（ ）

【知识点：位运算】

A. &
B. &&
C. \*
D. **

答案：D

解析：本题选 D。`&` 可以表示按位与或取地址，`&&` 是逻辑与，`*` 可以表示乘法、指针声明或解引用，都是 C++ 中的运算符。C++ 没有 `**` 这个幂运算符，求幂通常要用函数或自己计算，因此 D 不是 C++ 运算符。

## 第5题

如果字符串定义为 `char str[] = "Hello";`，则字符数组 str 的长度为（ ）。

【知识点：数组与字符数组】

A. 0
B. 5
C. 6
D. 7

答案：C

解析：本题选 C。字符串字面量 `"Hello"` 包含 5 个可见字符 `H`、`e`、`l`、`l`、`o`，末尾还会自动带上字符串结束标记 `'\0'`。因此用它初始化 `char str[]` 时，数组长度是 $5 + 1 = 6$。

## 第6题

一个数组定义为 `double array[3];`，则这个数组占用内存的大小为（ ）。

【知识点：数组与字符数组】

A. 24
B. 12
C. 6
D. 3

答案：A

解析：本题选 A。在常见 C++ 环境和本级别考查中，`double` 类型通常占 8 字节。`double array[3];` 有 3 个 `double` 元素，数组元素连续存放，所以总大小是 $3 \times 8 = 24$ 字节。

## 第7题

以下数组定义，符合 C++ 语言语法的是（ ）。

【知识点：数组与字符数组】

A. `int a[];`
B. `int b['3'];`
C. `int c[3.0];`
D. `int[3] d;`

答案：B

解析：本题选 B。`int b['3'];` 中的 `'3'` 是字符常量，属于整型常量表达式，可以作为数组长度使用。`int a[];` 没有给出长度也没有初始化列表，无法确定数组大小；`int c[3.0];` 用浮点数作数组长度不合法；`int[3] d;` 的数组声明顺序错误，正确写法应类似 `int d[3];`。

## 第8题

下列关于进制的叙述，不正确的是（ ）。

【知识点：进制转换与进制字面量】

A. 正整数的二进制表示中只会出现 $0$ 和 $1$。
B. $10$ 不是 $2$ 的整数次幂，所以十进制数无法转换为二进制数。
C. 从二进制转换为 $8$ 进制时，可以很方便地由低到高将每 $3$ 位二进制位转换为对应的一位 $8$ 进制位。
D. 从二进制转换为 $16$ 进制时，可以很方便地由低到高将每 $4$ 位二进制位转换为对应的一位 $16$ 进制位。

答案：B

解析：本题选 B。任意十进制整数都可以转换成二进制表示，并不要求 10 本身是 2 的整数次幂。A 正确，因为二进制只使用 0 和 1；C、D 也正确，因为 8 进制每一位对应 3 个二进制位，16 进制每一位对应 4 个二进制位。

## 第9题

下列关于 C++ 语言中数组的叙述，不正确的是（ ）。

【知识点：数组与字符数组】

A. 数组必须先定义后使用。
B. 数组的所有元素在内存中是连续存放的。
C. 除了字符数组，在定义数组时“[]”内必须有常数。
D. 不能对数组赋值，但可以对数组的每个基础类型的元素赋值。

答案：C

解析：本题选 C。数组必须先定义后使用，元素在内存中连续存放，定义后的数组名也不能像普通变量那样整体赋值，这些说法都正确。C 的问题在于“除了字符数组”过于特殊化：普通数组也可以通过初始化列表省略长度，例如 `int a[] = {1, 2, 3};`，编译器会根据初值个数确定数组大小。

## 第10题

一个 int 类型的值，做以下哪个操作，一定会变回原来的值？（ ）

【知识点：位运算】

A. 左移 3 位，再右移 3 位。
B. 右移 3 位，再左移 3 位。
C. 按位或 7，再按位与 -8。
D. 按位异或 7，再按位异或 7。

答案：D

解析：本题选 D。异或有“相同为 0，不同为 1”的性质，同一个数连续异或同一个值两次会恢复原值，即 `(x ^ 7) ^ 7 == x`。左移再右移可能丢掉高位信息，右移再左移可能丢掉低位信息；先按位或 7 再按位与 -8 会把低 3 位清零，也不一定等于原数。

## 第11题

如果 a 和 b 均为 int 类型的变量，下列表达式能正确判断 “a 等于 b” 的是（ ）。

【知识点：位运算】

A. `(a / b) == 1`
B. `(a & b) == a`
C. `(a ^ b) == 0`
D. `(a | b) == b`

答案：C

解析：本题选 C。两个整数按位异或时，相同的二进制位结果为 0，不同的二进制位结果为 1；因此 `a ^ b` 为 0 当且仅当每一位都相同，也就是 `a` 等于 `b`。`a / b == 1` 会受整除影响且还可能遇到 `b` 为 0；`a & b == a` 和 `a | b == b` 只能说明某些位包含关系，不能保证两个数相等。

## 第12题

如果 a 为 int 类型的变量，下列哪个表达式可以正确求出满足“小于等于 a 且是 4 的倍数”的整数中最大的？

【知识点：位运算】

A. `(a & (~3))`
B. `((a << 2) >> 2)`
C. `(a ^ 3)`
D. `((a - 1) | 3) + 1`

答案：A

解析：本题选 A。4 的倍数在二进制中最低两位一定是 `00`。`~3` 的作用可以理解为除最低两位外都是 1，`a & (~3)` 会把 `a` 的最低两位清零，得到不超过 `a` 的最大 4 的倍数。B 只是移位后再移回，不能保证得到 4 的倍数；C 会翻转低位；D 实际上更接近把数调整到不小于 `a` 的 4 的倍数。

## 第13题

在下列代码的横线处填写（ ），可以使得输出是 “24 12”。

```cpp
#include <iostream>
using namespace std;
int main() {
    int a = 12, b = 24;
    ________; // 在此处填入代码
    a = a ^ b;
    b = a ^ b;
    cout << a << " " << b << endl;
    return 0;
}
```

【知识点：位运算】

A. `a = a ^ b`
B. `b = a ^ b`
C. `a = a + b`
D. `b = a + b`

答案：B

解析：本题选 B。若先执行 `b = a ^ b`，初始 `a = 12, b = 24`，则 `b` 变成 `12 ^ 24`。随后 `a = a ^ b` 会得到原来的 24，最后 `b = a ^ b` 会得到原来的 12，输出正好是 `24 12`。这是异或交换变量的一种写法；A 会把步骤顺序破坏，C、D 用加法也不能配合后面的异或语句完成交换。

## 第14题

在下列代码的横线处填写（ ），可以使得输出是 “2”。

```cpp
#include <iostream>
using namespace std;
int main() {
    int array[5] = {3, 7, 5, 2, 4};
    int min = 0;
    for (int i = 0; i < 5; i++)
        if (______) // 在此处填写代码
            min = array[i];
    cout << min << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. `min > array[i]`
B. `min < array[i]`
C. `min = array[i]`
D. 以上均不对。

答案：D

解析：本题选 D。`min` 初值是 0，而数组元素都是正数。填 A 时 `0 > array[i]` 一开始就不成立，最后仍输出 0；填 B 时会先更新到 3，再更新到 7，最后输出 7；填 C 时 `min = array[i]` 是赋值表达式，数组元素都非 0，条件每次为真，最后输出 4。三个选项都不能输出 2，所以应选“以上均不对”。

## 第15题

在下列代码的横线处填写（ ），可以使得输出不是 “31”。

```cpp
#include <iostream>
using namespace std;
int main() {
    int array[5] = {1, 2, 4, 8, 16};
    int res = 0;
    for (int i = 0; i < 5; i++)
        ________; // 在此处填写代码
    cout << res << endl;
    return 0;
}
```

【知识点：位运算】

A. `res = res + array[i]`
B. `res = res & array[i]`
C. `res = res | array[i]`
D. `res = res ^ array[i]`

答案：B

解析：本题选 B。A 是把 1、2、4、8、16 相加，结果为 31；C 是把这些互不重叠的二进制位按位或起来，结果也是 31；D 对这些互不重叠的二进制位做异或，结果仍是 31。B 中 `res` 初值为 0，而 `0 & x` 恒为 0，所以最终输出 0，不是 31。
