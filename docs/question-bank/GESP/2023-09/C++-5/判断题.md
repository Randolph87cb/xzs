## 第1题

TCP/IP 的传输层的两个不同的协议分别是 UDP 和 TCP。

答案：正确

解析：TCP/IP 体系中传输层最常见的两个协议就是 TCP 和 UDP。TCP 提供面向连接、可靠传输等机制，UDP 提供无连接、开销较小的数据报传输；二者都属于传输层协议，所以题干表述正确。

## 第2题

在特殊情况下流程图中可以出现三角框和圆形框。

答案：错误

解析：流程图有约定俗成的基本图形，例如起止框、处理框、判断框、输入输出框和流程线等，不能随意用未定义的图形代替。三角框不是常用流程图规范中的基本符号；圆形即使在某些图中可作连接符，也不能说明“三角框和圆形框”都可以作为特殊情况下的流程图框使用，因此题干说法错误。

## 第3题

找出自然数 $ N $ 以内的所有质数，常用算法有埃氏筛法和线性筛法，其中埃氏筛法效率更高。

答案：错误

解析：埃氏筛和线性筛都可以求出 $N$ 以内的所有质数。埃氏筛通过从每个质数开始标记其倍数，常见复杂度为 $O(N \log \log N)$；线性筛保证每个合数只被其最小质因子筛掉一次，复杂度为 $O(N)$。从通常的复杂度分析看，线性筛效率更高，题干说“埃氏筛法效率更高”是错误的。

## 第4题

在 C++ 中，可以使用二分查找链表中的元素。

答案：错误

解析：二分查找依赖“能快速访问中间位置”的顺序存储结构，例如数组或支持随机访问的 `vector`。链表节点只保存指向下一个节点的指针，查找中间节点需要从头遍历，无法像数组那样用下标直接跳到中点，因此通常不能用二分查找来高效查找链表中的元素。

## 第5题

在 C++ 中，通过恰当的实现，可以将链表首尾相接，形成循环链表。

答案：正确

解析：普通单链表通常让尾节点的 `next` 指针指向空指针。若把尾节点的 `next` 指向头节点，或在带头结点的实现中指回头结点，就可以让链表首尾相接，形成循环链表，所以题干说法正确。

## 第6题

贪心算法的解可能不是最优解。

答案：正确

解析：贪心算法每一步只选择当前看来最优的方案，但局部最优不一定能推出全局最优。只有当问题满足相应的贪心选择性质和最优子结构时，贪心结果才一定最优；否则可能得到非最优解，所以题干说法正确。

## 第7题

一般说来，冒泡排序算法优于归并排序。

答案：错误

解析：冒泡排序通过相邻元素反复比较和交换完成排序，平均和最坏时间复杂度通常为 $O(N^2)$；归并排序采用分治合并，时间复杂度通常为 $O(N \log N)$。在数据规模较大时，归并排序一般明显优于冒泡排序，因此题干把冒泡排序说成一般优于归并排序是错误的。

## 第8题

C++ 语言中的 `qsort` 库函数是不稳定排序。

答案：正确

解析：稳定排序要求关键字相等的元素排序后仍保持原来的相对次序。C/C++ 标准库中的 `qsort` 只要求按比较函数完成排序，并不保证相等元素的原有顺序不变，因此它是不稳定排序；若需要稳定性，应使用稳定排序算法或 C++ 的 `stable_sort`。

## 第9题

质数的判定和筛法的目的并不相同，质数判定旨在判断特定的正整数是否为质数，而质数筛法意在筛选出范围内的所有质数。

答案：正确

解析：质数判定面对的是一个给定整数，例如判断 97 是否为质数；筛法面对的是一个范围，例如一次求出 $1$ 到 $N$ 中的所有质数。二者都与质数有关，但目标和典型算法过程不同，题干区分这两类任务是正确的。

## 第10题

下面的 C++ 代码执行后将输出 0 5 1 6 2 3 4。

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

bool compareModulo5(int a, int b) {
    return a % 5 < b % 5;
}

int main() {
    int lst[7];
    for (int i = 0; i < 7; i++)
        lst[i] = i;

    // 对序列所有元素按compareModulo5结果排序
    sort(lst, lst + 7, compareModulo5);

    for (int i = 0; i < 7; i++)
        cout << lst[i] << " ";
    cout << endl;
    return 0;
}
```

答案：正确

解析：比较函数按 `a % 5` 的值从小到大排序。原数组为 `0 1 2 3 4 5 6`，各元素模 5 的余数分别为 `0,1,2,3,4,0,1`，按余数分组后依次是余数 0 的 `0 5`、余数 1 的 `1 6`、余数 2 的 `2`、余数 3 的 `3`、余数 4 的 `4`，所以试题给出的输出为正确。需要注意，`sort` 本身不是稳定排序；从严格的 C++ 标准角度看，余数相同的元素不保证保持原相对顺序，本题应按官方试题预期理解为得到上述分组顺序。
