## 第1题

近年来，线上授课变得普遍，很多有助于改善教学效果的设备也逐渐流行，其中包括比较常用的手写板，那么它属于哪类设备？（ ）。

A. 输入
B. 输出
C. 控制
D. 记录

答案：A

解析：手写板的作用是把用户书写、点击或绘图的动作采集到计算机中，本质上是向计算机输入信息，所以属于输入设备。显示器、打印机等才是输出设备；“控制”“记录”不是这里按计算机外设功能分类时的正确类别。

## 第2题

如果 a 和 b 均为 int 类型的变量，且 b 的值不为 0，那么下列能正确判断“a 是 b 的 3 倍”的表达式是（ ）。

A. `(a >> 3 == b)`
B. `(a - b) % 3 == 0`
C. `(a / b == 3)`
D. `(a == 3 * b)`

答案：D

解析：要判断“`a` 是 `b` 的 3 倍”，需要直接判断 `a` 是否等于 `3 * b`，所以 D 正确。A 中 `a >> 3` 是位右移，通常相当于把整数除以 8 后取整，并不是乘 3 关系；B 只说明 `a-b` 能被 3 整除，例如 `a=5,b=2` 时成立但 `a` 不是 `b` 的 3 倍；C 会受整数除法截断影响，例如 `a=7,b=2` 时 `a / b == 3` 成立但实际并不是 3 倍。

## 第3题

以下不属于面向对象程序设计语言的是（ ）。

A. C++
B. Python
C. Java
D. C

答案：D

解析：C++、Python、Java 都支持面向对象程序设计，可以定义类并创建对象。C 语言主要是面向过程的程序设计语言，标准 C 本身没有类、继承、多态等面向对象语言的核心机制，因此不属于面向对象程序设计语言。

## 第4题

下面有关 C++ 类定义的说法，错误的是（ ）。

A. C++ 类实例化时，会执行构造函数。
B. C++ 自定义类可以通过定义构造函数实现自动类型转换。
C. C++ 自定义类可以通过重载 >、< 等运算符实现大小比较。
D. C++ 自定义类可以包含任意类型的成员。

答案：D

解析：错误的是 D，“任意类型”说法过于绝对。例如一个类不能直接包含该类自身类型的普通数据成员，否则对象大小会无限递归；抽象类对象也不能作为普通成员被实例化。A 中对象实例化会调用相应构造函数；B 中非 `explicit` 的单参数构造函数可参与自动类型转换；C 中自定义类可以通过重载比较运算符实现大小比较，这些说法都成立。

## 第5题

有关下面 C++ 代码的说法，错误的是（ ）。

```cpp
#include <iostream>
#include <string>
using namespace std;

class MyStr {
    string data;
public:
    MyStr(string _data): data(_data) {}
};

int main() {
    MyStr st("ABC");
    cout << st << endl;
    return 0;
}
```

A. 代码 `cout << st << endl;` 会报错，因为没有为 `MyStr` 类重载 `<<` 运算符。
B. 第 6 行代码的 `data` 是 `MyStr` 类的成员变量。
C. 代码 `MyStr st("ABC");` 不会报错，将执行构造函数。
D. 代码 `cout<< st.data <<endl;` 可输出 `ABC`。

答案：D

解析：错误的是 D。`MyStr` 在 `public:` 之前声明的 `string data;` 默认是私有成员，类外的 `main` 函数不能直接访问 `st.data`，所以 `cout << st.data << endl;` 会因访问私有成员而编译失败。A 也说明了 `cout << st` 不能直接输出对象，因为没有为 `MyStr` 提供适用的 `operator<<`；B、C 分别对应成员变量和构造函数调用，都是正确说法。

## 第6题

下列关于命名空间的说法错误的是（ ）。

A. 命名空间可以嵌套，例如 `namespace A { namespace B { int i; }; }`。
B. 命名空间只可以在全局定义。
C. 命名空间中可以存放变量和函数。
D. 如果程序中使用了 `using` 命令同时引用了多个命名空间，并且命名空间中存在相同的函数，会出现程序运行错误。

答案：D

解析：错误的是 D。多个命名空间中可以存在同名函数，单纯使用 `using` 引入这些命名空间不会导致“运行错误”；只有在调用时无法判定应使用哪一个同名函数，才会产生编译期的二义性错误。A 的嵌套命名空间、C 的命名空间中存放变量和函数都成立；B 在本题语境下强调命名空间定义不能写在函数或类的局部块内。

## 第7题

有关下面 C++ 代码的说法，正确的是（ ）。

```cpp
#include <iostream>
using namespace std;

class ManyData {
    int * __data;
    int head, tail, capacity;
public:
    ManyData(int cap) {
        capacity = cap;
        __data = new int[capacity];
        head = tail = 0;
    }
    void push(int val) {
        __data[tail++] = val;
    }
    int pop() {
        return __data[--tail];
    }
    int size() {
        return tail - head;
    }
};

int main() {
    auto myData = ManyData(100);
    myData.push(1);
    myData.push(2);
    myData.push(3);
    myData.push(100);
    cout << myData.size() << endl;
    cout << myData.pop() << endl;
    return 0;
}
```

A. 这段代码不能正常运行。
B. ManyData 类可用于构造队列（Queue）数据结构。
C. 在上面代码环境，代码 `cout<< myData.__data[0] << endl;` 可以增加到代码 main 函数末尾（return 0; 之前），且不会导致报错。
D. 可以为 ManyData 类的 push()、pop() 函数增加异常处理代码，否则在使用 ManyData 类时可能导致运行时错误或逻辑错误（不一定局限于上述代码中的 main 函数）。

答案：D

解析：这段 `main` 中依次压入 1、2、3、100，`size()` 输出 4，`pop()` 从 `tail` 前移后取值，输出最后压入的 100，因此代码本身可以运行，A 错。`pop()` 取的是尾部元素，表现为后进先出，更像栈而不是队列，B 错。`__data` 在 `public:` 之前声明，是私有成员，类外不能访问，C 错。`push()` 没检查数组是否满，`pop()` 没检查是否为空，其他使用场景可能越界或产生逻辑错误，所以 D 正确。

## 第8题

有关下面 C++ 代码的说法，错误的是（ ）。

```cpp
#include <iostream>
using namespace std;

class MoreData {
    int * __data;
    int head, tail, capacity;
public:
    MoreData(int cap) {
        capacity = cap;
        __data = new int[capacity];
        head = tail = 0;
    }
    MoreData & push(int val) {
        __data[tail++] = val;
        return *this;
    }
    int pop() {
        return __data[head++];
    }
    int size() {
        return tail - head;
    }
};

int main() {
    auto myData = MoreData(100);
    myData.push(1);
    myData.push(2);
    myData.push(3);
    myData.push(11).push(12).push(13);
    cout << myData.pop() << endl;
    return 0;
}
```

A. `MoreData` 类可用于构造队列（Queue）数据结构。
B. 代码第 29 行，连续 `push()` 的用法将导致编译错误。
C. `__data` 是 `MoreData` 类的私有成员，只能在类内访问。
D. 以上说法均没有错误。

答案：B

解析：错误的是 B。`push()` 的返回类型是 `MoreData &`，函数末尾返回 `*this`，所以 `myData.push(11).push(12).push(13)` 每次都会在同一个对象上继续调用 `push()`，可以通过编译。`pop()` 从 `head` 位置取出元素并后移，配合尾部插入，可用于构造队列；`__data` 在 `public:` 之前声明，确实是私有成员，只能在类内直接访问。

## 第10题

下面有关格雷码的说法，错误的是（ ）。

A. 在格雷码中，任意两个相邻的代码只有一位二进制数不同。
B. 格雷码是一种唯一性编码。
C. 在格雷码中，最大数和最小数只有一位二进制数不同。

答案：B

## 第12题

$N$ 个节点的二叉搜索树，其查找的平均时间复杂度为（ ）。

A. $O(1)$
B. $O(N)$
C. $O(\log N)$
D. $O(N^2)$

答案：C

解析：二叉搜索树查找时，每比较一个节点，就根据大小关系转向左子树或右子树。若树形较均衡，搜索路径长度与树高同阶，平均约为 $O(\log N)$。$O(1)$ 低估了需要逐层比较的过程；$O(N)$ 是退化成单链时的最坏情况；$O(N^2)$ 则明显高估。

## 第13题

青蛙每次能跳 1 或 2 步。下面是青蛙跳到第 $N$ 步台阶 C++ 实现代码。该段代码采用的算法是（ ）。

```cpp
int jumpFrog(int N) {
    if (N <= 3)
        return N;
    return jumpFrog(N - 1) + jumpFrog(N - 2);
}
```

A. 递推算法
B. 贪心算法
C. 动态规划算法
D. 分治算法

答案：C

解析：青蛙到达第 $N$ 步的走法可以按最后一次跳法划分：最后跳 1 步来自第 $N-1$ 步，最后跳 2 步来自第 $N-2$ 步，所以满足 `f(N)=f(N-1)+f(N-2)`。这种把大问题拆成相邻状态并用状态转移关系求解的思想，按题目给出的选项最符合动态规划算法，因此选 C。需要注意，这段代码没有数组或记忆化保存已经算过的 `jumpFrog(k)`，实际运行会重复计算许多子问题，效率较低；若改为从小到大保存状态值，就会是更典型、更高效的动态规划实现。贪心算法不适用，因为不能只看当前一步就决定总走法；递推算法更强调迭代推出下一项；分治算法通常把子问题独立求解后合并，这里核心更在于重叠状态转移。

## 第14题

$N$ 个节点的双向循环链，在其中查找某个节点的平均时间复杂度是（ ）。

A. $O(1)$
B. $O(N)$
C. $O(\log N)$
D. $O(N^2)$

答案：B

解析：双向循环链表虽然可以从当前节点向前或向后移动，也没有空指针边界问题，但它仍然不能像数组那样通过下标直接跳到目标节点。若只知道要查找的值或目标条件，平均需要遍历一部分节点，数量与 $N$ 成正比，因此平均时间复杂度为 $O(N)$。

## 第15题

关于 C++ 语言，以下说法不正确的是（ ）。

A. 若对象被定义为常量，则它只能调用以 `const` 修饰的成员函数。
B. 所有的常量静态变量都只能在类外进行初始化。
C. 若类 A 的对象 a 是类 B 的静态成员变量，则 a 在 `main()` 函数调用之前应被初始化。
D. 静态全局对象、常量全局对象都是在 `main` 函数调用之前完成初始化，执行完 `main` 函数后被析构。

答案：D

解析：本题要求选择不正确的说法，答案为 D。全局对象和命名空间作用域的 `static` 对象都具有静态存储期，初始化发生在程序开始阶段；如果它们是类类型对象，通常会在程序结束阶段析构。但 D 把“静态全局对象、常量全局对象”一概说成都会在 `main` 之后被析构，表述过于绝对：常量全局变量可能只是内置类型常量，并没有析构函数可执行，因此不能统一称为“被析构”。A 中 `const` 对象只能调用不会修改对象状态的 `const` 成员函数，这是正确规则。C 中类类型静态成员对象也具有静态存储期，应在进入 `main()` 前完成相应初始化。B 的边界要注意：早期 C++ 中不少静态常量成员需要在类外定义，整型 `static const` 常量成员又允许在类内给出常量初始化；但在本题单选语境下，更明显需要判错的是 D 对析构对象范围的概括。
