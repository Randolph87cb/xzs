## 第1题

二进制数 $101.101$ 在十进制下是 $5.005$。

【知识点：进制转换与进制字面量】

答案：错误

解析：二进制整数部分 $101_2 = 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0 = 5$。小数部分 $.101_2 = 1 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3} = 0.625$，所以 $101.101_2$ 等于十进制 $5.625$，不是 $5.005$。

## 第2题

在 C++ 语言中，位运算符也有类似“先乘除、后加减”的优先级规则。因此，使用时应注意合理使用括号。

【知识点：位运算】

答案：正确

解析：C++ 运算符有明确的优先级，位运算符也不例外。例如移位运算 `<<`、`>>`，按位与 `&`，按位异或 `^`，按位或 `|` 的优先级并不相同，并且还会与加减、比较、相等判断等运算符发生组合。为了避免表达式按非预期顺序结合，写位运算表达式时应主动加括号。

## 第3题

字符常量 `'3'` 的值和 int 类型常量 3 的值是相同的，只是占用的字节数不同。

【知识点：ASCII/字符编码】

答案：错误

解析：字符常量 `'3'` 表示字符 3，在常见 ASCII 编码中对应的编码值是 51；整型常量 `3` 的数值就是 3。二者不仅类型不同，数值也不同，不能说只是占用字节数不同。

## 第4题

在 C++ 语言中，长度为 $n$ 的数组，访问下标为 $n$ 的元素会引起编译错误。

【知识点：数组与字符数组】

答案：错误

解析：长度为 $n$ 的数组，合法下标是从 $0$ 到 $n - 1$。访问下标 $n$ 已经越界，但 C++ 通常不会因为这类表达式本身产生编译错误；越界访问属于未定义行为，可能运行异常，也可能表面上正常结束。

## 第5题

在 C++ 语言（洛谷注：竞赛环境是 C++14 语言标准）中，所有 int 类型的值，经过若干次左移操作（`<<`）后，它们的值总会变为 0。

【知识点：位运算】

答案：正确

解析：左移运算可以按固定二进制位宽来理解：每左移一次，二进制位整体向高位移动一位，低位补 0。`int` 能保存的位数是有限的，反复左移时，原来为 1 的高位会逐步移出这个有限范围；当所有 1 都移出后，留下的可见二进制位全是 0，因此按这种有限位宽模型判断，结果会变为 0。

## 第6题

在 C++ 语言中，数组下标的大小决定元素在逻辑上的先后顺序，与元素在内存中位置的先后顺序无关。

【知识点：数组与字符数组】

答案：错误

解析：C++ 数组元素在内存中连续存放，并且下标较小的元素排在前面，下标较大的元素排在后面。例如 `a[0]`、`a[1]`、`a[2]` 在逻辑顺序和内存顺序上都是依次排列的。因此题干说“与内存中位置的先后顺序无关”是错误的。

## 第7题

在 C++ 语言中，定义数组时，`[]` 中必须指定元素个数。

【知识点：数组与字符数组】

答案：错误

解析：定义普通数组时通常需要给出元素个数，例如 `int a[10];`。但如果定义时同时给出初始化列表，可以省略方括号中的个数，由编译器根据初始值个数推断，例如 `int a[] = {1, 2, 3};`。所以“必须指定元素个数”过于绝对。

## 第8题

著名的哥德巴赫猜想：任一大于 2 的偶数都可写成两个素数之和。我们可以通过枚举法来证明它。

【知识点：枚举与基础算法】

答案：错误

解析：枚举法可以验证某个有限范围内的偶数是否都能写成两个素数之和，但哥德巴赫猜想讨论的是任意大于 2 的偶数，范围是无限的。对无限命题，简单枚举有限多个例子不能构成证明，因此题干说“可以通过枚举法来证明它”是错误的。

## 第9题

在 C++ 语言中，表达式 `(0xff == 255)` 的值为 true。

【知识点：进制转换与进制字面量】

答案：正确

解析：`0xff` 是十六进制整型字面量，`f` 表示十进制的 15，所以它的值是 $15 \times 16 + 15 = 255$。因此表达式 `0xff == 255` 的比较结果为 `true`。

## 第10题

如果 a 为 int 类型的变量，且表达式 `((a & 1) == 0)` 的值为 true，则说明 a 是偶数。

【知识点：位运算】

答案：正确

解析：`a & 1` 会保留 `a` 的最低二进制位。偶数的最低位是 0，奇数的最低位是 1；如果 `(a & 1) == 0` 为 `true`，说明最低位为 0，因此 `a` 是偶数。
