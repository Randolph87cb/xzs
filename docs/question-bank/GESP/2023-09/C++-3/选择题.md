## 第1题

人们所使用的手机上安装的 App 通常指的是（ ）。

【知识点：计算机基础常识】

A. 一款操作系统
B. 一款应用软件
C. 一种通话设备
D. 以上都不对

答案：B

解析：App 是 Application 的常见缩写，通常指安装在手机、平板等设备上的应用软件。操作系统对应的是 Android、iOS 等底层系统；通话设备指手机等硬件本身，都不是“App”的含义。

## 第2题

下列流程图的输出结果是？（ ）

![](https://cdn.luogu.com.cn/upload/image_hosting/i3r3ksr7.png)

【知识点：控制结构与程序阅读】

A. 60
B. 20
C. 5
D. 1

答案：B

解析：流程图先令 `a = 5, s = 1`。第一次判断 `a >= 4` 成立，执行 `s = s * a` 得到 `s = 5`，再令 `a = 4`；第二次判断仍成立，执行后 `s = 20`，再令 `a = 3`；第三次判断不成立，输出当前的 `s`，所以结果是 20。

## 第3题

已知大写字符 'A' 的 ASCII 编码的十六进制表示为 0x41，则字符 'L' 的 ASCII 编码的十六进制表示为（ ）。

【知识点：ASCII/字符编码】

A. 4A
B. 4B
C. 4C
D. 52

答案：C

解析：大写英文字母的 ASCII 编码是连续的。`'A'` 是 `0x41`，`'L'` 比 `'A'` 往后数 11 个位置，所以编码是 `0x41 + 0x0B = 0x4C`。`4A`、`4B` 分别对应更靠前的字母，`52` 对应的偏移更大。

## 第4题

以下哪个不是 C++ 语言中的运算符？（ ）

【知识点：位运算】

A. ~
B. ~~
C. <
D. <<

答案：B

解析：`~` 是 C++ 的按位取反运算符，`<` 是小于运算符，`<<` 可以作为左移运算符，也可以用于输出流。`~~` 不是一个单独的 C++ 运算符；如果写成 `~~x`，会被理解成连续使用两个 `~`，而不是名为 `~~` 的运算符。

## 第5题

如果数组定义为 `long long array[] = {3, 5, 7, 2};`，则数组 array 占用的字节数为（ ）。

【知识点：数组与字符数组】

A. 32
B. 16
C. 8
D. 4

答案：A

解析：数组初始化列表中有 4 个元素，因此数组长度为 4。`long long` 类型通常占 8 个字节，所以整个数组占用 $4 \times 8 = 32$ 字节。16、8、4 都少算了元素个数或单个元素大小。

## 第6题

一个数组定义为 `double array[3];`，则可合理访问这个数组的元素的下标最大为（ ）。

【知识点：数组与字符数组】

A. 2
B. 3
C. 23
D. 24

答案：A

解析：`double array[3];` 定义了 3 个元素，下标从 0 开始，分别是 `array[0]`、`array[1]`、`array[2]`。因此可合理访问的最大下标是 2，而不是元素个数 3。

## 第7题

以下数组定义，符合 C++ 语言语法的是（ ）。

【知识点：数组与字符数组】

A. `double a[];`
B. `double b[] = {1, 2.0, '3'};`
C. `double c[3.0];`
D. `double[] d = new double[3];`

答案：B

解析：`double b[] = {1, 2.0, '3'};` 可以由初始化列表推断数组长度，且 `1` 和字符 `'3'` 都能转换为 `double` 存入数组。`double a[];` 既没有指定长度也没有初始化列表；`double c[3.0];` 的数组长度不能写成浮点数；`double[] d = new double[3];` 是类似 Java 的写法，不是 C++ 数组定义语法。

## 第8题

下列关于进制的叙述，正确的是（ ）。

【知识点：进制转换与进制字面量】

A. 只有十进制和二进制能够用来表示小数，八进制和十六进制不可以。
B. 常用的进制包括二进制、八进制、十进制、十六进制，其他进制在日常生活中很少使用。
C. 对任意正整数，其二进制表示不会比它的十进制表示更短。
D. 正整数的八进制表示中，每一位可能出现的最大数字是 8。

答案：C

解析：任意进制都可以表示小数，所以 A 错。B 是经验性描述，而且日常生活中也能见到时间的 60 进制、月份的 12 进制等，并不是严谨的进制性质。八进制每一位只能是 0 到 7，不能出现 8，所以 D 错。对一个正整数来说，二进制每一位只提供 2 种选择，十进制每一位有 10 种选择；表示同样大小的正整数时，二进制位数不会比十进制位数更短，因此 C 正确。

## 第9题

下列关于 C++ 语言中数组的叙述，不正确的是（ ）。

【知识点：数组与字符数组】

A. 可以定义 0 个元素的数组。
B. 不能定义 -1 个元素的数组。
C. 数组下标越界访问会产生编译错误。
D. 程序运行时发生数组下标的越界访问，程序依然可能正常结束。

答案：C

解析：C++ 编译器主要检查语法和类型，数组下标是否越界很多时候要到运行时才知道。例如 `a[i]` 中 `i` 的值来自输入时，编译阶段无法确定它是否超出范围，所以“越界访问会产生编译错误”这个说法过于绝对，C 是不正确的。越界访问属于未定义行为，程序可能崩溃，也可能表面上正常结束，因此 D 的描述是可能发生的；B 中数组元素个数不能为负数。A 涉及具体编译环境对 0 长度数组写法的支持情况，不是判断 C 这个错误项的关键。

## 第10题

如果 a 是 int 类型的变量，下列哪个表达式的值一定为 true？（ ）

【知识点：位运算】

A. `a + 1000 - 1000 == a`
B. `a * 2 / 2 == a`
C. `(a & 1) == 1`
D. `(a | 1) == a + 1`

答案：A

解析：本题按原卷的入门整数运算语境，`a + 1000 - 1000` 是先加上同一个数再减去同一个数，结果仍为 `a`。B 的中间结果 `a * 2` 可能超出 `int` 可表示范围，不能作为“值一定为 true”的判断依据；C 只有在 `a` 为奇数时为真；D 例如 `a = 3` 时，`a | 1` 仍为 3，而 `a + 1` 为 4，并不相等。严格 C++ 标准下，A 在极端取值也涉及有符号整数溢出边界，本解析保留原卷答案。

## 第11题

如果 a 和 b 均为 int 类型的变量，下列表达式不能正确判断“a 等于 b”的是（ ）。

【知识点：位运算】

A. `(a >= b) && (a <= b)`
B. `(a >> 1) == (b >> 1)`
C. `(a + b) == (a + a)`
D. `(a ^ b) == 0`

答案：B

解析：右移一位会丢掉最低位，所以不同的整数可能右移后相同。例如 `a = 2, b = 3` 时，`a >> 1` 和 `b >> 1` 都是 1，但 `a` 并不等于 `b`。A 同时满足 `a >= b` 和 `a <= b` 才能相等；C 在本题整数代数语境下等价于 `b == a`；D 中 `a ^ b` 为 0 说明每一位都相同，也就是两数相等。

## 第12题

如果 a 为 char 类型的变量，下列哪个表达式可以正确判断“a是大写字母”？（ ）

【知识点：运算符、表达式与类型转换】

A. `a - 'A' <= 26`
B. `'A' <= a <= 'Z'`
C. `'A' <= 'a' <= 'Z'`
D. `('A' <= a) && (a <= 'Z')`

答案：D

解析：判断大写字母必须同时满足不小于 `'A'` 且不大于 `'Z'`，D 用逻辑与把两个条件完整写出。A 只限制了上界，像 `'0'` 这样的字符也可能满足 `a - 'A' <= 26`；B 在 C++ 中不会按数学链式比较处理，而是先算 `'A' <= a` 得到 `true` 或 `false`，再与 `'Z'` 比较；C 更没有使用变量 `a`，不能判断输入字符。

## 第13题

在下列代码的横线处填写（ ），可以使得输出是“20 10”。
```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 10, b = 20;
    a = (a << 8) | b;
    ______  // 在此处填入代码
    cout << a << " " << b << endl;
    return 0;
}
```

【知识点：位运算】

A. `a = a >> 8; b = a & 0xff`
B. `b = a >> 8; a = a & 0xff;`
C. `a = b; b = a & 0xff;`
D. `b = a; a = b;`

答案：B

解析：执行 `a = (a << 8) | b` 后，原来的 `a = 10` 被放到高 8 位，原来的 `b = 20` 保存在低 8 位，此时 `a` 可看作 `0x0A14`。选项 B 先用 `a >> 8` 取出高 8 位给 `b`，得到 10；再用 `a & 0xff` 取出低 8 位给 `a`，得到 20，输出就是 `20 10`。A 先改掉 `a` 后再取低位，会丢失 20；C、D 没有正确分离高低 8 位。

## 第14题

在下列代码的横线处填写（ ），可以使得输出是“120”。
```cpp
#include <iostream>
using namespace std;

int main() {
    int array[5] = {1, 2, 3, 4, 5};
    int res = 0;
    for (int i = 0; i < 5; i++)
        ______; // 在此处填入代码
    cout << res << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. `res += array[i];`
B. `res *= array[i]`
C. `res = array[i]`
D. 以上均不对。

答案：D

解析：初始 `res = 0`。如果填 A，循环后得到数组元素和 $1 + 2 + 3 + 4 + 5 = 15$；如果填 B，由于一开始就是 0，反复乘以后仍是 0；如果填 C，每次都把 `res` 改成当前元素，最后只会留下 5。输出 120 需要类似从 1 开始连乘数组元素，但代码初值不是 1，因此三个代码片段都不符合要求。

## 第15题

下列代码的输出是（ ）。
```cpp
#include <iostream>
using namespace std;

int main() {
    int array[10];
    for (int i = 0; i < 10; i++)
        array[i] = i;
    for (int p = 2; p < 10; p++)
        if (array[p] == p)
            for (int n = p; n < 10; n += p)
                array[n] = array[n] / p * (p - 1);
    int res = 0;
    for (int n = 1; n < 10; n++)
        res += array[n];
    cout << res << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. 15
B. 28
C. 45
D. 55

答案：B

解析：数组先被初始化为 `array[i] = i`。当 `p = 2` 时，更新 2、4、6、8 这些位置，得到 `array[2]=1, array[4]=2, array[6]=3, array[8]=4`；当 `p = 3` 时，更新 3、6、9，得到 `array[3]=2, array[6]=2, array[9]=6`；之后只有 `p = 5` 和 `p = 7` 满足 `array[p] == p`，分别把 `array[5]` 改为 4、`array[7]` 改为 6。最后 `array[1]` 到 `array[9]` 为 `1, 1, 2, 2, 4, 2, 6, 4, 6`，总和是 28。
