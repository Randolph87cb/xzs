## 第1题

人们所使用的手机上安装的 App 通常指的是（ ）。

A. 一款操作系统
B. 一款应用软件
C. 一种通话设备
D. 以上都不对

答案：B

解析：App 通常是 application 的简称，指安装在手机操作系统上的应用软件，例如浏览器、聊天工具或学习软件。操作系统是管理手机硬件和软件资源的基础系统，不是普通 App；通话设备指硬件或设备类别，也不符合题意。

## 第3题

对包含 $n$ 个元素的数组进行冒泡排序，平均时间复杂度一般为（ ）。

A. $O(n)$
B. $O(n \log n)$
C. $O(n^2)$
D. 以上都不正确

答案：C

解析：冒泡排序通过相邻元素比较和交换逐步把较大的元素“冒”到后面。一般实现中外层进行约 $n$ 轮，内层每轮比较次数也是 $n$ 量级，总比较次数约为 $n(n-1)/2$，平均时间复杂度为 $O(n^2)$。$O(n)$ 和 $O(n\log n)$ 都低估了普通冒泡排序的平均比较次数。

## 第4题

下列关于 C++ 语言中指针的叙述，不正确的是（ ）。

A. 可以定义指向 int 类型的指针。
B. 可以定义指向自定义结构体类型的指针。
C. 自定义结构体类型可以包含指针类型的元素。
D. 不能定义指向 void 类型的指针，那没有意义。

答案：D

解析：`int *p;` 可以定义指向 `int` 的指针，自定义结构体也可以有对应的指针类型，例如 `Node *p;`，结构体成员中也常见指针成员。`void *` 是 C++ 中可以定义的通用指针类型，可保存对象地址，只是不能在没有类型转换的情况下直接解引用，所以 D 的说法不正确。

## 第5题

下列关于 C++ 语言中数组的叙述，不正确的是（ ）。

A. 一维数组可以用来表示数列。
B. 二维数组可以用来表示矩阵。
C. 三维数组可以用来表示空间中物体的形状。
D. 世界是三维的，所以定义四维数组没有意义。

答案：D

解析：数组维数只是组织数据的一种方式，一维数组可表示数列，二维数组可表示矩阵，三维数组也可表示空间网格或体数据。四维数组同样有意义，例如在三维空间数据外再加时间、类别或批次这一维；“世界是三维的”不能推出四维数组没有意义。

## 第6题

下列关于 C++ 语言中函数的叙述，正确的是（ ）。

A. 函数调用前必须定义。
B. 函数调用时必须提供足够的实际参数。
C. 函数定义前必须声明。
D. 函数声明只能写在函数调用前。

答案：B

解析：调用函数时，实参与形参必须能够匹配；没有默认值的形参必须由调用者提供对应实参，所以 B 正确。函数可以先声明后定义，因此 A“调用前必须定义”不对；如果函数定义本身出现在调用前，可以不再提前声明，因此 C 不对；函数声明可以在不同位置重复出现，并不只能写在调用前，所以 D 也不对。

## 第7题

下列关于 C++ 语言中函数的叙述，不正确的是（ ）。

A. 两个函数的声明可以相同。
B. 两个函数的局部变量可以重名。
C. 两个函数的参数可以重名。
D. 两个函数可以重名。

答案：A

解析：两个不同函数不能靠完全相同的声明来区分；相同的函数名和参数列表表示同一个函数声明，不能表示两个不同函数，所以 A 不正确。不同函数各自的局部变量、形参名互不冲突，可以重名；函数名也可以通过重载而相同，但参数列表必须不同。

## 第8题

一个二维数组定义为 `char array[3][10];`，则这个二维数组占用内存的大小为（ ）。

A. 10
B. 30
C. 32
D. 48

答案：B

解析：`char` 类型通常占 1 字节，`char array[3][10];` 一共有 `3*10=30` 个 `char` 元素，因此数组元素本身占 30 字节。这里不是字符串长度题，也没有额外的行指针或补齐到 32、48 字节的规则。

## 第9题

如果 n 为 int 类型的变量，一个指针变量定义为 `int *p = &n;`，则下列说法正确的是（ ）。

A. 指针变量 p 的值与变量 n 是相同的。
B. 指针变量 p 的值与变量 n 的地址是相同的。
C. 指针变量 p 指向的值为 'n'。
D. 指针变量 p 指向的值与变量 n 的地址是相同的。

答案：B

解析：`&n` 表示变量 `n` 的地址，`int *p = &n;` 把这个地址保存到指针变量 `p` 中，所以 `p` 的值等于 `n` 的地址。`p` 的值不是 `n` 的整数值；`*p` 才表示通过指针访问到的 `n` 的值，也不是字符 `'n'` 或地址本身。

## 第10题

一个三维数组定义为 `long long array[6][6][6];`，则 `array[1][2][3]` 和 `array[3][2][1]` 在内存中的位置相差多少字节？（ ）

A. 70 字节
B. 198 字节
C. 560 字节
D. 无法确定

答案：C

解析：C++ 多维数组按行优先连续存储，`array[i][j][k]` 的线性下标为 `i*6*6+j*6+k`。`array[1][2][3]` 的下标是 `1*36+2*6+3=51`，`array[3][2][1]` 的下标是 `3*36+2*6+1=121`，相差 `70` 个 `long long` 元素。每个 `long long` 占 8 字节，因此相差 `70*8=560` 字节。

## 第11题

如果 a 为 int 类型的变量，且 a 的值为6，则执行 `a = ~a;` 之后，a 的值会是（ ）。

A. -6
B. 6
C. -7
D. 7

答案：C

解析：`~` 是按位取反运算，每一位 0 变 1、1 变 0。在常见补码表示中有恒等关系 `~x == -x-1`，因此 `~6` 等于 `-6-1=-7`。它不是保持原值的运算，也不是普通的取相反数，所以不是 6、7 或 -6。

## 第12题

一个数组定义为 `int a[5] = {1, 2, 3, 4, 5};`，一个指针定义为 `int *p = &a[2];`，则执行 `*p = a[1];` 后，数组 a 中的值会变为（ ）。

A. {1, 2, 2, 4, 5}
B. {1, 3, 3, 4, 5}
C. {1, 2, 3, 3, 5}
D. {1, 2, 4, 4, 5}

答案：A

解析：数组下标从 0 开始，`a[2]` 初值为 3，`p=&a[2]` 表示 `p` 指向第三个元素。语句 `*p = a[1];` 是把 `a[1]` 的值 2 赋给 `p` 指向的位置，也就是把 `a[2]` 改成 2；其他元素不变，所以数组变为 `{1, 2, 2, 4, 5}`。

## 第13题

下列关于 C++ 语言中异常处理的叙述，正确的是（ ）。

A. 一个 try 子句可以有多个 catch 子句与之对应。
B. 如果 try 子句在执行时发生异常，就一定会进入某一个 catch 子句执行。
C. 如果 try 子句中没有可能发生异常的语句，会产生编译错误。
D. catch 子句处理异常后，会重新执行与之对应的 try 子句。

答案：A

解析：一个 `try` 块后面可以跟多个 `catch` 子句，用于分别处理不同类型的异常，所以 A 正确。发生异常时只有类型匹配的 `catch` 才会执行，并不保证一定能被某个 `catch` 捕获；`try` 中没有异常语句也不会导致编译错误；`catch` 处理完后通常继续执行后续语句，不会自动重新执行原来的 `try` 块。

## 第14题

执行以下 C++ 语言程序后，输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int fib[10];
    fib[0] = 0;
    fib[1] = 1;
    for (int i = 2; i < 10; i++)
        fib[i] = fib[i - 1] + fib[i - 2];
    cout << fib[10] << endl;
    return 0;
}
```

A. 0
B. 5
C. 55
D. 无法确定。

答案：D

解析：循环只给 `fib[0]` 到 `fib[9]` 赋值，数组 `fib[10]` 的合法下标范围也是 0 到 9。最后输出的是 `fib[10]`，这是越界访问，程序行为未定义，不能根据斐波那契数列推出稳定输出 55，也不能保证输出 0 或 5，因此结果无法确定。
