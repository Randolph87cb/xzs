## 第1题

定义 `int arr[5] = {1,2,3};` ，则 `arr[3]` 的值为 $0$，`arr[5]` 是合法下标。

【知识点：数组与字符数组】

答案：错误

解析：`int arr[5] = {1,2,3};` 只显式初始化前三个元素，剩余元素会被初始化为 0，所以 `arr[3]` 的值确实为 0。但数组下标范围是 `0~4`，`arr[5]` 越界，不是合法下标；命题中后一部分错误，因此整体为错误。

## 第2题

定义 `double arr[10];` ，未手动初始化时，数组中所有元素的默认值为 $0.0$。

【知识点：数组与字符数组】

答案：错误

解析：局部定义的普通数组 `double arr[10];` 如果没有显式初始化，其元素值是不确定的，并不会自动变成 `0.0`。只有具有静态存储期的数组，或使用初始化语法进行值初始化时，才会被初始化为 0。

## 第3题

定义 `int arr[] = {1,2,3};` ，则 `sizeof(arr)` 的结果为 $12$（`int` 占 $4$ 字节）。

【知识点：数组与字符数组】

答案：正确

解析：`int arr[] = {1,2,3};` 会根据初始化列表推导出 3 个 `int` 元素。若一个 `int` 占 4 字节，则整个数组大小为 `3 * 4 = 12` 字节，所以 `sizeof(arr)` 的结果为 12。

## 第5题

下面流程图的功能是计算 $5$ 对 $2$ 取余数，输出结果为 $1$。

![](https://cdn.luogu.com.cn/upload/image_hosting/1yeu7o6d.png)

【知识点：控制结构与程序阅读】

答案：正确

解析：取余运算 `a % b` 表示 `a` 除以 `b` 后的余数。`5 = 2 * 2 + 1`，所以 `5` 对 `2` 取余的结果是 `1`，流程图若输出该余数，则判断正确。

## 第6题

已知大写字母 A 的 ASCII 编码的十六进制表示为 `0x41` ，计算字符 m 的 ASCII 编码的八进制表示为 $155$ （八进制）。

【知识点：ASCII/字符编码】

答案：正确

解析：大写 `A` 的 ASCII 十六进制编码 `0x41` 等于十进制 65。小写 `m` 的编码为十进制 109，109 转八进制为 `155`，因此说法正确。

## 第7题

在 C++ 位运算中，各种不同的运算符有优先级的区分，使用括号能够解决优先级的问题。

【知识点：位运算】

答案：正确

解析：C++ 中位运算符如 `~`、`<<`、`&`、`^`、`|` 等有不同优先级，和算术、比较运算混用时也可能受优先级影响。使用括号可以明确指定先算哪一部分，避免因默认优先级导致理解或计算错误。

## 第8题

由于在 $0 \sim 255$ 范围内，`char` 类型和 `int` 类型可以互换，因此在这里 x 和 y 相等。
 
```cpp
char x='1';
int y=1;
```

【知识点：ASCII/字符编码】

答案：错误

解析：`char x = '1';` 中的 `x` 保存的是字符 `'1'` 的编码值，ASCII 中为 49；`int y = 1;` 保存的是整数 1。二者数值并不相等，不能因为某些字符编码值处在 `0~255` 范围内就认为字符和整数含义可以互换。

## 第9题

在 C++ 语言中，表达式 `((0xff0 + 0x15) == 255)` 的值为 `true` 。

【知识点：进制转换与进制字面量】

答案：错误

解析：`0xff0` 是十六进制数，等于十进制 `15*16^2 + 15*16 = 4080`；`0x15` 等于十进制 21。二者相加为 4101，不等于 255，所以表达式 `((0xff0 + 0x15) == 255)` 的值为 `false`。

## 第10题

如果 a 为 `int` 类型的变量，且 a 的二进制最低位为 $0$ ，则表达式 `((a & 3 & 1) == 0)` 的值为 `true` 。

【知识点：位运算】

答案：正确

解析：`a` 的二进制最低位为 0，说明 `a` 是偶数，`a & 1` 的结果一定为 0。表达式 `a & 3 & 1` 等价于先保留低两位再检查最低位，最低位仍为 0，因此结果等于 0，判断为 `true`。
