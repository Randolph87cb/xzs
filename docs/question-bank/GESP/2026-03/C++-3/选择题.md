## 第1题

如果字符变量 `_1` 的值是字符 $1$,那么 `(int) _1` 的值是? （ ）。

【知识点：ASCII/字符编码】

A. $1$
B. -$1$
C. $49$
D. $+1$ 或者 -$1$

答案：C

解析：字符 `'1'` 存储的是字符编码而不是数值 1。在 ASCII 编码中，字符 `'1'` 的编码值是 49，所以强制转换为 `int` 后得到 49。A、B、D 都把字符值和数字值混淆了。

## 第2题

a,b 是整型变量，各自有互不相同的初始值。下列程序实现了什么效果（ ）。

```cpp
a = a ^ b;
b = a ^ b;
a = a ^ b;
```

【知识点：位运算】

A. a,b 的值从始至终都没有改变。
B. a,b 的值实现了互换。
C. a,b 的值互换了以后，又还回去了，相当于没有变化。
D. a,b 的值最后和原值不一样，没有任何意义。

答案：B

解析：异或满足 `x ^ x = 0`、`x ^ 0 = x`，且交换律、结合律成立。第一句后 `a` 为原来的 `a ^ b`；第二句 `b = (a ^ b) ^ b` 得到原来的 `a`；第三句 `a = (a ^ b) ^ a` 得到原来的 `b`，因此实现交换。题目说明初始值互不相同，也避免了对“看起来没有变化”的误判。

## 第3题

关于下列正确的程序段，说法正确的是（ ）。

```cpp
char str1[] = "Hello";
char str2[]={'H','e','l','l','0'};
```

【知识点：数组与字符数组】

A. 字符数组 str1 和 str2 完全相同。
B. `cout<<str1<<endl; cout<<str2<<endl;` 这段程序多次执行将输出不同的结果。
C. 字符数组 str1 和 str2 不相等。
D. 这两个赋值方式完全相同。

答案：C

解析：`str1` 由字符串字面量初始化，数组内容是 `H e l l o '\0'`，末尾自动带字符串结束符。`str2` 是字符列表初始化，最后一个字符写的是数字字符 `'0'`，不是字母 `'o'`，也没有自动补上同样的字符串内容，因此两个字符数组不相等。A、D 忽略了字符串字面量和字符列表初始化的差异。

## 第4题

关于以下程序段，说法正确的是（ ）。

```cpp
int x=10;
cout<<(x++) + (++x)<<endl;
```

【知识点：C++语法规则与编译模型】

A. C++11 标准中，这是未定义行为，不同的环境有可能出现不同的结果
B. $22$
C. $21$
D. $20$

答案：A

解析：表达式 `(x++) + (++x)` 在同一个完整表达式中对 `x` 进行了多次修改，并且求值顺序没有保证。在 C++11 标准下这是未定义行为，不能依赖某个固定输出。B、C、D 都假定了特定求值顺序，因此不可靠。

## 第5题

$8$ 位二进制下，十进制数 $-15$ 的补码是（ ）。

【知识点：原码反码补码与整数表示】

A. $11110000$
B. $10001111$
C. $10010000$
D. $11110001$

答案：D

解析：8 位补码表示负数时，可先写出 `15` 的二进制 `00001111`，按位取反得 `11110000`，再加 1 得 `11110001`，所以 `-15` 的补码是 `11110001`。A 少了最后加 1 的步骤，B 是符号位加绝对值形式，C 不是 `-15` 的补码。

## 第6题

三进制数 $2102_{(3)}$ 转换成十进制是：（ ）。

【知识点：进制转换与进制字面量】

A. $63$
B. $65$
C. $67$
D. $69$

答案：B

解析：三进制 `2102_(3)` 转十进制为 `2*3^3 + 1*3^2 + 0*3^1 + 2*3^0 = 54 + 9 + 0 + 2 = 65`，所以选 B。

## 第7题

二进制数 $10110101$ 是某数的 $8$ 位补码，该数的十进制是（ ）。

【知识点：原码反码补码与整数表示】

A. $-73$
B. $-75$
C. $-77$
D. $75$

答案：B

解析：8 位补码 `10110101` 最高位为 1，表示负数。求其绝对值可按位取反得到 `01001010`，再加 1 得 `01001011`，十进制为 75，因此原数是 `-75`。A、C 是计算绝对值时出错，D 忽略了最高位表示负数。

## 第8题

已知 `unsigned char c = 0x0F;` （十六进制 `0F` = 二进制 `00001111`），执行 `c = c << 3;` 后，`c` 的十进制值是：（ ）。

【知识点：位运算】

A. $64$
B. $72$
C. $80$
D. $120$

答案：D

解析：`0x0F` 的二进制是 `00001111`，左移 3 位得到 `01111000`，十进制为 `64 + 32 + 16 + 8 = 120`。`unsigned char` 能表示 0 到 255，120 不会溢出，所以结果为 120。

## 第9题

补码的情况下，关于按位取反运算，用笔计算的情况下，以下说法错误的是：（ ）

【知识点：原码反码补码与整数表示】

A. `~5` 的结果是 $-6$（`int` 类型，$32$ 位）
B. `~0` 的结果是 $0$（`int` 类型，$32$ 位）
C. `~(-3)` 的结果是 $2$（`int` 类型，$32$ 位）
D. `~8` 的结果是 $-9$（`int` 类型，$32$ 位）

答案：B

解析：在补码表示下，按位取反有规律 `~x == -x - 1`。因此 `~5 = -6`，`~(-3) = 2`，`~8 = -9` 都正确；而 `~0` 应为所有二进制位变成 1，在 32 位 `int` 补码中表示 `-1`，不是 0，所以 B 错误。

## 第10题

执行以下 C++ 代码后，`sub` 的值是（）。

```cpp
string str = "GESP2026";
string sub = str.substr(4, 2);
```

【知识点：进制转换与进制字面量】

A. $20$
B. $02$
C. $2026$
D. $026$

答案：A

解析：`substr(pos, len)` 从下标 `pos` 开始取 `len` 个字符。字符串 `"GESP2026"` 的下标为 `G(0), E(1), S(2), P(3), 2(4), 0(5), 2(6), 6(7)`，从下标 4 取 2 个字符得到 `"20"`。

## 第11题

执行以下代码后，输出结果是：（ ）。

```cpp
int arr[] = {5, 10, 15, 20, 25, 30};
int count = sizeof(arr) / sizeof(arr[0]);
cout << count;
```

【知识点：数组与字符数组】

A. $4$
B. $6$
C. $24$
D. $30$

答案：B

解析：数组 `arr` 中有 6 个 `int` 元素，`sizeof(arr)` 是整个数组占用字节数，`sizeof(arr[0])` 是一个元素占用字节数，两者相除得到元素个数 6。C 把字节数误当成了元素个数，D 把最后一个元素值误当成了数量。

## 第12题

执行以下代码后，输出结果是：（ ）。

```cpp
char s[10] = "abcde";
int a = sizeof(s) / sizeof(s[0]);
int b = strlen(s);
cout << a - b;
```

【知识点：数组与字符数组】

A. $4$
B. $5$
C. $6$
D. $10$

答案：B

解析：`char s[10] = "abcde";` 定义了长度为 10 的字符数组，因此 `a = 10`。`strlen(s)` 统计字符串结束符 `'\0'` 之前的字符个数，`"abcde"` 长度为 5，因此 `b = 5`，输出 `a - b = 5`。

## 第13题

以下问题中，最不适合用枚举法解决的是：（ ）

【知识点：枚举与基础算法】

A. 找出 $1 \sim 100$ 之间所有能被 $7$ 整除的数
B. 找出 $100 \sim 200$ 之间的所有质数
C. 计算 $1+2+3+\dots+1000$ 的和
D. 找出三位数中个位、十位、百位数字之和等于 $10$ 的数

答案：C

解析：枚举法适合在有限范围内逐个尝试并筛选结果。A、B、D 都需要检查一批候选数是否满足条件；C 是等差数列求和，直接用公式或循环累加即可，没有必要把它作为“枚举候选并筛选”的问题处理，因此最不适合枚举法。

## 第14题

用枚举法解决“鸡兔同笼问题：头共 $35$ 个，脚共 $94$ 只，求鸡和兔的数量”，以下枚举逻辑最合理的是：（ ）

【知识点：枚举与基础算法】

A. 枚举鸡的数量 `x` （$0 \sim 35$），兔的数量 `y=35-x` ，判断 `2*x + 4*y == 94`
B. 枚举兔的数量 `y` （$0 \sim 94$），鸡的数量 `x=35-y` ，判断 `2*x + 4*y == 94`
C. 枚举所有整数 `x` （$0 \sim 100$）和 `y` （$0 \sim 100$），判断 `x+y==35 && 2*x+4*y==94`
D. 枚举脚的总数 `sum` （$0 \sim 94$），判断 `sum == 94`

答案：A

解析：头数为 35，鸡数 `x` 和兔数 `y` 必须满足 `x + y = 35`。只枚举 `x` 的 `0~35`，令 `y = 35 - x`，再判断脚数 `2*x + 4*y == 94`，范围最小且条件完整。B 的兔子数量枚举到 94 会产生不合理数量，C 虽然能找到答案但枚举范围过大，D 只枚举脚总数不能推出鸡兔数量。

## 第15题

模拟“字符串加密”：规则为“每个字符 ASCII 码 +3 ，若超过 z（$122$）则从 a 重新开始”，以下代码中正确的条件判断是：（ ）

```cpp
for (int i = 0; i < str.length(); i++) {
    // 需补充条件:
    ________________
    else {
        str[i] += 3;
    }
}
```

【知识点：ASCII/字符编码】

A. `if (str[i]> 122) str[i] = str[i] + 3 - 26;`
B. `if (str[i] > 122) str[i] = str[i] - 26;`
C. `if (str[i] + 3 > 122) str[i] = str[i] - 26;`
D. `if (str[i] + 3 > 'z') str[i] = 'a' + (str[i] + 3 - 'z') - 1;`

答案：D

解析：应先判断字符加 3 后是否超过 `'z'`，超过时再从 `'a'` 开始回绕。D 用 `str[i] + 3 > 'z'` 判断，并通过 `'a' + (str[i] + 3 - 'z') - 1` 计算回绕后的字符，逻辑完整。A、B 都在加密前判断当前字符是否已超过 122，时机错误；C 判断时机对了，但赋值 `str[i] - 26` 少加了 3，结果不对。
