## 第1题

某班级有 8 名男生和 6 名女生，现要选出 3 人组成学习小组，要求小组中至少有 1 名男生和 1 名女生，则不同的选法共有（ ）种。

A. 112

B. 168

C. 224

D. 288

答案：D

解析：要选 3 人且男女都至少 1 人，只可能是“1 名男生 + 2 名女生”或“2 名男生 + 1 名女生”。前者有 $C_8^1 \times C_6^2 = 8 \times 15 = 120$ 种，后者有 $C_8^2 \times C_6^1 = 28 \times 6 = 168$ 种，共 $120 + 168 = 288$ 种，所以选 D。常见误区是直接算 $C_{14}^3$，那会把 3 人全是男生或全是女生的情况也算进去。

## 第2题

在杨辉三角中，从第 0 行开始计数，第 10 行的所有数之和为（ ）。

A. 512

B. 1024

C. 2048

D. 4096

答案：B

解析：杨辉三角第 $n$ 行的元素是二项式系数 $C_n^0, C_n^1, \ldots, C_n^n$，这一行所有数之和为 $(1 + 1)^n = 2^n$。从第 0 行开始计数时，第 10 行的和是 $2^{10} = 1024$，所以选 B。容易出错的地方是把第 10 行误认为有 10 个数或按第 11 行计算。

## 第3题

下列代码实现了快速幂算法，其时间复杂度为（ ）。

```cpp
long long fastPow(long long b, long long e, long long mod) {
    long long result = 1;
    while (e > 0) {
        if (e & 1)
            result = result * b % mod;
        b = b * b % mod;
        e >>= 1;
    }
    return result;
}
```

A. $O(\log b)$

B. $O(\log e)$

C. $O(\log mod)$

D. $O(e)$

答案：B

解析：循环条件由指数 `e` 控制，每一轮执行 `e >>= 1`，相当于把 `e` 除以 2 并向下取整。因此循环次数与 `e` 的二进制位数成正比，时间复杂度为 $O(\log e)$，选 B。底数 `b` 和模数 `mod` 只参与常数次乘法、取模和赋值，不决定循环轮数。D 是普通逐次乘法的复杂度，不是快速幂的复杂度。

## 第4题

从 5 本不同的数学书和 4 本不同的物理书中选取 3 本书，要求至少包含 1 本数学书，则不同的选法有（ ）种。

A. 60

B. 74

C. 80

D. 84

答案：C

解析：不加限制时，从 9 本不同的书中选 3 本共有 $C_9^3 = 84$ 种。违反条件的情况是 3 本全是物理书，共 $C_4^3 = 4$ 种，所以满足“至少包含 1 本数学书”的选法有 $84 - 4 = 80$ 种，选 C。也可以按选 1、2、3 本数学书分类求和，但要注意不能漏掉选 3 本数学书的情况。

## 第5题

在二叉搜索树（BST）中，若中序遍历的序列为 `{1, 2, 3, 4, 5}`，且先序遍历的第一个序列元素为 3，则下列说法正确的是（ ）。

A. 该树一定是一棵完全二叉树

B. 元素 4 和 5 不可能是兄弟节点

C. 元素 1 所在节点的深度可能大于 3（根节点深度为 1）

D. 元素 2 一定是元素 1 的父节点

答案：B

解析：先序遍历第一个元素是根节点，所以根为 3。BST 的中序序列有序，根 3 左侧只能是 `{1, 2}`，右侧只能是 `{4, 5}`。右子树只有 4 和 5 两个节点，其中一个必须作为右子树的根，另一个是它的孩子，因此 4 和 5 不可能同时作为同一父节点的两个孩子，B 正确。A 错在树形不唯一，不一定完全。左子树只有两个节点，元素 1 的最大深度只能到 3，不可能大于 3，C 错。左子树可以是 2 为父、1 为左孩子，也可以是 1 为父、2 为右孩子，D 的“一定”不成立。

## 第6题

在一个有向带权图中，使用 Dijkstra 算法求单源最短路时，若使用优先队列（小根堆）优化，其时间复杂度为（ ）。

A. $O(V^2)$

B. $O(V \cdot E)$

C. $O((V + E) \log V)$

D. $O(V^2 \log V)$

答案：C

解析：使用优先队列优化的 Dijkstra 算法时，每次取出当前距离最小的节点需要堆操作，边松弛后也可能把新的距离压入堆。若图用邻接表保存边，顶点和边带来的堆操作总量可按 $O(V + E)$ 估计，每次堆操作代价为 $O(log V)$，所以时间复杂度为 $O((V + E) log V)$，选 C。A 更接近不用堆、用数组扫描最小距离点的写法。B 没有体现堆优化。D 把平方级顶点扫描和堆操作混在了一起。

## 第7题

对于含 $n$ 个顶点（$n \ge 2$）的连通加权有向图，若图中不存在负权环，则任意两点之间的最短路径（简单路径）最多包含（ ）条边。

A. $n$

B. $n - 1$

C. $n + 1$

D. 无法确定，取决于图的具体边数

答案：B

解析：题目明确问的是最短路径中的“简单路径”。简单路径不会重复经过顶点，一个含 $n$ 个顶点的图中，简单路径最多经过 $n$ 个不同顶点，因此最多包含 $n - 1$ 条边，选 B。A 和 C 都超过了简单路径的边数上限。D 的说法忽略了“简单路径”这一限制，边数再多也不能让一条简单路径重复顶点。

## 第8题

在使用 Floyd 算法求任意两点间最短路径时，时间复杂度为 $O(V^3)$。若在某次算法执行前，已经用 Dijkstra 算法正确求出了所有点对的最短路并存入了 `dist` 数组。如果此时继续对该 `dist` 数组执行一次完整的 Floyd 算法过程（无任何提前终止），执行完毕后 `dist` 数组内的值（ ）。

A. 会发生改变，因为 Floyd 又做了一次松弛

B. 不会发生改变

C. 可能变大，因为未针对已有最短路优化

D. 可能在某些负权图中陷入死循环

答案：B

解析：如果 `dist` 已经存放了所有点对之间的正确最短路，那么对任意中转点 $k$，都应满足三角不等式 `dist[i][j] <= dist[i][k] + dist[k][j]`。Floyd 的松弛只会在经过 $k$ 更短时更新 `dist[i][j]`，但此时已经没有更短路径可更新，所以数组不会发生改变，选 B。A 错在“执行松弛过程”不等于“一定更新值”。C 错在 Floyd 不会把最短路变大。D 错在 Floyd 是固定三重循环，不会因为负权边而死循环；负权环会使最短路问题本身失去通常意义，但不是这里的情形。

## 第9题

关于图论中的最短路径算法，下列说法中严格正确的是（ ）。

A. Dijkstra 算法能够高效处理包含负权边的有向图。

B. Floyd 算法可以求出任意两点间的最短路径，且允许图中存在负权边（但不能有负权环）。

C. 单源最短路径算法无法用于无向图，无向图只能通过 BFS 求解。

D. Dijkstra 算法的每一步必定从当前未访问的节点中，选取距离起始点最远的节点进行松弛操作。

答案：B

解析：Floyd 算法可以求所有点对之间的最短路径，允许边权为负，但不能存在负权环，否则最短路径可能没有良好定义，所以 B 正确。A 错在 Dijkstra 的正确性依赖非负边权，含负权边时可能出错。C 错在无向图同样可以求单源最短路，例如无权图用 BFS，非负权图可用 Dijkstra。D 正好说反了，Dijkstra 每一步选的是当前未确定节点中距离起点最近的节点。

## 第10题

有 6 个人排成一排照相，其中甲、乙两人必须相邻，且丙不能站在排头的不同排法有（ ）种。

A. 120

B. 144

C. 192

D. 240

答案：C

解析：先把甲、乙看成一个整体，则共有这个整体、丙和另外 3 人，一共 5 个对象排列；甲乙内部有 2 种顺序。若不考虑丙不能站排头，共有 $5! \times 2 = 240$ 种。需要减去丙站在排头的情况：丙固定在第一位，其余 4 个对象排列，甲乙内部仍有 2 种，共 $4! \times 2 = 48$ 种。因此满足条件的排法为 $240 - 48 = 192$ 种，选 C。常见误区是只把甲乙捆绑后得到 240，忘记排除丙在排头。

## 第11题

下列代码试图实现 Floyd 算法求所有点对之间的最短路径，横线处应填入（ ）。

```cpp
void floyd(int n, int dist[][MAXN]) {
    for (int k = 0; k < n; k++)
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                if (__________) // 在此处填入选项
                    dist[i][j] = dist[i][k] + dist[k][j];
}
```

A. `dist[i][k] + dist[k][j] < dist[i][j]`

B. `dist[i][k] != INF && dist[k][j] != INF`

C. `dist[i][k] != INF && dist[k][j] != INF && dist[i][k] + dist[k][j] < dist[i][j]`

D. `dist[i][j] == INF`

答案：C

解析：Floyd 的更新条件有两层含义：首先 `i` 到 `k`、`k` 到 `j` 两段路都必须可达，否则不能用 `INF` 参与相加；其次，经过 `k` 的路径必须比当前 `dist[i][j]` 更短，才应该更新。因此条件应为 `dist[i][k] != INF && dist[k][j] != INF && dist[i][k] + dist[k][j] < dist[i][j]`，选 C。A 缺少可达性判断，可能产生无意义的 `INF` 加法。B 只判断可达，不判断是否更短，会把较短距离覆盖成较长距离。D 只看当前是否不可达，不能处理已有路径被更短路径改进的情况。

## 第12题

用数字 0、1、2、3、4 组成无重复数字的五位偶数，共有（ ）个。

A. 48

B. 60

C. 72

D. 96

答案：B

解析：五位偶数的个位必须是 0、2、4。若个位是 0，首位可从 1、2、3、4 中选 1 个，其余 3 位任意排列，共 $4 \times 3! = 24$ 个。若个位是 2 或 4，有 2 种选择；此时首位不能为 0，也不能等于个位，所以首位有 3 种选择，其余 3 位排列，共 $2 \times 3 \times 3! = 36$ 个。总数为 $24 + 36 = 60$，选 B。容易错的是把个位为 0 和个位不为 0 的首位限制混为一谈。

## 第13题

在一个无向带权图中，若使用 Prim 算法从顶点 0 开始构造最小生成树（边权均为正整数，且 `graph[u][v] == 0` 表示无边），下列代码中横线处应填入（ ）。

```cpp
int prim(vector<vector<int>>& graph, int n) {
    vector<bool> inMST(n, false);
    vector<int> minEdge(n, INT_MAX);
    minEdge[0] = 0;
    int result = 0;
    for (int i = 0; i < n; i++) {
        int u = -1;
        for (int j = 0; j < n; j++)
            if (!inMST[j] && (u == -1 || minEdge[j] < minEdge[u]))
                u = j;
        inMST[u] = true;
        result += minEdge[u];
        for (int v = 0; v < n; v++)
            if (__________) // 在此处填入选项
                minEdge[v] = graph[u][v];
    }
    return result;
}
```

A. `graph[u][v] && !inMST[v] && graph[u][v] < minEdge[v]`

B. `!inMST[v] && graph[u][v] < minEdge[v]`

C. `graph[u][v] > 0 && !inMST[v]`

D. `!inMST[v] && minEdge[v] > 0`

答案：A

解析：`minEdge[v]` 表示当前已构造的生成树连接到未加入顶点 `v` 的最小边权。更新时必须同时满足三点：`graph[u][v]` 非 0，说明边存在；`v` 还没有加入最小生成树；新边权 `graph[u][v]` 比当前记录的 `minEdge[v]` 更小。因此 A 正确。B 没有排除 `graph[u][v] == 0` 的无边情况，会把不存在的边当成权值 0 的边。C 不比较是否更小，可能用更大的边覆盖更优记录。D 完全没有检查 `u` 到 `v` 是否有边，也没有比较新边权。

## 第14题

已知三个点 $A(x_1, y_1)$、$B(x_2, y_2)$、$C(x_3, y_3)$ 在平面直角坐标系中的坐标。下列 C++ 表达式中，在精度误差范围 `1e-8` 内能正确计算判断这三个点是三点共线的表达式是（ ）。

A. `(x2-x1)/(y2-y1) == (x3-x1)/(y3-y1)`

B. `(x2-x1)*(y3-y1)-(x3-x1)*(y2-y1) == 0`

C. `fabs((x2-x1)*(y3-y1)-(x3-x1)*(y2-y1)) < 1e-8`

D. `fabs((x2-x1)/(y2-y1)-(x3-x1)/(y3-y1)) < 1e-8`

答案：C

解析：三点共线可以用向量叉积为 0 判断：向量 $\overrightarrow{AB}$ 与 $\overrightarrow{AC}$ 共线时，`(x2-x1)*(y3-y1)-(x3-x1)*(y2-y1)` 应为 0。由于坐标计算可能有浮点误差，应判断其绝对值是否小于给定误差 `1e-8`，所以 C 正确。A、D 使用除法，遇到 `y2 == y1` 或 `y3 == y1` 时可能除以 0，而且斜率比较也不如叉积稳健。B 使用精确等于 0，不适合有精度误差的浮点判断。

## 第15题

在 64 位操作系统下（LP64 / LLP64 模型），下面代码的输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int a[4] = {1, 2, 3, 4};
    int (*p)[4] = &a;
    int *q = a;

    cout << sizeof(a) << " ";
    cout << sizeof(p) << " ";
    cout << sizeof(p + 1) << " ";
    cout << sizeof(q + 1) << " ";
    cout << (p + 1) - p << " ";
    cout << (q + 1) - q << endl;
}
```

A. `16 8 8 8 1 1`

B. `16 8 16 8 1 1`

C. `16 8 8 4 4 1`

D. `16 8 8 8 4 1`

答案：A

解析：数组 `a` 含 4 个 `int`，通常每个 `int` 为 4 字节，所以 `sizeof(a) = 16`。在 64 位 LP64 / LLP64 模型下，指针大小都是 8 字节，因此 `sizeof(p)`、`sizeof(p + 1)`、`sizeof(q + 1)` 都是 8。`p` 的类型是指向含 4 个 `int` 的数组的指针，`p + 1` 在地址上会跨过整个数组，但指针相减的结果表示跨过了 1 个该类型元素，所以 `(p + 1) - p` 为 1。`q` 是 `int*`，`q + 1` 跨过 1 个 `int`，因此 `(q + 1) - q` 也为 1。输出为 `16 8 8 8 1 1`，选 A。常见误区是把指针加法跨过的字节数直接当作指针相减的结果。
