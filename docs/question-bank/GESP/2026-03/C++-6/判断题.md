## 第1题

题干：下面定义了一个表示二维坐标点的类 `Point`，并提供了一个带参数的构造函数，但第 ② 行 `Point b;` 会调用编译器自动生成的默认构造函数，将 `b.x` 和 `b.y` 被初始化为 `0.0`，程序可以正常编译运行。

```cpp
class Point {
public:
    double x, y;
    Point(double px, double py) : x(px), y(py) {}
    void print() {
        cout << "(" << x << ", " << y << ")";
    }
};

int main() {
    Point a(3.0, 4.0);    // ①
    Point b;              // ②
    a.print();
}
```

答案：B

解析：本题判断为错误。类中已经定义了带参数的构造函数 `Point(double px, double py)`，编译器就不会再自动生成无参默认构造函数。语句 `Point b;` 需要调用无参构造函数，但类中并没有这样的构造函数，因此程序不能通过编译，更不会把 `b.x` 和 `b.y` 自动初始化为 `0.0`。正确说法是：如果希望 `Point b;` 合法，需要显式提供默认构造函数，或写成类似 `Point b(0.0, 0.0);` 的形式。

## 第2题

题干：C++ 中的继承支持单继承和多继承，但子类无法直接访问父类的私有成员。

答案：A

解析：本题判断为正确。C++ 允许一个派生类只继承一个基类，也允许一个派生类同时继承多个基类，因此支持单继承和多继承。父类的 `private` 成员只能由父类自己的成员函数或友元直接访问，派生类成员函数不能直接访问这些私有成员；如果需要让派生类使用相关数据，通常要通过父类提供的 `public` 或 `protected` 接口完成。

## 第3题

题干：对如下结构的树，执行 `travel` 函数，输出结果是 `1 2 3 4 5`。

```text
    1
   / \
  2   3
 / \
4   5
```

```cpp
struct Node {
    int val;
    Node *left, *right;
    Node(int v) : val(v), left(nullptr), right(nullptr) {}
};

void travel(Node* root) {
    if (!root) return;
    stack<Node*> s;
    s.push(root);

    while (!s.empty()) {
        Node* cur = s.top(); s.pop();
        cout << cur->val << " ";

        if (cur->right) s.push(cur->right);
        if (cur->left)  s.push(cur->left);
    }
}
```

答案：B

解析：本题判断为错误。栈是后进先出结构，代码在访问当前结点后先压入右孩子，再压入左孩子，所以左孩子会先被弹出并访问。这段代码实现的是二叉树的非递归前序遍历，访问顺序应为根、左子树、右子树。对题图中的树，实际输出是 `1 2 4 5 3`，不是 `1 2 3 4 5`。

## 第4题

题干：若所有字符出现频率相同，则哈夫曼编码一定会得到完全二叉树。

答案：B

解析：本题判断为错误。哈夫曼树每次选取权值最小的两个结点合并，所有字符频率相同时，确实会让叶子深度尽量接近，但不能保证树的形态一定是完全二叉树。完全二叉树要求除最后一层外其余层都满，且最后一层结点从左到右连续排列；而哈夫曼树的形态还受字符个数和相同权值时的合并顺序影响。例如字符个数不是 $2$ 的幂时，即使频率相同，也可能得到并不完全的树。正确说法是：等频率会使哈夫曼编码长度较均衡，但不必然生成完全二叉树。

## 第5题

题干：哈夫曼编码是一种变长的前缀编码，在解码时不需要额外的分隔符就能唯一还原，这是因为在哈夫曼树中，任何一个字符的叶子结点都不会成为另一个字符结点的祖先。

答案：A

解析：本题判断为正确。哈夫曼编码把每个字符放在哈夫曼树的叶子结点上，从根到某个叶子的路径对应这个字符的编码。如果一个字符的编码是另一个字符编码的前缀，那么前者对应的结点就会是后者对应结点的祖先；但字符都在叶子结点上，叶子结点不可能再成为其他字符结点的祖先。因此哈夫曼编码具有前缀码性质，解码时沿树读到叶子就能确定一个字符，不需要额外分隔符。

## 第6题

题干：在 C++ 中使用一维数组 `vector<int> tree` 存储按层序遍历的完全二叉树时，若根节点存储在 `tree[0]`，则对于任意非空节点 `tree[i]`，其右孩子（如果存在）必然位于 `tree[2 * i + 2]`。

答案：A

解析：本题判断为正确。完全二叉树按层序从左到右存入下标从 $0$ 开始的一维数组时，根结点在下标 $0$，下标为 $i$ 的结点左孩子位置是 $2i+1$，右孩子位置是 $2i+2$。题目只说“右孩子如果存在”，因此没有要求所有结点都必须有右孩子；只要该右孩子存在，它在数组中的位置就应为 `tree[2 * i + 2]`。

## 第7题

题干：在 C++ 中使用栈来非递归地实现二叉树的前序遍历时，为了保证遍历顺序正确，在处理完当前结点后，应该先将该结点的左孩子压入栈中，然后再将右孩子压入栈中。

答案：B

解析：本题判断为错误。前序遍历顺序是根、左子树、右子树，而栈的访问顺序是后进先出。如果先把左孩子压栈、再把右孩子压栈，右孩子会先被弹出，遍历顺序会变成先访问右子树。正确做法通常是：处理完当前结点后，先压入右孩子，再压入左孩子，这样左孩子位于栈顶，下一步会被优先访问。

## 第8题

题干：设二叉树共有 $n$ 个结点，函数 `preorderTraversal` 以下代码的时间复杂度为 $O(n)$，空间复杂度为 $O(n)$。

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x): val(x), left(nullptr), right(nullptr) {}
};

void preorder(TreeNode *root, vector<int> &res) {
    if (root == nullptr) {
        return;
    }
    res.push_back(root->val);
    preorder(root->left, res);
    preorder(root->right, res);
}

vector<int> preorderTraversal(TreeNode *root) {
    vector<int> res;
    preorder(root, res);
    return res;
};
```

答案：A

解析：本题判断为正确。前序遍历中，每个非空结点只会被访问一次，并向结果数组 `res` 中加入一次，所以总时间复杂度是 $O(n)$。空间方面，结果数组需要保存 $n$ 个结点的值，递归调用栈在最坏情况下（例如树退化成链）也可能达到 $O(n)$，因此总体空间复杂度可以记为 $O(n)$。

## 第9题

题干：下列代码实现了一个 0-1 背包的一维动态规划代码，内层循环是经典的逆序写法。若将内层循环改成正序遍历（即 `for (int j = w[i]; j <= W; j++)`），仍能得到正确答案。

```cpp
int main() {
    int W = 5;
    int w[] = {2, 3, 4};
    int v[] = {10, 1, 1};
    int n = 3;
    int dp[6] = {0};

    for (int i = 0; i < n; i++) {
        for (int j = W; j >= w[i]; j--) {    // <- 逆序!
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    cout << dp[W];
}
```

答案：B

解析：本题判断为错误。0-1 背包中每件物品最多只能选一次，一维优化时必须让容量 `j` 逆序遍历，才能保证 `dp[j - w[i]]` 仍然来自“没有使用第 `i` 件物品之前”的状态。如果改成正序遍历，同一轮中刚更新过的 `dp[j - w[i]]` 可能马上又被用于更新更大的容量，相当于同一件物品被重复选择，问题就变成了完全背包的更新方式。以代码中的第一件物品为例，若正序遍历，更新 `dp[2] = 10` 后又可能用它更新 `dp[4] = 20`，这已经把重量为 `2` 的同一件物品选了两次，不符合 0-1 背包要求。

## 第10题

题干：在动态规划问题中，状态空间相同且没有重复计算的情况下，“状态转移方程+递推”与“递归+记忆化搜索”的时间复杂度通常相同。

答案：A

解析：本题判断为正确。时间复杂度主要取决于需要计算多少个状态，以及每个状态的转移代价。如果两种写法的状态空间相同，并且递归加记忆化搜索也保证每个状态只计算一次，那么它们处理的状态数量和转移次数通常是同一数量级，因此时间复杂度通常相同。两者的区别更多体现在实现方式、常数开销、递归栈使用以及状态访问顺序上，而不是渐进时间复杂度。
