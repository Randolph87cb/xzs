## 第1题

下列关于 C++ 中类的描述，正确的是（ ）。

A. 如果类没有用户声明的构造函数，那么编译器会隐式声明一个默认构造函数

B. 类的析构函数可以被重载，一个类可以有多个析构函数

C. 类中的所有成员都必须声明为 `public`

D. 类和结构体在 C++ 中没有区别，包括默认访问权限也相同

答案：A

解析：如果一个类没有用户声明的构造函数，编译器会隐式声明一个默认构造函数，使对象可以默认构造，因此 A 正确。析构函数没有参数，一个类只能有一个析构函数，不能重载，所以 B 错。类成员不必都声明为 `public`，还可以是 `private` 或 `protected`，C 错。`class` 和 `struct` 的主要区别之一就是默认访问权限不同，`class` 默认为 `private`，`struct` 默认为 `public`，D 错。

## 第2题

下列代码中，`s1->draw();` 和 `s2->draw();` 输出不同结果的主要原因是（ ）。

```cpp
class Shape {
public:
    virtual void draw() {
        cout << "绘制图形" << endl;
    }

    virtual ~Shape() {}
};

class Circle : public Shape {
public:
    void draw() override {
        cout << "绘制圆形" << endl;
    }
};

class Rectangle : public Shape {
public:
    void draw() override {
        cout << "绘制矩形" << endl;
    }
};

int main() {
    Shape* s1 = new Circle();
    Shape* s2 = new Rectangle();

    s1->draw();
    s2->draw();

    delete s1;
    delete s2;
    return 0;
}
```

A. `draw()` 是普通成员函数

B. `Shape` 中的 `draw()` 被声明为虚函数

C. `Circle` 和 `Rectangle` 中使用了 `public` 继承

D. 指针变量名不同

答案：B

解析：`s1` 和 `s2` 的静态类型都是 `Shape*`，但它们实际分别指向 `Circle` 和 `Rectangle` 对象。由于基类 `Shape` 中的 `draw()` 被声明为 `virtual`，通过基类指针调用时会发生动态绑定，分别调用派生类重写后的函数，所以输出不同。A 与题干相反，若不是虚函数，通常会按基类指针类型调用基类版本。C 的 `public` 继承保证了基类接口可访问，但不是产生多态调用的主要原因。D 指针变量名不会影响函数绑定。

## 第3题

下面的代码在 `main()` 中有一行会导致编译错误，请找出来。

```cpp
class Pet {
public:
    Pet(string n, int a) : name(n), age(a) {}
    string getName()  { return name; }
    void   birthday() { age++; }
private:
    string name;
    int    age;
};

int main() {
    Pet cat("奶茶", 2);
    cout << cat.getName();    // ①
    cat.birthday();           // ②
    cat.name = "大橘";         // ③
    cout << cat.getName();    // ④
}
```

A. 第 ① 行

B. 第 ② 行

C. 第 ③ 行

D. 第 ④ 行

答案：C

解析：`name` 和 `age` 位于 `private` 区域，只能被类的成员函数或友元等访问，不能在 `main()` 中通过 `cat.name` 直接修改，所以第 ③ 行会编译错误，选 C。第 ① 行和第 ④ 行调用的是 `public` 成员函数 `getName()`，第 ② 行调用的是 `public` 成员函数 `birthday()`，这些访问都是允许的。常见误区是看到 `getName()` 返回了 `name` 就误以为外部也能直接访问 `name`，但封装规则区分“通过公有接口访问”和“直接访问私有成员”。

## 第4题

游乐园的过山车每次限坐 4 人，用循环队列管理排队（容量 `MAX=5`，空一格判满）。下面代码执行后，循环队列是否已满？`rear` 的值是多少？

```cpp
const int MAX = 5;
int queue[MAX];
int front = 0, rear = 0;

// 入队
void enqueue(int x) {
    queue[rear] = x;
    rear = (rear + 1) % MAX;
}

// 出队
void dequeue() {
    front = (front + 1) % MAX;
}

int main() {
    enqueue(1); enqueue(2); enqueue(3); enqueue(4);
    dequeue(); dequeue();
    enqueue(5); enqueue(6);
}
```

A. 已满，`rear = 1`

B. 未满，`rear = 1`

C. 已满，`rear = 2`

D. 未满，`rear = 4`

答案：A

解析：开始时 `front = 0, rear = 0`。连续入队 4 次后 `rear` 变为 4；出队 2 次后 `front` 变为 2；再入队 5 时 `rear = (4 + 1) % 5 = 0`，入队 6 后 `rear = (0 + 1) % 5 = 1`。空一格判满的条件是 `(rear + 1) % MAX == front`，此时 `(1 + 1) % 5 = 2`，正好等于 `front`，因此队列已满且 `rear = 1`，选 A。

## 第5题

在以下计算机系统应用场景中，最适合使用循环队列的是（ ）。

A. 函数调用过程中，保存局部变量和返回地址

B. 表达式求值中的运算符优先级处理

C. 操作系统中的进程优先级调度（高优先级先执行）

D. 生产者和消费者问题中的共享缓冲区

答案：D

解析：循环队列适合在固定容量缓冲区中按先进先出的顺序反复入队、出队，生产者和消费者共享缓冲区正是典型场景，所以 D 正确。A 中函数调用栈保存局部变量和返回地址，主要使用栈结构。B 中运算符优先级处理常用栈。C 中高优先级先执行需要按优先级取出元素，更接近优先队列，而不是普通循环队列。

## 第6题

在二叉搜索树（BST）中，若中序遍历的序列为 `{1, 2, 3, 4, 5}`，且先序遍历的第一个序列元素为 3，则下列说法正确的是（ ）。

A. 该树一定是一棵完全二叉树。

B. 元素 4 和 5 不可能是兄弟节点。

C. 元素 1 所在节点的深度可能大于 3（根节点深度为 1）。

D. 元素 2 一定是元素 1 的父节点。

答案：B

解析：先序遍历第一个元素是根节点，所以根为 3。BST 的中序序列有序，左子树只能包含 `{1, 2}`，右子树只能包含 `{4, 5}`。右子树只有 4 和 5 两个节点，其中一个必然是右子树的根，另一个是它的孩子，不可能同时拥有同一个父节点成为兄弟节点，因此 B 正确。A 错在形态并不唯一，不一定完全。左子树只有两个节点，元素 1 的最大深度为 3，不可能大于 3，所以 C 错。左子树可能是 2 为父、1 为左孩子，也可能是 1 为父、2 为右孩子，因此 D 的“一定”不成立。

## 第7题

某二叉树共有 10 个结点，记为 A~J，已知它的先序遍历序列为：`A B D H I E C F J G`，中序遍历序列为：`H D I B E A F J C G`，则该二叉树的后序遍历序列是（ ）。

A. `H I D E B J F G C A`

B. `H I D B E J F G C A`

C. `I H D E B J F G C A`

D. `H I D E B F J G C A`

答案：A

解析：先序序列第一个元素 A 是根。用 A 分割中序序列，左子树中序为 `H D I B E`，右子树中序为 `F J C G`。左子树先序为 `B D H I E`，根为 B；其中 D 的左右孩子分别为 H、I，B 的右孩子为 E，所以左子树后序为 `H I D E B`。右子树先序为 `C F J G`，根为 C；左子树为 F、J，右孩子为 G，因此右子树后序为 `J F G C`。最后接根 A，得到 `H I D E B J F G C A`，选 A。

## 第8题

下列关于树的遍历的说法中，正确的一项是（ ）。

A. 对任意一棵树进行深度优先遍历，所得序列一定唯一。

B. 已知一棵二叉树的先序遍历和后序遍历序列，可以唯一确定这棵二叉树。

C. 已知一棵二叉树的先序遍历和中序遍历序列，可以唯一确定这棵二叉树。

D. 已知一棵二叉树的先序遍历序列，可以唯一确定这棵二叉树。

答案：C

解析：在节点值互不相同的二叉树中，先序遍历可以确定根节点，中序遍历可以根据根节点划分左右子树，再递归确定整棵树，所以 C 正确。A 错在“深度优先遍历”还要看先访问哪个孩子以及采用先序、中序、后序等具体规则，不能笼统说序列一定唯一。B 中先序加后序通常不能唯一确定二叉树，例如只有一个孩子时无法判断它是左孩子还是右孩子。D 只知道先序遍历也无法确定左右子树的划分。

## 第9题

有 6 个字符，它们出现的次数分别为：`{2, 3, 3, 4, 6, 8}`，现在用哈夫曼编码为这些字符编码，最小加权路径长度 WPL（每个字符的出现次数乘它的编码长度，再把每个字符结果加起来）的值为（ ）。

A. 58

B. 60

C. 62

D. 64

答案：D

解析：构造哈夫曼树时，每次合并当前最小的两个权值，WPL 等于所有合并产生的新权值之和。按顺序合并：`2 + 3 = 5`，`3 + 4 = 7`，`5 + 6 = 11`，`7 + 8 = 15`，`11 + 15 = 26`。因此 WPL 为 `5 + 7 + 11 + 15 + 26 = 64`，选 D。常见误区是只把最终根节点 26 当成 WPL，或在有两个 3 时漏掉其中一个 3。

## 第10题

对 `n` 个不同符号的符号进行哈夫曼编码。若生成的哈夫曼树共有 115 个结点，则 `n` 的值是（ ）。

A. 60

B. 58

C. 57

D. 56

答案：B

解析：哈夫曼树是只有度为 0 和度为 2 的二叉树。若有 `n` 个不同符号，则有 `n` 个叶子结点，对应 `n - 1` 个内部结点，总结点数为 `2n - 1`。题中总数为 115，所以 `2n - 1 = 115`，解得 `n = 58`，选 B。若误用普通满二叉树层数公式，或者把 115 直接除以 2，都会得到错误结果。

## 第11题

关于格雷编码（Gray Code），下列说法正确的是（ ）。

A. 格雷编码中，编码位数越多，相邻编码之间变化的位数也越多

B. 格雷编码中，相邻两个编码的二进制位恰好有一位不同

C. 格雷编码就是把普通二进制编码按位取反后得到的结果

D. 格雷编码不能用于数字电路和状态转换的设计中

答案：B

解析：格雷编码的核心性质是相邻两个编码之间只有一位二进制位不同，因此 B 正确。这一性质与编码总位数无关，不是位数越多相邻变化位数越多，所以 A 错。格雷编码不是简单按位取反，C 错。它常用于减少状态转换时多位同时变化带来的风险，可用于数字电路和状态转换设计，D 错。

## 第12题

给定一棵二叉树，采用广度优先搜索 (BFS) 算法，返回右视图所有节点的值。其中右视图定义为：二叉树的右视图是从树的右侧看过去时可见的节点集合，即右视图中的每个节点都是某一层中最右侧的节点。

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x): val(x), left(nullptr), right(nullptr) {}
};

vector<int> rightSideView(TreeNode* root) {
    unordered_map<int, int> rightmostValueAtDepth;
    int max_depth = -1;

    queue<TreeNode*> nodeQueue;
    queue<int> depthQueue;
    nodeQueue.push(root);
    depthQueue.push(0);

    while (!nodeQueue.empty()) {
        TreeNode* node = nodeQueue.front(); nodeQueue.pop();
        int depth = depthQueue.front(); depthQueue.pop();

        if (node != NULL) {
            max_depth = max(max_depth, depth);

            rightmostValueAtDepth[depth] = node->val;

            nodeQueue.push(node->left);
            nodeQueue.push(node->right);

            depthQueue.push(________);
            depthQueue.push(________);
        }
    }

    vector<int> rightView;
    for (int depth = 0; ________; ++depth) {
        rightView.push_back(rightmostValueAtDepth[depth]);
    }
    return rightView;
};
```

A.

```cpp
depth
depth
depth < max_depth
```

B.

```cpp
depth + 1
depth + 1
depth <= max_depth
```

C.

```cpp
depth + 1
depth + 1
depth < max_depth
```

D.

```cpp
depth
depth
depth <= max_depth
```

答案：B

解析：左右孩子都位于当前节点的下一层，所以两次压入 `depthQueue` 的值都应为 `depth + 1`。由于根节点深度从 0 开始，`max_depth` 表示实际存在的最大层号，收集答案时必须遍历 `0` 到 `max_depth`，条件应为 `depth <= max_depth`。因此 B 正确。A 和 D 把孩子深度仍记为当前层，会导致层次信息错误。C 的前两空正确，但 `depth < max_depth` 会漏掉最深一层的右视图节点。

## 第13题

下列关于树的深度优先搜索（DFS）的说法中，正确的是（ ）。

A. 对树进行 DFS 时，一定是按层从上到下依次访问结点

B. 对任意一棵树进行 DFS，得到的遍历序列唯一

C. 对一棵树进行 DFS 时，常借助递归或栈实现

D. DFS 只能用于二叉树，不能用于普通树

答案：C

解析：DFS 会沿着一条分支尽量深入，再回溯访问其他分支，常用递归或显式栈实现，所以 C 正确。A 描述的是按层访问，更符合 BFS。B 错在 DFS 的结果与孩子访问顺序、先序或后序等具体规则有关，并不总是唯一。D 错在 DFS 不限于二叉树，普通树、图等结构也可以使用。

## 第14题

小朋友们去邻里拜年，每个家里有不同数量的糖果。规则是：不能连续进入两个相邻的房子（即不能同时取相邻两家的糖果）。目标是拿到最多糖果。以下是代码实现，请补全横线。

```cpp
int visit(vector<int>& nums) {
    if (nums.empty()) {
        return 0;
    }
    int size = nums.size();
    if (size == 1) {
        return nums[0];
    }
    vector<int> dp = vector<int>(size, 0);
    dp[0] = nums[0];
    dp[1] = max(nums[0], nums[1]);

    for (int i = 2; i < size; i++) {
        dp[i] = ______;  // 在此处填写代码
    }

    return dp[size - 1];
}
```

A. `dp[i] = dp[i - 1] + nums[i];`

B. `dp[i] = max(dp[i - 1], dp[i - 2] * nums[i]);`

C. `dp[i] = max(dp[i - 1], dp[i - 2] + nums[i]);`

D. `dp[i] = dp[i - 2] + nums[i];`

答案：C

解析：设 `dp[i]` 表示考虑到第 `i` 个房子时能拿到的最多糖果。对第 `i` 个房子只有两种合理选择：不进入它，则最多为 `dp[i - 1]`；进入它，则不能进入相邻的第 `i - 1` 个房子，最多为 `dp[i - 2] + nums[i]`。取两者最大值，得到 `dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])`，选 C。A 会把相邻房子都计入，违反规则。B 使用乘法没有题意依据。D 强制进入当前房子，可能错过“不进入当前房子更优”的情况。

## 第15题

元宵节晚上，小朋友沿着一条发光石板路前进，每次可向前走 1 块或 2 块石板。动态规划定义如下：`dp[i] = dp[i - 1] + dp[i - 2]`，下面关于 `dp[i]` 的含义最合适的是（ ）。

A. 走到第 `i` 块石板的不同走法数量

B. 走到第 `i` 块石板时，已经走过的石板总数

C. 从第 `i` 块石板走回起点的最少步数

D. 从第 `i` 块石板走回起点的最大步数

答案：A

解析：每次可以走 1 块或 2 块。若最后一步走 1 块，则前一状态是第 `i - 1` 块；若最后一步走 2 块，则前一状态是第 `i - 2` 块。因此到达第 `i` 块的不同走法数量等于到达第 `i - 1` 块和第 `i - 2` 块的走法数量之和，正对应 `dp[i] = dp[i - 1] + dp[i - 2]`，选 A。B 只是石板数量统计，不会满足这种递推。C、D 讨论最少或最大步数，通常应使用取最小值或最大值的转移，而不是两个方案数相加。
