## 第1题

小明的妈妈最近刚刚给他买了一块电话手表，除了可以看时间，小明也可以用它和妈妈打电话、收发信息，那么可以推测这块手表中装有一款特定操作系统。

答案：正确

解析：本题判断为正确。电话手表不仅显示时间，还能打电话、收发信息，这些功能需要由软件统一管理硬件、通信、界面和应用。这样的设备通常会运行专门的操作系统或嵌入式系统，所以可以推测其中装有特定操作系统。

## 第2题

C++表达式 `4 % 2` 和 `2 * 2 % 2` 的结果相同。

答案：正确

解析：本题判断为正确。`4 % 2` 表示 4 除以 2 的余数，结果是 0。表达式 `2 * 2 % 2` 先算乘法得到 4，再算 `4 % 2`，结果也是 0。因此两个表达式结果相同。

## 第3题

下面C++代码段成功执行后将输出 `0`。

```cpp
for (i = 1; i < 10; i++)
    if (i % 3 == 0)
        break;
cout << i;
```

答案：错误

解析：本题判断为错误。题干既然说“代码段成功执行后”，就默认 `i` 已在题目上下文中正确定义，并且循环后仍可输出它的值。在这个前提下，循环从 `i = 1` 开始，`i = 1` 和 `i = 2` 时都不能被 3 整除，不会 `break`；当 `i = 3` 时，`i % 3 == 0` 成立，执行 `break` 跳出循环。此时 `i` 的值是 3，所以输出 3，不是 0。

## 第4题

下面能够正常执行的C++代码段用于求1到N之和，N为正整数。因为 `i < N + 1`，所以是1到N且包含N之和。

```cpp
tnt = 0;
cout << "请输入正整数：";
cin >> N;
total = 0;
for (i =1; i < N + 1; i++) ;
    total += i;
cout << total;
```

答案：错误

解析：本题判断为错误。`for (i =1; i < N + 1; i++)` 后面紧跟了一个分号，这个分号就是空循环体，真正循环时什么都不加。循环结束后，`i` 已经变成 `N + 1`，随后 `total += i;` 只执行一次，所以输出的不是 `1 + 2 + ... + N`。常见误区是只看循环条件，没有注意到 `for` 后面多出的分号。

## 第5题

执行下面的C++代码段，其语句 `cout << (N)` 将被执行0次或无数次（即死循环）。

```cpp
cin >> N;
while (N)
    cout << (N);
```

答案：正确

解析：本题判断为正确。`while (N)` 会在 `N` 非0时反复执行循环体，但循环体中只输出 `N`，没有修改 `N` 的值。如果输入的 `N` 是 0，循环体执行0次；如果输入的 `N` 非0，条件会一直为真，`cout << (N)` 会不断执行，形成死循环。

## 第6题

下面的C++代码段的变量都是整型，它能用于判断输入的正整数是否为对称数。所谓对称数是指从左到右和从右到左读该数，其值相同。例如，`121` 和 `414` 都是对称数，而 `123` 不是对称数。（ ）

```cpp
cout << "请输入正整数:";
cin >> n;

old_number = n;
new_number = 0;
while (n != 0){
    new_number = new_number * 10 + n % 10;
    n /= 10;
}

if (old_number == new_number)
    cout << "对称数";
else
    cout << "非对称数";
```

答案：正确

解析：本题判断为正确。程序用 `old_number` 保存原数，然后在循环中每次取出 `n % 10` 这个最低位，把它接到 `new_number` 的末尾，再用 `n /= 10` 去掉最低位。循环结束后，`new_number` 就是原数反过来得到的数。若原数和反转后的数相等，说明从左到右和从右到左读相同，可以判断为对称数。

## 第7题

执行下面的C++代码段，如果变量都为整型变量，输入为大于0的整数，则输出数值一定为 `-N` 的值。

```cpp
cin >> N;
total = 0;
for (i = -N; i < N; i +=2)
    total += i;
cout << total;
```

答案：正确

解析：本题判断为正确。循环累加的是 `-N, -N + 2, -N + 4, ...`，一直到小于 `N` 的最后一个同奇偶数。若 `N = 5`，累加 `-5, -3, -1, 1, 3`，结果为 `-5`；若 `N = 4`，累加 `-4, -2, 0, 2`，结果为 `-4`。这些数可以前后配对相消，最后总和正好是 `-N`。

## 第8题

执行C++语句 `printf("%d\n", 3.14)` 将报错。

答案：错误

解析：本题判断为错误。`%d` 是按整数格式输出，而 `3.14` 是浮点数，格式符和实参类型不匹配。这种写法不正确，结果也不可靠；有的编译环境会给出警告，某些更严格的设置下甚至可能直接拒绝通过。但脱离具体工具链，不能把它绝对化地说成一定“将报错”。一级题里要区分“类型不匹配、结果不可靠”和“程序必然报错”。

## 第9题

执行下面的C++代码后将输出 `2500`。

```cpp
int cnt = 0;
for (int i = 1; i < 100; i++)
    cnt += i++;
cout << cnt;
```

答案：正确

解析：本题判断为正确。循环体中的 `cnt += i++` 会先把当前 `i` 加入 `cnt`，然后让 `i` 自增一次；每轮结束时，`for` 语句自己的 `i++` 还会再执行一次。因此真正被累加的是 `1, 3, 5, ..., 99` 这些奇数。前 50 个正奇数的和是 $50^2 = 2500$，所以输出 2500。

## 第10题

小明在测试C++的 `printf` 的功能时执行了 `printf("%-5d\n", 314)`，则代码输出的结果是 `-5314`。

答案：错误

解析：本题判断为错误。`%-5d` 中的 `-` 不是要输出负号，而是表示在宽度为 5 的位置内左对齐输出整数。`314` 占 3 个字符，宽度 5 还剩 2 个空格，所以输出效果是 `314` 后面跟两个空格，再换行，不是 `-5314`。
