# 判断题

## 第1题

题干：有一个存储了 $n$ 个整数的线性表，分别用数组和单链表两种方式实现。在已知下标（或结点指针）的前提下，数组的随机访问是 $O(1)$，而在链表中已知某结点的指针时，在该结点之后插入一个新结点的操作也是 $O(1)$。

答案：A

解析：本题判断为正确。数组支持通过下标直接计算元素地址，因此随机访问某个位置的时间复杂度是 $O(1)$。单链表如果已经给出了某个结点的指针，要在它后面插入新结点，只需要修改新结点的 `next` 和该结点的 `next`，不需要从头遍历链表，所以时间复杂度也是 $O(1)$。

## 第2题

题干：若数组 `a` 已按升序排列，则下面代码可以正确实现“在 `a` 中查找第一个大于等于 `x` 的元素的位置”。

```cpp
int lowerBound(vector<int>& a,int x){
    int l=0, r=a.size();
    while(l < r) {
        int mid = (l + r) / 2;
        if( a[mid] >= x) r = mid;
        else l = mid + 1;
    }
    return l;
}
```

答案：A

解析：本题判断为正确。代码维护的是左闭右开区间 `[l, r)`：当 `a[mid] >= x` 时，答案可能是 `mid` 或更靠左的位置，因此令 `r = mid`；否则 `a[mid] < x`，`mid` 及其左侧都不可能是第一个大于等于 `x` 的位置，因此令 `l = mid + 1`。循环结束时 `l == r`，正好是第一个满足 `a[pos] >= x` 的下标；如果所有元素都小于 `x`，则返回 `a.size()`，也符合 lower_bound 的语义。

## 第3题

题干：快速排序只要每次都选取中间元素作为枢轴，就一定是稳定排序。

答案：B

解析：本题判断为错误。排序算法是否稳定，关键看相等元素的相对次序是否一定保持不变，而不是只由枢轴选取位置决定。普通快速排序在划分和交换元素时，可能把相等关键字的元素交换到彼此前后顺序相反的位置，所以通常不是稳定排序。正确说法是：选取中间元素作枢轴可能影响快速排序的划分效果和时间表现，但不能保证快速排序稳定。

## 第4题

题干：若某算法满足递推式：$T(n)=2T(n/2)+O(n)$，则其时间复杂度为 $O(n\log n)$。

答案：A

解析：本题判断为正确。该递推式表示每层把规模为 $n$ 的问题分成两个规模为 $n/2$ 的子问题，并且本层合并或处理代价为 $O(n)$。递归树每一层总代价都是 $O(n)$，规模每次减半，共有 $O(\log n)$ 层，因此总时间复杂度为 $O(n\log n)$。

## 第5题

题干：在一个数组中，如果两个元素 `a[i]` 和 `a[j]` 满足 `i < j` 且 `a[i] > a[j]`，则 `a[i]` 和 `a[j]` 是一个逆序对。下面代码可以正确统计数组 `a` 区间 `[l,r]` 内的逆序对总数。

```cpp
long long cnt=0;
void merge_count(vector<int>& a, int l, int m, int r){
    int i = l, j = m + 1;
    while(i <= m && j <= r) {
        if(a[i] <= a[j]) i++;
        else {
            cnt += (m - i + 1);
            j++;
        }
    }
}
```

答案：B

解析：本题判断为错误。题目前半句对逆序对的定义是正确的，但下面这段代码不能正确统计整个区间 `[l,r]` 的逆序对总数。`cnt += (m - i + 1)` 这一步只适用于左右两段 `[l,m]`、`[m+1,r]` 都已经分别有序时，用来统计跨越左右两段的逆序对。完整的归并统计还需要递归统计左半段、右半段内部的逆序对，并在统计后把两段真正合并成有序区间；这段代码既没有递归统计内部逆序对，也没有完成归并，因此不能直接统计 `[l,r]` 内的逆序对总数。

## 第6题

题干：唯一分解定理保证：若一个数未被任何不超过其平方根的质数筛去，则它一定是质数。

答案：A

解析：本题判断为正确。对大于 1 的整数，如果它是合数，就可以写成两个大于 1 的因子相乘，其中至少有一个因子不超过它的平方根；这个因子还会包含某个不超过平方根的质因子。因此，一个数如果没有被任何不超过其平方根的质数整除，就不可能是合数，只能是质数。这也是试除法和筛法中只需要检查到平方根的原因。

## 第7题

题干：假设数组 `a` 的值域范围是 $D$，以下程序的时间复杂度是 $O(n\log n+n\log D)$。

```cpp
bool check(int n, int a[], int k, int dist) {
    int cnt = 1;
    int last = a[0];

    for (int i = 1; i < n; i++) {
        if (a[i] - last >= dist) {
            cnt++;
            last = a[i];
        }
    }

    return cnt >= k;
}

int solve(int n, int a[], int k) {
    std::sort(a, a + n);

    int l = 0;
    int r = a[n - 1] - a[0];

    while (l < r) {
        int mid = (l + r + 1) / 2;

        if (check(n, a, k, mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}

int main() {
    int a[] = {1, 2, 8, 4, 9};
    int n = 5;
    int k = 3;

    std::cout << solve(n, a, k) << std::endl;

    return 0;
}
```

答案：A

解析：本题判断为正确。程序先调用 `std::sort(a, a + n)` 排序，时间复杂度为 $O(n\log n)$。随后在距离范围 `[0, a[n-1]-a[0]]` 上二分，数组值域范围为 $D$ 时，二分轮数为 $O(\log D)$。每一轮二分都调用一次 `check`，而 `check` 只线性扫描数组一次，时间复杂度为 $O(n)$。因此二分检查部分为 $O(n\log D)$，总时间复杂度为 $O(n\log n+n\log D)$。

## 第8题

题干：若一个问题满足最优子结构性质，则一定可以用贪心算法得到最优解。

答案：B

解析：本题判断为错误。最优子结构只说明一个问题的最优解可以由子问题的最优解组成，它是动态规划和许多递归优化问题常见的性质。贪心算法还需要更强的条件，也就是每一步做出的局部最优选择能够保证最终全局最优，通常称为贪心选择性质。很多问题有最优子结构但不能用贪心求最优解，例如一般的 0/1 背包问题。因此题目把“最优子结构”直接推出“一定可以贪心”是错误的。

## 第9题

题干：线性筛相比埃氏筛的核心改进在于：埃氏筛中一个合数可能被多个质数重复标记，线性筛通过“每个合数只被其最大质因子筛去”的策略，保证每个合数恰好被标记一次，从而实现 $O(n)$ 的时间复杂度。

答案：B

解析：本题判断为错误。线性筛确实通过让每个合数只被标记一次来避免埃氏筛中的重复标记，从而达到 $O(n)$ 时间复杂度；但题干中“被其最大质因子筛去”的说法不对。标准线性筛中，合数通常是在遇到它的最小质因子时被标记，例如用当前数 `i` 乘以不超过 `i` 的最小质因子的质数来生成合数，并在 `i % p == 0` 时停止继续枚举质数。正确说法应是：线性筛保证每个合数按其最小质因子对应的方式被标记一次，而不是最大质因子。

## 第10题

题干：任何递归程序都可以改写为等价的非递归程序，但改写后的非递归程序一定需要显式地使用栈来模拟递归调用过程。

答案：B

解析：本题判断为错误。递归调用本质上可以用程序状态和控制流程来模拟，因此递归程序通常可以改写为等价的非递归程序。但是“改写后一定需要显式地使用栈”说得过于绝对。某些尾递归或简单线性递归可以直接改写成循环，只用若干变量保存当前状态，不需要显式栈；只有当需要保存多层调用现场、回溯分支或后续执行位置时，才常常需要手动使用栈来模拟调用过程。
