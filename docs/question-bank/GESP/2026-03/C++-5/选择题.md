## 第1题

关于单链表、双链表和循环链表，下列说法正确的是（ ）。

A. 在单链表中，若已知任意结点的指针，则可以在 `O(1)` 时间内删除该结点。

B. 循环链表中一定不存在空指针。

C. 在循环双链表中，尾结点的 `next` 指针一定为 `nullptr`。

D. 在带头结点的循环单链表中，判定链表是否为空只需判断头结点的 `next` 是否指向自身。

答案：D

解析：在带头结点的循环单链表中，空表没有数据结点，头结点的 `next` 会指回头结点本身，因此 D 正确。A 忽略了单链表删除结点通常需要找到前驱结点，仅知道任意结点指针不能保证都能在 `O(1)` 内完成删除，尤其是尾结点。B 说“一定不存在空指针”过于绝对，例如空循环链表可能用空头指针表示。C 与循环双链表定义相反，尾结点的 `next` 应指向头结点或首结点，而不是 `nullptr`。

s -> next = p;

## 第4题

对如下代码实现的欧几里得算法（辗转相除法），执行 `gcd(48, 18)` 得到的调用序列为（ ）。

```cpp
int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
}
```

A.

```text
gcd(48,18) -> gcd(18,12) -> gcd(12,6) -> gcd(6,0)
```

B.

```text
gcd(48,18) -> gcd(30,18) -> gcd(12,18)
```

C.

```text
gcd(48,18) -> gcd(18,30) -> gcd(30,6)
```

D.

```text
gcd(48,18) -> gcd(12,18) -> gcd(6,12)
```

答案：A

解析：递归调用规则是 `gcd(a, b)` 在 `b != 0` 时转为 `gcd(b, a % b)`。因此 `gcd(48,18)` 的下一步是 `gcd(18, 48 % 18)`，也就是 `gcd(18,12)`；之后依次为 `gcd(12,6)`、`gcd(6,0)`。当第二个参数为 0 时返回，所以 A 正确。其他选项把减法、参数顺序或取余结果混在了一起。

## 第6题

埃氏筛中将内层循环从 `j = i*i` 开始而不是 `j = 2*i` 的主要原因是（ ）。

```cpp
vector<int> eratosthenes_sieve(int n) {
    vector<bool> is_composite(n + 1, false);
    vector<int>  primes;

    for (int i = 2; i <= n; i++) {
        if (is_composite[i]) continue;

        primes.push_back(i);
        for (long long j = (long long)i * i; j <= n; j += i)
            is_composite[j] = true;
    }
    return primes;
}
```

A. 因为 `2*i` 一定不是合数

B. `i*i` 一定是质数

C. 小于 `i*i` 的 `i` 的倍数已被更小质因子筛过

D. 这样可以把时间复杂度降为 `O(n)`

答案：C

解析：当处理质数 `i` 时，小于 `i*i` 的 `i` 的倍数都形如 `k*i`，其中 `k < i`，它们已经在处理更小因子 `k` 或其质因子时被标记过，因此可以从 `i*i` 开始。A 错在 `2*i` 通常就是合数。B 错在 `i*i` 是大于 1 的平方数，除 `i=1` 外不可能是质数，而筛法中 `i` 从 2 开始。D 混淆了埃氏筛和线性筛，埃氏筛这样优化后也不是 `O(n)`。

## 第7题

下面程序的运行结果为（ ）。

```cpp
bool check(int n, int a[], int k, int dist) {
    int cnt = 1;
    int last = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] - last >= dist) {
            cnt++;
            last = a[i];
        }
    }
    return cnt >= k;
}

int solve(int n, int a[], int k) {
    std::sort(a, a + n);
    int l = 0;
    int r = a[n - 1] - a[0];

    while (l < r) {
        int mid = (l + r + 1) / 2;

        if (check(n, a, k, mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}

int main() {
    int a[] = {1, 2, 8, 4, 9};
    int n = 5;
    int k = 3;

    std::cout << solve(n, a, k) << std::endl;
    return 0;
}
```

A. 2

B. 3

C. 4

D. 5

答案：B

解析：数组排序后为 `{1, 2, 4, 8, 9}`。`check` 判断能否选出至少 3 个数，使相邻被选数的距离不小于 `dist`。当 `dist = 3` 时，可以选 `1, 4, 8`，满足要求；当 `dist = 4` 时，从 1 开始最多选到 `1, 8` 或 `1, 9`，无法选出 3 个。因此最大可行距离是 3，程序输出 3，选 B。

## 第9题

关于递归函数调用，下列说法错误的是（ ）。

A. 递归调用层次过深时，可能会耗尽栈空间导致栈溢出

B. 尾递归函数可以通过编译器优化来避免栈溢出

## 第11题

下面代码用分治求“最大连续子段和”，其时间复杂度为（ ）。

```cpp
int solve(vector<int>& a, int l, int r){
    if(l == r) return a[l];

    int mid = l + (r - l) / 2;

    int left = solve(a, l, mid);
    int right = solve(a, mid + 1, r);

    int sum = 0, lmax = INT_MIN;
    for(int i = mid; i >= l; i--){
        sum += a[i];
        lmax = max(lmax, sum);
    }

    sum = 0;
    int rmax = INT_MIN;
    for(int i = mid + 1; i <= r; i++){
        sum += a[i];
        rmax = max(rmax, sum);
    }

    return max({left, right, lmax + rmax});
}
```

A. `O(n^2)`

B. `O(n log n)`

C. `O(log n)`

D. `O(n)`

答案：B

解析：函数每次把区间分成左右两半，递归求左半和右半的最大子段和；合并时又从中点向两侧各扫一遍，合并代价与当前区间长度成正比。因此递推式可看作 `T(n) = 2T(n/2) + O(n)`，时间复杂度为 `O(n log n)`。A 高估了分治合并的代价，C 只看到了递归层数，D 忽略了每一层都要做线性合并。

## 第12题

游戏大赛决赛，两组选手分别按得分从小到大排好队，现在要把他们合并成一个有序排行榜。

A 组：`A = {12, 35, 67, 89}`，B 组：`B = {20, 45, 55, 78}`，下面是归并合并函数的核心循环，横线处应填入（ ）。

```cpp
int i = 0, j = 0;
vector<int> result;

while (i < A.size() && j < B.size()) {
    if (___________________) {
        result.push_back(A[i++]);
    } else {
        result.push_back(B[j++]);
    }
}

while (i < A.size()) {
    result.push_back(A[i++]);
}
while (j < B.size()) {
    result.push_back(B[j++]);
}
```

A. `A[i] >= B[j]`

B. `A[i] <= B[j]`

C. `i >= j`

D. `i <= j`

答案：B

解析：两个数组都已经升序排列，归并时应每次把当前较小的元素放入结果。如果 `A[i] <= B[j]`，就先放 `A[i]`，否则放 `B[j]`，这样结果仍然升序，所以 B 正确。A 会优先放较大的元素，破坏升序。C、D 比较的是下标大小，不是选手得分大小，不能决定哪个元素应先进入结果。

## 第13题

有 `n` 位同学的成绩已经从小到大排好序，现在对它执行下面这段以第一个元素为 `pivot` 的快速排序，请问此次排序的时间复杂度是（ ）。

```cpp
void quicksort(vector<int>& a, int l, int r) {
    if (l >= r) return;
    int pivot = a[l];
    int i = l, j = r;
    while (i < j) {
        while (i < j && a[j] >= pivot) j--;
        while (i < j && a[i] <= pivot) i++;
        if (i < j) swap(a[i], a[j]);
    }
    swap(a[l], a[i]);
    quicksort(a, l, i - 1);
    quicksort(a, i + 1, r);
}
```

A. `O(n)`

B. `O(n log n)`

C. `O(n^2)`

D. `O(log n)`

答案：C

解析：数组已经升序，而代码每次都选当前区间第一个元素作为 `pivot`，这个 `pivot` 正好是当前区间最小值。划分后左区间为空，右区间规模只减少 1，递归会退化成处理 `n, n-1, n-2, ...` 的区间。每层划分仍要线性扫描当前区间，因此总时间为 `O(n^2)`。B 是快速排序平均情况下常见复杂度，但不适用于这里给定的有序输入和固定首元素 pivot。

## 第14题

下面关于排序算法的描述中，不正确的是（ ）。

A. 冒泡排序和插入排序都是稳定的排序算法

B. 快速排序和归并排序都是不稳定的排序算法

C. 冒泡排序和插入排序最好时间复杂度均为 `O(n)`

D. 归并排序在最好、最坏和平均三种情况的时间复杂度均为 `O(n log n)`

答案：B

解析：快速排序在划分和交换元素时可能改变相等元素的相对顺序，因此一般不是稳定排序；归并排序在合并两个有序段时若相等元素先取左段元素，就能保持相等元素的原有顺序，因此可以做到稳定。所以“快速排序和归并排序都是不稳定的排序算法”不正确，选 B。A 正确，冒泡排序和插入排序在相等元素不乱交换时是稳定的。C 正确，冒泡排序带提前结束标记、插入排序遇到已有序数组时都可达到 `O(n)`。D 正确，归并排序每层合并总量为 `O(n)`，层数为 `O(log n)`，三种情况都是 `O(n log n)`。
