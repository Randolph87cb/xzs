## 第1题

假设一个算法时间复杂度的递推式是 $T(n)=2T(n-1)+1$（$n$ 为正整数），且 $T(0)=1$，那么这个算法的时间复杂度是（ ）。

A. $O(n)$
B. $O(n log n)$
C. $O(n^2)$
D. $O(2^n)$

答案：D

解析：把递推式逐层展开：$T(n)=2T(n-1)+1=2^2T(n-2)+2+1=...=2^nT(0)+(2^n-1)=2^{n+1}-1$，所以时间复杂度是 $O(2^n)$。A、B、C 都是多项式或准线性级别，增长速度远小于这个递推式产生的指数级增长。

## 第2题

下面关于“唯一分解定理”和“素数筛法”的说法中，错误的是（ ）。

A. 如果预处理出 $n$ 以内每个数的最小质因子，那么可以在 $O(log n)$ 时间内完成任意一个不超过 $n$ 的整数的质因数分解。
B. 线性筛（欧拉筛）能够保证每个合数只被其最小质因子筛掉一次，这一性质依赖于唯一分解定理。
C. 唯一分解定理保证：若一个数未被任何不超过其平方根的质数筛去，则它一定是质数。
D. 唯一分解定理是埃氏筛时间复杂度为 $O(n log log n)$ 的根本原因。

答案：D

解析：埃氏筛的 $O(n log log n)$ 来自对各个质数标记倍数时的总工作量估计，也就是与素数倒数和有关，并不是由唯一分解定理直接推出的，所以 D 错。A 中每次用最小质因子把当前数除小，最多除出 $O(log n)$ 个质因子；B 是线性筛“一数只被筛一次”的核心依据；C 是试除判质的基本结论，如果合数 $x$ 存在分解 $x=ab$，必有一个因子不超过 $sqrt(x)$。

## 第3题

若字符串 $A$ 与字符串 $B$ 的最长公共子序列（LCS）长度为 5，则（ ）。

A. 它们的编辑距离为 5
B. 它们至少有 5 个公共字符
C. 它们最长公共子串长度为 5
D. 它们一定长度相等

答案：B

解析：LCS 长度为 5，说明两个字符串中可以各选出 5 个位置，按相同顺序组成同一个长度为 5 的子序列，因此至少有 5 个公共字符。A 不成立，编辑距离还和插入、删除、替换的代价及字符串总长度有关；C 不成立，公共子序列不要求连续，而公共子串要求连续；D 也不成立，例如一个较长字符串和一个较短字符串仍可能有长度为 5 的 LCS。

## 第4题

对于一棵包含 $n$ 个顶点（$n >= 2$）的树，其所有顶点的度数之和必定等于（ ）。

A. $n-1$
B. $2n-2$
C. $2n$
D. $n^2$

答案：B

解析：树有 $n$ 个顶点时必有 $n-1$ 条边。无向图中每条边会给两个端点的度数各贡献 1，所以所有顶点度数之和为 $2(n-1)=2n-2$。A 只数了边数，没有把一条边对两个端点的贡献都算进去；C 多算了 2；D 与树的边数性质无关。

## 第5题

关于哈希表（Hash Table）在不考虑扩容且采用简单均匀哈希函数的前提下，下列说法中错误的是（ ）。

A. 装载因子越大，发生冲突的概率通常越高
B. 开放定址法在删除元素时实现相对复杂
C. 链地址法在最坏情况下查找时间复杂度为 $O(n)$
D. 查找哈希表的时间复杂度总是 $O(1)$

答案：D

解析：哈希表查找通常说的是平均或期望 $O(1)$，并不是“总是” $O(1)$。如果大量元素发生冲突，链地址法可能退化为在一条链上顺序查找，最坏为 $O(n)$，所以 C 正确。装载因子越大，桶或探测位置越拥挤，冲突概率通常越高，A 正确；开放定址法删除时要处理探测链不能被直接断开的问题，常需懒删除标记，B 正确。

## 第6题

在 Kruskal 算法中，会将边排序后按顺序扫描选取边加入最小生成树中，算法的本质思想是（ ）。

A. 分治
B. 贪心
C. 动态规划
D. 回溯

答案：B

解析：Kruskal 每一步都选择当前能连接两个不同连通块的最小边，这是典型的“当前最优选择”策略，依靠最小生成树的割性质保证整体最优，因此本质是贪心。分治强调拆分子问题后合并，动态规划强调状态转移和重叠子问题，回溯强调尝试与撤销，它们都不是 Kruskal 的核心思想。

## 第7题

下面程序的运行结果为（ ）。

```cpp
#include <iostream>
#include <algorithm>

bool check(int n, int a[], int k, int dist) {
    int cnt = 1;
    int last = a[0];

    for (int i = 1; i < n; i++) {
        if (a[i] - last >= dist) {
            cnt++;
            last = a[i];
        }
    }

    return cnt >= k;
}

int solve(int n, int a[], int k) {
    std::sort(a, a + n);

    int l = 0;
    int r = a[n - 1] - a[0];

    while (l < r) {
        int mid = (l + r + 1) / 2;

        if (check(n, a, k, mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}

int main() {
    int a[] = {1, 2, 8, 4, 9};
    int n = 5;
    int k = 3;

    std::cout << solve(n, a, k) << std::endl;

    return 0;
}
```

A. 2
B. 3
C. 4
D. 5

答案：B

解析：程序先把数组排成 $1,2,4,8,9$，然后二分最大的最小间距。若最小间距为 3，可以选 $1,4,8$，已经能选出 3 个数；若最小间距为 4，从 1 出发最多选到 8，再无法选出第 3 个数，因此不可行。最大可行距离是 3，输出为 3。A 太小但不是最大值，C、D 都不可行。

## 第8题

下面程序的时间复杂度是（ ），假设数组 $a$ 的值域范围是 $D$。

```cpp
#include <iostream>
#include <algorithm>

bool check(int n, int a[], int k, int dist) {
    int cnt = 1;
    int last = a[0];

    for (int i = 1; i < n; i++) {
        if (a[i] - last >= dist) {
            cnt++;
            last = a[i];
        }
    }

    return cnt >= k;
}

int solve(int n, int a[], int k) {
    std::sort(a, a + n);

    int l = 0;
    int r = a[n - 1] - a[0];

    while (l < r) {
        int mid = (l + r + 1) / 2;

        if (check(n, a, k, mid))
            l = mid;
        else
            r = mid - 1;
    }

    return l;
}

int main() {
    int a[] = {1, 2, 8, 4, 9};
    int n = 5;
    int k = 3;

    std::cout << solve(n, a, k) << std::endl;

    return 0;
}
```

A. $O(n log n+n log D)$
B. $O(n log n log D)$
C. $O(n log n)$
D. $O(n log D)$

答案：A

解析：排序部分需要 $O(n log n)$。之后二分答案，距离范围不超过值域范围 $D$，二分次数是 $O(log D)$；每次调用 `check` 都要线性扫描数组，耗时 $O(n)$，所以二分检查总耗时 $O(n log D)$。两部分相加为 $O(n log n+n log D)$。B 把两部分误乘了；C 忽略了二分检查；D 忽略了排序。

## 第9题

某二叉树共有 10 个结点，记为 A~J，已知它的先序遍历序列为：A B D H I E C F J G，中序遍历序列为：H D I B E A F J C G，则该二叉树的后序遍历序列是（ ）。

A. H I D E B J F G C A
B. H I D B E J F G C A
C. I H D E B J F G C A
D. H I D E B F J G C A

答案：A

解析：先序第一个结点 A 是根。中序中 A 左边为 H D I B E，右边为 F J C G。左子树先序为 B D H I E，根为 B；其中 D 的左右孩子分别为 H、I，B 的右孩子为 E，所以左子树后序为 H I D E B。右子树根为 C，中序左边是 F J、右边是 G；F 的右孩子是 J，所以右子树后序为 J F G C。合并左右子树和根，后序为 H I D E B J F G C A。B 把 B 提前了，C 交换了 H、I，D 把 F、J 的后序顺序写反。

## 第10题

下面哪一个可能是下图的深度优先遍历序列（ ）。根据 PDF 图，图中有向边可描述为：$1->4$、$1->5$、$1->7$、$2->1$、$2->5$、$2->9$、$3->1$、$3->2$、$4->7$、$5->4$、$5->8$、$6->3$、$8->4$、$8->7$、$8->9$、$9->6$。

A. 1, 5, 4, 8, 7, 9, 6, 3, 2
B. 1, 5, 8, 4, 7, 9, 6, 3, 2
C. 2, 5, 8, 7, 9, 6, 3, 4, 1
D. 8, 9, 6, 3, 2, 5, 1, 4, 7

答案：B

解析：从 1 开始，若按 $1->5$、$5->8$、$8->4$、$4->7$ 的顺序深入，7 无后继后回溯到 8；再走 $8->9->6->3->2$，且 2 的后继 1、5、9 都已经访问过，就能得到 B 的序列。A 中到达 4 后，4 还有未访问的后继 7，不能马上回到 5 再访问 8；C 中访问到 3 后，若还有未访问的 1，则下一步不能直接到 4；D 中 3 到 2 后可以到 5，但 5 没有到 1 的边，因此不可能紧接着访问 1。

## 第11题

下面这个有向图的强连通分量的个数是（ ）。根据 PDF 图，边可描述为：$1->2$、$2->6$、$3->2$、$3->4$、$4->3$、$4->7$、$5->1$、$6->5$、$6->9$、$7->8$、$8->7$、$9->10$、$10->11$、$10->12$、$11->9$、$11->15$、$12->6$、$12->7$、$12->13$、$13->14$、$14->15$、$14->16$、$15->13$、$16->15$。

A. 3
B. 4
C. 5
D. 6

答案：B

解析：强连通分量要求分量内任意两点都能互相到达。图中 $3$ 和 $4$ 互相可达，构成一个分量；$7$ 和 $8$ 互相可达，构成一个分量；$13,14,15,16$ 之间可以通过 $13->14->16->15->13$ 等路径互相到达，构成一个分量。剩下 $1,2,5,6,9,10,11,12$ 也互相可达，例如 $1->2->6->9->10->12->6$，$6->5->1$，$10->11->9$，并且可由 $12->6$ 回到前面各点，所以它们是同一个强连通分量。总数为 4。A 少算了，C、D 则把同一个可互达的大分量拆开了。

## 第12题

关于泛洪算法（Flood Fill）的说法，正确的是（ ）。

A. 泛洪算法只适用于二维网格中的四连通或八连通问题。
B. 泛洪算法必须使用递归方式实现。
C. 泛洪算法本质上是对图进行一次从起点出发的搜索。
D. 泛洪算法只能用于统计连通块个数，不能用于计算面积或周长。

答案：C

解析：Flood Fill 的本质是从一个起点出发，沿允许的相邻关系做 DFS 或 BFS，把可达区域搜索出来，因此 C 正确。A 错在“只适用于”，它也可以推广到一般图或其他邻接定义；B 错在“必须”，队列 BFS、显式栈 DFS 都可以实现；D 错在用途过窄，搜索过程中可以统计格子数得到面积，也可以检查边界得到周长。

## 第13题

有 6 个字符，它们出现的次数分别为：{2, 3, 3, 4, 6, 8}，现在用哈夫曼编码为这些字符编码，最小加权路径长度 WPL（每个字符的出现次数乘以它的编码长度，再把每个字符结果加起来）的值为（ ）。

A. 58
B. 60
C. 62
D. 64

答案：D

解析：哈夫曼树每次合并当前最小的两个权值，WPL 等于每次合并产生的新权值之和。依次合并：$2+3=5$，$3+4=7$，$5+6=11$，$7+8=15$，$11+15=26$，所以 WPL 为 $5+7+11+15+26=64$。A、B、C 都少算了某些合并带来的深度增加。

## 第14题

关于单链表、双链表和循环链表，下列说法正确的是（ ）。

A. 在单链表中，若已知某结点的指针，则可以在 $O(1)$ 时间内删除该结点。
B. 循环链表中一定不存在空指针。
C. 在循环双链表中，尾结点的 next 指针一定为 NULL。
D. 在带头结点的循环单链表中，判定链表是否为空只需判断头结点的 next 是否指向自身。

答案：D

解析：带头结点的循环单链表为空时，头结点的 `next` 指回头结点自身；非空时，头结点的 `next` 指向第一个数据结点，因此 D 正确。A 不严谨，单链表删除某结点通常需要知道它的前驱，若该结点是尾结点更不能只靠当前指针完成常规删除。B 的“一定”过强，链表实现可能仍用空指针表示特殊状态或外部指针为空。C 与循环双链表定义相反，尾结点的 `next` 应指向头结点或首结点，而不是 NULL。

## 第15题

下列关于树的遍历的说法中，正确的一项是（ ）。

A. 对任意一棵树进行深度优先遍历，所得序列一定唯一。
B. 已知一棵二叉树的先序遍历和后序遍历序列，可以唯一确定这棵二叉树。
C. 已知一棵二叉树的先序遍历和中序遍历序列，可以唯一确定这棵二叉树。
D. 一棵二叉树的中序遍历序列是单调递增的，则该二叉树一定是二叉平衡树。

答案：C

解析：先序遍历能确定根结点，中序遍历能把根的左、右子树划分出来，再递归处理左右子树，因此可以唯一确定一棵二叉树，C 正确。A 错在 DFS 序列会受孩子或邻接点访问顺序影响；B 错在先序加后序通常不能唯一确定普通二叉树，例如只有一个孩子时无法判断它是左孩子还是右孩子；D 错在中序递增只能说明它可能满足二叉搜索树的访问结果，不能推出高度平衡。
