# 判断题

来源：

## 第1题

题干：下面代码执行结束时，变量 `a` 的值变成 15。

```cpp
void add10(int &x) { x += 10; }

int main() {
    int a = 5;
    add10(a);
}
```

答案：A

解析：本题判断为正确。函数 `add10` 的形参是 `int &x`，表示 `x` 是实参的引用，调用 `add10(a)` 时，函数体中的 `x += 10` 实际修改的就是变量 `a` 本身。`a` 初始为 5，加 10 后变成 15。

## 第2题

题干：引用一旦绑定某个变量，就不能再绑定其他变量。

答案：A

解析：本题判断为正确。C++ 引用在定义时必须完成初始化，例如 `int &r = a;`，之后 `r` 就一直作为 `a` 的别名使用，不能再让 `r` 改为引用另一个变量。给引用赋值只会修改它所绑定对象的值，并不会改变绑定关系。

## 第3题

题干：执行下面代码，输出结果为 5。

```cpp
int main() {
    int a[2][3];
    cout << &a[1][2] - &a[0][1] << endl;
    return 0;
}
```

答案：B

解析：本题判断为错误。二维数组 `a[2][3]` 在内存中按行连续存放，元素顺序依次是 `a[0][0]`、`a[0][1]`、`a[0][2]`、`a[1][0]`、`a[1][1]`、`a[1][2]`。从 `a[0][1]` 到 `a[1][2]` 相差 4 个 `int` 元素，因此两个同类型元素指针相减的结果是 4，不是 5。

## 第4题

题干：下面程序可以正常编译并输出 10。

```cpp
int calc(int x, int y = 10);
int calc(int x) { return x * 2; }
int calc(int x, int y) { return x * y; }

int main() {
    cout << calc(5);
}
```

答案：B

解析：本题判断为错误。这里同时存在 `calc(int)` 和带默认参数的 `calc(int, int)`。调用 `calc(5)` 时，既可以匹配单参数版本 `calc(int)`，也可以把第二个参数使用默认值来匹配 `calc(int, int)`，编译器无法确定应调用哪一个重载函数，会产生二义性错误。因此程序不能正常编译并输出 10。

## 第5题

题干：下面程序执行后输出 `2010`。

```cpp
int x = 10;
void f() { int x = 20; cout << x; }

int main() {
    f();
    cout << x;
}
```

答案：A

解析：本题判断为正确。全局变量 `x` 的值是 10；函数 `f` 内部又定义了一个局部变量 `x`，它会在函数作用域内遮蔽同名全局变量，所以 `f()` 中输出的是局部变量 20。回到 `main` 后执行 `cout << x;`，此处访问的是全局变量 10，因此连续输出为 `2010`。

## 第6题

题干：在 C++ 中，如果声明了一个指针变量但没有显式初始化，该指针会自动被初始化为 `nullptr`。

答案：B

解析：本题判断为错误。普通局部指针变量如果只写 `int *p;` 而不初始化，它的值是不确定的，并不会自动变成 `nullptr`。只有显式写成 `int *p = nullptr;`，或者对象具有静态存储期并被零初始化时，才会得到空指针值。

## 第7题

题干：下面代码没有语法错误。

```cpp
struct GameCharacter {
    string name;
    int level;
    float position_x;
    float position_y;

    struct Equipment {
        string weapon;
        int attack_bonus;
        int defense_bonus;
    } equipment;

    struct Skill {
        string name;
        int damage;
    } skills[8];
    int skill_count;
};
```

答案：A

解析：本题判断为正确。C++ 允许在一个结构体内部定义嵌套结构体类型，也允许在定义嵌套结构体后直接声明成员变量。代码中 `equipment` 是 `Equipment` 类型的成员，`skills[8]` 是包含 8 个 `Skill` 元素的数组成员，最后外层结构体用分号结束，语法上没有问题。

## 第8题

题干：下面程序能够把 `Hello` 写入 `data.txt` 文件中。

```cpp
ofstream fout("data.txt");
cout << "Hello";
fout.close();
```

答案：B

解析：本题判断为错误。第一行确实创建了输出文件流 `fout` 并打开 `data.txt`，但第二行使用的是 `cout << "Hello";`，这会把内容输出到标准输出流，而不是写入文件流。若要写入文件，应使用 `fout << "Hello";`。

## 第9题

题干：由于选择排序和插入排序的时间复杂度均为 $O(n^2)$，在任何实际场景下两者的性能表现几乎相同，可以互相替代。

答案：B

解析：本题判断为错误。时间复杂度同为 $O(n^2)$ 只说明它们在数量级上的最坏或平均增长趋势相近，不代表实际性能在所有场景下都几乎相同。插入排序在数据基本有序时移动次数少，常常接近线性表现；选择排序每一轮都要扫描未排序部分寻找最小值，比较次数相对固定。二者稳定性、数据移动次数和适用场景也不同，不能说在任何实际场景下都可以互相替代。

## 第10题

题干：下面用递推方式计算斐波那契数列第 `n` 项的程序，时间复杂度是 $O(2^n)$。

```cpp
int fib(int n) {
    if (n <= 1) return n;
    int f0 = 0, f1 = 1, cur = 0;
    for (int i = 2; i <= n; i++) {
        cur = f0 + f1;
        f0 = f1;
        f1 = cur;
    }
    return cur;
}
```

答案：B

解析：本题判断为错误。这段程序使用循环递推，从 `i = 2` 到 `i = n` 依次计算当前项，每轮只做常数次赋值和加法操作，因此循环执行次数与 `n` 成正比，时间复杂度是 $O(n)$。$O(2^n)$ 通常出现在直接递归重复计算大量子问题的斐波那契写法中，不适用于这里的递推实现。
