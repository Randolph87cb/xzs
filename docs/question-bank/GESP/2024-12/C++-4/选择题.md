## 第1题

下面的语句中，（ ）正确定义了一个计算浮点数 $x$ 的平方（$x^2 = x \times x$）的函数，并成功调用该函数。

A.
```cpp
float square(float x) {
    return x * x;
}
float area = square(2);
```

B.
```cpp
square(float x) {
    return x * x;
}
float area = square(2);
```

C.
```cpp
void square(float x) {
    return x * x;
}
area = square(2.0);
```

D.
```cpp
void square(float x) {
    x * x;
    return;
}
area = square(2);
```

答案：A

解析：要计算并返回浮点数平方，函数必须声明返回值类型，并在函数体中 `return x * x`。A 中 `float square(float x)` 返回 `float`，调用 `square(2)` 时整数 `2` 可以转换为 `float`，结果再赋给 `float area`，语法和功能都满足要求。B 缺少返回值类型，C 把函数声明为 `void` 却返回表达式且 `area` 未声明，D 的 `void` 函数没有返回平方值，也不能把调用结果赋给 `area`。

## 第2题

下面代码的描述中，正确的是（ ）。

```cpp
void n_chars(char c, int n) {
    while (n-- > 0)
        cout << c;
}

char my_char = 'w';
int times = 5;
n_chars(my_char, times);
```

A. 代码执行结束后，times 的值为 0
B. n 是形参， times 是实参
C. n 是实参， times 是形参
D. 代码最后一行换成 `n_chars(times, my_char);` 也可以

答案：B

解析：函数定义 `void n_chars(char c, int n)` 中的 `c`、`n` 是形参，调用 `n_chars(my_char, times)` 时传入的 `my_char`、`times` 是实参，所以 B 正确。这里采用值传递，函数内自减的是形参 `n` 的副本，不会把外部变量 `times` 改成 $0$，因此 A 错。C 把形参和实参说反了。D 中第一个参数需要字符、第二个参数需要整数，交换后含义和类型都不合适。

## 第3题

给定以下代码，

```cpp
void func(int& x) {
    x = x * 2;
}

int a = 5;
func(a);
```

执行上述代码后，变量 $a$ 的值为（ ）。

A. $5$
B. $10$
C. $15$
D. $20$

答案：B

解析：参数 `int& x` 是引用形参，调用 `func(a)` 后，函数体中的 `x` 就是变量 `a` 的别名。执行 `x = x * 2` 时，原来的 $5$ 被改为 $5 \times 2 = 10$，因此函数返回后 `a` 的值为 $10$。

## 第4题

运行下面代码，屏幕上输出是（ ）。

```cpp
double* p_arr = new double[3];
p_arr[0] = 0.2;
p_arr[1] = 0.5;
p_arr[2] = 0.8;
p_arr += 1;
cout << p_arr[0] << endl;
p_arr -= 1;
delete p_arr;
```

A. $0.2$
B. $0.5$
C. $1.2$
D. $1.5$

答案：B

解析：`new double[3]` 得到数组首元素地址，初始时 `p_arr[0]`、`p_arr[1]`、`p_arr[2]` 分别是 $0.2$、$0.5$、$0.8$。执行 `p_arr += 1` 后，指针移动到原数组第 $2$ 个元素处，此时新的 `p_arr[0]` 等价于原来的第二个元素，值为 $0.5$。随后 `p_arr -= 1` 只是把指针移回首地址以便释放，不影响已经输出的内容。

## 第5题

运行下面代码片段后，$x$ 和 `*p` 的结果分别是（ ）。

```cpp
int x = 20;
int* p = &x;
*p = *p + 2;
```

A. $20\ 20$
B. $20\ 22$
C. $22\ 20$
D. $22\ 22$

答案：D

解析：`p` 保存的是 `x` 的地址，`*p` 表示通过这个地址访问到的 `x` 本身。语句 `*p = *p + 2` 先读出当前的 `x = 20`，计算得到 $22$，再把 $22$ 写回 `x`。因此执行结束后 `x` 是 $22$，再次取 `*p` 也会读到同一个变量的值 $22$。

## 第6题

下面的描述中，（ ）不能正确定义一个名为 Student 的结构体以及一个包含 $20$ 个元素的结构数组。

A.
```cpp
struct Student {
    string name;
    int age;
    float score;
};
struct Student students[20];
```

B.
```cpp
struct Student {
    string name;
    int age;
    float score;
};
Student students[20];
```

C.
```cpp
struct Student {
    string name;
    int age;
    float score;
};
Student* students = new Student[20];
```

D.
```cpp
struct Student {
    string name;
    int age;
    float score;
};
Student students = new Student[20];
```

答案：D

解析：题目问“不能正确定义”的写法。A 先定义 `struct Student`，再用 `struct Student students[20]` 定义含 $20$ 个元素的结构数组；B 在 C++ 中可以省略 `struct` 关键字，`Student students[20]` 也正确；C 定义的是指向动态数组首元素的指针，`new Student[20]` 确实创建了 $20$ 个 `Student` 对象。D 左边 `Student students` 是单个结构体对象，右边 `new Student[20]` 的结果是 `Student*`，类型不匹配，也不是结构数组变量。

## 第7题

假定整型是 $32$ 位，对一个 $2$ 行 $3$ 列的二维整数数组 `array`，假设数组第一个元素在内存中的地址为 `0x7ffee4065820`，则第 $2$ 行第 $2$ 个元素的地址 `&array[1][1]` 为（ ）。

```cpp
int array[2][3] = {
    {0, 1, 2},
    {3, 4, 5}
};
```

A. `0x7ffee4065824`
B. `0x7ffee4065828`
C. `0x7ffee406582c`
D. `0x7ffee4065830`

答案：D

解析：C++ 二维数组按行连续存储，`array[1][1]` 前面依次有 `array[0][0]`、`array[0][1]`、`array[0][2]`、`array[1][0]` 共 $4$ 个 `int`。题设 `int` 为 $32$ 位，即 $4$ 字节，所以地址偏移量为 $4 \times 4 = 16$ 字节，也就是十六进制的 `0x10`。首地址 `0x7ffee4065820` 加上 `0x10` 得到 `0x7ffee4065830`。

## 第8题

下面（ ）正确定义二维数组。

A. `int a[3][];`
B. `int a[][];`
C. `int a[][4];`
D. `int a[][2] = {{1,2},{1,2},{3,4}};`

答案：D

解析：定义二维数组时，除第一维可以在有初始化列表时由编译器推断外，后面的维度必须明确给出。A、B 都缺少第二维，编译器无法计算每一行的长度。C 给出了第二维，但没有初始化列表，也没有给出第一维，不能作为完整数组定义。D 给出第二维为 $2$，并通过初始化列表推断第一维为 $3$，因此是正确的二维数组定义。

## 第9题

下面代码采用递推算法来计算斐波那契数列 $f(n) = f(n-1) + f(n-2)$，则横线上应填写（ ）。

```cpp
int fib(int n) {
    if (n == 0 || n == 1)
        return n;

    int f1 = 0;
    int f2 = 1;
    int result = 0;
    for (int i = 2; i <= n; i++) {
        // 在此处填入代码
    }
    return result;
}
```

A.

```cpp
result = f1 + f2;
f1 = f2;
f2 = result;
```

B.

```cpp
result += f1 + f2;
f1 = f2;
f2 = result;
```

C.

```cpp
result += f1 + f2;
f2 = result;
f1 = f2;
```

D.

```cpp
result = f1 + f2;
f2 = result;
f1 = f2;
```

答案：A

解析：循环中应始终让 `f1`、`f2` 表示相邻两项，先用 `result = f1 + f2` 算出新一项，再把窗口向后移动为 `f1 = f2`、`f2 = result`。这样当 `i = 2` 时得到 $1$，之后依次得到 $2,3,5,\ldots$。B、C 使用 `result += f1 + f2` 会把旧的 `result` 也累加进去；C、D 还把 `f1` 赋成更新后的 `f2`，会丢失原来相邻两项的关系。

## 第10题

下面关于排序算法（冒泡排序、插入排序和选择排序）的描述中，不正确的是（ ）。

A. 冒泡排序基于元素交换实现，需借助临时变量，共涉及 $3$ 个单元操作；而插入排序基于元素赋值实现，仅需 $1$ 个单元操作。因此冒泡排序的计算开销通常比插入排序更高。
B. 选择排序在任何情况下的时间复杂度都为 $O(n^2)$。
C. 冒泡排序在任何情况下的时间复杂度都为 $O(n^2)$。
D. 如果给定数据部分有序，插入排序通常比选择排序效率更高。

答案：C

解析：题目问“不正确”的描述。带有提前结束标记的冒泡排序在数组已经有序时，只需一轮扫描就能确认没有交换，最优时间复杂度可达 $O(n)$，所以“任何情况下都是 $O(n^2)$”过于绝对，C 不正确。选择排序每轮都要在剩余元素中找最小值，比较次数不随初始有序程度减少，通常为 $O(n^2)$；插入排序在数据部分有序时移动次数较少，常比选择排序更有优势；冒泡交换一次通常需要临时变量参与多次赋值，开销也常高于插入排序中的顺序后移。

## 第11题

冒泡排序的第一轮操作是从左到右遍历数组，通过两两比较相邻元素，将当前最大的元素移动到末尾。给定数组 `arr[]={4, 1, 3, 1, 5, 2}`，执行第一轮冒泡排序后数组 `arr` 中的内容为（ ）。

A. `1, 4, 3, 1, 5, 2`
B. `1, 3, 1, 4, 2, 5`
C. `1, 4, 3, 1, 2, 5`
D. `4, 1, 3, 1, 5, 2`

答案：B

解析：第一轮从左到右比较相邻元素。`4` 和 `1` 交换得 `1,4,3,1,5,2`；`4` 和 `3` 交换得 `1,3,4,1,5,2`；`4` 和 `1` 交换得 `1,3,1,4,5,2`；`4` 和 `5` 不交换；最后 `5` 和 `2` 交换，得到 `1,3,1,4,2,5`。这一轮结束后最大值 $5$ 被移动到末尾。

## 第12题

给定如下代码，其时间复杂度为（ ）。

```cpp
int cellRecur(int n) {
    if (n == 1)
        return 1;
    return cellRecur(n - 1) + cellRecur(n - 1) + 1;
}
```

A. $O(n^2)$
B. $O(2^n)$
C. $O(1)$
D. $O(n)$

答案：B

解析：当 $n > 1$ 时，函数会调用两次 `cellRecur(n - 1)`，此外只做常数次加法和返回操作，所以时间递推式可写成 $T(n) = 2T(n-1) + O(1)$。展开可得调用规模近似翻倍：$T(n) = O(2^n)$。它不是单层循环那样的 $O(n)$，也不是双层循环式的 $O(n^2)$，更不是常数时间。

## 第13题

下面代码实现了插入排序函数，则横线上应填写（ ）。

```cpp
void insertion_sort(vector<int> &nums) {
    for (int i = 1; i < nums.size(); i++) {
        ________________________ {  // 在此处填入代码
            while (j >= 0 && nums[j] > base)
                nums[j + 1] = nums[j];
                j--;
        }
        nums[j + 1] = base;
    }
}
```

A. `int base = nums[i], j = i - 1;`
B. `int base = nums[i], j = i;`
C. `int base = nums[0], j = i - 1;`
D. `int base = nums[0], j = i;`

答案：A

解析：按插入排序的初始化意图，第 `i` 轮应把当前位置的元素暂存为 `base = nums[i]`，再从已排序区间的最后一个位置 `j = i - 1` 向左比较并后移较大的元素，最后把 `base` 放到 `nums[j + 1]`，因此官方答案按这一意图选 A。需要注意的是，原题代码片段中 `while (j >= 0 && nums[j] > base)` 后没有花括号，严格按 C++ 语法只有 `nums[j + 1] = nums[j];` 属于循环体，`j--` 不在 `while` 循环内；若条件成立，循环变量不会在循环体内推进，这段代码不能作为完整正确的插入排序实现。B 从 `j = i` 开始会先拿元素和自己比较，位置不对；C、D 都把 `base` 固定成 `nums[0]`，无法逐轮插入当前元素。

## 第14题

下面哪种方式不能实现将字符串 "Welcome to GESP!" 输出重定向到文件 log.txt（ ）。

A.
```cpp
freopen("log.txt", "w", stdout);
cout << "Welcome to GESP!" << endl;
fclose(stdout);
```

B.
```cpp
std::ofstream outFile("log.txt");
outFile << "Welcome to GESP!" << endl;
outFile.close();
```

C.
```cpp
std::ofstream outFile("log.txt");
cout << "Welcome to GESP!" << endl;
outFile.close();
```

D.
```cpp
ofstream log_file("log.txt");
streambuf* org_cout = cout.rdbuf();
cout.rdbuf(log_file.rdbuf());
cout << "This output will go to the log file." << endl;
cout.rdbuf(oorg_cout);
```

答案：C

解析：C 虽然打开了 `log.txt`，但输出语句仍然写的是 `cout << "Welcome to GESP!" << endl;`，没有向 `outFile` 写入，也没有把 `cout` 的缓冲区重定向到文件，所以不能实现题目要求。A 通过 `freopen` 把标准输出重定向到文件，B 直接向 `ofstream` 对象写入，按意图都能写入文件。D 的思路是用 `cout.rdbuf(log_file.rdbuf())` 临时替换 `cout` 的缓冲区；但官方文本最后写成 `cout.rdbuf(oorg_cout);`，变量名与前面的 `org_cout` 不一致，按字面不能作为可直接编译的正确写法，不过本题官方答案仍按重定向思路判定 C 为不能实现的选项。

## 第15题

运行下面的代码，将出现什么情况？（ ）

```cpp
double hmean(double a, double b) {
    if (a == -b)
        throw runtime_error("Runtime error occurred");
    return 2.0*a*b/(a + b);
}

int main() {
    double x = 10;
    double y = -10;

    try {
        int result = hmean(x, y);
        cout << "hmean: " << result << endl;
    }
    catch (const runtime_error& e) {
        cout << "Caught: " << e.what() << endl;
    }
    catch (...) {
        cout << "Caught an unknown exception." << endl;
    }
    return 0;
}
```

A. 屏幕上输出 `Caught: Runtime error occurred`
B. 屏幕上输出 `Caught an unknown exception`
C. 程序调用 `std::terminate()`
D. 编译错误

答案：A

解析：调用 `hmean(10, -10)` 时，`a == -b` 成立，因为 $10 = -(-10)$，函数立即抛出 `runtime_error("Runtime error occurred")`，不会继续计算调和平均数。该异常类型能被第一个 `catch (const runtime_error& e)` 捕获，于是输出 `Caught: ` 加上 `e.what()` 返回的错误信息，即 `Caught: Runtime error occurred`。由于异常已被专门的 `catch` 捕获，不会进入 `catch (...)`，也不会调用 `std::terminate()`。
