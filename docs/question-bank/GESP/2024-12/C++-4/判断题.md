## 第1题

在 C++ 中，下面代码可以正确定义指针和初始化指针。
```cpp
int* ptr;
*ptr = 10;
```

答案：错误

解析：`int* ptr;` 只是定义了一个指针变量，但没有让它指向任何有效的 `int` 对象。随后执行 `*ptr = 10;` 是对未初始化指针解引用，写入位置不确定，属于未定义行为。若要正确初始化，可以让它指向已有变量，例如 `int x; int* ptr = &x;`，或使用动态分配得到的有效地址。

## 第2题

一个函数必须在调用之前既声明又定义。

答案：错误

解析：函数在调用前需要让编译器知道它的声明，包括函数名、返回值类型和参数列表，但不一定必须已经给出完整定义。可以先写函数原型声明，再在调用之后补上函数定义；如果函数定义本身出现在调用之前，那么这个定义也同时提供了声明。因此“必须在调用之前既声明又定义”说得过严。

## 第3题

函数参数可以通过值传递、引用传递和指针传递，这样函数内对参数的修改可以直接修改传入变量的值。

答案：错误

解析：值传递、引用传递和指针传递三者效果不同。值传递会复制实参，函数内修改形参不会影响原变量；引用传递可以直接修改被引用的实参；指针传递本身也是按值传入地址，只有通过解引用修改指针指向的对象时，才会改变外部变量。因此不能笼统地说这些传参方式都能让函数内修改直接作用到传入变量。

## 第4题

`int arr[3][]` 是一个正确的二维数组的声明。

答案：错误

解析：二维数组需要知道每一行有多少个元素，才能根据下标计算地址。`int arr[3][]` 省略了第二维，编译器无法确定一行的长度，所以不是正确的二维数组声明。常见正确写法应给出第二维，例如 `int arr[3][4];`；若通过初始化推断，也只能省略第一维，如 `int arr[][4] = {{1, 2, 3, 4}};`。

## 第5题

递推是一种通过已知的初始值和递推公式，逐步求解目标值的算法。

答案：正确

解析：递推的核心就是从一个或多个已知初始值出发，按照递推公式依次推出后面的值。例如斐波那契数列可由 $f(0)$、$f(1)$ 和 $f(n)=f(n-1)+f(n-2)$ 逐步算到目标项。题干对递推算法的描述符合这个过程。

## 第6题

某算法的递推关系式为 $T(n) = T(n-1) + n$ （$n$ 为正整数）及 $T(0) = 1$，则该算法的时间复杂度为 $O(n^2)$。

答案：正确

解析：由 $T(n) = T(n-1) + n$ 和 $T(0) = 1$ 展开，可得 $T(n) = 1 + 1 + 2 + \cdots + n$。求和结果为 $1 + \frac{n(n+1)}{2}$，最高次项是 $n^2$，因此时间复杂度为 $O(n^2)$。题干判断正确。

## 第7题

冒泡排序的平均时间复杂度为 $O(n^2)$，但最优情况下为 $O(n)$。

答案：正确

解析：普通冒泡排序平均情况下需要多轮相邻比较和交换，比较规模与 $n^2$ 同阶，因此平均时间复杂度为 $O(n^2)$。若实现中使用“本轮没有交换就提前结束”的标记，当数组一开始已经有序时，只需一轮扫描即可结束，最优时间复杂度为 $O(n)$。题干说法成立。

## 第8题

冒泡排序和插入排序都是稳定的排序算法。

答案：正确

解析：稳定排序要求相等元素排序后相对先后顺序不变。冒泡排序通常只在前一个元素大于后一个元素时交换，相等元素不会交换；插入排序通常只把大于待插入元素的元素后移，相等元素不会越过待插入元素。因此按常见实现，冒泡排序和插入排序都是稳定排序。

## 第9题

选择排序是稳定的排序算法。

答案：错误

解析：选择排序每一轮会从未排序区间中选出最小元素，并把它交换到当前位置。这个交换可能让后面的相等元素越过前面的相等元素，从而破坏相对顺序。例如序列中有两个关键字相等但附带不同标记的元素时，前面的元素可能因为与更小元素交换而移动到后一个相等元素之后。因此选择排序通常不是稳定排序。

## 第10题

在 C++ 语言中，如果一个函数可能抛出异常，那么一定要在 try 子句里调用这个函数。

答案：错误

解析：C++ 并不要求所有可能抛出异常的函数都必须在本地 `try` 子句中调用。异常可以在当前函数中捕获，也可以沿调用栈向上传播，由更外层的调用者捕获；若一直没有匹配的处理器，程序才会终止。因此“可能抛出异常就一定要在 `try` 子句里调用”过于绝对。
