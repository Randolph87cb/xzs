## 第1题

小杨家响应国家“以旧换新”政策，将自家的汽油车置换为新能源汽车，正在准备自编车牌。自编车牌包括 5 位数字或英文字母，要求第 5 位必须是数字，前 4 位中可以有最多 1 位英文字母。英文字母必须是大写，而且不能是 O 或 I（因为容易与数字 0 或 1 混淆）。请问自编车牌共有多少种可能性？（ ）

A. $100,000$
B. $1,060,000$
C. $1,360,000$
D. $1,460,000$

答案：B

解析：第 5 位必须是数字，有 $10$ 种选择。前 4 位“最多 1 位英文字母”，分两类计数：没有字母时为 $10^4$ 种；恰好 1 位字母时，先选字母所在位置有 $4$ 种，字母可用 $26-2=24$ 种，其余 3 位是数字，有 $4 \times 24 \times 10^3$ 种。再乘最后一位数字，得到 $(10^4 + 4 \times 24 \times 10^3) \times 10 = 1,060,000$。

## 第2题

新年到，四家人在一起聚会。其中两家有三口人，另外两家有两口人。现在要安排大家在一张十人圆桌坐下，要求一家人必须相邻就座。由于有“主座”的习俗，每个座位都被认为是不同的。请问共有多少种就座方案？（ ）

A. $8640$
B. $6912$
C. $144$
D. $60$

答案：A

解析：把每一家看成一个整体块，四个家庭块围成一圈的相对顺序有 $(4-1)! = 6$ 种。因为题目说明有“主座”，每个座位不同，所以整体围桌方案还可以相对固定座位旋转到 $10$ 个起始位置。每个家庭内部也要排列，两家三口各有 $3!$ 种，两家两口各有 $2!$ 种。因此总数为 $10 \times 6 \times 3! \times 3! \times 2! \times 2! = 8640$。

## 第3题

下面关于 C++ 类继承的说法，错误的是（ ）。

A. 一个类可以继承多个类。
B. 一个类可以被多个类继承。
C. 一个类可以继承另一个类的子类。
D. 抽象类必须被至少一个类继承，否则会编译错误。

答案：D

解析：C++ 支持多继承，所以一个类可以继承多个类；同一个基类也可以被多个派生类继承；类还可以继续继承某个派生类，形成多层继承。抽象类只是含有纯虚函数、不能直接实例化，它可以单独定义，也可以只用于指针或引用类型，并不要求一定被某个类继承。因此 D 的说法错误。

## 第4题

使用邻接表表达一个简单有向图，图中包含 $v$ 个顶点、$e$ 条边，则该出边表中边节点的个数为（ ）。

A. $v \times (v - 1)$
B. $v \times v$
C. $2 \times e$
D. $e$

答案：D

解析：有向图的出边邻接表中，每一条有向边只会出现在它的起点对应的出边链表里，所以边节点与有向边一一对应，个数为 $e$。$2e$ 是无向图常见邻接表中一条无向边存两次时的结果，本题明确是简单有向图。

## 第5题

以下将二维数组作为参数的函数声明，哪个是符合语法的？（ ）。

A. `void Bubble(int a[10][], int m);`
B. `void Bubble(int a[][], int n, int m);`
C. `void Bubble(int (*a)[20], int n);`
D. `void Bubble(int * a[20], int n);`

答案：C

解析：二维数组作为函数参数时，第一维可以省略，但后续维度必须给出，因为编译器需要知道每一行有多少个元素来计算地址。`int (*a)[20]` 表示 `a` 是“指向含 20 个 `int` 的数组”的指针，等价于接收形如 `int a[][20]` 的二维数组参数。A、B 都省略了第二维，无法确定行宽；D 声明的是指针数组参数退化后的 `int **` 形式，语法本身可以成立，但不是接收连续二维 `int` 数组的正确声明。

## 第6题

已知两个点 A、 B 在平面直角坐标系下的坐标分别为 $(xa, ya)$ 和 $(xb, yb)$，并分别定义变量 `double xa, ya, xb, yb;` 存储坐标。假设直线 AB 的斜率存在，下列哪个表达式可以用来表达它？（ ）。

A. $(xa - xb) / (ya - yb)$
B. $(xa - xb) / (yb - ya)$
C. $(ya - yb) / (xa - xb)$
D. $(ya - yb) / (xb - xa)$

答案：C

解析：直线斜率为纵坐标差除以横坐标差，即 $\dfrac{yb-ya}{xb-xa}$。分子和分母同时取相反数，值不变，所以也可以写成 $\dfrac{ya-yb}{xa-xb}$，对应 C。A、B 把横纵坐标差的位置颠倒成了斜率的倒数或相反形式，D 只对分子取了相反数，得到的是原斜率的相反数。

## 第7题

二项式 $(x + y)^6$ 的展开式中 $x^3 y^3$ 项的系数是（ ）。

A. $6$
B. $15$
C. $20$
D. $120$

答案：C

解析：在 $(x+y)^6$ 中要得到 $x^3y^3$，就是从 6 个因子里选 3 个贡献 $y$，其余 3 个贡献 $x$。系数为组合数 $\binom{6}{3}=20$。

## 第8题

以下关于动态规划的说法中，错误的是（ ）。

A. 动态规划方法有递推和递归两种实现形式。
B. 递归实现动态规划方法的时间复杂度总是不低于递推实现。
C. 动态规划方法将原问题分解为一个或多个相似的子问题。
D. 动态规划方法通常能够列出递推公式。

答案：B

解析：动态规划既可以自底向上递推，也可以用递归加记忆化搜索实现；它依赖把原问题拆成有重叠子问题，并通常能写出状态转移或递推公式。B 中“总是不低于”说得过于绝对：递归记忆化和递推在很多问题上访问的状态数相同，渐进时间复杂度可以一样，只是常数和调用栈开销不同。因此错误的是 B。

## 第9题

在下面的程序中，使用整数表示一种组合。整数二进制表示的某一位为 1，表示该位对应的数被选中，反之为 0 表示未选中。例如，从 0 - 5 这 6 个数中选出 3 个，则 `0b111000` 代表选中 3, 4, 5 三个数，`0b011001` 代表选中 0, 3, 4 三个数。`zuhe_next` 函数按组合对应的整数由大到小的顺序，求出组合 `c` 的下一个组合。横线处可以填入的是（ ）。

```cpp
int intlow2(int c) {
    return _______;  // 在此处填入选项
}

int zuhe_next_incur(int c, int n, int l) {
    if (n == 1) return c;
    if ((c & (1 << l)) == 0) {
        int d = intlow2(c);
        c = (c & ~d);
        c = (c | (d >> 1));
    } else {
        c = (c & ~(1 << l));
        c = zuhe_next_incur(c, n - 1, l + 1);
        int d = intlow2(c);
        c = (c | (d >> 1));
    }
    return c;
}

// 从n个数中选m个，当前组合为c
int zuhe_next(int c, int n, int m) {
    return zuhe_next_incur(c, n, 0);
}
```

A. `((c - 1) ^ c)`
B. `(((c - 1) ^ c) + 1)`
C. `(((c - 1) ^ c) >> 1)`
D. `((((c - 1) ^ c) + 1) >> 1)`

答案：D

解析：函数后续代码需要的 `d` 是 `c` 的最低位 1 对应的掩码，也就是常见的 `lowbit(c)`。设最低位 1 在第 $k$ 位，则 `c - 1` 会把第 $k$ 位从 1 变成 0，并把更低位都变成 1；`(c - 1) ^ c` 因而得到低 $k+1$ 位全为 1 的数。再加 1 得到第 $k+1$ 位为 1 的数，右移 1 位就得到第 $k$ 位为 1 的掩码。例如 `c = 0b111000` 时，D 得到 `0b001000`，正是最低位 1。A、B、C 分别会得到低位连续 1、其进位结果或少移一步的结果，都不是最低位 1 的单独掩码。

## 第10题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;
int main() {
    int N = 15, cnt = 0;
    for (int x = 0; x + x + x <= N; x++)
        for (int y = x; x + y + y <= N; y++)
            for (int z = y; x + y + z <= N; z++)
                cnt++;
    cout << cnt << endl;
    return 0;
}
```

A. $174$
B. $447$
C. $816$
D. $4096$

答案：A

解析：循环枚举的是满足 $0 \le x \le y \le z$ 且 $x+y+z \le 15$ 的三元组。外层 $x$ 最大为 5。固定 $x,y$ 后，$z$ 从 $y$ 到 $15-x-y$，共有 $16-x-2y$ 个取值。逐个 $x$ 求和：$x=0$ 得 $72$，$x=1$ 得 $49$，$x=2$ 得 $30$，$x=3$ 得 $16$，$x=4$ 得 $6$，$x=5$ 得 $1$，总数为 $72+49+30+16+6+1=174$。

## 第11题

下面最长公共子序列程序中，横线处应该填入的是（ ）。

```cpp
#define MAX(A, B) (((A) > (B)) ? (A) : (B))
#define MIN(A, B) (((A) < (B)) ? (A) : (B))
int dp[MAX_L + 1][MAX_L + 1];
int LCS(char str1[], char str2[]) {
    int len1 = strlen(str1);
    int len2 = strlen(str2);
    for (int i = 0; i < len1; i++)
        for(int j = 0; j < len2; j++)
            if (str1[i] == str2[j])
                dp[i + 1][j + 1] = dp[i][j] + 1;
            else
                _______;  // 在此处填入选项
    return dp[len1][len2];
}
```

A. `dp[i + 1][j + 1] = dp[i][j + 1] + dp[i + 1][j]`
B. `dp[i + 1][j + 1] = MIN(dp[i][j + 1], dp[i + 1][j])`
C. `dp[i + 1][j + 1] = MAX(dp[i][j + 1], dp[i + 1][j])`
D. `dp[i + 1][j + 1] = MAX(dp[i][j + 1], dp[i + 1][j]) + 1`

答案：C

解析：令 `dp[i + 1][j + 1]` 表示 `str1` 的前 `i + 1` 个字符和 `str2` 的前 `j + 1` 个字符的最长公共子序列长度。当 `str1[i] == str2[j]` 时，可以在 `dp[i][j]` 的基础上加 1；不相等时，最后一个字符不能同时选入，只能在“去掉 `str1[i]`”和“去掉 `str2[j]`”两种情况中取较大值，即 `MAX(dp[i][j + 1], dp[i + 1][j])`。A 把两个子问题相加会重复计数，B 取最小值会丢掉更优解，D 在字符不等时仍加 1 不符合 LCS 转移。

## 第12题

下列 Dijkstra 算法中，横线处应该填入的是（ ）。

```cpp
typedef struct Edge {
    int in, out;     // 从下标in顶点到下标out顶点的边
    int len;         // 边长度
    struct Edge * next;
} Edge;

// v: 顶点个数, graph: 出边邻接表, start: 起点下标, dis: 输出每个顶点的最短距离
void dijkstra(int v, Edge * graph[], int start, int * dis) {
    const int MAX_DIS = 0x7fffff;
    for (int i = 0; i < v; i++)
        dis[i] = MAX_DIS;
    dis[start] = 0;
    int * visited = new int[v];
    for (int i = 0; i < v; i++)
        visited[i] = 0;
    visited[start] = 1;
    for (int t = 0; ; t++) {
        int min = MAX_DIS, minv = -1;
        for (int i = 0; i < v; i++) {
            if (visited[i] == 0 && min > dis[i]) {
                min = dis[i];
                minv = i;
            }
        }
        if (minv < 0)
            break;
        visited[minv] = 1;
        for (Edge * e = graph[minv]; e != NULL; e = e->next) {
            // 在此处填入选项
        }
    }
    delete[] visited;
}
```

A.
```cpp
if (dis[e->out] > e->len)
    dis[e->out] = e->len;
```

B.
```cpp
if (dis[e->out] > min + e->len)
    dis[e->out] = min + e->len;
```

C.
```cpp
if (dis[e->in] > e->len)
    dis[e->in] = e->len;
```

D.
```cpp
if (dis[e->in] > min + e->len)
    dis[e->in] = min + e->len;
```

答案：B

解析：每轮选出的 `minv` 是当前未确定顶点中距离最小的顶点，变量 `min` 就是 `dis[minv]`。遍历 `graph[minv]` 的每条出边时，应尝试用“起点到 `minv` 的距离 + 这条边长度”更新终点 `e->out`，即判断 `dis[e->out] > min + e->len`。A 忽略了到 `minv` 之前的路径长度，C、D 更新的是 `e->in`，方向也错了。

## 第13题

假设图 graph 中顶点数 $v$、边数 $e$，上题程序的时间复杂度为（ ）。

A. $O(e)$
B. $O(v^2)$
C. $O(v \log v + e)$
D. $O((v + e) \log v)$

答案：B

解析：这份 Dijkstra 程序没有使用堆。每确定一个顶点前，都用一个 `for` 循环扫描全部 $v$ 个顶点寻找当前最小距离，最多进行 $v$ 次，因此这一部分是 $O(v^2)$。邻接表松弛所有边的总次数是 $O(e)$。若按题目中的简单图，$e \le v(v-1)$，总复杂度 $O(v^2+e)$ 可写作 $O(v^2)$。若不使用“简单图”这一限制，更严谨的表达应保留 $O(v^2+e)$，但选项中对应官方意图的是 B。

## 第14题

下面的快速排序程序中，两处横线处分别应填入的是（ ）。

```cpp
void quick_sort(int a[], int n) {
    if (n <= 1)
        return;
    int pivot = 0, l = 0, r = n - 1;
    while (________) { // 在此处填入选项
        while (r > pivot && a[r] >= a[pivot])
            r--;
        if (r > pivot) {
            int temp = a[pivot];
            a[pivot] = a[r];
            a[r] = temp;
            pivot = r;
        }
        while (l < pivot && a[l] <= a[pivot])
            l++;
        if (l < pivot) {
            int temp = a[pivot];
            a[pivot] = a[l];
            a[l] = temp;
            pivot = l;
        }
    }
    quick_sort(a, pivot);
    quick_sort(________); // 在此处填入选项
}
```

A.
```cpp
l < r
a + pivot + 1, n - pivot - 1
```

B.
```cpp
l < r
a + pivot + 1, n - pivot
```

C.
```cpp
l <= r
a + pivot + 1, n - pivot - 1
```

D.
```cpp
l <= r
a + pivot + 1, n - pivot
```

答案：A

解析：分区过程在左右指针还未相遇时继续，所以条件应为 `l < r`；如果写成 `l <= r`，当三者相遇时内部两个 `while` 都可能不移动指针，也不会交换，循环无法结束。分区结束后，`pivot` 左侧有 `pivot` 个元素，右侧从 `a + pivot + 1` 开始，共 `n - pivot - 1` 个元素，因此第二个递归调用应为 `quick_sort(a + pivot + 1, n - pivot - 1)`。把长度写成 `n - pivot` 会把枢轴之后的区间算多一个元素。

## 第15题

上题程序的时间复杂度为（ ）。

A. $O(n)$
B. $O(n^2)$
C. $O(2^n)$
D. $O(n \log n)$

答案：D

解析：快速排序每一层分区需要线性扫描当前区间。若枢轴把区间大致分成两半，递归深度约为 $\log n$，平均时间复杂度为 $O(n \log n)$，对应 D。需要注意的是，这段程序固定用第一个元素作为初始枢轴，没有随机化；在已经有序或每次分区极不平衡的输入上，递归深度会退化到 $n$，最坏时间复杂度是 $O(n^2)$。题目给出的答案按快速排序的常见平均复杂度理解。
