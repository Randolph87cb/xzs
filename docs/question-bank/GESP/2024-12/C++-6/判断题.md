## 第1题

构造函数是一种特殊的成员函数，构造函数的名称和类名相同。但通过函数重载，可以创建多个同名的构造函数，条件是每个构造函数的参数列表不同。

答案：正确

解析：构造函数用于对象创建时的初始化，它的函数名必须与类名相同，并且没有返回类型。C++ 允许函数重载，构造函数也一样可以重载：只要参数个数、参数类型或参数顺序不同，就可以在同一个类中定义多个构造函数，让对象按不同方式初始化。因此题干说法正确。

## 第2题

类的静态成员函数既能访问类的静态数据成员，也能访问非静态数据成员。

答案：错误

解析：静态成员函数属于类本身，调用时没有绑定到某一个具体对象，因此函数体内没有可用的 `this` 指针。静态数据成员同样属于类，静态成员函数可以直接访问；但非静态数据成员必须依附于某个对象，不能在静态成员函数中直接访问。若要访问非静态成员，必须显式传入或创建某个对象再通过对象访问。题干把两者都说成能访问，所以错误。

## 第3题

栈中元素的插入和删除操作都在栈的顶端进行，所以方便使用单向链表实现。

答案：正确

解析：栈只允许在栈顶进行插入和删除。用单向链表实现时，可以把链表头作为栈顶：入栈就是在表头插入新结点，出栈就是删除表头结点，这两个操作都只需要修改头指针和一个 `next` 指针，时间复杂度为 $O(1)$。由于不需要从尾部回退，也不需要随机访问，单向链表已经足够方便，所以题干说法正确。

## 第4题

下面代码构建的树一定是完全二叉树：
```cpp
struct TreeNode {
    int value;
    TreeNode* left;
    TreeNode* right;
};

TreeNode* buildCompleteBinaryTree() {
    TreeNode* root = new TreeNode{1};
    root->left = new TreeNode{2};
    root->right = new TreeNode{3};
    root->left->left = new TreeNode{4};
    root->left->right = new TreeNode{5};
    root->right->left = new TreeNode{6};
    return root;
}
```

答案：正确

解析：代码构造出的结点关系是：根结点 $1$ 有左右孩子 $2,3$，结点 $2$ 有左右孩子 $4,5$，结点 $3$ 有左孩子 $6$ 而没有右孩子。完全二叉树要求除最后一层外每层都满，最后一层结点从左到右连续排列。这里前两层是满的，最后一层为 $4,5,6$，正好占据从左到右的前三个位置，没有中间空缺，因此是完全二叉树。题干说法正确。

## 第5题

在二叉排序树中，左子树所有节点的值都大于根节点的值，右子树所有节点的值都小于根节点的值。

答案：错误

解析：二叉排序树，也称二叉搜索树，通常满足：左子树中所有结点值小于根结点值，右子树中所有结点值大于根结点值；若允许相等值，还需要额外规定相等值放在哪一侧。题干把左右大小关系完全说反了，因此错误。

## 第6题

在生成一个派生类的对象时，只调用派生类的构造函数。

答案：错误

解析：创建派生类对象时，需要先把它的基类部分构造好，再构造派生类自己新增的成员和派生类本体。一般顺序是先调用基类构造函数，再按成员声明顺序构造成员对象，最后执行派生类构造函数体。析构时顺序相反。题干说“只调用派生类的构造函数”，漏掉了基类构造过程，所以错误。

## 第7题

下面的代码实现了二叉树的前序遍历，它通过递归方法访问每个节点并打印节点值。
```cpp
void preorder(TreeNode* root) {
    if (root == NULL) return;
    cout << root->val << " ";
    preorder(root->left);
    preorder(root->right);
}
```

答案：正确

解析：二叉树前序遍历的访问顺序是“根结点、左子树、右子树”。代码先判断空结点并返回，然后立即输出 `root->val`，接着递归遍历 `root->left`，最后递归遍历 `root->right`，正好符合前序遍历定义。递归会把同样的规则应用到每一棵子树，因此能访问并打印整棵树的每个结点。题干说法正确。

## 第8题

在二叉树中，宽度优先搜索算法（BFS）保证从起点到每个节点的访问路径是边数最少的路径（即最短路径）。

答案：正确

解析：BFS 按距离起点的边数逐层扩展：先访问距离为 $0$ 的起点，再访问距离为 $1$ 的结点，再访问距离为 $2$ 的结点，以此类推。在无权图中，某个结点第一次被 BFS 访问时，对应路径的边数就是从起点到它的最少边数。二叉树可以看作一种无权图，而且树中任意两点之间路径唯一，因此从给定起点到每个结点的访问路径也是最短路径。题干说法正确。

## 第9题

在解决简单背包问题时，动态规划的状态转移方程如下：
```cpp
dp[i][w] = max(dp[i-1][w], dp[i-1][w - weights[i-1]] + values[i-1]);
```
该方程表示：在考虑第 $i$ 个物品时，当前背包容量为 $w$，如果不放物品 $i$，则最大价值是 $dp[i-1][w]$；如果放入物品 $i$，则最大价值是 $dp[i-1][w - weights[i-1]] + values[i-1]$，其中数组 `weights` 和 `values` 分别表示所有物品的重量和价值，数组下标从 0 开始。

答案：正确

解析：这个转移式对应 0/1 背包的两种选择。考虑前 $i$ 个物品、容量为 $w$ 时，如果不选第 $i$ 个物品，价值保持为 $dp[i-1][w]$；如果选第 $i$ 个物品，就要占用 `weights[i-1]` 的容量，并在前 $i-1$ 个物品、剩余容量 $w - weights[i-1]$ 的最优价值上加上 `values[i-1]`。由于物品编号从 $1$ 讲起而数组下标从 $0$ 开始，第 $i$ 个物品对应下标 `i-1`。实际实现中还需要保证 $w \ge weights[i-1]$ 时才能使用第二项。题干对方程含义的解释是正确的。

## 第10题

栈中元素的插入和删除操作都在栈的顶端进行，所以方便用双向链表比单向链表更合适表实现。

答案：错误

解析：栈的核心操作只发生在栈顶，用单向链表把表头作为栈顶即可在 $O(1)$ 时间完成入栈和出栈。双向链表多保存一个前驱指针，能方便从某个结点向前移动，但栈操作并不需要这种能力，反而会增加每个结点的存储开销和指针维护成本。题干中“更合适表实现”文字不够通顺，但按其核心意思“用双向链表比单向链表更合适”来判断，该说法是错误的。
