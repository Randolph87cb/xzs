## 第1题

面向对象编程 (OOP) 是一种特殊的程序设计方法。下面（ ）不是重要的 OOP 特性。

A. 抽象
B. 封装
C. 继承
D. 模块化

答案：D

解析：OOP 的核心思想是把数据和操作数据的方法组织成对象，常见特性包括抽象、封装、继承，多态也常被列为重要特性。模块化是一种更一般的软件设计思想，可以用于过程式程序、函数库划分或工程拆分，并不是面向对象编程特有的重要特性。因此 A、B、C 都属于 OOP 相关特性，D 不属于题目所问的 OOP 重要特性。

## 第2题

以下关于 C++ 中类的说法，哪一项是正确的？

A. 类中定义的所有成员变量和成员函数默认是 `public` 访问权限。
B. 类的构造函数必须显式声明返回类型为 `void`。
C. 在 C++ 中，类的数据一般设置为私有，其公有成员函数提供访问私有数据的唯一途径。
D. 同一个类的实例有各自的成员数据和成员函数。

答案：C

解析：C++ 中 `class` 的成员默认访问权限是 `private`，不是 `public`，所以 A 错。构造函数没有返回类型，连 `void` 也不能写，所以 B 错。封装通常把数据成员设为私有，再通过公有成员函数控制读写，这是 C++ 类设计中的常见做法，C 正确。不同对象各自保存一份非静态数据成员，但成员函数代码由类共享，并不是每个实例各自拥有一份成员函数，D 的表述不准确。

## 第3题

以下 C++ 代码段中存在语法错误或逻辑错误，（ ）是正确的。

```cpp
#include <iostream>
using namespace std;
class MyClass {
public:
    MyClass() {
        cout << "Constructor called!" << endl;
    }
    void display() {
        cout << "Display function called!" << endl;
    }
};
int main() {
    MyClass* obj = NULL;
    obj->display();
    return 0;
}
```

A. `NULL` 在 C++ 中无法用于指针初始化，应使用 `nullptr`。
B. `obj` 的定义应该是 `MyClass obj;` 而不是指针类型。
C. `obj->display()` 语句存在空指针访问错误，`obj` 应该初始化为一个有效的对象。
D. `obj->display()` 语句会调用 `display()` 函数，但它没有输出任何内容。

答案：C

解析：`MyClass* obj = NULL;` 定义了一个指针，但它没有指向任何有效的 `MyClass` 对象。随后执行 `obj->display()` 等价于通过空指针调用成员函数，这属于未定义行为；正确做法应当先创建对象，例如 `MyClass obj; obj.display();`，或令指针指向有效对象后再用 `->`。A 错在 `NULL` 可以用于旧式指针空值表示，只是现代 C++ 更推荐 `nullptr`。B 的写法可以避免空指针，但指针类型本身并非错误。D 错在代码中的 `display()` 明明会输出文字，真正的问题是调用发生在空指针上。

## 第4题

阅读以下代码，下面哪一项是正确的？

```cpp
void processData() {
    stack<int> s;
    queue<int> q;
    for (int i = 1; i <= 5; ++i) {
        s.push(i);
        q.push(i);
    }

    while (!s.empty()) {
        cout << "Stack pop: " << s.top() << endl;
        s.pop();
    }

    while (!q.empty()) {
        cout << "Queue pop: " << q.front() << endl;
        q.pop();
    }
}
```

A. 栈 `s` 的输出顺序是 $1\ 2\ 3\ 4\ 5$，队列 `q` 的输出顺序是 $5\ 4\ 3\ 2\ 1$。
B. 栈 `s` 的输出顺序是 $5\ 4\ 3\ 2\ 1$，队列 `q` 的输出顺序是 $1\ 2\ 3\ 4\ 5$。
C. 栈 `s` 的输出顺序是 $1\ 2\ 3\ 4\ 5$，队列 `q` 的输出顺序是 $1\ 2\ 3\ 4\ 5$。
D. 栈 `s` 的输出顺序是 $1\ 2\ 3\ 4\ 5$，队列 `q` 的输出顺序是 $1\ 2\ 3\ 4\ 5$，程序不会正常执行。

答案：B

解析：循环依次把 $1,2,3,4,5$ 压入栈和队列。栈是后进先出，最后压入的 $5$ 最先由 `top()` 取出，所以栈输出为 $5\ 4\ 3\ 2\ 1$。队列是先进先出，最先入队的 $1$ 最先由 `front()` 取出，所以队列输出为 $1\ 2\ 3\ 4\ 5$。因此 B 正确，A、C 把栈的顺序判断错了，D 还额外声称程序不能正常执行，也不符合代码行为。

## 第5题

$N$ 个节点的双向循环链，在其中查找某个节点的平均时间复杂度是（ ）。

A. $O(1)$
B. $O(N)$
C. $O(\log N)$
D. $O(N^3)$

答案：B

解析：双向循环链表只是在每个节点中保存前驱和后继指针，并且尾节点能回到头节点；如果没有额外索引或有序性，查找某个指定节点仍然只能沿链逐个比较。最坏情况下要检查 $N$ 个节点，平均情况下也与 $N$ 成正比，所以时间复杂度是 $O(N)$。双向和循环能方便插入、删除或双向移动，但不能把普通查找降到 $O(1)$ 或 $O(\log N)$。

## 第6题

以下关于树的说法，（ ）是正确的。

A. 在一棵二叉树中，叶子结点的度一定是 $2$。
B. 满二叉树中每一层的结点数等于 $O(2^{(\text{层数}-1)})$。
C. 在一棵树中，所有结点的度之和等于所有叶子结点的度之和。
D. 一棵二叉树的先序遍历结果和中序遍历结果一定相同。

答案：B

解析：叶子结点没有孩子，度为 $0$，不是 $2$，A 错。满二叉树按根为第 $1$ 层计，第 $k$ 层有 $2^{k-1}$ 个结点，用渐进记号可写为 $O(2^{k-1})$，B 正确。树中所有结点的度之和等于边数，不等于叶子结点度之和；叶子结点度之和反而是 $0$，C 错。先序遍历是“根、左、右”，中序遍历是“左、根、右”，一般不会相同，D 错。

## 第7题

已知字符集 $\{A, B, C, D\}$ 的出现频率如下表所示：

| 字符 | 频率 |
| :----------: | :----------: |
| A | 8 |
| B | 3 |
| C | 1 |
| D | 6 |

根据哈夫曼编码法，下面（ ）是正确的哈夫曼树。

![](https://cdn.luogu.com.cn/upload/image_hosting/2a1wnq6a.png)


A. A
B. B
C. C
D. D

答案：A

解析：哈夫曼树每次选当前频率最小的两个结点合并。四个频率为 $C=1$、$B=3$、$D=6$、$A=8$，先合并 $C$ 和 $B$ 得到权值 $4$ 的结点；再把 $4$ 与 $D=6$ 合并得到权值 $10$ 的结点；最后把 $10$ 与 $A=8$ 合并成根。也就是说，$B$、$C$ 应先成为兄弟结点，随后与 $D$ 合并，最后才与 $A$ 合并。图中只有 A 满足这个合并过程。

## 第8题

上一题中各字符的哈夫曼编码是（ ）。

A. A: $0$, B: $10$, C: $110$, D: $111$
B. A: $0$, B: $10$, C: $11$, D: $10$
C. A: $0$, B: $101$, C: $100$, D: $11$
D. A: $11$, B: $10$, C: $01$, D: $00$

答案：C

解析：上一题的合并过程决定了码长结构：频率最大的 $A=8$ 最靠近根，编码长度为 $1$；$D=6$ 次之，编码长度为 $2$；$B=3$ 与 $C=1$ 先合并，二者编码长度为 $3$。选项 C 中 $A:0$ 长度为 $1$，$D:11$ 长度为 $2$，$B:101$、$C:100$ 长度为 $3$，并且没有任何一个编码是另一个编码的前缀，符合这棵哈夫曼树的一种左右分支标记。A 把 $B$ 设为 $2$ 位、$D$ 设为 $3$ 位，不符合频率更高者不应更深的构造结果；B 中 $B$ 和 $D$ 编码重复；D 的四个字符都为 $2$ 位，不能体现哈夫曼树的最优码长分配。

## 第9题

（ ）是 3 位格雷编码。

A. $000\ 001\ 011\ 010\ 110\ 111\ 101\ 100$
B. $000\ 001\ 010\ 011\ 100\ 101\ 110\ 111$
C. $000\ 001\ 100\ 101\ 011\ 010\ 111\ 110$
D. $000\ 010\ 001\ 011\ 100\ 110\ 101\ 111$

答案：A

解析：$n$ 位格雷码要求相邻两个编码只相差 $1$ 位，通常还要求首尾也只差 $1$ 位形成循环序列。A 中依次比较：$000 \to 001$ 只变最后一位，$001 \to 011$ 只变中间一位，之后每一步也都只改变一位，且 $100 \to 000$ 也只差最高位，所以是合法的 3 位格雷码。B 是普通二进制顺序，$001 \to 010$ 相差两位；C 中 $001 \to 100$ 相差两位；D 中 $011 \to 100$ 相差三位，都不符合格雷码要求。

## 第10题

根据下面二叉树和给定的代码。

```cpp
#include <iostream>
using namespace std;

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(NULL), right(NULL) {}
};

TreeNode* search(TreeNode* root, int val) {
    cout << root->val << " ";
    if (root == NULL || root->val == val) return root;

    if (val < root->val)
        return search(root->left, val);
    else
        return search(root->right, val);
}
```

给定以下二叉搜索树，调用函数 `search(root, 7)` 时，输出的结果是（ ）。

```
    5
   / \
  3   7
 / \ / \
2  4 6  8
```

A. $5\ 3\ 7$
B. $5\ 7$
C. $2\ 3\ 4\ 5\ 6\ 7$
D. $8\ 7$

答案：B

解析：二叉搜索树中，若目标值大于当前结点值，就去右子树查找；若小于当前结点值，就去左子树查找。调用 `search(root, 7)` 时，先访问根结点 $5$ 并输出 `5`，因为 $7 > 5$，递归进入右孩子 $7$；接着输出 `7`，此时 `root->val == val`，函数返回。因此输出为 $5\ 7$。代码中 `cout << root->val` 写在空指针判断之前，若查找不存在的值并递归到空指针会出错；但本题查找 $7$ 时路径上没有空指针，所以不影响该次输出。

A. `TreeNode* node = s.top();`

## 第13题

使用上题中的宽度优先搜索算法遍历以下这棵树，可能的输出是（ ）。

```
    1
   / \
  2   3
 / \   \
8   9   6
   / \   \
  4   5   7
```

A. $1\ 2\ 8\ 9\ 4\ 5\ 3\ 6\ 7$
B. $1\ 2\ 3\ 4\ 5\ 6\ 6\ 8\ 9$
C. $1\ 2\ 3\ 8\ 9\ 6\ 4\ 5\ 7$
D. $8\ 4\ 5\ 9\ 2\ 1\ 3\ 6\ 7$

答案：C

解析：上题 BFS 代码先访问当前结点，再按左孩子、右孩子的顺序入队，因此会逐层从左到右输出。第 $1$ 层是 $1$；第 $2$ 层是 $2,3$；第 $3$ 层依次是 $2$ 的孩子 $8,9$ 和 $3$ 的右孩子 $6$；第 $4$ 层依次是 $9$ 的孩子 $4,5$ 和 $6$ 的右孩子 $7$。合起来就是 $1\ 2\ 3\ 8\ 9\ 6\ 4\ 5\ 7$，对应 C。

## 第14题

以下关于动态规划的描述，（ ）是正确的。

A. 动态规划适用于没有重叠子问题的优化问题。
B. 动态规划要求问题具有最优子结构和无后效性。
C. 动态规划通常通过递归来实现。
D. 动态规划与贪心算法不同，贪心算法不适用于有重叠子问题的问题。

答案：B

解析：动态规划适合把原问题拆成子问题，并利用子问题结果推出更大问题的答案。典型条件包括最优子结构和无后效性：当前状态确定后，未来决策只依赖这个状态，而不需要关心到达它的完整历史。A 错在动态规划常常正是利用重叠子问题来避免重复计算。C 错在动态规划可以用递推表实现，也可以用记忆化搜索，不“通常必须递归”。D 对贪心和重叠子问题的关系表述不准确，贪心是否适用取决于贪心选择性质，而不是简单由“有无重叠子问题”决定。

## 第15题

假设背包的最大容量 $W = 8kg$，共有有 4 个物品可供选择，4 个物品的重量分别为 $weights = [2, 3, 5, 7]$，对应的价值分别为 $values = [30, 40, 60, 80]$，则该 0/1 背包问题中，背包的最大价值为（ ）。

A. $70$
B. $90$
C. $100$
D. $120$

答案：C

解析：这是 0/1 背包，每个物品最多选一次，容量为 $8$。逐一看可行组合：重量 $2+3=5$，价值 $70$；重量 $2+5=7$，价值 $90$；重量 $2+7=9$ 超重；重量 $3+5=8$，价值 $100$；重量 $3+7=10$ 超重；重量 $5+7=12$ 超重。三个物品中 $2+3+5=10$ 已超重，含 $7$ 再加其它也超重。单个物品最高价值是 $80$，可行组合最高是重量 $3$ 和 $5$ 的两个物品，价值 $40+60=100$，所以最大价值为 $100$。
