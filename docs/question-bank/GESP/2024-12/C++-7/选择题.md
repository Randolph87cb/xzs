## 第1题

已知小写字母 b 的 ASCII 码为 98，下列 C++ 代码的输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;
int main() {
    char a = 'b';
    cout << a + 1;
    return 0;
}
```

A. b
B. c
C. 98
D. 99

答案：D

解析：`a` 是 `char`，但表达式 `a + 1` 会先把字符提升为整数参与运算。已知 `'b'` 的 ASCII 码是 $98$，所以 `a + 1` 的值是 $99$，`cout` 输出整数 `99`，不是字符 `c`。因此选 D。

## 第2题

已知 a 为 int 类型变量，p 为 int * 类型变量，下列赋值语句不符合语法的是（ ）。

A. `+a = *p;`
B. `*p = +a;`
C. `a = *(p + a);`
D. `*(p + a) = a;`

答案：A

解析：赋值号左侧必须是可修改的左值。`+a` 是对 `a` 取一元正号后得到的表达式结果，不再是可赋值对象，所以 `+a = *p;` 不符合语法。`*p` 表示指针指向的 `int` 对象，`a` 也是普通 `int` 变量，`*(p + a)` 虽然可能有越界风险，但从语法上仍是可赋值的 `int` 左值。因此选 A。

## 第3题

已知数组 a 的定义 `int a[10] = {0};`，下列说法不正确的是（ ）。

A. 语句 `a[-1] = 0;` 会产生编译错误。
B. 数组 a 的所有元素均被初始化为 0。
C. 数组 a 至少占用 10 个 int 大小的内存，一般为 40 个字节。
D. 语句 `a[13] = 0;` 不会产生编译错误，但会导致难以预测的运行结果。

答案：A

解析：`a[-1]` 在 C++ 中等价于 `*(a - 1)`，语法上可以通过编译，但访问了数组首元素之前的位置，运行行为未定义，所以 A 的“会产生编译错误”不正确。`int a[10] = {0};` 会把 10 个元素都初始化为 0；数组至少占 10 个 `int` 的空间，常见平台上 `int` 为 4 字节即 40 字节；`a[13] = 0;` 同样通常不会因下标越界而编译报错，但运行结果不可预测。因此选 A。

## 第4题

下列关于 C++ 类的说法，错误的是（ ）。

A. 构造函数不能声明为虚函数，但析构函数可以。
B. 函数参数如声明为类的引用类型，调用时不会调用该类的复制构造函数。
C. 静态方法属于类、不属于对象，因此不能使用 对象.方法(...) 的形式调用静态方法。
D. 析构派生类的对象时，一定会调用基类的析构函数。

答案：C

解析：静态成员函数确实属于类本身，不依赖某个对象的 `this` 指针，但 C++ 允许用 `类名::方法(...)` 调用，也允许通过对象写成 `对象.方法(...)` 调用，因此 C 的后半句错误。构造函数不能是虚函数，析构函数可以声明为虚函数；引用参数不会触发复制构造；派生类对象析构时会先析构派生类部分，再析构基类部分。因此选 C。

## 第5题

下列关于有向图的说法，错误的是（ ）。

A. $n$ 个顶点的弱连通有向图，最少有 $n - 1$ 条边。
B. $n$ 个顶点的强连通有向图，最少有 $n$ 条边。
C. $n$ 个顶点的有向图，最多有 $n \times (n - 1)$ 条边。
D. $n$ 个顶点的有向完全图，有 $n \times (n - 1)$ 条边。

答案：C

解析：本题选 C。A 说弱连通有向图最少有 $n-1$ 条边是合理的，因为忽略方向后要让 $n$ 个顶点连通，至少需要像树一样的 $n-1$ 条边。B 说强连通有向图最少有 $n$ 条边也合理，例如 $n$ 个顶点首尾相接形成一个有向环即可从任意顶点到达任意顶点。D 中有向完全图通常指任意两个不同顶点之间两个方向都连边，因此有 $n(n-1)$ 条边。C 只说“有向图”，没有限定为简单图、无重边、无自环；在一般说法下，允许重边时边数没有这个上界，允许自环时也会改变最大边数。因此“最多有 $n(n-1)$ 条边”缺少必要条件，作为一般说法不正确。

## 第6题

一棵二叉树的每个结点均满足：结点的左子树和右子树，要么同时存在，要么同时不存在。该树有 197 个结点。则其叶结点有多少个？（ ）

A. $98$
B. $99$
C. 不存在这样的树。
D. 无法确定叶结点数量。

答案：B

解析：题目描述的是满二叉树性质：每个结点要么有两个孩子，要么没有孩子。设内部结点数为 $I$、叶结点数为 $L$，总边数为 $197 - 1 = 196$。每个内部结点贡献 2 条到孩子的边，所以 $2I = 196$，得到 $I = 98$，于是 $L = 197 - 98 = 99$。因此选 B。

## 第7题

下列关于二叉树的说法，错误的是（ ）。

A. 二叉排序树的中序遍历顺序与元素排序的顺序是相同的。
B. $n$ 个元素的二叉排序树，其高一定为 $\lfloor \log_2 n \rfloor$。
C. 自平衡二叉查找树（AVL 树）是一种二叉排序树。
D. 任意的森林，都可以映射为一颗二叉树进行表达和存储。

答案：B

解析：二叉排序树的形态取决于插入顺序，不一定平衡。若元素按递增顺序插入，树可能退化成一条链，高度接近 $n$；只有接近平衡时高度才约为 $\log_2 n$，所以 B 的“一定”为错。二叉排序树中序遍历会得到有序序列；AVL 树是在二叉排序树基础上维持平衡；森林也可以用“左孩子右兄弟”方式转成二叉树表示。因此选 B。

## 第8题

一个简单无向图有 10 个结点、6 条边。在最差情况，至少增加多少条边可以使其连通？（ ）

A. $3$
B. $4$
C. $6$
D. $9$

答案：C

解析：要让一个无向图连通，若当前有 $k$ 个连通分量，至少要新增 $k - 1$ 条边。6 条边在 10 个结点中最差可以全部集中在同一个连通分量内；一个含 6 条边的简单连通分量至少可以有 4 个点，例如 $K_4$ 有 6 条边。这样剩下 6 个点都是孤立点，总共有 $1 + 6 = 7$ 个连通分量，需要至少 $7 - 1 = 6$ 条边连接起来。因此选 C。

## 第9题

一个哈希表，包括 $n$ 个位置（分别编号 $0 \sim (n-1)$），每个位置最多仅能存储一个元素。该哈希表只有插入元素和查询两种操作，没有删除或修改元素的操作。以下说法错误的是（ ）。

A. 如果哈希函数取值范围为 $0 \sim (n-1)$，且当发生哈希函数碰撞时循环向后寻找空位，则查询操作的最差时间复杂度为 $O(n)$。（“循环向后”指：0 向后一位为 1，1 向后一位为 2，……，$(n-2)$ 向后一位为 $(n-1)$，$(n-1)$ 向后一位为 0）
B. 如果哈希函数取值范围为 $0 \sim (n-1)$，且当发生哈希函数碰撞时仅循环向后一个位置寻找空位，则查询操作的最差时间复杂度为 $O(1)$。
C. 如果哈希函数取值范围为 $0 \sim (m-1)$（$m < n$），且当发生哈希函数碰撞时仅在 $m \sim (n-1)$ 的范围内寻找空位，则查询操作的最差时间复杂度为 $O(n - m)$。
D. 查询操作时，如果发现查询元素经哈希函数对应的位置为空位，该查询元素仍可能出现在哈希表内。

答案：D

解析：题目说明没有删除操作。按开放寻址的查询逻辑，如果一个元素按哈希函数应从某个位置开始查找，而起始位置已经是空位，就说明从未有需要越过这个空位的插入过程发生，该元素不可能在后面的探查位置中，因此 D 错误。A 的线性探查最坏可能扫描整个表，复杂度为 $O(n)$；B 只允许看当前位置和后一个位置，查询次数有常数上界；C 只在 $m \sim n-1$ 的备用区间中查找，最坏扫描 $n-m$ 个位置。因此选 D。

## 第10题

以下关于动态规划的说法中，错误的是（ ）。

A. 动态规划方法将原问题分解为一个或多个相似的子问题。
B. 动态规划方法通常能够列出递推公式。
C. 动态规划方法有递推和递归两种实现形式。
D. 递推实现动态规划方法的时间复杂度总是不低于递归实现。

答案：D

解析：动态规划的递推实现通常按状态顺序直接填表；递归实现若不加记忆化，可能反复计算同一子问题，时间复杂度反而更高。即使递归加了记忆化，通常也只是和递推达到同一数量级，并不存在“递推总是不低于递归”的规律。A、B、C 都是动态规划常见特征：拆成重叠子问题，写出状态转移，并可用递推或记忆化递归实现。因此选 D。

## 第11题

下面程序的输出为（ ）。

```cpp
#include <iostream>
#include <cmath>
using namespace std;
int main() {
    cout << (int)exp(2) << endl;
    return 0;
}
```

A. $4$
B. $7$
C. $100$
D. 无法通过编译。

答案：B

解析：`exp(2)` 计算的是自然常数 $e$ 的平方，即 $e^2 \approx 7.389$。表达式前的 `(int)` 会把浮点数转换为整数，直接截去小数部分，得到 $7$。程序包含了 `<cmath>`，`exp` 可以正常使用，因此选 B。

## 第12题

下面程序的输出为（ ）。

```cpp
#include <iostream>
#define N 10
using namespace std;
int h[N];
int main() {
    h[0] = h[1] = 1;
    for (int n = 2; n < N; n++)
        for (int j = 0; j < n; j++)
            h[n] += h[j] * h[n - j - 1];
    cout << h[6] << endl;
    return 0;
}
```

A. $132$
B. $1430$
C. $16796$
D. 结果是随机的。

答案：A

解析：全局数组 `h` 会先被初始化为 0，程序再令 `h[0] = h[1] = 1`。递推式为 $h[n] = \sum_{j=0}^{n-1} h[j] \times h[n-j-1]$，这是卡特兰数递推。逐项算得 $h[2]=2$，$h[3]=5$，$h[4]=14$，$h[5]=42$，$h[6]=132$，所以输出为 132。因此选 A。

## 第13题

上题中程序的时间复杂度为（ ）。

A. $O(N)$
B. $O(N \log N)$
C. $O(N^{3/2})$
D. $O(N^2)$

答案：D

解析：外层循环中 `n` 从 2 到 $N-1$，内层循环对每个 `n` 执行 $n$ 次。总执行次数为 $\sum_{n=2}^{N-1} n = \frac{(N-1)N}{2} - 1$，数量级是 $O(N^2)$。因此选 D。

## 第14题

下面 `init_sieve` 函数的时间复杂度为（ ）。

```cpp
int sieve[MAX_N];
void init_sieve(int n) {
    for (int i = 1; i <= n; i++)
        sieve[i] = i;
    for (int i = 2; i <= n; i++)
        for (int j = i; j <= n; j += i)
            sieve[j]--;
}
```

A. $O(n)$
B. $O(n \log n)$
C. $O(n^2)$
D. 无法正常结束。

答案：B

解析：第一重初始化循环执行 $n$ 次。第二部分中，固定 `i` 时，`j` 取 `i, 2i, 3i, ...`，执行约 $\lfloor n/i \rfloor$ 次，所以总次数为 $\sum_{i=2}^{n} \lfloor n/i \rfloor$，不超过 $n \sum_{i=2}^{n} 1/i = O(n \log n)$，也达到同一数量级。两部分合并后时间复杂度为 $O(n \log n)$。因此选 B。

## 第15题

下列选项中，哪个不可能是下图的深度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/drz4v5nh.png)

A. 1, 2, 3, 5, 7, 8, 6, 9, 4
B. 1, 4, 7, 8, 9, 5, 2, 3, 6
C. 1, 5, 7, 8, 9, 4, 2, 3, 6
D. 1, 2, 3, 6, 9, 8, 5, 7, 4

答案：A

解析：按题图的有向边，关键关系有 $1 \to 2,4,5$，$2 \to 3$，$3 \to 5,6$，$5 \to 7$，$7 \to 8$，$8 \to 5,9$，$6 \to 8,9$。DFS 序列记录的是首次访问顺序。A 中从 `1, 2, 3, 5, 7, 8` 走到 8 时，8 仍有一条指向未访问结点 9 的边，因此递归 DFS 必须先访问 9，不能直接回退到 3 再访问 6，所以 A 不可能。B、C、D 都可以通过调整每个结点邻接边的访问顺序得到，例如 D 可先走 $1 \to 2 \to 3 \to 6 \to 9$，再回到 6 访问 8。因此选 A。
