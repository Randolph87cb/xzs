# 判断题

来源：

- 官网原卷 PDF：https://gesp.ccf.org.cn/101/attach/1650239779897376.pdf

说明：本文件优先采用洛谷结构化数据整理，并以官网 PDF 做交叉校对。

## 第1题

表达式 `5^3` 的结果为 125。

答案：错误

解析：在 C++ 中，`^` 不是乘方运算符，而是按位异或运算符。`5` 的二进制可看作 `0101`，`3` 可看作 `0011`，异或结果为 `0110`，即十进制 $6$。因此 `5^3` 的结果不是 $125$，题干错误。

## 第2题

在 C++ 语言中，函数定义和函数调用可以不在同一个文件内。

答案：正确

解析：C++ 支持分离编译。一个函数可以在头文件或当前文件中提供声明，在另一个源文件中给出定义；调用处只要在编译时能看到函数声明，最终链接时能找到对应定义即可。因此函数定义和函数调用可以不在同一个文件内，题干正确。

## 第3题

在 $n$ 个元素中进行二分查找，平均时间复杂度是 $O(\log n)$，但须要事先进行排序。

答案：正确

解析：二分查找每次把候选区间大致缩小一半，因此查询过程的时间复杂度是 $O(\log n)$。它成立的前提是数据按关键字有序，或至少满足可以根据中点比较结果排除一半区间的单调性；普通无序数组不能直接二分。因此题干正确。

## 第4题

`unsigned long long` 类型是 C++ 语言中表达范围最大的非负整数类型之一，其表达范围是 $[0, 2^{64} - 1]$。超出该范围的非负整数运算，将无法使用 C++ 语言进行计算。

答案：错误

解析：`unsigned long long` 通常是 64 位无符号整数，表达范围为 $[0, 2^{64}-1]$，也是 C++ 内置整数类型中范围最大的非负整数类型之一。但“超出该范围的非负整数运算，将无法使用 C++ 语言进行计算”过于绝对：可以用数组、字符串、自定义大整数类或第三方大整数库在 C++ 中实现任意精度计算。因此题干错误。

## 第5题

使用 `math.h` 或 `cmath` 头文件中的函数，表达式 $\log_2(32)$ 的结果为 5、类型为 `int`。

答案：错误

解析：`log2(32)` 的数值结果是 $5$，但 `<cmath>` 中的 `log2` 对整数实参会按浮点函数处理，返回类型不是 `int`，常见情况下为 `double`。题干把“值为 5”和“类型为 `int`”同时断言，其中类型判断错误，所以题干错误。

## 第6题

C++ 是一种面向对象编程语言，C 则不是。继承是面向对象三大特性之一。因此，使用 C 语言无法实现继承。

答案：错误

解析：C 语言本身没有 C++ 那样的类继承语法，但“无法实现继承”说得过于绝对。通过结构体嵌套、把公共字段放在结构体开头、函数指针表等方式，C 程序可以模拟出类似继承和多态的组织方式。题干把“语言没有内置继承语法”扩大成“无法实现继承思想”，因此错误。

## 第7题

邻接表和邻接矩阵都是图的存储形式。邻接表在遍历单个顶点的所有边时，时间复杂度更低；邻接矩阵在判断两个顶点之间是否有边时，时间复杂度更低。

答案：正确

解析：邻接表只存储实际存在的边，遍历某个顶点的所有出边时，时间与该顶点的边数相关；邻接矩阵需要扫描这一行或这一列，通常是 $O(n)$。反过来，邻接矩阵可直接查看矩阵中的一个位置，判断两点之间是否有边为 $O(1)$；邻接表则可能要在该顶点的边表中查找。因此题干正确。

## 第8题

MD5 是一种常见的哈希函数，可以由任意长度的数据生成 128 位的哈希值，曾广泛应用于数据完整性校验。中国科学家的系列工作首次发现了可实用的 MD5 破解方法。之后，MD5 逐渐被其他哈希函数所取代。

答案：正确

解析：MD5 会把任意长度输入映射为 128 位摘要，早期常用于完整性校验。王小云等中国科学家的工作给出了实用的 MD5 碰撞攻击，使得 MD5 的抗碰撞安全性不再可靠。此后安全场景逐步改用 SHA-2、SHA-3 等更安全的哈希函数。因此题干正确。

## 第9题

递归调用在运行时会由于层数过多导致程序崩溃，可以通过循环配合栈缓解这一问题。

答案：正确

解析：递归调用会占用调用栈，每深入一层都要保存返回地址、局部变量等信息；当递归层数过多时，可能导致栈空间耗尽并崩溃。把递归改写为循环并使用显式栈，可以把“调用栈深度”转移到程序可控的数据结构上，通常更容易管理空间。因此题干正确。

## 第10题

一个图中，每个顶点表达一个城市，连接两个顶点的边表达从一个城市到达另一个城市的一种交通方式。这个图可以用来表达交通网络，且是简单有向图。

答案：错误

解析：用图表示交通网络是合理的，顶点表示城市，边表示交通方式或线路。但题干进一步断定它“是简单有向图”并不一定成立：两个城市之间可能同时有公路、铁路、航线等多种交通方式，对应重边；交通也可能是双向的或需要用无向边/双向边表达。因此不能必然称为简单有向图，题干错误。
