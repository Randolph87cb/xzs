## 第1题

在 Windows 的资源管理器中为已有文件 A 建立副本的操作是 `Ctrl+C`，然后 `Ctrl+V`。（ ）

答案：正确

解析：在 Windows 资源管理器中，`Ctrl+C` 是复制选中的文件，`Ctrl+V` 是在当前位置粘贴复制出的文件。对已有文件 A 先复制再粘贴，会生成它的副本，而不是移动原文件。因此题干说法正确。

## 第2题

在 C++ 中，表达式 $8/3$ 和 $8\%3$ 的值相同。（ ）

答案：正确

解析：`8/3` 是整数除法，只保留商的整数部分，结果为 `2`；`8%3` 是取余，表示 `8` 除以 `3` 后的余数，也为 `2`。两个表达式的含义不同，但在这组数值下结果相同。因此题干说法正确。

## 第3题

X 是 C++ 语言的基本类型变量，则语句 `cin>>X, cout <<X` 能接收键盘输入并原样输出。（ ）

答案：错误

解析：`cin >> X, cout << X` 在语法上会先执行输入，再执行输出，逗号运算符能把两个表达式连在一起。但“原样输出”说得过于绝对：例如输入给整数变量时，前导零、空白等格式不会原样保留，输入给数值类型还会按数值规则转换后再输出。因此不能说对任意 C++ 基本类型变量都能接收键盘输入并原样输出，题干说法错误。

## 第4题

下面 C++ 代码执行后将输出 10。（ ）

```cpp
int N = 0;
for (int i = 0; i < 10; i++){
    continue;
    N += 1;
}
cout << N;
```

答案：错误

解析：循环体中先执行 `continue;`，它会立刻结束本轮循环，跳到下一轮的自增和条件判断处。位于 `continue` 后面的 `N += 1;` 每一轮都不会被执行，所以 `N` 始终是初始值 `0`。最终输出 `0`，不是 `10`，题干说法错误。

## 第5题

下面 C++ 代码执行后将输出 100。（ ）

```cpp
int i;
for (i = 0; i <= 100; i++)
    continue;
cout << i;
```

答案：错误

解析：循环条件是 `i <= 100`，当 `i` 从 `0` 增加到 `100` 时仍会进入循环；这一轮执行 `continue` 后，还会执行 `for` 语句末尾的 `i++`，使 `i` 变成 `101`。此时条件不成立，循环结束，输出的是 `101`。因此题干说输出 `100` 是错误的。

## 第6题

下面 C++ 代码被执行时，将执行三次输出（即标记行 L2 将被执行一次）。（ ）

```cpp
for (int i = 0; i < 10; i+=3)
    cout << i;  //L2
```

答案：错误

解析：循环变量从 `0` 开始，每次增加 `3`，满足 `i < 10` 的取值是 `0,3,6,9`，一共 4 次。因此标记行 L2 会执行 4 次，输出也会发生 4 次，而不是 3 次。题干说法错误。

## 第7题

C++ 语句 `cout << (3,2)` 执行后，将输出 3 和 2，且 3 和 2 之间有逗号间隔。（ ）

答案：错误

解析：括号中的 `(3,2)` 使用的是逗号运算符，它会先计算左边的 `3`，再计算右边的 `2`，整个表达式的值是右边的 `2`。所以 `cout << (3,2)` 只会输出 `2`，不会输出 `3,2`，也不会自动加逗号。题干说法错误。

## 第8题

在 C++ 代码中，`studentName`、`student_name` 以及 `sStudentName` 都是合法的变量名称。（ ）

答案：正确

解析：C++ 标识符可以由字母、数字和下划线组成，但不能以数字开头，也不能是关键字。`studentName`、`student_name`、`sStudentName` 都由字母或下划线相关的合法字符组成，并且都以字母开头，不是关键字。因此它们都是合法变量名，题干说法正确。

## 第9题

在 C++ 中，对浮点变量 `float f`，则语句 `cin >> f; cout << (f<1);`；在输入是 `2e-1` 时，输出是 0。（ ）

答案：错误

解析：`2e-1` 是科学计数法，表示 $2 \times 10^{-1}$，也就是 `0.2`。读入 `float f` 后，表达式 `f < 1` 为真，`cout` 默认输出布尔值时会输出 `1`，而不是 `0`。因此题干说法错误。

## 第10题

在 C++ 的循环体内部，如果 `break` 和 `continue` 语句连续在一起，那么作用抵消，可以顺利执行下一次循环。（ ）

答案：错误

解析：`break` 和 `continue` 的作用不同：`break` 会直接结束整个循环，`continue` 只结束本轮循环并进入下一轮判断。如果在循环体中先执行到 `break`，循环已经退出，后面的 `continue` 不会再执行；如果先执行到 `continue`，它后面的 `break` 也不会执行。它们不会“作用抵消”。题干说法错误。
