## 第1题

下面关于链表和数组的描述，错误的是（ ）。

A. 当数据数量不确定时，为了应对各种可能的情况，需要申请一个较大的数组，可能浪费空间；此时用链表比较合适，大小可动态调整。
B. 在链表中访问节点的效率较低，时间复杂度为 $O(n)$。
C. 链表插入和删除元素效率较低，时间复杂度为 $O(n)$。
D. 链表的节点在内存中是分散存储的，通过指针连在一起。

答案：C

解析：题目要求选“错误”的描述。链表在已知插入或删除位置、并已拿到前驱节点时，只需要改指针，操作本身是 $O(1)$；如果还要先查找位置，查找才可能是 $O(n)$。C 把链表插入和删除一概说成效率较低、时间复杂度为 $O(n)$，表述过于绝对，所以错误。A 说明数组可能需要预留空间、链表可动态增减，合理；B 说链表不能像数组一样按下标直接访问，访问节点通常要顺着指针走，复杂度为 $O(n)$，合理；D 说明链表节点可分散存储并通过指针连接，也正确。

## 第2题

在循环单链表中，节点的 `next` 指针指向下一个节点，最后一个节点的 `next` 指针指向（ ）。

A. 当前节点
B. `nullptr`
C. 第一个节点
D. 上一个节点

答案：C

解析：循环单链表的“循环”体现在尾节点不再指向 `nullptr`，而是把 `next` 指回链表的第一个节点，这样从任意节点不断沿 `next` 前进都能回到开头。因此最后一个节点的 `next` 指向第一个节点，选 C。A 会形成只有当前节点参与的自环；B 是普通单链表的尾节点写法；D 是双向链表中 `prev` 指针才可能表达的关系，不是单链表 `next` 的方向。

## 第3题

为了方便链表的增删操作，一些算法生成一个虚拟头节点，方便统一删除头节点和其他节点。下面代码实现了删除链表中值为 `val` 的节点，横线上应填的最佳代码是（ ）。

```cpp
struct LinkedNode {
    int val;
    LinkedNode* next;
    LinkedNode(int val):val(val), next(nullptr){}
};

void removeElements(LinkedNode* head, int val) {
    if (head == nullptr) {
        return;
    }
    LinkedNode* cur;
    LinkedNode* dummyHead = new LinkedNode(0); //虚拟头节点
    // 在此处填入代码

    while(cur ->next ！= nullptr) {
        if(cur->next->val == val) {
            LinkedNode* tmp = cur->next;
            cur->next = cur->next->next;
            delete tmp;
            tmp = nullptr;
        }
        else {
            cur = cur ->next;
        }
    }
    head = dummyHead->next;
    delete dummyHead;
    dummyHead = nullptr;
}
```

A. `dummyHead->next = head; cur = dummyHead;`
B. `dummyHead->next = head->next; cur = dummyHead;`
C. `dummyHead->next = head; cur = dummyHead->next;`
D. `dummyHead->next = head->next; cur = dummyHead->next;`

答案：A

解析：按官方意图，循环条件应理解为 `cur->next != nullptr`，逐个检查 `cur` 后面的节点；若严格按题面中的全角 `！=`，这段代码不是合法 C++，无法编译。使用虚拟头节点的目的，是让原头节点也变成“某个节点的后继”，从而删除头节点和删除中间节点可以统一处理。因此应先令 `dummyHead->next = head`，再令 `cur = dummyHead`，这样第一次检查的就是原来的 `head`。B、D 把 `head` 本身跳过，若头节点值等于 `val` 就无法删除；C 让 `cur` 从原头节点开始，也失去了统一删除头节点的作用。

## 第4题

对下面两个函数，说法错误的是（ ）。

```cpp
int fibA(int n) {
    if (n <= 1) return n;
    int f1 = 0, f2 = 1;
    for (int i = 2; i <= n; ++i) {
        int temp = f2;
        f2 = f1 + f2;
        f1 = temp;
    }
    return f2;
}

int fibB(int n) {
    if (n <= 1) return n;
    return fibB(n - 1) + fibB(n - 2);
}
```

A. 两个函数的实现的功能相同。
B. `fibA` 采用递推方式。
C. `fibB` 采用的是递归方式。
D. `fibA` 时间复杂度为 $O(n)$，`fibB` 的时间复杂度为 $O(n^2)$。

答案：D

解析：题目要求选“错误”的说法。`fibA` 从 $0,1$ 开始迭代更新，循环执行到 $n$，是递推实现，时间复杂度为 $O(n)$。`fibB` 通过调用 `fibB(n - 1)` 和 `fibB(n - 2)` 求解，是递归实现，也计算同一个斐波那契数列，所以 A、B、C 都正确。D 的前半句正确，但 `fibB` 没有记忆化，会反复计算大量相同子问题，时间复杂度是指数级，常写作 $O(2^n)$ 或更精细地写作 $O(\varphi^n)$，不是 $O(n^2)$。

## 第5题

两块长方形土地的长宽分别为 $24$ 和 $36$ 米，要将它们分成正方形的小块，使得正方形的尺寸尽可能大。小杨采用如下的辗转相除函数 `gcd(24, 36)` 来求正方形分块的边长，则函数 `gcd` 调用顺序为（ ）。

```cpp
int gcd(int a, int b) {
    int big = a > b ? a : b;
    int small = a < b ? a : b;
    if (big % small == 0) {
        return small;
    }
    return gcd(small, big % small);
}
```

A. `gcd(24, 36)`、`gcd(24, 12)`、`gcd(12, 0)`
B. `gcd(24, 36)`、`gcd(12, 24)`、`gcd(0, 12)`
C. `gcd(24, 36)`、`gcd(24, 12)`
D. `gcd(24, 36)`、`gcd(12, 24)`

答案：C

解析：第一次调用 `gcd(24, 36)` 时，函数内部得到 `big = 36`、`small = 24`，因为 $36 \bmod 24 = 12$，不满足整除，于是递归调用 `gcd(24, 12)`。第二次调用中 `big = 24`、`small = 12`，因为 $24 \bmod 12 = 0$，直接返回 $12$，不会再调用 `gcd(12, 0)`。所以调用顺序只有 `gcd(24, 36)`、`gcd(24, 12)`，选 C。

## 第7题

下述代码实现素数表的埃拉托色尼（埃氏）筛法，筛选出所有小于等于 $n$ 的素数。

```cpp
vector<int> sieve_Eratosthenes(int n) {
    vector<bool> is_prime(n +1, true);
    vector<int> primes;

    for (int i = 2; i * i <= n; i++) {
        if (is_prime[i]) {
            primes.push_back(i);

            for (int j = i * i; j <= n; j += i) {
                is_prime[j] = false;
            }
        }
    }

    for (int i = sqrt(n) + 1; i <= n; i++) {
        if (is_prime[i]) {
            primes.push_back(i);
        }
    }

    return primes;
}
```

下面说法，正确的是（ ）。

A. 代码的时间复杂度是 $O(n\sqrt{n})$。
B. 在标记非素数时，代码从 $i^2$ 开始，可以减少重复标记。
C. 代码会输出所有小于等于 $n$ 的奇数。
D. 调用函数 `sieve_Eratosthenes(10)`，函数返回值的数组中包含的元素有：$2$，$3$，$5$，$7$，$9$。

答案：B

解析：埃氏筛在处理质数 $i$ 时，从 $i^2$ 开始标记倍数，是因为小于 $i^2$ 的倍数都可以写成 $i \times k$ 且 $k < i$，已经在处理更小因子 $k$ 时被标记过了，所以这样能减少重复标记，B 正确。A 的 $O(n\sqrt n)$ 不是埃氏筛的典型复杂度；C 错在函数返回的是素数，不是所有奇数；D 中 $9$ 是合数，会在处理 $3$ 时被标记为非素数，`sieve_Eratosthenes(10)` 应得到 $2,3,5,7$。

## 第8题

下述代码实现素数表的线性筛法，筛选出所有小于等于 $n$ 的素数。下面说法正确的是（ ）。

```cpp
vector<int> sieve_linear(int n) {
    vector<bool> is_prime(n +1, true);
    vector<int> primes;

    for (int i = 2; i <= n/2; i++) {
        if (is_prime[i])
            primes.push_back(i);

        for (int j = 0; j < primes.size() && i * primes[j] <= n; j++) {
            is_prime[ i * primes[j] ] = 0;
            if (i % primes[j] == 0)
                break;
        }
    }

    for (int i = n/2 +1; i <= n; i++) {
        if (is_prime[i])
            primes.push_back(i);
    }

    return primes;
}
```

A. 线性筛的时间复杂度是 $O(n)$。
B. 每个合数会被其所有的质因子标记一次。
C. 线性筛和埃拉托色尼筛的实现思路完全相同。
D. 以上都不对

答案：A

解析：线性筛的核心是内层循环遇到 `i % primes[j] == 0` 就 `break`，使每个合数只被它的最小质因数筛掉一次，因此总标记次数与 $n$ 同阶，时间复杂度为 $O(n)$，A 正确。B 说每个合数会被所有质因子标记一次，这是埃氏筛中可能出现的重复标记特征，不是线性筛；C 说两种筛法思路完全相同也不对，线性筛专门通过最小质因数控制唯一标记；因此 D 也不成立。

## 第9题

考虑以下 C++ 代码实现的快速排序算法：

```cpp
int partition(vector<int>& arr, int left, int right) {
    int pivot = arr[right]; // 基准值
    int i = left - 1;

    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[right]);
    return i + 1;
}

// 快速排序
void quickSort(vector<int>& arr, int left, int right) {
    if (left < right) {
        int pi = partition(arr, left, right);
        quickSort(arr, left, pi - 1);
        quickSort(arr, pi + 1, right);
    }
}
```

以下关于快速排序的说法，正确的是（ ）。

A. 快速排序通过递归对子问题进行求解。
B. 快速排序的最坏时间复杂度是 $O(n \log n)$。
C. 快速排序是一个稳定的排序算法。
D. 在最优情况下，快速排序的时间复杂度是 $O(n)$。

答案：A

解析：代码中 `partition` 先把数组按基准值分成左右两部分，随后 `quickSort(arr, left, pi - 1)` 和 `quickSort(arr, pi + 1, right)` 分别递归排序两个子区间，所以 A 正确。快速排序最坏情况下每次划分极不均衡，时间复杂度会退化为 $O(n^2)$，不是 B 的 $O(n \log n)$；普通快速排序会交换远距离元素，不能保证相等元素的相对顺序，通常不是稳定排序；最优情况下每层处理 $n$ 个元素、共有约 $\log n$ 层，时间复杂度是 $O(n \log n)$，不是 $O(n)$。

## 第10题

下面关于归并排序，描述正确的是（ ）。

A. 归并排序是一个不稳定的排序算法。
B. 归并排序的时间复杂度在最优、最差和平均情况下都是 $O(n \log n)$。
C. 归并排序需要额外的 $O(1)$ 空间。
D. 对于输入数组 $\{12, 11, 13, 5, 6, 7\}$，代码输出结果为：$7\ 6\ 5\ 13\ 12\ 11$。

答案：B

解析：归并排序无论原数组是否接近有序，都要不断二分到小区间，再逐层合并；共有约 $\log n$ 层，每层合并总量为 $n$，所以最优、最差和平均时间复杂度都是 $O(n \log n)$，B 正确。归并时若相等元素优先取左半部分，可以保持相对顺序，因此归并排序通常是稳定排序，A 错；数组归并排序一般需要额外辅助数组，空间复杂度为 $O(n)$，不是 $O(1)$；D 给出的结果既不是升序归并排序的输出，也不是题目中归并排序概念的正确结论。

## 第11题

给定一个长度为 $n$ 的有序数组 `nums`，其中所有元素都是唯一的。下面的函数返回数组中元素 `target` 的索引。

```cpp
int binarySearch(vector<int> &nums, int target, int left, int right) {
    if (left > right) {
        return -1;
    }

    int middle = left + ((right - left) / 2);
    if (nums[middle] == target) {
        return middle;
    }
    else if (nums[middle] < target) {
        return binarySearch(nums, target, middle + 1, right);
    }
    else
        return binarySearch(nums, target, left, middle - 1);
}

int Find(vector<int> &nums, int target) {
    int n = nums.size();
    return binarySearch(nums, target, 0, n - 1);
}
```

关于上述函数，描述不正确的是（ ）。

A. 函数采用二分查找，每次计算搜索当前搜索区间的中点，然后根据中点的元素值排除一半搜索区间。
B. 函数采用递归求解，每次问题的规模减小一半。
C. 递归的终止条件是中间元素的值等于 `target`，若数组中不包含该元素，递归不会终止。
D. 算法的复杂度为 $O(\log n)$。

答案：C

## 第14题

关于分治算法，以下说法中不正确的是（ ）。

A. 分治算法将问题分成子问题，然后分别解决子问题，最后合并结果。
B. 归并排序采用了分治思想。
C. 快速排序采用了分治思想。
D. 冒泡排序采用了分治思想。

答案：D

解析：题目要求选“不正确”的说法。分治算法的典型过程是把原问题拆成若干规模更小的子问题，分别求解后再合并结果，A 正确。归并排序先分成左右两半、分别排序、再合并，符合分治，B 正确；快速排序先划分出基准左右两侧，再递归处理两个子区间，也符合分治，C 正确。冒泡排序是通过相邻元素比较和交换逐步把元素“冒泡”到正确位置，不是把问题拆分后合并，所以 D 不正确。

## 第15题

小杨编写了一个如下的高精度减法函数：

```cpp
vector<int> highPrecisionSubtract(vector<int> a, vector<int> b) {
    vector<int> result;
    int borrow = 0;

    for (int i = 0; i < a.size(); ++i) {
        int digitA = a[i];
        int digitB = i < b.size() ? b[i] : 0;

        int diff = digitA - digitB - borrow;
        if (diff < 0) {
            diff += 10;
            borrow = 1;
        }
        else {
            borrow = 0;
        }

        result.push_back(diff);
    }

    while (result.size() > 1 && result.back() == 0) {
        result.pop_back();
    }

    return result;
}
```

下面说法，正确的是（ ）。

A. 如果数组 `a` 表示的整数小于 `b` 表示的整数，代码会正确返回二者的差为负数。
B. 代码假设输入数字是以倒序存储的，例如 $500$ 存储为 $\{0, 0, 5\}$。
C. 代码的时间复杂度为 $O(a.size() + b.size())$。
D. 当减法结果为 $0$ 时，结果数组仍然会存储很多个元素 $0$。

答案：B

解析：本题选 B。循环从 `i = 0` 开始逐位相减，并把 `result.push_back(diff)` 直接加入结果，说明数组下标 $0$ 存的是最低位，下标越大表示越高位。例如 $500$ 的个位、十位、百位分别是 $0,0,5$，应存成 `{0, 0, 5}`，所以 B 是明确正确项。A 不合适，因为代码没有处理 `a < b` 时最终仍有借位的情况，不能正确表示负数结果。D 不合适，因为末尾的 `while` 会删除多余前导零，只在结果全为 $0$ 时保留一个 $0$。C 如果只当作宽松上界，可以覆盖这段循环，但本题比较的是代码性质，单选时更贴切、明确的是 B。
