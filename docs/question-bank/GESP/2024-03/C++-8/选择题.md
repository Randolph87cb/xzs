## 第1题

为丰富食堂菜谱，炒菜部进行头脑风暴。肉类有鸡肉、牛肉、羊肉、猪肉 4 种，切法有肉排、肉块、肉末 3 种，配菜有圆白菜、油菜、豆腐 3 种，辣度有麻辣、微辣、不辣 3 种。不考虑口感的情况下，选 1 种肉、1 种切法、1 种配菜、1 种辣度产生一道菜（例如：麻辣牛肉片炒豆腐），这样能产生多少道菜？（ ）。

A. $13$
B. $42$
C. $63$
D. $108$

答案：D

解析：四类选择彼此独立，按乘法原理计数：肉类有 $4$ 种，切法有 $3$ 种，配菜有 $3$ 种，辣度有 $3$ 种，所以总数为 $4 \times 3 \times 3 \times 3 = 108$，选 D。A 的 $13$ 是把数量相加得到的结果，B、C 都少乘了某一类或混淆了独立选择，不能覆盖所有组合。

## 第2题

已知袋中有 2 个相同的红球、3 个相同的绿球、5 个相同的黄球。每次取出一个不放回，全部取出。可能产生多少种序列？（ ）。

A. $6$
B. $1440$
C. $2520$
D. $3628800$

答案：C

解析：共有 $10$ 个球，其中同色球不可区分。若先把 $10$ 个位置排好，再分别放入 $2$ 个红球、$3$ 个绿球、$5$ 个黄球，序列数为 $\frac{10!}{2!3!5!}=2520$，选 C。D 把同色球也当成不同球排列，B 少除了重复排列，A 只是在排列三种颜色本身。

## 第3题

以下二维数组的初始化，哪个是符合语法的？（ ）。

A. `int a[][] = {{1, 2}, {3, 4}};`
B. `int a[][2] = {};`
C. `int a[2][2] = {{1, 2, 3}, {4, 5, 6}};`
D. `int a[2][] = {{1, 2, 3}, {4, 5, 6}};`

答案：B

解析：二维数组初始化时，除第一维外，其余维度必须能由声明直接确定。B 写出了第二维长度 `2`，符合本题官方答案口径，表示每行有 2 个 `int`。A、D 都省略了第二维，编译器无法确定一行有多少个元素；C 声明每行只有 2 个元素，却给每行放了 3 个初始化值。需要注意，严格 C++ 标准下 `int a[][2] = {};` 是否能推出 0 行数组存在编译器争议；考试官方答案按“第二维已给出”的语法考点判 B。

## 第4题

下面有关 C++ 拷贝构造函数的说法，错误的是（ ）。

A. 必须实现拷贝构造函数，否则一定会出现编译错误。
B. 对象作为函数参数、以值传递方式传入函数时，会自动调用拷贝构造函数。
C. 对象作为函数返回值、以值传递方式从函数返回时，会自动调用拷贝构造函数。
D. 使用一个对象初始化另一个对象时，会自动调用拷贝构造函数。

答案：A

解析：题目问“错误的是”。如果类没有显式定义拷贝构造函数，编译器通常会生成默认拷贝构造函数，因此“必须实现，否则一定会编译错误”过于绝对，A 错。B 中值传参需要用实参初始化形参对象，D 中用一个对象初始化另一个对象，都是拷贝构造函数的典型调用场景；C 在教材口径下也属于值返回可能调用拷贝构造函数，实际 C++ 编译器可能进行返回值优化，但不影响本题选 A。

## 第5题

使用邻接表表达一个无向简单图，图中包含 $v$ 个顶点、$e$ 条边，则该表中边节点的个数为（ ）。

A. $v \times (v - 1)$
B. $v \times v$
C. $2 \times e$
D. $e$

答案：C

解析：无向图的一条边连接两个顶点，在邻接表中通常要分别出现在两个端点的链表里，因此每条无向边对应 $2$ 个边节点，总数为 $2e$，选 C。A、B 是完全图或邻接矩阵规模一类的表达式，和实际边数 $e$ 无关；D 只计算了一条无向边的一侧记录。

## 第6题

关于生成树的说法，错误的是（ ）。

A. 一个无向连通图可以有多个生成树。
B. 一个无向图，只要连通，就一定有生成树。
C. $n$ 个顶点的无向完全图，有 $n^{n-2}$ 棵生成树。
D. $n$ 个顶点的无向图，生成树包含 $n-1$ 条边。

答案：D

解析：题目问“错误的是”。生成树只对连通图有意义；如果只说“$n$ 个顶点的无向图”，它可能不连通，此时根本不存在生成树，所以 D 少了连通条件，表述错误。B 明确说无向图只要连通就一定有生成树；A 中“可以有多个”表示存在这种可能，例如一个环图删去不同边可得不同生成树；C 是 Cayley 公式，$n$ 个顶点的无向完全图 $K_n$ 有 $n^{n-2}$ 棵生成树。

## 第7题

已知三个 `double` 类型的变量 `a`、`b` 和 `theta` 分别表示一个三角形的两条边长及二者的夹角（弧度），则下列哪个表达式可以计算这个三角形的周长？（ ）。

A. `a * b * sin(theta) / 2`
B. `a + b + (a + b) * sin(theta) / 2`
C. `a * b * cos(theta) / 2`
D. `a + b + sqrt(a * a + b * b - 2 * a * b * cos(theta))`

答案：D

解析：已知两边及夹角，第三边由余弦定理得到：$c=\sqrt{a^2+b^2-2ab\cos\theta}$，周长为 $a+b+c$，对应 D。A 是三角形面积公式 $\frac{1}{2}ab\sin\theta$；B 把正弦项错误地当成第三边；C 既不是面积公式，也不是边长公式。

## 第8题

在有 $n$ 个元素的二叉排序树中进行查找，其最好、最差时间复杂度分别为（ ）。

A. $O(1)$、$O(n)$
B. $O(1)$、$O(\log n)$
C. $O(\log n)$、$O(\log n)$
D. $O(\log n)$、$O(n)$

答案：A

解析：二叉排序树查找从根开始比较。最好情况是要找的元素恰好在根节点，只比较一次，时间复杂度为 $O(1)$；最差情况是树退化成一条链，需要沿链查找最多 $n$ 个节点，时间复杂度为 $O(n)$，选 A。B、C 把最差情况误当成平衡树；D 的最好情况不应写成 $O(\log n)$。

## 第9题

如下图所示，半径为 $r$、圆心角为 $t$（弧度）的扇形，下面哪个表达式能够求出顶部阴影部分的面积？（ ）

![](https://cdn.luogu.com.cn/upload/image_hosting/1kf4r9wx.png)

A. `r * r * sin(t) / 2`
B. `r * r * t / 2`
C. `r * r * (t - sin(t))`
D. `r * r * (t - sin(t)) / 2`

答案：D

解析：阴影部分是扇形面积减去由两条半径和弦围成的三角形面积。圆心角为弧度 $t$ 时，扇形面积为 $\frac{r^2t}{2}$，三角形面积为 $\frac{r^2\sin t}{2}$，相减得 $\frac{r^2(t-\sin t)}{2}$，选 D。A 只是三角形面积，B 只是扇形面积，C 比正确结果多了 $2$ 倍。

## 第10题

下面程序的时间复杂度为（ ）。

```cpp
int fib(int n) {
    if (n <= 1)
        return 1;
    return fib(n - 1) + fib(n - 2);
}
```

A. $O(2^n)$
B. $O(\phi^n)$，其中 $\phi = \frac{\sqrt{5}+1}{2}$
C. $O(n)$
D. $O(1)$

答案：B

解析：该递归没有记忆化，`fib(n)` 会分别调用 `fib(n - 1)` 和 `fib(n - 2)`，调用次数满足 $T(n)=T(n-1)+T(n-2)+O(1)$，增长率与 Fibonacci 数列相同，为 $\Theta(\phi^n)$，其中 $\phi=\frac{\sqrt5+1}{2}$，选 B。A 的 $O(2^n)$ 是较粗的上界，不是本题选项中最精确的增长量级；C、D 忽略了重复递归调用。

## 第11题

下面程序的时间复杂度为（ ）。

```cpp
int choose(int n, int m) {
    if (m == 0 || m == n)
        return 1;
    return choose(n - 1, m - 1) + choose(n - 1, m);
}
```

A. $O(2^n)$
B. $O(2^m \times (n - m))$
C. $O(C(n, m))$
D. $O(m \times (n - m))$

答案：C

解析：该函数直接按组合数递推式 $C(n,m)=C(n-1,m-1)+C(n-1,m)$ 展开，没有保存中间结果。递归树中每条到边界条件的路径对应一种选择 $m$ 个位置的方式，叶子数量为 $C(n,m)$，整棵二叉递归树节点数与叶子数同阶，因此时间复杂度按官方口径记为 $O(C(n,m))$，选 C。D 是动态规划填表的复杂度；A 只给出更粗的指数上界；B 不是这个递归树的标准计数。

## 第12题

下面程序的时间复杂度为（ ）。

```cpp
int primes[MAXP], num = 0;
bool isPrime[MAXN] = {false};
void sieve() {
    for (int n = 2; n <= MAXN; n++) {
        if (!isPrime[n])
            primes[num++] = n;
        for (int i = 0; i < num && n * primes[i] <= MAXN; i++) {
            isPrime[n * primes[i]] = true;
            if (n % primes[i] == 0)
                break;
        }
    }
}
```

A. $O(n)$
B. $O(n \times \log n)$
C. $O(n \times \log \log n)$
D. $O(n^2)$

答案：A

解析：这段代码是线性筛。外层枚举每个 `n`，内层用已得到的质数标记 `n * primes[i]`，并在 `n % primes[i] == 0` 时停止，使每个合数只会被它的最小质因子对应地标记一次。因此总标记次数与范围大小同阶，时间复杂度为 $O(n)$，选 A。B、C 更接近普通筛或其他筛法的估计，D 则把内外循环误认为总会完整相乘。

## 第13题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;

int a[10][10];
int main() {
    int m = 5, n = 4;
    for (int x = 0; x <= m; x++)
        a[x][0] = 1;
    for (int y = 1; y <= n; y++)
        a[0][y] = 1;
    for (int x = 1; x <= m; x++)
        for (int y = 1; y <= n; y++)
            a[x][y] = a[x - 1][y] + a[x][y - 1];
    cout << a[m][n] << endl;
    return 0;
}
```

A. $4$
B. $5$
C. $126$
D. $3024$

答案：C

解析：数组边界先设为 `a[x][0] = 1` 和 `a[0][y] = 1`，随后递推 `a[x][y] = a[x - 1][y] + a[x][y - 1]`，这等价于从左上角走到 $(x,y)$ 的路径计数。到 $(5,4)$ 需要 $5$ 次向下和 $4$ 次向右，共 $9$ 步，路径数为 $C(9,4)=C(9,5)=126$，所以输出 126，选 C。A、B 只取了维度值，D 是把 $9$ 步做了错误的排列乘法。

## 第14题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;

int main() {
    int cnt = 0;
    for (int x = 0; x <= 10; x++)
        for (int y = 0; y <= 10; y++)
            for (int z = 0; z <= 10; z++)
                if (x + y + z == 15)
                    cnt++;
    cout << cnt << endl;
    return 0;
}
```

A. $90$
B. $91$
C. $96$
D. $100$

答案：B

解析：程序统计满足 $0 \le x,y,z \le 10$ 且 $x+y+z=15$ 的有序三元组个数。若只看非负整数解，数量为 $C(17,2)=136$；再减去某个变量至少为 $11$ 的情况，令该变量减 $11$ 后有 $x'+y+z=4$，每个变量对应 $C(6,2)=15$ 种，共减 $45$ 种，不会有两个变量同时超过 $10$。因此计数为 $136-45=91$，选 B。

## 第15题

下面的程序使用邻接矩阵表达的带权无向图，则从顶点 0 到顶点 3 的最短距离为（ ）。

```cpp
int weight[4][4] = {
    { 0,  1,  7, 100},
    { 1,  0,  5,  15},
    { 7,  5,  0,   6},
    {100, 15,  6,   0}};
```

A. $100$
B. $16$
C. $12$
D. $13$

答案：C

解析：从顶点 0 到顶点 3 的直接边权为 100，但可以经过中间点缩短距离。路径 $0 \to 2 \to 3$ 的长度是 $7+6=13$，路径 $0 \to 1 \to 3$ 的长度是 $1+15=16$，路径 $0 \to 1 \to 2 \to 3$ 的长度是 $1+5+6=12$，是这些可行路径中最短的，选 C。A 是直接边，B、D 分别对应较长的绕行路径。
