## 第1题

整数 $-5$ 的 $16$ 位补码表示是（ ）。

【知识点：原码反码补码与整数表示】

A. `1005`
B. `1006`
C. `FFFA`
D. `FFFB`

答案：D

解析：本题答案选 D。按 16 位补码计算，$5$ 的十六进制表示是 `0005`，逐位取反得到 `FFFA`，再加 $1$ 得到 `FFFB`，所以 $-5$ 的 16 位补码是 `FFFB`。选项 C 的 `FFFA` 只是 `0005` 取反后的结果，还没有加 $1$；A、B 的 `1005`、`1006` 不是按 16 位补码由 $-5$ 推出的结果。

## 第2题

如果 $16$ 位短整数 $-2$ 的二进制是"FFFE"，则短整数 $-4$ 的十六进制是（ ）

【知识点：进制转换与进制字面量】

A. `FF04`
B. `FFFA`
C. `FFFC`
D. `FFFH`

答案：C

解析：本题答案选 C。题目已说明采用 16 位补码口径，$-4$ 的补码可以看成 $2^{16}-4=65532$，换成十六进制就是 `FFFC`。选项 B 的 `FFFA` 对应的是 $-6$，A 的低位字节不符合 $65536-4$ 的计算结果；D 中 `H` 不是十六进制数码，本题要求填写的表示也不是 `FFFH`。

## 第3题

下面 C++ 代码执行后的输出是（ ）。

```cpp
int main()
{
    cout << (3 | 16) << endl;

    cout << endl;
    return 0;
}
```

【知识点：位运算】

A. 3
B. 16
C. 19
D. 48

答案：C

解析：本题答案选 C。`|` 是按位或运算，$3$ 的二进制可写成 `00011`，$16$ 的二进制可写成 `10000`，逐位或后得到 `10011`，即十进制 $19$。A、B 分别只保留了其中一个操作数；D 的 $48$ 不是按位或结果。

## 第4题

定义整数 `int x = -5`，则执行C++代码 `cout << (x == (x << 1 >> 1))` 输出是（ ）。

【知识点：位运算】

A. 0
B. 1
C. -5
D. 5

答案：B

解析：本题答案选 B。在题目假定的 16 位补码和算术移位模型下，`x = -5` 左移一位可理解为得到 $-10$，随后再右移一位会回到 $-5$，所以 `x == (x << 1 >> 1)` 为真，`cout` 输出布尔值 `1`。A 表示比较为假，不符合这个位运算模型下的结果；C、D 把表达式当成整数值本身来输出，也不符合 `cout << (比较表达式)` 的含义。实际 C++ 编程中，对负的有符号整数做左移不宜作为可移植写法，应避免依赖这种结果。

## 第5题

已知字符 `'0'` 的ASCII编码的十进制表示为 $48$，则执行下面 C++ 代码后，输出是（ ）。

```cpp
int main()
{
    string s = "316";
    int n = s.length();
    int x = 0;
    for (int i = 0; i < n; i++)
        x += s[i];
    cout << x << endl;

    cout << endl;
    return 0;
}
```

【知识点：ASCII/字符编码】

A. 10
B. 58
C. 154
D. 316

答案：C

解析：本题答案选 C。循环执行的是 `x += s[i]`，加入的是字符的 ASCII 码值，不是数字本身。已知 `'0'` 为 $48$，则 `'3'`、`'1'`、`'6'` 的码值分别是 $51$、$49$、$54$，总和为 $51+49+54=154$。A 的 $10$ 是把字符当成数字后求和；D 的 $316$ 是把字符串当整数；B 的 $58$ 也不是三个字符码值之和。

## 第6题

下面 C++ 代码执行后数组中大于 $0$ 的数的特征是（ ）。

```cpp
int main()
{
    int a[20], i;
    for (i = 0; i < 20; i++)
        a[i] = i + 1;

    for (int i = 0; i < 20; i++)
        if ((a[i] % 2) && (a[i] % 3))
            a[i] = 0;

    for (i = 0; i < 20; i++)
        if (a[i])
            cout << a[i] << " ";

    cout << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. $2$ 的倍数
B. $3$ 的倍数
C. 能被 $2$ 或 $3$ 整除的数
D. 能被 $2$ 和 $3$ 同时整除的数

答案：C

解析：本题答案选 C。数组最初存放 $1$ 到 $20$。条件 `(a[i] % 2) && (a[i] % 3)` 为真，表示这个数除以 $2$ 有余数并且除以 $3$ 也有余数，也就是既不是 $2$ 的倍数，也不是 $3$ 的倍数；这些数会被改成 $0$。最后还能输出的大于 $0$ 的数，正好是能被 $2$ 或 $3$ 整除的数。A、B 都只说了其中一类，D 只保留同时能被 $2$ 和 $3$ 整除的数，范围太小。

## 第7题

执行下面 C++ 代码后输出的第一个数是（ ）。

```cpp
int main()
{
    int a[20], i;
    for (i = 0; i < 20; i++)
        a[i] = i + 1;

    for (; i > 0; i--)
        cout << a[i - 1] << " ";

    cout << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. $20$
B. $19$
C. $1$
D. 不确定

答案：A

解析：本题答案选 A。第一个 `for` 循环结束后，数组 `a[0]` 到 `a[19]` 分别是 $1$ 到 $20$，同时变量 `i` 已经变成 $20$。第二个循环第一次执行时输出的是 `a[i - 1]`，也就是 `a[19]`，其值为 $20$。B 的 $19$ 是把下标和数组值混淆了；C 是正序输出时的第一个数；D 不成立，因为第一次访问的 `a[19]` 在数组范围内。

## 第8题

在下列代码的横线处填写（ ），可以使得输出是 `GESP IS INTERESTING`。

```cpp
int main()
{
    string str = "gEsP is Interesting";
    int x = str.length();
    for (int i = 0; i < x; i++)
        if ((str[i] >= 'a') && (str[i] <= 'z'))
            ___________________;
    cout << str << endl;
    cout << endl;
    return 0;
}
```

【知识点：ASCII/字符编码】

A. `str[i] += 'a' - 'A'`
B. `str[i] += 20`
C. `str[i] += 'A' - 'a'`
D. 无法实现

答案：C

解析：本题答案选 C。代码只在 `str[i]` 是小写字母时修改它，要把小写变成对应大写，需要让码值减少 $32$，也就是加上 `'A' - 'a'`。因此 `g`、`i`、`s` 等小写字母会转成 `G`、`I`、`S`，原本的大写字母和空格不变，最终得到 `GESP IS INTERESTING`。A 中 `'a' - 'A'` 是 $+32$，方向相反；B 的 $20$ 不是大小写字母的码值差；D 错在这个转换可以直接通过字符码差实现。

## 第9题

假设英文句子由若干词构成。下面 C++ 代码统计输出的词数是（ ）。

```cpp
int main()
{
    string str = "gEsP is Interesting !";
    int x = str.length();
    int nwords = 0;

    for (int i = 0; i < x; i++)
        if (str[i] == ' ') {
            nwords++;
            while (str[++i] == ' ');
        }
    cout << nwords << endl;

    cout << endl;
    return 0;
}
```

【知识点：字符串处理】

A. $1$
B. $2$
C. $3$
D. $4$

答案：C

解析：本题答案选 C。字符串是 `gEsP is Interesting !`，其中空格分别出现在 `gEsP` 后、`is` 后和 `Interesting` 后。循环遇到一个空格就让 `nwords++`，并用 `while (str[++i] == ' ')` 跳过连续空格，所以这里会累计 $3$ 次。程序实际输出的是 $3$；A、B 少算了空格分隔处，D 则更像是把感叹号也当作一个独立词来数，但代码并不会直接统计词本身的个数。

## 第10题

C++ 的字符变量的码值是整数，下面字面量形式的字符码值最大的是（ ）。

【知识点：ASCII/字符编码】

A. `100`
B. `075`
C. `0x70`
D. `0x60`

答案：C

解析：本题答案选 C。比较这些字面量的实际整数值：`100` 是十进制 $100$；`075` 以 `0` 开头，是八进制，等于 $7 \times 8 + 5 = 61$；`0x70` 是十六进制，等于 $7 \times 16 = 112$；`0x60` 等于 $96$。最大的是 $112$，所以选 `0x70`。

## 第11题

下面 C++ 程序执行的结果是（ ）。

```cpp
int main()
{
    int a[20], i;
    int cnt = 0;
    for (i = 0; i < 20; i++)
        a[i] = i + 1;

    for (; i > 1; i--)
        if ((a[i - 1] + a[i - 2]) % 3)
            cnt++;
    cout << cnt << endl;

    cout << endl;
    return 0;
}
```

【知识点：数组与字符数组】

A. $5$
B. $6$
C. $10$
D. $12$

答案：D

解析：本题答案选 D。初始化后 `a[0]` 到 `a[19]` 是 $1$ 到 $20$，此时 `i=20`。第二个循环依次检查相邻两项之和：$20+19$、$19+18$、一直到 $2+1$，共 $19$ 组。按从小到大看，这些和是 $3,5,7,9,\ldots,39$；其中能被 $3$ 整除的是 $3,9,15,21,27,33,39$，共 $7$ 组。条件 `(sum % 3)` 为真表示余数不为 $0$，因此计数为 $19-7=12$。A、B、C 都没有正确统计“不能被 $3$ 整除”的相邻和数量。

## 第12题

定义字符数组 `char str[20] = {'G', 'E', 'S', 'P'};`，则 `str` 的字符串长度为（ ）。

【知识点：数组与字符数组】

A. $4$
B. $5$
C. $19$
D. $20$

答案：A

解析：本题答案选 A。`char str[20] = {'G', 'E', 'S', 'P'};` 只显式给出前 $4$ 个字符，数组剩余位置会被初始化为 `'\0'`。按 C 风格字符串长度计算时，从开头数到第一个 `'\0'` 为止，所以字符串内容是 `GESP`，长度为 $4$。B 把字符串结束符也算进长度了；C、D 混淆了字符串长度和字符数组容量。

## 第13题

定义整型变量 `int a=3, b=16`，则 `a|b` 的值和 `a+b` 的关系是（ ）。

【知识点：位运算】

A. 大于
B. 等于
C. 小于
D. 等于或小于

答案：B

解析：本题答案选 B。$3$ 的二进制是 `00011`，$16$ 的二进制是 `10000`，两者为 $1$ 的位没有重叠，所以 `a | b` 得到 `10011`，也就是 $19$。同时 `a + b = 3 + 16 = 19`，两者相等。A、C 都与实际计算结果相反；D 虽然包含“等于”，但题目问的是确定关系，最准确的选项是“等于”。

## 第14题

小杨的父母最近刚刚给他买了一块华为手表，他说手表上跑的是鸿蒙，这个鸿蒙是（ ）。

【知识点：计算机基础常识】

A. 小程序
B. 计时器
C. 操作系统
D. 神话人物

答案：C

解析：本题答案选 C。华为手表上运行的“鸿蒙”指 HarmonyOS，是设备使用的操作系统，负责管理硬件资源并支撑应用运行。小程序是运行在平台上的应用形态，计时器只是功能或部件，神话人物不是这里的计算机概念。

## 第15题

中国计算机学会（CCF）在 2024 年 1 月 27 日的颁奖典礼上颁布了王选奖，王选先生的重大贡献是（ ）。

【知识点：计算机基础常识】

A. 制造自动驾驶汽车
B. 创立培训学校
C. 发明汉字激光照排系统
D. 成立方正公司

答案：C

解析：本题答案选 C。王选先生的重要贡献是主持研制汉字激光照排系统，推动中文出版印刷技术从铅字排版走向计算机照排。A 的自动驾驶汽车、B 的培训学校都不是王选奖名称所对应的人物贡献；D“成立方正公司”与相关产业化背景有关，但不是本题所问的重大技术贡献。
