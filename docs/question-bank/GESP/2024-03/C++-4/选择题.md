## 第1题

若函数声明为 `int f(int &x){ x+=3; return x; }`，则对声明的变量 `int a=3`，下面哪个调用能够改变 $a$ 的值（ ）。

A. `f(&a);`
B. `f(*a);`
C. `f(a);`
D. `f(a-3);`

答案：C

解析：`f` 的形参是 `int &x`，它需要绑定到一个可修改的 `int` 左值。`f(a);` 中的 `a` 正好是这样的变量，进入函数后 `x` 就是 `a` 的别名，执行 `x += 3` 会把 `a` 从 $3$ 改为 $6$。`f(&a);` 传入的是 `int *`，类型不匹配；`f(*a);` 把 `a` 当成指针解引用，语法不成立；`f(a-3);` 的实参是表达式结果，不能绑定到非 `const` 左值引用。因此选 C。

## 第2题

下面 C++ 代码执行后，输出的是（ ）。

```cpp
int main()
{
    int x[] = {2, 0, 2, 4};
    char geSP[] = "Grade Examination of SP";

    cout << geSP[sizeof(x)] << endl;

    cout << endl;
    return 0;
}
```

A. G
B. e
C. n
D. P

答案：C

解析：`x` 是含 $4$ 个 `int` 的数组，通常每个 `int` 占 $4$ 字节，因此 `sizeof(x)` 的值是 $16$。`geSP` 中字符串 `"Grade Examination of SP"` 从下标 $0$ 开始计数：`geSP[0]` 是 `G`，`geSP[4]` 是 `e`，`geSP[16]` 是单词 `Examination` 的最后一个字符 `n`。程序输出的是 `geSP[16]`，所以选 C。

## 第3题

下面 C++ 代码执行后输出是（ ）。

```cpp
int foo(float *f)
{
    return int(*f * 2);
}

int main()
{
    float fnum[10] = {1.1};
    fnum[1] = foo(fnum);
    cout << fnum[0] + fnum[1] << endl;

    cout << endl;
    return 0;
}
```

A. $1$
B. $1.1$
C. $3$
D. $3.1$

答案：D

解析：`float fnum[10] = {1.1};` 只显式初始化第 $0$ 个元素，故 `fnum[0] = 1.1`。调用 `foo(fnum)` 时，数组名转换为指向首元素的指针，`*f` 就是 `fnum[0]`，于是 `*f * 2 = 2.2`，再转换为 `int` 得到 $2$。所以 `fnum[1]` 被赋值为 $2$，最终 `fnum[0] + fnum[1] = 1.1 + 2 = 3.1`，选 D。

## 第4题

对二维数组 `int arr[3][16];`，则 `arr[1]` 占用内存的大小为（ ）字节。

A. $4$
B. $16$
C. $48$
D. $64$

答案：D

解析：`arr` 是 $3$ 行、每行 $16$ 个 `int` 的二维数组。`arr[1]` 表示第 $1$ 行这一整个一维数组，包含 $16$ 个 `int`。在本题选项对应的环境中，`int` 占 $4$ 字节，因此这一行占用 $16 \times 4 = 64$ 字节。A 只是一格 `int` 的大小，B 只数了元素个数，C 是 $12$ 个 `int` 的大小，都不符合 `arr[1]` 这一整行的含义。

## 第5题

对二维数组 `int arr[3][16];`，若 `arr` 的地址是 $0x28cbc0$，则 `arr[1]` 的值是（ ）。

A. 0x28cbc4
B. 0x28cbd0
C. 0x28cc00
D. 不确定

答案：C

解析：二维数组在内存中按行连续存放，`arr[1]` 是第 $1$ 行的起始地址，比 `arr` 的起始地址多跳过一整行。每行有 $16$ 个 `int`，一行大小是 $16 \times 4 = 64$ 字节，也就是十六进制的 `0x40`。所以 `arr[1]` 的地址为 `0x28cbc0 + 0x40 = 0x28cc00`，选 C。A、B 分别只偏移了 $4$ 和 $16$ 字节；D 错在二维数组行地址偏移是确定的。

## 第6题

下面 C++ 代码执行后输出是（ ）。

```cpp
int main()
{
    char *p = "I love GESP!";
    cout << p + 5 << endl;

    cout << endl;
    return 0;
}
```

A. `e`
B. `I lov`
C. `e GESP!`
D. `GESP!`

答案：C

解析：`p` 指向字符串 `"I love GESP!"` 的首字符。按下标计数，`p[0]` 是 `I`，`p[1]` 是空格，`p[2]` 到 `p[5]` 依次是 `l`、`o`、`v`、`e`。表达式 `p + 5` 指向字符 `e`，而 `cout` 输出 `char *` 时会从该地址一直输出到字符串结束的空字符，因此结果是 `e GESP!`。A 只输出一个字符，不符合 `char *` 的输出规则；B 是前 $5$ 个字符；D 对应的起始位置应更靠后。

## 第7题

下面 C++ 代码执行以后输出的是（ ）。

```cpp
int rc = 5;
int main()
{
    int rc;
    cout << ++rc << endl;

    cout << endl;
    return 0;
}
```

A. $0$
B. $1$
C. $6$
D. 不确定

答案：D

解析：全局变量 `rc` 的值是 $5$，但 `main` 内又定义了一个同名的局部变量 `int rc;`。在 `main` 的作用域内，局部变量会遮蔽全局变量，所以 `++rc` 操作的是这个未初始化的自动变量。读取未初始化局部变量的值没有确定结果，严格说会导致未定义行为，因此不能推出输出 $0$、$1$ 或 $6$ 中的任何一个，选 D。

## 第8题

下面 C++ 函数中采用的算法是（ ）。

```cpp
int fib(int n)
{
    int i, f[n] = {0, 1};

    for (int i = 2; i <= n; i++)
        f[i] = f[i - 1] + f[i - 2];

    return f[n];
}
```

A. 递推
B. 递归
C. 迭代
D. 循环

答案：A

解析：函数先给出 `f[0]`、`f[1]` 的初始值，再用关系式 `f[i] = f[i - 1] + f[i - 2]` 从小到大求出后面的项，这正是用已知项推出未知项的递推思想。代码里虽然使用了 `for` 循环，也可以说是迭代实现，但本题问“采用的算法”，更准确的算法类别是递推。它没有在函数内部调用 `fib` 自身，所以不是递归。

## 第9题

插入排序在最好情况下的时间复杂度是（ ）。

A. $O(1)$
B. $O(N/2)$
C. $O(N)$
D. $O(N^2)$

答案：C

解析：插入排序最好情况出现在序列已经有序时。此时每个元素只需要和前一个位置做一次比较，就能确认不需要继续向前移动；总共处理约 $N - 1$ 个元素，因此时间复杂度是 $O(N)$。$O(1)$ 忽略了仍要扫描各元素这一事实；$O(N^2)$ 是平均或最坏情况下大量移动时的复杂度；$O(N/2)$ 在复杂度记号中会省略常数因子，规范写法仍是 $O(N)$。

## 第10题

在如下的 C++ 代码执行后，设第 11 和 12 行的输出地址值分别为 $X$ 和 $Y$，则下面正确的是（ ）。

```cpp
struct pass {
    int no;
    char name[20];
    int level;
};

int main()
{
    struct pass XiaoYang;

    cout << "&XiaoYang=" << &XiaoYang << endl;     //第11行
    cout << "&(XiaoYang.no)=" << &(XiaoYang.no) << endl; //第12行

    cout << endl;
    return 0;
}
```

A. $X > Y$
B. $X = Y$
C. $X < Y$
D. 不确定

答案：B

解析：`XiaoYang` 是一个结构体对象，`no` 是它声明的第一个非静态成员。该结构体没有继承关系，成员按声明顺序存放，第一个成员从结构体对象起始位置开始，因此对象地址 `&XiaoYang` 与首成员地址 `&(XiaoYang.no)` 数值相同。后面的 `name`、`level` 才会排在更高地址处，所以不存在 $X > Y$ 或 $X < Y$ 的情况，选 B。

## 第11题

如果文件 `1.txt` 中的内容如下，则执行下面 C++ 代码时，注释了 `####` 那行代码所输出的 $x$ 的值为（ ）。

```
50 2024 3.16 I
love
GESP!
```

执行的 C++ 代码如下：

```cpp
int main()
{
    ifstream fin;
    string line;
    int x;
    fin.open("1.txt", ios::in);

    for (int i = 0; i < 2; i++) {
        fin >> line;
        cout << line << endl;
    }
    fin >> x;
    cout << x << endl; //####
    
    cout << endl;
    return 0;
}
```

A. $5$
B. $2024$
C. $3$
D. $0$

答案：C

解析：输入运算符 `>>` 以空白字符分隔数据。循环执行两次，第一次把 `"50"` 读入 `line`，第二次把 `"2024"` 读入 `line`，此时文件中的下一个内容是 `3.16`。接着执行 `fin >> x`，向 `int` 变量读取时会先读出整数部分 $3$，遇到小数点停止，因此 `x` 的值为 $3$。$2024$ 已经在循环中被读走；$5$ 不是完整 token；本次读取能够得到整数前缀，不会输出 $0$。

## 第12题

执行下列 C++ 代码时输出中的第 2 行是（ ）。

```cpp
int main()
{
    char *s[] = {(char*)"2024", (char*)"3.16", (char*)"GESP"};

    for (int i = 0; i < 2; i++) {
        cout << *s + i << endl;
    }

    cout << endl;
    return 0;
}
```

A. $2024$
B. $3.16$
C. $024$
D. $3$

答案：C

解析：`s` 是一个字符指针数组，`*s` 等价于 `s[0]`，也就是指向字符串 `"2024"` 首字符的指针。循环中输出的是 `*s + i`，这是对第一个字符串做指针偏移，而不是访问 `s[i]`。当 `i = 0` 时输出 `2024`；当 `i = 1` 时从下标 $1$ 的字符开始输出，结果是 `024`。因此第 $2$ 行是 `024`，选 C；B 是如果输出 `s[1]` 才会得到的结果。

## 第13题

C++ 语言中下面哪个关键字能够限定对象的作用域（ ）。

A. extern
B. static
C. inline
D. public

答案：B

解析：按本题考查的关键字作用，`static` 可以用于限定对象或函数的可见范围。例如文件作用域中的 `static` 变量具有内部链接性，只能在当前源文件内使用；块内 `static` 变量虽然有静态存储期，但名字仍受块作用域限制。`extern` 表示外部声明，常用于引用别处定义的对象；`inline` 主要与内联函数或内联变量规则有关；`public` 是类成员访问控制说明符，不是用来限定普通对象作用域的关键字。

## 第14题

小杨的父母最近刚刚给他买了一块华为手表，他说手表上跑的是鸿蒙，这个鸿蒙是（ ）。

A. 小程序
B. 计时器
C. 操作系统
D. 神话人物

答案：C

解析：华为手表上运行的“鸿蒙”指 HarmonyOS，它属于操作系统，负责管理设备硬件、应用运行和用户交互。小程序是在操作系统或宿主应用上运行的应用形态，计时器只是具体功能，神话人物是“鸿蒙”一词的文化含义而不是设备上运行的软件系统。因此选 C。

## 第15题

中国计算机学会（CCF）在 2024 年 1 月 27 日的颁奖典礼上颁布了王选奖，王选先生的重大贡献是（ ）。

A. 制造自动驾驶汽车
B. 创立培训学校
C. 发明汉字激光照排系统
D. 成立方正公司

答案：C

解析：王选先生的代表性贡献是主持研制汉字激光照排系统，推动中文出版印刷从铅字排版转向计算机照排。自动驾驶汽车、培训学校都不是他的主要贡献；方正公司与相关成果产业化有关，但题目问的是王选先生的重大技术贡献，因此应选“发明汉字激光照排系统”。
