## 第1题

唯一分解定理描述的内容是（ ）？

A. 任意整数都可以分解为素数的乘积
B. 每个合数都可以唯一分解为一系列素数的乘积
C. 两个不同的整数可以分解为相同的素数乘积
D. 以上都不对

答案：B

解析：唯一分解定理通常表述为：每个大于 $1$ 的整数都能唯一地写成若干素数的乘积，且只允许交换因子的顺序。选项 B 说每个合数可以唯一分解为一系列素数的乘积，符合本题考查口径。A 说“任意整数”过宽，$0$、$1$ 以及负整数都不能直接按该说法处理；C 与“唯一”矛盾；D 因 B 正确而不成立。

## 第2题

贪心算法的核心思想是（ ）？

A. 在每一步选择中都做当前状态下的最优选择
B. 在每一步选择中都选择局部最优解
C. 在每一步选择中都选择全局最优解
D. 以上都对

答案：A

解析：贪心算法是在构造解的过程中，每一步都根据当前状态做出看起来最优的选择，再希望这些选择组合成整体最优解。A 准确强调了“每一步”和“当前状态下的最优选择”。B 的“局部最优解”容易让人误以为每一步已经得到一个完整的解，而贪心每一步通常只是选择一个决策；C 说每一步都选择全局最优解并不现实，如果每一步都已经能确定全局最优，就不需要贪心策略；D 因 C 不成立而不选。

## 第3题

下面的 C++ 代码片段用于计算阶乘。请在横线处填入（ ），实现正确的阶乘计算。

```cpp
int factorial(int n) {
    if (n == 0 || n == 1) {
        return 1;
    } else {
        ________________________ // 在此处填入代码
    }
}
```

A. `return n * factorial(n - 1);`
B. `return factorial(n - 1) / n;`
C. `return n * factorial(n);`
D. `return factorial(n / 2) * factorial(n / 2);`

答案：A

解析：阶乘满足递推关系 $n! = n \times (n-1)!$，并且代码已经把 $0!$ 和 $1!$ 作为递归终止条件返回 $1$。因此递归分支应返回 `n * factorial(n - 1)`。B 把乘法写成除法；C 继续调用 `factorial(n)`，参数没有变小，会无限递归；D 把问题拆成两个 `n / 2` 的阶乘相乘，不等于 $n!$。

## 第5题

辗转相除法也被称为（ ）

A. 高斯消元法
B. 费马定理
C. 欧几里德算法
D. 牛顿迭代法

答案：C

解析：辗转相除法通过反复使用 $\gcd(a,b)=\gcd(b,a\bmod b)$ 求最大公约数，这一算法也称欧几里德算法。高斯消元法用于解线性方程组，费马定理属于数论定理，牛顿迭代法常用于求方程近似根或函数零点，均不是辗转相除法的别名。

## 第6题

下面的代码片段用于计算斐波那契数列。该代码的时间复杂度是（ ）？

```cpp
int fibonacci(int n) {
    if (n <= 1) {
        return n;
    } else {
        return fibonacci(n - 1) + fibonacci(n - 2);
    }
}
```

A. $O(1)$
B. $O(n)$
C. $O(2^n)$
D. $O(\log n)$

答案：C

解析：设运行时间为 $T(n)$，代码在 $n>1$ 时会递归计算 `fibonacci(n - 1)` 和 `fibonacci(n - 2)`，因此有 $T(n)=T(n-1)+T(n-2)+O(1)$。由于没有记忆化，大量子问题会被重复计算，递归调用树规模按指数级增长，考试口径下记为 $O(2^n)$。$O(1)$、$O(n)$ 和 $O(\log n)$ 都低估了重复递归调用的数量。

A. `result = to_string(sum % 10) + result;`
B. `result = to_string(carry % 10) + result;`
C. `result = to_string(sum / 10) + result;`
D. `result = to_string(sum % 10 + carry) + result;`

答案：A

## 第8题

给定序列：$1, 3, 6, 9, 17, 31, 39, 52, 61, 79, 81, 90, 96$。使用以下代码进行二分查找查找元素 $82$ 时，需要循环多少次，即最后输出的 `times` 值为（ ）。

```cpp
int binarySearch(const std::vector<int>& arr, int target) {
    int left = 0;
    int right = arr.size() - 1;
    int times = 0;
    while (left <= right) {
        times++;
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            cout << times << endl;
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    cout << times << endl;
    return -1;
}
```

A. 2
B. 5
C. 3
D. 4

答案：D

解析：数组下标为 $0$ 到 $12$。第 1 次 `left=0,right=12,mid=6`，`arr[6]=39<82`，令 `left=7`；第 2 次 `mid=9`，`arr[9]=79<82`，令 `left=10`；第 3 次 `mid=11`，`arr[11]=90>82`，令 `right=10`；第 4 次 `mid=10`，`arr[10]=81<82`，令 `left=11`。此时 `left=11,right=10`，循环结束并输出 `times=4`。

## 第9题

下面的代码片段用于判断一个正整数是否为素数。请对以下代码进行修改，使其能正确实现相应功能。（ ）

```cpp
bool isPrime(int num) {
    if (num < 2) {
        return false;
    }
    for (int i = 2; i * i < num; ++i) {
        if (num % i == 0) {
            return false;
        }
    }
    return true;
}
```

A. `num < 2` 应该改为 `num <= 2`
B. 循环条件 `i * i < num` 应该改为 `i * i <= num`
C. 循环条件应该是 `i <= num`
D. 循环体中应该是 `if (num % i != 0)`

答案：B

解析：判断素数只需要检查是否存在不超过 $\sqrt{num}$ 的因子，因此循环条件应包含 `i * i == num` 的情况。原代码用 `i * i < num` 会漏掉平方数的平方根，例如 `num=4` 时 `i=2` 不进入循环，错误返回 `true`。A 会把 $2$ 判为非素数；C 虽然也能检查到因子，但范围过大，不是这段代码的必要修正；D 把“能整除则不是素数”改成“不能整除则不是素数”，逻辑相反。

## 第11题

素数的线性筛法时间复杂度为（ ）。

A. $O(n)$
B. $O(n \log \log n)$
C. $O(n \log n)$
D. $O(n^2)$

答案：A

解析：线性筛的关键是每个合数只被它的最小质因子筛掉一次，因此总的标记次数与 $n$ 同阶，时间复杂度为 $O(n)$。$O(n\log\log n)$ 是埃氏筛的常见复杂度口径；$O(n\log n)$ 和 $O(n^2)$ 都不是线性筛的复杂度。

## 第12题

归并排序的基本思想是（ ）。

A. 动态规划
B. 分治
C. 贪心算法
D. 回溯算法

答案：B

解析：归并排序先把序列不断二分为规模更小的子序列，分别排序后再把两个有序子序列合并成更大的有序序列，这正是“分解、解决、合并”的分治思想。动态规划强调重叠子问题和状态转移，贪心强调每步局部选择，回溯强调搜索与撤销选择，都不是归并排序的基本思想。

## 第13题

在快速排序中，选择的主元素（pivot）会影响算法的（ ）。

A. 不影响
B. 时间复杂度
C. 空间复杂度
D. 时间复杂度和空间复杂度

答案：B

解析：快速排序每次按 pivot 把序列划分成两部分。若 pivot 选得接近中位数，左右子问题规模较均衡，递归层数约为 $log n$，平均时间复杂度可达 $O(n log n)$；若每次都选到当前序列中的极端元素，就会退化成一边为空、一边规模减一的划分，时间复杂度可变为 $O(n^2)$。因此 pivot 的选择会直接影响快速排序的时间复杂度。A 说“不影响”不对；C 只看空间复杂度，忽略了最主要的时间影响；D 把空间也并列作为主要答案，不如 B 准确。

## 第14题

递归函数在调用自身时，必须满足（ ），以避免无限递归？

A. 有终止条件
B. 函数参数递减（或递增）
C. 函数返回值固定
D. 以上都对

答案：A

解析：递归函数要避免无限调用自身，必须存在能直接返回、不再递归调用的终止条件，也就是递归基。参数递减或递增只是常见的“逐步接近终止条件”的方式，并非所有递归都必须表现为简单单调变化；返回值固定也不是递归能停止的条件。因此按“必须满足”的单选口径应选 A。

## 第15题

假设给定链表为：$1 \rightarrow 3 \rightarrow 5 \rightarrow 7 \rightarrow \text{nullptr}$。若调用 `searchValue(head, 5)`，函数返回值为（ ）。

```cpp
int searchValue(ListNode* head, int target) {
    while (head != nullptr) {
        if (head->val == target) {
            return 1;
        }
        head = head->next;
    }
    return 0;
}
```

A. 返回 1
B. 返回 0
C. 死循环，无法返回
D. 返回 -1

答案：A

解析：函数从头节点开始顺序遍历链表。第一次访问值为 $1$，不等于目标 $5$，移动到下一个节点；第二次访问 $3$，仍不等；第三次访问 $5$，满足 `head->val == target`，立即 `return 1`。因此不会继续走到 $7$ 或 `nullptr`，也不会执行末尾的 `return 0`；函数每轮都会执行 `head = head->next`，所以不是死循环。
