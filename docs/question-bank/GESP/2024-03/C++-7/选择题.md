## 第1题

下列关于排序的说法，正确的是（ ）。

A. 冒泡排序是最快的排序算法之一。
B. 快速排序通常是不稳定的。
C. 最差情况，$N$ 个元素做归并排序的时间复杂度为 $O(N)$。
D. 以上均不正确。

答案：B

解析：正确答案是 B。快速排序的常见原地划分过程会把元素跨区间交换，相等关键字的相对次序可能改变，因此通常不是稳定排序。A 错在冒泡排序平均和最坏时间复杂度都是 $O(N^2)$，不能称为最快的排序算法之一；C 错在归并排序最差时间复杂度为 $O(N\log N)$，不是 $O(N)$。由于 B 正确，D 也不能选。

## 第2题

下面的程序属于哪种算法（ ）。

```cpp
int pos[8];
void queen(int n) {
    for (int i = 0; i < 8; i++) {
        pos[n] = i;
        bool attacked = false;
        for (int j = 0; j < n; j++)
            if (pos[n] == pos[j] || pos[n] + n == pos[j] + j || pos[n] - n == pos[j] - j) {
                attacked = true;
                break;
            }
        if (attacked)
            continue;
        if (n == 7) {
            return;
        } else {
            queen(n + 1);
        }
    }
}
```

A. 贪心算法
B. 动态规划
C. 深度优先搜索
D. 广度优先搜索

答案：C

解析：正确答案是 C。函数 `queen(n)` 表示正在尝试第 $n$ 行皇后的位置，枚举当前行的列号后，先检查是否与前面各行在同列或同斜线上冲突；若不冲突，就递归调用 `queen(n + 1)` 继续处理下一行。这种“先沿一个可行分支不断深入，失败后回退再试下一个分支”的过程是深度优先搜索。它不是只做局部最优选择的贪心算法，也没有保存子问题最优值的动态规划；广度优先搜索会按层扩展状态，而本程序使用递归沿单一路径深入。

## 第3题

下面有关 C++ 类的说法，错误的是（ ）。

A. C++ 类对象销毁时，会执行析构函数。
B. C++ 类可以通过定义构造函数实现自动类型转换。
C. C++ 类可以通过重载 `[ ]` 运算符实现通过给定下标访问数组成员的元素。
D. C++ 类可以包含任意类型的成员变量。

答案：D

解析：正确答案是 D。本题问“错误的是”。A 正确，类对象生命周期结束时会调用析构函数。B 正确，单参数构造函数在未加 `explicit` 时可参与隐式类型转换。C 正确，重载 `operator[]` 后可以写出类似数组下标访问的形式。D 的“任意类型”过于绝对：例如类不能包含自身类型的非静态成员对象，也不能以 `void` 或函数类型作为普通数据成员，因此该说法错误。

## 第4题

一个连通的简单无向图，共有 $28$ 条边，则该图至少有（ ）个顶点。

A. $6$
B. $7$
C. $8$
D. $9$

答案：C

解析：正确答案是 C。含 $n$ 个顶点的简单无向图最多有 $\frac{n(n-1)}{2}$ 条边。若 $n=7$，最多只有 $\frac{7\times 6}{2}=21$ 条边，不足 $28$ 条；若 $n=8$，最多有 $\frac{8\times 7}{2}=28$ 条边，完全图 $K_8$ 本身也是连通图。因此至少需要 $8$ 个顶点，A、B 不够，D 不是“至少”。

## 第5题

以下哪个方案不能合理解决或缓解哈希表冲突（ ）。

A. 在每个哈希表项处，使用单链表管理该表项的冲突元素。
B. 建立额外的单链表，用来管理所有发生冲突的元素。
C. 使用不同的哈希函数再建立一个哈希表，用来管理所有发生冲突的元素。
D. 用新元素覆盖发生冲突的哈希表项。

答案：D

解析：正确答案是 D。本题问“不能合理解决或缓解冲突”。A 是链地址法，把同一槽位上的元素挂成链表；B 将冲突元素另行组织，虽然描述不如标准教材术语严谨，但仍是在保留冲突元素的前提下管理它们；C 类似再哈希或使用辅助哈希结构，也是在重新安置冲突元素。D 直接用新元素覆盖旧表项会丢失原有数据，既不能保证查找正确，也不是合理的冲突处理方法。

## 第6题

已知一颗二叉树的中序遍历序列为：$\{C\ F\ B\ A\ E\ D\ G\}$，后序遍历序列为：$\{F\ C\ B\ E\ G\ D\ A\}$，则下列说法中正确的是（ ）。

A. 该树是平衡二叉树。
B. 该树的高为 $4$。
C. 该树有 $4$ 个叶节点。
D. 以上说法都不对。

答案：B

解析：正确答案是 B。后序遍历最后一个结点是根，所以根为 $A$；在中序序列中，$A$ 左侧为 $\{C,F,B\}$，右侧为 $\{E,D,G\}$。左子树后序为 $\{F,C,B\}$，根为 $B$，其左子树为以 $C$ 为根、$F$ 为右孩子的链；右子树后序为 $\{E,G,D\}$，根为 $D$，左右孩子分别为 $E$、$G$。按“结点层数”计高度，最长路径 $A\to B\to C\to F$ 共有 $4$ 层。叶结点只有 $F,E,G$ 共 $3$ 个，所以 C 错；结点 $B$ 的左右子树高度差超过 $1$，不是平衡二叉树，A 错；因此 D 也错。

## 第7题

以下关于二叉排序树的说法，正确的是（ ）。

A. 二叉排序树的中序遍历序列一定是有序的。
B. 在含 $n$ 个节点的二叉排序树中查找元素，最差情况的时间复杂度为 $O(\log(n))$。
C. 二叉排序树一定是二叉平衡树。
D. 以上说法都不对。

答案：A

解析：正确答案是 A。二叉排序树满足左子树关键字小于根、右子树关键字大于根（若允许重复，一般按约定放在一侧），中序遍历按“左子树、根、右子树”的顺序访问，因此得到有序序列。B 错在最差情况下二叉排序树可能退化成一条链，查找复杂度为 $O(n)$，不是 $O(\log n)$。C 错在二叉排序树不一定保持平衡，只有 AVL、红黑树等额外维护平衡性质的数据结构才有相应保证。

## 第8题

已知 $x$ 为 `double` 类型的变量，且值大于 $0$，则下列表达式的值一定大于 $0$ 的是（ ）。

A. $\sin(x) / x$
B. $\exp(x) - x$
C. $\log(x) - x$
D. $x \times x - x$

答案：B

解析：正确答案是 B。对任意 $x>0$，有 $e^x\ge 1+x$，所以 $\exp(x)-x\ge 1>0$。A 中 $\sin(x)$ 可能为负，例如 $x=\frac{3\pi}{2}$ 时 $\sin(x)<0$；C 中取 $x=1$ 得 $\log(1)-1=-1$；D 中 $x^2-x=x(x-1)$，当 $0<x<1$ 时为负。因此只有 B 一定大于 $0$。

## 第9题

一个简单有向图有 $10$ 个结点、$30$ 条边。再增加多少条边可以成为完全图。（ ）

A. $60$
B. $70$
C. $15$
D. $20$

答案：A

解析：正确答案是 A。简单有向完全图通常不含自环，任意两个不同顶点之间两个方向的弧都存在，因此 $10$ 个结点最多有 $10\times 9=90$ 条有向边。已有 $30$ 条边，还需增加 $90-30=60$ 条边。B 把总边数或差值算错，C、D 则明显不足以补到完全图。

## 第10题

下列选项中，哪个可能是下图的深度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/mbvv0hd9.png)

A. $8, 6, 1, 5, 3, 4, 2, 10, 7, 12, 11, 9$
B. $7, 8, 6, 4, 2, 1, 5, 3, 12, 9, 11, 10$
C. $8, 10, 12, 9, 11, 4, 5, 3, 2, 1, 6, 7$
D. $7, 8, 10, 9, 11, 12, 4, 5, 1, 2, 3, 6$

答案：C

解析：正确答案是 C。深度优先遍历允许按不同的邻接点顺序访问，因此题目问“可能是”。从 $8$ 出发，可以先访问 $10$，再访问 $12$；在 $12$ 处先走到 $9$，再到 $11$，回溯后再从 $12$ 走向 $4$，随后沿 $4,5,3,2,1$ 这一支继续深入，最后回溯到 $8$ 再访问 $6$、$7$，可得到 C 的序列。A 在访问 $8,6,1$ 后很快跳到另一侧分支，不能解释为当前深度分支自然回溯后的结果；B、D 也都存在尚未沿当前分支深入完就跨到其他分支的情况。

## 第11题

下面 `schedule` 函数的时间复杂度为（ ）。

```cpp
#include <algorithm>
using namespace std;
struct activity {
    int id, start, end;
};
bool compare(activity a, activity b) {
    return a.end < b.end;
}
int schedule(int n, activity * p) {
    sort(p, p + n, compare);
    int cnt = 0, end = 0;
    for (int i = 0; i < n; i++) {
        if (p[i].start >= end) {
            end = p[i].end;
            cnt++;
        }
    }
    return cnt;
}
```

A. $O(n)$
B. $O(\log(n))$
C. $O(n \log(n))$
D. $O(n^2)$

答案：C

解析：正确答案是 C。函数先调用 `sort(p, p + n, compare)` 按结束时间排序，标准库排序平均和最坏复杂度通常按 $O(n\log n)$ 计；比较函数只比较两个 `end` 字段，是 $O(1)$。随后 `for` 循环只线性扫描一次数组，复杂度为 $O(n)$。总复杂度由排序主导，为 $O(n\log n)$，不是单纯的 $O(n)$、$O(\log n)$ 或 $O(n^2)$。

## 第12题

下面 `search` 函数的平均时间复杂度为（ ）。

```cpp
int search(int n, int * p, int target) {
    int low = 0, high = n;
    while (low <= high) {
        int middle = (low + high) / 2;
        if (target == p[middle]) {
            return middle;
        } else if (target > p[middle]) {
            low = middle + 1;
        } else {
            high = middle - 1;
        }
    }
    return -1;
}
```

A. $O(n)$
B. $O(\log(n))$
C. $O(1)$
D. 可能无法返回

答案：B

解析：这段代码采用二分查找思想：每次取当前区间中点 `middle`，如果没有命中，就把查找范围缩小到左半段或右半段，区间长度大约减半。因此平均查找次数随区间不断折半增长，时间复杂度为 $O(log n)$。A 的 $O(n)$ 是顺序查找量级，C 的 $O(1)$ 只有能直接定位目标时才可能；D 描述的是程序是否返回的问题，不是二分查找的平均时间复杂度。编写真实代码时还要注意数组边界，若数组合法下标是 `0` 到 `n - 1`，`high` 通常应初始化为 `n - 1`。

## 第13题

下面 `count_triple` 函数的时间复杂度为（ ）。

```cpp
int count_triple(int n) {
    int cnt = 0;
    for (int a = 1; a <= n; a++)
        for (int b = a; a + b <= n; b++)
            for (int c = b; a + b + c <= n; c++)
                if (a * a + b * b == c * c)
                    cnt++;
    return cnt;
}
```

A. $O(N)$
B. $O(N^2)$
C. $O(N^3)$
D. $O(N^4)$

答案：C

解析：正确答案是 C。程序有三层循环，分别枚举 $a$、$b$、$c$，循环条件 `a + b <= n` 和 `a + b + c <= n` 会剪掉一部分组合，但满足这些条件的三元组数量仍与 $n^3$ 同阶，例如 $a,b,c$ 都在一定比例范围内时仍有大量可行组合。循环体中的平方和判断是常数时间，所以总时间复杂度为 $O(N^3)$。A、B 低估了三重枚举的规模，D 则多算了一层循环。

## 第14题

下面程序的输出为（ ）。

```cpp
#include <iostream>
using namespace std;
int down(int n) {
    if (n <= 1)
        return n;
    return down(n - 1) + down(n - 2) + down(n - 3);
}
int main() {
    cout << down(6) << endl;
    return 0;
}
```

A. $6$
B. $13$
C. $20$
D. 无法正常结束。

答案：A

解析：正确答案是 A。递归基为 `n <= 1` 时直接返回 `n`，因此 $f(-1)=-1$、$f(0)=0$、$f(1)=1$。继续计算：$f(2)=f(1)+f(0)+f(-1)=0$，$f(3)=f(2)+f(1)+f(0)=1$，$f(4)=2$，$f(5)=3$，$f(6)=6$。程序递归会终止，输出 `down(6)` 的结果为 $6$，所以不是 $13$、$20$，也不是无法正常结束。

## 第15题

下面的程序使用邻接矩阵表达的带权无向图，则从顶点 0 到顶点 3 的最短距离为（ ）。

```cpp
int weight[4][4] = {
    {0, 2, 5, 8},
    {2, 0, 1, 7},
    {5, 1, 0, 4},
    {8, 7, 4, 0}
};
```

A. $6$
B. $7$
C. $8$
D. $9$

答案：B

解析：正确答案是 B。直接从 $0$ 到 $3$ 的边权为 $8$；经 $1$ 到 $3$ 的距离是 $2+7=9$；经 $2$ 到 $3$ 的距离是 $5+4=9$。若走 $0\to 1\to 2\to 3$，距离为 $2+1+4=7$，比直接边更短。其他绕路如 $0\to 2\to 1\to 3$ 为 $5+1+7=13$。因此最短距离为 $7$。
