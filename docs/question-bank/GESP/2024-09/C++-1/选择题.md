## 第1题

据有关资料，山东大学于 1972 年研制成功 DJL-1 计算机，并于 1973 年投入运行，其综合性能居当时全国第三位。DJL-1 计算机运算控制部分所使用的磁心存储元件由磁心颗粒组成，设计存贮周期为 $2\mu s$（微秒）。那么该磁心存储元件相当于现代计算机的（ ）。

A. 内存
B. 磁盘
C. CPU
D. 显示器

答案：A

解析：题干中的“磁心存储元件”用于存放计算机运行时正在使用的数据和指令，属于早期计算机的主存储设备。现代计算机中承担类似作用的是内存，因此 A 正确。磁盘主要负责长期保存文件和数据；CPU 负责运算和控制；显示器只是输出设备，都不符合“存储元件”的作用。

## 第2题

C++ 程序执行出现错误，不太常见的调试手段是（ ）。

A. 阅读源代码
B. 单步调试
C. 输出执行中间结果
D. 跟踪汇编码

答案：D

解析：阅读源代码可以检查语法和逻辑，单步调试可以观察程序逐行执行过程，输出中间结果可以定位变量值从哪里开始异常，这三种都是常见调试方法。汇编码比 C++ 源代码更底层，和具体机器指令、编译结果有关，初学阶段排查普通 C++ 程序错误时很少直接跟踪汇编码，因此 D 符合“不太常见”。

## 第3题

在 C++ 中，下列表达式错误的是（ ）。

A. `cout << "Hello,GESP!" << endl;`
B. `cout << 'Hello,GESP!' << endl;`
C. `cout << """Hello,GESP!""" << endl;`
D. `cout << "Hello,GESP!' << endl;`

答案：D

解析：字符串常量必须用一对匹配的双引号括起来。D 中开头使用双引号，结尾却使用单引号，导致双引号没有闭合，表达式明显错误。A 是正常输出字符串；C 中连续的双引号可按相邻字符串字面量理解；B 使用单引号包住多个字符，写法不适合作为字符串，但本题考查的明显错误是 D 的引号不配对。

## 第4题

C++ 表达式 `10 - 3 * 2` 的值是（ ）。

A. 14
B. 4
C. 1
D. 0

答案：B

解析：C++ 中乘法 `*` 的优先级高于减法 `-`，所以表达式不是从左到右先算 `10 - 3`，而是先算 `3 * 2 = 6`，再算 `10 - 6 = 4`。A 对应误把减法先算；C、D 都不是按运算优先级计算得到的结果，因此选 B。

## 第5题

C++ 中，假设 $N$ 为正整数 $10$，则 `cout << (N / 3 + N % 3)` 将输出（ ）。

A. 6
B. 4.3
C. 4
D. 2

答案：C

解析：`N` 是整数 $10$，因此 `N / 3` 执行整数除法，只保留商，结果是 $3$，不会得到小数。`N % 3` 表示取余数，$10$ 除以 $3$ 的余数是 $1$。所以 `(N / 3 + N % 3) = 3 + 1 = 4`。B 把整数除法误当成小数运算；A、D 都没有同时正确计算商和余数。

## 第6题

C++ 语句 `printf("6%2={%d}", 6%2)` 执行后的输出是（ ）。

A. "6%2={6%2}"
B. 6%2={6%2}
C. 0=0
D. 6%2={0}

答案：D

解析：按题目选项的命题意图，格式串中前面的 `6%2={` 和最后的 `}` 是希望原样输出，`%d` 用来输出后面表达式 `6%2` 的值。因为 $6$ 能被 $2$ 整除，`6 % 2 = 0`，所以目标输出应为 `6%2={0}`，对应 D。严格按照 C/C++ 的 `printf` 规则，如果要输出字面量 `%`，格式串应写成 `%%`，即 `printf("6%%2={%d}", 6%2);`；本题答案按原题意图判断。

## 第7题

成功执行下面的 C++ 代码，先后从键盘上输入 5 回车和 2 回车，输出是（ ）。

```cpp
cin >> a;
cin >> b;
cout << a + b;
```

A. 将输出整数 7
B. 将输出 52，5 和 2 之间没有空格
C. 将输出 5 和 2，5 和 2 之间有空格
D. 执行结果不确定，因为代码段没有显示 a 和 b 的数据类型

答案：D

解析：`cin >> a` 和 `cin >> b` 只是说明把两个输入值分别读入变量，不能推出变量类型。若 `a`、`b` 是 `int`，输入 $5$ 和 $2$ 后 `a + b` 为 $7$；若是字符、字符串或其他类型，`+` 的含义或结果可能不同。A、B、C 都默认了某一种类型，但题目代码段没有给出定义，所以输出无法唯一确定，选 D。

## 第8题

下面 C++ 代码执行后输出是（ ）。

```cpp
int Sum = 0;
for (int i = 0; i < 10; i++)
    Sum += i;
cout << Sum;
```

A. 55
B. 45
C. 10
D. 9

答案：B

解析：`for` 循环从 `i = 0` 开始，每次循环结束后执行 `i++`，只要 `i < 10` 就继续执行循环体。因此 `i` 依次取 $0,1,2,\ldots,9$，不会取到 $10$。循环体 `Sum += i` 把这些数累加到初值为 $0$ 的 `Sum` 中，得到 $0+1+\cdots+9=45$。A 的 $55$ 是误把 $10$ 也加了进去，C、D 则混淆了循环次数或最后一个 `i` 值。

## 第9题

下面 C++ 代码执行后输出的是（ ）。

```cpp
int N = 0;
for (int i = 0; i < 10; i++)
    N += 1;
cout << N;
```

A. 55
B. 45
C. 10
D. 9

答案：C

解析：本题与上一题不同，循环体不是把 `i` 加到 `N` 中，而是每次固定执行 `N += 1`。`i` 从 $0$ 到 $9$，共满足条件并执行循环 $10$ 次，所以 `N` 从 $0$ 增加 $10$ 次，最后变为 $10$。A、B 是把本题误看成累加 `i`；D 是把最后一次 `i` 的取值误当成 `N` 的值。

## 第10题

下面 C++ 代码执行后输出的是（ ）。

```cpp
int N = 0;
for (int i = 1; i < 10; i += 2){
    if (i % 2 == 1)
        continue;
    N += 1;
}
cout << N;
```

A. 5
B. 4
C. 2
D. 0

答案：D

解析：`i` 的初值是 $1$，每次增加 $2$，在 `i < 10` 的条件下会依次取 $1,3,5,7,9$，这些值全部是奇数。每次循环中 `i % 2 == 1` 都成立，于是执行 `continue`，直接进入下一轮循环，跳过后面的 `N += 1`。因此 `N` 从头到尾没有增加，最终输出 $0$。A、B、C 都忽略了 `continue` 会跳过本轮剩余语句。

## 第11题

下面 C++ 代码执行时输入 14+7 后，正确的输出是（ ）。

```cpp
int P;
printf("请输入正整数P：");
scanf("%d", &P);
if (P % 3 || P % 7)
    printf("第5行代码%d, %d", P % 3, P % 7);
else
    printf("第7行代码%2d", P % 3 && P % 7);
```

A. 第5行代码2, 0
B. 第5行代码1, 0
C. 第7行代码 1
D. 第7行代码 0

答案：A

解析：`scanf("%d", &P)` 按整数格式读取输入，遇到 `14+7` 时会先读入整数 $14$，在 `+` 处停止，后面的 `+7` 不会参与这次赋值。此时 `P % 3 = 14 % 3 = 2`，`P % 7 = 14 % 7 = 0`。逻辑或 `||` 只要有一个操作数非零就为真，所以 `P % 3 || P % 7` 为真，执行 `if` 分支，输出 `第5行代码2, 0`。C、D 错在进入了 `else` 分支；B 的余数计算错误。

## 第12题

执行下面 C++ 代码后得到的输出是（ ）。

```cpp
int count = 0, i, s;
for (i = 0, s = 0; i < 20; i++, count++)
    s += i++;
cout << s << " " << count;
```

A. 190 20
B. 95 10
C. 90 19
D. 90 10

答案：D

解析：循环初始化后 `i = 0, s = 0, count = 0`。循环体中的 `s += i++` 使用后置自增，会先把当前 `i` 加入 `s`，再让 `i` 增加 $1$；每轮结束处的 `i++, count++` 又让 `i` 增加 $1$，并让 `count` 增加 $1$。所以每轮开始进入循环体时，`i` 依次为 $0,2,4,\ldots,18$，`s = 0+2+4+\cdots+18 = 90`，一共执行 $10$ 轮，`count = 10`，输出 `90 10`。

## 第13题

下面 C++ 代码拟用于计算整数 $N$ 的位数，比如对 123 则输出 `123是3位整数`，但代码中可能存在 bug。下面有关描述正确的是（ ）。

```cpp
int N, N0, rc = 0;
cout << "请输入整数：";
cin >> N;
N0 = N;
while (N) {
    rc++;
    N /= 10;
}
printf("%d是%d位整数\n", N, rc);  // L11
```

A. 变量 N0 占用额外空间，可以去掉
B. 代码对所有整数都能计算出正确位数
C. L11 标记的代码行简单修改后可以对正整数给出正确输出
D. L11 标记的代码行的输出格式有误

答案：C

解析：循环用 `N /= 10` 不断去掉最后一位，确实可以通过 `rc++` 统计正整数的位数，但它也会把变量 `N` 最终改成 $0$。因此 L11 直接输出 `N` 时，不能输出原来的输入值。代码前面已经用 `N0 = N` 保存了原始整数，只要把 L11 中第一个输出项从 `N` 改成 `N0`，就能对正整数输出类似 `123是3位整数` 的结果，所以 C 正确。A 错在 `N0` 并非无用；B 错在没有处理所有整数，例如 $0$ 和负数并不能按题意正确统计；D 的格式本身不是主要问题。

## 第14题

下面的 C++ 代码用于求连续输入的若干正五位数的百位数之和。例如输入 32488 25731 41232 0，则输出 3 个正五位数的百位数之和为 13。有关描述错误的是（ ）。

```cpp
int M, Sum = 0, rc = 0;
cout << "请输入正整数：";
cin >> M;

while (M) {
    M = (M / 100 % 10);  // L6
    Sum += M;
    rc++;
    cin >> M;
}

cout << rc << "个正五位数的百位数之和为" << Sum;
```

A. 执行代码时如果输入 23221 23453 12345 11111 0，则最后一行 Sum 的值是 10
B. 执行代码时如果输入 2322 2345 1234 1111 0，程序也能运行
C. 将代码标记为 L6 那行改为 `M = (M % 1000 / 100);`，同样能实现题目要求
D. 将代码标记为 L6 那行改为 `M = (M % 100 / 10);`，同样能实现题目要求

答案：D

解析：五位数的百位可以用 `M / 100 % 10` 取得：先去掉十位和个位，再对 $10$ 取余保留当前个位，也就是原数的百位。`M % 1000 / 100` 先保留后三位，再取其中的百位，同样正确。`M % 100 / 10` 只保留最后两位后再除以 $10$，取得的是十位数字，不是百位数字，所以 D 的描述错误。A 给出的输入百位分别为 $2,4,3,1$，和为 $10$；B 中虽然输入不是五位数，但代码仍能按循环和算式运行，只是不符合题目“正五位数”的数据含义。

## 第15题

如果一个正整数 $N$ 能够表示为 $X \times (X+1)$ 的形式，这里称它是一个“兄弟数”。例如，输入 6，则输出“6 是一个兄弟数”。下面 C++ 代码用来判断 $N$ 是否为一个“兄弟数”，在横线处应填入的代码可从 i)-iv) 中选择，则有几个能完成功能？（ ）。

```cpp
int N;
cin >> N;
for (int i = 0; i <= N; i++)
    if (__________________)
        cout << N << "是一个兄弟数\n";
```

i) `N == i * (i + 1)`  
ii) `N == i * (i - 1)`  
iii) `N / (i + 1) == i`  
iv) `N / (i - 1) == i`

A. 1
B. 2
C. 3
D. 4

答案：B

解析：i) 直接判断 `N == i * (i + 1)`，与“兄弟数”定义完全一致，可以使用。ii) 虽然写成 `i * (i - 1)`，但当 `i = X + 1` 时，它也等于 $X \times (X+1)$，循环范围包含这样的 `i`，也可以找到兄弟数。iii) 用整数除法判断会误判，例如 $N=8$、`i=2` 时，`8 / 3 == 2` 成立，但 $8$ 不能表示为 $X \times (X+1)$。iv) 当 `i=1` 时会计算 `N / (i - 1)`，也就是除以 $0$，程序不安全。可用的只有 i) 和 ii)，共 $2$ 个，选 B。
