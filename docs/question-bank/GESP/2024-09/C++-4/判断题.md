# 判断题

来源：

## 第1题

代码 `int a = 10; int* p = &a;` 可以正确定义指针和初始化指针。

答案：正确

解析：`int a = 10;` 定义了一个整型变量，`int* p` 定义了一个指向 `int` 的指针变量。`&a` 取得变量 `a` 的地址，用它初始化 `p` 后，`p` 正好指向 `a`，所以这段代码可以正确定义并初始化指针。

## 第2题

在 C++ 中，引用传递允许函数修改传递给它的参数的值。

答案：正确

解析：引用形参相当于给实参起了一个别名，函数体内通过引用形参读写的就是调用者传入的那个对象。例如 `void f(int &x) { x++; }` 调用后会改变实参的值。因此引用传递允许函数修改传入参数。

## 第3题

指针的大小与其所指向的变量的数据类型的大小相同。

答案：错误

解析：指针变量保存的是地址，指针本身的大小通常由运行平台和地址宽度决定，例如同一平台上 `int*`、`double*` 的大小通常相同。它所指向对象的大小由对象类型决定，`int` 和 `double` 可能不同。因此不能说指针大小与所指向变量的数据类型大小相同。

## 第4题

二维数组的行的大小必须在定义时确定，列的大小可以动态变化。

答案：错误

解析：普通 C++ 二维数组如 `int a[3][4];` 在定义时需要确定每一维的大小，尤其列数必须明确，这样编译器才能计算 `a[i][j]` 的位置。题干说“列的大小可以动态变化”不符合普通二维数组的声明规则，所以判断为错误。

## 第5题

递推算法通过逐步求解当前状态和前一个或几个状态之间的关系来解决问题。

答案：正确

解析：递推通常先给出初始状态，再用相邻状态之间的关系逐步推出后续状态。例如斐波那契数列可由前两项推出当前项，阶乘可由前一个乘积乘上当前因子得到。题干强调“当前状态和前一个或几个状态之间的关系”，正是递推的核心。

## 第6题

选择排序是稳定的排序算法。

答案：错误

解析：标准选择排序每轮会从未排序部分选出最小元素并与当前位置交换，这个交换可能让相等元素的相对顺序改变。例如序列 `[2a, 2b, 1]` 第一轮会把 `1` 与 `2a` 交换，结果为 `[1, 2b, 2a]`，两个值同为 `2` 的元素顺序被颠倒。因此选择排序不是稳定排序。

## 第7题

插入排序的时间复杂度总是比冒泡排序低。

答案：错误

解析：插入排序和冒泡排序的平均、最坏时间复杂度通常都是 $O(n^2)$，不能说插入排序“总是”更低。即使在某些近乎有序的数据上插入排序可能更快，冒泡排序配合提前结束标志时最好情况也可以达到 $O(n)$。题干用了绝对表述，因此错误。

## 第8题

在 C++ 中，如果没有捕获到异常（没有匹配的 `catch` 块），程序会继续执行而不会终止。

答案：错误

解析：C++ 异常抛出后会沿调用链寻找匹配的 `catch`。如果一直没有找到匹配的处理块，程序不会当作正常流程继续执行，而会调用异常终止机制，通常导致程序终止。因此题干说“继续执行而不会终止”是错误的。

## 第9题

以下代码用递推法求斐波那契数列的第 $n$ 项，时间复杂度为指数级。

```cpp
int fibonacci(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;

    int f0 = 0; // F(0)
    int f1 = 1; // F(1)
    int current;

    for (int i = 2; i <= n; i++) {
        current = f0 + f1; // F(n) = F(n-1) + F(n-2)
        f0 = f1;
        f1 = current;
    }

    return current;
}
```

答案：错误

解析：这段代码使用循环递推求斐波那契数列，循环变量 `i` 从 $2$ 到 $n$，每轮只做一次加法和两次赋值，因此循环次数与 $n$ 成正比，时间复杂度是 $O(n)$。指数级复杂度通常出现在朴素递归反复计算相同子问题的写法中，本题代码没有这种重复递归调用。

## 第10题

执行下面 C++ 代码后，输出的是 $20$。

```cpp
int point(int* p){
    return *p * 2;
}

int main() {
    int a = 10;
    int* p = &a;
    *p = point(p);
    cout << *p << endl;
}
```

答案：正确

解析：`p = &a` 后，`p` 指向变量 `a`，此时 `*p` 的值是 `10`。调用 `point(p)` 时，函数返回 `*p * 2`，即 `20`；随后执行 `*p = point(p);`，把 `20` 写回 `p` 指向的 `a`。最后 `cout << *p` 输出的就是 `20`。
