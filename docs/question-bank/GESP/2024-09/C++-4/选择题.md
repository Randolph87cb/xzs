## 第1题

在 C++ 中，（ ）正确定义了一个返回整数值并接受两个整数参数的函数。

A. `int add(int a, int b) { return a + b; }`
B. `void add(int a, int b) { return a + b; }`
C. `int add(a, b) { return a + b; }`
D. `void add(int a, int b) { return a - b; }`

答案：A

解析：函数定义需要同时写出返回类型、函数名和每个形参的类型。A 中 `int` 表示函数返回整数，`int a, int b` 表示接收两个整数参数，函数体返回 `a + b` 的整数结果。B、D 的返回类型是 `void`，不能返回整数值；C 中形参缺少类型，语法不完整。

## 第2题

在 C++ 中，形参与实参的关系描述正确的是（ ）。

A. 形参在函数调用时指定，实参在函数定义时传递
B. 形参在函数定义时指定，实参在函数调用时传递
C. 形参和实参可以互换
D. 形参和实参必须是完全相同的类型，不能有任何差异。

答案：B

解析：形参写在函数定义或声明的参数列表中，用来描述函数内部接收的变量；实参出现在函数调用处，是实际传给函数的值或对象。调用时实参按参数传递规则赋给对应形参，因此 B 的描述正确。A 把定义和调用的位置说反了，C 不能互换概念，D 说“不能有任何差异”过于绝对，例如存在合法的类型转换。

## 第3题

运行以下代码，屏幕上将输出（ ）。

```cpp
#include <iostream>
using namespace std;

int var = 100;

void function() {
    int var = 200;
    cout << var << " ";
    cout << ::var << " ";
}

int main() {
    cout << var << " ";
    function();
    var += 100;
    cout << var << " ";
    return 0;
}
```

A. 100 200 100 200
B. 100 200 100 300
C. 100 200 200 200
D. 100 200 200 300

答案：A

解析：全局变量 `var` 初值为 `100`，`main` 中第一次 `cout << var` 输出全局变量，所以先输出 `100`。`function` 里又定义了局部变量 `var = 200`，它遮蔽全局同名变量，因此 `cout << var` 输出 `200`；`::var` 用作用域解析符访问全局变量，输出 `100`。函数返回后执行全局 `var += 100`，全局变量变为 `200`，最后输出 `200`。

## 第4题

运行下面代码，屏幕上输出是（ ）。

```cpp
int arr[3] = {24, 9, 7};
int* p = arr;
p++;
cout << *p << endl;
```

A. 24
B. 9
C. 7
D. 不确定

答案：B

解析：数组名 `arr` 在这里转换为指向首元素 `arr[0]` 的指针，所以 `p` 起初指向值为 `24` 的位置。`p++` 让 `int*` 前进一个 `int` 元素，改为指向 `arr[1]`。此时 `*p` 取到第二个元素，输出 `9`。

## 第5题

运行下面代码片段的结果是（ ）。

```cpp
int x = 20;
int y = 24;
int* p = &x;
int* q = &y;
p = q;
```

A. 将 $x$ 赋值为 $24$
B. 将 $y$ 赋值为 $20$
C. 将 $q$ 指向 $x$ 的地址
D. 将 $p$ 指向 $y$ 的地址

答案：D

解析：`p` 和 `q` 都是指针变量，保存的是地址。开始时 `p` 保存 `x` 的地址，`q` 保存 `y` 的地址；执行 `p = q;` 是把 `q` 中保存的地址复制给 `p`，不会修改 `x` 或 `y` 的值，也不会改变 `q` 的指向。因此执行后 `p` 也指向 `y`。

## 第6题

在 C++ 中，（ ）正确定义一个名为 student 的结构体，其中包含一个 name 字符数组和一个 age 整数？

A. `struct student { char name[20]; int age; };`
B. `student struct { char name[20]; int age; };`
C. `student struct { string name; int age; };`
D. `struct student { char[20] name; int age; };`

答案：A

解析：结构体定义的基本格式是 `struct 结构体名 { 成员声明; };`，字符数组成员应写成 `char name[20];`。A 同时满足结构体名为 `student`、包含 `name` 字符数组和 `age` 整数。B、C 把 `struct` 和结构体名的顺序写反；D 的数组声明顺序错误，`char[20] name` 不是 C++ 成员声明写法。

## 第7题

在 C++ 中，（ ）正确声明了一个 3 行 4 列的二维数组。

A. `int arr[3, 4];`
B. `int arr[3][4];`
C. `int arr[4][3];`
D. `int arr(3, 4);`

答案：B

解析：二维数组声明按“类型 数组名[行数][列数]”书写，`int arr[3][4];` 表示有 $3$ 行、每行 $4$ 个 `int`。A 中 `[3, 4]` 不是二维数组的声明方式；C 把行列写成了 $4$ 行 $3$ 列；D 是函数式写法，不是在定义二维数组。

## 第8题

一个二维数组定义为 `int arr[3][4];`（假设一个 int 变量占 4 个字节），则 `int arr[0]` 占用（ ）个字节的内存。

A. $3$
B. $4$
C. $12$
D. $16$

答案：D

解析：`arr` 是 $3$ 行 $4$ 列的二维数组，`arr[0]` 表示第 $0$ 行这一整行，类型相当于含 $4$ 个 `int` 的数组。每个 `int` 占 $4$ 字节，所以一行占 $4 \times 4 = 16$ 字节。

## 第9题

下面代码采用递推算法来实现整数 $n$ 的阶乘（$n! = n \times (n - 1) \times \ldots \times 2 \times 1$），则横线上应填写（ ）。

```cpp
int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        // 在此处填入代码
    }
    return result;
}
```

A. `result *= i;`
B. `result += i;`
C. `result *= result;`
D. `result += result;`

答案：A

解析：递推求阶乘时，循环变量 `i` 依次取 $2,3,\ldots,n$，`result` 应始终表示当前已经乘出的部分积。初值 `result = 1`，每轮应乘上新的因子 `i`，所以填 `result *= i;`。使用加法会变成求和，`result *= result` 则会反复平方，都不符合 $n!$ 的定义。

## 第10题

在排序算法中，稳定性指的是（ ）。

A. 排序后数据不会丢失
B. 排序后相同元素的相对顺序保持不变
C. 排序后数据不会被修改
D. 排序后数据的时间复杂度不变

答案：B

解析：排序算法的稳定性只讨论“关键字相同”的元素在排序前后的先后关系。例如两个成绩同为 $90$ 的记录，若排序后原来在前的那条记录仍在前，就保持稳定。它不表示数据是否丢失、元素值是否被修改，也不描述时间复杂度是否变化。

## 第11题

下面代码实现了冒泡排序函数，则横线上应填写（ ）。

```cpp
//交换数组arr的第i个元素和第j个元素
void swap(vector<int> &arr, int i, int j) {
    int tmp = arr[i];
    arr[i] = arr[j];
    arr[j] = tmp;
}

int bubble_sort(vector<int> &arr) {
    for (int i = arr.size() - 1; i > 0; i--) {
        bool flag = false; // 标志位
        // 在此处填入代码
        {
            if (arr[j] > arr[j + 1]) {
                swap(arr, i, j);
                flag = true;
            }
        }
        if (!flag)
            break; // 此轮“冒泡”未交换任何元素
    }
}
```

A. `for (int j = 0; j < arr.size() - 1; j++)`
B. `for (int j = arr.size() - 1; j > 0; j--)`
C. `for (int j = 0; j < i; j++)`
D. `for (int j = i-1; j <=0; j--)`

答案：C

解析：外层变量 `i` 从 `arr.size() - 1` 递减，表示当前这一轮还需要处理到的右边界。冒泡排序的内层循环应从左端开始，依次比较相邻位置 `j` 和 `j + 1`，为了保证 `j + 1` 不超过本轮右边界，条件应写成 `j < i`，所以横线处选 C：`int j = 0; j < i; j++`。A 每轮都扫描到数组末尾，没有利用外层边界缩小范围；B 从右向左走，但循环更新和相邻比较位置不匹配；D 从 `i - 1` 开始却要求 `j <= 0`，不能正常覆盖本轮应比较的区间。

## 第12题

上一题算法的时间复杂度为（ ）。

A. $O(n^2)$
B. $O(2^n)$
C. $O(1)$
D. $O(n)$

答案：A

解析：冒泡排序有两层循环。最坏情况下，外层大约执行 $n-1$ 轮，内层比较次数依次约为 $n-1,n-2,\ldots,1$，总比较次数为 $\frac{n(n-1)}{2}$，数量级是 $O(n^2)$。即使代码中有 `flag` 可在已有序时提前结束，算法的一般时间复杂度仍按最坏情况记为 $O(n^2)$。

## 第13题

下面代码实现了插入排序函数（升序），则横线上应填写（ ）。

```cpp
void insertion_sort(vector<int> &nums) {
    for (int i = 1; i < nums.size(); i++) {
        int base = nums[i], j = i - 1;
        // 在此处填入代码
        {
            nums[j + 1] = nums[j];
            j--;
        }
        nums[j + 1] = base;
    }
}
```

A. `while (j >= 0 && nums[j] > base)`
B. `while (j > 0 && nums[j] > base)`
C. `while (j >= 0 && nums[j] < base)`
D. `while (j > 0 && nums[j] < base)`

答案：A

解析：插入排序中，`base` 保存待插入元素，`j` 从它左边的已排序部分向前移动。升序排列时，只要 `j` 还没有越过数组左端，并且 `nums[j] > base`，就要把较大的 `nums[j]` 右移一格。条件必须写成 `j >= 0 && nums[j] > base`；若用 `j > 0` 会漏掉下标 `0`，若用 `< base` 则会按降序方向移动。

## 第14题

小杨用文件重定向实现在 `log.txt` 文件中输出日志，则下面横线上应填写（ ）。

```cpp
#include <iostream>
#include <fstream>
#include <string>
using namespace std;

int main() {
    ofstream log_file("log.txt");
    streambuf* original_cout = cout.rdbuf();
    cout.rdbuf(log_file.rdbuf());
    // 在此处填入代码
    cout.rdbuf(original_cout); // 恢复原始的标准输出缓冲区
    return 0;
}
```

A. `cout << "This output will go to the log file." << endl;`
B. `log_file << "This output will go to the log file." << endl;`
C. `cout >> "This output will go to the log file." >> endl;`
D. `log_file >> "This output will go to the log file." >> endl;`

答案：A

解析：代码先保存 `cout` 原来的缓冲区，再用 `cout.rdbuf(log_file.rdbuf())` 把 `cout` 的输出重定向到文件流 `log_file`。因此在恢复缓冲区之前，继续使用 `cout << ...` 会写入 `log.txt`。B 虽然直接写文件流也能向文件输出，但没有体现题干这段 `cout` 重定向的用法；C、D 使用 `>>`，这是输入运算符，方向错误。

## 第15题

运行下面的代码，屏幕上将输出（ ）。

```cpp
#include <iostream>
using namespace std;

int divide(int a, int b) {
    if (b == 0) {
        throw runtime_error("division by zero error ");
    }
    return a / b;
}

int main() {
    int x = 10;
    int y = 0; // 设为 0 会导致除零错误
    try {
        int result = divide(x, y);
        cout << "result: " << result << endl;
    } catch (const runtime_error& e) {
        cout << "caught an exception: " << e.what() << endl;
    }
    return 0;
}
```

A. `division by zero error result: caught an exception:`
B. `result: caught an exception: division by zero error`
C. `caught an exception: division by zero error`
D. `division by zero error caught an exception: division by zero error`

答案：C

解析：`main` 中 `y = 0`，调用 `divide(10, 0)` 时满足 `b == 0`，函数抛出 `runtime_error`，后面的 `return a / b` 不会执行，`try` 块中输出 `result:` 的语句也不会执行。异常被 `catch (const runtime_error& e)` 捕获，于是输出前缀 `caught an exception: ` 和 `e.what()` 中的错误信息，所以屏幕上显示对应的异常提示。
