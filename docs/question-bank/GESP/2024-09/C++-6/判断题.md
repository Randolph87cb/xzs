## 第1题

C++、Python 和 JAVA 等都是面向对象的编程语言。

答案：正确

解析：C++ 支持类、对象、封装、继承和多态；Python 也以对象模型组织数据和行为；Java 的类、接口、继承和多态是语言核心机制。因此题干把三者归为面向对象编程语言是正确的。

## 第2题

在 C++ 中，类的静态成员变量只能被该类对象的成员函数访问。

答案：错误

解析：静态成员变量属于类本身，而不是某一个对象独有。只要访问权限允许，它可以通过类名作用域如 `ClassName::member` 访问，也可以通过对象访问，并不限定“只能被该类对象的成员函数访问”。私有静态成员也可由类的静态成员函数访问。

## 第3题

栈是一种线性结构，可通过数组或链表来实现。二者相比，数组实现占用的内存较少，链表实现的入队和出队操作的时间复杂度较低。

答案：错误

解析：栈确实是线性结构，既可以用数组实现，也可以用链表实现；但栈的操作应称为入栈、出栈，不是入队、出队。更关键的是，两种实现的栈顶插入和删除通常都能做到 $O(1)$，链表实现并不比数组实现时间复杂度更低，还需要额外指针空间。

## 第4题

运行以下 C++ 代码，屏幕将输出 “derived class”。

```cpp
#include <iostream>
using namespace std;

class base {
public:
    virtual void show() {
        cout << "base class" << endl;
    }
};

class derived : public base {
public:
    void show() override {
        cout << "derived class" << endl;
    }
};

int main() {
    base* b;
    derived d;
    b = &d;

    b->show();
    return 0;
}
```

答案：正确

解析：`base::show()` 被声明为 `virtual`，`derived::show()` 使用 `override` 重写它。虽然变量 `b` 的静态类型是 `base*`，但它实际指向 `derived d`，调用 `b->show()` 会发生动态绑定，执行派生类版本并输出 `derived class`。

## 第5题

如下列代码所示的基类（base）及其派生类（derived），则生成一个派生类的对象时，只调用派生类的构造函数。

```cpp
#include <iostream>
using namespace std;

class base {
public:
    base() {
        cout << "base constructor" << endl;
    }
    ~base() {
        cout << "base destructor" << endl;
    }
};

class derived : public base {
public:
    derived() {
        cout << "derived constructor" << endl;
    }
    ~derived() {
        cout << "derived destructor" << endl;
    }
};
```

答案：错误

解析：创建派生类对象时，会先构造它的基类子对象，再构造派生类自身，所以会依次调用 `base()` 和 `derived()`。对象销毁时顺序相反，先调用派生类析构函数，再调用基类析构函数。因此“只调用派生类的构造函数”是错误的。

## 第6题

哈夫曼编码本质上是一种贪心策略。

答案：正确

解析：哈夫曼编码每一步都选择当前权值最小的两个结点合并，使较高频字符尽量靠近根、较低频字符可以更深，从而最小化带权路径长度。这个“每次选局部最小的两个权值”的过程正是贪心策略。

## 第7题

如果根结点的深度记为 $1$，则一棵恰有 $2024$ 个叶结点的二叉树的深度最少是 $12$。

答案：正确

解析：根结点深度记为 `1` 时，深度为 `d` 的二叉树最多有 `2^{d-1}` 个叶结点。若深度为 `11`，最多只有 `2^{10}=1024` 个叶结点，不够 `2024` 个；深度为 `12` 时最多 `2^{11}=2048` 个叶结点，已经可以容纳。因此最少深度是 `12`。

## 第8题

在非递归实现的树的广度优先搜索中，通常使用栈来辅助实现。

答案：错误

解析：树的广度优先搜索要按层访问结点，通常用队列保存“已经发现但尚未访问”的结点，保证先入队的上一层或同层结点先被处理。栈是后进先出结构，通常用于深度优先搜索的非递归实现，而不是广度优先搜索。

## 第9题

状态转移方程是动态规划的核心，可以通过递推方式表示问题状态的变化。

答案：正确

解析：动态规划需要定义状态，并用状态转移方程说明当前状态如何由更小或更早的状态得到。无论是一维递推、二维表格还是背包问题中的 `dp[i][w]`，核心都在于用递推关系描述状态变化，所以题干正确。

## 第10题

应用动态规划算法时，识别并存储重叠子问题的解是必须的。

答案：正确

解析：动态规划区别于朴素递归的重要点，就是把会重复出现的子问题结果保存下来，之后直接复用。用数组自底向上填表或用记忆化搜索缓存结果，都是为了避免重复计算重叠子问题，因此识别并存储这些解是动态规划的必要环节。
