## 第1题

下面关于 C++ 类和对象的说法，错误的是（ ）。

A. 类的析构函数可以为虚函数。
B. 类的构造函数不可以为虚函数。
C. class 中成员的默认访问权限为 private。
D. struct 中成员的默认访问权限为 private。

答案：D

解析：C++ 中析构函数可以声明为虚函数，常用于通过基类指针删除派生类对象时正确调用派生类析构函数，所以 A 正确。构造函数不能是虚函数，因为构造对象时虚函数机制依赖的对象动态类型尚未完整建立，所以 B 正确。`class` 的成员默认访问权限是 `private`，C 正确；`struct` 的成员默认访问权限是 `public`，不是 `private`，因此错误的是 D。

## 第2题

对于一个具有 $n$ 个顶点的无向图，若采用邻接矩阵表示，则该矩阵的大小为（ ）。

A. $n \times \frac{n}{2}$
B. $n \times n$
C. $(n - 1) \times (n - 1)$
D. $(n + 1) \times (n + 1)$

答案：B

解析：邻接矩阵用一行和一列分别对应一个顶点，矩阵中第 $i$ 行第 $j$ 列表示顶点 $i$ 与顶点 $j$ 之间是否有边或边权。图有 $n$ 个顶点，就需要 $n$ 行、$n$ 列，因此大小为 $n \times n$。无向图的矩阵通常关于主对角线对称，但这只减少存储优化的可能性，不改变标准邻接矩阵的尺寸。

## 第3题

设有编号为 A、B、C、D、E 的 5 个球和编号为 A、B、C、D、E 的 5 个盒子。现将这 5 个球投入 5 个盒子，要求每个盒子放一个球，并且恰好有两个球的编号与盒子编号相同，问有多少种不同的方法？（ ）。

A. 5
B. 120
C. 20
D. 60

答案：C

解析：先选出恰好编号相同的两个球和盒子，有 $C(5,2)=10$ 种选法。剩下 3 个球必须都不能放入同编号盒子，即 3 个元素的错排，数量为 $!3=2$。所以总数为 $10 \times 2 = 20$。若选 B 的 $120$，是没有限制时的全排列数；D 的 $60$ 没有扣除剩余 3 个位置中出现额外匹配的情况。

## 第4题

从甲地到乙地，可以乘高铁，也可以乘汽车，还可以乘轮船。一天中，高铁有 10 班，汽车有 5 班，轮船有 2 班。那么一天中乘坐这些交通工具从甲地到乙地共有多少种不同的走法？（ ）。

A. 100
B. 60
C. 30
D. 17

答案：D

解析：一次出行只选择一种交通工具的一班车，因此应使用加法原理，把互斥的选择数相加。高铁、汽车、轮船分别有 $10$、$5$、$2$ 种选择，共 $10+5+2=17$ 种。A、B、C 都体现了不同程度的乘法或多余组合，但题目不是连续乘坐多段交通工具。

## 第5题

$n$ 个结点的二叉树，执行释放全部结点操作的时间复杂度是（ ）。

A. $O(n)$
B. $O(n \log n)$
C. $O(\log n)$
D. $O(2^n)$

答案：A

解析：释放一棵二叉树的全部结点时，不论采用后序遍历递归释放还是显式栈遍历释放，每个结点都需要被访问并释放一次。每个结点上的释放和指针处理是常数时间，总工作量与结点数 $n$ 成正比，所以时间复杂度是 $O(n)$。树是否平衡只影响递归深度，不会让释放全部结点少于或多于线性数量级。

## 第6题

在一个单位圆上，随机分布 $n$ 个点，求这 $n$ 个点能被一个单位半圆周全部覆盖的概率（ ）。

A. $\frac{n}{2^{n-1}}$
B. $\frac{1}{n^2}$
C. $\frac{1}{n}$
D. $\frac{1}{2^n}$

答案：A

解析：固定圆上的某一个点，把它作为覆盖半圆的一端，其他 $n-1$ 个点都落在同一侧长度为半个圆周的弧内的概率是 $\left(\frac{1}{2}\right)^{n-1}$。连续随机分布下几乎不会出现边界重合，若所有点能被某个半圆覆盖，则可以唯一地取其中一个点作为这种半圆的起点来计数。共有 $n$ 个点可作为起点，所以概率为 $n \cdot \left(\frac{1}{2}\right)^{n-1}=\frac{n}{2^{n-1}}$。

## 第8题

上一题中，如果主函数为如下的程序，则最后的排列数是多少个？（ ）。

```cpp
int main() {
    int a[5] = {1, 2, 3, 4, 5};
    pailie(0, 5, a);
    return 0;
}
```

A. 120
B. 60
C. 240
D. 180

答案：A

解析：主函数调用 `pailie(0, 5, a)`，表示要对 5 个元素的所有位置依次选择元素。第 1 个位置有 5 种选择，第 2 个位置剩 4 种，之后依次为 3、2、1 种，因此排列总数为 $5! = 120$。递归基准 `begin == end` 时输出一次排列，所以最后输出 120 个排列。

## 第14题

下面程序的 `Merge_Sort` 函数时间复杂度为（ ）。

```cpp
void Merge(int a[], int left, int mid, int right) {
    int temp[right - left + 1];
    int i = left;
    int j = mid + 1;
    int k = 0;
    while (i <= mid && j <= right) {
        if (a[i] < a[j])
            temp[k++] = a[i++];
        else
            temp[k++] = a[j++];
    }
    while (i <= mid)
        temp[k++] = a[i++];
    while (j <= right)
        temp[k++] = a[j++];
    for (int m = left, n = 0; m <= right; m++, n++)
        a[m] = temp[n];
}

void Merge_Sort(int a[], int left, int right) {
    if (left == right)
        return;
    int mid = (left + right) / 2;
    Merge_Sort(a, left, mid);
    Merge_Sort(a, mid + 1, right);
    Merge(a, left, mid, right);
}
```

A. $O(n \log n)$
B. $O(n^2)$
C. $O(2^n)$
D. $O(\log n)$

答案：A

解析：`Merge_Sort` 每次把长度为 $n$ 的区间分成两个约为 $n/2$ 的子区间递归排序，然后用 `Merge` 在线性时间内合并。递推式为 $T(n)=2T(n/2)+O(n)$，递归层数是 $O(\log n)$，每层合并总量是 $O(n)$，所以总时间复杂度为 $O(n \log n)$。它不是二重循环式的 $O(n^2)$，也不是只沿一条递归链计算的 $O(\log n)$。

## 第15题

下面 `fibonacci` 函数的时间复杂度为（ ）。

```cpp
int fibonacci(int n) {
    if (n <= 1)
        return n;
    else
        return fibonacci(n - 1) + fibonacci(n - 2);
}
```

A. $O(1)$
B. $O(\phi^n), \phi = \frac{\sqrt{5}-1}{2}$
C. $O(n)$
D. $O(n \log n)$

答案：B

解析：这个递归函数在 $n>1$ 时会分别调用 `fibonacci(n - 1)` 和 `fibonacci(n - 2)`，除递归调用外只做常数次运算，因此时间满足 $T(n)=T(n-1)+T(n-2)+O(1)$。这个递推与 Fibonacci 数列同阶，调用次数呈指数增长，而不是 $O(1)$、$O(n)$ 或 $O(n \log n)$。按官方答案应选 B；不过官方选项中写的 $\phi=\frac{\sqrt{5}-1}{2}$ 字面值小于 $1$，若把它直接当作指数复杂度的底数会产生习惯上的歧义，通常 Fibonacci 型增长使用的底数是黄金比例 $\frac{\sqrt{5}+1}{2}$。
