## 第1题

下面关于链表和数组的描述，错误的是（ ）。

A. 数组大小固定，链表大小可动态调整。
B. 数组支持随机访问，链表只能顺序访问。
C. 存储相同数目的整数，数组比链表所需的内存多。
D. 数组插入和删除元素效率低，链表插入和删除元素效率高。

答案：C

解析：数组把元素连续存放，除了元素本身通常不需要为每个元素额外保存指针；链表的每个结点除数据外还要保存后继指针，双向链表还要保存前驱指针。因此存储相同数量的整数时，链表一般比数组占用更多内存，C 的说法反了，所以错误项是 C。A、B 描述的是数组固定连续存储和链表按指针顺序访问的典型特点；D 在已定位到插入、删除位置时成立。

## 第2题

通过（ ）操作，能完成在双向循环链表结点 $p$ 之后插入结点 $s$ 的功能（其中 `next` 域为结点的直接后继，`prev` 域为结点的直接前驱）。

A. `p->next->prev = s; s->prev = p; p->next = s; s->next = p->next;`
B. `p->next->prev = s; p->next = s; s->prev = p; s->next = p->next;`
C. `s->prev = p; s->next = p->next; p->next = s; p->next->prev = s;`
D. `s->next = p->next; p->next->prev = s; s->prev = p; p->next = s;`

答案：D

解析：在 `p` 后插入 `s` 时，必须先保留原来的后继结点：`s->next = p->next`，再让原后继的 `prev` 指向 `s`，同时令 `s->prev = p`，最后把 `p->next` 改为 `s`。D 的顺序没有丢失原后继。A、B 都先改了 `p->next`，后面再用 `p->next` 会指向新结点而不是原后继；C 在 `p->next = s` 之后执行 `p->next->prev = s`，实际会把 `s->prev` 设成 `s`。

## 第3题

对下面两个函数，说法错误的是（ ）。

```cpp
int sumA(int n) {
    int res = 0;
    for (int i = 1; i <= n; i++) {
        res += i;
    }
    return res;
}

int sumB(int n) {
    if (n == 1)
        return 1;
    int res = n + sumB(n - 1);
    return res;
}
```

A. sumA 体现了迭代的思想。
B. SumB 采用的是递归方式。
C. SumB 函数比 SumA 的时间效率更高。
D. 两个函数的实现的功能相同。

答案：C

解析：`sumA` 用 `for` 循环从 $1$ 加到 $n$，是迭代；`sumB` 调用自身 `sumB(n - 1)`，是递归。两个函数都计算 $1+2+\cdots+n$，并且都要处理约 $n$ 个数，时间复杂度都是 $O(n)$。递归版还会产生函数调用栈开销，不能说时间效率更高，所以错误项是 C。

## 第4题

有如下函数 `fun`，则 `fun(20, 12)` 的返回值为（ ）。

```cpp
int fun(int a, int b) {
    if (a % b == 0)
        return b;
    else
        return fun(b, a % b);
}
```

A. 20
B. 12
C. 4
D. 2

答案：C

解析：这个函数是递归形式的辗转相除法。`fun(20, 12)` 中 $20 \bmod 12=8$，转为 `fun(12, 8)`；$12 \bmod 8=4$，转为 `fun(8, 4)`；此时 $8 \bmod 4=0$，返回 `b`，即 $4$。

## 第7题

下面函数可以将 $n$ 的所有质因数找出来，其时间复杂度是（ ）。

```cpp
#include <iostream>
#include <vector>

vector<int> get_prime_factors(int n) {
    vector<int> factors;

    while (n % 2 == 0) {
        factors.push_back(2);
        n /= 2;
    }

    for (int i = 3; i * i <= n; i += 2) {
        while (n % i == 0) {
            factors.push_back(i);
            n /= i;
        }
    }

    if (n > 2) {
        factors.push_back(n);
    }

    return factors;
}
```

A. $O(n^2)$
B. $O(n \log n)$
C. $O(\sqrt{n} \log n)$
D. $O(n)$

答案：C

解析：代码先不断除以 $2$，再从 $3$ 开始只枚举奇数因子，并且循环条件是 `i * i <= n`，也就是候选因子只需要检查到当前 $n$ 的平方根。每次发现因子后，内层 `while` 会把这个因子全部除掉，使剩余的 `n` 变小；这些除法次数不会改变“枚举到平方根”为主的上界判断。在给出的选项中，C 比线性枚举的 D 更紧，也更符合这段因数分解代码的实际循环范围，所以选 C。

## 第8题

现在用如下代码来计算 $x^n$（$n$ 个 $x$ 相乘），其时间复杂度为（ ）。

```cpp
double quick_power(double x, unsigned n) {
    if (n == 0) return 1;
    if (n == 1) return x;
    return quick_power(x, n / 2) * quick_power(x, n / 2) * ((n & 1) ? x : 1);
}
```

A. $O(n)$
B. $O(n^2)$
C. $O(\log n)$
D. $O(n \log n)$

答案：A

解析：这段代码每次递归都会计算两遍 `quick_power(x, n / 2)`，没有把第一次结果保存下来复用，因此递推式是 $T(n)=2T(n/2)+O(1)$。由主定理或递归树可知总调用数与 $n$ 同阶，所以时间复杂度是 $O(n)$。若只递归一次并保存半次幂结果，才会是 $O(\log n)$。

## 第9题

假设快速排序算法的输入是一个长度为 $n$ 的已排序数组，且该快速排序算法在分治过程总是选择第一个元素作为基准元素。下面选项（ ）描述的是在这种情况下的快速排序行为。

A. 快速排序对于此类输入的表现最好，因为数组已经排序。
B. 快速排序对于此类输入的时间复杂度是 $O(n \log n)$。
C. 快速排序对于此类输入的时间复杂度是 $O(n^2)$。
D. 快速排序无法对此类数组进行排序，因为数组已经排序。

答案：C

解析：数组已经有序且每次都选第一个元素作基准时，基准会成为当前区间的最小元素，划分后左侧为空、右侧规模为 $n-1$。于是递归规模依次为 $n,n-1,n-2,\ldots$，每层划分又要线性扫描当前区间，总工作量为 $O(n+(n-1)+\cdots+1)=O(n^2)$。

## 第10题

考虑以下 C++ 代码实现的归并排序算法：

```cpp
void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;

    int L[n1], R[n2];

    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        }
        else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

void merge_sort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;

        merge_sort(arr, left, mid);
        merge_sort(arr, mid + 1, right);

        merge(arr, left, mid, right);
    }
}
```

对长度为 $n$ 的数组 `arr`，调用函数 `merge_sort(a, 0, n-1)`。在排序过程中 `merge` 函数的递归调用次数大约是（ ）。

A. $O(1)$
B. $O(n)$
C. $O(\log n)$
D. $O(n \log n)$

答案：B

解析：`merge_sort` 每次把一个长度大于 $1$ 的区间分成左右两半，并在两个子区间排好序后调用一次 `merge`。递归树的叶子对应 $n$ 个单元素区间，内部结点约为 $n-1$ 个；每个内部结点调用一次 `merge`，所以 `merge` 的调用次数是 $O(n)$，不是递归深度的 $O(\log n)$，也不是总合并工作量的 $O(n\log n)$。

## 第11题

现在有 $n$ 个人要过河，每只船最多载2人，船的承重为 $100\,\text{kg}$。下列代码中，数组 `weight` 中保存有 $n$ 个人的体重（单位为 $\text{kg}$），已经按从小到大排好序，代码输出过河所需要的船的数目，采用的思想为（ ）。

```cpp
int i, j;
int count = 0;
for (i = 0, j = n - 1; i < j; j--) {
    if (weight[i] + weight[j] <= 100) {
        i++;
    }
    count++;
}
printf("过河的船数: %d\n", count);
```

A. 枚举算法
B. 贪心算法
C. 迭代算法
D. 递归算法

答案：B

解析：数组已经按体重升序排列，代码每次处理当前最重的人 `weight[j]`：如果最轻的人 `weight[i]` 能和他同船，就让两人同船并移动 `i`；否则最重的人只能单独坐一条船。每一步都为当前最重的人作出能减少船数的局部最优选择，这正是贪心思想。

## 第12题

关于分治算法，以下哪个说法正确？

A. 分治算法将问题分成子问题，然后分别解决子问题，最后合并结果。
B. 归并排序不是分治算法的应用。
C. 分治算法通常用于解决小规模问题。
D. 分治算法的时间复杂度总是优于 $O(n \log(n))$。

答案：A

解析：分治的核心流程是“分解、解决、合并”：把原问题拆成规模更小的子问题，分别求解后再合并成原问题答案。归并排序就是典型分治算法，所以 B 错；分治常用于把大规模问题拆小，而不是只适合小规模问题；它的复杂度取决于分解方式和合并代价，不一定总优于 $O(n\log n)$。

## 第13题

根据下述二分查找法，在排好序的数组 $1, 3, 6, 9, 17, 31, 39, 52, 61, 79$ 中查找数值 $31$，循环 `while (left <= right)` 执行的次数为（ ）。

```cpp
int binary_search(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }
        else if (nums[mid] < target) {
            left = mid + 1;
        }
        else {
            right = mid - 1;
        }
    }

    return -1; // 如果找不到目标元素，返回-1
}
```

A. 1
B. 2
C. 3
D. 4

答案：C

解析：数组下标为 $0$ 到 $9$。第一次 `mid=(0+9)/2=4`，比较到 `17`，因为 $17<31$，令 `left=5`；第二次 `mid=(5+9)/2=7`，比较到 `52`，因为 $52>31$，令 `right=6`；第三次 `mid=(5+6)/2=5`，比较到 `31` 并返回。因此 `while` 循环共执行 $3$ 次。

## 第14题

以下关于高精度运算的说法错误的是（ ）。

A. 高精度计算主要是用来处理大整数或需要保留多位小数的运算。
B. 大整数除以小整数的处理的步骤可以是，将被除数和除数对齐，从左到右逐位尝试将除数乘以某个数，通过减法得到新的被除数，并累加商。
C. 高精度乘法的运算时间只与参与运算的两个整数中长度较长者的位数有关。
D. 高精度加法运算的关键在于逐位相加并处理进位。

答案：C

解析：普通高精度乘法要让一个数的每一位和另一个数的每一位相乘，再处理进位；若两个整数位数分别为 $m$ 和 $n$，时间通常与 $m \times n$ 有关，而不是只由较长者的位数决定。A、B、D 分别描述了高精度适用场景、除以小整数的竖式思想和加法进位处理，都是合理说法。

## 第15题

当 $n = 7$ 时，下面函数的返回值为（ ）。

```cpp
int fun(int n) {
    if (n == 1) return 1;
    else if (n >= 5) return n * fun(n - 2);
    else return n * fun(n - 1);
}
```

A. 105
B. 840
C. 210
D. 420

答案：C

解析：当 $n \ge 5$ 时函数返回 `n * fun(n - 2)`，当 $n<5$ 且不等于 $1$ 时返回 `n * fun(n - 1)`。因此 `fun(7)=7*fun(5)`，`fun(5)=5*fun(3)`，`fun(3)=3*fun(2)`，`fun(2)=2*fun(1)=2`。最终结果为 $7 \times 5 \times 3 \times 2 \times 1=210$。
