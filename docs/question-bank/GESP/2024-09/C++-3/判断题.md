## 第1题

$+1$ 和 $-1$ 的原码进行 $1 + (-1)$ 计算的结果是 $-2$。（ ）

【知识点：原码反码补码与整数表示】

答案：正确

解析：在原码中，`+1` 可表示为 `0000 0001`，`-1` 可表示为 `1000 0001`。如果直接按原码形式做加法，会得到 `1000 0010`，它在原码中表示 $-2$，所以题干说法正确。

## 第2题

`~1` 的输出值是 $-2$。（ ）

【知识点：位运算】

答案：正确

解析：`~` 会把每一位取反。补码表示下，`1` 的低位形如 `0000 0001`，取反后形如 `1111 1110`，这正是 $-2$ 的补码表示，所以 `~1` 的值为 $-2$。

## 第3题

`~1 = 1111 1110`。（ ）

【知识点：位运算】

答案：正确

解析：`~1` 是把整数 `1` 的各二进制位全部取反。若只写出低 $8$ 位，`0000 0001` 取反后就是 `1111 1110`，因此题干给出的位模式是正确的。

## 第4题

取 $X$ 的低四位，可以用 `Y = 0000 1111`，用 `X & Y` 获取 $X$ 的低四位。（ ）

【知识点：位运算】

答案：正确

解析：掩码 `0000 1111` 的低四位为 `1`，高四位为 `0`。`X & Y` 会保留 `X` 中与 `1` 相与的低四位，并把高四位清成 `0`，因此可以取得 `X` 的低四位。

## 第5题

下列程序输出的是 `A`。（ ）

```cpp
char x=65;
x++;
cout<<x++<<endl;
```

【知识点：运算符、表达式与类型转换】

答案：错误

解析：`char x = 65` 时，`x` 对应字符 `A`。执行 `x++` 后，`x` 先变为 $66$，即字符 `B`；`cout << x++` 输出递增前的当前值 `B`，随后才把 `x` 增加到 `C`。所以程序不会输出 `A`。

## 第6题

下列程序输出的是 $3$。（ ）

```cpp
string str="CHADai";
size_t pos = str.find('D');
cout<<pos<<endl;
```

【知识点：字符串处理】

答案：正确

解析：字符串下标从 $0$ 开始，`"CHADai"` 中 `C`、`H`、`A`、`D` 的下标分别是 $0,1,2,3$。`str.find('D')` 返回字符 `D` 第一次出现的位置，所以输出 $3$。

## 第7题

下列程序将输出 $1$。（ ）

```cpp
int a[10]={1};
cout<<a[1]<<endl;
```

【知识点：数组与字符数组】

答案：错误

解析：`int a[10] = {1};` 只把 `a[0]` 初始化为 $1$，其余元素会被补零初始化。`a[1]` 的值是 $0$，因此程序输出 $0$，不是 $1$。

## 第8题

下列程序将输出 `A`。（ ）

```cpp
int a=65;
cout<<(char)a<<endl;
```

【知识点：ASCII/字符编码】

答案：正确

解析：在常用 ASCII 编码中，十进制 $65$ 对应字符 `A`。`(char)a` 把整数 $65$ 转成对应字符输出，所以程序将输出 `A`。

## 第9题

16 进制数 `AB`，表示成 2 进制数是 `10101011`。（ ）

【知识点：进制转换与进制字面量】

答案：正确

解析：十六进制每一位对应 $4$ 位二进制。`A` 对应 `1010`，`B` 对应 `1011`，合在一起就是 `10101011`，所以题干说法正确。

## 第10题

`def`（十六进制）$= 103231$（五进制）。（ ）

【知识点：进制转换与进制字面量】

答案：错误

解析：`def`（十六进制）的十进制值是 $13\times16^2+14\times16+15=3567$。`103231`（五进制）的十进制值是 $1\times5^5+3\times5^3+2\times5^2+3\times5+1=3566$，两者不相等，所以题干说法错误。
