## 第1题

已知小写字母 b 的 ASCII 码为 $98$，下列 C++ 代码的输出结果是（ ）。

```cpp
#include <iostream>
using namespace std;
int main() {
    char a = 'b';
    a++;
    cout << a;
    return 0;
}
```

A. b
B. c
C. 98
D. 99

答案：B

解析：变量 `a` 的类型是 `char`，初值为字符 `'b'`。题目给出 `'b'` 的 ASCII 码为 $98$，执行 `a++` 后字符编码变为 $99$，对应小写字母 `c`，`cout << a` 按字符输出，因此输出 `c`。`98` 和 `99` 是按整数理解字符编码时的数值，不是这里的输出形式。

## 第2题

已知 a 为 int 类型变量，下列表达式不符合语法的是（ ）。

A. `&a + 3`
B. `+a & 3`
C. `a - - 4`
D. `a++3`

答案：D

解析：`a++` 是后缀自增表达式，后面不能直接再接一个整数常量，所以 `a++3` 不能被 C++ 语法正确解析。`&a + 3` 是指针加整数；`+a & 3` 可理解为 `(+a) & 3`，对整数做按位与；`a - - 4` 可理解为 `a - (-4)`，这些表达式都符合语法。

## 第3题

下列关于 C++ 语言中指针的叙述，不正确的是（ ）。

A. 指针变量中存储的是内存地址。
B. 指针变量指向的内存地址不一定能够合法访问。
C. 结构类型中的指针成员不能指向该结构类型。
D. 定义指针变量时必须指定其指向的类型。

答案：C

解析：结构体可以包含指向同类型结构体的指针成员，例如链表结点常写成 `struct Node { Node *next; };`，因此 C 的说法不正确。A 是指针的基本含义；B 也正确，指针中即使保存了某个地址，也可能是空指针、野指针或越界地址，访问不一定合法；D 正确，定义指针变量时需要给出被指向对象的类型，如 `int *p`。

## 第4题

下列关于 C++ 类的说法，错误的是（ ）。

A. 将 C++ 类对象通过值传递给函数参数时，会自动调用复制构造函数。
B. 将一个类的对象赋值给该类的另一个对象时，不会自动调用构造函数。
C. 定义 C++ 类对象时，一定会调用默认构造函数。
D. 构造派生类的对象时，一定会调用基类的构造函数。

答案：C

解析：定义类对象时会调用构造函数，但不一定是默认构造函数，例如 `Point p(1, 2);` 会调用带参数构造函数，所以 C 错。按值传参需要用实参构造形参对象，通常会调用复制构造函数或发生等价的拷贝初始化；已有对象之间赋值调用的是赋值运算符，不是构造函数；派生类对象构造时必须先构造其基类子对象，因此会调用基类构造函数。

## 第5题

某二叉树 $T$ 的先序遍历序列为：$\text{\{A B D C E G H F\}}$，中序遍历序列为：$\text{\{D B A H G E C F\}}$，则下列说法中正确的是（ ）。

A. $T$ 的高为 $5$
B. $T$ 有 $4$ 个叶节点
C. $T$ 是平衡树
D. 以上说法都不对

答案：A

解析：先序序列首元素为根，所以根为 $A$。在中序序列中，$A$ 左侧为 $\{D,B\}$，右侧为 $\{H,G,E,C,F\}$；左子树由先序 `B D` 与中序 `D B` 得到，结构为 $B$ 的左孩子是 $D$。右子树根为 $C$，其中左子树根为 $E$，$E$ 的左孩子为 $G$，$G$ 的左孩子为 $H$，$C$ 的右孩子为 $F$。最长根到叶路径为 $A \to C \to E \to G \to H$，按结点数计高为 $5$。叶结点只有 $D,H,F$ 共 $3$ 个，且结点 $C$ 的左右子树高度差超过 $1$，不是平衡树。

## 第6题

一棵完全二叉树有 $431$ 个结点，则叶结点有多少个？（ ）

A. 176
B. 215
C. 216
D. 255

答案：C

解析：完全二叉树按层序编号时，编号大于 $\lfloor n/2 \rfloor$ 的结点没有孩子，都是叶结点。这里 $n=431$，非叶结点数为 $\lfloor 431/2 \rfloor = 215$，叶结点数为 $431-215=216$。也可以用完全二叉树叶结点数公式 $\lceil n/2 \rceil$，得到 $\lceil 431/2 \rceil=216$。

## 第7题

下列关于树的说法，错误的是（ ）。

A. 二叉树的中序遍历与其深度优先遍历总是相同的。
B. 所有树都可以构造一颗二叉树与之一一对应。
C. 如果树的一个叶结点有两个不同的祖先结点，那么其中一个一定是另一个的祖先结点。
D. 树的结点不能有两个父结点。

答案：A

解析：二叉树的深度优先遍历可以有先序、中序、后序等不同访问次序，中序遍历只是其中一种，不能说总是与“深度优先遍历”相同，因此 A 错。一般树可以用孩子兄弟表示法转换为二叉树；树中从根到任一结点路径唯一，所以同一叶结点的两个祖先必然在这条路径上，其中一个是另一个的祖先；树的每个非根结点只有一个父结点。

## 第8题

一个简单无向图有 $10$ 个结点、$30$ 条边。再增加多少条边可以成为完全图。（ ）

A. 10
B. 15
C. 51
D. 60

答案：B

解析：含 $n$ 个结点的简单无向完全图有 $\frac{n(n-1)}{2}$ 条边。代入 $n=10$，最多边数为 $\frac{10 \times 9}{2}=45$。当前已有 $30$ 条边，还需要补上 $45-30=15$ 条边。

## 第9题

以下哪个方案可以合理解决或缓解哈希表冲突（ ）。

A. 丢弃发生冲突的新元素。
B. 用新元素覆盖发生冲突的元素。
C. 用新元素覆盖在冲突位置的下一个位置。
D. 将新元素放置在冲突位置之后的第一个空位。

答案：D

解析：哈希冲突指不同元素被映射到同一个表位置。开放定址法中的线性探测会从冲突位置继续向后查找，把新元素放入之后遇到的第一个空位，因此 D 是合理方案。丢弃新元素会丢失数据；用新元素覆盖原元素同样破坏已有数据；覆盖下一个位置也可能破坏另一个合法元素，都不能正确解决冲突。

## 第10题

一个迷宫，已知从起点不经过重复结点到达终点的路径有且仅有一条，则下面说法错误的是（ ）。

A. 可以使用深度优先搜索找到这条路径。
B. 可以使用广度优先搜索找到这条路径。
C. 该迷宫内与起点连通的结点，一定也与终点连通。
D. 该迷宫内与起点连通的结点及它们之间的路径可以抽象为无向无环图。

答案：D

解析：从起点到终点的简单路径唯一，并不推出起点所在连通部分没有环。例如某个旁支区域可以包含一个只通过一个入口接到主路径上的环，这不会产生第二条从起点到终点的简单路径，但连通部分已经不是无向无环图，所以 D 错。DFS 和 BFS 都能在图中搜索到存在的路径；在无向迷宫模型中，若某结点与起点连通，而终点也与起点连通，则该结点与终点同属一个连通分量。

## 第11题

下面程序的输出为（ ）。

```cpp
#include <iostream>
#include <cmath>
using namespace std;
int main() {
    cout << (int)log(8) << endl;
    return 0;
}
```

A. 2
B. 3
C. 8
D. 无法通过编译。

答案：A

解析：`<cmath>` 中的 `log` 默认表示自然对数，$\log(8) \approx 2.079$。表达式前的 `(int)` 会把这个 `double` 值转换为整数，正数转换时小数部分被截去，所以得到 $2$。程序包含了需要的头文件，能够通过编译；结果也不是以 $2$ 为底的对数 $3$。

## 第12题

下面程序的输出为（ ）。

```cpp
#include <iostream>
#define N 10
using namespace std;
int path[N][N];
int main() {
    for (int i = 1; i < N; i++)
        path[i][0] = i;
    for (int j = 1; j < N; j++)
        path[0][j] = j;
    for (int i = 1; i < N; i++)
        for (int j = 1; j < N; j++)
            path[i][j] = path[i - 1][j] + path[i][j - 1];
    cout << path[8][4] << endl;
    return 0;
}
```

A. 84
B. 495
C. 1012
D. 结果是随机的。

答案：C

解析：全局数组 `path` 初始为 $0$，随后设置边界 `path[i][0]=i`、`path[0][j]=j`。内部位置满足递推式 `path[i][j] = path[i-1][j] + path[i][j-1]`。逐行算到第 $8$ 行、第 $4$ 列：第 $4$ 列从上到下依次为 $4,11,26,56,112,210,372,627,1012$，因此 `path[8][4]` 为 $1012$。数组下标都在 $0$ 到 $9$ 之间，结果不是随机值。

## 第13题

上题中程序的时间复杂度为（ ）。

A. $O(1)$
B. $O(N)$
C. $O(N \log N)$
D. $O(N^2)$

答案：D

解析：按程序中的规模符号 $N$ 分析，前两个单层循环分别执行约 $N$ 次，是 $O(N)$；后面两重循环中，`i` 和 `j` 都从 $1$ 到 $N-1$，循环体执行 $(N-1)^2$ 次，是 $O(N^2)$。总时间为 $O(N)+O(N)+O(N^2)$，由二重循环主导，所以时间复杂度为 $O(N^2)$。

## 第14题

下面 `fib` 函数的时间复杂度为（ ）。

```cpp
int fib_rcd[MAX_N];
int fib(int n) {
    if (n <= 1)
        return 1;
    if (fib_rcd[n] > 0)
        return fib_rcd[n];
    return fib(n - 1) + fib(n - 2);
}
```

A. $O(n)$
B. $O(\phi^n)$，$\phi = \frac{\sqrt{5}-1}{2}$
C. $O(2^n)$
D. 无法正常结束。

答案：B

解析：这段函数虽然有数组 `fib_rcd`，也检查了 `fib_rcd[n] != 0`，但递归计算后没有把结果写回 `fib_rcd[n]`，所以并没有真正记忆化。计算 `fib(n)` 时会继续调用 `fib(n-1)` 和 `fib(n-2)`，而这些分支里又会反复计算相同的小规模问题，调用次数满足 Fibonacci 型递推 $T(n)=T(n-1)+T(n-2)+O(1)$，呈指数增长。选项 B 表示黄金比例级增长，比简单写成 $O(2^n)$ 更贴近朴素 Fibonacci 递归的增长阶。A 是有效记忆化或动态规划后才可能达到的线性复杂度；C 只是较粗的指数上界，不如 B 精确；D 错在递归参数会不断减小并到达 `n <= 1` 的终止条件。

## 第15题

下列选项中，哪个可能是下图的广度优先遍历序列（ ）。

![](https://cdn.luogu.com.cn/upload/image_hosting/opb9eqv0.png)

A. $1, 3, 5, 7, 4, 2, 6, 8, 9$
B. $9, 4, 2, 1, 3, 7, 5, 6, 8$
C. $1, 3, 5, 7, 6, 8, 9, 4, 2$
D. $9, 4, 7, 2, 1, 3, 5, 6, 8$

答案：A

解析：广度优先遍历要求按层扩展：先访问起点，再访问所有距离为 $1$ 的点，然后才访问距离为 $2$ 的点。若从 $1$ 出发，并按图中相邻点的一种合法顺序扩展，可以先得到 $3,5$，再从 $3$ 扩展出 $7,4,2$，之后从 $7$、$6$、$8$ 继续扩展，形成 `1, 3, 5, 7, 4, 2, 6, 8, 9`。C 把更深层的 $6,8,9$ 放到了同层的 $4,2$ 之前；B、D 从 $9$ 出发时也出现了下一层结点越过当前层未处理结点的情况，因此不符合 BFS 队列顺序。
