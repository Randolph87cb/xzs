## 第 1 题

一个 $32$ 位无符号整数可以表示的最大值，最接近下列哪个选项？

A. $4 \times 10^9$

B. $3 \times 10^{10}$

C. $2 \times 10^9$

D. $2 \times 10^{10}$

答案：A

解析：### 解题思路

32 位无符号整数的最大值是 `2^32 - 1`。先估算它的数量级：

- `2^10 ≈ 10^3`
- 所以 `2^32 = 2^2 × 2^30 ≈ 4 × 10^9`
- 更精确地说，`2^32 - 1 = 4294967295`

在给出的选项中，最接近的是 `4 × 10^9`。

### 关键知识点

- `无符号整数`：只能表示非负整数。
- `32 位无符号整数` 的取值范围是 `0 ~ 2^32 - 1`。
- 估算数量级时，可以用 `2^10 ≈ 10^3` 进行换算。

### 选项分析

- `A. 4 × 10^9`：`2^32 - 1 ≈ 4.29 × 10^9`，最接近，正确。
- `B. 3 × 10^{10}`：数量级过大，远大于 `4.29 × 10^9`，错误。
- `C. 2 × 10^9`：数量级偏小，和实际值相差较大，错误。
- `D. 2 × 10^{10}`：数量级也过大，错误。

### 正确答案

`A`。32 位无符号整数最大值是 `2^32 - 1 = 4294967295`，约为 `4.29 × 10^9`，因此最接近 `4 × 10^9`。

## 第 2 题

在 C++ 中，执行 `int x = 255; cout << (x & (x - 1));` 后，输出的结果是？

A. $255$

B. $254$

C. $128$

D. $0$

答案：B

解析：### 解题思路
题目要求计算 `x & (x - 1)`。先令 `x = 255`，则 `x - 1 = 254`。将它们按位与运算：

- `255` 的二进制是 `11111111`
- `254` 的二进制是 `11111110`
- 按位与后得到 `11111110`，即十进制 `254`

因此输出结果是 `254`。

### 关键知识点
- `&` 表示按位与运算，只有对应位都为 `1` 时结果位才为 `1`。
- `x - 1` 会将 `x` 减去 `1`。
- 对于一个全为 `1` 的二进制数（如 `255 = 11111111`），与 `x - 1` 做按位与后，结果就是 `x - 1`。

### 选项分析
A. `255`：`255 & 254 = 254`，不会得到 `255`，错误。

B. `254`：`255` 与 `254` 按位与后结果为 `254`，正确。

C. `128`：`255 & 254` 的结果不是 `128`，错误。

D. `0`：只有在对应位没有共同为 `1` 时才可能为 `0`，这里显然不是，错误。

### 正确答案
`x = 255` 时，`x - 1 = 254`。按位与运算 `255 & 254` 的二进制结果是 `11111110`，换算成十进制就是 `254`，所以标准答案 `B` 正确。

## 第 3 题

函数 `calc(n)` 的定义如下，则 `calc(5)` 的返回值是多少？（ ）

```cpp
int calc(int n) {
    if (n <= 1) return 1;
    if (n % 2 == 0) return calc(n / 2) + 1;
    else return calc(n - 1) + calc(n - 2);
}
```

A. $5$

B. $6$

C. $7$

D. $8$

答案：B

解析：### 解题思路
先按函数定义逐步计算 `calc(5)`。

- `5 > 1`，且 `5` 是奇数，所以执行 `return calc(5 - 1) + calc(5 - 2);`，即 `calc(4) + calc(3)`。
- 计算 `calc(4)`：`4 > 1` 且为偶数，所以 `calc(4) = calc(4 / 2) + 1 = calc(2) + 1`。
- 计算 `calc(2)`：`2 > 1` 且为偶数，所以 `calc(2) = calc(1) + 1`。
- 由于 `1 <= 1`，所以 `calc(1) = 1`。
- 因此 `calc(2) = 1 + 1 = 2`，`calc(4) = 2 + 1 = 3`。

再算 `calc(3)`：
- `3 > 1` 且为奇数，所以 `calc(3) = calc(2) + calc(1) = 2 + 1 = 3`。

最后：
- `calc(5) = calc(4) + calc(3) = 3 + 3 = 6`。

### 关键知识点
- 递归函数的计算方法：从函数定义出发，逐层展开到终止条件。
- 基本条件 `if (n <= 1) return 1;` 决定了 `calc(1)` 的值。
- 注意区分奇数分支和偶数分支的递归规则。

### 选项分析
- A：`5`，与逐步计算得到的 `calc(5)=6` 不符。
- B：`6`，与计算结果一致，是正确答案。
- C：`7`，比正确结果大 1，不符合递归展开结果。
- D：`8`，与正确结果不符。

### 正确答案
由递归展开可得 `calc(5) = calc(4) + calc(3) = 3 + 3 = 6`，因此返回值是 `6`，选择 `B`。

## 第 4 题

用 $5$ 个权值 $10, 12, 15, 20, 25$ 构造哈夫曼树，该树的带权路径长度是多少？

A. $176$

B. $186$

C. $196$

D. $206$

答案：B

解析：### 解题思路
哈夫曼树的构造方法是：每次从当前权值中选出最小的两个合并，合并后的新结点权值等于这两个权值之和。所有合并过程中新生成的结点权值之和，就是这棵哈夫曼树的带权路径长度 `WPL`。

题目给出的 5 个权值为 `10, 12, 15, 20, 25`。

按哈夫曼算法依次合并：
1. `10 + 12 = 22`
2. `15 + 20 = 35`
3. `22 + 25 = 47`
4. `35 + 47 = 82`

因此带权路径长度为：
`22 + 35 + 47 + 82 = 186`

所以答案是 `B`。

### 关键知识点
- 哈夫曼树的构造原则：每次合并当前最小的两个权值。
- 哈夫曼树的带权路径长度 `WPL` 等于所有内部结点权值之和，也等于每次合并得到的新权值之和。
- 计算时要按从小到大逐步合并，不能跳步。

### 选项分析
A. `176`：与哈夫曼树逐步合并得到的总和不一致，错误。

B. `186`：按 `10, 12, 15, 20, 25` 依次合并得到 `22, 35, 47, 82`，总和为 `186`，正确。

C. `196`：不符合哈夫曼树的合并过程，错误。

D. `206`：不符合哈夫曼树的合并过程，错误。

### 正确答案
标准答案为 `B`。按照哈夫曼树构造规则依次合并最小权值，得到的新结点权值分别为 `22、35、47、82`，它们的和为 `186`，这就是带权路径长度 `WPL`。

## 第 5 题

在一个有向图中，所有顶点的入度之和等于所有顶点的出度之和，这个总和等于？

A. 顶点数

B. 边数

C. 顶点数 + 边数

D. 顶点数 $\times 2$

答案：B

解析：### 解题思路
在有向图中，每一条边都会从某个顶点“出发”一次，因此会让这个顶点的出度加 1；同时，这条边也会指向另一个顶点，因此会让那个顶点的入度加 1。也就是说，**每条边都会同时贡献 1 个入度和 1 个出度**。

所以，把所有顶点的入度加起来，得到的总和就等于边数；把所有顶点的出度加起来，得到的总和也等于边数。两者相等，且都等于图中的 `边数`。

### 关键知识点
- 在有向图中，`所有顶点的入度之和 = 边数`。
- 在有向图中，`所有顶点的出度之和 = 边数`。
- 每条有向边只会给一个顶点贡献 `1` 个入度、给一个顶点贡献 `1` 个出度。

### 选项分析
- `A`：`顶点数`。错误。入度和出度的总和与顶点数量无关，而与边的数量有关。
- `B`：`边数`。正确。由有向图的度数性质可知，所有顶点的入度之和、出度之和都等于边数。
- `C`：`顶点数 + 边数`。错误。没有这种度数总和公式。
- `D`：`顶点数 × 2`。错误。`2` 倍顶点数不是有向图入度和/出度和的通用结果。

### 正确答案
`B`。因为在有向图中，每条边都会让一个顶点的入度加 `1`，同时让另一个顶点的出度加 `1`，所以所有顶点的入度之和等于边数，所有顶点的出度之和也等于边数。

## 第 6 题

从 $5$ 位男生和 $4$ 位女生中选出 $4$ 人组成一个学习小组，要求学习小组中男生和女生都有。有多少种不同的选法？

A. $126$

B. $121$

C. $120$

D. $100$

答案：C

解析：### 解题思路
从 5 位男生和 4 位女生中选 4 人，并且要求小组里**既有男生又有女生**。可以先算出不限制性别时的总选法，再减去“全是男生”或“全是女生”的情况。

1. 不限制性别，共有 `C(9,4)` 种选法。
2. 全是男生的情况：从 5 位男生中选 4 位，共 `C(5,4)` 种。
3. 全是女生的情况：从 4 位女生中选 4 位，共 `C(4,4)` 种。

所以符合条件的选法数为：
`C(9,4) - C(5,4) - C(4,4) = 126 - 5 - 1 = 120`。

### 关键知识点
- 组合数 `C(n,m)` 表示从 `n` 个不同元素中选 `m` 个的选法数。
- “至少包含两类对象”时，常用“总数减去不合要求的情况”来计算。
- 要注意题目要求“男生和女生都有”，即不能全选同一性别。

### 选项分析
A. `126`：这是不考虑性别限制时，从 9 人中任意选 4 人的总数，少减了不符合条件的情况。

B. `121`：这个结果不对应本题的正确组合计算，说明在去除不合要求情况时少减或误算了。

C. `120`：正确。`C(9,4) - C(5,4) - C(4,4) = 120`，满足既有男生又有女生。

D. `100`：与正确结果不符，明显少算了部分符合条件的选法。

### 正确答案
`C`。先算总选法 `C(9,4)=126`，再减去全男生 `C(5,4)=5` 和全女生 `C(4,4)=1`，得到 `120`，因此选 `C`。

## 第 7 题

假设 $a, b, c$ 都是布尔变量，逻辑表达式 `(a && b) || (!c && a)` 的值与下列哪个表达式不始终相等？

A. `a && (b || !c)`

B. `(a || !c) && (b || !c) && (a || a)`

C. `a && (!b || c)`

D. `!(!a || !b) || (a && !c)`

答案：C

解析：### 解题思路
先把原式化简：`(a && b) || (!c && a)` 中两项都含有 `a`，可提取公因子，得到 `a && (b || !c)`。因此只要某个选项与这个化简结果不始终相等，就说明它不是等价表达式。接下来逐项判断。

### 关键知识点
1. 布尔代数中的分配律：`(x && y) || (x && z) = x && (y || z)`。
2. 布尔代数中的结合律、交换律、幂等律。
3. 判断“始终相等”就是判断两个逻辑表达式是否对所有布尔取值都等价。

### 选项分析
A. `a && (b || !c)`

由分配律可知：`(a && b) || (a && !c) = a && (b || !c)`，与原式完全等价，所以 A 始终相等。

B. `(a || !c) && (b || !c) && (a || a)`

先化简 `a || a = a`，原式变为 `(a || !c) && (b || !c) && a`。再利用 `a && (a || !c) = a`，可得整体化简为 `a && (b || !c)`，与原式等价，所以 B 始终相等。

C. `a && (!b || c)`

原式化简后是 `a && (b || !c)`，而 C 是 `a && (!b || c)`。这两个括号内的表达式一般不相同，例如取 `a = 1, b = 1, c = 0` 时，原式为 `1 && (1 || 1) = 1`，而 C 为 `1 && (0 || 0) = 0`，结果不同，所以 C 不始终相等。

D. `!(!a || !b) || (a && !c)`

由德摩根定律，`!(!a || !b) = a && b`，所以 D 变为 `(a && b) || (a && !c)`，再提取 `a` 得到 `a && (b || !c)`，与原式等价，所以 D 始终相等。

### 正确答案
`C`。因为原式可化简为 `a && (b || !c)`，而 `a && (!b || c)` 与它不等价，存在取值使二者结果不同。

## 第 8 题

已知 $f[0] = 1$, $f[1] = 1$，并且对于所有 $n \geq 2$ 有 $f[n] = (f[n-1] + f[n-2]) \% 7$。那么 $f[2025]$ 的值是多少？

A. $2$

B. $4$

C. $5$

D. $6$

答案：D

解析：### 解题思路
题目给出了递推式 `f[n] = (f[n-1] + f[n-2]) % 7`，这表示数列每一项都只保留除以 `7` 的余数。由于初值 `f[0] = 1`、`f[1] = 1`，我们可以先按递推关系依次算出前几项，观察是否出现循环。

计算如下：
- `f[0] = 1`
- `f[1] = 1`
- `f[2] = (1 + 1) % 7 = 2`
- `f[3] = (1 + 2) % 7 = 3`
- `f[4] = (2 + 3) % 7 = 5`
- `f[5] = (3 + 5) % 7 = 1`
- `f[6] = (5 + 1) % 7 = 6`
- `f[7] = (1 + 6) % 7 = 0`
- `f[8] = (6 + 0) % 7 = 6`
- `f[9] = (0 + 6) % 7 = 6`
- `f[10] = (6 + 6) % 7 = 5`
- `f[11] = (6 + 5) % 7 = 4`
- `f[12] = (5 + 4) % 7 = 2`
- `f[13] = (4 + 2) % 7 = 6`
- `f[14] = (2 + 6) % 7 = 1`

可以继续验证，`f[14] = 1`、`f[15] = 1`，与 `f[0]`、`f[1]` 完全相同，因此数列从这里开始重复，周期为 `14`。

所以只需要计算 `2025` 除以 `14` 的余数：
`2025 = 14 × 144 + 9`

因此 `f[2025] = f[9] = 6`。

### 关键知识点
- 递推数列可以通过手动推算前若干项寻找周期。
- 取模运算会使数列值限定在 `0` 到 `6` 之间。
- 周期数列中，若 `n` 对周期取余即可得到对应项。

### 选项分析
- `A. 2`：由上面的计算可知，`f[2025] = f[9] = 6`，不是 `2`。
- `B. 4`：对应的值不是 `4`，与实际计算不符。
- `C. 5`：对应的值不是 `5`，与实际计算不符。
- `D. 6`：`2025 % 14 = 9`，而 `f[9] = 6`，因此该项正确。

### 正确答案
由递推式计算可知，该数列模 `7` 后以 `14` 为周期重复。因为 `2025 % 14 = 9`，所以 `f[2025] = f[9] = 6`，对应选项 `D`。

## 第 9 题

下列关于 C++ string 类的说法，正确的是？

A. string 对象的长度在创建后不能改变。

B. 可以使用 + 运算符直接连接一个 string 对象和一个 char 类型的字符。

C. string 的 length() 和 size() 方法返回的值可能不同。

D. string 对象必须以 '\0' 结尾，且这个结尾符计入 length()。

答案：B

解析：### 解题思路
题目考查 C++ `string` 类的基本用法。逐项判断各选项是否符合 `string` 的定义和常用成员函数、运算符重载规则。`string` 表示可变长度字符串，支持与字符、字符串进行拼接，因此需要重点关注 `+` 运算符的使用。

### 关键知识点
- `std::string` 是长度可变的字符串类，对象内容可以修改，长度也可以改变。
- `string` 支持与 `char`、`const char*`、`string` 等类型进行拼接，常见形式如 `s + 'a'`、`'a' + s`、`s + "abc"`。
- `string` 的 `length()` 与 `size()` 返回值相同，都是字符串中字符的个数。
- `string` 内部通常以 `\0` 作为底层存储的一部分约定，但 `length()` 统计的是有效字符数，不包含结尾的 `\0`。

### 选项分析
- `A`：错误。`string` 对象的长度创建后可以改变，例如可以继续追加字符或删除字符。

- `C`：错误。`string` 的 `length()` 和 `size()` 返回值相同，没有区别。
- `D`：错误。`string` 并不要求“对象必须以 `\0` 结尾且这个结尾符计入 `length()`”，`length()` 统计的是字符数量，不包含结尾的 `\0`。

### 正确答案
选项 `B` 正确。`C++` 的 `string` 类重载了拼接运算符，可以直接把 `string` 对象与 `char` 字符相加，得到新的字符串。

## 第 10 题

考虑以下 C++ 函数：

```cpp
void solve(int &a, int b) {
    a = a + b;
    b = a - b;
    a = a - b;
}
int main() {
    int x = 5, y = 10;
    solve(x, y);
}
```

在 main 函数调用 solve 后，$x$ 和 $y$ 的值分别是？

A. $5,10$

B. $10,5$

C. $10,10$

D. $5,5$

答案：C

解析：### 解题思路
先看函数参数的传递方式：`solve(int &a, int b)` 中，`a` 是引用传递，`b` 是值传递。

因此，调用 `solve(x, y)` 时：
- `a` 直接绑定到 `x`，函数内对 `a` 的修改会影响 `x`；
- `b` 只是 `y` 的一个副本，函数内对 `b` 的修改不会影响 `y`。

初始时 `x = 5`，`y = 10`。

进入函数后：
1. `a = a + b`，此时 `a` 对应 `x`，所以 `x = 5 + 10 = 15`；`b` 仍为 `10`。
2. `b = a - b`，即 `b = 15 - 10 = 5`；此时 `b` 变成 `5`，但不影响 `y`。
3. `a = a - b`，即 `x = 15 - 5 = 10`。

所以函数结束后，`x = 10`，`y = 10`。

### 关键知识点
- 引用传递 `int &a` 会直接修改实参。
- 值传递 `int b` 只会修改形参副本，不影响实参。
- 交换两个数时，若其中一个参数是引用，另一个是值传递，需要特别注意实际改变的是哪一个变量。

### 选项分析
- `A. 5,10`：如果 `x` 和 `y` 都没变，才会是这个结果，但 `x` 作为引用参数会被修改，故错误。
- `B. 10,5`：`x` 最终确实会变成 `10`，但 `y` 是值传递，不会变成 `5`，故错误。
- `C. 10,10`：`x` 被函数改为 `10`，`y` 保持原值 `10`，正确。
- `D. 5,5`：`x` 不会保持 `5`，`y` 也不会变成 `5`，故错误。

### 正确答案
由参数传递方式可知，`a` 引用绑定到 `x`，所以函数内部对 `a` 的运算会改变 `x`；`b` 只是 `y` 的副本，不会影响 `y`。按语句顺序计算后，`x` 最终为 `10`，`y` 仍为 `10`，因此标准答案是 `C`。

## 第 11 题

一个 $8 \times 8$ 的棋盘，左上角坐标为 $(1,1)$，右下角为 $(8,8)$。一个机器人从 $(1,1)$ 出发，每次只能向右或向下走一格。要到达 $(4,5)$，有多少种不同的路径？

A. $20$

B. $35$

C. $56$

D. $70$

答案：B

解析：### 解题思路

A. $j < i$

先看第 18 行代码：

`for (; j < i && a[i] - a[j + 1] > k; ++j) ;`

这是一个空循环体的 `for` 循环，作用是不断增大 `j`，直到不再满足 `j < i` 且 `a[i] - a[j + 1] > k`。由于 `j` 每次只会自增，且有 `j < i` 作为边界，所以执行这一行代码时，必然有 `j < i`。

- `j < i`：这是循环条件的一部分，一定成立。
- `a[i] - a[j] > k`：代码判断的是 `a[i] - a[j + 1] > k`，并不是 `a[i] - a[j] > k`，所以不一定成立。
- `j < n`：由于 `j < i` 且 `i <= n`，所以一定有 `j < n`。
- `a[j] < a[i]`：因为数组去重排序后严格递增，且 `j < i`，所以 `a[j] < a[i]` 一定成立。

### 关键知识点

- `std::sort` 后配合 `std::unique` 可以将数组排序并去重。
- 严格递增数组中，若 `j < i`，则有 `a[j] < a[i]`。
- `for` 循环的条件表达式和循环变量更新要区分清楚，题目中判断的是 `a[i] - a[j + 1] > k`，不是 `a[i] - a[j] > k`。

- 指针 `j` 会尽量向右移动，使得 `a[i] - a[j + 1] <= k`。
- 然后令 `ans[i] = ans[j] + 1`。

- 当 `i = 4` 时，`a[4] - a[1] = 3 > 2`，`j` 会移动到 `1`，于是 `ans[4] = ans[1] + 1 = 2`。

### 关键知识点

- `std::sort`：对数组排序。
- `std::unique`：去除相邻重复元素，配合 `sort` 后可实现全局去重。
- 双指针/滑动窗口思想：`j` 只会单调递增，表示当前可接续的最左位置。
- 递推中 `ans[i] = ans[j] + 1`：表示当前位置需要在前面最优结果基础上再加一段。
- 对于连续整数 `1..100` 且 `k = 2`，每段最多包含 `k + 1 = 3` 个数。

B. `100`

2. 如果 `a[i] == b[j]`，则还可以转移到 `f[i-1][j-1] + 1`，表示把这两个相等的元素配成一对。

这类转移实际上是在求两个序列的最长公共子序列长度（LCS）。

现在在第 16 行循环前加入：
`std::sort(a+1, a+n+1);`
`std::sort(b+1, b+n+1);`

排序后，两个数组都变成非降序。对任意两个序列来说，排序不会让它们的最长公共子序列长度变小：
- 原来能匹配到的元素个数，排序后至少还能按相同数值继续匹配；
- 由于排序把相同元素集中在一起，公共子序列通常更容易变长。

因此答案只能是“变大或不变”，对应 A。

### 关键知识点

- 动态规划中，`f[i][j]` 结合 `f[i-1][j]`、`f[i][j-1]`、`f[i-1][j-1]` 的转移，是典型的最长公共子序列（LCS）模型。
- `sort` 会改变序列的相对顺序，但不会改变元素的多重集合。
- 对于本题这种“按相等关系匹配”的 DP，排序后公共子序列长度不会减少。

### 选项分析

A. `变大或不变`：正确。排序后不会降低最长公共子序列长度，可能变大，也可能保持不变。

B. `变小或不变`：错误。排序把元素重新排列后，一般不会让可匹配的最优值减少。

C. `一定变大`：错误。若原本两个数组已经有序，或者本来就已经达到最优，排序后结果可能不变。

D. `不变`：错误。排序后结果有可能变大，例如原来交错排列导致只能匹配较短公共子序列，排序后可匹配更多。

### 正确答案

A

- A. `query(candidate, i) == false`

- B. `query(i, candidate) == true`

- D. `query(candidate, i) == false || query(i, candidate) == false`
  - 只要任意一方认为对方是糊涂人，就说明这对人可以抵消，符合题意。

`D`
