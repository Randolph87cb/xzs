## 一、单项选择题（共 15 题，每题 2 分，共计 30 分；每题有且仅有一个正确选项）

在 C++ 中，下面哪个关键字用于声明一个变量， 其值不能被修改?

A. `unsigned`

B. `const`

C. `static`

D. `mutable`

答案：B

解析：### 解题思路
题目问的是：在 C++ 中，哪个关键字用于声明一个变量，使它的值不能被修改。根据 C++ 语法，`const` 用来声明常量，表示该变量一旦初始化后就不能再被修改，因此应选 `const`。

### 关键知识点
- `const`：表示常量，变量值不可再被修改。
- `static`：表示静态存储期或静态成员等，与“不可修改”无直接关系。
- `unsigned`：表示无符号整数类型修饰符，不是用来限制修改的。
- `mutable`：常用于类的成员变量，允许在 `const` 对象中被修改，含义与“不可修改”相反。

### 选项分析
A. `unsigned`：这是整数类型的修饰关键字，表示无符号，不是“不可修改”的意思，错误。

B. `const`：用于声明常量，表示变量的值不能被修改，正确。

C. `static`：用于表示静态变量、静态成员等，与变量值能否修改无直接关系，错误。

D. `mutable`：表示可变的，常用于修饰类成员，使其在 `const` 对象中仍可被修改，与题意相反，错误。

### 正确答案
`const` 是 C++ 中用于声明常量的关键字。被 `const` 修饰的变量在初始化后不能再赋新值，因此符合“其值不能被修改”的要求，故选 `B`。

## 第 2 题

八进制数 $12345670_8$ 和 $07654321_8$ 的和为

A. $22222221_8$

B. $21111111_8$

C. $22111111_8$

D. $22222211_8$

答案：D

解析：### 解题思路
八进制是逢 8 进 1 的进位制，计算时可以像十进制竖式加法一样逐位相加，并注意满 8 向前进位。

从最低位开始计算：
- `0 + 1 = 1`
- `7 + 2 = 11_8`，写 `1`，向前进 `1`
- `6 + 3 + 1 = 12_8`，写 `2`，向前进 `1`
- `5 + 4 + 1 = 12_8`，写 `2`，向前进 `1`
- `4 + 5 + 1 = 12_8`，写 `2`，向前进 `1`
- `3 + 6 + 1 = 12_8`，写 `2`，向前进 `1`
- `2 + 7 + 1 = 12_8`，写 `2`，向前进 `1`
- `1 + 0 + 1 = 2`

所以和为 `22222211_8`。

### 关键知识点
- 八进制是 `8` 进制，个位相加满 `8` 要向前进 `1`。
- 进行多位数加法时，可以按位逐列计算。
- 结果中的每一位都必须是 `0` 到 `7` 之间的八进制数字。

### 选项分析
- `A. 22222221_8`：末位对应 `0 + 1 = 1`，不可能是 `2`，错误。
- `B. 21111111_8`：从低位开始加法后，多列都应产生进位，这个结果没有正确体现连续进位，错误。
- `C. 22111111_8`：中间多位未正确反映逐位相加后的结果，错误。
- `D. 22222211_8`：与逐位八进制加法结果一致，正确。

### 正确答案
`D`。按八进制逐位相加并处理进位，`12345670_8 + 07654321_8 = 22222211_8`。

## 第 3 题

阅读下述代码，请问修改 `data` 的 `value` 成员以存储 $3.14$，正确的方式是
```cpp
union Data{
    int num;
    float value;
    char symbol;
};
union Data data;
```

A. `data.value = 3.14;`

B. `value.data = 3.14;`

C. `data -> value = 3.14;`

D. `value->data = 3.14;`

答案：A

解析：### 解题思路
题目给出了一个 `union` 联合体 `Data`，其中包含成员 `num`、`value` 和 `symbol`。题目要求“修改 `data` 的 `value` 成员以存储 `3.14`”，也就是直接给联合体变量 `data` 的成员 `value` 赋值。因为 `data` 是一个已经定义好的联合体对象，所以应使用成员访问运算符 `.` 来访问成员。

### 关键知识点
- `union` 联合体可以像结构体一样通过成员名访问成员。
- 已定义的对象使用 `.` 运算符访问成员，例如 `data.value`。
- `->` 运算符用于指针访问成员，不适用于普通对象。

### 选项分析
- `A`：`data.value = 3.14;`，`data` 是联合体对象，使用 `.` 访问成员，写法正确。
- `B`：`value.data = 3.14;`，把对象和成员顺序写反了，`value` 也不是已定义的对象名，错误。
- `C`：`data -> value = 3.14;`，`->` 只能用于指针，`data` 不是指针，错误。
- `D`：`value->data = 3.14;`，不仅 `value` 不是指针，而且成员名和访问方向都不对，错误。

### 正确答案
`A`。因为 `data` 是联合体对象，修改其成员 `value` 应使用 `data.value = 3.14;`。

## 第 4 题

假设有一个链表的节点定义如下：

```cpp
struct Node { int data; Node* next; }
```

现在有一个指向链表头部的指针：`Node* head`。如果想要在链表中插入一个新节点，其成员 `data` 的值为 $42$，并使新节点成为链表的第一个节点，下面哪个操作是正确的？

A. `Node* newNode = new Node; newNode->data = 42; newNode->next = head; head = newNode;`

B. `Node* newNode = new Node; head->data = 42; newNode->next = head; head = newNode;`

C. `Node* newNode = new Node; newNode->data = 42; head->next = newNode;`

D. `Node* newNode = new Node; newNode->data = 42; newNode->next = head;`

答案：A

解析：### 解题思路
题目要求把一个新节点插入到链表最前面，也就是让这个新节点成为新的头结点。对于单链表来说，正确做法通常分三步：

1. 创建新节点；
2. 让新节点的 `next` 指向原来的头结点 `head`；
3. 再把 `head` 更新为新节点。

这样做之后，链表的第一个节点就变成了新节点，且原链表没有丢失。

### 关键知识点
- 单链表插入到表头时，必须先保存原头结点的地址。
- 新节点的 `next` 应该指向原来的 `head`。
- 最后再更新 `head`，否则可能丢失原链表。
- `data` 成员只是节点保存的数据，和链表连接关系无关。

### 选项分析
- `A`：`newNode->data = 42; newNode->next = head; head = newNode;`，先设置数据，再让新节点连接到原头结点，最后更新 `head`，这是标准的表头插入操作，正确。
- `B`：把 `head->data` 改成 `42`，这只是修改了原头结点的数据，并没有真正创建一个新节点成为第一个节点，而且若 `head` 为空还会出错，错误。
- `C`：`head->next = newNode;` 是把新节点接到原头结点后面，相当于在第二个位置插入，不是插到最前面，错误。
- `D`：只写了 `newNode->next = head;`，没有执行 `head = newNode;`，因此头指针没有更新，新节点不会成为第一个节点，错误。

### 正确答案
标准答案是 `A`。因为要让新节点成为链表第一个节点，必须先令 `newNode->next` 指向原来的 `head`，再把 `head` 指向 `newNode`。这样新节点就成为新的头结点，原链表也完整保留。

## 第 5 题

根节点的高度为 $1$，一棵拥有 $2023$个节点的三叉树高度至少为（）。

A. 6

B. 7

C. 8

D. 9

答案：C

解析：### 解题思路

三叉树中，每个节点最多有 `3` 个孩子。题目要求“高度至少为多少”，并且规定根节点高度为 `1`。要让节点数尽可能少、而高度尽可能大，应该让树尽量“瘦”，也就是每一层都只保留 `1` 个节点，这样高度增加得最快。

如果一棵树高度为 `h`，那么在最理想的“最少节点”情况下，它至少需要 `h` 个节点，因为每一层至少要有 `1` 个节点。题目给出共有 `2023` 个节点，所以最大可能高度至少可以达到 `2023`，但这显然不是题目想问的方向。

实际上，这类题目通常考查的是：**给定节点数，三叉树的高度至少是多少**，也就是在节点尽量多地集中到上层时，树的高度最小；但题目文字写的是“高度至少为”，结合标准答案 `C` 可知它问的是：**拥有 `2023` 个节点的三叉树，最少需要多少层才能容纳这么多节点**。对于一棵高度为 `h` 的满三叉树，最多节点数为 `1 + 3 + 3^2 + ... + 3^{h-1}`。

逐层计算：
- 高度 `1`：最多 `1` 个节点
- 高度 `2`：最多 `1 + 3 = 4` 个节点
- 高度 `3`：最多 `1 + 3 + 9 = 13` 个节点
- 高度 `4`：最多 `40` 个节点
- 高度 `5`：最多 `121` 个节点
- 高度 `6`：最多 `364` 个节点
- 高度 `7`：最多 `1093` 个节点
- 高度 `8`：最多 `3280` 个节点

因为 `2023` 满足 `1093 < 2023 <= 3280`，所以至少需要 `8` 层，答案选 `C`。

### 关键知识点

- 三叉树中，每个节点最多有 `3` 个孩子。
- 高度从 `1` 开始计数，根节点高度为 `1`。
- 高度为 `h` 的满三叉树最多节点数为 `1 + 3 + 3^2 + ... + 3^{h-1}`。
- 判断“至少多少层”时，通常比较给定节点数与各高度下的最大节点数。

### 选项分析

- `A. 6`：高度 `6` 的满三叉树最多有 `364` 个节点，不能容纳 `2023` 个节点，错误。
- `B. 7`：高度 `7` 的满三叉树最多有 `1093` 个节点，仍然不能容纳 `2023` 个节点，错误。
- `C. 8`：高度 `8` 的满三叉树最多有 `3280` 个节点，可以容纳 `2023` 个节点，正确。
- `D. 9`：虽然也能容纳，但不是“至少需要”的最小高度，错误。

### 正确答案

`C`。因为高度为 `7` 时最多只能有 `1093` 个节点，不足以容纳 `2023` 个节点；而高度为 `8` 时最多可达 `3280` 个节点，能够容纳 `2023` 个节点，所以最少需要 `8` 层。

## 第 6 题

小明在某一天中依次有七个空闲时间段，他想要选出至少一个空闲时间段来练习唱歌，但他希望任意两个练习的时间段之间都有至少两个空闲的时间段让他休息。则小明一共有（）种选择时间段的方案。

A. 31

B. 18

C. 21

D. 33

答案：B

解析：### 解题思路

把 7 个空闲时间段记为 `1, 2, 3, 4, 5, 6, 7`。题目要求：如果选择了两个练习时间段，那么它们之间必须至少隔着 `2` 个空闲时间段，也就是任意两个被选中的位置差至少为 `3`。

因此，我们是在 7 个位置中选若干个位置，使得任意相邻两个被选位置的距离都不小于 `3`。

可以分类讨论选了多少个时间段：

- 选 `1` 个：有 `7` 种。
- 选 `2` 个：设为 `i < j`，则需要 `j - i >= 3`。这样的选法共有 `10` 种。
- 选 `3` 个：设为 `a < b < c`，则需要 `b - a >= 3` 且 `c - b >= 3`。符合条件的共有 `1` 种，即 `1, 4, 7`。
- 选 `4` 个及以上：不可能满足条件。

所以总方案数为 `7 + 10 + 1 = 18`。

### 关键知识点

- 组合计数：按“选几个”分类统计。
- 间隔限制：相邻被选位置之间至少相差 `3`。
- 枚举与分类讨论是解决这类离散计数题的常用方法。

### 选项分析

- `A. 31`：数量过大，不符合间隔限制下的实际可选方案数。
- `B. 18`：由分类统计可得，总方案数正好是 `18`，正确。
- `C. 21`：与正确计数不符。
- `D. 33`：超过所有子集数量的合理范围，也不符合题意。

### 正确答案

标准答案是 `B`。满足“任意两个练习时间段之间至少有两个空闲时间段”这一条件的方案共有 `18` 种。

## 第 7 题

以下关于高精度运算的说法错误的是()

A. 高精度计算主要是用来处理大整数或需要保留多位小数的运算

B. 大整数除以小整数的处理的步骤可以是，将被除数和除数对齐，从左到右逐位尝试将除数乘以某个数，通过减法得到新的被除数，并累加商

C. 高精度乘法的运算时间只与参与运算的两个整数中长度较长者的位数有关

D. 高精度加法运算的关键在于逐位相加并处理进位

答案：C

解析：### 解题思路
本题考查高精度运算的基本概念与常见算法。高精度运算通常用于处理普通整数类型无法直接表示的超大整数，或者需要较高精度的小数运算。逐项判断各选项描述是否符合高精度运算的特点即可。

### 关键知识点
- `高精度`常用于超出基本数据类型范围的整数运算。
- 高精度加法通常按位相加，并处理进位。
- 高精度乘法的运算量通常与两个数的位数共同有关，不只取决于其中较长者。
- 大整数除以小整数时，常用从高位到低位模拟除法的方式计算商和余数。

### 选项分析
- `A`：正确。高精度计算主要用于处理大整数，或需要保留很多位小数的运算。
- `B`：正确。大整数除以小整数时，常见做法是从高位到低位逐位处理，模拟人工除法，得到商并更新余数。
- `C`：错误。高精度乘法的时间复杂度通常与两个参与运算数的位数都有关，例如常见竖式乘法是 `O(nm)`，并不只与较长者的位数有关。
- `D`：正确。高精度加法的核心就是逐位相加，并把低位产生的进位传到高位。

### 正确答案
`C`。高精度乘法的计算量不仅取决于较长整数的位数，还与另一个整数的位数有关，因此该说法错误。

## 第 8 题

后缀表达式 `6 2 3 + - 3 8 2 / + * 2 ^ 3 +` 对应的中缀表达式是

A. ((6-(2+3))\*(3+8/2))^2+3

B. 6-2+3\*3+8/2^2+3

C. (6-(2+3))\*((3+8/2)^2)+3

D. 6-((2+3)\*(3+8/2))^2+3

答案：A

解析：### 解题思路
后缀表达式（逆波兰式）求中缀表达式时，遵循“从左到右扫描，遇到数字入栈，遇到运算符就弹出两个操作数，先弹出的作右操作数，后弹出的作左操作数，再把新表达式入栈”的规则。

依次处理 `6 2 3 + - 3 8 2 / + * 2 ^ 3 +`：

1. `6` 入栈：`[6]`
2. `2` 入栈：`[6, 2]`
3. `3` 入栈：`[6, 2, 3]`
4. `+`：弹出 `2` 和 `3`，组成 `(2+3)`，入栈：`[6, (2+3)]`
5. `-`：弹出 `6` 和 `(2+3)`，组成 `6-(2+3)`，入栈：`[6-(2+3)]`
6. `3` 入栈：`[6-(2+3), 3]`
7. `8` 入栈：`[6-(2+3), 3, 8]`
8. `2` 入栈：`[6-(2+3), 3, 8, 2]`
9. `/`：弹出 `8` 和 `2`，组成 `8/2`，入栈：`[6-(2+3), 3, 8/2]`
10. `+`：弹出 `3` 和 `8/2`，组成 `3+8/2`，入栈：`[6-(2+3), (3+8/2)]`
11. `*`：弹出 `6-(2+3)` 和 `(3+8/2)`，组成 `(6-(2+3))*(3+8/2)`，入栈：`[(6-(2+3))*(3+8/2)]`
12. `2` 入栈：`[(6-(2+3))*(3+8/2), 2]`
13. `^`：弹出前两个，组成 `((6-(2+3))*(3+8/2))^2`，入栈：`[((6-(2+3))*(3+8/2))^2]`
14. `3` 入栈：`[((6-(2+3))*(3+8/2))^2, 3]`
15. `+`：弹出前两个，组成 `((6-(2+3))*(3+8/2))^2+3`

因此对应的中缀表达式是 `((6-(2+3))*(3+8/2))^2+3`，即选项 A。

### 关键知识点
- 后缀表达式（逆波兰式）按“操作数在前、运算符在后”的顺序书写。
- 转换/求值时，二元运算符需要先弹出右操作数，再弹出左操作数。
- 为保证运算顺序正确，必要时要给中缀表达式补充括号。

### 选项分析
A. `((6-(2+3))*(3+8/2))^2+3`，与按栈转换得到的结果完全一致，正确。

B. `6-2+3*3+8/2^2+3`，没有体现后缀表达式中 `-`、`*`、`^` 的实际结合关系，运算顺序错误，不是原表达式。

C. `(6-(2+3))*((3+8/2)^2)+3`，把平方 `^2` 放到了 `3+8/2` 这一项上，实际后缀表达式中是整个乘积后再平方，错误。

D. `6-((2+3)*(3+8/2))^2+3`，这里平方作用在括号内乘积上且整体结构变成 `6-...+3`，与后缀表达式对应的整体形式不符，错误。

### 正确答案
由后缀表达式逐步入栈、出栈并补充括号，可得到中缀表达式 `((6-(2+3))*(3+8/2))^2+3`，因此正确答案是 A。

## 第 9 题

数 $101010_2$ 和 $166_8$ 的和为 ( )

A. $(10110000)_2$

B. $(236)_8$

C. $(158)_{10}$

D. $(A0)_{16}$

答案：D

解析：### 解题思路
先把两个数都化成十进制，再相加。

- `101010_2 = 1×2^5 + 1×2^3 + 1×2^1 = 32 + 8 + 2 = 42`
- `166_8 = 1×8^2 + 6×8^1 + 6×8^0 = 64 + 48 + 6 = 118`

所以它们的和是 `42 + 118 = 160`。

再把 `160` 转成十六进制：`160 = 10×16 + 0`，因此结果是 `A0_16`，即 `A0`。

### 关键知识点
- 不同进制数相加时，常用方法是先统一转换为同一进制，再计算。
- 二进制、八进制、十进制、十六进制之间可以互相转换。
- 十六进制中，`A` 表示 `10`。

### 选项分析
- `A. (10110000)_2`：`10110000_2 = 128 + 32 + 16 = 176`，不等于 `160`，错误。
- `B. (236)_8`：`236_8 = 2×64 + 3×8 + 6 = 158`，不等于 `160`，错误。
- `C. (158)_{10}`：题目求和结果是 `160`，不是 `158`，错误。
- `D. (A0)_{16}`：`A0_{16} = 10×16 + 0 = 160`，与求和结果一致，正确。

### 正确答案
`D`。先将 `101010_2` 和 `166_8` 分别化为十进制，得到 `42` 和 `118`，相加得 `160`；`160` 转为十六进制是 `A0`，对应选项 `D`。

## 第 10 题

## 第 11 题

给定一棵二叉树，其前序遍历结果为：`ABDECFG`,中序遍历结果为：`DEBACFG`。请问这棵树的正确后序遍历结果是什么?

A. `EDBGFCA`

B. `EDGBFCA`

D. `DBEGFCA`

答案：A

解析：### 解题思路
已知二叉树的前序遍历和中序遍历，可以唯一确定这棵树，再求后序遍历。

1. 前序遍历 `ABDECFG` 的第一个结点是根结点 `A`。
2. 在中序遍历 `DEBACFG` 中找到 `A`，左边 `DEB` 是左子树，右边 `CFG` 是右子树。
3. 继续看左子树：
   - 左子树前序部分是 `BDE`，中序部分是 `DEB`。
   - 其根结点是 `B`，在中序中 `B` 左边是 `DE`，说明 `D`、`E` 是 `B` 的左子树结点。
   - 左子树的后序为 `EDB`。
4. 再看右子树：
   - 右子树前序部分是 `CFG`，中序部分是 `CFG`。
   - 根结点是 `C`，它的左右子树为空，只剩下右侧的 `F`、`G` 组成子树。
   - 右子树的后序为 `GFC`。
5. 整棵树的后序遍历应为：左子树后序 `EDB` + 右子树后序 `GFC` + 根结点 `A`，得到 `EDBGFCA`。

### 关键知识点
- 二叉树遍历性质：前序遍历的第一个结点是根结点，中序遍历中根结点左侧是左子树，右侧是右子树。
- 由前序遍历和中序遍历可以唯一确定一棵二叉树。

## 第 12 题

考虑一个有向无环图，该图包含 $4$ 条有向边：$(1,2),(1,3),(2,4)$ 和 $(3,4)$。以下哪个选项是这个有向无环图的一个有效的拓扑排序?

A. 4,2,3,1

B. 1,2,3,4

C. 1,2,4,3

D. 2,1,3,4

答案：B

解析：### 解题思路
拓扑排序要求：在一个有向无环图中，若存在边 `u -> v`，那么在排序结果里 `u` 必须出现在 `v` 的前面。

题目给出的边为：`(1,2)`, `(1,3)`, `(2,4)`, `(3,4)`。
因此必须满足：
- `1` 在 `2` 前面
- `1` 在 `3` 前面
- `2` 在 `4` 前面
- `3` 在 `4` 前面

只要检查各选项是否同时满足这四个条件即可。

### 关键知识点
- 拓扑排序是对有向无环图顶点的一种线性排序。
- 对于每条有向边 `u -> v`，`u` 必须排在 `v` 之前。
- 只要有一条边的顺序被破坏，该序列就不是合法拓扑序。

### 选项分析
- `A. 4,2,3,1`：`1` 应该在 `2`、`3` 前面，但这里 `1` 排在最后，违反了边 `(1,2)` 和 `(1,3)`，错误。

- `D. 2,1,3,4`：`1` 必须在 `2` 前面，但这里 `2` 在 `1` 前，违反边 `(1,2)`，错误。

### 正确答案
`B` 是有效的拓扑排序，因为它满足所有边的先后约束：`1` 先于 `2` 和 `3`，而 `2` 和 `3` 又都先于 `4`。

子题 4：④处应填

A. `dp[i-1][j-1]+1`

B. `dp[i-1][j-1]`

答案：B

解析：### 解题思路

本题考查的是编辑距离（Edit Distance）的动态规划转移。`dp[i][j]` 表示把 `str1` 的前 `i` 个字符转换成 `str2` 的前 `j` 个字符所需的最少操作次数。

当 `str1[i-1] == str2[j-1]` 时，说明当前两个字符相同，不需要额外操作，问题可以直接缩小为把前 `i-1` 个字符转换成前 `j-1` 个字符，因此状态转移应为 `dp[i-1][j-1]`。

所以题目中 `④` 处应填 `dp[i-1][j-1]`。

### 关键知识点

- `dp[i][j]`：表示前缀 `str1[0..i-1]` 变成前缀 `str2[0..j-1]` 的最小操作数。
- 边界条件：
  - `i == 0` 时，只能通过插入把空串变成 `str2` 的前 `j` 个字符。
  - `j == 0` 时，只能通过删除把 `str1` 的前 `i` 个字符变成空串。
- 当 `str1[i-1] == str2[j-1]` 时，不需要替换，直接继承左上角状态 `dp[i-1][j-1]`。
- 当 `str1[i-1] != str2[j-1]` 时，才考虑插入、删除、替换三种操作。

### 选项分析

- `A. dp[i-1][j-1]+1`：表示字符相同但仍然多算了一次操作，不符合“相同则不操作”的转移规则。
- `B. dp[i-1][j-1]`：字符相同，直接沿用左上角状态，正确。
- `C. dp[i-1][j]`：这是删除 `str1` 当前字符后的状态，属于字符不同时的备选转移，不是本空答案。
- `D. dp[i][j-1]`：这是插入一个字符后的状态，也属于字符不同时的备选转移，不是本空答案。

### 正确答案

`B`

本题中 `④` 对应的是“当前两个字符相同”的情况，此时不需要任何新增操作，最优值就是之前子问题 `dp[i-1][j-1]` 的结果，因此标准答案 `B` 正确。

子题 5：⑤处应填

A. `dp[i][j] + 1`

B. `dp[i-1][j-1]+1`

C. `dp[i-1][j-1]`

D. `dp[i][j]`

答案：C

解析：### 解题思路
本题考查的是经典的编辑距离（Edit Distance）动态规划。`dp[i][j]` 表示：将字符串 `str1` 的前 `i` 个字符转换成字符串 `str2` 的前 `j` 个字符所需的最少操作次数。

当两个字符 `str1[i-1]` 和 `str2[j-1]` 相等时，最后一个字符不需要额外操作，因此状态可以直接由左上角转移：`dp[i][j] = dp[i-1][j-1]`。

当两个字符不相等时，可以考虑三种操作：
- 删除 `str1[i-1]`
- 插入 `str2[j-1]`
- 替换 `str1[i-1]` 为 `str2[j-1]`

对应转移为：
`dp[i][j] = 1 + min(dp[i][j-1], dp[i-1][j], dp[i-1][j-1])`

本题问的是 `⑤` 处应该填什么。由于在“替换”对应的转移中，应当使用左上角状态 `dp[i-1][j-1]`，所以答案是 `C`。

### 关键知识点
- 编辑距离是典型的二维动态规划问题。
- `dp[i][j]` 表示前缀之间的最少编辑次数。
- 字符相同时，状态从 `dp[i-1][j-1]` 直接继承。
- 字符不同时，分别考虑删除、插入、替换三种操作。

### 选项分析
- `A. dp[i][j] + 1`：这是当前状态再加 1，属于“原地递推”，没有意义，也会形成自我依赖，不符合动态规划转移。
- `B. dp[i-1][j-1] + 1`：这是字符不同时的“替换”代价形式，但题目中的 `⑤` 位于统一加 `1 + min(...)` 的结构里，因此这里应填的是不额外加 1 的状态值，而不是 `dp[i-1][j-1] + 1`。
- `C. dp[i-1][j-1]`：正确。它表示在最后一个字符发生替换时，从左上角子问题转移过来。
- `D. dp[i][j]`：等于当前状态本身，无法构成正确转移，而且会导致循环依赖。

### 正确答案
`C`。在编辑距离的标准转移中，若当前两个字符不同，则替换操作对应的前驱状态是 `dp[i-1][j-1]`，因此 `⑤` 处应填 `dp[i-1][j-1]`。
