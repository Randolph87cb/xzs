## 一、单项选择题（共 15 题，每题 2 分，共计 30 分；每题有且仅有一个正确选项）

以下哪种功能没有涉及 C++ 语言的面向对象特性支持：（ ）。

A. C++ 中调用 `printf` 函数

B. C++ 中调用用户定义的类成员函数

C. C++ 中构造一个 `class` 或 `struct`

D. C++ 中构造来源于同一基类的多个派生类

答案：A

解析：### 解题思路
本题考查 C++ 的“面向对象特性”相关概念。面向对象特性通常包括类与对象、封装、继承、多态等。先逐项判断：哪些选项体现了类、成员函数、派生类等面向对象内容，哪些只是普通的函数调用。只有不涉及这些特性的选项才是答案。

### 关键知识点
- `class` / `struct` 是 C++ 定义类类型的基础。
- 类成员函数属于面向对象编程中的“对象行为”。
- 继承是面向对象的核心特性之一，派生类来源于基类。
- `printf` 是 C 语言风格的库函数调用，本身不依赖 C++ 的面向对象机制。

### 选项分析
- `A`：`printf` 是标准库中的输出函数，属于过程式函数调用，不涉及类、对象、继承或多态等面向对象特性，因此正确。
- `B`：调用用户定义的类成员函数，显然是通过对象或类来完成的方法调用，属于面向对象特性。
- `C`：构造一个 `class` 或 `struct`，是在定义或使用类类型，这是 C++ 面向对象的基础内容。
- `D`：构造来源于同一基类的多个派生类，涉及继承关系，属于典型的面向对象特性。

### 正确答案
`A`。`printf` 只是普通函数调用，不涉及 C++ 的类、对象、继承等面向对象机制，因此它是本题中没有涉及面向对象特性的功能。

## 第 2 题

有 $6$ 个元素，按照 $6,5,4,3,2,1$ 的顺序进入栈 $S$，请问下列哪个出栈序列是非法的（ ）。

A. $5,4,3,6,1,2$

B. $4,5,3,1,2,6$

C. $3,4,6,5,2,1$

D. $2,3,4,1,5,6$

答案：C

解析：### 解题思路
栈是“后进先出”结构。题目给出元素按 `6,5,4,3,2,1` 的顺序进入栈 `S`，因此在不违反入栈顺序的前提下，可以在任意时刻进行出栈操作。判断某个序列是否合法，只需按目标出栈顺序模拟：能出栈就先出栈，不能出栈就继续按给定顺序入栈，直到无法满足为止。

对于四个选项，逐个检查是否存在一种“入栈-出栈”过程能够得到该序列。只要某一步需要一个尚未入栈的元素先出栈，或者栈顶元素与目标不匹配且后续也无法弥补，那么该序列就是非法的。

### 关键知识点
- 栈的基本性质：`后进先出（LIFO）`。
- 栈序列判定：给定入栈顺序和目标出栈顺序，可以通过模拟入栈、出栈过程判断是否合法。
- 对于固定入栈顺序，某些出栈排列无法由栈操作得到。

### 选项分析
- `A. 5,4,3,6,1,2`：可以合法得到。先入栈 `6` 后出栈得到 `6` 之前需要继续入栈 `5,4,3`，可依次出栈 `5,4,3`，之后再处理 `6,1,2`，整体可以通过模拟实现，因此合法。
- `B. 4,5,3,1,2,6`：可以合法得到。入栈 `6,5,4` 后先出 `4`，再出 `5`，然后入 `3` 出 `3`，继续入 `2,1` 后出 `1,2`，最后出 `6`，因此合法。
- `C. 3,4,6,5,2,1`：非法。为了先输出 `3`，必须先依次入栈 `6,5,4,3`，并弹出 `3`。此时栈顶是 `4`，所以接着输出 `4` 没问题；但之后目标是输出 `6`，而此时栈顶依次还会是 `5`、再下面才是 `6`。由于 `5` 还没有输出，却要求 `6` 先于 `5` 出栈，这与栈的后进先出性质矛盾，因此该序列非法。
- `D. 2,3,4,1,5,6`：可以合法得到。先入栈到 `2`，出 `2`；再继续入 `1`、`3`、`4`，可依次出 `3,4`；之后出 `1`，最后出 `5,6`，因此合法。

### 正确答案
`C`。因为在目标序列中，`6` 要先于 `5` 出栈，但在按 `6,5,4,3,2,1` 入栈并模拟后，`6` 位于 `5` 的下面，不能在 `5` 尚未出栈时先出 `6`，所以该序列不可能由栈操作得到。

## 第 3 题

运行以下代码片段的行为是（ ）。
```
int x = 101;
int y = 201;
int *p = &x;
int *q = &y;
p = q;
```

A. 将 $x$ 的值赋为 $201$

B. 将 $y$ 的值赋为 $101$

C. 将 $q$ 指向 $x$ 的地址

D. 将 $p$ 指向 $y$ 的地址

答案：D

解析：### 解题思路
题目中定义了两个整型变量 `x` 和 `y`，以及两个整型指针 `p` 和 `q`，其中 `p` 指向 `x`，`q` 指向 `y`。随后执行 `p = q;`，这是**指针赋值**，表示把 `q` 中保存的地址复制给 `p`，并不会改变 `x`、`y` 的值，也不会改变 `q` 本身指向的位置。

因此，执行后 `p` 也指向 `y` 的地址。

### 关键知识点

- 指针变量保存的是地址，不直接保存普通整型值
- `p = q;` 表示把 `q` 中的地址赋给 `p`，不会修改被指向变量的值

### 选项分析

A. 错误。`p = q;` 不会把 `x` 的值改成 `201`，它只改变指针 `p` 的指向。

B. 错误。`p = q;` 不会把 `y` 的值改成 `101`，`y` 的值保持不变。

C. 错误。执行的是 `p = q;`，不是 `q = &x;`，所以 `q` 不会指向 `x` 的地址。

D. 正确。`q` 原本指向 `y`，把 `q` 赋给 `p` 后，`p` 也指向 `y` 的地址。

### 正确答案

`int *p = &x;` 和 `int *q = &y;` 表示 `p` 保存 `x` 的地址，`q` 保存 `y` 的地址。执行 `p = q;` 后，`p` 获得 `q` 中保存的地址，也就是 `y` 的地址，因此 `p` 指向 `y`。这只是指针之间的赋值，不会修改 `x` 或 `y` 的数值。

## 第 4 题

链表和数组的区别包括（ ）。

A. 数组不能排序，链表可以

B. 链表比数组能存储更多的信息

C. 数组大小固定，链表大小可动态调整

D. 以上均正确

答案：C

解析：### 解题思路
本题考查数组与链表的基本区别。判断每个选项时，要看它是否准确描述了两种数据结构的特点。数组通常需要一段连续内存，长度一般在创建时就确定；链表由结点串联而成，结点可以按需分配，因此更容易动态扩展。

### 关键知识点
- `数组`：通常在内存中连续存储，支持按下标快速访问，长度往往固定或扩容成本较高。
- `链表`：由多个结点通过指针连接，插入和删除较灵活，大小可根据需要动态变化。
- `数组`和`链表`都可以进行排序，区别不在于“能不能排序”，而在于访问、插入删除和空间组织方式不同。

### 选项分析
- `A`：错误。数组当然可以排序，链表也可以排序，所以“数组不能排序，链表可以”不对。
- `B`：错误。链表并不是“比数组能存储更多的信息”。两者存储的信息取决于元素类型；链表只是每个结点通常还要额外存储指针等连接信息，这不表示它能存储“更多信息”。
- `C`：正确。数组通常大小固定，若要改变大小需要重新分配；链表可以通过增加或删除结点动态调整大小。
- `D`：错误。因为 `A`、`B` 都不正确，所以不可能“以上均正确”。

### 正确答案
`C`

链表和数组的重要区别之一就是存储结构与容量变化方式不同：数组一般大小固定，而链表可以按需要不断增加或减少结点，因此标准答案选 `C`。

## 第 5 题

对假设栈 $S$ 和队列 $Q$ 的初始状态为空。存在 $e_1\sim e_6$ 六个互不相同的数据，每个数据按照进栈 $S$、出栈 $S$、进队列 $Q$、出队列 $Q$ 的顺序操作，不同数据间的操作可能会交错。已知栈 $S$ 中依次有数据 $e_1$、$e_2$、$e_3$、$e_4$、$e_5$ 和 $e_6$ 进栈，队列 $Q$ 依次有数据 $e_2$、$e_4$、$e_3$、$e_6$、$e_5$ 和 $e_1$ 出队列。则栈 $S$ 的容量至少是（ ）个数据。

A. $2$

B. $3$

C. $4$

D. $6$

答案：B

解析：### 解题思路
题目要求确定栈 `S` 的最小容量。关键在于：**栈的容量取决于某一时刻栈中最多同时存放了多少个数据**，而不是总共压入了多少个数据。

题目给出栈 `S` 的入栈顺序是 `e_1, e_2, e_3, e_4, e_5, e_6`，并且这些数据在栈中的进出顺序要符合栈的 `后进先出` 特性。为了让出栈顺序仍然是 `e_1, e_2, e_3, e_4, e_5, e_6` 这样的顺序（题面所说“栈 S 中依次有数据 ... 进栈”表示按这个顺序完成入栈操作，并结合栈的特性分析容量），最省空间的做法是：每次只要某个元素一进栈，就尽快把它出栈，再让下一个元素入栈。这样栈里同时最多只需要容纳 1 个数据；但题目同时给出了队列 `Q` 的操作顺序 `e_2, e_4, e_3, e_6, e_5, e_1`，说明各数据的操作可以交错，必须找出两种结构并行时栈 `S` 的峰值占用。

对这道题，真正需要观察的是：`Q` 的出队顺序与 `S` 的操作交错后，会导致某些数据在进入 `S` 后不能立刻离开，需要在栈中暂存多个元素。按题目给定的交错关系分析，栈中最多会同时存在 `3` 个数据，因此栈容量至少为 `3`。

### 关键知识点
- `栈` 是 `后进先出（LIFO）` 结构。
- `队列` 是 `先进先出（FIFO）` 结构。
- 容量问题本质上看的是**运行过程中同时占用的最大元素数**。
- 当多个数据结构操作可交错时，需要考虑它们的并行约束，而不是只看单一序列。

### 选项分析
- `A. 2`：容量为 `2` 不够。根据题目给出的交错操作关系，栈中峰值同时存放元素数会超过 `2`。
- `B. 3`：正确。最坏情况下栈中最多同时需要容纳 `3` 个数据，因此最小容量是 `3`。
- `C. 4`：容量 `4` 当然也能满足，但不是“至少”所要求的最小值。
- `D. 6`：这是把总数据个数当成了容量上限，过大。栈并不需要同时容纳全部 `6` 个数据。

### 正确答案
`B`

本题答案为 `3`。因为栈 `S` 的容量只需满足操作过程中出现的最大同时存放数即可，结合题目所给的入栈、出栈以及与队列操作的交错关系，栈的峰值占用为 `3`，所以最小容量是 `3`。

## 第 6 题

对表达式 `a+(b-c)*d` 的前缀表达式为（ ），其中 +、-、* 是运算符。

A. `*+a-bcd`

B. `+a*-bcd`

C. `abc-d*+`

D. `abc-+d`

答案：B

解析：### 解题思路
要把中缀表达式 `a+(b-c)*d` 改写成前缀表达式，可以先按运算优先级判断整体结构：`(b-c)*d` 是一个整体，再与 `a` 做加法。前缀表达式的规则是：运算符写在前，两个操作数写在后。

因此，整个表达式的结构是：`+` 的左操作数是 `a`，右操作数是 `*` 的结果；而 `*` 的左操作数是 `-` 的结果，右操作数是 `d`。继续展开 `-` 的两个操作数 `b` 和 `c`，最终得到前缀表达式 `+a*-bcd`。

### 关键知识点
- 前缀表达式（波兰式）的特点是：`运算符 + 操作数1 + 操作数2`。
- 中缀表达式转前缀时，要先根据括号和运算优先级确定表达式的树形结构。
- `*` 的优先级高于 `+`，括号 `(b-c)` 会先计算，表达式整体可视为 `a + ((b-c) * d)`。

### 选项分析
A. `*+a-bcd`：最前面是 `*`，表示整个表达式应以乘法为最外层，但原式最外层运算是 `+`，因此错误。

B. `+a*-bcd`：对应结构为 `+ a (* (- b c) d)`，即 `a + (b-c)*d`，与原式完全一致，正确。

C. `abc-d*+`：这是后缀表达式风格，不是前缀表达式，且顺序也不符合原式结构，错误。

D. `abc-+d`：符号和操作数排列不完整，不能正确表示 `a+(b-c)*d` 的前缀形式，错误。

### 正确答案
原式 `a+(b-c)*d` 的最外层运算是加法 `+`，左操作数是 `a`，右操作数是 `(b-c)*d`。右侧乘法部分又由减法 `b-c` 和 `d` 构成，因此前缀表达式应写成 `+a*-bcd`，对应选项 `B`。

## 第 7 题

假设字母表 $\{a,b,c,d,e\}$ 在字符串出现的频率分别为 $10\%$，$15\%$，$30\%$，$16\%$，$29\%$。若使用哈夫曼编码方式对字母进行不定长的二进制编码，字母 $d$ 的编码长度（ ）位。

A. $1$

B. $2$

C. $2$ 或 $3$

D. $3$

答案：B

解析：### 解题思路
哈夫曼编码的核心是：每次把当前频率最小的两个字符合并成一棵新树，直到只剩一棵树。字符的编码长度，等于它在哈夫曼树中的深度。题目给出 5 个字母的频率分别为 `10%`、`15%`、`30%`、`16%`、`29%`，先按从小到大排序：`10, 15, 16, 29, 30`。

依次合并最小的两个：
1. 合并 `10` 和 `15`，得到 `25`
2. 当前为 `16, 25, 29, 30`，再合并 `16` 和 `25`，得到 `41`
3. 当前为 `29, 30, 41`，再合并 `29` 和 `30`，得到 `59`
4. 最后合并 `41` 和 `59`

观察字母 `d` 的频率是 `16%`。它在第 2 步被合并一次，之后所在的子树还会继续参与后面的合并，因此它从叶子到根一共经过 `2` 条边，所以编码长度为 `2` 位。

### 关键知识点
- 哈夫曼编码是一种最优前缀编码。
- 字符编码长度等于其在哈夫曼树中的深度。
- 构造哈夫曼树时，每次选择当前权值最小的两个结点合并。

### 选项分析
- `A. 1`：`d` 不可能只有 `1` 位。若是 `1` 位，说明它直接作为根的一个孩子，但按哈夫曼构造，`16%` 不是最后两项中的最小之一，不能得到这么短的编码。
- `B. 2`：正确。`d` 的频率为 `16%`，在构造过程中对应的结点深度为 `2`，因此编码长度是 `2` 位。
- `C. 2 或 3`：不对。哈夫曼树的构造过程在本题数据下是确定的，`d` 的长度唯一为 `2`，不是不确定。
- `D. 3`：不对。`d` 位于树的第二层深度，不需要 `3` 位编码。

### 正确答案
`B`。根据哈夫曼树的构造过程，频率为 `16%` 的字母 `d` 最终处于深度 `2`，所以它的编码长度为 `2` 位。

## 第 8 题

一棵有 $n$ 个结点的完全二叉树用数组进行存储与表示，已知根结点存储在数组的第 $1$ 个位置。若存储在数组第 $9$ 个位置的结点存在兄弟结点和两个子结点，则它的兄弟结点和右子结点的位置分别是（ ）。

A. $8$、$18$

B. $10$、$18$

C. $8$、$19$

D. $10$、$19$

答案：C

解析：### 解题思路
完全二叉树按数组顺序存储时，结点在数组中的位置满足固定关系：若某结点位置为 `i`，则它的左孩子在 `2i`，右孩子在 `2i+1`；若 `i>1`，它的兄弟结点与它的父结点相同，且位于相邻位置。

本题中结点在第 `9` 个位置：
- 其父结点位置为 `⌊9/2⌋ = 4`；
- 因为 `9` 是奇数，所以它是父结点的右孩子，它的兄弟结点是左孩子，位置为 `8`；
- 右子结点位置为 `2×9+1 = 19`。

因此答案为 `8` 和 `19`。

### 关键知识点
- 完全二叉树的顺序存储规则：结点 `i` 的左孩子为 `2i`，右孩子为 `2i+1`。
- 结点 `i` 的兄弟结点与其同父，位置可由父结点的另一孩子位置确定。
- 对于奇数位置结点，它通常是右孩子；其兄弟结点在 `i-1`。

### 选项分析
A. `8、18`：兄弟结点 `8` 正确，但右子结点应为 `19`，不是 `18`，错误。

B. `10、18`：`10` 是第 `9` 个结点的右邻位置，但不是兄弟结点；右子结点也不是 `18`，错误。

C. `8、19`：兄弟结点为 `8`，右子结点为 `19`，完全正确。

D. `10、19`：右子结点 `19` 正确，但兄弟结点应为 `8`，不是 `10`，错误。

### 正确答案
由顺序存储规则可知，第 `9` 个结点的左兄弟位置是 `8`，右子结点位置是 `19`，所以正确答案是 `C`。

## 第 9 题

考虑由 $N$ 个顶点构成的有向连通图，采用邻接矩阵的数据结构表示时，该矩阵中至少存在（ ）个非零元素。

A. $N-1$

B. $N$

C. $N+1$

D. $N^2$

答案：B

解析：### 解题思路
有向图用邻接矩阵表示时，矩阵中的每个元素对应一条从一个顶点到另一个顶点的有向边。题目问的是“有向连通图”在邻接矩阵中**至少**有多少个非零元素。

对于 `N` 个顶点的连通图，要让边数尽可能少，图中应尽量接近“树”的结构。对于有向图，若要求连通（通常理解为图整体连通/强连通语境下的最少边数问题），至少需要形成一条贯通所有顶点的连接关系。对应到邻接矩阵中，最少也要有 `N-1` 条边，也就是 `N-1` 个非零元素。

## 第 11 题

以下哪组操作能完成在双向循环链表结点 $p$ 之后插入结点 $s$ 的效果（其中，`next` 域为结点的直接后继，`prev` 域为结点的直接前驱）：（ ）。

A. `p->next->prev=s; s->prev=p; p->next=s; s->next=p->next;`

B. `p->next->prev=s; p->next=s; s->prev=p; s->next=p->next;`

C. `s->prev=p; s->next=p->next; p->next=s; p->next->prev=s;`

D. `s->next=p->next; p->next->prev=s; s->prev=p; p->next=s;`

答案：D

解析：### 解题思路
在双向循环链表中，要把结点 `s` 插入到结点 `p` 之后，需要同时修改 4 个指针关系：

1. 让 `s` 的前驱指向 `p`：`s->prev = p`
2. 让 `s` 的后继指向原来 `p` 的后继：`s->next = p->next`
3. 让原来 `p` 的后继结点的前驱改为 `s`：`p->next->prev = s`
4. 让 `p` 的后继指向 `s`：`p->next = s`

注意修改顺序很重要。先保存 `s->next = p->next`，否则一旦先执行 `p->next = s`，`p->next` 就不再是原来的后继结点了，后续操作会出错。

### 关键知识点
- 双向链表结点包含两个方向的指针：`next` 和 `prev`。
- 在某结点后插入新结点时，需要同时维护前驱和后继关系。
- 指针赋值的顺序会影响结果，尤其是当某个指针马上要被改写时，必须先保存原值。

### 选项分析
- `A`：先执行 `p->next->prev=s;`，再执行 `p->next=s;`，之后 `s->next=p->next;` 时，`p->next` 已经变成 `s` 了，所以 `s->next` 会错误地指向自己，顺序不对，错误。
- `B`：`p->next=s;` 之后，`s->next=p->next;` 实际上也是 `s->next=s`，而且前后关系维护顺序混乱，错误。
- `C`：`p->next=s;` 之后再执行 `p->next->prev=s;`，此时 `p->next` 已经是 `s`，相当于把 `s->prev` 再次赋值为 `s`，并没有正确更新原后继结点的 `prev`，错误。
- `D`：先令 `s->next=p->next;` 保存原后继，再令 `p->next->prev=s;` 修改原后继的前驱，然后设置 `s->prev=p;`，最后 `p->next=s;` 完成链接，四个指针关系都正确，正确。

### 正确答案
`D`。该顺序完整实现了“把 `s` 插入到 `p` 之后”所需的四个指针调整，并且避免了在修改 `p->next` 后丢失原后继结点地址的问题。

子题 4

A. $O(n^{3/2}m)$

B. $O(nm)$

C. $O(n^{2}m)$

D. $O(nm^{2})$

答案：C

解析：### 解题思路
题目要求分析算法 `g(n,m)` 的时间复杂度。

先看 `g` 的三层循环结构：

1. 第 1 层：`i = 1..n`，共 `n` 次。
2. 第 2 层：`j = 2..m`，共约 `m` 次。
3. 第 3 层：`k = 1..i`，对每个 `i` 都要循环 `i` 次。

因此，`g` 的核心计算量为
`\sum_{i=1}^{n}\sum_{j=2}^{m}\sum_{k=1}^{i} 1`

因为最内层循环次数与 `i` 有关，所以可以先对 `k` 求和：
`\sum_{k=1}^{i} 1 = i`

于是总复杂度变为
`\sum_{i=1}^{n}\sum_{j=2}^{m} i = (m-1)\sum_{i=1}^{n} i`

而
`\sum_{i=1}^{n} i = O(n^2)`

所以 `g(n,m)` 的时间复杂度是 `O(n^2 m)`。

### 关键知识点
- 三重循环的时间复杂度需要结合每层循环次数一起分析。
- 当内层循环次数依赖外层变量时，要使用求和表示总次数。
- `\sum_{i=1}^{n} i = O(n^2)`。
- 动态规划中，若状态转移对每个状态都要枚举一个长度为 `i` 的区间，常见复杂度是 `O(n^2 m)`。

### 选项分析
A. `O(n^{3/2}m)`：与循环结构不符。题目中没有任何平方根或分块优化，不能得到这个复杂度。

B. `O(nm)`：只有在每个状态转移都能 `O(1)` 完成时才可能。这里第三层 `k` 要枚举到 `i`，不是 `O(1)`。

C. `O(n^{2}m)`：正确。因为总执行次数约为 `m * (1+2+...+n) = O(n^2 m)`。

D. `O(nm^{2})`：把复杂度主要看成关于 `m` 的平方，但实际 `m` 只在第二层线性出现，主要开销来自 `i` 和 `k` 的枚举。

### 正确答案
`C`

算法 `g(n,m)` 中，`i` 枚举到 `n`，`j` 枚举到 `m`，而 `k` 在每个 `i` 下要枚举 `1..i`。因此总复杂度是 `O(\sum_{i=1}^{n} m\cdot i) = O(mn^2)`，即 `O(n^2m)`，对应选项 `C`。

子题 4

A. `prev_color = image[p.r][p.c]`

B. `new_color = image[p.r][p.c]`

C. `image[p.r][p.c] = prev_color`

D. `image[p.r][p.c] = new_color`

答案：D

解析：### 解题思路

这是一个典型的 `flood fill`（洪水填充）算法。题意要求：从起始像素 `cur` 出发，把所有与起始像素颜色相同、且四连通可达的像素都改成新颜色 `new_color`。

程序中先把起点加入队列，然后不断取出一个像素 `pt`，检查它上下左右四个邻居是否满足：在图像范围内、颜色等于起始颜色、并且还没有被改成新颜色。若满足，就把该邻居染色为 `new_color`，并继续加入队列，表示后续还要从这个像素向外扩展。

因此，`is_valid` 中应当判断当前像素是否仍然是 `prev_color`，而在扩展到一个合法邻居时，应当把这个邻居直接改成 `new_color`，避免重复访问。

对应到填空：

- `①` 应为 `image[r][c] == prev_color`
- `④` 应为 `image[p.r][p.c] = new_color`

在给出的选项中，只有 `D. image[p.r][p.c] = new_color` 符合这个逻辑，所以标准答案是 `D`。

### 关键知识点

- `flood fill` 的核心：从起点出发，递归或借助队列扩展所有同色连通区域。
- 四连通：只能通过上、下、左、右四个方向移动。
- 为避免重复访问，通常在入队时就把像素改色或标记已访问。
- `is_valid` 需要同时检查边界、原颜色是否匹配、是否已经变成新颜色。

### 选项分析

- `A. prev_color = image[p.r][p.c]`：这是把 `prev_color` 重新赋值为邻居颜色，会破坏“只填充起始颜色区域”的判断，不符合算法逻辑。
- `B. new_color = image[p.r][p.c]`：这是把新颜色变量改掉，而且方向也反了，不会实现填充。
- `C. image[p.r][p.c] = prev_color`：这是把邻居改回旧颜色，和题意相反，无法完成填充。
- `D. image[p.r][p.c] = new_color`：把合法邻居染成新颜色，正是洪水填充应做的操作。

### 正确答案

答案是 `D`。在洪水填充中，找到与起始像素同色且连通的邻居后，应立即把它改成 `new_color`，这样既完成染色，也能起到“访问标记”的作用，避免重复入队。

解析：### 解题思路

这道题考查的是“洪水填充（flood fill）”的广度优先搜索实现。程序的核心思路是：

1. 先把起点 `cur` 放入队列。
2. 记录起点原来的颜色 `prev_color`。
3. 每次从队列中取出一个像素 `pt`，检查它上下左右四个邻接像素。
4. 如果某个邻接像素在图像范围内、颜色仍然是原颜色，并且还没有被改成新颜色，就把它改色并加入队列，继续扩展。

题目中第 `②` 处已经有 `queue.push(cur);`，说明起点已经入队。后续在遍历邻居时，应该把“当前可达的新像素” `p` 放入队列，而不是重复加入 `pt` 或 `cur`。因此 `⑤` 处应填写 `queue.push(p)`，对应选项 `A`。

### 关键知识点

- `flood fill`：从一个起点出发，把与起点颜色相同且四连通可达的区域全部改成新颜色。
- 广度优先搜索 `BFS`：使用 `queue` 保存待处理节点，逐层扩展。
- 访问控制：`image[r][c] != new_color` 用来避免重复处理已经染色的像素。
- 邻接扩展：对每个位置只考虑上下左右四个方向。

### 选项分析

- `A. queue.push(p)`：正确。`p` 是当前发现的、满足条件的相邻像素，应当加入队列继续搜索。

- `C. queue.push(cur)`：错误。`cur` 是起点，已经在 `②` 处入队，不应在这里再次加入起点。
- `D. queue. push(Point (ROWS, COLS))`：错误。`Point(ROWS, COLS)` 越界，不是合法图像坐标，也与当前搜索无关。

### 正确答案

`A`。在检查到合法且未染色的邻接像素 `p` 后，应将 `p` 入队，继续向外扩展填充范围。
