## 第 1 题

在内存储器中每个存储单元都被赋予一个唯一的序号，称为（）。

A. 地址

B. 序号

C. 下标

D. 编号

答案：A

解析：### 解题思路

在计算机的内存储器中，为了能够准确找到某一个存储单元，需要给每个存储单元分配一个唯一的标识，这个标识就叫作**地址**。题目问“唯一的序号”对应的名称，实际上就是内存地址。

### 关键知识点

- 内存储器中的每个存储单元都有唯一标识。
- 这个唯一标识称为`地址`。
- 通过地址，CPU 才能读写对应的存储单元。

### 选项分析

- `A. 地址`：正确。内存中每个存储单元的唯一序号就是地址。
- `B. 序号`：错误。题目描述的是“称为什么”，标准术语是地址，不是简单的序号。
- `C. 下标`：错误。下标常用于数组等数据结构中的位置表示，不是内存单元的标准名称。
- `D. 编号`：错误。虽然“编号”有类似含义，但在计算机存储器中规范术语是地址。

### 正确答案

`A`

## 第 2 题

编译器的主要功能是（ ）。

A. 将源程序翻译成机器指令代码

B. 将源程序重新组合

C. 将低级语言翻译成高级语言

D. 将一种高级语言翻译成另一种高级语言

答案：A

解析：### 解题思路
本题考查“编译器”的基本作用。编译器的核心任务是把用高级语言编写的源程序，转换成计算机能够直接执行的目标代码，也就是机器语言或机器指令代码。根据这一概念，可判断正确选项。

### 关键知识点
- `编译器`：将高级语言源程序翻译成目标程序的工具。
- `机器指令代码`：计算机硬件能够直接识别和执行的指令。
- 编译与解释不同：编译通常先整体翻译，再执行；解释通常边解释边执行。

### 选项分析
- `A`：`将源程序翻译成机器指令代码`，这是编译器的主要功能，正确。
- `B`：`将源程序重新组合`，这不是编译器的功能，错误。
- `C`：`将低级语言翻译成高级语言`，方向相反，不是编译器的主要作用，错误。
- `D`：`将一种高级语言翻译成另一种高级语言`，这更接近源到源翻译工具，不是编译器的一般定义，错误。

### 正确答案
`A`。编译器负责把源程序翻译成计算机可直接执行的机器指令代码，因此选 `A`。

## 第 3 题

设 `x=true,y=true,z=false`，以下逻辑运算表达式值为真的是（ ）。

A. (y∨z)∧x∧z

B. x∧(z∨y) ∧z

C. (x∧y) ∧z

D. (x∧y)∨(z∨x)

答案：D

解析：### 解题思路
题目给出 `x=true, y=true, z=false`，只需将每个选项中的逻辑量代入，按逻辑运算顺序计算结果即可。`∧` 表示“且”，只有两边都为真时结果才为真；`∨` 表示“或”，只要有一边为真结果就为真。

### 关键知识点
- `∧`（与）：只有两个操作数都为 `true` 时，结果才为 `true`。
- `∨`（或）：只要有一个操作数为 `true`，结果就为 `true`。
- 代入已知布尔值后，按括号优先计算，再进行整体判断。

### 选项分析
A. `(y∨z)∧x∧z`，代入得 `(true∨false)∧true∧false = true∧true∧false = false`，不成立。

B. `x∧(z∨y)∧z`，代入得 `true∧(false∨true)∧false = true∧true∧false = false`，不成立。

C. `(x∧y)∧z`，代入得 `(true∧true)∧false = true∧false = false`，不成立。

D. `(x∧y)∨(z∨x)`，代入得 `(true∧true)∨(false∨true) = true∨true = true`，成立。

### 正确答案
将 `x=true, y=true, z=false` 代入后，只有 `D` 的结果为 `true`。因为左边 `(x∧y)=true`，右边 `(z∨x)=true`，所以整个表达式 `true∨true` 仍为 `true`，因此标准答案是 `D`。

## 第 4 题

现有一张分辨率为 $2048\times 1024$ 像素的 $32$ 位真彩色图像。请问要存储这张图像，需要多大的存储空间？（ ）。

A. 16MB

B. 4MB

C. 8MB

D. 2MB

答案：C

解析：### 解题思路

图像的存储空间 = 像素总数 × 每个像素所占字节数。

题目给出分辨率为 `2048×1024`，表示共有 `2048 × 1024 = 2,097,152` 个像素。又因为是 `32` 位真彩色图像，所以每个像素占 `32` 位，即 `4` 字节。

因此所需存储空间为：

`2048 × 1024 × 4 = 8,388,608` 字节。

由于 `1MB = 1,024KB = 1,048,576` 字节，所以：

`8,388,608 ÷ 1,048,576 = 8MB`

所以答案是 `8MB`。

### 关键知识点

- 图像存储空间计算公式：`像素总数 × 每像素字节数`
- `32` 位真彩色图像中，每个像素占 `32/8 = 4` 字节
- 存储单位换算：`1MB = 1024KB = 1024×1024B`

### 选项分析

- `A. 16MB`：对应的像素数据量过大，实际计算结果只有 `8MB`。
- `B. 4MB`：只相当于把 `4` 字节/像素少算了一半，不正确。
- `C. 8MB`：`2048×1024×4B = 8,388,608B = 8MB`，正确。
- `D. 2MB`：明显偏小，不符合计算结果。

### 正确答案

`C`

## 第 5 题

冒泡排序算法的伪代码如下：

```
输入：数组L, n ≥ k。输出：按非递减顺序排序的 L。
算法 BubbleSort：
   1. FLAG ← n //标记被交换的最后元素位置
   2. while FLAG > 1 do
   3.     k ← FLAG -1
   4.     FLAG ← 1
   5.     for j=1 to k do
   6.         if L(j) > L(j+1) then do
   7.              L(j)  ↔ L(j+1)
   8.              FLAG ← j
```
对 $n$ 个数用以上冒泡排序算法进行排序，最少需要比较多少次?（ ）。

A. $n^2$

B. $n-2$

C. $n-1$

D. $n$

答案：C

解析：### 解题思路

冒泡排序在每一趟扫描中，都会从左到右比较相邻的两个元素。题目中的伪代码用 `FLAG` 记录“最后一次发生交换的位置”，这样下一趟只需要比较到 `FLAG-1` 为止。

要问“最少需要比较多少次”，就要考虑**最理想情况**：数组一开始已经是按非递减顺序排列的。这样在第一次 `for j=1 to k do` 的扫描中，虽然每次比较都不会交换，但仍然必须把相邻元素全部比较一遍，才能确认序列已经有序。

对于 `n` 个数，相邻元素共有 `n-1` 对，因此最少比较次数是 `n-1` 次。

### 关键知识点

- 冒泡排序每一趟都要进行相邻元素比较。
- 最好情况下，若序列已排序，第一趟仍需比较 `n-1` 次才能确认有序。
- `FLAG` 优化只能减少后续趟数的比较范围，不能减少第一次完整扫描的比较次数。

### 选项分析

- `A. n^2`：这是冒泡排序在最坏情况下的比较次数量级，不是最少比较次数。
- `B. n-2`：相邻比较次数应为 `n-1`，少一次就无法完成对全部相邻元素的检查。
- `C. n-1`：正确。`n` 个元素共有 `n-1` 对相邻元素，最少也要比较这么多次。
- `D. n`：比实际最少比较次数多 1 次，错误。

### 正确答案

在最理想情况下，数组已经有序，冒泡排序仍需要完成一次从 `1` 到 `n-1` 的相邻比较，因此最少比较次数为 `n-1` 次，选择 `C`。

## 第 6 题

设 $A$ 是 $n$ 个实数的数组，考虑下面的递归算法：
```
XYZ (A[1..n])
1.  if n=1 then return A[1]
2.  else temp ← XYZ (A[1..n-1])
3.  if temp < A[n]
4.  then return temp
5.  else return A[n]
```
 请问算法 XYZ 的输出是什么？（）。

A. A 数组的平均

B. A 数组的最小值

C. A 数组的中值

D. A 数组的最大值

答案：B

解析：### 解题思路

题目给出一个递归算法 `XYZ(A[1..n])`：

1. 当 `n = 1` 时，直接返回 `A[1]`；
2. 当 `n > 1` 时，先递归求出 `A[1..n-1]` 的结果 `temp`；
3. 再比较 `temp` 和 `A[n]`：
   - 如果 `temp < A[n]`，返回 `temp`；
   - 否则返回 `A[n]`。

这个过程本质上是在“从前 `n-1` 个数里找一个较小值，再和第 `n` 个数比较，保留更小的那个”。

因此，递归不断向前合并时，最终得到的是整个数组中的**最小值**。

也可以用归纳理解：
- 当 `n=1` 时，结果显然是唯一元素，也就是最小值；
- 假设 `XYZ(A[1..n-1])` 返回前 `n-1` 个数的最小值，那么再与 `A[n]` 取较小者，得到的就是前 `n` 个数的最小值。

所以算法输出的是 `A` 数组的最小值。

### 关键知识点

- 递归算法的归纳分析
- 通过比较并保留较小值来求数组最小值
- `n=1` 的递归边界条件

### 选项分析

- `A`：平均值需要对所有元素求和再除以元素个数，算法中没有求和和除法操作，因此不是平均值。
- `B`：每一步都在两个值中保留较小者，最终得到整个数组的最小值，正确。
- `C`：中值通常指排序后位于中间位置的元素，算法没有排序，也没有判断中间位置，因此不是中值。
- `D`：求最大值应当在比较后保留较大者，但题目代码在 `temp < A[n]` 时返回 `temp`，保留的是较小值，因此不是最大值。

### 正确答案

`B`。算法 `XYZ` 通过递归比较并保留较小的元素，最终返回整个数组中的最小值。

## 第 7 题

链表不具有的特点是（）。

A. 可随机访问任一元素

B. 不必事先估计存储空间

C. 插入删除不需要移动元素

D. 所需空间与线性表长度成正比

答案：A

解析：### 解题思路
链表是一种线性存储结构，但它与顺序表不同：元素在内存中通常不是连续存放的，结点之间通过指针（或引用）连接。因此，链表适合插入和删除操作，不适合按下标快速定位任意元素。题目问“链表不具有的特点”，应选出不符合链表性质的一项。

### 关键知识点
- 链表中的结点通常由 `数据域 + 指针域` 组成。
- 链表的存储空间可以按需动态分配，不必事先精确估计长度。
- 链表插入和删除结点时，通常只需修改指针，不必移动大量元素。
- 链表不支持像顺序表那样的随机访问，访问第 `i` 个元素一般需要从头结点依次遍历。

### 选项分析
A. `可随机访问任一元素`：错误。链表不能像数组或顺序表那样通过下标直接访问任一元素，通常需要顺序查找。

B. `不必事先估计存储空间`：正确。链表可以动态申请结点空间，长度可随数据插入而增长。

C. `插入删除不需要移动元素`：正确。链表进行插入、删除时主要调整指针连接关系，一般不需要搬移其他元素。

D. `所需空间与线性表长度成正比`：正确。链表的结点数与元素个数一致，元素越多，所需空间越大，整体上与线性表长度成正比。

### 正确答案
`A`。链表不具有随机访问任一元素的特点，访问元素需要按结点顺序逐个遍历。

## 第 8 题

有 $10$ 个顶点的无向图至少应该有（ ）条边才能确保是一个连通图。

A. 9

B. 10

C. 11

D. 12

答案：A

解析：### 解题思路
一个无向图要成为连通图，至少需要让 `10` 个顶点全部连在一起。对于 `n` 个顶点的连通无向图，最少边数出现在“树”这种结构中，而树恰好满足边数为 `n-1`。这里 `n=10`，所以至少需要 `10-1=9` 条边。

### 关键知识点
- 连通图：任意两个顶点之间都存在路径。
- 无向连通图的最少边数：`n-1`。
- 当且仅当图是一棵树时，`n` 个顶点的无向连通图边数最少，且边数为 `n-1`。

### 选项分析
- A. `9`：`10` 个顶点的无向图要保证连通，最少需要 `10-1=9` 条边，正确。
- B. `10`：`10` 条边当然可以构成连通图，但不是“至少应该有”的最小值，错误。
- C. `11`：边数更多，也可能连通，但不满足最小边数要求，错误。
- D. `12`：同样不是最少边数，错误。

### 正确答案
`A`。因为 `10` 个顶点的无向连通图至少需要 `10-1=9` 条边，这正好对应树的边数最小情况。

## 第 9 题

二进制数 $1011$ 转换成十进制数是（ ）。

A. 11

B. 10

C. 13

D. 12

答案：A

解析：### 解题思路
二进制数按“从右往左，每一位乘以对应的 2 的幂”来转换成十进制。

`1011` 可以写成：`1×2^3 + 0×2^2 + 1×2^1 + 1×2^0`。

计算可得：`8 + 0 + 2 + 1 = 11`，所以答案是 `11`。

### 关键知识点
- 二进制转十进制的方法：按位展开成 `各位数字 × 2^位数` 的和。
- 二进制的位权从右到左依次是 `2^0, 2^1, 2^2, 2^3, ...`。
- `1011₂ = 11₁₀`。

### 选项分析
A. `11`：将 `1011` 按二进制位权展开后得到 `11`，正确。

B. `10`：`10` 对应的二进制是 `1010`，不是 `1011`，错误。

C. `13`：`13` 的二进制是 `1101`，不是 `1011`，错误。

D. `12`：`12` 的二进制是 `1100`，不是 `1011`，错误。

### 正确答案
`1011` 是二进制数，展开后为 `1×2^3 + 0×2^2 + 1×2^1 + 1×2^0 = 11`，因此标准答案 `A` 正确。

## 第 10 题

$5$ 个小朋友并排站成一列，其中有两个小朋友是双胞胎，如果要求这两个双胞胎必须相邻，则有（ ）种不同排列方法?

A. 48

B. 36

C. 24

D. 72

答案：A

解析：### 解题思路
把“两个双胞胎”看作一个整体来处理。这样原来 5 个小朋友就变成了 4 个对象：1 个“双胞胎整体”加上另外 3 个普通小朋友。

先排这 4 个对象，有 `4! = 24` 种方法。

再考虑双胞胎内部的顺序：虽然必须相邻，但这两个小朋友还可以互换位置，所以有 `2! = 2` 种方法。

因此总排列数为 `4! × 2! = 24 × 2 = 48`。

### 关键知识点
- 相邻问题常用“捆绑法”处理。
- 把必须相邻的元素看成一个整体，再和其他元素一起排列。
- 整体内部如果还有顺序变化，需要再乘以内部分配数。

### 选项分析
- `A. 48`：将双胞胎捆成一个整体后有 `4!` 种排法，双胞胎内部还有 `2!` 种排法，总数为 `48`，正确。
- `B. 36`：与 `4! × 2!` 不符，少算了排列数，不正确。
- `C. 24`：只算了把双胞胎当整体后的 `4!` 种排法，漏掉了双胞胎内部互换的 `2` 种情况，不正确。
- `D. 72`：多算了，不符合相邻排列的实际计数，不正确。

### 正确答案
答案是 `A`。将两个双胞胎视为一个整体，与另外 3 个小朋友共 `4` 个对象排列，有 `4! = 24` 种；双胞胎内部还可以互换，有 `2! = 2` 种，所以总数为 `24 × 2 = 48`。

## 第 12 题

独根树的高度为 $1$。具有 $61$ 个结点的完全二叉树的高度为（ ）。

A. 7

B. 8

C. 5

D. 6

答案：D

解析：### 解题思路

本题考查完全二叉树的高度。已知“独根树的高度为 `1`”，说明树的高度按“结点层数”计算，而不是按边数计算。

对于完全二叉树，若高度为 `h`，则它最多有 `2^h - 1` 个结点。题目给出结点数为 `61`，需要找到满足

`2^(h-1) - 1 < 61 <= 2^h - 1`

的最小整数 `h`。

逐步计算：

- `2^5 - 1 = 31`
- `2^6 - 1 = 63`

因为 `61` 介于 `31` 和 `63` 之间，所以该完全二叉树的高度是 `6`。

### 关键知识点

- 独根树高度为 `1`，表示高度按层数计算。
- 完全二叉树第 `h` 层最多有 `2^(h-1)` 个结点。
- 高度为 `h` 的完全二叉树最多有 `2^h - 1` 个结点。
- 已知结点数时，可通过比较 `2^h - 1` 判断树高。

### 选项分析

- 2 个颜色各取 2 只，共 `2 + 2 = 4` 只；
- 剩下 2 只必须来自另外 2 个不同颜色，各取 1 只。

1. 先选出 2 个颜色配成手套对；

`C(5,2) × C(3,2) × 2^2 = 10 × 3 × 4 = 120`

- 组合计数：先分类，再分步计算。
- “恰好成两副手套”对应颜色分布必须是 `2,2,1,1`。
- 每个单只手套颜色都有左右手两种选择，因此要乘以 `2`。

- `A. 120`：按 `2,2,1,1` 的分布分步计数，结果为 `120`，正确。
- `B. 180`：与正确计数不符，说明在分步时多算或重复计算了某些情况，错误。
- `C. 150`：不是 `2,2,1,1` 分布下的计数结果，错误。
- `D. 30`：明显少算了单只手套左右手的选择情况，错误。

### 正确答案

先观察程序本身：`dfs(n, sum)` 每次从当前 `n` 个数对中任选相邻两个进行合并，把这两个数对变成一个新数对：

- 新的第 0 维：`a + x`
- 新的第 1 维：`b + y`
- 本次合并带来的增量：`s = a + x + abs(b - y)`

然后递归处理剩下的 `n - 1` 个数对，直到只剩 `1` 个数对为止，最后把所有合并过程中的 `s` 累加，取最大值 `ans`。

对本题所给单选题 4）来说：

- `n = 20`
- 有 `20` 个 `9` 和 `20` 个 `0`

也就是说，初始有 20 个数对，每个数对都是 `(9, 0)`。

关键在于：如果当前两个数对分别是 `(a, b)` 和 `(x, y)`，那么合并代价为 `a + x + abs(b - y)`。当所有 `b` 都是 `0` 时，`abs(b - y) = 0`，每次合并代价就变成 `a + x`。

因为每个数对的两维分别都是 `9` 和 `0`，无论怎么合并，新的数对第一维始终是若干个 `9` 的和，第二维始终是 `0` 的和，所以每次合并代价就是被合并两个部分第一维之和。最终整个递归的总值等于把 20 个 `9` 按某种二叉合并方式全部加起来的总和。

对于这种“每次合并代价等于两部分权值和”的过程，最大值并不是固定一个普通求和，而取决于合并顺序；但由于这里所有元素完全相同，任意合并顺序得到的总和都相同。20 个 `9` 合并成一个数时，总贡献是每个 `9` 被计入多次，最终答案为 `9 * (20 - 1) * 2?` 需要更准确地按程序含义计算。

注意程序的递归并不是普通“哈夫曼式只记叶子权值一次”，而是每次合并都把 `a + x + abs(b - y)` 加入 `sum`。在本题的输入下，`b = y = 0`，所以每次合并的增量就是 `a + x`。如果把每个初始数对看作权值 `9`，那么将 20 个权值通过 19 次合并连成一棵树，所有内部节点代价都是两个子树权值之和，总和等于 `9` 乘以所有叶子深度之和。对于 20 个完全相同的叶子，最优（也就是程序枚举所有方案取最大）时，相当于尽量让树不平衡，从而使一些叶子被重复计入更多次。

### 关键知识点

- `dfs` 用回溯枚举所有相邻合并方案，并用 `ans = max(sum, ans)` 记录最大值。
- 合并两个数对 `(a, b)`、`(x, y)` 的代价是 `a + x + abs(b - y)`。
- 当第二维全为 `0` 时，`abs(b - y) = 0`，代价简化为 `a + x`。
- 题目要求的是程序输出结果，不是手算中间合并过程的某一步。

using namespace std;

int n;
int d[50][2];
int ans;

void dfs(int n, int sum) {
  if (n == 1) {
    ans = max(sum, ans);
    return;
  }
  for (int i = 1; i < n; ++i) {
    int a = d[i - 1][0], b = d[i - 1][1];
    int x = d[i][0], y = d[i][1];
    d[i - 1][0] = a + x;
    d[i - 1][1] = b + y;
    for (int j = i; j < n - 1; ++j)
      d[j][0] = d[j + 1][0], d[j][1] = d[j + 1][1];
    int s = a + x + abs(b - y);
    dfs(n - 1, sum + s);
    for (int j = n - 1; j > i; --j)
      d[j][0] = d[j - 1][0], d[j][1] = d[j - 1][1];
    d[i - 1][0] = a, d[i - 1][1] = b;
    d[i][0] = x, d[i][1] = y;
  }
}

int main() {
  cin >> n;
  for (int i = 0; i < n; ++i)
  cin >> d[i][0];
  for (int i = 0; i < n;++i)
     cin >> d[i][1];
  ans = 0;
  dfs(n, 0);
  cout << ans << endl;
  return 0;
}

- `C. q<n&&A[q].a<=r`：下标位置不对，`q` 初始为 `0`，而循环里通过 `q++` 向后扫，判断应使用下一个未检查位置 `q+1`，错误。
- `D. q<n&&A[q].b<=r`：同样判断对象和逻辑都不对，不能用于找候选区间，错误。

### 正确答案
`A`。在每轮贪心中，需要继续把所有左端点 `a_i <= r` 的区间纳入候选集合，才能从中挑出右端点最大的区间来扩展覆盖范围，所以 `④` 应填 `q+1<n&&A[q+1].a<=r`。

解析：### 解题思路

这道题是经典的“最小区间覆盖”贪心问题。

做法分两步：

1. **先排序并去重/筛掉无用区间**
   - 按区间左端点 `a` 从小到大排序；
   - 如果左端点相同，再让右端点更大的区间排在前面，这样更有利于覆盖。
   - 排序后，可以把“左端点相同但右端点更小”的区间删掉，因为它一定不如前面的区间有用。

2. **从左到右贪心覆盖 `[0,m]`**
   - 设当前已经覆盖到 `r`，初始 `r = 0`；
   - 在所有 `a <= r` 的区间中，选择右端点最大的那个，把覆盖范围扩展到最大；
   - 每选一个区间，答案 `ans++`；
   - 重复直到 `r >= m`。

题目给出的代码中，`while (r < m)` 这部分正是这种贪心过程。内层循环负责把所有左端点不超过当前覆盖点 `r` 的区间都纳入候选，然后用右端点最大的那个来更新 `r`。

因此，第 `⑤` 处应当把 `r` 更新为当前选中的区间的右端点，即 `r=max(r,A[q].b)`。

### 关键知识点

- 贪心策略：每一步选择“能让当前覆盖范围延伸最远”的区间。
- 区间排序：按左端点升序、右端点降序，有助于筛掉无效区间。
- 最小区间覆盖：只要保证每次都把当前能选的区间里最右的那个选出来，就能得到最少数量。

### 选项分析

#### A. `r=max(r,A[q+1].b)`

错误。`q+1` 不是当前已经确认要选中的区间位置，且内层循环结束后通常 `q` 指向最后一个满足条件的区间，应该使用当前区间下标 `q`，而不是 `q+1`。

#### B. `r=max(r,A[q].b)`

正确。内层循环结束后，`A[q]` 对应的是当前所有满足 `a <= r` 的区间中，应该选中的那个区间；用它的右端点更新 `r`，正是贪心核心。

#### C. `r=max(r,A[q+1].a)`

错误。更新覆盖范围时应该用右端点 `b`，而不是左端点 `a`。用左端点无法推进覆盖，也不符合题意。

#### D. `q++`

错误。这里只是继续移动指针的动作，并没有完成对当前覆盖范围的更新；题目要求的是贪心选中一个区间后扩大 `r`，所以不能只写 `q++`。

### 正确答案

`B`

原因是：在完成“找到当前可选区间中最远右端点”的扫描后，应该用该区间的右端点更新当前覆盖边界 `r`，即 `r=max(r,A[q].b)`。这正是最小区间覆盖的标准贪心步骤。
