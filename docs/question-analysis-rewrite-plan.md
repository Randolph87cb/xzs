# 题目解析重构方案

## 背景与现状

- 已连接 Fly 远端 Postgres app `xzs-pg-cb867393296`，业务库为 `xzs_cb867393296`，只执行只读抽查 SQL。
- 题目答案保存在 `t_question.correct`，题干、选项、解析保存在 `t_text_content.content` 的 JSON 字段中，其中解析字段为 `analyze`。
- 当前线上有效题目共 4249 道；其中有 `question_code/import_source` 的 GESP Markdown 真题为 2124 道，本地 `docs/question-bank/GESP` 解析出的题量也是 2124 道。
- 另有 2125 道早期题没有 `question_code/import_source`，不是当前 Markdown 题库源直接可重建的对象。
- 线上 `t_question_review_record` 只有 1 条 `KNOWLEDGE_POINT` 审核记录，没有 `ANALYSIS` 审核记录；批量改解析前仍应备份并做差异审查。

远端抽查统计：

| 指标 | 数量 |
| --- | ---: |
| 有效题目总数 | 4249 |
| 无导入来源/早期题 | 2125 |
| 解析包含“暂无解析” | 2124 |
| 解析长度小于 60 字 | 2273 |
| 选择题通用模板命中 | 1010 |
| 判断题通用模板命中 | 320 |
| 有 `question_code` 的真题 | 2124 |

典型问题：

| 类型 | 样例 | 问题 |
| --- | --- | --- |
| 占位解析 | `id=4669`，二分查找复杂度题，解析为 `<p>暂无解析</p>` | 学生无法看到推导过程，尤其是复杂度、程序阅读题不具备教学价值。 |
| 模板复述 | `id=9014`，循环执行次数题，解析只说“选项 C 与题干要求一致” | 没有计算 `i=-10` 到 `9` 的执行次数，不能解释为什么是 20。 |
| 模板错配 | `id=10052`，`sort` 是否可直接排序 `set`，解析却讲排序稳定性 | 解析知识点和题干不一致，会形成误导。 |
| 模板错配 | `id=10431`，删除 `printf("\n")` 是否影响乘法表输出，解析套用“C++ 基础语法” | 没有说明换行对二维表格输出结构的影响。 |
| 质量目标样例 | `2026-03` 批次 | 能围绕具体代码、边界或计算步骤说明结论，可作为改写目标样式。 |

## 结论

推荐分两阶段重构：先修当前有稳定来源的 2124 道 GESP Markdown 真题解析，再处理 2125 道无来源早期题。所有批量回写都应以本地 Markdown 或导出清单为源，生成 SQL 差异后审查，再写入 Fly 远端，避免直接在生产 JSON 中手工改动。

## 需求拆解

### 1. 建立解析质量标准

- 当前现状：解析没有机器可检查的质量门槛，模板句式可以整批进入题库。
- 判断：单纯“把暂无解析替换成一段话”不够，必须要求解析能指向题干条件、关键推导、正确结论和易错点。
- 修改方案：
  - 为客观题解析建立统一短格式：`结论 -> 关键推导 -> 排除/易错点`。
  - 选择题至少说明正确选项成立的具体原因；当题目问“错误/不符合”时必须显式说明题干问法。
  - 判断题必须说明题干中哪一部分成立或不成立，不能只写“与规则不一致”。
  - 程序阅读题必须给出关键变量变化、循环次数、输出格式或复杂度推导。
  - 概念题必须点明定义边界，避免套用无关知识点模板。
- 影响范围：`docs/question-bank/GESP/**/*.md`、后续导入脚本的质量检查、后台解析审核流程。
- 验证方案：抽样人工审核每个级别和题型；机器检查解析长度、禁用模板短语、是否仍包含 `暂无解析`。

### 2. 重写有来源的 2124 道 GESP 真题解析

- 当前现状：`scripts/import-gesp-objective-questions.ps1 -DryRun` 可解析 2124 道题；导入脚本按 `import_batch + import_source + import_question_order` 或 `question_code` 匹配并更新 `t_text_content.content`。
- 判断：这批题有稳定源文件，应优先改 `docs/question-bank/GESP`，再通过现有脚本回写远端。
- 修改方案：
  - 先按批次优先级处理：`2024-03` 到 `2024-12` 几乎整批模板化，优先级最高；`2025-03` 到 `2025-12` 其次；`2023` 批次为短模板，第三优先级；`2026-03` 暂不重写，只作为质量样例。
  - 批量生成候选解析后，按 `year-month + level + question_type` 分文件审校。
  - 每次只合入一个月份或一个级别，避免 2124 道一次性回写无法定位问题。
  - 使用 `-MigrationSqlOnly` 生成迁移 SQL，先审查 `content_rows_to_update` 和样本 diff，再执行远端写入。
- 影响范围：当前真题训练、智能训练、错题回看、后台预览中展示的解析。
- 验证方案：
  - 本地运行 `.\scripts\import-gesp-objective-questions.ps1 -DryRun`，确认仍为 2124 道且题型/级别分布不变。
  - 执行模板短语扫描，确认目标批次不再命中“与题干要求一致”“其余选项要么改变了关键条件”“按题目涉及的 C++ 语法”等泛化短语。
  - 对每个批次随机抽 5 道选择题和 5 道判断题，人工确认解析与题干、答案、知识点一致。
  - 远端回写后抽查 `t_text_content.content::jsonb ->> 'analyze'`，并打开学生端答题解析页做浏览器验收。

### 3. 处理 2125 道无来源早期题

- 当前现状：这批题没有 `question_code/import_source`，2124 道是 `暂无解析`，分布在 GESP 1 到 8 级，数量大致与本地真题题量相当。
- 判断：不能直接用现有 Markdown 导入脚本覆盖，因为缺少稳定源文件和题号映射。需要先决定保留、归档还是补源。
- 修改方案：
  - 先导出这批题到新的可版本化目录，例如 `docs/question-bank/legacy-export/GESP/...`，文件中保留远端 `id`、题型、知识点、题干、选项、答案和原解析。
  - 对导出题做去重：与 2124 道真题按题干规范化文本、答案、选项集合比对，重复题建议软删除或标记为重复来源，不重复题再补解析。
  - 对保留题补充稳定标识，例如 `LEGACY-GESP-L{level}-{type}-{id}`，并建立单独回写脚本，禁止混用 GESP 真题导入脚本。
  - 如果早期题质量整体低于真题，可先从智能训练抽题池中排除这批题，等补完解析后再恢复。
- 影响范围：智能训练抽题质量、题库总量、历史答题记录引用。
- 验证方案：
  - 导出后确认 2125 道数量、id、答案和题干 hash 不丢失。
  - 去重报告列出重复题、疑似重复题、保留题。
  - 回写前备份 `t_question` 和 `t_text_content` 相关行；回写后确认题目 id 不变，历史答题记录仍能关联。

### 4. 加质量门禁和后台审核入口

- 当前现状：后台 `QuestionReviewController` 支持解析编辑和审核记录，但缺少批量筛选“模板解析/占位解析/疑似错配”的能力；导入脚本也不会阻止低质量解析。
- 判断：只做一次性重写，后续导入仍可能回退到模板化。
- 修改方案：
  - 在导入脚本中增加质量检查模式：统计 `暂无解析`、短解析、禁用模板短语、解析与知识点明显错配的规则命中。
  - 后台题目审核页增加快速筛选：占位解析、模板解析、短解析、有来源批次、无来源早期题。
  - 批量回写时自动写入或导入审核记录，至少保留 before/after、批次、操作者和时间。
- 影响范围：`scripts/import-gesp-objective-questions.ps1`、后台题目审核接口和管理端审核页面。
- 验证方案：构造包含 `暂无解析` 和模板短语的 Markdown 样例，确认质量检查失败；后台筛选能定位对应题目；编辑解析后 `t_question_review_record` 有记录。

第一阶段已先在本地导入脚本加入报告型质量检查，不改生产数据库，也不批量重写 `docs/question-bank/GESP` 解析正文：

```powershell
.\scripts\import-gesp-objective-questions.ps1 -QualityCheck
```

默认报告模式会解析本地 Markdown 并统计总题数、占位解析、短解析、模板短语命中、批次/文件汇总和问题样本，但即使命中当前存量问题也返回 0。这样可以先把问题规模和优先批次稳定暴露出来，不阻断现有导入、DryRun、SQL 生成和迁移 SQL 生成流程。

短解析阈值默认 60 个有效字符，可按需调整：

```powershell
.\scripts\import-gesp-objective-questions.ps1 -QualityCheck -QualityShortAnalyzeThreshold 80
```

后续接入 CI 或正式门禁时再显式启用严格失败：

```powershell
.\scripts\import-gesp-objective-questions.ps1 -QualityCheck -FailOnQualityIssues
```

严格模式只在存在质量问题时返回非 0，用于防止后续重写批次回退到 `暂无解析`、过短解析或模板句式。

## 执行顺序

1. 生产库备份：导出 `t_question`、`t_text_content`、`t_question_review_record`，并记录 Fly app、数据库和备份时间。
2. 质量门禁脚本先行：先能稳定识别占位、短解析和模板解析，避免重写过程中反复回退。
3. 重写并审校 `2024-03` 到 `2024-12` 批次。
4. 重写并审校 `2025-03` 到 `2025-12` 批次。
5. 补齐或改写 `2023` 批次短解析。
6. 生成 SQL diff，分批回写 Fly 远端，回写后做页面和 SQL 抽查。
7. 导出并治理 2125 道无来源早期题，先去重和决定保留范围，再补解析。
8. 把质量门禁接入日常导入流程和后台审核筛选。

## 风险与待确认

- 2125 道无来源早期题是否需要全部保留待确认；如果只是历史导入残留，优先软删除或排除出抽题池会比逐题补解析更划算。
- 批量生成解析需要人工审校，特别是程序阅读、复杂度、图论、排序稳定性、C++ 未定义/不确定行为等题型。
- 远端回写应避免覆盖未来人工编辑；正式执行前需要再次检查 `t_question_review_record` 是否新增 `ANALYSIS` 记录。
- 题库源 Markdown 当前没有独立“解析质量测试”，应在第一阶段补齐，否则后续导入仍可能重新引入模板解析。
